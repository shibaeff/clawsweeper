## Summary

Keep open. Current `main` appears to have implemented the threaded-reply half of the request, but the issue also asks for receiving non-mention messages in Google Chat spaces, and that is not implemented in the current Google Chat channel. The channel is still service-account based with `chat.bot` auth, docs still describe spaces as mention-gated, and the inbound access path still drops group messages when mention policy is not satisfied.

## Evidence

- **Threaded replies are implemented on main:** Inbound Google Chat messages capture the thread id as `ReplyToId`, and outbound replies pass that thread id back into `sendGoogleChatMessage`, which sets `body.thread = { name: thread }` plus `messageReplyOption=REPLY_MESSAGE_FALLBACK_TO_NEW_THREAD`. This is visible in `extensions/googlechat/src/monitor.ts:226`, `extensions/googlechat/src/monitor.ts:401`, and `extensions/googlechat/src/api.ts:141`.
