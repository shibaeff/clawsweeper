## Summary

Current main already guards the voice-call webhook server against duplicate starts, cleans up bound resources after failed runtime initialization, and has regression tests covering both cases. The same implementation is present in the shipped v2026.4.22 release tag.

## Evidence

- **Webhook start is idempotent:** `VoiceCallWebhookServer.start()` returns the existing listening URL when the server is already listening, explicitly calling out EADDRINUSE prevention for repeated starts on the same instance.
  - file: [extensions/voice-call/src/webhook.ts:431](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/voice-call/src/webhook.ts#L431)

  - file: [extensions/voice-call/src/runtime.ts:265](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/voice-call/src/runtime.ts#L265)

Closing as implemented.
