## Summary

Keep open. Current `main` does not contain the PR's Feishu fixes: reaction synthesis still drops events when neither the webhook nor fetched message carries `chat_type`, fetched messages still do not infer chat type from `chat_id`, and reply routing still targets the synthetic reaction `message_id` directly. That means this PR is neither implemented on `main` nor safely auto-closable as obsolete/incoherent.

## Evidence
