## Summary

Current `main` already contains the Telegram polling/watchdog changes that address this regression class: long-running Telegram reply delivery is no longer treated as a hard polling stall, recoverable `getUpdates` failures do not immediately trip the watchdog, and dirty polling cycles rebuild/retire the transport instead of reusing stale connections.

## Evidence

- **Direct fix noted in shipped changelog:** The changelog explicitly says Telegram polling now keeps the watchdog from aborting long-running reply delivery by treating recent non-polling API activity as bounded liveness instead of a hard stall.
  - file: [CHANGELOG.md:1470](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/CHANGELOG.md#L1470)
  - command: `nl -ba CHANGELOG.md | sed -n '1468,1471p'`
- **Current watchdog logic uses API liveness, not only getUpdates cadence:** `TelegramPollingLivenessTracker.detectStall()` suppresses a stall when recent non-`getUpdates` API activity or an in-flight outbound API call is still within the threshold.
  - file: [extensions/telegram/src/polling-liveness.ts:110](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/telegram/src/polling-liveness.ts#L110)
  - command: `nl -ba extensions/telegram/src/polling-liveness.ts | sed -n '92,130p'`
- **Polling session wires non-getUpdates traffic into liveness tracking:** `TelegramPollingSession` records non-`getUpdates` API calls through middleware before the watchdog decides to restart polling.
  - file: [extensions/telegram/src/polling-session.ts:249](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/telegram/src/polling-session.ts#L249)
  - command: `nl -ba extensions/telegram/src/polling-session.ts | sed -n '243,325p'`
- **Regression tests cover the reported false-stall behavior:** There are explicit tests asserting no stall restart when `sendMessage` activity is recent or still in flight, and no false stall immediately after a recoverable `getUpdates` error.
  - file: [extensions/telegram/src/polling-session.test.ts:695](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/telegram/src/polling-session.test.ts#L695)
  - command: `nl -ba extensions/telegram/src/polling-session.test.ts | sed -n '526,915p'`
