## Summary

Keep open. Current `main` does not contain the gateway-owned ACP node-backed runtime from PR #48940: the shipped runtime is still `acpx`-plugin-backed, the node command surface lacks ACP session/turn commands, the PR’s core store/replay/runtime files are absent from `main`, and the PR discussion still includes unresolved P1 review concerns that need maintainer judgment rather than conservative auto-close.

## Evidence

- **Main ACP runtime is still plugin-backed:** `requireAcpRuntimeBackend(...)` on current `main` still throws `ACP runtime backend is not configured. Install and enable the acpx runtime plugin.`, which is inconsistent with the PR’s goal of adding a built-in gateway-owned `acp-node` backend.
