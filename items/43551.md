## Summary

Current main does not support arbitrary top-level plugin-entry keys by design, but it does support and validate plugin-specific settings under `plugins.entries.<id>.config`. The reported nested-`config` rejection does not match current main: docs point users to `plugins.entries.<id>.config`, the runtime validator passes `entry.config` into each plugin manifest's `configSchema`, and regression tests cover accepted/rejected nested plugin config under that path.
