## Summary

Keep open: current `main` still misses dynamic-provider model preparation in the image path, but this PR as proposed appears to regress the existing explicit-model fast path for providers with expensive `prepareDynamicModel` hooks.

## Evidence

- **Current image path still bypasses dynamic-provider preparation:** `resolveImageRuntime` in `src/media-understanding/image.ts` still does `modelRegistry.find(...)` directly and throws `Unknown model` without calling `prepareProviderDynamicModel` or `resolveModelWithRegistry`, so the underlying bug this PR targets is not already fixed on `main`.
  - file: [src/media-understanding/image.ts:139](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/media-understanding/image.ts#L139)
  - command: `nl -ba src/media-understanding/image.ts | sed -n '120,240p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Core async resolver intentionally preserves an explicit-model fast path:** `resolveModelAsync` first checks `resolveExplicitModelWithRegistry`; when an explicit model is already resolved and runtime comparison is unnecessary, it returns that model without awaiting `prepareProviderDynamicModel`. That is the existing current-main contract for avoiding unnecessary provider-runtime work.
  - file: [src/agents/pi-embedded-runner/model.ts:788](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-runner/model.ts#L788)
  - command: `nl -ba src/agents/pi-embedded-runner/model.ts | sed -n '640,850p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Tests lock in the no-preload behavior for already-known OpenRouter models:** `src/agents/pi-embedded-runner/model.test.ts` has a dedicated test, `skips OpenRouter preload for models already present in the registry`, asserting `mockLoadOpenRouterModelCapabilities` is not called for a registry hit.
  - file: [src/agents/pi-embedded-runner/model.test.ts:681](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-runner/model.test.ts#L681)
  - command: `nl -ba src/agents/pi-embedded-runner/model.test.ts | sed -n '650,730p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **OpenRouter prepare hook is potentially expensive:** The bundled OpenRouter provider's `prepareDynamicModel` hook awaits `loadOpenRouterModelCapabilities`, and that helper can wait on a catalog fetch when the model is not already cached. An unconditional prepare in the image path can therefore add avoidable latency and offline failure surface.
  - file: [extensions/openrouter/index.ts:122](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/extensions/openrouter/index.ts#L122)
  - command: `nl -ba extensions/openrouter/index.ts | sed -n '100,170p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Capability preload may block on fetch before first use:** `loadOpenRouterModelCapabilities` calls `ensureOpenRouterModelCache()`, triggers a fetch if needed, and awaits the in-flight fetch before returning on a cache miss.
  - file: [src/agents/pi-embedded-runner/openrouter-model-capabilities.ts:264](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-runner/openrouter-model-capabilities.ts#L264)
  - command: `nl -ba src/agents/pi-embedded-runner/openrouter-model-capabilities.ts | sed -n '240,310p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **PR discussion already surfaced the regression risk:** The Codex review comment on PR commit `9c9e3c4f71b3ff16f8e3541dc6695c31e1f8cef8` points out that the patch awaits `prepareProviderDynamicModel` for every image request before normal registry resolution, regressing the explicit-model fast path that `resolveModelAsync` currently preserves.
  - sha: [9c9e3c4f71b3](https://github.com/openclaw/openclaw/commit/9c9e3c4f71b3ff16f8e3541dc6695c31e1f8cef8)

## Risks / Open Questions

- If this PR lands as-is, image requests for statically resolvable models may pay unnecessary provider-runtime preload cost, especially for OpenRouter.
- If the PR is closed now, the underlying dynamic-model lookup gap in `src/media-understanding/image.ts` remains unfixed on `main`, leaving issue `#66486` unresolved.
