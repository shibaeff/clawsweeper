## Summary

Current main already covers the reported WhatsApp phone-notification problem via `channels.whatsapp.selfChatMode`: when enabled, the WhatsApp inbox monitor sends global `unavailable` presence on connect instead of `available`, and the code comment explicitly says this is to avoid muting phone pushes.

## Evidence

"unavailable" : "available"` and immediately calls `sock.sendPresenceUpdate(presence)`. The adjacent comment says: `Keep the global presence unavailable so self-chat sessions do not mute phone pushes.`

## Close Comment

Closing as implemented after Codex review. Current main already covers this same-phone WhatsApp case with `channels.whatsapp.selfChatMode`: the connect path sends global `"unavailable"` presence instead of `"available"`, and the code comment explicitly says this is to keep self-chat setups from muting phone pushes (`extensions/whatsapp/src/inbound/monitor.ts:77`, `extensions/whatsapp/src/inbound/monitor.ts:133`). The setting is part of the shipped config schema (`src/config/zod-schema.providers-whatsapp.ts:63`), the runtime wires it through from account config (`extensions/whatsapp/src/auto-reply/monitor.ts:255`), tests assert the first connect-time presence is `"unavailable"` in self-chat mode (`extensions/whatsapp/src/monitor-inbox.streams-inbound-messages.test-support.ts:188`), and the docs/FAQ already direct personal-number users to enable `channels.whatsapp.selfChatMode` (`docs/channels/whatsapp.md:127`, `docs/help/faq.md:1853`). This is also present in the latest shipped release, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
