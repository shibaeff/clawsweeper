## Summary

Current `main` already centralizes implied operator-scope expansion for device auth, routes pairing token scope handling through that shared normalizer, and has regression tests covering both stored token expansion and admin-token verification. The requested behavior is also present in released `v2026.4.23`, so this PR is redundant.

## Evidence

- **Shared device-auth normalization already expands implied operator scopes on main:** `normalizeDeviceAuthScopes()` now adds `operator.read` for `operator.write`, and adds both `operator.read` and `operator.write` for `operator.admin`. That is the core behavior this PR proposes.
