## Summary

Current `main` does not implement this PR's `user_id`-first sender-name lookup. The Feishu bot still calls sender-name resolution with a single `ctx.senderOpenId` value, and mixed-ID events still normalize that field to `open_id` when both IDs are present. This remains a coherent, plausible bugfix PR rather than something that should be auto-closed.
