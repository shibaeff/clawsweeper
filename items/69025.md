## Summary

Current `main` and the shipped `v2026.4.23` release restore the `message` tool for isolated cron runs with `delivery.mode: "none"`, and the repo now has matching docs and regression coverage for that exact behavior.

## Evidence

- **Cron tool policy now keeps `message` enabled for `delivery.mode: "none"`:** `resolveCronToolPolicy` enables the shared `message` tool for every isolated cron delivery mode except `webhook`, so `delivery.mode: "none"` no longer disables it.
  - file: [src/cron/isolated-agent/run.ts:285](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cron/isolated-agent/run.ts#L285)

  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Regression tests cover the reported no-deliver isolated-session path:** Tests assert that `delivery.mode: "none"` keeps the tool enabled, preserves explicit/current target routing, and marks the run delivered when the agent sends via `message`.
  - file: [src/cron/isolated-agent/run.message-tool-policy.test.ts:266](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cron/isolated-agent/run.message-tool-policy.test.ts#L266)
