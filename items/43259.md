## Summary

Current `main` does not implement this PR’s requested behavior. Web UI `chat.send` attachments are still omitted from `MsgContext.MediaPath/MediaPaths` for ordinary sessions, while media-understanding still keys off those fields. Main has a partial text-only fallback via `media://inbound/...` refs, but that is not the same fix and does not prove the PR is obsolete.

## Evidence

- **chat.send only injects media fields for plugin-bound sessions:** `pluginBoundMediaFields` is populated only when `explicitOriginTargetsPlugin && parsedImages.length > 0`, and `ctx` spreads only that object. Ordinary Web UI sessions therefore still dispatch without `MediaPath`/`MediaPaths`.
