## Summary

Keep open. Current `main` still treats Slack `allowBots` as boolean-only and does not implement a Slack `"mentions"` mode comparable to Discord.

## Evidence

- **Slack config types are boolean-only:** `SlackChannelConfigResolved.allowBots` and `SlackChannelConfigEntry.allowBots` are both typed as `boolean`, so the Slack config surface does not currently accept `"mentions"`.
  - file: [extensions/slack/src/monitor/channel-config.ts:15](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/slack/src/monitor/channel-config.ts#L15)
  - command: `nl -ba extensions/slack/src/monitor/channel-config.ts | sed -n '1,240p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Slack inbound gating only checks true/false:** Slack conversation prep stores `allowBots` as `boolean`, resolves it with `?? false`, and the authorization path only drops on `!allowBots`; there is no mention-only branch.
  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:100](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/slack/src/monitor/message-handler/prepare.ts#L100)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare.ts | sed -n '88,210p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Slack user-facing hints still describe boolean behavior:** Slack config UI help says only `Allow bot-authored messages to trigger Slack replies (default: false)` and does not mention a `"mentions"` option.
  - file: [extensions/slack/src/config-ui-hints.ts:28](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/slack/src/config-ui-hints.ts#L28)
  - command: `nl -ba extensions/slack/src/config-ui-hints.ts | sed -n '1,80p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Slack docs do not document mention-only bot gating:** The Slack channel docs list `allowBots` as a per-channel control, but they do not document `allowBots: "mentions"` or mention-only bot handling.
  - file: [docs/channels/slack.md:498](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/channels/slack.md#L498)
