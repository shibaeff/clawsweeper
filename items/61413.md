## Summary

Keep open. Current `main` still intentionally uses DNS-pinned/custom dispatchers in `TRUSTED_ENV_PROXY` when no proxy env vars are set, so the transparent-proxy behavior requested by PR #61413 is not implemented. The draft patch also has unresolved review findings: it would drop host-policy enforcement in trusted mode, add an environment-dependent test, and introduce an unguarded `JSON.stringify` in the error path.
