## Summary

Keep open. Current `main` still lacks the Feishu reply-mention fallback this PR is trying to add, so the underlying bug is not obsolete, but the proposed `rootId` clearing conflicts with current topic-session routing and reply-target contracts on `main`. The review discussion already surfaced those correctness risks, so this is not a safe cleanup close.

## Evidence

- **Current main still lacks reply-text fallback for dropped Feishu mentions:** `checkBotMentioned()` on `main` only recognizes `@_all`, structured `message.mentions`, or rich-text `post` mentions. There is no reply-text/orphan-placeholder fallback, so the PR's reported gap is not already implemented on `main`.
