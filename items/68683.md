## Summary

Keep open. Current `main` does not show a maintainer-fix for this report, and I could not disprove it conservatively. The code and tests show that OpenClaw intentionally routes direct `PERPLEXITY_API_KEY` usage to Perplexity's native `/search` transport, while the issue's control request only proves `/chat/completions` works. That mismatch means the reported 401 is still plausible rather than obsolete.

## Evidence

- **Direct Perplexity keys route to native Search API on current main:** The runtime reads `PERPLEXITY_API_KEY` from the gateway environment, selects `https://api.perplexity.ai` for direct auth, and uses `transport: "search_api"` unless legacy `baseUrl`/`model` overrides are present.
  - file: [extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:75](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/perplexity/src/perplexity-web-search-provider.runtime.ts#L75)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Native requests go to `/search`, not `/chat/completions`:** Current main sends native Perplexity web search requests to `https://api.perplexity.ai/search`. The issue body only demonstrated success against `https://api.perplexity.ai/chat/completions`, which is a different transport than OpenClaw's default direct-key path.
  - file: [extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:34](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/perplexity/src/perplexity-web-search-provider.runtime.ts#L34)

  - file: [extensions/perplexity/src/perplexity-web-search-provider.test.ts:56](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/perplexity/src/perplexity-web-search-provider.test.ts#L56)

## Risks / Open Questions

- I could not perform a live Perplexity request against current main because the report depends on reporter-specific credentials and gateway environment state.
- The issue's successful control request targets `/chat/completions`, but OpenClaw's direct `PERPLEXITY_API_KEY` path targets `/search`; that leaves a transport-specific failure mode unresolved.
- The runtime still advertises a legacy config path in its missing-key message, which may send users to the wrong credential surface during debugging.
