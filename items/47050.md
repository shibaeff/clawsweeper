## Summary

Current `main` already has deadline-based chat-run cleanup in Gateway: each `chat.send` run gets an `expiresAtMs`, the Gateway maintenance loop aborts expired runs every minute with `stopReason: "timeout"`, and stale/orphaned chat buffers are reaped afterward. The shipped release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` (`v2026.4.22`) contains the same implementation.
