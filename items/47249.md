## Summary

Keep open. Current `main` already fixes the picker-side provider qualification bug and newer slash-containing model-ref edge cases, but it still does not canonicalize bare `/model` output or the `/model` success text the way this PR proposes. The PR therefore is not implemented on `main`, yet it is also stale against newer `chat-model-ref` logic and needs a refresh/review instead of auto-close.
