## Summary

Keep open. Current `main` still drops `requesterAgentIdOverride` on the ACP `sessions_spawn` path, and the ACP requester/heartbeat logic still derives the requester agent from the parsed session key instead of honoring an explicit override. The PR’s described fix does not appear to be implemented or superseded on `main`.

## Evidence

- **ACP `sessions_spawn` path still omits the override:** In current `main`, the ACP branch forwards `agentSessionKey`, channel/account/thread/group context, and `sandboxed`, but not `requesterAgentIdOverride`, when calling `spawnAcpDirect`. In the same file, the subagent branch does forward `requesterAgentIdOverride`, so the ACP path remains inconsistent with the intended behavior described in the PR.
  - file: [src/agents/tools/sessions-spawn-tool.ts:268](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tools/sessions-spawn-tool.ts#L268)
