## Summary

Keep open. Current `main` still treats browser WebChat sends as internal `webchat` turns and does not inherit the session’s external `deliveryContext` unless `ExplicitDeliverRoute` is set, so the reported WeChat-routing failure still matches current behavior.

## Evidence

- **Browser chat uses `webchat` client mode:** The browser Control UI/WebChat client connects with `mode: "webchat"`, so the report's browser-originated send path goes through the WebChat-specific routing branch.
  - file: [ui/src/ui/app-gateway.ts:290](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/ui/src/ui/app-gateway.ts#L290)

- **WebChat clients do not inherit external delivery routes:** `resolveChatSendOriginatingRoute` explicitly documents that WebChat clients never inherit external delivery routes, and falls back to `originatingChannel: webchat` with `explicitDeliverRoute: false` when that inheritance is blocked.
