## Summary

Current `main` already implements the requested iMessage DM-policy behavior: iMessage DMs default to `pairing`, `dmPolicy: "open"` is only valid when `allowFrom` includes `"*"`, and current docs/setup/doctor surfaces explain that unknown DMs will be blocked or require pairing unless explicitly opened. The same behavior is present in release `v2026.4.23`.

## Evidence
