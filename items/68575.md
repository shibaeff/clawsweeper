## Summary

Keep open. Current `main` still does not implement the two session-reset paths this PR targets, and the PR discussion surfaces unresolved correctness and coverage concerns that make an auto-close unsafe.

## Evidence

- **Thrown overflow path on main still does not reset the session:** In the current catch handler, `isContextOverflow` is computed, but only `isCompactionFailure` triggers `resetSessionAfterCompactionFailure(...)`. Plain pre-reply overflow still falls through to the generic overflow fallback text instead of resetting the session.
