## Summary

Current main already routes Web UI context usage through fresh session `totalTokens` snapshots instead of cumulative run totals, and the latest shipped release includes regression tests for the exact inflated-display case.

## Evidence

- **Session usage now prefers fresh context snapshots:** `persistSessionUsageUpdate` documents and implements `lastCallUsage`/`promptTokens` for `totalTokens` so context utilization reflects the current context, not accumulated multi-call usage.

## Close Comment

Closing as implemented on current OpenClaw. Codex review checked `src/auto-reply/reply/session-usage.ts:71` and `src/auto-reply/reply/session-usage.ts:114`, where session `totalTokens` is derived from `lastCallUsage`/`promptTokens` instead of accumulated run usage, and `ui/src/ui/chat/context-notice.ts:53`, where the Web UI context notice reads that fresh `totalTokens` value. The shipped regression test in `ui/src/ui/chat/context-notice.test.ts:24` explicitly guards against the reported failure by asserting the UI shows `190k / 200k` and not cumulative `757.3k / 200k`, and `src/auto-reply/reply/session.test.ts:2563` verifies cumulative usage no longer overwrites current-context totals. This behavior is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you can still reproduce it on a newer build, please open a fresh issue with exact repro steps and current session/status payloads.
