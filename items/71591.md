## Summary

Current `main` already preserves DeepSeek V4 reasoning replay across turns by storing replayable reasoning as signed thinking blocks and rehydrating `reasoning_content` for DeepSeek completion payloads; the exact regression is covered by targeted tests. The fix is present on `main` but not in the latest tagged release `v2026.4.23`.

## Evidence

- **Reasoning is preserved when assistant responses are converted into transcript messages:** `buildAssistantMessageFromResponse` maps replayable reasoning output items into assistant `thinking` blocks and stores a `thinkingSignature`, so reasoning metadata is retained in conversation history instead of being dropped.
  - file: [src/agents/openai-ws-message-conversion.ts:572](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/openai-ws-message-conversion.ts#L572)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **DeepSeek replay patch restores required `reasoning_content` on outgoing payloads:** The DeepSeek V4 wrapper patches outgoing payloads so assistant messages with tool calls always carry `reasoning_content` when thinking is enabled, and removes it when thinking is disabled.
  - file: [extensions/deepseek/stream.ts:31](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/deepseek/stream.ts#L31)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **DeepSeek provider wires the replay patch into runtime requests:** The bundled DeepSeek provider registers `createDeepSeekV4ThinkingWrapper` via `wrapStreamFn`, so the replay patch is active for DeepSeek V4 runtime traffic on current `main`.
  - file: [extensions/deepseek/index.ts:48](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/deepseek/index.ts#L48)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Regression test matches the reported multi-turn failure mode:** A targeted test asserts that a prior assistant turn with `thinkingSignature: "reasoning_content"` is replayed as `reasoning_content: "call reasoning"` in the next DeepSeek V4 request, and a companion test asserts blank `reasoning_content` is added for non-DeepSeek prior tool-call turns.
  - file: [extensions/deepseek/index.test.ts:119](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/extensions/deepseek/index.test.ts#L119)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Fix is newer than the latest release tag:** Commit `443b837bd521b0485c87bb7359f370a4ccec10fa` contains the DeepSeek replay implementation and lands after release tag `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`), so this appears fixed on `main` but not yet in a tagged release.
