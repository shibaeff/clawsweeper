## Summary

Current `main` does not implement the requested Control UI mission board. The backend/session types already expose sub-agent hierarchy and live-run metadata, but the browser UI still only offers the generic Sessions table plus chat slash-command controls, not a live per-sub-agent board with inline kill/steer actions.

## Evidence

- **Session API already exposes needed sub-agent fields:** `GatewaySessionRow` includes `spawnedBy`, `status`, `startedAt`, `endedAt`, `runtimeMs`, `childSessions`, `model`, and `modelProvider`, so the data plane exists.
