## Summary

Current `main` no longer forces fallback candidates back to the primary model. Live model reconciliation is now gated on an explicit `liveModelSwitchPending` flag, and fallback explicitly treats `LiveSessionModelSwitchError` as a failover condition so the chain can continue. The follow-up overload timeout concern is also addressed by the current overloaded-profile rotation policy.

## Evidence
