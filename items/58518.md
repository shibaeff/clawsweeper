## Summary

Current `main` already handles isolated heartbeat/cron runs with dedicated session/model-selection logic that honors the requested model instead of falling back to the global primary model, and it has regression coverage for those paths.

## Evidence

- **Heartbeat isolated sessions use a dedicated session key:** `runHeartbeatOnce` switches isolated heartbeat runs onto a dedicated `:heartbeat` session key and persists that isolated session entry before the reply run, so the run no longer reuses the main session lane/model state.
  - file: [src/infra/heartbeat-runner.ts:803](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/heartbeat-runner.ts#L803)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '798,923p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Heartbeat model override wins over stored/default model selection:** `getReplyFromConfig` resolves `heartbeatModelOverride` first and suppresses later stored/channel override replacement when that heartbeat model was resolved, so heartbeat runs keep the payload/configured heartbeat model even when the session/default primary differs.
  - file: [src/auto-reply/reply/get-reply.ts:200](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/auto-reply/reply/get-reply.ts#L200)
  - command: `nl -ba src/auto-reply/reply/get-reply.ts | sed -n '195,220p;338,350p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Cron isolated runs prioritize payload.model:** `resolveCronModelSelection` applies `payload.model` before any session override fallback, so an isolated cron job's requested model becomes the active provider/model selection instead of inheriting `agents.defaults.model.primary`.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/isolated-agent/model-selection.ts#L104)
  - command: `nl -ba src/cron/isolated-agent/model-selection.ts | sed -n '90,135p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Cron persists the chosen model before execution:** The isolated cron path marks the session with the chosen provider/model and persists it pre-run, which prevents the session entry from drifting back to the agent default during execution/retry paths.
  - file: [src/cron/isolated-agent/run.ts:640](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/isolated-agent/run.ts#L640)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '632,645p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression tests cover the reported behavior:** Heartbeat tests assert both `heartbeatModelOverride` forwarding and `:heartbeat` isolated session keys; cron tests assert payload-model forwarding to `runWithModelFallback`/embedded runner and pre-run persistence of the payload model on isolated sessions.
  - file: [src/infra/heartbeat-runner.model-override.test.ts:168](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/heartbeat-runner.model-override.test.ts#L168)
