## Summary

This is a locale-specific privacy/filtering feature request that fits OpenClaw's documented plugin/ClawHub scope, not core maintainer cleanup on `main`. Current core only provides generic configurable log/tool-summary redaction, while the project vision explicitly says optional capability should usually ship as plugins and community distribution belongs on ClawHub.

## Risks / Open Questions

- If maintainers later decide locale-specific PII filtering must be a core security default across all model traffic, this issue could become in-scope again.
- Current core does offer `logging.redactPatterns`, but that is limited to documented redaction surfaces and is not the same as full bilingual traffic filtering/prompt-injection detection.

## Close Comment

Closing this as `clawhub` after a Codex review of current `main`.

The repo direction is explicit that optional capability should usually live in plugins rather than core (`VISION.md:52`). The supported extension/distribution path is external plugins via `openclaw plugins install` with ClawHub-first discovery (`docs/tools/plugin.md:11`, `docs/plugins/community.md:9`). Current core redaction is intentionally generic and operator-configurable through `logging.redactPatterns`, and `main` does not ship built-in Chinese-specific 身份证 / 手机号 / 银行卡 detection patterns (`src/logging/redact.ts:16`, `docs/gateway/logging.md:53`).

Given that this request already points to an external implementation (ShellWard), this is a better fit for a ClawHub/community plugin than OpenClaw core. If you want to keep pushing it forward, the right next step is publishing/maintaining it on ClawHub or npm and sharing the install path with users.
