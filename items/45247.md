## Summary

Current main already implements the Control UI `/compact` path end-to-end: the local slash command dispatches `sessions.compact`, queued local commands are drained when the active run ends, failures are surfaced back into chat instead of being silent, and the gateway explicitly allows the `openclaw-control-ui` client to compact sessions.

## Close Comment

Closing from a Codex review: current main already has the Control UI `/compact` flow implemented. The local slash handler calls `sessions.compact` and always renders a visible success/skip/failure message (`ui/src/ui/chat/slash-command-executor.ts:151`). When `/compact` is entered during an active run, it is queued as a local command and drained when the run reaches a terminal chat event (`ui/src/ui/app-chat.ts:354`, `ui/src/ui/app-gateway.ts:405`). On the gateway side, the webchat mutation guard explicitly exempts the `openclaw-control-ui` client, so Control UI compaction is allowed (`src/gateway/server-methods/sessions.ts:225`, `src/gateway/server-methods/sessions.ts:1492`). The same `/compact` RPC path is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If this still reproduces on a newer build, please open a fresh issue with the exact version plus browser network/console capture so the remaining environment-specific failure can be isolated.
