## Summary

Current `main` does not expose or consume a Gemini web-search `baseUrl` override. The issue is still a valid enhancement request and should remain open.

## Evidence

- **Gemini web-search runtime still hardcodes Google API base URL:** The runtime sets `GEMINI_API_BASE = DEFAULT_GOOGLE_API_BASE_URL`, and `runGeminiSearch()` only accepts `query`, `apiKey`, `model`, and `timeoutSeconds`, so there is no path for a configurable base URL.
  - file: [extensions/google/src/gemini-web-search-provider.runtime.ts:26](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/google/src/gemini-web-search-provider.runtime.ts#L26)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Gemini web-search config type has no baseUrl field:** `GeminiConfig` only defines `apiKey` and `model`, and the resolver helpers only read those two fields.
  - file: [extensions/google/src/gemini-web-search-provider.shared.ts:3](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/google/src/gemini-web-search-provider.shared.ts#L3)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Bundled Google plugin schema/UI do not expose webSearch.baseUrl:** `extensions/google/openclaw.plugin.json` advertises `webSearch.apiKey` and `webSearch.model` in `uiHints`, and its `configSchema.properties.webSearch.properties` also only contains `apiKey` and `model`.
  - file: [extensions/google/openclaw.plugin.json:35](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/google/openclaw.plugin.json#L35)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Current docs still document only apiKey and model for Gemini search:** The Gemini search docs show `plugins.entries.google.config.webSearch` with `apiKey` and `model` only, with no documented `baseUrl` override.
  - file: [docs/tools/gemini-search.md:34](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/docs/tools/gemini-search.md#L34)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **No current HEAD references for a Gemini web-search baseUrl config path:** A repo grep for `plugins.entries.google.config.webSearch.baseUrl` and `tools.web.search.gemini.baseUrl` returned no matches on the checked-out `main`.
  - command: `git grep -n 'plugins.entries.google.config.webSearch.baseUrl\|tools.web.search.gemini.baseUrl' HEAD`
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
