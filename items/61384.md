## Summary

Current `main` no longer depends on an exact `memory/YYYY-MM-DD.md` file at bare `/new` or `/reset` startup. The runtime now preloads recent daily memory, including same-day slugged session-memory artifacts, which covers the mismatch reported here even though the hook still writes `YYYY-MM-DD-slug.md` files.

## Evidence

- **Default docs still describe the original mismatch:** The default AGENTS template still tells agents to read today/yesterday from `memory/` and calls the canonical daily log `memory/YYYY-MM-DD.md`, while the bundled session-memory hook docs still describe output as `memory/YYYY-MM-DD-slug.md`.
