## Summary

Keep open. Current `main` has stale session-lock cleanup and some WebChat error-surfacing hardening, but I did not find strong evidence that issue #47129 is fully fixed. The report is recent, has two independent confirmations, and the current gateway chat path still allows `final` events with no `message`, so the reported blank-final symptom remains plausible.
