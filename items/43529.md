## Summary

Current `main` does not implement this timeout-triggered session reset. The reply runner still only rotates sessions for compaction failures and role-order conflicts, and timeout-only failures still fall through existing generic/error-payload paths. Given that, and the later review note about an additional timeout payload surface, this PR should stay open for maintainer judgment rather than conservative auto-close.
