## Summary

Keep open. Current `main` still has the queue-pruning race the PR is trying to address, so this is not `implemented_on_main`; and the PR as written is not ready for cleanup-closing because it captures only the approval id, which would still break plugin approvals in the same race window and still lacks the requested regression test.

## Evidence
