## Summary

Keep open. Current `main` only implements a config default timeout for `subagent` spawns, not ACP runtime spawns. The issue asks for a default timeout when `sessions_spawn` launches ACP sessions, and that path still does not accept or forward `runTimeoutSeconds` to `spawnAcpDirect`.

## Evidence

- **Subagent default exists:** `resolveConfiguredSubagentRunTimeoutSeconds` reads `agents.defaults.subagents.runTimeoutSeconds` and falls back to it when the call omits `runTimeoutSeconds`.
