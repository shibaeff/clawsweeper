## Summary

Keep open. Current `main` still has the split reported in #45595: trust/persistence paths are derived from non-canonical candidate paths, while execution pins to the executable realpath when available. I did not find evidence on `main` that this asymmetry was fixed.

## Evidence

  - file: [src/infra/exec-command-resolution.ts:62](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/infra/exec-command-resolution.ts#L62)

- **Allowlist matching still compares against `resolvedPath`:** `matchAllowlist()` returns early if `resolution.resolvedPath` is missing and then matches patterns against `const resolvedPath = resolution.resolvedPath`; it never switches to `resolvedRealPath`. That preserves the reported symlink-vs-realpath mismatch for allowlist entries.
