## Summary

Current `main` and the shipped `v2026.4.23` release already avoid the reported `/app/extensions` vs `/app/dist/extensions` crash-loop by resolving installed-package bundled plugins to the built tree and scanning only child entries, not the bundled root itself.

## Evidence

- **Installed/package roots prefer built bundled plugin trees:** `resolveBundledDirFromPackageRoot` chooses `dist-runtime/extensions` or `dist/extensions` for installed/package roots, and only falls back to source `extensions/` for source-checkout cases.
  - file: [src/plugins/bundled-dir.ts:79](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/plugins/bundled-dir.ts#L79)

- **Explicit override remains supported:** `OPENCLAW_BUNDLED_PLUGINS_DIR` is still honored when set, matching the workaround described in the issue and the confirming comment.
  - file: [src/plugins/bundled-dir.ts:116](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/plugins/bundled-dir.ts#L116)
