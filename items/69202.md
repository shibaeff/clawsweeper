## Summary

Keep open. Current main still treats compaction failure as an error path rather than an automatic recovery path. The codebase has restart-time recovery for aborted main sessions, but not an in-process watchdog that resets or rolls over a session after compaction failure. The issue discussion also points to an open broader compaction-invariants fix (#69270), which is not present on current main.

## Evidence

- **Compaction failure still returns an error outcome:** `compactEmbeddedPiSessionDirect()` uses a local `fail()` helper that logs the failure and returns `{ ok: false, compacted: false, reason }`; there is no reset, rollover, or notification path here.
