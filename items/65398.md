## Summary

Keep open. Current `main` still lacks the Feishu named-account promotion fix and still re-runs full Feishu registration on every load, so the underlying bug report is not implemented on `main`. At the same time, the PR as proposed needs maintainer follow-up because its module-level one-shot guard conflicts with current activating-reload behavior that clears global plugin state before re-registration.
