## Summary

Keep open. Current `main` still matches the reported failure mode closely: exec approvals are process-local only, restart recovery resumes sessions whose tail is a `toolResult` without invalidating approval-pending entries, and replay sanitization preserves the approval text that contains the stale approval id. I did not find code on `main` that re-registers or invalidates stale approval ids after a gateway restart.

## Evidence
