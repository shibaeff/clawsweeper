## Summary

Keep open. Current main does send `sessions_spawn({ runtime: "acp", task })` into the ACP child run, but it still does not persist that initial task into session history at spawn time. The code dispatches the task via an `agent` call, then only writes the ACP transcript after `runTurn()` finishes, so the issue's immediate-empty-history symptom is still plausible on current main.

## Evidence

- **ACP spawn forwards task into child run:** `spawnAcpDirect` initializes the ACP session and then calls Gateway `agent` with `message: params.task` for the child `sessionKey`, which shows the task is dispatched to the ACP runtime path.
  - file: [src/agents/acp-spawn.ts:1264](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/acp-spawn.ts#L1264)

That means the initial task is not pre-seeded into transcript/history at spawn time.
