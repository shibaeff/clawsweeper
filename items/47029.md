## Summary

Keep open. Current `main` has partial superseding work: Feishu config/account inspection now uses inspect-mode resolution, and gateway-status surfaces resolve `gateway.auth.*` SecretRefs through a separate auth-resolution path. But the PR is not fully implemented on `main`: `getStatusCommandSecretTargetIds()` still excludes `gateway.auth.token` and `gateway.auth.password`, and current gateway probe logic explicitly keeps device identity disabled for unauthenticated loopback probes, which is the opposite of this PR's loopback-identity change. I also did not find a merged replacement PR; the cross-referenced PR #47055 is closed with `merged_at: null`.
