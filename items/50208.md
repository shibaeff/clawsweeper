## Summary

Current `main` already implements the WhatsApp outbound-listener fix described here: the active listener/controller registry is now pinned to a process-global `globalThis` singleton, outbound sends read from that shared registry, and regression tests cover duplicate-module and split light/heavy runtime-chunk cases. The changelog shows this shipped in release `2026.3.22`.
