## Summary

Current `main` does not implement per-candidate transient retries in the core model fallback path. `runWithModelFallback()` still makes one attempt per candidate and then advances to the next candidate after a failover-classified error. Tests on `main` explicitly expect immediate fallback for transient 5xx and 429 cases. There is a separate auto-reply path that retries the entire fallback cycle once for some transient HTTP errors, but that is narrower than this issue and does not cover the requested same-model retry behavior.

## Evidence
