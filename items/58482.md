## Summary

Keep open. Current `main` still does not implement end-to-end `AbortSignal` support for memory embedding batch polling in the live OpenAI, Gemini, and Voyage providers, so the PR’s core request is not already fixed on `main`. One unrelated UI locale regression mentioned in review is already absent on `main`, but that does not resolve the main cancellation gap.

## Evidence

- **Batch options still have no signal surface:** `MemoryEmbeddingBatchOptions` exposes `agentId`, `chunks`, `wait`, `concurrency`, `pollIntervalMs`, `timeoutMs`, and `debug`, but no `signal`, so provider adapters still cannot receive disconnect cancellation from callers.
