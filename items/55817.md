## Summary

Current `main` still has the rollover consistency gap described by the PR: it persists the new `sessionId`/`sessionFile` to the session store before any transcript file is created, and it archives the previous transcript during the same rollover path. The PR discussion also still carries an unresolved review concern about non-atomic header creation, so this is not a conservative close candidate.

## Evidence

- **Rollover persists the new session entry first:** `initSessionState()` on current `main` calls `resolveAndPersistSessionFile()` during rollover with no transcript-bootstrap step before continuing the session-reset flow.
