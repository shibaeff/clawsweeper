## Summary

Keep open. Current `main` has the argv-inspection helper, but the gateway service startup path still relies on the older command-name-based stale-PID cleanup. The canonical systemd unit still launches the gateway as `node .../openclaw/dist/entry.js gateway`, so the PR's npm-global orphan-process scenario remains plausible on `main`. The discussion also shows follow-up work (`fff8fecb`) that addressed earlier review blockers, so this PR looks stale/rework-needed rather than obsolete.
