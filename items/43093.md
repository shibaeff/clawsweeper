## Summary

Keep open. Current `main` does not implement this PR’s active-log-file-path fix: log tailing still starts from `getResolvedLoggerSettings().file`, which recomputes the rolling filename from the current date, and the existing fallback only helps when that recomputed file is missing.

## Evidence

- **Current tail path still uses resolved settings:** `src/logging/log-tail.ts:150-156` still does `resolveLogFile(getResolvedLoggerSettings().file)` before reading logs, so it does not use a cached active logger file path.
  - file: [src/logging/log-tail.ts:150](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/logging/log-tail.ts#L150)
  - command: `nl -ba src/logging/log-tail.ts | sed -n '150,156p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Resolved logger settings still recompute today's rolling file:** `src/logging/logger.ts:127` sets `file = cfg?.file ?? defaultRollingPathForToday()`, `getResolvedLoggerSettings()` at `src/logging/logger.ts:296-297` returns `resolveSettings()`, and `defaultRollingPathForToday()` at `src/logging/logger.ts:337-339` derives the filename from `new Date()`. That is the behavior the PR is trying to avoid for long-running gateways.
  - file: [src/logging/logger.ts:127](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/logging/logger.ts#L127)

  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **No active-path helper exists on main:** `src/logging.ts:15-25` exports `getResolvedLoggerSettings` but no `getActiveLogFilePath`, and `rg -n "getActiveLogFilePath" src` returned no matches.
  - file: [src/logging.ts:15](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/logging.ts#L15)

## Risks / Open Questions

- The current fallback can still select the wrong file whenever the recomputed current-day rolling file exists but is not the file the running gateway is actively writing.
- This PR was based on the older `src/gateway/server-methods/logs.ts` call site, but the same underlying issue now lives in `src/logging/log-tail.ts`, so the work is still relevant rather than obsolete.
