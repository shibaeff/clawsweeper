## Summary

Current `main` already ships a native background memory-consolidation feature under `memory-core` named `Dreaming`: it auto-manages a cron sweep, ingests recent memory/transcript signals, writes reviewable `DREAMS.md` artifacts, and promotes qualified items into `MEMORY.md`. The same feature is present in release `v2026.4.23`.

## Evidence

- **Dreaming is documented as native background consolidation:** `docs/concepts/dreaming.md` describes `Dreaming` as the background memory consolidation system in `memory-core`, including machine state in `memory/.dreams/`, review output in `DREAMS.md`, transcript ingestion, and an auto-managed cron sweep.

  - file: [docs/cli/memory.md:126](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/cli/memory.md#L126)
