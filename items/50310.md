## Summary

Current `main` does not implement the requested cleanup. On Discord thread archive, OpenClaw still only marks matching session-store rows stale by setting `updatedAt = 0`; it does not remove those keys from `sessions.json`. `sessions_list` still enumerates store entries directly, so those stale rows can continue surfacing. There is a gateway-level `sessions.delete` RPC, but the agent-facing session tool surface still does not expose a delete/GC tool, and the Discord archive/delete paths inspected here do not call that RPC for channel sessions.
