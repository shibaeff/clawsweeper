## Summary

Current `main` already contains a transport-timeout fix that matches this Ollama agent-mode failure: embedded runs now propagate their timeout into guarded Undici dispatchers instead of falling back to Undici's default per-request timeout, and the current tree includes regression coverage for that wiring. The fix appears to be on `main` after `v2026.4.23`, so it has not necessarily shipped in a tagged release yet.

## Evidence

- **Current changelog calls out the exact Ollama timeout class:** `CHANGELOG.md` explicitly records an agents/transport fix: configured attempt timeouts now flow into guarded per-request dispatchers so slow local LLM calls such as Ollama no longer fail at Undici's default 60-second body timeout.
  - file: [CHANGELOG.md:175](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/CHANGELOG.md#L175)
  - command: `nl -ba CHANGELOG.md | sed -n '168,180p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Current main passes timeout into guarded dispatchers:** `fetchWithSsrFGuard()` now resolves `timeoutMs` and passes it into every dispatcher construction path (`createHttp1EnvHttpProxyAgent`, `createPolicyDispatcherWithoutPinnedDns`, `createPinnedDispatcher`). That is the code path used for provider HTTP requests inside embedded agent runs.
  - file: [src/infra/net/fetch-guard.ts:359](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/infra/net/fetch-guard.ts#L359)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '352,385p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Latest release did not yet have that propagation:** At `v2026.4.23`, the same guarded fetch path constructed dispatchers without passing any timeout override, which left requests exposed to Undici's default per-request timeout behavior.
  - file: [src/infra/net/fetch-guard.ts:352](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/net/fetch-guard.ts#L352)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/infra/net/fetch-guard.ts | nl -ba | sed -n '348,382p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Embedded runs now preserve a long shared stream timeout:** Embedded attempt setup keeps the global Undici stream timeout at least at the shared 30-minute default, and `undici-global-dispatcher.ts` stores that value so guarded fetches can reuse it.
  - file: [src/agents/pi-embedded-runner/run/attempt-http-runtime.ts:7](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/pi-embedded-runner/run/attempt-http-runtime.ts#L7)

- **Regression test covers the timeout wiring:** `attempt.spawn-workspace.timeout.test.ts` asserts embedded runs do not lower global Undici tuning below the shared default and preserve longer run timeouts when configured.
