## Summary

Current `main` still appears to preserve host-workspace skill paths in sandboxed non-`rw` runs. OpenClaw mirrors skill files into the sandbox workspace, but the run path reuses a prebuilt `skillsSnapshot.prompt` from the original workspace instead of rebuilding or remapping it for the sandbox workspace.

## Evidence

- **Snapshot is built from the original workspace:** `buildWorkspaceSkillSnapshot(workspaceDir, ...)` is created in the session setup path using the prepared `workspaceDir`, before the embedded runner switches to the sandbox copy.
  - file: [src/agents/agent-command.ts:575](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/agent-command.ts#L575)

- **Snapshot contents use workspace-local skill paths:** Skill discovery resolves `workspaceDir/skills`, and prompt generation formats those discovered `skill.filePath` values directly into the skills prompt.
