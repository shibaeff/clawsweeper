## Summary

Keep open. Current main clearly treats system-prompt/tool-instruction text as soft guidance and enforces hard restrictions through tool policy, so the reported `curl` fallback likely means `exec` was still available rather than a real `web_fetch` bypass. But the issue also reports repeated browser timeouts in a WSL/Windows11 Docker setup, and while main has added multiple browser timeout/recovery fixes and better diagnostics, I do not have strong enough evidence that this exact current-main scenario is resolved well enough to close conservatively.
