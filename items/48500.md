## Summary

Keep open. Current `main` still leaves the reported restart gap in place: the refactored skills refresh state starts at version `0`, untouched workspaces still report snapshot version `0` after restart, and both agent session paths still use a predicate that does not rebuild a persisted `version: 0` snapshot when the next version is also `0`. The PR is not redundant, though it would need rebasing onto the newer `refresh-state.ts`/`agent-command.ts` structure.

## Evidence

- **Cold-start global version is still zero:** Current `main` initializes the shared skills snapshot version with `let globalVersion = 0`, so a fresh gateway boot has no positive version marker to distinguish pre-restart snapshots from post-restart state.
  - file: [src/agents/skills/refresh-state.ts:9](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/skills/refresh-state.ts#L9)

  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)
- **Untouched workspaces still resolve to version 0:** `getSkillsSnapshotVersion(workspaceDir)` returns `Math.max(globalVersion, local)`. With no watcher/config/remote bump after restart, both values stay `0`, matching the stale-session scenario described in the PR body.
  - file: [src/agents/skills/refresh-state.ts:57](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/skills/refresh-state.ts#L57)

  - file: [src/agents/skills/refresh-state.ts:65](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/skills/refresh-state.ts#L65)
