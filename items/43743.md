## Summary

Keep open. Current main still renders assistant thinking/working output through the same `.chat-thinking` container, and that container still has no explicit CJK-safe font-family or other obvious fix for the reported Chinese text rendering problem. I found no evidence on `main` or in the latest shipped tag `v2026.4.22` that this issue was implemented or obsoleted.
