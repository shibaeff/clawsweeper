## Summary

Keep open. Current `main` still drops wake routing metadata for the `cron` tool path, so `cron` `action:"wake"` cannot target a non-default agent/session even though the lower cron gateway adapter can route targeted wake/enqueue requests when those fields are provided.

## Evidence

- **Cron tool wake call only sends mode and text:** The `cron` tool's `wake` branch forwards only `{ mode, text }` to gateway method `wake`; it does not read or pass `agentId` or `sessionKey`.
  - file: [src/agents/tools/cron-tool.ts:678](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/agents/tools/cron-tool.ts#L678)
  - command: `nl -ba src/agents/tools/cron-tool.ts | sed -n '665,695p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Gateway wake handler only forwards mode and text:** Gateway `wake` validation accepts the request, but the handler narrows params to `{ mode, text }` and calls `context.cron.wake({ mode: p.mode, text: p.text })`, dropping any extra routing fields.
  - file: [src/gateway/server-methods/cron.ts:118](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/gateway/server-methods/cron.ts#L118)
  - command: `nl -ba src/gateway/server-methods/cron.ts | sed -n '118,138p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Cron service wake ignores routing options:** `CronServiceContract.wake` only exposes `{ mode, text }`, and `wake()` enqueues the system event with no options and requests heartbeat with only `{ reason: "wake" }`. That forces fallback routing instead of a targeted agent/session wake.
  - file: [src/cron/service-contract.ts:35](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/cron/service-contract.ts#L35)
  - command: `nl -ba src/cron/service-contract.ts | sed -n '18,40p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Main-job execution path already supports targeted routing:** Scheduled main-session cron execution calls `state.deps.enqueueSystemEvent(text, { agentId: params.job.agentId })`. The gateway cron adapter also resolves `agentId`/`sessionKey` when those options are supplied, so the missing propagation is specific to `wake`, not the underlying routing machinery.
  - file: [src/cron/service/timer.ts:380](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/cron/service/timer.ts#L380)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '374,386p;1409,1421p' && nl -ba src/gateway/server-cron.ts | sed -n '248,307p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Wake schema does not define agent targeting:** The protocol wake schema defines only `mode` and `text`. It allows additional properties for forward compatibility, but current handlers ignore them, so there is no implemented agent-targeted wake contract on `main`.
  - file: [src/gateway/protocol/schema/agent.ts:197](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/gateway/protocol/schema/agent.ts#L197)
  - command: `nl -ba src/gateway/protocol/schema/agent.ts | sed -n '197,206p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
