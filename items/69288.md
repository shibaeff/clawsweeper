## Summary

Keep open. Current `main` still has the reported gap: final-text sanitization already strips `<antml:thinking>` tags, but the streaming/block-reply parsers in `src/agents/pi-embedded-subscribe.ts` and `src/agents/pi-embedded-utils.ts` still only recognize `think|thinking|thought|antthinking`, so namespaced Anthropic-style tags can still leak during streaming as described in #69235. The PR is addressing a live mismatch rather than duplicate or stale work.

## Evidence

- **Streaming block-strip regex still excludes `antml:thinking`:** `THINKING_TAG_SCAN_RE` in the streaming path matches only `think(?:ing)?|thought|antthinking`, with no `antml:` namespace support.
  - file: [src/agents/pi-embedded-subscribe.ts:38](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/pi-embedded-subscribe.ts#L38)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Streaming reasoning extraction helpers still exclude namespaced tags:** `splitThinkingTaggedText`, `extractThinkingFromTaggedText`, and `extractThinkingFromTaggedStream` all use regexes that do not include `antml:thinking`, so message-end promotion and reasoning-stream extraction still miss that form.
  - file: [src/agents/pi-embedded-utils.ts:192](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/pi-embedded-utils.ts#L192)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Current shared final-text sanitizer already supports namespaced tags:** `stripReasoningTagsFromText` uses `(?:(?:antml:)?(?:think(?:ing)?|thought)|antthinking)` and is the helper behind final visible-text sanitization, matching the PR description that final extraction is already fixed while streaming is not.
  - file: [src/shared/text/reasoning-tags.ts:5](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/shared/text/reasoning-tags.ts#L5)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Existing tests confirm final-path support but not the streaming-path fix:** Current tests already cover stripping `antml:thinking` from visible text, which supports the report that only the streaming/block path remains inconsistent.
  - file: [src/shared/text/reasoning-tags.test.ts:54](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/shared/text/reasoning-tags.test.ts#L54)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Direct regex check on current main reproduces the mismatch:** A read-only Node check against current regexes showed `streaming matches 0` and `shared matches 2` for `<antml:thinking>hidden</antml:thinking>Visible`, confirming the streaming parser misses the namespaced tags that the shared sanitizer catches.
  - command: `node - <<'NODE'
const streamingRe = /<\s*(\/?)\s*(?:think(?:ing)?|thought|antthinking)\s*>/gi;
const text = '<antml:thinking>hidden</antml:thinking>Visible';
console.log('streaming matches', [...text.matchAll(streamingRe)].length);
const sharedRe = /<\s*(\/?)\s*(?:(?:antml:)?(?:think(?:ing)?|thought)|antthinking)\b[^<>]*>/gi;

NODE`
