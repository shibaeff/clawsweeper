## Summary

## Close Comment

Closing this as a Codex triage review because it fits the ClawHub/community-plugin path, not OpenClaw core. Current main explicitly says optional capability should usually ship as plugins and be maintained outside the core repo (`VISION.md:54`), documents community providers/plugins as ClawHub or npm installs (`docs/plugins/community.md:9`, `docs/cli/plugins.md:54`), and already supports generic OpenAI-compatible backends through custom provider config (`docs/concepts/model-providers.md:566`, `docs/providers/inferrs.md:10`). I also checked current main and found no in-tree Maple provider/plugin surface under `extensions/`, `docs/`, or `src/`, which is consistent with that scope boundary rather than a missing core hook. If Maple support should advance, the documented route is an external OpenClaw plugin or ClawHub package, not a core issue in `openclaw/openclaw`.
