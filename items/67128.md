## Summary

Keep open. Current `main` already exposes Telegram `/subagents` and `/usage` commands, and `/subagents list` now includes per-subagent token totals/input/output, so the original “no way to check token consumption in Telegram” claim is only partially true. But there is still no evidence of the specific richer feature requested: a Telegram usage/session-stats surface with per-agent API call counts or a consolidated breakdown panel.

## Risks / Open Questions

- Closing as implemented would overstate the current behavior: token visibility exists, but the requested per-agent API call counts and consolidated Telegram stats surface do not appear to exist.
