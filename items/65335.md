## Summary

Current `main` already fixes this: the Dreams UI now skips optional `memory-wiki` RPC calls unless the plugin is enabled or the gateway advertises those methods, refreshes config before wiki reloads, and shows an enablement CTA instead of surfacing `unknown method` errors.

## Evidence

- **Dreams controller gates optional wiki RPCs:** `isMemoryWikiEnabled` and `canCallMemoryWikiMethod` gate `wiki.importInsights` / `wiki.palace`; the loaders return early and clear stale errors when `memory-wiki` is off or the gateway does not advertise the method.
  - file: [ui/src/ui/controllers/dreaming.ts:234](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/controllers/dreaming.ts#L234)
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Dreams view shows enablement CTA instead of method error:** When a wiki subtab is selected and `memory-wiki` is disabled, the UI renders `Memory Wiki is not enabled` plus `plugins.entries.memory-wiki.enabled = true` guidance and an Open Config button.
  - file: [ui/src/ui/views/dreaming.ts:1443](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/views/dreaming.ts#L1443)
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Reload path refreshes config before wiki calls:** The Dreams tab now reloads config before refreshing Imported Insights or Memory Palace, matching the changelog note about avoiding stale unknown-method failures after config changes.
  - file: [ui/src/ui/app-render.ts:2425](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/app-render.ts#L2425)
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Regression tests cover the reported failure mode:** Tests assert that Imported Insights and Memory Palace do not call `wiki.importInsights` / `wiki.palace` when `memory-wiki` is not enabled or when the gateway does not advertise those methods.
  - file: [ui/src/ui/controllers/dreaming.test.ts:340](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/controllers/dreaming.test.ts#L340)
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Changelog records the fix in release 2026.4.14:** The `2026.4.14` changelog includes: `Control UI/Dreaming: stop Imported Insights and Memory Palace from calling optional memory-wiki gateway methods when the plugin is off, and refresh config before wiki reloads...`
  - file: [CHANGELOG.md:822](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/CHANGELOG.md#L822)
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **These RPCs belong to the optional memory-wiki plugin:** `wiki.importInsights` and `wiki.palace` are registered by the bundled `memory-wiki` plugin, so disabling that plugin should not leave the Dreams UI calling them directly.
  - file: [extensions/memory-wiki/src/gateway.ts:134](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/memory-wiki/src/gateway.ts#L134)
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
