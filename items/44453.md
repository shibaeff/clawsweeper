## Summary

## Close Comment

Closing as implemented on main after Codex review. The old `ReferenceError` path reported for `v2026.3.11` is not present in current OpenClaw source or the latest shipped release. On current `main`, the Control UI imports `buildAgentMainSessionKey` in `ui/src/ui/app-render.ts:4` and uses it on the agent-switch path in `ui/src/ui/app-render.ts:2290`; the helper is defined in `ui/src/ui/session-key.ts:60` and `src/routing/session-key.ts:121`. `resolveAgentMainSessionKey` still exists only in config/session resolution code (`src/config/sessions/main-session.ts:29`), not in the UI path that would cause this runtime `ReferenceError`. I also checked the `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), and it already contains the same corrected `buildAgentMainSessionKey(...)` usage, so this regression appears fixed in the latest release.
