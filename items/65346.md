## Summary

Current `main` already fixes this by invalidating cached session skill snapshots whenever `skills.*` config changes. The prompt path already respected `shouldIncludeSkill`; the stale entries came from reusing an old `skillsSnapshot` until #67401 added config-change invalidation.

## Evidence

- **Prompt builder already filters with `shouldIncludeSkill`:** `filterSkillEntries()` applies `shouldIncludeSkill({ entry, config, eligibility })` before building the available-skills prompt, so bundled allowlist filtering was already in the prompt-building path.
  - file: [src/agents/skills/workspace.ts:101](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/skills/workspace.ts#L101)

- **Session snapshots are cached and reused until version changes:** `agent-command` reuses `sessionEntry.skillsSnapshot` unless `shouldRefreshSnapshotForVersion(...)` says it is stale, which matches the issue's symptom of old skills persisting across turns.
  - file: [src/agents/agent-command.ts:578](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/agent-command.ts#L578)
