## Summary

Current main already suppresses exact assistant `NO_REPLY` responses in WebChat both during live chat events and when loading chat history, which covers the reported broken post-delivery empty-turn UX.

## Evidence

- **WebChat controller drops silent assistant replies:** `handleChatEvent` ignores `NO_REPLY` deltas, drops `NO_REPLY` finals, and avoids persisting a `NO_REPLY` stream as a visible assistant message. `loadChatHistory` also filters assistant silent replies from fetched history.
  - file: [ui/src/ui/controllers/chat.ts:40](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/ui/src/ui/controllers/chat.ts#L40)
- **UI regression tests cover the reported cases:** Tests explicitly verify ignored `NO_REPLY` delta updates, dropped `NO_REPLY` final payloads from another run and the active run, no persisted `NO_REPLY` stream text, and filtered `NO_REPLY` assistant history rows.
  - file: [ui/src/ui/controllers/chat.test.ts:95](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/ui/src/ui/controllers/chat.test.ts#L95)
- **Gateway-side history normalization omits silent assistant rows:** Gateway `chat.history` sanitization extracts visible assistant text for silent-token checks and drops assistant messages whose visible text is only `NO_REPLY` when there is no non-text payload to keep.
  - file: [src/gateway/server-methods/chat.ts:1162](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/gateway/server-methods/chat.ts#L1162)
- **Gateway event tests suppress silent control replies in WebChat streams:** Gateway chat event tests verify that suppressed control text like `NO_REPLY` does not emit chat deltas and does not appear in final chat messages.
  - file: [src/gateway/server-chat.agent-events.test.ts:218](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/gateway/server-chat.agent-events.test.ts#L218)
- **Docs describe the fixed WebChat behavior:** WebChat docs now state that `chat.history` omits assistant entries whose whole visible text is only the exact silent token `NO_REPLY` / `no_reply`.
  - file: [docs/web/webchat.md:27](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/web/webchat.md#L27)

## Close Comment

The regression coverage is explicit in `ui/src/ui/controllers/chat.test.ts:95`, `ui/src/ui/controllers/chat.test.ts:136`, `ui/src/ui/controllers/chat.test.ts:420`, and `ui/src/ui/controllers/chat.test.ts:525`. Gateway-side normalization also backs this up in `src/gateway/server-methods/chat.ts:1162` and `src/gateway/server-chat.agent-events.test.ts:218`, and the behavior is documented in `docs/web/webchat.md:27`. I could not tie the implementation to a released tag from this checkout, so this is being closed as implemented on current `main`, not as a released-fix attribution.
