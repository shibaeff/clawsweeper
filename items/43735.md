## Summary

Keep open. Current `main` does refresh stale persisted skill snapshots after restart and documents/watches mid-session skill changes, so the specific restart-staleness theory is partly addressed. But `main` still deliberately limits skill discovery and `<available_skills>` prompt rendering, so the reported symptom that only a subset of workspace skills appears in agent context is still plausible and not strongly resolved.

## Evidence

- **Restart-stale snapshots are now explicitly refreshed:** `shouldRefreshSnapshotForVersion` returns true when the persisted cached version is nonzero and the current process version has reset to `0`, which makes old session snapshots refresh after a restart.
