## Summary

Keep open. Current `main` does not implement this LAN-interface preference change, and the reported behavior is still plausible in three live code paths (`gateway`, pairing setup, and bundled `device-pair`). The PR is directionally useful, but the provided review thread raises unresolved maintainability/correctness concerns about duplicated selection logic and incomplete CNI de-prioritization.

## Risks / Open Questions

- The PR currently duplicates the new LAN-selection heuristic across core and the bundled `device-pair` plugin, which increases future drift risk.
- The proposed de-prioritization regex appears incomplete for common Kubernetes/CNI interfaces, so the fix may still mis-advertise overlay IPs on some hosts.
- Live GitHub API inspection was unavailable locally because `gh` lacked `GH_TOKEN`; this review relied on the supplied PR context plus local `main` source inspection.
