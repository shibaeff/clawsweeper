## Summary

Current `main` already routes `message thread create --channel telegram` through the Telegram plugin remap, converting `thread-create` to `topic-create` and `threadName` to `name`; the same implementation is present in released `v2026.4.23` and is covered by a targeted regression test.

## Evidence

- **CLI thread-create dispatch now consults channel remap hooks:** `resolveThreadCreateRequest` asks the selected channel plugin to rewrite `thread-create` before `runMessageAction` is called, instead of always dispatching the generic action unchanged.
  - file: [src/cli/program/message/register.thread.ts:7](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/cli/program/message/register.thread.ts#L7)

- **Telegram plugin rewrites thread-create to topic-create:** The Telegram channel action adapter maps `thread-create` to `topic-create` and rewrites `threadName` into the required `name` field, matching the issue's requested bridge behavior.
  - file: [extensions/telegram/src/channel-actions.ts:163](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/extensions/telegram/src/channel-actions.ts#L163)
