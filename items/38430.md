## Summary

Keep open. Current `main` still intentionally shortens home-directory paths to `~/` in both prompt-facing and user-visible display code, so the reported behavior is not fixed on `main`. I did not find any `OPENCLAW_EXPAND_PATHS`-style control on the checked-out tree.
