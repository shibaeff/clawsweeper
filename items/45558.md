## Summary

Keep open. I did not find strong evidence on current `main` that compaction strips or otherwise neutralizes `thinking`/`redacted_thinking` blocks before summarization for Anthropic Claude 4.6-class models, which is the core request in #45558.

## Evidence

- **Compaction sanitizer only removes tool result details:** `summarizeChunks()` sanitizes compaction input with `stripToolResultDetails(params.messages)` and immediately chunks that result; there is no analogous thinking/redacted_thinking filter in this path.
  - file: [src/agents/compaction.ts:309](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/compaction.ts#L309)
  - command: `nl -ba src/agents/compaction.ts | sed -n '292,340p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Safeguard compaction passes raw summarizable messages into LLM summarization:** The current safeguard path calls `summarizeViaLLM({ messages: messagesToSummarize, ... })` without first calling `dropThinkingBlocks()` or any equivalent helper on the compaction transcript.
  - file: [src/agents/pi-hooks/compaction-safeguard.ts:1009](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-hooks/compaction-safeguard.ts#L1009)
  - command: `nl -ba src/agents/pi-hooks/compaction-safeguard.ts | sed -n '1001,1022p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Current replay policy still preserves thinking blocks for Claude Opus 4.6:** The transcript-policy tests explicitly assert `dropThinkingBlocks === false` for `claude-opus-4-6`, the model family called out in the issue environment. That means current main does not globally strip these blocks for this model family.
  - file: [src/agents/transcript-policy.test.ts:290](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/transcript-policy.test.ts#L290)
  - command: `nl -ba src/agents/transcript-policy.test.ts | sed -n '288,312p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Thinking stripping exists on replay/send paths, not on compaction summarization:** Current main uses `dropThinkingBlocks()` when sanitizing replay history and outbound follow-up requests, which shows the repo has a helper for this class of problem, but I did not find it wired into the compaction summarization path itself.
  - file: [src/agents/pi-embedded-runner/replay-history.ts:479](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-embedded-runner/replay-history.ts#L479)
