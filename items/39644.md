## Summary

Keep open. Current `main` still contains both Windows behaviors the PR describes: PowerShell completion install falls through to "not supported," and Windows time-format detection still hardcodes `powershell` instead of the existing `resolvePowerShellPath()` helper. The latest release `v2026.4.23` ships the same codepaths, so this is neither implemented nor obsolete. The PR also appears salvageable rather than stale-only: the discussion's one concrete review nit was already addressed in a follow-up commit, but the patch now needs a rebase because completion install logic lives in `src/cli/completion-runtime.ts` on current `main`.
