## Summary

Current `main` already contains the pairing and token-drift fixes that match this report: superseded pairing requests now surface the live `requestId`, broader reconnects are explained as approval upgrades instead of looking like lost pairing, and docs/doctor now point operators to rotate/remove/re-approve stale device tokens.
