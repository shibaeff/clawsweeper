## Summary

Keep open. Current `main` still does not implement `--agent-id` for `openclaw system event`: the CLI, wake RPC schema/handler, cron wake contract, generated protocol models, and docs all still model manual wake as main-session-only. The PR discussion also surfaced a real follow-up concern that typoed or unknown agent IDs would currently fall back to the default agent in cron wake routing, so this is neither implemented-on-main nor safe to dismiss as obsolete.

## Risks / Open Questions

- Any eventual fix needs the protocol schema, generated client models, docs, and CLI/tests updated together; current `main` is still inconsistent with the PR’s intended surface.
