## Summary

Keep open. Current `main` partially improved Web UI image handling, but it still does not wire pasted `chat.send` attachments into `MsgContext.MediaPath/MediaPaths`, so media understanding remains bypassed and text-only model runs still cannot process those images through the auxiliary media-understanding path.

## Evidence

- **Media understanding still only runs when inbound media is present on `MsgContext`:** `applyMediaUnderstandingIfNeeded` returns early unless `hasInboundMedia(ctx)` is true, and `hasInboundMedia` only checks `MediaPath`, `MediaUrl`, `MediaPaths`, `MediaUrls`, `MediaTypes`, sticker fields. It does not inspect `replyOptions.images` or `[media attached: ...]` markers.
  - file: [src/auto-reply/reply/get-reply.ts:137](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/auto-reply/reply/get-reply.ts#L137)
  - command: `nl -ba src/auto-reply/reply/get-reply.ts | sed -n '128,148p'; nl -ba src/auto-reply/reply/inbound-media.ts | sed -n '1,80p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Gateway still leaves Web UI attachment media off the dispatched `MsgContext`:** Gateway tests on current `main` explicitly assert that the saved transcript user turn gets `MediaPath/MediaPaths`, but the dispatched runtime context does not: `mockState.lastDispatchCtx?.MediaPath` and `mockState.lastDispatchCtx?.MediaPaths` are both `undefined`. That means media understanding still sees no inbound media for ordinary webchat sends.
  - file: [src/gateway/server-methods/chat.directive-tags.test.ts:1750](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/server-methods/chat.directive-tags.test.ts#L1750)

- **`chat.send` still passes pasted images separately as reply options, not inbound media context:** The gateway dispatch path forwards parsed attachments via `replyOptions.images`, not by populating `ctx.MediaPath/MediaPaths` for normal webchat sends.
  - file: [src/gateway/server-methods/chat.ts:2503](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/server-methods/chat.ts#L2503)
