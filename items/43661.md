## Summary

Keep open. Current `main` still intentionally retries the prompt after successful timeout-triggered compaction, and the shared message tool still does not supply a stable idempotency key for outbound sends. I did not find strong evidence that the duplicate-send / wedged-session family reported in #43661 is fixed on `main` or in the latest release.

## Evidence

- **Timeout compaction still replays the prompt:** The embedded runner still enters timeout-triggered compaction and then unconditionally `continue`s into a fresh prompt attempt when compaction succeeds. There is no guard here for prior mutating tool work or prior outbound sends.
  - file: [src/agents/pi-embedded-runner/run.ts:965](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/pi-embedded-runner/run.ts#L965)

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Current tests still assert retry-after-compaction behavior:** The current test suite explicitly expects a second prompt attempt after successful timeout compaction (`mockedRunEmbeddedAttempt` called twice), which matches the replay behavior described in the issue rather than a one-shot failure path.
  - file: [src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts:105](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/pi-embedded-runner/run.timeout-triggered-compaction.test.ts#L105)

- **Message tool does not pass a stable idempotency key:** The agent message tool calls `runMessageActionForTool` without adding any per-turn/per-tool idempotency key. That means a replayed tool invocation is not deduped at the tool boundary.
  - file: [src/agents/tools/message-tool.ts:806](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/tools/message-tool.ts#L806)

That does not provide protection against duplicate sends caused by a replayed compaction retry.
  - file: [src/infra/outbound/message.ts:227](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/infra/outbound/message.ts#L227)
