## Summary

Current `main` already routes `openclaw status` and `openclaw status --deep` through the snapshot-based plugin compatibility path instead of the diagnostics-heavy path described in the issue.

## Evidence

- **Non-JSON status path uses scanStatus for normal and deep output:** `statusCommand(...)` sends regular text status invocations, including `--deep`, through `scanStatus(...)`; the deep flag is handled later in runtime snapshotting, not by switching plugin compatibility code paths.
  - file: [src/commands/status.command.ts:99](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/commands/status.command.ts#L99)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **scanStatus now uses snapshot compatibility notices:** The status scan progress step calls `buildPluginCompatibilitySnapshotNotices({ config: overview.cfg })` before building the final status result.
  - file: [src/commands/status.scan.ts:65](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/commands/status.scan.ts#L65)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Snapshot helper avoids diagnostics rebuilds:** `buildPluginCompatibilitySnapshotNotices(...)` first builds a snapshot report, then passes that report into `buildPluginCompatibilityNotices(...)`, so compatibility notices reuse the snapshot report instead of defaulting to `buildPluginDiagnosticsReport(...)`.
  - file: [src/plugins/status.ts:364](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/plugins/status.ts#L364)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Snapshot report is metadata-only; diagnostics report is module-loading:** `buildPluginReport(..., false)` uses `loadPluginMetadataRegistrySnapshot(... loadModules: false)`, while `buildPluginDiagnosticsReport(...)` uses `loadOpenClawPlugins(...)` with module loading enabled.
  - file: [src/plugins/status.ts:183](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/plugins/status.ts#L183)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Test coverage asserts snapshot path does not call full plugin loader:** The test `uses a metadata snapshot load for snapshot reports` verifies `buildPluginSnapshotReport(...)` calls `loadPluginMetadataRegistrySnapshot` with `loadModules: false` and does not call `loadOpenClawPlugins`.
  - file: [src/plugins/status.test.ts:395](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/plugins/status.test.ts#L395)
