## Summary

Keep open. Current `main` does not show a landed fix for this item: `--log-file` and `OPENCLAW_LOG_FILE` are still not implemented, the logger/config path logic is effectively unchanged from the latest release, and the UID-suffixed temp-dir fallback reported in comments still exists. I also could not find source evidence that conclusively disproves the remaining `logging.file` bug report.
