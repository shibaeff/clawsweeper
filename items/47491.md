## Summary

Keep open: current `main` fixed the `before_reset` context and the runner-owned compaction hook path, but the embedded subscribe compaction hook path still passes only `sessionKey` to plugin hooks, so the PR's stated goal of preserving full agent/session context across compaction paths is not fully implemented on `main`.

## Evidence

- **`before_reset` now resolves agent from the session key:** Current `main` passes `agentId: resolveAgentIdFromSessionKey(params.sessionKey)` plus `sessionId` and `workspaceDir` into `runBeforeReset`, which addresses the bad raw-prefix extraction described in the PR for reset hooks.
  - file: [src/auto-reply/reply/commands-reset-hooks.ts:149](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/auto-reply/reply/commands-reset-hooks.ts#L149)
  - command: `nl -ba src/auto-reply/reply/commands-reset-hooks.ts | sed -n '140,160p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Runner-owned compaction hooks on `main` already carry full scoped context:** `runBeforeCompactionHooks` and `runAfterCompactionHooks` send `sessionId`, `agentId`, `sessionKey`, and `workspaceDir` in the compaction hook context, so part of the PR's goal is already implemented in the newer runner compaction path.
  - file: [src/agents/pi-embedded-runner/compaction-hooks.ts:202](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-runner/compaction-hooks.ts#L202)
  - command: `nl -ba src/agents/pi-embedded-runner/compaction-hooks.ts | sed -n '200,214p'`

- **Embedded subscribe compaction start still drops `agentId` and `sessionId`:** `handleCompactionStart` still calls `runBeforeCompaction(..., { sessionKey: ctx.params.sessionKey })`, omitting `ctx.params.agentId` and `ctx.params.sessionId` even though those fields exist on the subscribe params and are the exact data this PR wanted to preserve.
  - file: [src/agents/pi-embedded-subscribe.handlers.compaction.ts:26](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-subscribe.handlers.compaction.ts#L26)

- **Embedded subscribe compaction end still drops `agentId` and `sessionId`:** `handleCompactionEnd` likewise calls `runAfterCompaction(..., { sessionKey: ctx.params.sessionKey })`, so the embedded compaction completion path still does not provide the same scoped runtime context that other compaction paths provide.
  - file: [src/agents/pi-embedded-subscribe.handlers.compaction.ts:92](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-subscribe.handlers.compaction.ts#L92)
