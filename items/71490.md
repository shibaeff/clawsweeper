## Summary

Keep open. Current `main` still strips LM Studio `@iq*` quant suffixes as auth profiles, so the reported bug is still present and this PR contains a focused fix plus regression coverage that is not yet on `main`.

## Evidence

- **Current parser still excludes `@iq*` suffixes:** `splitTrailingAuthProfile` on current `main` only preserves `@q*` and `<n>bit` suffixes. The protected regex is `^(?:q\d+(?:_[a-z0-9]+)*|\d+bit)(?:@|$)`, so `@iq3_xxs` is not treated as part of the model id.

- **Code path affects model resolution:** `resolveModelRefFromString` strips any trailing auth profile via `splitTrailingAuthProfile(params.raw)` before parsing the model ref, so a bad split propagates into provider/model selection.
