## Summary

Keep open. Current `main` still has the dashboard i18n problem from this issue: the document root is hard-coded to English and the Sessions page still renders many user-facing strings in raw English instead of going through `t(...)`. There is evidence that the reconnect/history-loss and heartbeat-origin parts have had follow-up work, so this is now a mixed issue, but it is not safe to close as implemented while the UI-language bug remains in current code.

## Evidence

- **Root HTML language is still fixed to English:** `ui/index.html` still ships `<html lang="en">`, so the document language does not follow the selected locale.
  - file: [ui/index.html:2](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/ui/index.html#L2)
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Locale manager does not update document language:** `I18nManager.setLocale()` persists the locale and notifies subscribers, but there is no document-level `lang` update in this runtime path.
  - file: [ui/src/i18n/lib/translate.ts:74](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/ui/src/i18n/lib/translate.ts#L74)
  - command: `rg -n "document\.documentElement\.lang|setAttribute\(\s*['\"]lang" ui/src ui/index.html`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Sessions view still contains hard-coded English UI copy:** The Sessions page title, subtitle, filters, search placeholder, table headers, empty state, pagination text, and buttons are still literal English strings rather than localized lookups, which matches the report that the sidebar is translated but inner pages remain English.
  - file: [ui/src/ui/views/sessions.ts:250](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/ui/src/ui/views/sessions.ts#L250)
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Reconnect/history path has explicit refresh logic now:** On reconnect hello, Control UI refreshes the active tab; for chat that path calls `refreshChat()`, which reloads `chat.history` and sessions. This suggests Bug 1 has seen follow-up work, but it does not resolve the still-open i18n portion of the issue.
  - file: [ui/src/ui/app-gateway.ts:330](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/ui/src/ui/app-gateway.ts#L330)
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)

That makes the reported heartbeat relabeling less likely on current `main`, but does not justify closing the whole issue.
