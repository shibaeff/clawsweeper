## Summary

Keep open. Current `main` does not implement zerobrew. macOS setup still auto-installs Homebrew, skill install metadata only supports `brew|node|go|uv|download`, and gateway skill install selection still prefers Homebrew when available.

## Evidence

- **No zerobrew support in repo:** `rg -n "zerobrew" docs src scripts skills ui VISION.md AGENTS.md` returned no matches, so current `main` has no zerobrew-specific code or docs.
  - command: `rg -n "zerobrew" docs src scripts skills ui VISION.md AGENTS.md`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Setup path still installs Homebrew on macOS:** `scripts/install.sh` still defines `install_homebrew()` and calls it as installer step 1. The script says `# Step 1: Homebrew (macOS only)` and invokes `install_homebrew` before Node/OpenClaw setup.
  - file: [scripts/install.sh:1217](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/scripts/install.sh#L1217)
  - command: `nl -ba scripts/install.sh | sed -n '1190,1245p' and nl -ba scripts/install.sh | sed -n '2315,2335p'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Skill installer schema has no zerobrew kind:** `src/agents/skills/types.ts` limits `SkillInstallSpec.kind` to `"brew" | "node" | "go" | "uv" | "download"`; there is no `zerobrew` option or config knob for it.
  - file: [src/agents/skills/types.ts:3](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/agents/skills/types.ts#L3)

- **Gateway still prefers Homebrew for skill installs:** `src/agents/skills-status.ts` selects installer preference with Homebrew first when `skills.install.preferBrew` is enabled and `brew` exists, then `uv`, then node, then later fallbacks.
  - file: [src/agents/skills-status.ts:62](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/agents/skills-status.ts#L62)

  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)

  - file: [src/agents/skills-install.ts:278](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/agents/skills-install.ts#L278)

## Risks / Open Questions

- The request is partially adjacent to newer non-brew fallback work (`uv`, node-manager installs, apt for Go), but the zerobrew-specific feature itself is still absent.
- If maintainers want to resolve this later, they may prefer a broader package-manager abstraction instead of hard-coding zerobrew into core install flows.
