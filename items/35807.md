## Summary

Keep open. Current `main` still routes Windows `host=node` exec through `cmd.exe /d /s /c`, which matches the issue's suspected root cause. Local gateway exec uses PowerShell now, but the Windows node-host path is still explicitly implemented and tested as `cmd.exe`, so there is not strong evidence that #35807 is fixed on `main`.
