## Summary

Current `main` does not implement the requested phase-aware timeout attribution. It still exposes only flat `agent.wait` snapshots (`status`, `startedAt`, `endedAt`, `error`), and embedded-run timeout handling simply skips auth-profile failure writes for all timeout cases rather than distinguishing provider-started timeouts from queue/preflight/post-turn timeouts.

## Evidence

- **Embedded runner skips all timeout auth-profile failure writes:** `maybeMarkAuthProfileFailure` returns early when `reason === "timeout"`, and `resolveAuthProfileFailureReason` also maps timeout failovers to `null`. That avoids poisoning, but it does not implement the requested provider-started-only timeout accounting.
