## Summary

Current `main` does not already implement this PR, and the proposed blanket `modelApi === "openai-completions"` eligibility rule is broader than OpenClaw’s current cache contract. On `main`, cache-TTL pruning is intentionally provider-gated, while OpenAI-compatible prompt-cache keys are only sent when a model explicitly opts in via `compat.supportsPromptCacheKey`. That makes this PR a live behavior/design change that still needs maintainer review, not a conservative auto-close candidate.

## Risks / Open Questions

- Merging the PR as written would enable cache-TTL pruning for OpenAI-compatible providers that never receive `prompt_cache_key`, so pruning could run without verified provider-side cache semantics.
- The PR examples in the body conflate `api: "openai-completions"` with prompt-cache support; current main treats those as separate capabilities.
- Because this is a pull request, not a stale issue, and the requested behavior is neither implemented nor disproven on `main`, auto-closing would be premature.
