## Summary

Current `main` has Control UI upload support for images only, not general file uploads. The web composer hard-limits selection and paste/drop handling to `image/*`, and the Control UI chat client serializes every attachment as `type: "image"`. The gateway RPC already accepts generic attachment fields (`type`, `mimeType`, `fileName`, `content`), so the requested browser-side file upload support is still incomplete on `main`.
