## Summary

Keep open. Current `main` has new multi-session switching in the chat header, but the Control UI left sidebar still renders only tab navigation and footer utilities, not a live list of active channel sessions. That does not match the issue's requested sidebar behavior strongly enough to close as implemented.

## Evidence

- **Left sidebar still contains only tab groups:** The sidebar body iterates `TAB_GROUPS` and renders only nav sections/items, followed by docs/theme/version utilities. There is no session list or per-channel conversation list in the sidebar.
  - file: [ui/src/ui/app-render.ts:1305](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/ui/src/ui/app-render.ts#L1305)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1280,1455p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Session switching lives in the chat header, not the sidebar:** When the current tab is chat, the page header renders `renderChatSessionSelect(state)`; non-chat tabs render the normal page title. This is a header control, not a left-sidebar conversation list.
  - file: [ui/src/ui/app-render.ts:1416](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/ui/src/ui/app-render.ts#L1416)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1280,1455p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Current UI uses a single grouped select for sessions:** `renderChatSessionSelect` renders one `<select>` populated from `resolveSessionOptionGroups(...)`, so users can switch sessions from a dropdown. That is materially different from showing all active sessions in the sidebar at once.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/ui/src/ui/chat/session-controls.ts#L23)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '1,120p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Grouped session options are built for a selector, not a sidebar list:** `resolveSessionOptionGroups` builds grouped option arrays with labels/titles for select options; it does not produce sidebar entries or any always-visible multi-session navigation surface.
  - file: [ui/src/ui/chat/session-controls.ts:447](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/ui/src/ui/chat/session-controls.ts#L447)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '440,585p'`

## Risks / Open Questions

- There is partial progress on `main`: users can now switch among multiple sessions via the chat-header dropdown, so maintainers might consider the issue partially addressed even though the requested left-sidebar UX is still absent.
- Closing as implemented would likely discard a still-valid discoverability/navigation request: active channel sessions are not visible simultaneously in the Control UI sidebar.
