## Summary

Current `main` already addresses the reported Control UI context-notice failure: the notice now uses fresh `totalTokens` data, suppresses itself when totals are stale after reload, and is styled/rendered as an inline pill in the chat flow rather than a fullscreen overlay.

## Evidence

- **UI now ignores stale refresh values and uses total tokens:** `getContextNoticeViewModel` returns `null` when `session.totalTokensFresh === false`, and computes the displayed detail from `session.totalTokens` plus `contextTokens`, not from `inputTokens`. That directly addresses the reported impossible `550.4k / 204.8k` refresh state.

## Close Comment

Closing as implemented on `main` after Codex review. The current Control UI code now suppresses the notice when token totals are stale on refresh (`ui/src/ui/chat/context-notice.ts:53`), uses `totalTokens` rather than the old inflated value source for the displayed numbers (`ui/src/ui/chat/context-notice.ts:56-80`), and renders the notice as an inline pill in the chat flow instead of a fullscreen overlay (`ui/src/ui/views/chat.ts:1192`, `ui/src/styles/chat/layout.css:150-166`). There is also a regression test covering the exact bad-value case and the stale-refresh hide behavior (`ui/src/ui/chat/context-notice.test.ts:24-99`). The changelog places the shipped fix in `v2026.3.22` (`CHANGELOG.md:2200`, `CHANGELOG.md:2247`). If someone can still reproduce this on a current build, please open a new issue with current version/build details and fresh repro steps.
