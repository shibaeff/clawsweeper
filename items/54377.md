## Summary

Keep open. Current `main` does not implement `npm:<package>` support for `openclaw plugins install`, and the docs/tests still describe the existing ClawHub-first then npm-fallback behavior. The PR also has a substantive review concern about explicit `npm:` specs being checked after local-path resolution, so this is not a high-confidence close candidate.
