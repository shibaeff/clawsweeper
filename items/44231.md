## Close Comment

Thanks for the contribution. This Codex review recommends closing this PR because it fits ClawHub, not the core OpenClaw repo.

The current maintainer guidance is explicit: `CONTRIBUTING.md:177` sends skill contributions to ClawHub, and `VISION.md:68-70` plus `VISION.md:101` says new skills should be published to ClawHub first and that new core skills that can live on ClawHub are not something we plan to merge by default. The current docs also treat ClawHub as the normal distribution path for skills (`docs/cli/skills.md:12-45`, `docs/tools/clawhub.md:10-18`, `docs/tools/clawhub.md:61-68`).

This PR is a standalone addition under `skills/aliyun-ecs-skill/`, and the PR description says it is purely additive with no core OpenClaw functionality changes. That is exactly the shape we want in ClawHub rather than in `openclaw/openclaw`.

If you want to ship this, the right next step is to publish it on ClawHub. If there is a strong product or security reason that this must be bundled in core, open a new discussion/PR with that exception case spelled out.
