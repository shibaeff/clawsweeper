## Summary

Keep open. Current `main` still does not let `before_prompt_build` or legacy `before_agent_start` abort an embedded LLM turn and return a synthetic reply. The hook contract only supports prompt-mutation fields, the embedded runner merges only those fields, and ordinary embedded user turns still proceed to `activeSession.prompt(...)`. There is a newer `before_agent_reply` synthetic-reply seam, but in the embedded runner it is only wired for `trigger === "cron"`, not normal user turns.

## Risks / Open Questions

- The existence of `before_agent_reply` may make the issue title/path slightly stale; the remaining gap is embedded-runner parity for ordinary user turns, not total absence of any synthetic-reply hook.
- `docs/concepts/agent-loop.md` describes `before_agent_reply` broadly, while embedded runner code currently limits that seam to cron, so maintainers may want to clarify docs even if this issue stays open.
