## Summary

Keep open. Current `main` does not implement the requested ACP auth-contract hardening: ACP spawn accepts no auth-policy/env-scrub/preflight fields, plugin hook surfaces cover subagents but not ACP spawn, and the bundled `acpx` plugin only exposes manual command overrides rather than generic auth-contract controls. This reads as a still-open design/product gap, not an implemented fix, stale report, or ClawHub-only item.

## Evidence

- **ACP spawn API lacks auth-contract fields:** `SpawnAcpParams` only exposes `task`, `label`, `agentId`, `resumeSessionId`, `model`, `cwd`, `mode`, `thread`, `sandbox`, and `streamTo`; there is no field for expected auth mode, fallback policy, env scrubbing, or spawn preflight.
  - file: [src/agents/acp-spawn.ts:95](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/acp-spawn.ts#L95)
  - command: `nl -ba src/agents/acp-spawn.ts | sed -n '88,108p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Plugin hook registry has subagent hooks, not ACP spawn hooks:** Current hook names include `subagent_spawning`, `subagent_delivery_target`, `subagent_spawned`, and `subagent_ended`, but no ACP-specific spawn/preflight hook family. That means there is no generic plugin hook surface today for ACP auth-contract enforcement before launch.
  - file: [src/plugins/hook-types.ts:72](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/plugins/hook-types.ts#L72)

  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Bundled acpx config only offers command overrides, not auth-policy metadata:** The bundled `acpx` plugin schema exposes fields such as `cwd`, `stateDir`, `probeAgent`, permission settings, MCP servers, and `agents.<id>.command`. There is no first-class config for per-agent auth contracts, ambient env stripping, or fail-closed auth preflight. The resolver also only normalizes `agents` into command strings.
