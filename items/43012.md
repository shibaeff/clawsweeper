## Summary

Current `main` no longer matches this report: existing-session/browser-relay snapshots now default to `format=ai`, AI snapshots register action refs for later `click`/`fill`, and `axN` refs from `format=aria` are rejected immediately with a clear re-snapshot message instead of hanging on `locator('aria-ref=...')`.

## Evidence

- **Existing-session snapshots default to AI refs:** `resolveDefaultSnapshotFormat()` returns `"ai"` for `local-existing-session` profiles, which covers the Chrome Extension relay/browser MCP path described in the issue.
  - file: [extensions/browser/src/browser/profile-capabilities.ts:71](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/browser/src/browser/profile-capabilities.ts#L71)

- **AI snapshots register refs for later act commands:** `snapshotAiViaPlaywright()` builds refs from the AI snapshot and stores them with `mode: "aria"`, so subsequent interactions resolve against the right Playwright `aria-ref` mapping.
  - file: [extensions/browser/src/browser/pw-tools-core.snapshot.ts:104](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/browser/src/browser/pw-tools-core.snapshot.ts#L104)
