## Summary

Current `main` still falls through to the next fallback model after a surfaced timeout/overload/rate-limit failure; it does not implement configurable same-model retry with backoff before cross-model fallback. The only same-model retry I found is a narrow idle-timeout path that is disabled when fallbacks are configured.

## Evidence

- **Timeout still proves immediate cross-model fallback:** The embedded fallback e2e test explicitly expects a timeout on the primary model to switch directly to the fallback model (`groq/mock-2`) with the first recorded attempt reason `timeout`.
