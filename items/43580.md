## Summary

Keep open. Current `main` does not implement Deepgram as a TTS/speech provider, and the PR’s approach is stale against the newer generic speech-provider architecture rather than clearly superseded by an equivalent implementation on `main`. None of the allowed close reasons fit.

## Evidence

- **Deepgram plugin on main is not a speech provider:** The bundled Deepgram plugin currently registers only media-understanding and realtime transcription capabilities. It does not call `registerSpeechProvider(...)`, so Deepgram Aura TTS is not implemented on `main`.
