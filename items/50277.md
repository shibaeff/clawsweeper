## Summary

Keep open. Current `main` still renders the chat delete confirmation popover above the trigger with no viewport-aware repositioning, so the reported overflow near the top/edge of the chat view remains plausible. The linked fix attempts were closed without landing, and the specific viewport-handling logic they proposed is not present on `main`.

- **Current CSS still places the popover above the delete button:** `ui/src/styles/chat/grouped.css:400` keeps `.chat-delete-confirm` at `position: absolute` with `bottom: calc(100% + 6px)`. There is no `top: calc(100% + 6px)` fallback and no viewport-clamping rule in this block.
