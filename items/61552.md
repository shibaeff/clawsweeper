## Summary

Current main already covers the requested outcome: Feishu direct messages can share the main session via `session.dmScope: "main"`, and Feishu conversations can be pinned to a chosen agent via top-level `bindings`, so a separate `channels.feishu.sessionBinding` feature is redundant for this use case.

## Evidence

- **Session docs: DMs share the main session by default:** The session docs state that direct messages share one session by default, list `main` as the default `dmScope`, and recommend `session.identityLinks` when the same person should share one session across channels.
