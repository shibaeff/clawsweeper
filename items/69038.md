## Summary

Current `main` still has the exact split-state setup this PR is trying to fix: Mattermost slash-command route registration loads `slash-state` through the synchronous bundled-entry loader, while the runtime monitor path reaches Mattermost monitor code through a native dynamic import path. `extensions/mattermost/src/mattermost/slash-state.ts` still stores state in a plain module-local `Map`, so there is no strong evidence that the dual-instance 503 bug is already fixed or obsolete on `main`.

## Evidence
