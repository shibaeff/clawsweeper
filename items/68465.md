## Summary

Keep open. Current `main` still lacks WhatsApp `silentErrorReplies` support, while Telegram already has the parity behavior this PR is trying to add. The PR discussion also shows the earlier generated-metadata gap was addressed on the PR branch, so there is still a concrete, unresolved change here rather than an obsolete report.
