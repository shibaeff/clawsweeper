## Summary

Keep open. Current `main` still resolves `pnpm` to an absolute path and passes that path unchanged into `spawn`/`spawnSync` in `scripts/ui.js`, so this PR is not implemented on `main`, and I did not find a replacement fix in the checked-out code.

## Evidence

- **Current runner path is passed through unquoted:** `resolveRunner()` returns the resolved `pnpm` path as `runner.cmd`, and both `run()` and `runSync()` pass `cmd` directly to `spawn` / `spawnSync` while `createSpawnOptions()` may set `shell: true` for `.cmd` files. There is no quoting step on current `main`.
