## Summary

Keep open. Current `main` still contains the malformed multi-line placeholder blocks in `src/auto-reply/reply/export-html/template.html`, while the export path still performs literal `.replace("{{...}}", ...)` substitutions. That means the reported export failure remains consistent with current code, and the same broken template is also present in release `v2026.4.22`.

## Evidence

- **Malformed placeholders still on main:** The HTML template currently renders the vendored script placeholders as multi-line brace blocks (`MARKED_JS`, `HIGHLIGHT_JS`, `JS`) instead of literal `{{MARKED_JS}}`-style tokens, so string replacement cannot match them.
  - file: [src/auto-reply/reply/export-html/template.html:62](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/auto-reply/reply/export-html/template.html#L62)
  - command: `nl -ba src/auto-reply/reply/export-html/template.html | sed -n '55,85p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Export code still expects exact literal tokens:** `generateHtml()` still does exact string replacements for `{{JS}}`, `{{SESSION_DATA}}`, `{{MARKED_JS}}`, and `{{HIGHLIGHT_JS}}`; it does not use a whitespace-tolerant or regex replacement path.
  - file: [src/auto-reply/reply/commands-export-session.ts:91](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/auto-reply/reply/commands-export-session.ts#L91)
