## Summary

The issue is asking OpenClaw maintainers to try and file issues against an external VSCode plugin (`openclaw luna`), not requesting a concrete OpenClaw core change. Current main's scope docs route optional integrations and community-maintained plugins to external repos/ClawHub rather than `openclaw/openclaw` core.

## Close Comment

Codex review: this request is about a separate community VSCode plugin (`openclaw luna`) and asks maintainers to try it and file issues for that plugin, rather than identifying a concrete OpenClaw core change. Current scope docs keep optional integrations in community plugins hosted and maintained outside core (`VISION.md:54`, `VISION.md:87`, `docs/plugins/community.md:9`, `docs/plugins/community.md:129`). OpenClaw already documents ClawHub/npm as the right discovery path for third-party plugins and expects those plugins to have their own public repo and issue tracker. Because of that, this belongs in the plugin's own repo/ClawHub listing rather than `openclaw/openclaw`. If there is a specific missing OpenClaw API or plugin contract that blocks the VSCode integration, please open a new focused issue describing that gap.
