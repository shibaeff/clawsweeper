## Summary

Current `main` does not implement Gemini Flex tier support. OpenClaw's Google transport never adds `service_tier`/`serviceTier` to Gemini requests, and the model-param plumbing for Google only forwards temperature, maxTokens, transport, cachedContent, and thinking-related behavior.

## Evidence

- **Google Flex API requires an explicit tier field:** Google's Flex inference docs say Gemini Flex is enabled by sending `service_tier: "flex"` in REST or `config.serviceTier = "flex"` in the JavaScript SDK.
  - command: `open https://ai.google.dev/gemini-api/docs/flex-inference`
- **Google request builder omits service tier:** `buildGoogleGenerativeAiParams` only sets `temperature`, `maxOutputTokens`, `thinkingConfig`, `cachedContent`, `systemInstruction`, `tools`, and `toolConfig`. There is no `service_tier` or `serviceTier` field in the emitted Gemini request payload.
  - file: [extensions/google/transport-stream.ts:458](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/google/transport-stream.ts#L458)
  - command: `nl -ba extensions/google/transport-stream.ts | sed -n '455,505p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Generic model params do not forward Gemini service tier:** The extra-param stream wrapper only maps `temperature`, `maxTokens`, `transport`, `openaiWsWarmup`, and `cachedContent` into stream options. Nothing in this path forwards `serviceTier`/`service_tier` for Google models.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:286](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/pi-embedded-runner/extra-params.ts#L286)
  - command: `nl -ba src/agents/pi-embedded-runner/extra-params.ts | sed -n '286,345p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Google provider hooks only add thinking behavior:** The bundled Google provider hook surface currently applies replay-family hooks, resolves thinking profiles, and wraps the stream with `createGoogleThinkingStreamWrapper`. There is no Google-specific service-tier wrapper analogous to the OpenAI/Anthropic implementations.
  - file: [extensions/google/provider-hooks.ts:8](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/google/provider-hooks.ts#L8)
  - command: `nl -ba extensions/google/provider-hooks.ts | sed -n '1,80p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **User-facing docs do not expose Gemini service-tier config:** The Google provider docs describe API-key/OAuth setup and capabilities, but do not document any `params.serviceTier` or Flex-tier option. In the model-provider overview, `serviceTier` is documented for OpenAI, Anthropic, and Codex only.
  - file: [docs/concepts/model-providers.md:85](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/docs/concepts/model-providers.md#L85)
  - command: `nl -ba docs/concepts/model-providers.md | sed -n '80,140p'; nl -ba docs/providers/google.md | sed -n '1,120p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
