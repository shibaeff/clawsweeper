## Summary

Keep open. Current `main` still falls back from failed npm plugin installs into hook-pack probing, and the cited `@martian-engineering/lossless-claw@0.9.2` package currently publishes normal plugin metadata (`openclaw.extensions` plus `openclaw.plugin.json`). The only clear mitigation on `main` is a narrow message fix for the `plugin already exists` case, so this report is not clearly implemented or obsolete.

## Evidence

- **Npm plugin failures still trigger hook-pack fallback:** `src/cli/plugins-install-command.ts:571` calls `installPluginFromNpmSpec(...)`; on failure, `src/cli/plugins-install-command.ts:588` immediately tries `tryInstallHookPackFromNpmSpec(...)` unless the failure maps to a bundled-plugin recovery path.
  - file: [src/cli/plugins-install-command.ts:571](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cli/plugins-install-command.ts#L571)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Default combined error still appends hook-pack text:** `src/cli/plugins-command-helpers.ts:102` formats general failures as `<plugin error>\nAlso not a valid hook pack: <hook error>`. The only exception is the regex special-case for `plugin already exists` at line 106.
  - file: [src/cli/plugins-command-helpers.ts:102](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cli/plugins-command-helpers.ts#L102)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Regression coverage is narrow:** `src/cli/plugins-cli.install.test.ts:681` covers the already-installed npm plugin case and asserts the hook-pack suffix is omitted there, but there is no nearby test proving valid npm plugins bypass hook-pack fallback after other plugin-side failures.
  - file: [src/cli/plugins-cli.install.test.ts:681](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cli/plugins-cli.install.test.ts#L681)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Published package metadata matches a native plugin:** Read-only npm/tarball inspection showed `@martian-engineering/lossless-claw@0.9.2` publishes `openclaw.extensions: ["./dist/index.js"]` in `package.json` and includes `openclaw.plugin.json` in the package tarball, matching the issue's claim that this is a plugin package rather than a hook pack.
  - command: `npm view @martian-engineering/lossless-claw@0.9.2 name version openclaw repository dist.tarball --json; curl -L <dist.tarball> | tar -tzf -; curl -L <dist.tarball> | tar -xOzf - package/package.json`
- **Docs still advertise this exact package as a plugin:** `docs/plugins/community.md:64` lists `@martian-engineering/lossless-claw` as an installable OpenClaw plugin, while `docs/cli/plugins.md:104` documents that `plugins install` is also the install surface for hook packs, which is consistent with the confusing fallback path described in the issue.
  - file: [docs/plugins/community.md:64](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/docs/plugins/community.md#L64)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
