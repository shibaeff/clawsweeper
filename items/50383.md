## Close Comment

Codex review: this is already implemented on current `main`. The WhatsApp outbound path now resolves listeners through a shared registry, and that registry is stored on `globalThis` with `Symbol.for(...)`, so duplicated bundled module instances share the same controller map instead of losing the active listener (`extensions/whatsapp/src/connection-controller-registry.ts:11`, `extensions/whatsapp/src/send.ts:39`, `extensions/whatsapp/src/connection-controller.ts:369`). There is also a regression test that imports duplicate copies of the registry module and verifies the registered controller is visible across both instances (`extensions/whatsapp/src/connection-controller-registry.test.ts:11`). The changelog records this exact fix in release `2026.3.22` (`CHANGELOG.md:2350`). Closing as implemented on main. If this still reproduces on a build newer than `2026.3.22`, please open a fresh issue with the exact version and current gateway logs.
