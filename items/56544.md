## Summary

Current `main` still disables Slack preview streaming for top-level DMs without a thread, the latest release `v2026.4.23` still has that guard, and the surrounding draft-stream code supports the PR's claim that a pre-existing `threadTs` is not required. This PR remains actionable cleanup rather than a close candidate.

## Evidence

That matches the PR's reported bug for flat DMs with `replyToMode: off`.
