## Summary

Keep open. Current `main` still lacks a Moonshot/Kimi replay patch that restores or preserves `reasoning_content` for assistant tool-call turns, while analogous provider-specific handling exists elsewhere. The relevant Moonshot replay/wrapper files are unchanged from `v2026.4.23`, so there is no evidence the reported post-compaction regression is already fixed on `main`.

## Evidence

- **Moonshot still uses only the generic OpenAI-compatible replay policy:** `extensions/moonshot/index.ts` wires `buildProviderReplayFamilyHooks({ family: "openai-compatible", sanitizeToolCallIds: false })` and adds `MOONSHOT_THINKING_STREAM_HOOKS`, but there is no Moonshot-specific replay sanitizer or payload repair for `reasoning_content`.
  - file: [extensions/moonshot/index.ts:63](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/extensions/moonshot/index.ts#L63)
  - command: `nl -ba extensions/moonshot/index.ts | sed -n '50,80p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Moonshot thinking wrapper does not touch replayed messages:** `src/agents/pi-embedded-runner/moonshot-thinking-stream-wrappers.ts` only adjusts `thinking`, `tool_choice`, and `thinking.keep`. It never iterates `payloadObj.messages` or injects `reasoning_content` for assistant tool-call messages, which is the missing behavior described in the issue.
  - file: [src/agents/pi-embedded-runner/moonshot-thinking-stream-wrappers.ts:97](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-runner/moonshot-thinking-stream-wrappers.ts#L97)

- **OpenClaw already has an explicit pattern for this class of provider requirement:** DeepSeek's provider wrapper explicitly adds blank `reasoning_content` to assistant messages with `tool_calls` and has regression tests/docs for that replay contract. That analogous fix exists in `extensions/deepseek/stream.ts` and `extensions/deepseek/index.test.ts`, but Moonshot has no corresponding implementation.
  - file: [extensions/deepseek/stream.ts:31](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/extensions/deepseek/stream.ts#L31)
