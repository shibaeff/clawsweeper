## Summary

Current `main` addresses the reported prompt-cache churn: inbound system metadata is kept session-stable, volatile heartbeat/channel context is moved below an internal cache boundary, and Anthropic-family payload shaping caches only the stable prefix.

## Close Comment

Closing after a Codex review: current `main` already implements this well enough. Inbound system metadata is kept session-stable and excludes per-turn ids that would churn the cache (`src/auto-reply/reply/inbound-meta.ts:142`), with regression coverage that changing chat ids does not change the system prompt (`src/auto-reply/reply/inbound-meta.test.ts:92`). Volatile heartbeat/channel context is now placed below the internal cache boundary (`src/agents/system-prompt.ts:942`), and Anthropic-family payload shaping only applies `cache_control` to the stable prefix (`src/agents/anthropic-payload-policy.ts:89`). Prompt-composition coverage also checks stable post-warmup system prompts (`src/agents/prompt-composition.test.ts:26`). This shipped in `v2026.4.5` (`CHANGELOG.md:1391`). If you still see a fresh cache-miss pattern on current `main`, please open a new issue with an updated prompt trace and provider/model path.
