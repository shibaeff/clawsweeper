## Summary

Keep open. Current `main` still lacks the PR's poll-aware notification suppression, so the reported ghost-notification race remains plausible. The PR also still needs maintainer judgment because its boolean `pollActive` approach conflicts with concurrent poll behavior already exercised in `main` tests.

## Evidence

- **`maybeNotifyOnExit()` on `main` has no poll-state guard:** The exit notification path still suppresses only on `backgrounded`, `notifyOnExit`, and `exitNotified`; there is no `pollActive` or equivalent check, so an active `process(poll)` is invisible to the notifier.
