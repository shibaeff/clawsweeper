## Summary

Keep open. Current `main` does not expose Telegram topic routing/binding state in `/status`. The status path was refactored, but it still only renders the generic session line plus optional plugin debug lines, and those plugin debug lines are verbose-only and are not populated by the Telegram plugin. The underlying Telegram routing/binding state still exists in current code, so the request remains plausible, but this PR is stale against the newer status architecture rather than already implemented.
