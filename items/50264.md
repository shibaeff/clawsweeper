## Summary

Close. Current `main` already supports markdown-table insertion for `feishu_doc` write/append by converting markdown and inserting the resulting table/table-cell blocks through Feishu's Descendant API; the issue appears to reflect stale Feishu skill docs, not the shipped runtime behavior.

## Evidence

- **`write` routes markdown output through Descendant insertion:** `writeDoc()` converts markdown, normalizes the block tree, and inserts via `insertBlocksWithDescendant(...)` for normal-sized writes instead of the older children-create path that could not create tables.
  - file: [extensions/feishu/src/docx.ts:976](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/feishu/src/docx.ts#L976)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Descendant path explicitly supports table blocks:** `insertBlocksWithDescendant()` is documented as supporting `Table`/`TableCell` blocks and calls `client.docx.documentBlockDescendant.create(...)` with the converted descendants.
  - file: [extensions/feishu/src/docx.ts:440](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/feishu/src/docx.ts#L440)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Table cleanup exists specifically for Descendant writes:** `cleanBlocksForDescendant()` removes read-only table fields, normalizes table-cell children, and preserves table metadata so converted markdown tables can be inserted successfully.
  - file: [extensions/feishu/src/docx-table-ops.ts:184](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/feishu/src/docx-table-ops.ts#L184)

## Close Comment

Closing as implemented on `main` after a Codex review. The current `feishu_doc` write/append path already converts markdown and inserts the result through `documentBlockDescendant.create`, and that code explicitly supports table/table-cell blocks (`extensions/feishu/src/docx.ts:440`, `extensions/feishu/src/docx.ts:976`, `extensions/feishu/src/docx-table-ops.ts:184`). The same implementation is present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The confusing part is that the Feishu skill docs are stale and still say markdown tables are unsupported (`extensions/feishu/skills/feishu-doc/SKILL.md:31`, `extensions/feishu/skills/feishu-doc/references/block-types.md:78`). If you can still reproduce a 400 on `v2026.4.22` or newer, please open a fresh issue with the exact markdown sample and the full Feishu response body so maintainers can isolate the remaining case.
