## Summary

Current `main` already addresses the reported Control UI issue cluster: the context notice is now a small inline pill with a fixed-size icon above the chat input, and both the chat notice and Sessions view use `totalTokens/contextTokens` rather than the mismatched `inputTokens` path described in the report.

## Close Comment

Closing after a Codex review because current `main` already contains the fix path for this report. The chat warning is now rendered as a small inline `context-notice` pill with a fixed `16x16` icon in `ui/src/ui/chat/context-notice.ts:85` and `ui/src/styles/chat/layout.css:150`, and the input bar remains a separate element immediately after it in `ui/src/ui/views/chat.ts:1192`. This implementation is already present in the shipped `v2026.4.22` release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. If you still hit a broken chat pane on a newer build, please open a fresh issue with the exact OpenClaw version, viewport size, and a screenshot, because that would be a different current regression.
