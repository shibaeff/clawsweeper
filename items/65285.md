## Summary

Keep open. Current `main` still routes unknown unhandled promise rejections to `process.exit(1)`, and there is no shipped classifier or other clear guard for the reported `"Agent listener invoked outside active run"` race. The only linked fix in the timeline is PR #65323, and it is still open in the provided issue context.

## Evidence

- **Unhandled rejections still crash by default:** `installUnhandledRejectionHandler()` only suppresses AbortError plus transient network/SQLite cases. Anything else logs `Unhandled promise rejection` and exits via `exitWithTerminalRestore(...)`, which matches the reporter's crash path rather than a fixed behavior.
