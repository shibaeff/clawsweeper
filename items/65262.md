## Summary

Keep open. Current `main` still treats `openclaw commands` as a plugin/allowlist problem instead of a built-in runtime slash command, while docs and the command registry still define `/commands` as a built-in chat command. The PR is addressing a real gap, but it is not already implemented on `main`, and the posted review history still leaves at least one maintainability concern unresolved.
