## Close Comment

Closing after Codex review. On current `main`, the reported bad outcome does not reproduce anymore: `src/auto-reply/media-note.ts:29-31` still renders the duplicate local-path note text, but the current image-ref parser only extracts the attachment path from `[media attached: ...]` brackets and dedupes refs (`src/agents/pi-embedded-runner/run/images.ts:229-293`). Since current `main` already avoids the double-injection behavior this PR was trying to fix, I’m closing this PR as no longer needed.
