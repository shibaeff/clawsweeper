## Summary

Keep open. Current `main` has Feishu inbound broadcast-group fan-out, but it does not implement the reported feature: rebroadcasting bot-sent Feishu messages as synthetic inbound events to other bots for cross-bot conversation parity.

## Evidence

- **Feishu only handles inbound webhook delivery:** The Feishu monitor registers `im.message.receive_v1` as the message ingress path. I did not find a send-time hook that re-injects bot-authored outbound messages back into other Feishu bot handlers.
  - file: [extensions/feishu/src/monitor.account.ts:328](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/feishu/src/monitor.account.ts#L328)
  - command: `nl -ba extensions/feishu/src/monitor.account.ts | sed -n '320,345p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Outbound Feishu replies do not synthesize peer bot events:** `sendMessageFeishu` sends directly through the Feishu API and returns. The reply dispatcher calls that send path, but there is no follow-up synthetic event injection for other bot accounts.
  - file: [extensions/feishu/src/send.ts:454](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/feishu/src/send.ts#L454)
  - command: `nl -ba extensions/feishu/src/send.ts | sed -n '454,560p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Existing Feishu `broadcast` feature is inbound agent fan-out, not bot-to-bot message visibility:** When a group message is eligible, `handleFeishuMessage` fans out the same inbound event to configured agents. Only the active agent gets a real Feishu dispatcher; observer agents use a no-op dispatcher and therefore do not send or receive each other's channel messages.
  - file: [extensions/feishu/src/bot.ts:1038](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/feishu/src/bot.ts#L1038)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '1038,1180p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Tests cover broadcast fan-out and cross-account dedupe only:** Current tests assert multi-agent dispatch from one inbound Feishu event and one active reply path across accounts. They do not cover rebroadcast of a bot-authored outbound message into other bot accounts.
  - file: [extensions/feishu/src/bot.broadcast.test.ts:189](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/feishu/src/bot.broadcast.test.ts#L189)
  - command: `nl -ba extensions/feishu/src/bot.broadcast.test.ts | sed -n '189,372p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Docs explicitly describe isolated histories for broadcast groups:** The broadcast-groups doc says broadcast only changes which agents run when a message is already eligible, and each agent keeps separate conversation history that does not include other agents' messages. That is the opposite of the requested cross-bot message visibility.
  - file: [docs/channels/broadcast-groups.md:164](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/channels/broadcast-groups.md#L164)
