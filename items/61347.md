## Summary

Keep open. Current `main` added partial mitigations for workspace bootstrap token cost, but it does not implement the issue's core request for general per-file selective/lazy injection. The available controls are a global `contextInjection` mode (`always` or `continuation-skip`), heartbeat-specific `lightContext`, and per-file/total character caps with truncation. There is no generic file-level policy/frontmatter, no TOC/on-demand read mode, and no broader selective injection for arbitrary workspace files.
