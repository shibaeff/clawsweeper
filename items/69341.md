## Summary

Keep open. Current `main` still matches the report: `channels status` does not register a `--deep` flag, but both the subcommand description and the runtime tip still mention `status --deep`. The real `--deep` flag exists on top-level `openclaw status`, so the ambiguity remains unresolved in CLI help/output.
