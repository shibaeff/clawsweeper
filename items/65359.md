## Summary

Current `main` still rejects `messages.groupChat.historyLimit: 0` in the runtime Zod schema, and the generated base config schema still publishes the same `exclusiveMinimum: 0` constraint. This PR appears to fix a real, unfixed inconsistency rather than duplicate shipped behavior, so it should stay open.

## Evidence
