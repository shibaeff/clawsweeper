## Summary

Current `main` now has an explicit WhatsApp relink flow, clears cached auth on logout and on logged-out QR/auth failures, and covers the exact stale-session relink case with a regression test. The fixing work is present on `main` but I could not prove it shipped in a tagged release yet.

## Evidence

- **Control UI now distinguishes relink from refresh:** The WhatsApp card has a dedicated `Relink` button wired to `onWhatsAppStart(true)`, while `Refresh` is a separate status reload action. That matches the reported relogin use case and removes ambiguity in the current UI.

  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)

  - file: [src/gateway/server-methods/web.ts:91](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/gateway/server-methods/web.ts#L91)
