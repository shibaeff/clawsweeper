## Summary

Keep open. Current `main` still treats `metadata.openclaw.os` as raw platform strings and compares them directly against `process.platform`, so a skill that declares `os: macos` will remain ineligible on a macOS host where Node reports `darwin`. The repo docs for skills currently document `darwin|linux|win32`, so this is not implemented on `main`; at minimum there is still a real docs/UX mismatch because other OpenClaw surfaces use `macos` as the platform name.

## Evidence

  - file: [src/shared/frontmatter.ts:86](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/shared/frontmatter.ts#L86)
