## Summary

Keep open. Current `main` still creates independent OpenTelemetry spans per diagnostic event without passing parent context, and the diagnostic event contract does not carry trace/span linkage needed to reconstruct parent-child relationships.

## Evidence

- **Spans are started without parent context:** `spanWithDuration()` calls `tracer.startSpan(name, { attributes, ...(startTime ? { startTime } : {}) })` with only the options object and no parent/context argument.
  - file: [extensions/diagnostics-otel/src/service.ts:368](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/extensions/diagnostics-otel/src/service.ts#L368)
  - command: `nl -ba extensions/diagnostics-otel/src/service.ts | sed -n '361,372p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Diagnostic handlers emit standalone spans and end them immediately:** `recordModelUsage`, `recordWebhookProcessed`, and `recordMessageProcessed` each create a span and immediately `end()` it; there is no stored active span or child linkage between related events.
  - file: [extensions/diagnostics-otel/src/service.ts:435](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/extensions/diagnostics-otel/src/service.ts#L435)

  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Diagnostic event schema lacks trace/parent identifiers:** `DiagnosticEventPayload` variants include timestamps, sequence numbers, session ids, and event-specific fields, but no trace id, span id, parent span id, or OpenTelemetry context carrier.
  - file: [src/infra/diagnostic-events.ts:5](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/infra/diagnostic-events.ts#L5)
