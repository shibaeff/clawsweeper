## Summary

Keep open. Current `main` still auto-detects Gemini CLI for audio, the public docs still document that behavior, and there is no equivalent regression coverage on `main` proving audio skips Gemini in headless flows. The PR remains relevant, and the linked follow-up issue for proper Gemini multimodal audio support makes this a targeted behavior fix rather than something to close as obsolete.
