## Summary

Keep open: I did not find a landed fix on current `main`. Bundle MCP/browser tool schemas are still forwarded into the agent tool layer unchanged, the runtime normalization path does not strip `$schema` or add draft-2020-12 handling, and I found no focused regression coverage for `$schema`-annotated 2020-12 tool schemas.

## Evidence

- **Bundle MCP tool schemas are passed through unchanged:** `materializeBundleMcpToolsForRun()` still assigns `tool.inputSchema` directly to the runtime tool `parameters`. If a browser MCP tool advertises `$schema: https://json-schema.org/draft/2020-12/schema`, current main still forwards that schema as-is.
  - file: [src/agents/pi-bundle-mcp-materialize.ts:105](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/pi-bundle-mcp-materialize.ts#L105)
  - command: `nl -ba src/agents/pi-bundle-mcp-materialize.ts | sed -n '90,110p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **All tools still go through the same normalization path before runtime:** The agent tool assembly path maps every tool through `normalizeToolParameters(...)` before handing tools to `pi-agent/pi-ai`, so the relevant question is whether that normalizer now handles draft-2020-12 markers. It does not.
  - file: [src/agents/pi-tools.ts:697](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/pi-tools.ts#L697)

- **Schema normalization still does not handle draft-2020-12 markers:** `normalizeToolParameterSchema()` only normalizes object shape, unions, and provider-specific keyword cleanup. There is no `$schema` stripping, no `Ajv2020` path, and no draft-2020-12 meta-schema registration in this browser/MCP tool schema path.
  - file: [src/agents/pi-tools-parameter-schema.ts:145](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/pi-tools-parameter-schema.ts#L145)
