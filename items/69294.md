## Summary

Keep open. Current `main` still launches the ACPX background probe as a detached async task without the outer `.catch()` that the linked fix proposal used for this Windows crash path, and OpenClaw's global unhandled-rejection handler still exits the gateway process. The linked PR #69459 was closed for PR-queue policy, not because the bug was disproved or fixed on `main`.

## Evidence

- **Current main still has detached ACPX probe without outer catch:** `createAcpxRuntimeService.start()` starts the probe with `void (async () => { ... })();` and logs inside the inner try/catch, but there is no final `.catch(...)` on the detached promise in current `main`. That means any rejection escaping the body still reaches the process-level unhandled-rejection handler.
  - file: [extensions/acpx/src/service.ts:183](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/acpx/src/service.ts#L183)
  - command: `nl -ba extensions/acpx/src/service.ts | sed -n '131,214p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Unhandled rejections still terminate the gateway:** The repo-wide unhandled-rejection handler calls `process.exit(1)` for unhandled promise rejections that are not explicitly suppressed. If the detached ACPX probe leaks a rejection, gateway exit is expected from current `main` behavior.
  - file: [src/infra/unhandled-rejections.ts:340](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/infra/unhandled-rejections.ts#L340)
  - command: `nl -ba src/infra/unhandled-rejections.ts | sed -n '340,382p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Linked PR carried an issue-specific fix that is not present on main:** PR #69459's final commit `3034c161a2418f69f27bd8188747efd9a29c220c` is titled `fix(acpx): absorb unhandled rejection from background probe on Windows` and adds the missing outer `.catch(...)` to `extensions/acpx/src/service.ts`. That exact change is not in the checked-out `main` code.
  - file: [extensions/acpx/src/service.ts](https://github.com/openclaw/openclaw/blob/3034c161a2418f69f27bd8188747efd9a29c220c/extensions/acpx/src/service.ts)
