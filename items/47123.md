## Summary

Current main already avoids the reported silent ~4k final-message truncation in WebChat by collecting all delivered final payloads and rebuilding the final assistant message from the full payload set before the final broadcast/transcript write. That implementation was already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Current main reassembles all final webchat payloads before final broadcast:** `createReplyDispatcher` stores every `block`/`final` payload in `deliveredReplies`, then the non-streaming WebChat path filters `finalPayloads`, derives `displayReply`/`transcriptReply` from the full set, appends that full message to the transcript, and passes it to `broadcastChatFinal`. This means the final rendered message is no longer limited to a single 4000-char chunk.
  - file: [src/gateway/server-methods/chat.ts:2467](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/gateway/server-methods/chat.ts#L2467)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '2462,2693p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Final text is explicitly rebuilt by joining payload chunks:** `buildTranscriptReplyText(payloads)` maps each payload and joins the resulting chunks with `\n\n`, so the final message text is reconstructed from all final payloads rather than one truncated chunk.

## Risks / Open Questions

- WebChat still treats `webchat` as an internal channel rather than a normal configurable outbound channel, so the exact config shape proposed in the issue was not added; the close is based on the truncation behavior being fixed, not on parity of configuration knobs.

## Close Comment

Closing after Codex review. Current `main` already rebuilds the WebChat final assistant message from the full collected payload set before the final broadcast/transcript write, so the final UI message is no longer capped to a single ~4k chunk. The relevant logic is in `src/gateway/server-methods/chat.ts:2467` and `src/gateway/server-methods/chat.ts:2562`, with the text reconstruction helper at `src/gateway/server-methods/chat.ts:234`. This was already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Also, the remaining documented WebChat truncation setting is `gateway.webchat.chatHistoryMaxChars` for `chat.history` responses (`docs/web/webchat.md:64`), which is separate from final reply rendering. If you can still reproduce this on `v2026.4.22` or newer, please open a new issue with a fresh repro and a transcript/sample payload from the current build.
