## Summary

Keep open. Current `main` has partial Telegram reaction support, but it only turns inbound reactions into queued system events. It does not implement the requested first-class wake/execution behavior, ignores Telegram `custom_emoji` reactions, and exposes no per-emoji semantics mapping in config/docs. The linked follow-up bug report about reaction-only inputs not waking the agent is consistent with the current code path.
