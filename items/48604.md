## Summary

Keep open. Current `main` accepts `agents.list[].contextTokens` in config, but the runtime context-window guard still only honors `agents.defaults.contextTokens`, and the compaction/extension paths still compute budgets without any per-agent identity. The PR discussion correctly identified that gap, so this is not implemented on `main` and the PR still represents useful unfinished work.

## Evidence
