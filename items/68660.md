## Summary

Current `main` now uses a shared session-scoped bundle MCP runtime that is reused for the same `sessionId` and explicitly retired on run/session teardown paths. That directly addresses the duplicate/orphaned MCP child-process leak described here. The implementation landed after the latest tagged release, so this appears fixed on `main` but not yet shipped in `v2026.4.23`.

## Evidence
