## Summary

Current `main` already addresses the reported bundled-MCP leak: it now reuses one runtime per session/config, explicitly disposes stale and in-flight runtimes, kills lingering stdio MCP process trees on close, and wires runtime disposal into gateway session reset/delete, reload, shutdown, and run-end cleanup paths. The cleanup is documented in the changelog via commit `443b837bd521b0485c87bb7359f370a4ccec10fa`, but it does not appear in a tagged release yet.
