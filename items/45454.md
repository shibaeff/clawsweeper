## Summary

Keep open. Current `main` still appears to have the reported gap: fast-abort replies are emitted before `onModelSelected` runs, while the reply-prefix context is only populated from that callback and is not pre-seeded from persisted session runtime metadata. The draft PR’s exact patch no longer matches current file layout, but the underlying bug still looks real on `main`.
