## Summary

Keep open. Current `main` does not implement this PR's strict-first `web_fetch` retry design; it still exposes and tests a broader config-based `allowRfc2544BenchmarkRange` path that permits RFC2544 literal-IP access when opted in.

## Evidence

- **`web_fetch` still uses config opt-in, not strict-first retry:** Current `runWebFetch` reads `params.ssrfPolicy?.allowRfc2544BenchmarkRange`, bakes that into the cache key, and passes the policy directly into guarded fetch. There is no narrow retry after an initial `SsrFBlockedError` and no hostname-only env-proxy continuation path in this code.
