## Summary

Current `main` already covers the reported Discord announce-routing failure: phantom Discord DM session keys are normalized away, subagent completion delivery now backfills missing route fields from stored session metadata, and `Unknown Channel` is treated as a permanent announce-delivery error. The fix was also documented in the 2026.3.7 changelog.

## Close Comment

Closing from a Codex review because this is already implemented on current `main` (`76a4c167f7ef65040fd5b00d206c59b8bb28fd49`). The Discord session-key path now rewrites phantom direct-message keys like `discord:channel:<user>` to `discord:direct:*` when the sender matches (`extensions/discord/src/session-key-normalization.ts`, with regression coverage in `src/config/sessions/session-key.test.ts` and `extensions/discord/src/session-key-normalization.test.ts`). Current Discord inbound handling also persists real channel delivery targets via `updateLastRoute` (`extensions/discord/src/monitor/message-handler.process.ts`), and subagent completion delivery now backfills missing `to`/account fields from stored session routing plus treats `Unknown Channel` as a permanent error (`src/agents/subagent-announce-delivery.ts`). This was also called out in the 2026.3.7 changelog (`CHANGELOG.md`).
