## Summary

Current `main` does not implement the requested `cron lint` surface or the proposed multi-account `delivery.accountId` enforcement. The existing validation only rejects ambiguous announce channels, while cron delivery still falls back to session- or binding-derived account IDs when `delivery.accountId` is omitted, so the issue describes a still-plausible gap rather than an obsolete request.

## Evidence
