## Summary

Keep open. Current `main` still does not implement the shared MIME correction described in #43313: `src/media/mime.ts` returns `fileTypeFromBuffer()`'s `video/mp4` result directly, without inspecting the MPEG-4 `ftyp` brand, while the only brand-based reclassification that exists today is still channel-specific in LINE and Slack code paths.

## Evidence

- **Shared MIME sniffing still lacks MP4 audio reclassification:** `sniffMime()` calls `fileTypeFromBuffer(buffer)` and returns `type?.mime` directly. There is no `ftyp`/brand check in the shared path, so audio-only MP4/M4A buffers are still liable to stay `video/mp4`.
  - file: [src/media/mime.ts:74](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/media/mime.ts#L74)
  - command: `nl -ba src/media/mime.ts | sed -n '74,84p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Media-understanding selection still trusts MIME first:** `resolveAttachmentKind()` returns `kindFromMime(attachment.mime)` before falling back to file extension. If stored MIME is `video/mp4`, the attachment is treated as video and never reaches the `.m4a` extension fallback.
  - file: [src/media-understanding/attachments.normalize.ts:79](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/media-understanding/attachments.normalize.ts#L79)
  - command: `nl -ba src/media-understanding/attachments.normalize.ts | sed -n '79,100p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Saved media still persists detectMime output:** `saveMediaBuffer()` and `saveMediaSource()` both propagate `detectMime(...)` into `contentType`, so the shared misclassification flows into later attachment routing.
  - file: [src/media/store.ts:381](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/media/store.ts#L381)
  - command: `nl -ba src/media/store.ts | sed -n '381,425p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Only LINE has the major-brand fix today:** `extensions/line/src/download.ts` has an `AUDIO_BRANDS` set and converts matching MP4 `ftyp` brands to `audio/mp4`, showing the generic fix has not been deduplicated into core.
  - file: [extensions/line/src/download.ts:13](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/line/src/download.ts#L13)
