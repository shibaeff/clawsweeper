## Summary

Keep open. Current `main` has a newer but different post-restart orphan-recovery path, and it does not implement PR #43497’s transcript-based rehydration/classification flow. The specific failure mode the PR targets, missing session-store entries during restore, is still pruned as an orphan on `main` rather than recovered.

## Evidence

- **Restarted in-flight runs still go straight back to agent.wait:** `src/agents/subagent-registry.ts:497-500` still handles in-flight restored runs by calling `waitForSubagentCompletion` again after restart. There is no resumability classifier, transcript replay, or fresh redispatch routing in this path.
  - file: [src/agents/subagent-registry.ts:497](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/subagent-registry.ts#L497)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Missing session-store entries are still treated as orphan conditions:** `src/agents/subagent-registry.ts:517-523` calls `reconcileOrphanedRestoredRuns()`. `src/agents/subagent-registry-helpers.ts:140-164` returns `missing-session-entry` / `missing-session-id`, and `src/agents/subagent-registry-helpers.ts:208-261` prunes the run once either condition is hit.
  - file: [src/agents/subagent-registry-helpers.ts:140](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/subagent-registry-helpers.ts#L140)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Current main uses a different recovery design:** `src/agents/subagent-orphan-recovery.ts:1-7` and `src/agents/subagent-orphan-recovery.ts:88-145` implement post-restart recovery by scanning for `abortedLastRun` sessions and sending a synthetic `agent` message telling the child to continue, then remapping the run. That is materially different from the PR’s transcript rehydration plus replay/fresh classification.
  - file: [src/agents/subagent-orphan-recovery.ts:1](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/subagent-orphan-recovery.ts#L1)

  - file: [src/agents/subagent-orphan-recovery.test.ts:94](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/subagent-orphan-recovery.test.ts#L94)

## Risks / Open Questions

- Current main recovery depends on `abortedLastRun` and an existing session-store row; the PR’s documented missing-store-entry race is still unresolved in the restore path.
- The synthetic "continue where you left off" resume flow is behaviorally different from transcript replay or exact redispatch, so closing this PR as already implemented would be overstated.
