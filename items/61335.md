## Summary

Keep open. Current `main` still contains the DST-fallback hour-slicing bug in both Usage aggregation and selected-hour filtering, and a direct Node reproduction shows the cursor can stop advancing at the repeated local hour. The PR addresses a real bug, although its test file still appears to need the `TZ` restore follow-up already noted in review.
