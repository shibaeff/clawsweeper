## Summary

Keep open. Current `main` still uses one shared Copilot token cache file and returns any still-valid cached token without tying it to an auth profile or GitHub token identity, so the reported profile-rotation failure still appears plausible and unfixed.

## Evidence

- **Single shared cache path on main:** `resolveCopilotApiToken()` always uses `credentials/github-copilot.token.json` as the cache file, with no profile- or account-specific keying.
  - file: [src/agents/github-copilot-token.ts:18](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/github-copilot-token.ts#L18)

- **Cache hit ignores caller identity:** When a cached token is usable, `resolveCopilotApiToken()` returns it immediately from `cache:${cachePath}` and does not compare against `githubToken`, profile id, or any account marker before reuse.
