## Summary

Current main intentionally requires canonical `path` parameters for `read`/`write`/`edit`, so this PR's `file`-alias summary fix no longer matches supported behavior and is obsolete as a bug fix.

## Evidence

- **Runtime contract is canonical `path` only:** `REQUIRED_PARAM_GROUPS` now requires only `path` for `read`, `write`, and `edit`, and `assertRequiredParams()` validates exact keys instead of normalizing aliases.
  - file: [src/agents/pi-tools.params.ts:75](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-tools.params.ts#L75)

  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Validation explicitly rejects legacy aliases:** The unit test `does not normalize legacy aliases during validation` asserts that passing `file_path` still throws a missing-`path` validation error.
  - file: [src/agents/pi-tools.params.test.ts:28](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-tools.params.test.ts#L28)
