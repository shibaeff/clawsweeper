## Summary

Keep open. Current `main` still has the split memory model the report is complaining about: workspace Markdown memory files, a separate builtin SQLite search index, and an optional QMD backend configured through separate knobs. The proposed unified `memory.type` / mixed-mode redesign is not implemented on `main`, so there is not strong evidence for a conservative close.

## Evidence

- **Workspace memory is still Markdown-based:** The docs still describe durable memory as `MEMORY.md` plus daily notes under `memory/YYYY-MM-DD.md` in the agent workspace.
  - file: [docs/concepts/memory.md:9](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/concepts/memory.md#L9)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Builtin memory still uses a separate SQLite index:** The builtin backend docs say it indexes `MEMORY.md` and `memory/*.md` into a per-agent SQLite database at `~/.openclaw/memory/<agentId>.sqlite`.
  - file: [docs/concepts/memory-builtin.md:76](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/concepts/memory-builtin.md#L76)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Config surface is still split between builtin and QMD:** `MemoryConfig` on current `main` only exposes `memory.backend?: "builtin" | "qmd"`, `citations`, and `qmd` settings. There is no `memory.type` enum like `built-in-only | qmd-only | mixed`.
  - file: [src/config/types.memory.ts:3](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/types.memory.ts#L3)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Pre-compaction flush still writes daily Markdown notes:** The bundled `memory-core` flush plan still targets `memory/YYYY-MM-DD.md` and computes `relativePath` as `memory/<date>.md`, so Markdown note writes remain a distinct path from the SQLite index.
  - file: [extensions/memory-core/src/flush-plan.ts:13](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/memory-core/src/flush-plan.ts#L13)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Reference docs still expose separate QMD knobs:** The memory config reference still documents QMD as a separate backend under `memory.backend = "qmd"` with its own `memory.qmd.*` settings, while the builtin index path remains under `agents.defaults.memorySearch.store.path`.
  - file: [docs/reference/memory-config.md:381](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/reference/memory-config.md#L381)
