## Summary

Current `main` no longer self-blocks loopback `attachOnly` CDP profiles behind the browser SSRF policy, and the shipped `v2026.4.23` release already contains both the code path and regression coverage for loopback `/json/version`, websocket discovery, status, and tabs under strict-default SSRF.

## Evidence

- **Loopback CDP reachability no longer uses browser SSRF as a self-block for local OpenClaw profiles:** `resolveCdpReachabilityPolicy()` returns `undefined` for non-remote loopback `driver === "openclaw"` profiles, explicitly documenting that browser SSRF is for page/navigation protection, not the local CDP control plane.
