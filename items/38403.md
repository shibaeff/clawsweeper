## Summary

Current main already implements the core capability this issue asks for, under the existing `message_sending` plugin hook rather than a new `response:before-deliver` hook name. Outbound delivery awaits that hook before sending, the hook can rewrite content or cancel delivery, the behavior is documented, and the latest release tag already contains it.

## Evidence

- **Hook contract supports modify-or-cancel before delivery:** `message_sending` receives the outbound content and may return rewritten `content` or `cancel: true`, which is the functional equivalent of an allow/modify/reject interceptor.
  - file: [src/plugins/hook-message.types.ts:45](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/plugins/hook-message.types.ts#L45)
- **Hook runner is awaited and terminal on cancel:** `runModifyingHook(...)` executes handlers sequentially, awaits each one, merges results, and stops when a terminal condition is met. `runMessageSending(...)` defines `cancel === true` as terminal.
  - file: [src/plugins/hooks.ts:347](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/plugins/hooks.ts#L347)
- **Delivery path applies the hook before any channel send:** `deliverOutboundPayloadsCore(...)` calls `applyMessageSendingHook(...)` before dispatching payloads. That helper awaits `runMessageSending(...)`, drops delivery on `cancel`, and rewrites payload text on modified content.
  - file: [src/infra/outbound/deliver.ts:567](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/infra/outbound/deliver.ts#L567)
- **Docs describe outbound interception semantics:** The plugin docs list `message_received / message_sending / message_sent` as outbound message hooks and document `message_sending` terminal cancel behavior.
  - file: [docs/plugins/sdk-overview.md:205](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/plugins/sdk-overview.md#L205)
- **Tests cover modify/cancel behavior:** Tests verify `runMessageSending(...)` can return modified content, can cancel delivery, and short-circuits lower-priority handlers after `cancel: true`.
  - file: [src/plugins/wired-hooks-message.test.ts:38](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/plugins/wired-hooks-message.test.ts#L38)
- **Shipped in latest release tag:** The `v2026.4.22` release tree already contains `message_sending` in the hook runner, delivery path, and plugin docs (`git show v2026.4.22:src/plugins/hooks.ts`, `git show v2026.4.22:src/infra/outbound/deliver.ts`, `git show v2026.4.22:docs/plugins/sdk-overview.md`).
  - command: `git show v2026.4.22:src/plugins/hooks.ts | rg -n "message_sending|runMessageSending"; git show v2026.4.22:src/infra/outbound/deliver.ts | rg -n "message_sending|runMessageSending|applyMessageSendingHook"; git show v2026.4.22:docs/plugins/sdk-overview.md | rg -n "message_sending"`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- The shipped API is named `message_sending`, not the proposed `response:before-deliver`, so consumers expecting the exact proposed surface will not find a 1:1 match.
- The current implementation is broader than the issue text: it intercepts outbound messages generally, not only agent replies, and it does not add a built-in per-hook timeout option.

## Close Comment

Closing from a Codex review: current OpenClaw already has this capability via the blocking `message_sending` plugin hook rather than a new `response:before-deliver` hook name. `src/infra/outbound/deliver.ts:567` awaits `runMessageSending(...)` before any channel send and either rewrites the outgoing text or skips delivery on `cancel: true`; the hook contract is in `src/plugins/hook-message.types.ts:45`, and the sequential/terminal behavior is defined in `src/plugins/hooks.ts:347` and `src/plugins/hooks.ts:733`. This is documented in `docs/plugins/sdk-overview.md:205` and covered by `src/plugins/wired-hooks-message.test.ts:38` plus `src/plugins/hooks.security.test.ts:194`. The latest release tag, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), already contains that implementation, so there is no separate core cleanup item left here.
