## Summary

Current `main` has timeout guards, retry/backoff, failure alerts, and time-based isolated-session pruning, but it does not implement the issue’s requested auto-pause/circuit-breaker for recurring cron failures, does not delete associated sessions when `cron remove` runs, and shows no repo support for the reported `.learnings/changelog.md` automation. The evidence is not strong enough to close this as implemented or obsolete.

## Evidence

- **Recurring cron failures alert/back off but do not auto-pause:** `applyJobResult` increments `consecutiveErrors`, may emit failure alerts, and for recurring jobs only schedules the next run with backoff; it does not disable the job after repeated execution failures/timeouts.
  - file: [src/cron/service/timer.ts:433](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/service/timer.ts#L433)
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Timeout safety exists on current main:** Cron runs are wrapped in a timeout abort path, so the reported 2026.3.8 timeout behavior is at least partially changed on `main`, but this is still not the requested breaker/pause behavior.
  - file: [src/cron/service/timer.ts:79](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/service/timer.ts#L79)
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **`cron remove` does not clean sessions:** The remove operation only filters the job out of the store, persists, and emits `removed`; it does not call any session/transcript cleanup path.
  - file: [src/cron/service/ops.ts:332](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/service/ops.ts#L332)
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Existing cleanup is retention-based, not delete-on-remove:** Current session cleanup is a periodic reaper for expired isolated run sessions, and its header explicitly says the base `...:cron:<jobId>` session is kept. The timer invokes this reaper opportunistically after ticks.
  - file: [src/cron/session-reaper.ts:1](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/session-reaper.ts#L1)
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Docs describe retention pruning, not remove-time cleanup or changelog recording:** Cron docs document `cron.sessionRetention` and run-log pruning as maintenance behavior; there is no documented `cron remove --cleanup` or changelog-writing feature in the cron docs.
  - file: [docs/automation/cron-jobs.md:360](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/docs/automation/cron-jobs.md#L360)
