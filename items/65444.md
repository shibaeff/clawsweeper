## Summary

Keep open. Current `main` still starts configured channel runtimes during gateway startup, bundled channels still open their transports from `gateway.startAccount`, there is no `channels.lazyConnect` config on `main`, and the existing lazy-loading work only defers full plugin entry loading until after the gateway starts listening.

## Evidence

- **Gateway startup still launches channel sidecars:** `startGatewaySidecars()` measures `sidecars.channels` and calls `params.startChannels()` during startup unless `OPENCLAW_SKIP_CHANNELS`/`OPENCLAW_SKIP_PROVIDERS` is set, so channel runtime startup is still part of normal boot.
  - file: [src/gateway/server-startup-post-attach.ts:258](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/src/gateway/server-startup-post-attach.ts#L258)
