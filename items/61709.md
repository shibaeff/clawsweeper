## Summary

Keep open. Current `main` still does not expose a supported host-wide or atomic auth-profile switch across all agent stores. The closest built-in controls are per-agent only, and the issue body's `lastGood`-in-`auth-profiles.json` detail is outdated on current `main` because runtime scheduling state now lives in `auth-state.json`.

## Evidence

- **Per-agent auth-order CLI only:** `openclaw models auth order get/set/clear` is explicitly defined as a per-agent override surface, with only `--agent <id>` support and no `--all-agents` or host-wide variant.
