## Summary

Current `main` still appears to have the reported bug. The cron tool’s auto-inference path strips `:thread:` from the caller session key before parsing, so a Telegram direct-chat session like `agent:...:telegram:direct:<chatId>:thread:<chatId>:<topicId>` will still infer a plain `delivery.to=<chatId>` instead of preserving the DM topic/thread context.

## Evidence

- **Cron inference still drops `:thread:` before parsing:** `inferDeliveryFromSessionKey()` calls `parseAgentSessionKey(stripThreadSuffixFromSessionKey(rawSessionKey))`, so any Telegram DM topic encoded in a `:thread:` suffix is removed before `delivery.to` is built.
  - file: [src/agents/tools/cron-tool.ts:366](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/tools/cron-tool.ts#L366)
  - command: `nl -ba src/agents/tools/cron-tool.ts | sed -n '351,412p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Auto-inference still runs for cron add when delivery target is omitted:** `cron.add` still fills in delivery via `inferDeliveryFromSessionKey(opts.agentSessionKey)` whenever announce delivery is implied and no explicit target was supplied, so the buggy inference remains live on current `main`.
  - file: [src/agents/tools/cron-tool.ts:584](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/tools/cron-tool.ts#L584)
  - command: `nl -ba src/agents/tools/cron-tool.ts | sed -n '560,620p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Current Telegram DM-thread model still matches the issue report:** The Telegram channel code still represents DM topic routes with a thread-scoped session key and a delivery-facing `telegram:<chatId>:topic:<threadId>` form. For direct chats, `resolveTelegramOutboundSessionRoute()` keeps `threadId` on the route and formats `from` as `telegram:<chatId>:topic:<threadId>`, and the session-route test expects `currentSessionKey: agent:main:telegram:direct:12345:thread:12345:99` to remain thread-scoped.
  - file: [extensions/telegram/src/channel.ts:493](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/extensions/telegram/src/channel.ts#L493)
