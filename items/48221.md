## Summary

Keep open. Current `main` still leaves unowned `anthropic-messages` custom providers on the old `modelId.includes("claude")` heuristic, so alias-based proxies described in the PR are not covered. The PR discussion also surfaced a real problem with the submitted patch: it broadened the fallback beyond `anthropic-messages` proxies and added a misleading native-Anthropic test, so this PR is not a high-confidence close candidate as-is.

## Evidence

That means custom `anthropic-messages` proxies using aliases like the PR examples (`shannon-auto`, `assistant-v2`) still keep replayed thinking blocks.
