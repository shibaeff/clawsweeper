## Summary

Keep open. Current `main` has a pre-compaction memory flush, but it is still a single-threshold, single-shot mechanism per compaction cycle. The requested progressive multi-stage flush is not implemented.

## Evidence

- **Single-threshold gate only:** `shouldRunMemoryFlush()` computes one threshold from `contextWindow - reserveTokensFloor - softThresholdTokens` and returns false after one flush in the current compaction cycle.
