## Summary

Keep open. Current `main` still does not sync skills into the sandbox workspace for the common `workspaceAccess: "rw"` case, so the PR is not implemented on main. But this PR is also not a safe close candidate: its sandbox change matches the blockers already raised in review, and only the unrelated locale tweak appears to have landed separately on `main`.

## Evidence

- **`rw` still bypasses sandbox skill sync on current main:** `ensureSandboxWorkspaceLayout` sets `workspaceDir` to `agentWorkspaceDir` when `workspaceAccess === "rw"`, then only runs `ensureSandboxWorkspace` and `syncSkillsToWorkspace` inside the `workspaceDir === sandboxWorkspaceDir` branch, with an additional `cfg.workspaceAccess !== "rw"` guard. That means current main still skips skill sync for ordinary `rw` sandboxes.
  - file: [src/agents/sandbox/context.ts:45](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/sandbox/context.ts#L45)
  - command: `nl -ba src/agents/sandbox/context.ts | sed -n '45,80p'`
  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)
- **Agent and sandbox workspace roots are separate by default:** The default agent workspace is `~/.openclaw/workspace`, while the default sandbox workspace root is `STATE_DIR/sandboxes`. For normal session/agent sandboxes those resolve to different directories, so the current `workspaceDir === sandboxWorkspaceDir` gate is usually false in `rw` mode.
  - file: [src/agents/workspace.ts:29](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/workspace.ts#L29)
