## Summary

Keep open. Current `main` does not already implement the shared filename-decoding and Feishu inbound filename-normalization work described in PR #71517, and the PR is a fresh draft with concrete code/tests against a real gap. Conservative cleanup should not auto-close it.

## Evidence

- **Current main still uses a local `Content-Disposition` parser:** `src/media/fetch.ts` still defines and uses `parseContentDispositionFileName(...)` locally instead of a shared decoder. The implementation only strips quotes, percent-decodes `filename*`, and basenames the result; it does not show the broader shared helper proposed in the PR.
