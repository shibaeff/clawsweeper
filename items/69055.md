## Summary

Current `main` does not emit the requested structured `cron.delivery.sent` log line. Cron direct delivery still passes only generic outbound send params, the outbound layer emits hook events rather than a cron-specific log record, and existing cron logging only covers warn/error cases such as stale deliveries or best-effort failures. The issue remains actionable on core `main`.

## Evidence

- **Outbound delivery API has no cron telemetry fields:** `DeliverOutboundPayloadsCoreParams` accepts channel/to/accountId/payloads/session/mirror and related send options, but there is no `cronJobId`, `cronRunId`, `agentId`, or telemetry object for structured cron logging.
  - file: [src/infra/outbound/deliver.ts:330](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/infra/outbound/deliver.ts#L330)

No cron attribution fields are forwarded for logging.
