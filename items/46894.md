## Summary

Keep open. Current `main` still routes `/model <args>` through the WebUI local slash-command executor, and `executeModel()` treats every non-empty argument as a literal model id. That means `/model status` and `/model list` still go to `sessions.patch` instead of the documented info/list behavior.

## Evidence

- **WebUI intercepts `/model` locally:** `parseSlashCommand()` returns the raw remainder as `args`, and `app-chat` dispatches local slash commands directly when `executeLocal` is set. `model` is in the local-command set, so `/model status` stays in the client-side executor path.
  - file: [ui/src/ui/chat/slash-commands.ts:82](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/ui/src/ui/chat/slash-commands.ts#L82)
  - command: `nl -ba ui/src/ui/chat/slash-commands.ts | sed -n '60,95p'; nl -ba ui/src/ui/chat/slash-commands.ts | sed -n '520,565p'; nl -ba ui/src/ui/app-chat.ts | sed -n '350,372p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Local executor still patches any non-empty `/model` arg:** `executeModel()` only special-cases empty args. Otherwise it trims `args` and immediately calls `client.request("sessions.patch", { key: sessionKey, model: requestedModel })`, so `status` or `list` will be treated as model ids.
  - file: [ui/src/ui/chat/slash-command-executor.ts:185](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/ui/src/ui/chat/slash-command-executor.ts#L185)
  - command: `nl -ba ui/src/ui/chat/slash-command-executor.ts | sed -n '175,245p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Docs still promise `/model list` and `/model status` special behavior:** The user docs explicitly say `/model` and `/model list` show a compact picker, and `/model status` shows the detailed status view. Current WebUI local handling does not implement that contract.
  - file: [docs/tools/slash-commands.md:240](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/tools/slash-commands.md#L240)

  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Shared directive model handling expects summary/status modes:** Server-side directive handling distinguishes summary/list/detail flows and references `/model status` as the detailed path, which further shows the WebUI local executor is out of sync with intended behavior.
  - file: [src/auto-reply/reply/directive-handling.model.ts:233](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/auto-reply/reply/directive-handling.model.ts#L233)
