## Summary

Keep open. Current `main` still explicitly removes `qwen3.6-plus` from Qwen Coding Plan catalogs, suppresses the model at runtime with the same fallback/error described in the issue, and documents/tests that behavior as expected. There is no local evidence that #66367 or any equivalent fix landed on `main` or in `v2026.4.23`.
