## Summary

Keep open. Current `main` does not preserve embedded tool-result `ReplyPayload`s in the final returned payload list; it still only builds assistant payloads and optionally merges raw tool media URLs. The PR’s core behavior (`toolResultPayloads` buffering plus compaction-retry reset) is not present on `main`, so there is not strong evidence to close this as already implemented or obsolete.
