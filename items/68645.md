## Summary

Keep open. Current `main` at `b84e57fca38deacfa9834a82f7e270b32562ae1b` does not contain the PR's `runTimeoutSeconds` wall-clock kill or the separate timeout-path ledger-close behavior, and the existing cron tests only cover the existing `timeoutSeconds` abort path. This remains a plausible bug fix rather than cleanup-ready noise.
