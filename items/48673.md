## Summary

Keep open. Current `main` still emits the duplicate-warning path for an explicitly installed global plugin that overrides a bundled plugin, so this cleanup is not implemented on `main`. The PR discussion also shows a real design constraint: warning suppression must preserve duplicate-state consumers in loader and config validation, so this still needs maintainer review rather than auto-close.

## Risks / Open Questions

- Auto-closing would leave the user-visible duplicate warning behavior unchanged on current `main` for explicitly installed plugin overrides.
- Any future fix must preserve downstream duplicate-state consumers in loader and config validation; suppressing warnings naively can regress disabled-plugin status or compat handling.
