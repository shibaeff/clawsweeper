## Summary

Current `main` no longer uses the legacy `chrome-relay` auto-profile/extension-relay path and now supports browser-node routing for the live user browser, with regression coverage for attach-only loopback CDP tab listing. The reported Docker + host-node mismatch is obsolete on current `main`.

## Evidence

## Close Comment

Thanks for the report. Closing this because a Codex review of current `main` found that the old Chrome relay path described here has been replaced and the current browser stack already implements the supported split-host path. On current `main`, existing-session `profile="user"` browser tool calls can route through a connected browser node or explicit `target="node"` (`extensions/browser/src/browser-tool.ts:405`, covered in `extensions/browser/src/browser-tool.test.ts:548`). There is also regression coverage that attach-only loopback CDP profiles return tabs from `/json/list` (`extensions/browser/src/browser/routes/tabs.attach-only.test.ts:16`). Separately, the legacy extension-relay / auto-created `chrome-relay` path from older builds was removed in the `2026.3.22` migration (`CHANGELOG.md:2211`, `CHANGELOG.md:2437`, and `extensions/browser/src/browser/config.test.ts:57`). I did not find a current-main code path that still matches the old “direct CLI sees the tab but chat/browser returns no tabs” behavior, so this now looks implemented/obsolete rather than an active maintainer cleanup item.
