## Summary

Keep open. Current `main` still treats model fallback order as explicit operator configuration, and I did not find a shipped cost-first fallback optimization, benchmark matrix, or candidate-specific implementation matching issue #50286.

## Evidence

- **Failover docs still describe configured-order fallback:** The current docs say normal text runs evaluate the selected session model first, then `agents.defaults.model.fallbacks` in order, then the configured primary when starting from an override. That matches explicit ordering, not an automatic cost-first optimizer.
