## Summary

Keep open. Current main supports Telegram group sender allowlists and mention-skip ingest separately, but it does not implement the requested behavior of observing all group messages while only allowlisted users can trigger replies. Unauthorized group senders are rejected before the silent-ingest path runs.

## Evidence

- This review is based on static inspection of current main code/docs/tests in a read-only checkout; I did not perform a live Telegram runtime reproduction.
