## Summary

Current main has only a partial guardrail here. The config schema already restricts provider model `input` values to `text`/`image`, but I did not find the follow-up acceptance items from #50274: no benchmark/ranking artifact for the cited fallback models, no docs update that defines a latency/quality/cost fallback-order policy for them, and no targeted regression test covering invalid model-input modality configs in the config-validation suite.
