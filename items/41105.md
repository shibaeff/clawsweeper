## Summary

Keep open. Current `main` still codifies the final-only Telegram preview behavior this PR is trying to remove, and the PR discussion shows the reported regressions were real and iteratively fixed on the branch. This is not `implemented_on_main`, not `cannot_reproduce`, and not a safe conservative close.

## Evidence

That is the opposite of this PR's stated fix (`final-only answers should send normally without creating a first preview bubble`).
