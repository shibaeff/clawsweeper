## Summary

Keep open. Current `main` does not implement this SQLite-backed session-store feature: session persistence still reads/writes `sessions.json`, there is no `session.storeType` config surface, and the CLI/docs do not expose `sessions migrate` or `sessions store-info`.

## Evidence

- **Session store on main is still JSON-backed:** `loadSessionStore()` on current `main` reads the whole store with `fs.readFileSync(storePath, "utf-8")` and `JSON.parse(raw)`, then returns the in-memory object; there is no backend facade or SQLite path.
