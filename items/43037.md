## Summary

Keep open. Current main has explicit token/session handling for same-gateway Control UI navigation, but the reported failure is still plausible because gateway token scope is keyed by exact host/path. `ui/src/ui/storage.ts:103` normalizes scope to `protocol://host/path` without canonicalizing loopback aliases, so `localhost` and `127.0.0.1` remain different buckets. The repo has tests proving token persistence for same-gateway refresh/session flows, but I did not find coverage for switching between loopback aliases, which was the concrete root-cause theory in the issue comments.
