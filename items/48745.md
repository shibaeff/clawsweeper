## Summary

Keep open. Current `main` still uses the bare `"file"` fallback for WhatsApp document sends when no filename is supplied, and the current unit test explicitly locks that behavior in. The broader WhatsApp outbound normalization path now preserves an existing or URL-derived filename, but it still falls back to `"file"`, not a MIME-derived extension, so this PR's requested behavior is not already implemented on `main`.
