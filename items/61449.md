## Summary

Close as `clawhub`. On current `main`, this request is a bundle of optional, environment-specific operational policy for WSL2 hosts. OpenClaw’s documented direction is to keep that kind of capability out of core and ship it via plugins/skills/ClawHub, and the current plugin surface already exposes the needed setup, service, hook, command, and compaction seams. Core also already has generic doctor/update/onboarding health behavior, so this is not a missing core foundation.
