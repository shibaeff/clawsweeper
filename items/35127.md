## Summary

Issue #35127 requests a new optional messaging-channel integration. Current main explicitly steers optional capabilities, including new channel plugins, to externally maintained plugins distributed via ClawHub or npm rather than adding them to OpenClaw core.

## Evidence

## Close Comment

Closing this as a ClawHub/plugin-scope request after Codex review of current `main`. The current project direction is to keep optional capability out of core where possible: `VISION.md:54-59` says optional capability should usually ship as plugins, be hosted in the author’s own repository, and that the bar for adding optional plugins to core is intentionally high. The supported path for new messaging integrations is the external channel-plugin SDK and marketplace flow: `docs/plugins/building-plugins.md:16-19` says new plugins do not need to be added to this repo and should be published to ClawHub or npm, while `docs/plugins/community.md:9-22` and `docs/plugins/sdk-channel-plugins.md:11-13` document community channel plugins as the normal extension path. Since this request is for a new optional channel integration, it fits the ClawHub/community-plugin route rather than OpenClaw core. If a Keet plugin matures and is published on ClawHub or npm, that is the supported delivery path for users.
