## Summary

Current `main` does not already implement this PR, and the proposed change is not high-confidence as written: `listSessionFilesForAgent` still intentionally indexes usage-counted reset/deleted transcripts on `main`, while the PR's broad `.checkpoint.` substring guard would also exclude valid dotted session IDs. Conservative cleanup should leave this PR open for maintainer judgment rather than auto-close it.
