## Summary

The reported custom OpenAI-compatible provider failure path is already implemented in current main and was present in the shipped v2026.4.23 snapshot: the embedded subscriber now backfills final assistant text on `message_end` for non-streaming providers, the payload builder emits that text as the visible reply, and the release snapshot includes a regression test for the exact no-`text_delta` case.

## Evidence
