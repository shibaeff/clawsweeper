## Summary

Current `main` does not contain this PR's blockquote-prefix preservation logic, and the underlying bug is still evident in `src/markdown/ir.ts`. This remains a live, well-scoped fix rather than a cleanup close candidate.

## Evidence

- **Soft/hard breaks still drop the prefix:** `softbreak` and `hardbreak` both append a raw newline only, with no blockquote-prefix re-emission for continuation lines.
  - file: [src/markdown/ir.ts:634](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/markdown/ir.ts#L634)
  - command: `nl -ba src/markdown/ir.ts | sed -n '620,700p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Prefix is only added when entering a blockquote:** `blockquote_open` prepends `blockquotePrefix` once, but there is no tracked quote depth or continuation helper in current `main`. Nested/wrapped quote lines therefore cannot preserve full quoting structure.
  - file: [src/markdown/ir.ts:652](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/markdown/ir.ts#L652)
  - command: `nl -ba src/markdown/ir.ts | sed -n '620,700p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Paragraph separators inside quotes still bypass prefix logic:** `appendParagraphSeparator` writes raw `"\n\n"` with no quote-aware handling, so later paragraphs in a prefixed blockquote still lose their `> ` prefix.
  - file: [src/markdown/ir.ts:265](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/markdown/ir.ts#L265)
  - command: `nl -ba src/markdown/ir.ts | sed -n '220,320p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Regression coverage from the PR is not on main:** Current tests only cover the basic single-paragraph `blockquotePrefix` case; the continuation-line, hardbreak, multi-paragraph, and nested-prefix cases from the PR are absent.
  - file: [src/markdown/ir.blockquote-spacing.test.ts:140](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/markdown/ir.blockquote-spacing.test.ts#L140)
  - command: `nl -ba src/markdown/ir.blockquote-spacing.test.ts | sed -n '130,210p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)

## Risks / Open Questions

- Renderers using `blockquotePrefix` still flatten wrapped quoted text on soft/hard line breaks.
- Multi-paragraph and nested blockquotes with a configured prefix still appear vulnerable to missing or reduced quote markers on continuation content.
