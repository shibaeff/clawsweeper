## Summary

Current `main` already handles forced Matrix cross-signing reset through an explicit UIA fallback sequence, avoids mutating secret storage before that UIA step succeeds, documents the remaining password-UIA limitation, and the fix is listed as shipped in `v2026.4.15`.

## Evidence

- **Forced-reset bootstrap now uses explicit UIA fallback and defers secret-storage mutation:** `MatrixCryptoBootstrapper.bootstrap()` defers secret-storage bootstrap during `forceResetCrossSigning`, and `createSigningKeysUiAuthCallback()` now tries `null`, then `m.login.dummy`, then `m.login.password` with a Matrix user identifier. That directly covers the malformed `device_signing/upload` UIA path described in the issue.
