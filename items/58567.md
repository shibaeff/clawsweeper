## Summary

Keep open. Current `main` does not show a fix for assistant-ended Anthropic replay transcripts: the Anthropic turn validator only strips dangling tool calls and merges consecutive `user` turns, and the embedded runner only repairs orphaned trailing `user` leaves before submitting a new prompt.

## Evidence

- **Anthropic replay validation has no trailing-assistant repair:** `validateAnthropicTurns` strips dangling Anthropic tool calls, then runs consecutive-turn merging only for `role: "user"`. There is no branch that rewrites or appends a `user` turn when replay history ends with an `assistant` message.
  - file: [src/agents/pi-embedded-helpers/turns.ts:370](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-embedded-helpers/turns.ts#L370)

- **Test suite explicitly preserves consecutive assistant turns:** Current tests assert that `validateAnthropicTurns` does not merge assistant messages and leaves a `user, assistant, assistant` sequence at length 3, which matches the reported failure shape rather than repairing it.
  - file: [src/agents/pi-embedded-helpers.validate-turns.test.ts:283](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-embedded-helpers.validate-turns.test.ts#L283)
