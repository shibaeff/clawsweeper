## Summary

Current main already provides the generic plugin seams needed for this work, while repo policy says optional commercial integrations and new skill-like capabilities should live outside core. The remaining Morph-specific compaction/code-search integration is better scoped as an external/ClawHub plugin than a bundled OpenClaw PR.

## Close Comment

Closing this as a Codex review with reason `clawhub`. Current `main` already has the generic plugin seams this work needs: plugins can register tools via `src/plugins/types.ts:1948`, register compaction providers via `src/plugins/types.ts:2036`, and core now documents pluggable compaction providers in `docs/concepts/compaction.md:75` and `CHANGELOG.md:1077`. That means the remaining Morph-specific commercial integration and `codebase_search` tool no longer need to ship as bundled OpenClaw core code.

The scope guidance is also explicit here: `VISION.md:55-59` says optional capability should usually ship as plugins hosted in their own repo, and `VISION.md:103` calls out commercial service integrations outside the model-provider category as out of scope for core. Given that, this is better re-homed as an external/ClawHub plugin rather than kept open as a core PR.
