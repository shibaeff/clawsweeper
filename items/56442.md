## Summary

Current `main` does not implement this PR's opt-in ACP parent completion notify path. The `sessions_spawn` surface still only accepts `streamTo`, ACP run completions still resolve through generic background-task/system-event delivery, and the ACP/subagent announce contracts on `main` still do not support an ACP-specific task-completion announce flow. The later cross-reference to open issue #70264 also points to the same parent-completion wake gap still being active after this PR was opened.
