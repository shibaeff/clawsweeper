## Summary

Keep open. Current `main` does not implement the PR's PluralKit canonical inbound-id behavior: Discord preflight still skips PluralKit lookup for webhook-authored messages, and Discord inbound processing still records only the proxied Discord message id as `MessageSid`. The PR discussion also ends with an unresolved review note about lookup ordering around the bot-drop gate, so this remains active review work rather than conservative cleanup-close material.
