## Summary

Current `main` already fixes the reported Telegram proxy/media-download conflict by routing Telegram media fetches through a trusted explicit-proxy SSRF mode that preserves hostname-policy checks but skips target DNS pinning when an operator-configured explicit proxy is in use. The fix is on `main` but I could not tie it to any released tag yet.
