## Summary

The proposal is coherent, but current `main` already treats context reduction and alternative compaction/assembly strategies as plugin-owned extension points. OpenClaw documents context engines and pluggable compaction providers for this exact scope, and it already points users to an existing community plugin (`@martian-engineering/lossless-claw`) that reduces token usage while preserving context fidelity.

## Risks / Open Questions

- The existing plugin seams do not by themselves guarantee the exact reversible synonym-rewrite design requested in the issue; this close assumes plugin-level ownership is the right scope boundary.
- If maintainers later decide a narrower core seam is still missing for tool-result-only microcompression, that would justify a focused API issue rather than this broad core feature request.
