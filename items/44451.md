## Summary

Keep open. Current `main` still lacks a delivery-context-based lookup in the `agent` session resolution path, so the reported fallback to `agent:*:main` remains plausible rather than already fixed.

## Evidence

- **Agent command drops delivery metadata before session resolution:** `prepareAgentCommandExecution()` calls `resolveSession()` with only `cfg`, `to`, `sessionId`, `sessionKey`, and `agentId`. It does not pass `channel`, `accountId`, `threadId`, or `runContext`, so the resolver cannot choose a live channel/thread session by delivery context.
  - file: [src/agents/agent-command.ts:320](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/agent-command.ts#L320)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '318,345p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Resolver API has no delivery-context lookup path:** `resolveSessionKeyForRequest()` only accepts `to`, `sessionId`, `sessionKey`, and `agentId`. It derives `MsgContext` as `{ From: opts.to }` and never inspects stored `deliveryContext`, `lastTo`, `lastAccountId`, or `lastThreadId` when selecting a session.
  - file: [src/agents/command/session.ts:130](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/command/session.ts#L130)
  - command: `nl -ba src/agents/command/session.ts | sed -n '130,193p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Direct-chat fallback still collapses to main:** `resolveSessionKey()` explicitly returns the canonical main key for any non-group direct context. Without an explicit/group key, direct chats still resolve to `agent:<default>:main`.
  - file: [src/config/sessions/session-key.ts:25](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/config/sessions/session-key.ts#L25)
  - command: `nl -ba src/config/sessions/session-key.ts | sed -n '25,47p'`

- **Inputs already carry the missing routing fields:** `AgentCommandOpts` includes `channel`, `accountId`, `threadId`, and `runContext.currentChannelId/currentThreadTs`, so the lack of delivery-context resolution is a code-path omission, not an input-schema limitation.
  - file: [src/agents/command/types.ts:16](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/command/types.ts#L16)
