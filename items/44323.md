## Summary

Keep open. Current `main` still does not implement config-load validation for `memorySearch.provider`/provider-specific fields. The only related logic on `main` is doctor/runtime health checking, and the PR also carries unrelated cron/agent lifecycle additions that are not present on `main` and need separate maintainer judgment.
