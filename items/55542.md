## Summary

Keep open. Current `main` still relies on Sharp and still documents Sharp-specific install behavior, so this migration is not implemented on `main`. The PR discussion also records unresolved correctness regressions in the proposed `@napi-rs/image` port, and `main` has since refactored the touched image-ops path, which makes the PR stale but not safely auto-closable under the allowed reasons.
