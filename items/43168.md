## Summary

Keep open. Current `main` still routes queued heartbeat/system events into the session prompt, persists that prompt as a user turn, and `chat.history` does not filter generated user-side system-wrapper messages from the dashboard transcript.

## Evidence

- **System events are prepended into the reply prompt:** `get-reply-run` drains formatted system events and passes them into `buildReplyPromptBodies`, so event text becomes part of the next user-side prompt body for the session.
  - file: [src/auto-reply/reply/get-reply-run.ts:468](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/auto-reply/reply/get-reply-run.ts#L468)

- **Embedded runner persists the effective prompt into session history:** The embedded runner submits `effectivePrompt` with `activeSession.prompt(...)`, which is the path that records the synthesized prompt into the transcript.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2289](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/pi-embedded-runner/run/attempt.ts#L2289)
