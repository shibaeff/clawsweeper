## Summary

Keep open. Current `main` still treats isolated cron runs as `"ok"` when `meta.error` is present without any `isError` payloads, so the PR's core bugfix has not been implemented on `main` or in the latest release. The PR also already incorporated the review feedback about avoiding `"[object Object]"` fallbacks in error messages.

A run-level `meta.error` with no error payloads still falls through to `"ok"`.
