## Summary

Keep open. Current `main` still appears to have the same macOS companion-node mismatch: the gateway/node exec path requires `system.run.prepare`, the headless node host implements it, but the macOS app node advertises `system.run` without implementing `system.run.prepare`. That matches the reported failure mode rather than showing a fix on `main`.
