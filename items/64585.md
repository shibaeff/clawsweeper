## Summary

Current `main` has a narrower restart-recovery path for stale transcript locks, but it does not implement the issue's requested startup replay for unanswered gateway chat sessions. The new recovery only targets `abortedLastRun` main sessions, skips ordinary running sessions, and resumes via `agent` with `deliver: false` rather than the user-visible `sessions.send` pipeline.
