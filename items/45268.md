## Summary

Current `main` only fixes the missing-`totalTokens` case. It still treats explicit `totalTokens` as usable even when `totalTokensFresh=false`, so `/status` can still report a concrete context percentage for stale data. That does not meet the issue's stated expectation to show unknown usage when freshness is false.

## Evidence

- **Fresh-only helper exists:** `resolveFreshSessionTotalTokens()` returns `undefined` when `entry.totalTokensFresh === false`, which matches the issue's proposed rule to hide stale totals.
  - file: [src/config/sessions/types.ts:424](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/config/sessions/types.ts#L424)
  - command: `nl -ba src/config/sessions/types.ts | sed -n '410,445p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Status summary still ignores freshness for percent/remaining:** `getStatusSummary()` uses `resolveSessionTotalTokens(entry)` rather than the fresh-only helper, then computes `remainingTokens` and `percentUsed` whenever `total` is defined, even if `totalTokensFresh` is false.
  - file: [src/commands/status.summary.ts:206](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/commands/status.summary.ts#L206)

  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Text formatting also renders stale totals as concrete usage:** `formatTokensCompact()` renders `used/context (pct)` whenever `totalTokens` is non-null; it has no freshness check.
  - file: [src/commands/status.format.ts:17](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/commands/status.format.ts#L17)
  - command: `nl -ba src/commands/status.format.ts | sed -n '15,40p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Regression tests currently lock in stale percentage output:** The current test suite explicitly expects `totalTokensFresh: false` plus `totalTokens: 5000` to produce `percentUsed === 50` and `remainingTokens === 5000`, which is the misleading behavior the issue asks to avoid.
  - file: [src/commands/status.test.ts:1024](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/commands/status.test.ts#L1024)

## Risks / Open Questions

- The existing tests make future accidental reintroduction likely even if the missing-total fallback was fixed earlier.
