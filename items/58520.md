## Summary

Current `main` already implements the reported missing pieces: `sessions.resolve` resolves by session label, the TUI identifies itself to Gateway as `openclaw-tui`, and shipped tests cover the label-resolution and no-extra-history-reload paths. The issue's reported root cause does not match current `main`, and the same implementation is present in release `v2026.4.23`.
