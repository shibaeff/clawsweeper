## Summary

This PR is a new optional channel plugin, and current main explicitly steers that work toward external/community plugins published on ClawHub or npm rather than new in-repo bundled plugins. The proposed implementation also targets an outdated in-repo layout (`workspace/plugins/...`) that is not present on current main, where bundled plugins live under `extensions/` and use the current channel entry contract.

## Evidence

- **Vision sets the default scope boundary for optional plugins:** Current project vision says optional capability should usually ship as plugins, the preferred path is npm distribution plus local extension loading for development, plugin authors should host and maintain plugins in their own repository, and the bar for adding optional plugins to core is intentionally high.

## Close Comment

Closing this PR after Codex review because the Dispatch integration fits the external plugin / ClawHub path rather than a new in-repo bundled plugin. Current policy on `main` says optional capability should usually ship as plugins, the preferred path is an npm package in its own repository, and the bar for adding optional plugins to core is intentionally high (`VISION.md:54`). The plugin docs also now explicitly say you do not need to add a plugin to this repository and should publish it to ClawHub or npm instead (`docs/plugins/building-plugins.md:16`, `docs/plugins/community.md:9`, `docs/plugins/community.md:129`). This PR also targets an old `workspace/plugins/...` layout that does not exist on current `main`; bundled plugins now live under `extensions/` and use the current entry contract (for example `extensions/discord/index.ts:1` and `extensions/discord/package.json:28`). If you still want to ship Dispatch, please publish it as an external OpenClaw plugin and link the repo/package back here.
