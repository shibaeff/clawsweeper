## Summary

Keep open. Current `main` does not implement model-call counts in `/usage` response footers: the formatter still emits only token/cache/cost text, the caller passes no call-count field, the runtime metadata type has no model-call-count property, and existing tests only assert the current no-count output.

## Evidence

- **Usage footer formatter has no call-count support:** `formatResponseUsageLine` only accepts `usage`, `showCost`, and `costConfig`, and returns `Usage: ${inputLabel} in / ${outputLabel} out${cacheSuffix}${suffix}`. There is no call-count parameter or `(N calls)` suffix.
  - file: [src/auto-reply/reply/agent-runner-usage-line.ts:9](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/auto-reply/reply/agent-runner-usage-line.ts#L9)

  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **`/usage` footer call site passes no count:** `runReplyAgent` builds the footer with `formatResponseUsageLine({ usage, showCost, costConfig })` and only conditionally appends the session key for `full`; no model-call count is computed or forwarded.
