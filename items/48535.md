## Summary

Keep open. Current `main` still serves the full gateway catalog from `models.list` when `agents.defaults.models` is empty, even if `models.mode` is `"replace"`, so the reported dashboard dropdown bug is not implemented on `main`. The PR discussion is substantive, refreshed against April 13, 2026 `main`, and its proposed fix matches the gap visible in current source.

## Evidence

- **`models.list` still falls back to the full catalog on current main:** The gateway handler loads the catalog, calls `buildAllowedModelSet({ cfg, catalog, ... })`, and then returns `catalog` whenever `allowedCatalog.length === 0`. There is no `models.mode === "replace"` provider filtering in this path.
  - file: [src/gateway/server-methods/models.ts:26](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/gateway/server-methods/models.ts#L26)

  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)

That matches the PR's claim that `models.list` shows all built-in models when no defaults allowlist is configured.
