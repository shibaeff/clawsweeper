## Summary

Keep open. Current `main` still does not show an OpenClaw-side fix for deactivated ChatGPT workspace selection or retry in the OpenAI Codex OAuth path; the only Codex-specific hardening on `main` is for `accountId` extraction failures, and the issue discussion includes a second user report that the `deactivated_workspace` failure was still happening on `v2026.4.8`.

## Evidence

- **OAuth login path still delegates upstream and adds no workspace validation:** `loginOpenAICodexOAuth` calls upstream `loginOpenAICodex(...)` and returns the credentials without any OpenClaw-side enumeration, active-workspace filtering, or deactivated-workspace retry logic.
