## Summary

Close as implemented on main. Current Discord gateway lifecycle code now detects the reported silent WebSocket/READY stall, force-restarts or force-stops stale sockets, and shipped release notes indicate that behavior was released after the reporter's 2026.3.12 version.

## Close Comment

Closing from a Codex review because current `main` already implements the fix for this Discord gateway failure mode. The shipped note in `CHANGELOG.md:1661` says `v2026.3.31` force-stops Discord sockets that open without reaching READY instead of waiting on stale health timeouts, which matches the silent WebSocket stall/restart pattern reported here on `2026.3.12`. The current implementation in `extensions/discord/src/monitor/provider.lifecycle.ts:24` and `extensions/discord/src/monitor/provider.lifecycle.ts:198` drains stale sockets before reconnect and marks `runtime-not-ready` when a socket opens but never becomes READY, and `extensions/discord/src/monitor/provider.lifecycle.test.ts:504` covers that exact case. If this still reproduces on `v2026.3.31` or newer, please open a fresh issue with current logs and version details.
