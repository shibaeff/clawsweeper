## Summary

Keep open. Current `main` still blocks the LINE webhook HTTP 200 on event processing in both handler paths, and the existing tests explicitly encode that blocking/500 behavior. The linked PR #65397 carried the proposed async-ACK fix but was closed unmerged, so there is no strong basis to close this as already implemented or obsolete.

## Evidence

- **Node webhook still waits for LINE event processing before replying 200:** `createLineNodeWebhookHandler` still does `await params.bot.handleWebhook(body)` and only then writes the 200 JSON response, which matches the reported timeout/token-expiry failure mode.
