## Summary

Keep open. Current `main` already reloads assistant identity for the active session, but the resolver still gives `ui.assistant` higher priority than `agents.list[].identity`, so the reported multi-agent Control UI behavior is not fully implemented.

## Evidence

- **Global UI assistant still overrides per-agent identity:** `resolveAssistantIdentity()` resolves `name` from `cfg.ui.assistant.name` before `resolveAgentIdentity(cfg, agentId)?.name`, and resolves avatar candidates with `cfg.ui.assistant.avatar` before agent identity fields. That means a configured global UI assistant still wins over per-agent `agents.list[].identity`.
  - file: [src/gateway/assistant-identity.ts:92](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/assistant-identity.ts#L92)
  - command: `nl -ba src/gateway/assistant-identity.ts | sed -n '81,117p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Control UI does ask for identity for the active session:** On session switches, the Control UI calls `loadAssistantIdentity()`, which requests `agent.identity.get` for the current `sessionKey`. The gateway handler resolves the agent from that session key, so per-session identity refresh is wired up.
  - file: [ui/src/ui/app-render.helpers.ts:509](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/ui/src/ui/app-render.helpers.ts#L509)

- **Gateway identity RPC still uses the same resolver:** `agent.identity.get` calls `resolveAssistantIdentity({ cfg, agentId })`, so the same global-first precedence applies even when the UI switches sessions.

## Risks / Open Questions

- The issue can look partially fixed because session switching already reloads identity per session; closing it would miss the remaining precedence bug when `ui.assistant` is set.
- There does not appear to be a regression test covering the requested precedence: active agent `agents.list[].identity` overriding global `ui.assistant` in Control UI.
