## Summary

Current `main` already handles the reported Ollama `prompt too long` overflow path: it classifies the Ollama payload as a context overflow, surfaces friendly overflow copy instead of raw JSON, pre-checks for prompt-budget overflow before submission, and auto-compacts/retries on overflow. The fix was landed in commit `76500c7a785e1491de2b4634fbc76cc66e0959bb` and is present in release `v2026.3.22`.

## Risks / Open Questions

- This review did not do a live Ollama reproduction in the read-only checkout; it relies on current source, regression tests, changelog, and commit metadata.
- Preemptive compaction depends on token estimation heuristics, so separate edge cases with inaccurate provider metadata could still exist even though the reported Ollama `prompt too long` path is implemented.
