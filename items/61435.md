## Summary

Keep open. Current `main` still threads Google Chat inbound replies by default, and the Google Chat config/schema does not provide the requested DM-specific `threadReplies`/`replyToMode` control that Matrix already has.

## Evidence

- **Google Chat DM config has no thread override field:** `GoogleChatDmConfig` only exposes `enabled`, `policy`, and `allowFrom`. The account config has a top-level `replyToMode`, but there is no `dm.threadReplies` or DM-only threading override in the Google Chat types.
  - file: [src/config/types.googlechat.ts:11](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/types.googlechat.ts#L11)

- **Google Chat schema matches the missing DM override:** `GoogleChatDmSchema` only accepts `enabled`, `policy`, and `allowFrom`, and `GoogleChatAccountSchema` exposes only a top-level `replyToMode`. Current schema does not accept a DM-specific threading option.
