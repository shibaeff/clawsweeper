## Summary

Keep open. Current `main` partially addresses the workaround by supporting custom request-auth headers for model providers, but it does not implement the requested first-class fal.ai text provider path: the bundled `fal` provider is still documented and registered for image/video generation only, text-inference onboarding still filters it out, and the custom-provider onboarding docs still only describe base URL/API key/model inputs rather than an auth-scheme or live connection-test flow.
