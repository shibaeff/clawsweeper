## Summary

Keep open. Current `main` does not contain the same-agent cross-root remap described in PR #61322, and the existing review discussion identified a real correctness hole: the proposed remap path would ignore caller-supplied custom `session.store` roots and can redirect transcripts into the default state tree.

## Evidence

- **Current main still preserves canonical cross-root absolute paths instead of remapping them:** `resolvePathWithinSessionsDir()` still falls through to `resolveStructuralSessionFallbackPath(...)`, and that helper returns the original absolute path unchanged for canonical `.../agents/<agent>/sessions/<file>` shapes. That matches the bug report, not an implemented fix.
  - file: [src/config/sessions/paths.ts:132](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/paths.ts#L132)
  - command: `nl -ba src/config/sessions/paths.ts | sed -n '132,227p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Fallback logic still hard-resolves through OPENCLAW_STATE_DIR:** When absolute-path recovery runs, `tryAgentFallback()` uses `resolveAgentSessionsDir(normalizedAgentId)`, which derives from the current state dir rather than the caller's active `sessionsDir`. That aligns with the PR review comment warning that custom `--session.store` roots would be redirected into the default state tree.
  - file: [src/config/sessions/paths.ts:200](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/paths.ts#L200)
  - command: `nl -ba src/config/sessions/paths.ts | sed -n '191,203p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Session-file resolution is explicitly store-path aware:** OpenClaw derives `sessionsDir` from `storePath`, and transcript writes pass `path.dirname(storePath)` into `resolveAndPersistSessionFile()`. A same-agent remap that ignores that directory would split metadata and transcript writes across stores.
  - file: [src/config/sessions/transcript.ts:58](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/transcript.ts#L58)

  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **User-facing docs promise templated session.store roots:** Docs say routing/session discovery supports templated `session.store` roots, so a fix that only remaps into the default state dir would violate documented behavior for custom stores.
  - file: [docs/channels/channel-routing.md:123](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/channels/channel-routing.md#L123)

The same-agent remap case described in this PR is not present on `main`, so this is not already-shipped behavior.
