## Summary

Keep open. Current `main` still has the same lazy loader for `task-registry-control.runtime.{js,ts}`, but the build entry list on `main` does not emit that runtime, and the latest published tarball `v2026.4.23` still omits it from `dist/` and `dist/postinstall-inventory.json`. The user-facing docs still advertise `openclaw tasks cancel`, so this remains an active packaging bug rather than an obsolete report.

## Evidence

- **Lazy loader still requires missing runtime on main:** `task-registry.ts` still tries `require('./task-registry-control.runtime.js')` and `require('./task-registry-control.runtime.ts')`, then throws `Failed to load task registry control runtime.` if neither exists.
  - file: [src/tasks/task-registry.ts:75](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/tasks/task-registry.ts#L75)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Cancellation path still depends on that loader:** `cancelTaskById()` still calls `loadTaskRegistryControlRuntime()` for both ACP and subagent task cancellation, so packaged installs still need that runtime file for non-CLI task cancellation.
  - file: [src/tasks/task-registry.ts:1760](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/tasks/task-registry.ts#L1760)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)

A repo-wide search found no other reference that would force bundling of `src/tasks/task-registry-control.runtime.ts`.
  - file: [tsdown.config.ts:205](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/tsdown.config.ts#L205)

## Risks / Open Questions

- ACP and subagent task cancellation in packaged npm installs appears to remain broken until `task-registry-control.runtime` is emitted or the loader is rewritten.
- `openclaw tasks flow cancel` may also be affected anywhere it cascades into `cancelTaskById()` for ACP/subagent child tasks.
- There is no visible packaging guard on main for this runtime; current tests mock or override the control runtime instead of exercising the published tarball path.
