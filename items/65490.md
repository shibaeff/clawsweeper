## Summary

Keep open. Current `main` has operator-side diagnostics and better cron docs, but the reported execution-discipline/runtime gaps are not clearly resolved. The strongest concrete blocker is cron: `openclaw cron add` still exposes no `--dry-run`, and announce delivery still accepts an optional `--to` with no matching service-side validation for announce targets.

## Evidence

- **`cron add` still lacks a dry-run path and keeps `--to` optional:** `registerCronAddCommand` defines `openclaw cron add` with `--announce`, `--channel`, and optional `--to`, but there is no `--dry-run` option on the command.
  - file: [src/cli/cron-cli/register.cron-add.ts:69](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/cli/cron-cli/register.cron-add.ts#L69)

There is no equivalent announce-target presence/shape validation here.
