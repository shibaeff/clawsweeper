## Summary

Keep open. Current `main` still does not implement audio/file content blocks for `/v1/chat/completions`; the PR remains an unmerged gateway parity change, and the discussion includes substantive review concerns that require maintainer judgment rather than conservative auto-close.

There is no current-main handling for `input_audio`, `file`, or `video_url` on the chat-completions path.
