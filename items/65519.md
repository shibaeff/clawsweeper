## Summary

Current `main` already wires `sessions_spawn.model` and `agents.defaults.subagents.model`/per-agent `subagents.model` through subagent model selection, applies the chosen model to the child session before launch, and has regression coverage for nested provider/model refs. This issue reads as already implemented on `main`, but I could not determine a shipped release/landing commit from the available local history.

## Evidence

- **Subagent model precedence is implemented in current main:** `resolveSubagentSpawnModelSelection()` prefers an explicit `modelOverride`, then per-agent `subagents.model`, then the agent primary model, then global `agents.defaults.subagents.model`, then the runtime default.
  - file: [src/agents/model-selection.ts:251](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/model-selection.ts#L251)

- **Spawn plan pushes the selected model into the child-session patch:** `resolveSubagentModelAndThinkingPlan()` resolves the subagent model and includes it in `initialSessionPatch.model`, which is the payload later sent to `sessions.patch` before the run starts.
  - file: [src/agents/subagent-spawn-plan.ts:42](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/subagent-spawn-plan.ts#L42)

- **Spawn path fails on patch errors instead of silently falling back:** `spawnSubagentDirect()` applies `plan.initialSessionPatch` via `sessions.patch` before launch and returns an error immediately if that patch fails; it does not continue with a silent fallback after a failed model patch.
  - file: [src/agents/subagent-spawn.ts:716](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/subagent-spawn.ts#L716)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '716,810p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Gateway persists validated provider/model overrides for subagent sessions:** `applySessionsPatchToStore()` validates `patch.model` against the model catalog and writes the resolved provider/model override into the session entry, which is what runtime model resolution later reads.
  - file: [src/gateway/sessions-patch.ts:388](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/gateway/sessions-patch.ts#L388)

  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
