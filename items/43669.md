## Summary

Keep open. Current `main` acknowledges the same auto-retry/idle race, but only waits for idle during cleanup before flushing pending tool results. The run path still snapshots messages immediately after `activeSession.prompt()` returns, and `waitForAgentIdleBestEffort` remains private to the cleanup helper, so this PR's post-prompt fix is not implemented on `main`.
