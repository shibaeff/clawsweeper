## Summary

Current main already implements the reported exec-policy fix: unset gateway/node exec policy now resolves to host-approval defaults and ultimately `security=full`/`ask=off`, with regression coverage and release-note evidence that this shipped in 2026.4.2.

## Evidence

- **Gateway/node default policy is now full/off:** `resolveExecDefaults` loads `~/.openclaw/exec-approvals.json` defaults and falls back to `full` for non-sandbox hosts, so an unset `tools.exec.security` no longer hard-defaults gateway/node exec to `allowlist`.
  - file: [src/agents/exec-defaults.ts:116](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/exec-defaults.ts#L116)
- **Exec tool runtime uses approvals defaults before fallback:** The exec tool computes `configuredSecurity` from config defaults, then host approval defaults, then `full` for gateway hosts; `configuredAsk` likewise falls back to approval defaults before `off`.
  - file: [src/agents/bash-tools.exec.ts:1480](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/bash-tools.exec.ts#L1480)
- **Regression tests lock in the reported behavior:** Tests explicitly assert that unset gateway exec policy resolves to `security: "full"` and `ask: "off"`, and that exec-approvals defaults suppress gateway prompts.
  - file: [src/agents/exec-defaults.test.ts:87](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/exec-defaults.test.ts#L87)
- **Approval-default suppression is covered end-to-end:** The approval-id tests cover gateway exec using `exec-approvals.json` defaults (`security: "full"`, `ask: "off"`) to avoid approval gating when config is otherwise unset.
  - file: [src/agents/bash-tools.exec.approval-id.test.ts:648](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/bash-tools.exec.approval-id.test.ts#L648)
- **Release notes record this fix:** The changelog entry for 2026.4.2 says: "Exec defaults: make gateway/node host exec default to YOLO mode by requesting `security=full` with `ask=off`, and align host approval-file fallbacks plus docs/doctor reporting with that no-prompt default."
  - file: [CHANGELOG.md:1540](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/CHANGELOG.md#L1540)

## Close Comment

Closing from a Codex review because this is already implemented on current `main`. The current exec runtime no longer uses the old gateway-host fallback described here: `src/agents/exec-defaults.ts:116` now loads host approval defaults and falls back to `security=full` for non-sandbox hosts, and `src/agents/bash-tools.exec.ts:1480` applies the same precedence in the exec tool path. Regression coverage was added in `src/agents/exec-defaults.test.ts:87` and `src/agents/bash-tools.exec.approval-id.test.ts:648`, and the changelog records the fix in release `v2026.4.2` at `CHANGELOG.md:1540`. If someone still hits `exec denied: allowlist miss` on a current build, that would need a new issue with the exact effective `tools.exec.*` state, `~/.openclaw/exec-approvals.json` contents, and fresh logs, because the precedence bug reported here is no longer the current behavior.
