## Summary

Current main and the released `v2026.4.22` build already provide a visible Control UI password entry flow for gateway password auth, plus matching server hints and docs, so the reported missing-password UI is no longer an open bug on main.

## Close Comment

Closing this as a Codex review because the missing password-entry UI described here is already implemented on current `main` and in the released `v2026.4.22` build. The dashboard now renders an unauthenticated login gate before the main UI (`ui/src/ui/app-render.ts:663`) with a visible password field and connect button (`ui/src/ui/views/login-gate.ts:67`), and the Overview access/settings card also exposes `Password (not stored)` (`ui/src/ui/views/overview.ts:310`, `ui/src/i18n/locales/en.ts:195`). The gateway error text and docs also explicitly direct users to Control UI settings for password auth (`src/gateway/server/ws-connection/auth-messages.ts:26`, `docs/web/dashboard.md:40`, `docs/web/dashboard.md:90`). If this still reproduces on a newer build than `2026.3.8`, please open a new issue with screenshots plus the exact gateway auth mode and URL you opened.
