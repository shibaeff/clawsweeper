## Summary

Keep open. Current `main` still intentionally flattens remote markdown images to alt text in the Control UI renderer, and the Control UI CSP/docs explicitly block remote `http(s)` image fetches. This PR is therefore neither implemented on `main` nor obsolete; it also appears incomplete because changing the renderer alone would still leave browser-side blocking in place and would need explicit product/security judgment rather than conservative auto-close.

## Evidence

- **Current renderer only allows data URI images:** `md.renderer.rules.image` returns alt text unless the image `src` matches `INLINE_DATA_IMAGE_RE`, which is `data:image/...;base64`. No `http`/`https` allowlist exists on `main`.
