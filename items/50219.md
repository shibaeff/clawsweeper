## Summary

Keep open. Current main has a generic reconnect loop for node-host Gateway WebSocket clients, but I did not find strong evidence that the specific Cloudflare-502 stuck-retry report is fixed or obsolete. The reconnect path recreates the socket and applies bounded exponential backoff, yet there is no 502-specific regression test, no issue-linked changelog entry, and no reproducible proof that repeated upgrade-time 502s now recover without manual `openclaw node restart`.

## Evidence

  - file: [src/node-host/runner.ts:179](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/node-host/runner.ts#L179)

  - file: [src/gateway/client.ts:217](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/client.ts#L217)
