## Summary

Keep open. Current `main` still does not attach quoted Feishu image/file media to the agent payload, so the reported gap is not already fixed. At the same time, the PR as submitted is not safe to auto-close as implemented: current `main` still contains ACP conversation binding, thread bootstrap/context, structured card handling, and outbound identity flows that the PR discussion and diff context flagged as regressions.

## Evidence

- **Current main only downloads media from the triggering message:** The inbound Feishu handler builds `mediaPayload` solely from `resolveFeishuMediaList({ messageId: ctx.messageId, messageType: event.message.message_type, content: event.message.content, ... })`. When `parentId` is present it separately fetches `quotedMessageInfo`, but only copies `quotedMessageInfo.content` into `quotedContent`/`ReplyToBody`; no quoted media is merged into the agent media payload.
