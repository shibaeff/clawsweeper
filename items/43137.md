## Summary

Keep open. Current `main` still does not implement `heartbeat.enabled`, so the reported config-validation bug is not fixed on `main`. The proposed PR also needs more work before merge: current scheduler semantics would require disabled heartbeats to be excluded from scheduling or have their due time advanced, otherwise `skipped/disabled` runs can stay immediately due.

## Evidence

A config block containing `heartbeat.enabled` would still hit strict-key rejection on current `main`.
