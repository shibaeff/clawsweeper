## Summary

Keep open. Current `main` still does not implement `agents.defaults.heartbeat.enabled`, so the underlying bug/request is not already fixed, and this PR should not be auto-closed. The submitted patch also has unresolved correctness risks: it adds the config toggle without updating the scheduler path that builds active heartbeat agents, and it bundles a separate response-prefix templating change with a user-visible inconsistency already called out in review.
