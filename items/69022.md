## Summary

Keep open. Current `main` still does not wire `resolveHumanDelayConfig` into Telegram block-stream dispatch, so the reported bug remains plausible and the PR is still actionable. The latest shipped tag shown in this checkout (`v2026.4.23`) also lacks the wiring. Maintainer discussion on the PR explicitly calls this a safe small merge after the unrelated churn was removed, and the related PR `#69194` was closed unmerged rather than landing the fix.
