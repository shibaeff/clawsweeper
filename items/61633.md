## Summary

Current `main` contains meaningful mitigations for this report: cron now clears stale `runningAtMs` markers, blocks overlapping runs, prunes expired `cron:...:run:...` session entries, and applies generic session-store pruning/capping without status-based exemptions. But the crash-loop part is not fully closed: session-store loading still does a full `readFileSync` + `JSON.parse` of `sessions.json` before any maintenance runs, so an already oversized file can still exhaust memory on startup. That leaves the reported unbootable-gateway/OOM recovery path insufficiently resolved for a conservative close.

## Evidence
