## Summary

Keep open. Current main captures Telegram forum topic names, but it does not wire them into session metadata used by the session list/dropdown. The requested human-readable per-topic session names are not implemented on main.

## Evidence

- **Telegram runtime captures topic names only into inbound context:** Telegram message handling reads `forum_topic_created` / edited / closed metadata, caches the topic name, and resolves `topicName` for forum messages.
  - file: [extensions/telegram/src/bot-message-context.ts:165](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/extensions/telegram/src/bot-message-context.ts#L165)
  - command: `nl -ba extensions/telegram/src/bot-message-context.ts | sed -n '150,210p'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Topic name is stored as `TopicName`, not as a session display label:** The finalized Telegram inbound context includes `TopicName`, but this code path does not set `ThreadLabel` or `displayName` for the session entry.
  - file: [extensions/telegram/src/bot-message-context.session.ts:382](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/extensions/telegram/src/bot-message-context.session.ts#L382)
  - command: `nl -ba extensions/telegram/src/bot-message-context.session.ts | sed -n '370,390p'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Session display names only promote `ThreadLabel`:** Session persistence updates `sessionEntry.displayName` only when `ctx.ThreadLabel` is present.
  - file: [src/auto-reply/reply/session.ts:693](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/auto-reply/reply/session.ts#L693)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '688,699p'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Telegram plugin never sets `ThreadLabel` in production code:** A repo search found no non-test `ThreadLabel` assignment or usage in `extensions/telegram/src`, so the Telegram topic name never reaches the session display-name hook.
  - command: `rg -n "ThreadLabel" extensions/telegram/src -g '!**/*.test.ts'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Gateway session rows do not consume `TopicName`:** Session rows derive `displayName` from `entry.displayName`, `buildGroupDisplayName(...)`, `entry.label`, or `origin.label`. None of these sources read Telegram `TopicName`.
  - file: [src/gateway/session-utils.ts:1256](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/gateway/session-utils.ts#L1256)

  - file: [src/config/sessions/group.ts:45](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/config/sessions/group.ts#L45)

Since Telegram topic names never populate those row fields, the requested topic-specific dropdown label is still missing.
