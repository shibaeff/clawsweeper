## Summary

Current OpenClaw already provides a stable transcript-mapping path for subagent sessions: `sessions_list` returns `transcriptPath` from persisted session metadata, `sessions.resolve` resolves `sessionId` back to the canonical session key, and `/subagents info` exposes the child session key, session ID, and transcript path directly.

## Evidence

- **`sessions_list` returns `transcriptPath` from stable session metadata:** The tool reads `sessionId` and optional `sessionFile`, resolves the agent-specific path, and adds `transcriptPath` to each returned row.
  - file: [src/agents/tools/sessions-list-tool.ts:201](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/tools/sessions-list-tool.ts#L201)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Gateway session inventory includes persistent `sessionId`:** Session rows emitted from the gateway include `sessionId`, which is the stable identifier the tool uses for transcript lookup.
  - file: [src/gateway/session-utils.ts:1401](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/gateway/session-utils.ts#L1401)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **`sessions.resolve` supports `sessionId` lookup:** The resolver accepts `sessionId`, searches visible sessions for that exact ID, and returns the canonical session key without heuristic filename mapping.
  - file: [src/gateway/sessions-resolve.ts:150](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/gateway/sessions-resolve.ts#L150)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Subagent inspection already exposes transcript identity:** `/subagents info` loads the child session entry by `childSessionKey` and prints both `SessionId` and `Transcript` from persisted session state.
  - file: [src/auto-reply/reply/commands-subagents/action-info.ts:107](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/auto-reply/reply/commands-subagents/action-info.ts#L107)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Regression coverage exists for transcript path mapping:** The tests assert that `sessions_list` returns a concrete `transcriptPath` for a session row, including multi-store layouts.
  - file: [src/agents/openclaw-tools.sessions.test.ts:446](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/openclaw-tools.sessions.test.ts#L446)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Same implementation is present in the latest release commit:** Verified via `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/tools/sessions-list-tool.ts` and `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/auto-reply/reply/commands-subagents/action-info.ts` that release `v2026.4.23` already contains the transcript-path/session-id exposure.
