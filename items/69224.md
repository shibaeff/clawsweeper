## Summary

Keep open. Current `main` still copies `patches/` only into the build stage, but the runtime image still ships `package.json` with a `pnpm.patchedDependencies` entry pointing at `patches/@whiskeysockets__baileys@7.0.0-rc.9.patch`. The latest shipped release `v2026.4.23` still has the same mismatch, and the public Docker docs still describe downstream Dockerfiles that re-run `pnpm install`, so the report remains current and actionable.
