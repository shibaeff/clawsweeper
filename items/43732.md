## Summary

Current `main` already implements the requested subagent completion routing behavior, including the review-driven guard that preserves stored thread routes for the same destination but strips stale `threadId` values when the requester target changes. The same implementation is present in released tag `v2026.4.23`, so this unmerged PR is redundant.

## Evidence
