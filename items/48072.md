## Summary

Keep open. Current `main` still does not pre-create `/home/node/.openclaw` in the runtime image, so the PR's Dockerfile change is not already implemented. The repo still mounts `/home/node/.openclaw` in Compose and still documents manual EACCES remediation, while `scripts/docker/setup.sh` only provides a script-level ownership workaround for its managed flow. The PR discussion also shows the author already addressed the `0700` security concern in the patch, so this still looks like potentially useful work rather than something to auto-close.

## Evidence

This means the proposed fix is not present on `main`.
