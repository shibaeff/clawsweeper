## Summary

Current `main` already resolves `github-copilot` as a valid memory embedding provider for the standalone memory CLI path, and the fix is already shipped in `v2026.4.23`. The report matches older `2026.4.15` behavior, not current OpenClaw.

## Evidence

- **GitHub Copilot manifest declares a memory embedding provider:** The bundled GitHub Copilot plugin manifest explicitly declares `contracts.memoryEmbeddingProviders: ["github-copilot"]`, so capability-based provider resolution can discover it even when the live registry is cold.
  - file: [extensions/github-copilot/openclaw.plugin.json:5](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/github-copilot/openclaw.plugin.json#L5)
  - command: `sed -n '1,12p' extensions/github-copilot/openclaw.plugin.json`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **GitHub Copilot plugin registers its embedding adapter:** The plugin runtime calls `api.registerMemoryEmbeddingProvider(githubCopilotMemoryEmbeddingProviderAdapter)`, which is the concrete adapter that serves `github-copilot` memory embeddings.
  - file: [extensions/github-copilot/index.ts:88](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/github-copilot/index.ts#L88)
  - command: `sed -n '1,180p' extensions/github-copilot/index.ts`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Memory embedding lookup now falls back to declared plugin capabilities:** `getMemoryEmbeddingProvider()` and `listMemoryEmbeddingProviders()` merge registered adapters with capability-provider fallback resolution for `memoryEmbeddingProviders`, which is the path that prevents `Unknown memory embedding provider: github-copilot` when plugin runtime state is split or cold.
  - file: [src/plugins/memory-embedding-provider-runtime.ts:17](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/plugins/memory-embedding-provider-runtime.ts#L17)

- **Standalone memory CLI path is covered by integration test:** The QA integration test exercises the plugin-owned `openclaw memory status` command with staged CLI metadata and expects it to run successfully, covering the standalone memory-command startup path discussed on the issue.
