## Summary

Keep open. Current main did adopt one related mitigation by sending normal Feishu text replies as `post` messages, but the streaming/card path named in this issue still feeds raw markdown straight into Feishu card content with no escaping or sanitization for literal backticks or angle brackets, and there is no regression test for the reported truncation case.

## Evidence

- **Normal non-card replies now use Feishu `post` payloads:** `sendMessageFeishu` builds a `msg_type: "post"` payload with a single `tag: "md"` element. That addresses plain reply delivery, but it is separate from the streaming card path discussed in the issue.
  - file: [extensions/feishu/src/send.ts:432](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/feishu/src/send.ts#L432)
