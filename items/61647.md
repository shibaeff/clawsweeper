## Summary

Keep open. Current main does not implement the requested broader `allow-always` behavior on Windows: direct exec approvals are still persisted with Windows-only `argPattern` data derived from the full argv, and allowlist matching on Windows requires that argv pattern to match. That means approving `python.exe` (or similar) still does not generally cover later invocations with different arguments. Main has shipped a narrower fix for shell-wrapper commands, but not the broader executable-level trust requested here.
