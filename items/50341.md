## Summary

Current main already implements bounded recovery for the reported "partial output then silent hang" path: GPT-5 OpenAI embedded runs now auto-retry plan-only, reasoning-only, and empty-response turns, then surface an explicit blocked/incomplete-turn error instead of waiting for a user nudge.

## Close Comment

Closing from a Codex review because current `main` already implements this recovery path. For `openai/gpt-5*`, `src/agents/execution-contract.ts:71` auto-activates the stricter embedded contract, and `src/agents/pi-embedded-runner/run.ts:1827` now retries plan-only, reasoning-only, and empty-response turns instead of waiting for a manual “wake up”; if the retry still cannot produce a visible answer, it returns an explicit blocked/incomplete-turn error rather than going silent. The regression coverage in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts:158` and `:247` locks that in for unconfigured `openai/gpt-5.4` runs, and `CHANGELOG.md:733` plus `CHANGELOG.md:884`/`505` show the shipped fixes across `v2026.4.10`, `v2026.4.15`, and `v2026.4.22`. If this still reproduces on a current build, please open a fresh issue with the exact provider/model/channel and logs from the failing turn so it can be scoped as a new regression.
