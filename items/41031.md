## Summary

Keep open. Current `main` still resolves heartbeat reply context from the main session or an explicit `heartbeat.session`; it does not derive the session key from an explicit `heartbeat.target` + `heartbeat.to`, so the PR appears to address a real unfixed bug rather than duplicate shipped behavior.

## Evidence

- **Heartbeat session resolution still falls back to main unless `heartbeat.session` is set:** `resolveHeartbeatSession` handles global scope, forced session keys, and explicit `heartbeat.session`, then returns the main session when `heartbeat.session` is empty. There is no branch for explicit `heartbeat.target`/`heartbeat.to`.
