## Summary

Current main already supports Feishu single-message delivery when `channels.feishu.streaming` is disabled, and the streaming path updates one interactive card in place instead of emitting separate bubbles. Feishu message editing is also implemented and shipped in `v2026.4.22`.

## Evidence

- **Docs: single-message mode exists:** Feishu docs explicitly state that streaming uses interactive-card updates and that setting `channels.feishu.streaming: false` sends the complete reply in one message.
  - file: [docs/channels/feishu.md:238](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/channels/feishu.md#L238)

- **Dispatcher honors streaming=false:** Feishu reply dispatch enables streaming only when `account.config?.streaming !== false`; otherwise it falls back to normal chunked/final delivery instead of card streaming.
  - file: [extensions/feishu/src/reply-dispatcher.ts:190](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/feishu/src/reply-dispatcher.ts#L190)

## Close Comment

Codex review on current `main` and the shipped `v2026.4.22` release found that this behavior is already implemented. `docs/channels/feishu.md:238` documents that Feishu streaming uses interactive-card updates, and `docs/channels/feishu.md:251` documents `channels.feishu.streaming: false` to send the complete reply in one message. The runtime path updates a single card in place rather than spraying separate bubbles (`extensions/feishu/src/reply-dispatcher.ts:190`, `extensions/feishu/src/streaming-card.ts:248`, `extensions/feishu/src/streaming-card.ts:303`, `extensions/feishu/src/streaming-card.ts:419`), and explicit Feishu message editing is implemented in `extensions/feishu/src/send.ts:512`. Because current main and the latest shipped release already cover the requested single-bubble/editable behavior, I’m closing this as implemented. If you still reproduce 30+ separate Feishu messages on `v2026.4.22` or newer, please open a new issue with the exact OpenClaw version and your `channels.feishu` settings (`streaming`, `renderMode`, `textChunkLimit`) so maintainers can verify the specific config path.
