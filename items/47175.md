## Summary

Keep open. Current `main` has related loop-detection safeguards, but it does not implement the requested per-turn cap on consecutive tool calls or the requested forced assistant-response behavior. The only exposed config here is `tools.loopDetection`, which covers repeated/no-progress patterns, not a generic `maxToolCallsPerTurn` limit for varied tool bursts.

## Evidence

- **No `maxToolCallsPerTurn` in runtime config types:** `ToolLoopDetectionConfig` only exposes `enabled`, `historySize`, `warningThreshold`, `unknownToolThreshold`, `criticalThreshold`, `globalCircuitBreakerThreshold`, and detector toggles; there is no per-turn max-tool-call field.
