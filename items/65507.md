## Summary

Current `main` now defers context-engine turn maintenance off the hot session lane, waits for the lane to go idle before running deferred work, and regression tests cover the next-turn latency behavior described in the issue. The fix appears to have landed after `v2026.4.23`, so it is implemented on `main` but not in the latest provided release tag.

## Evidence
