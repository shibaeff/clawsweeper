## Summary

Current `main` already honors isolated cron `payload.model` ahead of any stored cron-session override, carries switched live selection correctly across retries, and has shipped cron staggering controls for bursty schedules. The reported behavior is implemented on `main` and present in release `v2026.4.23`.

## Evidence

- **Payload model now takes precedence over stored cron session override:** `resolveCronModelSelection` resolves `payload.model` first and only falls back to `sessionEntry.modelOverride` when no payload override was supplied.
