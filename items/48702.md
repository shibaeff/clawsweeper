## Summary

Keep open. Current `main` still has the UX gap this PR addresses: `/status` omits reasoning entirely when it is off, surfaces raw reasoning values when it is on, and the runtime prompt still says `/status` only shows Reasoning when enabled. The underlying implementation also moved after this PR was opened, so the branch is now stale but the request itself is still valid.
