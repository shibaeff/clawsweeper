## Summary

Keep open. Current `main` still does not attach `inputProvenance` when `spawnSubagentDirect()` starts the child `agent` run, even though the agent RPC supports provenance and other inter-session paths already set it. The PR is addressing a real current gap rather than something already implemented or obsolete.

## Evidence

- **Current spawn path omits provenance:** The `spawnSubagentDirect()` call into the gateway `agent` method passes message, session, delivery, lane, prompt, thinking, timeout, label, and spawned metadata, but no `inputProvenance` field.
  - file: [src/agents/subagent-spawn.ts:969](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/subagent-spawn.ts#L969)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '960,995p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Provenance changes gateway behavior:** `server-methods/agent.ts` normalizes `inputProvenance` and only treats a run as a normal tracked task when `inputProvenance.kind !== "inter_session"`, so omitting provenance is behaviorally meaningful.
  - file: [src/gateway/server-methods/agent.ts:244](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/gateway/server-methods/agent.ts#L244)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '232,260p'`

- **Analogous inter-session path already sets provenance:** `sessions_send` explicitly attaches `inputProvenance` with `kind: "inter_session"`, the requester session/channel, and `sourceTool: "sessions_send"` before calling `agent`.
  - file: [src/agents/tools/sessions-send-tool.ts:275](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/tools/sessions-send-tool.ts#L275)
