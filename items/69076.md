## Summary

Keep open. Current `main` still matches the bug described by the PR: manual-cancel exits are funneled into the generic failed path, notify-on-exit has no killed-state guard, process `kill`/`remove` return failed tool status on successful termination requests, and the Telegram sent-message cache still persists timestamp-only entries with no metadata support.

## Evidence

- **Exec runtime still classifies manual cancel as failed:** `buildExecExitOutcome()` only returns `completed` for normal exits and otherwise falls through to `failed`; there is no special case for `exit.reason === "manual-cancel"` on current `main`. This matches the PR's reported bug rather than proving it obsolete.
