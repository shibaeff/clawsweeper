## Summary

Keep open: current `main` does not implement this PR's Ctrl+C abort behavior. The TUI still routes Ctrl+C only through clear/warn/exit logic, `abortActive()` still ignores stale/non-aborted responses, and aborted runs still do not finalize partial assistant text.

## Evidence

- **Ctrl+C resolver still lacks abort path:** `resolveCtrlCAction` only returns `"clear" | "warn" | "exit"`; there is no active-run/abort branch on current `main`. Tests alongside it only cover clear/warn/exit.
  - file: [src/tui/tui.ts:259](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/tui/tui.ts#L259)
  - command: `nl -ba src/tui/tui.ts | sed -n '258,292p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Ctrl+C handler still ignores active run state:** `handleCtrlC()` passes only `hasInput`, `now`, and `lastCtrlCAt` into `resolveCtrlCAction`, then clears input, exits, or warns. It does not prioritize aborting an active run.
  - file: [src/tui/tui.ts:972](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/tui/tui.ts#L972)
  - command: `nl -ba src/tui/tui.ts | sed -n '966,995p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Abort RPC result still not handled conservatively:** `abortActive()` awaits `client.abortChat(...)` and unconditionally sets status to `"aborted"` on success. It does not inspect the returned `{ aborted: boolean }` or clear stale local run state on failure.
  - file: [src/tui/tui-session-actions.ts:386](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/tui/tui-session-actions.ts#L386)
  - command: `nl -ba src/tui/tui-session-actions.ts | sed -n '386,410p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Gateway can legitimately report `aborted: false`:** Current gateway `chat.abort` returns `{ ok: true, aborted: false, runIds: [] }` when the run is no longer active, so the missing client-side handling is still a real gap.
  - file: [src/gateway/server-methods/chat.ts:1993](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/gateway/server-methods/chat.ts#L1993)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '1956,2018p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)

  - file: [src/tui/tui-event-handlers.ts:382](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/tui/tui-event-handlers.ts#L382)
