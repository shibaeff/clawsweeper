## Summary

Current `main` does not pass through Feishu card-action fields such as `form_value`, `input_value`, `name`, `option`, or `options`. The parser and event type only retain `action.value`/`action.tag`, and the legacy fallback serializes only `action.value`, so the reported behavior is still present.

## Evidence

- **Card-action payload parser drops extra action fields:** `parseFeishuCardActionEventPayload()` reads `action.value` and `action.tag` only, then returns an event shape containing just those fields. `action.form_value`, `action.input_value`, `action.name`, `action.option`, and `action.options` are not parsed or preserved.
  - file: [extensions/feishu/src/monitor.account.ts:243](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/src/monitor.account.ts#L243)
  - command: `nl -ba extensions/feishu/src/monitor.account.ts | sed -n '232,292p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Runtime event type has no fields for form/input metadata:** `FeishuCardActionEvent` defines `action` as only `{ value: Record<string, unknown>; tag: string; }`, so current main has no typed path for the requested callback fields.
  - file: [extensions/feishu/src/card-action.ts:14](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/src/card-action.ts#L14)

- **Legacy fallback only forwards action.value:** `buildFeishuCardActionTextFallback()` only inspects `event.action.value`; when it is an object, it stringifies that object alone. Any sibling Feishu action fields are therefore omitted from the synthetic message content.
