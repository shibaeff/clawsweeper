## Summary

Current `main` does not contain this change, and the PR remains plausible but contested: `src/gateway/probe.ts:158` still strips loopback device identity whenever both token and password are absent, while current callers/tests intentionally produce empty auth objects (`{}`) for unresolved or credential-less probe paths. Because `GatewayClient.selectConnectAuth` falls back to stored device-token auth when no explicit token/password is present, the PR's `opts.auth === undefined` change would also change anonymous/diagnostic probe semantics. This needs maintainer judgment, not cleanup-close.
