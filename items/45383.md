## Summary

Keep open. Current `main` does not implement this PR's inbound-media prompt-injection guard: the read pipeline still returns file content after adaptive paging, image normalization, and image sanitization only, with no `media/inbound` path classification or untrusted-text wrapping. The sanitization utility exists on `main`, but it is not wired into `src/agents/pi-tools.read.ts`, and the dedicated inbound-guard test file from this PR is also absent.
