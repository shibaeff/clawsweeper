## Summary

Keep open. Current `main` fixed the warning/classification side of this report, but it did not remove the upstream provider gating that prevents `apply_patch` from being created outside exact `openai` / `openai-codex` providers. The issue’s policy-pipeline diagnosis is obsolete, yet the routed-provider failure mode described in the thread is not strongly shown to be resolved.
