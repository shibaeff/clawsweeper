## Summary

Current `main` no longer shows a live clock in the agent's `Current Date & Time` context. The system prompt now renders only the timezone and explicitly directs agents to use `session_status` for current time, which matches the requested cleanup of the incorrect offset-bearing prompt time.

## Close Comment

Closing as implemented on current `main` after Codex review. The agent system prompt no longer includes a live `Current Date & Time` clock; it now shows only `Time zone: ...` in `src/agents/system-prompt.ts:248`, with a regression test in `src/agents/system-prompt.test.ts:497` asserting that the formatted date/time is not present. The docs also reflect this in `docs/date-time.md:65`, and `CHANGELOG.md` records the shipped change in release `2026.1.23` (`Agents: keep system prompt time zone-only and move current time to session_status for better cache hits.`). Since the reported incorrect prompt time is no longer part of the agent context on `main`, this issue is resolved. If you can still reproduce a wrong clock on a current release, please open a new issue with the exact client surface, version, and a fresh screenshot/log snippet showing where that clock appears.
