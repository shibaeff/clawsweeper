## Summary

Current `main` already carries the Slack context plumbing this issue asked for, and the same implementation is present in release `v2026.4.22`: inbound Slack messages preserve `MessageThreadId`/`ReplyToId`, set `NativeChannelId` from the Slack event payload, and Slack's tool-context builder converts that into `currentChannelId`/`currentThreadTs` for message actions like reactions and uploads.

## Evidence

The Slack channel wires that tool context into runtime message actions (`extensions/slack/src/channel.ts:603`), and regression coverage exists in `extensions/slack/src/monitor/message-handler/prepare.test.ts:379`, `extensions/slack/src/monitor/message-handler/prepare.test.ts:405`, and `extensions/slack/src/threading-tool-context.test.ts:157`. On that basis this report looks fixed already.
