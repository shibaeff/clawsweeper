## Summary

Current `main` does not contain the canonical-path containment logic or regression coverage described in #68597. Builtin `memory_get` and QMD reads still rely on lexical root checks plus `lstat` of the final file, which does not block an intermediate directory symlink escape, so there is not strong evidence to close this PR.

## Evidence

- **Builtin memory reads still use lexical extra-path checks:** `readMemoryFile()` marks extra paths as allowed when `absPath` is string-prefixed by an allowed directory, then reads `absPath` directly. There is no `realpath()` containment check before `fs.readFile()`.
  - file: [src/memory-host-sdk/host/read-file.ts:33](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/memory-host-sdk/host/read-file.ts#L33)

  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Shared file guard only rejects direct symlinks:** `statRegularFile()` uses `fs.lstat()` and throws only when the final path itself is a symlink or non-file. That does not catch a regular file reached through an intermediate directory symlink.
