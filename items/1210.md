## Summary

Current `main` still handles inline transcript images with replay-time sanitization and delayed pruning, not storage-level media references. The code and docs both assume session JSONL can contain `{type:"image", data:"..."}` blocks, so issue #1210 is not implemented on `main` and should stay open.

## Evidence

- **Docs say transcript image hygiene is in-memory only:** `docs/reference/transcript-hygiene.md` explicitly says transcript hygiene fixes are in-memory and do not rewrite the stored JSONL transcript on disk; the image rule is downscale/recompress of base64 payloads, not media-reference storage.
