## Summary

Current `main` already covers this request: OpenClaw now has a model compat flag for `supportsPromptCacheKey`, and the OpenAI Responses payload policy strips `prompt_cache_key`/`prompt_cache_retention` on proxy-like local/custom endpoints by default unless the model explicitly opts back in. ([raw.githubusercontent.com](https://raw.githubusercontent.com/openclaw/openclaw/main/src/config/types.models.ts))

## Evidence

- **Config surface exists:** `src/config/types.models.ts` includes `compat.supportsPromptCacheKey?: boolean` in `ModelCompatConfig`, so there is now an explicit model-level compat control for this behavior. ([raw.githubusercontent.com](https://raw.githubusercontent.com/openclaw/openclaw/main/src/config/types.models.ts))
  - file: `src/config/types.models.ts`
  - sha: `db17b8cc4a31dccbb5f451bbb4c406d08c87e5b9`
- **Runtime strips prompt-cache fields:** `src/agents/provider-attribution.ts` computes `shouldStripResponsesPromptCache` from `compat.supportsPromptCacheKey` plus endpoint class, preserving default stripping on proxy-like OpenAI Responses endpoints; `src/agents/openai-responses-payload-policy.ts` then deletes `prompt_cache_key` and `prompt_cache_retention` when that policy is active. ([raw.githubusercontent.com](https://raw.githubusercontent.com/openclaw/openclaw/main/src/agents/provider-attribution.ts))
