## Summary

Current `main` has generic crash logging and an optional bundled `diagnostics-otel` plugin, but it does not implement native Sentry DSN configuration or startup-time Sentry wiring for gateway/agent processes. The request is recent, coherent, and still represents a real feature gap, so it should stay open.

## Evidence

- **Config surface is OTEL-only:** The validated config schema exposes `diagnostics.otel` fields (`enabled`, `endpoint`, `protocol`, `headers`, `serviceName`, `traces`, `metrics`, `logs`, `sampleRate`, `flushIntervalMs`) and no `observability.sentry` or similar Sentry/DSN config.

  - file: [docs/gateway/configuration-reference.md:788](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/gateway/configuration-reference.md#L788)
