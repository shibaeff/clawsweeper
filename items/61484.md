## Summary

Keep open. Current `main` still implements `sessions_send` as a text-only agent tool, so the requested inter-agent media/file path support is not present, even though lower-level gateway RPCs already have attachment plumbing.

## Evidence

- **`sessions_send` tool schema is text-only:** The tool only accepts `sessionKey`/`label`/`agentId`, required `message`, and optional `timeoutSeconds`; there is no media or attachments parameter.
  - file: [src/agents/tools/sessions-send-tool.ts:37](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/tools/sessions-send-tool.ts#L37)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '30,90p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **`sessions_send` forwards only text into the nested agent run:** When the tool builds `sendParams` for the `agent` gateway call, it passes `message`, `sessionKey`, delivery metadata, and provenance, but no `attachments`, `mediaUrl`, or `mediaUrls` field.
  - file: [src/agents/tools/sessions-send-tool.ts:275](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/tools/sessions-send-tool.ts#L275)

- **User-facing docs still describe `sessions_send` as message-only:** The session tools docs say `sessions_send` 'delivers a message to another session and optionally waits for the response' and document only timeout behavior and reply-back loops, with no attachment/media contract.
  - file: [docs/concepts/session-tool.md:66](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/concepts/session-tool.md#L66)

  - file: [src/gateway/protocol/schema/sessions.ts:97](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/gateway/protocol/schema/sessions.ts#L97)
