## Summary

Keep open. Current `main` still updates stable/beta git installs by checking out the release tag directly in the live checkout and then running `install`/`build`/`ui:build` there. Only the `dev` git-update path gets a disposable preflight worktree with cleanup. The docs also describe `/usr/lib/node_modules/openclaw` support for read-only global npm installs, not for mutable git checkouts in that path, so the reported in-place git-update failure is still plausible and not clearly implemented away on `main`.

## Evidence

- **Stable git updates are still in-place:** For git installs on `stable`/`beta`, the updater resolves a release tag, runs `git checkout --detach <tag>` in the live repo, then runs dependency install, build, and UI build in that same checkout. There is no preflight worktree or stale-artifact cleanup on this path.
