## Summary

Keep open. Current `main` does not already contain the PR’s core fixes for streaming-layout stability and session-scoped agent event handling. The shared chat UI still reparses assistant segments with fresh UUID identities on every update, still auto-scrolls on every streaming token without the PR’s debounce, and still filters agent stream events by `runId == sessionId` instead of the session-key-aware flow described in the PR.

## Evidence

- **Streaming segment IDs are still unstable on main:** `AssistantTextSegment` still uses `UUID()` for `id`, and `appendSegment` creates fresh segment objects on every parse. `ChatAssistantTextBody` renders those segments in `ForEach`, so repeated streaming reparses still change row identity and can churn layout.

  - command: `nl -ba apps/shared/OpenClawKit/Sources/OpenClawChatUI/AssistantTextParser.swift | sed -n '1,220p'`
- **Streaming bubble still re-renders on every token without PR debounce:** `OpenClawChatView` still animates directly on every `streamingAssistantText` change. The PR’s `pendingStreamingScrollTask` debounce/cancel path is not present.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift:181](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatView.swift#L181)
