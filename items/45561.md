## Summary

Gemini Live end-to-end audio support is not bundled on current main, but current main already exposes the intended implementation seam as an external provider plugin. Voice-call resolves generic realtime voice providers from plugin config, the public Plugin SDK defines `registerRealtimeVoiceProvider`, and repo policy says optional capabilities should usually ship as plugins maintained outside core/this repo.

## Risks / Open Questions

- If maintainers decide Gemini Live should ship as an official bundled Google capability rather than a community/provider plugin, closing as `clawhub` would defer that product decision.
- An external Gemini Live plugin may still discover missing ergonomics or helper gaps, but current main already provides the core realtime-voice registration seam.

## Close Comment

Closing after a Codex review because this request fits the existing plugin path better than OpenClaw core scope. Current main already exposes the needed seam for a provider plugin: `docs/plugins/building-plugins.md:11` says plugins should ship via ClawHub or npm rather than being added to this repo by default, `docs/plugins/sdk-provider-plugins.md:483` documents `api.registerRealtimeVoiceProvider(...)`, and `extensions/voice-call/src/runtime.ts:193` shows the voice-call plugin resolves a generic realtime voice provider from config instead of hardcoding vendors. The public contract is in `src/plugins/types.ts:1657`. The bundled Google plugin does not currently register realtime voice (`extensions/google/index.ts:147`), so a Gemini Live integration is best pursued as a ClawHub/community provider plugin rather than a new open core issue.
