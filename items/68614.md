## Summary

Keep open. Current `main` still does not implement the Google Chat deferred-response timeout/fallback behavior, and the PR discussion shows a remaining edge case: the new timer is only cleared from the delivery callback, but OpenClaw has legitimate reply-dispatch paths that complete without ever calling `deliver`.

## Evidence

- **Current main lacks the proposed timeout/fallback path:** `processMessageWithPipeline` still sends a typing message and then waits for `dispatchReplyWithBufferedBlockDispatcher`; there is no deferred-response timer, no fallback "still working" send, and no alternate post-timeout delivery path in `main`. A repo search for `deferred`, `still working`, and Google Chat timeout handling found nothing in the extension beyond webhook pre-auth timeout code.
  - file: [extensions/googlechat/src/monitor.ts:287](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/googlechat/src/monitor.ts#L287)
  - command: `nl -ba extensions/googlechat/src/monitor.ts | sed -n '250,315p'; rg -n "deferred|25-second|still working|isDeferredResponseExpired|timeout" extensions/googlechat/src -g '*.ts'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **OpenClaw reply dispatch can finish without invoking channel deliver:** `dispatch-from-config` has normal terminal branches that return `{ queuedFinal: false }` without any channel delivery callback, including plugin-bound `handled`, `declined`, and `error` outcomes. That means a timeout started in the Google Chat monitor must be cleared on function exit, not only inside `onDeliver`.

## Risks / Open Questions

- The underlying Google Chat long-turn/deferred-response problem is still present on current `main`; the latest release evidence in this checkout does not show an alternative fix shipped.
- Merging the PR as discussed without clearing the timer in a `finally`-style cleanup risks false "still working" follow-up messages on no-delivery control-flow paths.
- There is no current Google Chat test coverage in this checkout for deferred timeout/fallback behavior, so regressions around message-mode defaults, reaction fallback, and no-deliver exits would be easy to miss.
