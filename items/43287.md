## Summary

Current `main` already routes node-targeted browser tool calls through the node browser proxy and shared browser-control route stack, and the legacy Chrome extension relay path referenced in the report has since been removed in favor of `existing-session`/`user` attach flows.

## Close Comment

Codex review: closing as implemented on current `main`. The node-targeted browser tool path now routes through `node.invoke` with `command: "browser.proxy"` (`extensions/browser/src/browser-tool.ts:276`), and the node host handles that by starting the local browser-control service and dispatching through the shared browser route stack (`extensions/browser/src/node-host/invoke-browser.ts:62` and `extensions/browser/src/node-host/invoke-browser.ts:275`). The legacy Chrome extension relay path from this report is also no longer part of the supported runtime: doctor migrates `driver: "extension"` to `"existing-session"` and removes `browser.relayBindHost` (`docs/gateway/doctor.md:190`), and the old relay lifecycle hook is now an intentional no-op (`extensions/browser/src/browser/server-lifecycle.ts:9`). This was a Codex read-only review against `main` at `41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00`.
