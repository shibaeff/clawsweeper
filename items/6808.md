## Summary

Current `main` already covers web-based cron editing and run history, but I did not find cron pipeline orchestration on `main`; the request is only partially implemented, so this issue should stay open. ([raw.githubusercontent.com](https://raw.githubusercontent.com/openclaw/openclaw/main/docs/gateway/protocol.md))

## Evidence

- **Control UI already has cron edit + history:** `docs/web/control-ui.md:8` says the Control UI supports cron jobs `list/add/edit/run/enable/disable + run history`, and `docs/web/control-ui.md:9` adds advanced edit controls for existing jobs. 
  - file: `docs/web/control-ui.md:8`
  - sha: `db17b8cc4a31dccbb5f451bbb4c406d08c87e5b9`
- **Current cron surface includes run history tooling:** `docs/cli/cron.md:3` documents `openclaw cron runs --id ...` to follow queued/manual executions, and `docs/cli/cron.md:12` says `cron runs` includes delivery diagnostics. 
  - file: `docs/cli/cron.md:3`
  - sha: `db17b8cc4a31dccbb5f451bbb4c406d08c87e5b9`
- **Cron schedules are still time-based only:** `docs/automation/cron-jobs.md:5` enumerates only three schedule kinds on `main`: `at`, `every`, and `cron`. I did not find any documented job-dependency, chaining, or pipeline schedule kind. 
  - file: `docs/automation/cron-jobs.md:5`
