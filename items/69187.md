## Summary

Keep open. Current `main` still routes Dreaming narrative cleanup through `subagent.deleteSession(...)` in `finally`, and the gateway runtime still requires `operator.admin` for `sessions.delete`. The codebase has shipped mitigations in `v2026.4.23` (isolated cron runs, shorter narrative timeout, artifact scrubber, session-retention docs), but the current tests still explicitly model request-scoped cleanup failure as an expected best-effort path rather than proving the leak is fixed end-to-end.
