## Summary

Current `main` does not implement the requested `gateway.auth.mode="none"` behavior for non-loopback/internal-network deployments. The runtime still rejects non-loopback binds without shared-secret auth or `trusted-proxy`, and the docs describe `mode: "none"` as loopback-only.

## Evidence

- **Runtime blocks non-loopback `auth.mode=none`:** Gateway runtime config throws on any non-loopback bind without shared-secret auth unless the mode is `trusted-proxy`. `auth.mode="none"` does not satisfy this guard.
  - file: [src/gateway/server-runtime-config.ts:141](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/gateway/server-runtime-config.ts#L141)
  - command: `nl -ba src/gateway/server-runtime-config.ts | sed -n '132,150p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Regression test proves `mode=none` is rejected in container auto-bind:** There is an explicit test named `rejects container auto-bind without auth (security invariant preserved)` that passes `gateway.auth.mode: "none"` and expects startup to throw `refusing to bind gateway`.
  - file: [src/gateway/server-runtime-config.test.ts:290](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/gateway/server-runtime-config.test.ts#L290)
  - command: `nl -ba src/gateway/server-runtime-config.test.ts | sed -n '286,302p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **CLI docs still state non-loopback without auth is blocked:** The gateway CLI documentation says `Binding beyond loopback without auth is blocked (safety guardrail)`.
  - file: [docs/cli/gateway.md:41](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/cli/gateway.md#L41)
  - command: `nl -ba docs/cli/gateway.md | sed -n '36,44p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Config reference says `mode:none` is for trusted local loopback only:** The current configuration reference documents `gateway.auth.mode: "none"` as `Use only for trusted local loopback setups`, while non-loopback binds require token/password or `trusted-proxy`.
  - file: [docs/gateway/configuration-reference.md:306](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/gateway/configuration-reference.md#L306)
