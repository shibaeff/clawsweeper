## Summary

Keep open. Current `main` still does not propagate the winning auth profile out of embedded fallback runs or persist it back to the session override, so the PR’s core fix is not already implemented on `main`.

## Evidence

- **Post-run session persistence only updates runtime model/token state:** After the run finishes, `agentCommandInternal` calls `updateSessionStoreAfterAgentRun(...)` and then continues to delivery/transcript handling. There is no post-success persistence step for a rotated auth profile.
  - file: [src/agents/agent-command.ts:1048](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/agent-command.ts#L1048)

- **Session-store updater does not persist auth-profile overrides:** `updateSessionStoreAfterAgentRun` writes model/runtime fields, usage, harness id, and CLI session binding, but it never updates `authProfileOverride` or `authProfileOverrideSource`.
  - file: [src/agents/command/session-store.ts:33](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/command/session-store.ts#L33)
