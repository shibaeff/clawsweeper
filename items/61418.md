## Summary

Keep open. Current main still routes TUI `/models` to the generic gateway model catalog without filtering by configured auth, while the CLI `openclaw models list` default path still builds from configured entries instead of the full catalog. That matches the reported inconsistency rather than resolving it.

## Evidence

- **TUI `/models` is the local model picker:** The TUI slash-command registry defines `/models` as `Open model picker`, distinct from `/model <provider/model>`.
  - file: [src/tui/commands.ts:79](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/tui/commands.ts#L79)
