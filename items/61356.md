## Summary

Current `main` already gives each WhatsApp reconnect a fresh watchdog baseline, and the dedupe-reset helper is not used in the production reconnect path. The shipped `v2026.4.23` tag contains the same implementation and regression coverage.

## Evidence

- **Fresh connection state resets watchdog baseline:** Each live WhatsApp connection is created with a new `startedAt = Date.now()` and `lastInboundAt = null`, so reconnects start from a fresh baseline rather than reusing stale inbound timestamps.

  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Watchdog uses startedAt fallback, not only last inbound time:** The watchdog timeout checks `connection.lastInboundAt ?? connection.startedAt`, which prevents the reported 1-minute reconnect loop after a reconnect with no new inbound messages.
