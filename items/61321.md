## Summary

Keep open. Current `main` still preserves structurally valid same-agent absolute `sessionFile` paths from an old state root instead of remapping them into the current state dir, and the write-lock path still attempts to `mkdir` that preserved directory.

## Evidence

- **Cross-root fallback keeps the old absolute path:** `resolvePathWithinSessionsDir` handles absolute paths outside the current sessions dir by trying agent-based fallbacks, then falling through to `resolveStructuralSessionFallbackPath(...)` for canonical `.../agents/<agent>/sessions/<file>` shapes.
  - file: [src/config/sessions/paths.ts:190](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/paths.ts#L190)
  - command: `nl -ba src/config/sessions/paths.ts | sed -n '110,240p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Structural fallback returns the original root unchanged:** `resolveStructuralSessionFallbackPath` returns `path.normalize(path.resolve(candidateAbsPath))`, which preserves the old absolute root instead of rebuilding the path under the current state dir for the same agent.
  - file: [src/config/sessions/paths.ts:161](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/paths.ts#L161)
  - command: `nl -ba src/config/sessions/paths.ts | sed -n '110,240p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Resolved path is persisted back into the session entry:** `resolveAndPersistSessionFile` uses `resolveSessionFilePath(...)` and stores the returned `sessionFile` back into the session entry, so this preserved absolute path survives in `sessions.json` for matching `sessionId` entries.
  - file: [src/config/sessions/session-file.ts:30](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/session-file.ts#L30)
  - command: `nl -ba src/config/sessions/session-file.ts | sed -n '1,80p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Agent command uses that resolver before embedded execution:** `resolveSessionTranscriptFile` feeds the persisted session entry through `resolveAndPersistSessionFile` during agent startup, so `openclaw agent --session-id ...` will carry the stale absolute path forward into execution.
  - file: [src/config/sessions/transcript.ts:74](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/transcript.ts#L74)
  - command: `nl -ba src/config/sessions/transcript.ts | sed -n '45,95p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Write lock still mkdirs the resolved session directory:** `acquireSessionWriteLock` resolves `params.sessionFile`, takes its dirname, and immediately runs `fs.mkdir(..., { recursive: true })`; with a preserved `/home/...` path on macOS, that matches the reported failure mode.
  - file: [src/agents/session-write-lock.ts:489](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/session-write-lock.ts#L489)
  - command: `nl -ba src/agents/session-write-lock.ts | sed -n '475,530p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Current tests cover cross-agent and unsafe cross-root cases, not same-agent remap:** The existing session path tests explicitly preserve old-root cross-agent absolute paths and cover unsafe fallback cases, but there is no same-agent root-move test asserting remap into the current state dir.
  - file: [src/config/sessions.test.ts:618](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions.test.ts#L618)
  - command: `nl -ba src/config/sessions.test.ts | sed -n '600,690p'`
