## Summary

Keep open. Current `main` still uses hardcoded `10_000` ms subagent gateway call timeouts, does not expose `agents.defaults.subagents.gatewayTimeoutMs` in the config type or schema, and has no landed gateway-timeout test coverage for `subagent-spawn`. The PR targets a real core behavior gap that is not implemented on `main`, and the provided PR discussion also shows unresolved review feedback on the proposed semantics/tests rather than evidence that the work is obsolete.
