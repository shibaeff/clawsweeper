Keep open. Current `main` still does not implement the reported multi-agent dropdown fix, while the submitted PR is stale and bundles unrelated review-blocking changes, so there is not strong evidence for any auto-close path.

## Evidence

- **Current dropdown enumeration still omits configured agents without session rows:** On current `main`, `resolveSessionOptionGroups` builds options from `sessions?.sessions`, filters those rows, and then adds only the active `sessionKey`. It never seeds per-agent main sessions from `state.agentsList`, so a configured agent with no current session row can still disappear from the dropdown after switching away.
