## Summary

Current `main` still sets `ThreadStarterBody` unconditionally in the Discord inbound payload, while the reply runtime still consumes `ThreadStarterBody` on existing sessions as a `[Thread starter - for context]` note. That means the reported repeated reinjection is still plausible on `main`, so there is not strong evidence to close this PR as already implemented or obsolete.
