## Summary

Close: current `main` and the released `v2026.4.23` tree already implement the CLI-side `connect.challenge` nonce handshake, include timeout guards so the path fails instead of hanging forever, and carry regression coverage for the exact challenge/nonce flow the report says is missing.

## Evidence

- **Current CLI gateway client handles `connect.challenge` and sends nonce-bound device auth:** `GatewayClient` stores the server nonce from `connect.challenge`, requires it before sending `connect`, signs the device payload with that nonce, includes `device.nonce`, and closes on missing/timeout instead of waiting forever.
  - file: [src/gateway/client.ts:435](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/gateway/client.ts#L435)

- **CLI RPC path uses `GatewayClient` with device identity and outer timeout:** `callGateway()` routes CLI callers through `callGatewayCli()`, which creates a `GatewayClient` in CLI mode with a resolved device identity and wraps the request in a timeout, so this path is intended to fail fast rather than stall indefinitely.
