## Summary

Keep open. The report is recent, has a second corroborating comment, and current `main` does not provide strong enough evidence that secondary-workspace Slack DM inbound is fixed end-to-end.

## Evidence

- **Recent corroboration:** The issue was updated on 2026-04-21 with a second report saying the failure still reproduced on OpenClaw 2026.4.14 in a multi-workspace setup, while a single-workspace control instance worked.
- **Inbound routing is account/team aware:** Current Slack inbound routing resolves the agent route with both `accountId` and `teamId`, which is relevant to this bug class but not itself proof that the reported DM-reply failure is fixed.
  - file: [extensions/slack/src/monitor/message-handler/prepare-routing.ts:52](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/extensions/slack/src/monitor/message-handler/prepare-routing.ts#L52)
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **System-event routing also keys on account/team:** Slack monitor context routes system events through `resolveAgentRoute` using `accountId` and `teamId`, and separately drops mismatched `api_app_id` / `team_id` payloads.
  - file: [extensions/slack/src/monitor/context.ts:198](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/extensions/slack/src/monitor/context.ts#L198)
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Mismatch filtering exists but is indirect evidence:** The current code explicitly rejects Slack events whose `api_app_id` or `team_id` do not match the monitor context, which could help multi-workspace isolation, but this is still not an end-to-end verification of secondary-workspace DM replies.
  - file: [extensions/slack/src/monitor/context.ts:370](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/extensions/slack/src/monitor/context.ts#L370)
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Tests do not cover the reported scenario directly:** There are tests for mismatched-event filtering and account-bound system-event routing, but no direct regression test asserting that a DM reply from a second Slack workspace reaches OpenClaw in a multi-workspace configuration.
  - file: [extensions/slack/src/monitor/context.test.ts:53](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/extensions/slack/src/monitor/context.test.ts#L53)
