## Summary

Keep open. Current main contains several startup-footprint reductions that plausibly mitigate this class of cold-start pressure on small hosts, but I did not find strong evidence that #45440's specific Raspberry Pi 4 2 GB post-upgrade OOM/regression is fully fixed on current main.

## Evidence

- **2026.3.22 shipped startup-load reductions relevant to the report:** The 2026.3.22 changelog says gateway boot now loads bundled channel plugins from compiled `dist/extensions`, lazy-loads startup paths, and loads only configured channel plugins during gateway boot. Those changes are directionally relevant to the reported first-boot memory spike after upgrading from 2026.3.7 to 2026.3.12.
  - file: [CHANGELOG.md:2312](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/CHANGELOG.md#L2312)
  - command: `nl -ba CHANGELOG.md | sed -n '2300,2319p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Current main still encodes lean startup behavior in tests:** Tests on main assert that the lean JSON status path does not preload configured channel plugins, reducing cold-start work on constrained hosts.
  - file: [src/commands/status.scan.fast-json.test.ts:53](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/commands/status.scan.fast-json.test.ts#L53)

  - file: [src/plugins/channel-plugin-ids.test.ts:374](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/plugins/channel-plugin-ids.test.ts#L374)

## Risks / Open Questions

- Closing as implemented would rely on inference from general startup-performance work, not a direct fix or reproduction of the reported Raspberry Pi 4 2 GB upgrade OOM.
- The report is only about six weeks old as of 2026-04-24, so `stale_insufficient_info` does not apply.
- Current docs still position 2 GB Pi hosts as low-headroom systems that need swap, so the exact cold-start upgrade path in #45440 may still be vulnerable.
