## Summary

Keep open. Current `main` includes Telegram polling transport hardening and optional prompt-size mitigations, but the core reported failure mode is not clearly implemented away: Telegram DMs still default to the shared `agent:main:main` session, the bootstrap-size reduction is opt-in rather than default, and the provided timeline already links an open PR specifically targeting session reset after unrecoverable Telegram context overflow.
