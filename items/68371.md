## Summary

Keep open. Current `main` still routes `openclaw wiki bridge import` through the local CLI path, while CLI plugin loading remains in `cli-metadata` mode where `registerMemoryCapability` is a no-op. `main` does contain the snapshot-restore fix for the intermittent gateway capability wipe, but there is no corresponding landed fix for the CLI bridge path in this checkout, and the issue timeline already links an open parent bug/PR for that remaining gap.
