## Summary

Keep open. Current `main` still matches the CLI loader gap described in the issue comments: `openclaw wiki bridge import` runs local sync code, but plugin CLI registration is still built from a non-activating plugin load that restores prior memory capability state instead of leaving `memory-core`'s `publicArtifacts` capability active. The v2026.4.23 changelog shows the snapshot-restore fix from #69219 shipped, but I did not find corresponding main-branch evidence that the separate CLI path was rerouted or otherwise fixed.

## Evidence

- **CLI command still runs local bridge sync:** The `wiki bridge import` CLI action still calls `syncMemoryWikiImportedSources(...)` directly from the CLI process rather than going through a gateway RPC path.
  - file: [extensions/memory-wiki/src/cli.ts:502](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/memory-wiki/src/cli.ts#L502)

  - file: [src/plugins/cli-registry-loader.ts:115](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/plugins/cli-registry-loader.ts#L115)
