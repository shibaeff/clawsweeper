## Summary

Keep open: current `main` still lacks ephemeral peer agent-to-agent sessions. `sessions_send` and its follow-up reply/announce flow target existing session keys directly and persist those turns in transcript history, with only `inter_session` provenance markers rather than an isolated temporary session.

## Evidence

- **sessions_send targets the existing session directly:** `sessions_send` builds the outbound `agent` call with `sessionKey: resolvedKey` and `inputProvenance.kind = "inter_session"`; there is no temporary or derived ephemeral session created before delivery.
  - file: [src/agents/tools/sessions-send-tool.ts:275](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/tools/sessions-send-tool.ts#L275)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '220,420p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **A2A reply and announce steps also run inside real sessions:** The ping-pong reply loop and announce step call `runAgentStep` with `sessionKey` set to the requester or target session, so follow-up agent-to-agent traffic is injected into those existing sessions as well.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:70](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/tools/sessions-send-tool.a2a.ts#L70)

- **Nested agent steps persist as inter-session messages, not ephemeral sessions:** `runAgentStep` calls `method: "agent"` against the supplied `sessionKey` and tags the turn with `inputProvenance.kind = "inter_session"`; this is provenance on a normal session turn, not transient-session isolation.
