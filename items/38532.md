## Summary

Current main already fixes the native-provider `web_search` collision without renaming the public tool. OpenClaw now suppresses the managed `web_search` tool when native Codex search is active, while intentionally keeping the public tool/catalog/config name as `web_search` for compatibility. The PR’s rename is superseded, and its unrelated plugin-discovery/version/avatar changes are not needed on main.
