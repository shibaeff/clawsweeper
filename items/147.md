## Summary

Keep open. Current main has the gateway/node primitives that make an external Pi voice node possible, but there is still no first-party Brabble/Raspberry Pi voice node on `main`, and the repo’s ClawHub guidance is for skills/plugins rather than companion-node runtimes.

## Evidence

- **Companion-app scope remains in-project:** The vision doc still lists companion apps on macOS, iOS, Android, Windows, and Linux as a next priority, so a distributed voice node is not obviously out of scope for OpenClaw.
  - file: `VISION.md:253`
  - sha: `ef88cabe3947bf97494a0407d5ef7d56c83cc7c5`
- **ClawHub guidance is narrower than this request:** The same vision doc says optional capability should usually ship as plugins and that new skills should go to ClawHub first. This issue is a companion-node/runtime request, not clearly a skill/plugin request.
  - file: `VISION.md:276`
  - sha: `ef88cabe3947bf97494a0407d5ef7d56c83cc7c5`
- **Main documents generic node transport, not Brabble:** The gateway protocol docs describe a generic WebSocket node transport with node pairing, invoke, event, and pending-work APIs for headless nodes. That is enabling infrastructure, but it is not a bundled Brabble or Raspberry Pi implementation.
  - file: `docs/gateway/protocol.md:1`
  - sha: `ef88cabe3947bf97494a0407d5ef7d56c83cc7c5`
- **Main documents HTTP/SSE agent ingress:** The HTTP docs on main expose `POST /v1/chat/completions` with SSE streaming and agent routing. That matches the external prototype described in the issue comments, but it is a generic surface rather than the requested first-party node.
  - file: `docs/gateway/openai-http-api.md:1`
  - sha: `ef88cabe3947bf97494a0407d5ef7d56c83cc7c5`
- **Shipped voice/node support is still platform-specific:** `README.md` advertises Voice Wake on macOS/iOS and continuous voice on Android, with companion apps described as the macOS app plus iOS/Android nodes. There is no mention of a Raspberry Pi or Brabble node on main.
  - file: `README.md:17`
  - sha: `ef88cabe3947bf97494a0407d5ef7d56c83cc7c5`
- **Issue comments show an external solution, not a merged core feature:** On 2026-03-08, @afalk42 reported a working external Raspberry Pi voice node that uses the gateway `/v1/chat/completions` endpoint and explicitly said it required no OpenClaw PR. That supports 'possible externally' but not 'implemented on main'.
