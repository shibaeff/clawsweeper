## Summary

Keep open. Current `main` still overwrites the entire memory capability registration on each `registerMemoryCapability()` call, and `memory-wiki` still depends on `publicArtifacts`, so the underlying edge case is not implemented on `main`. The PR's specific `active-memory` reproduction does not match current `main`, but the loader and public plugin API still allow a second memory plugin to coexist with `memory-core` and replace its capability fields.
