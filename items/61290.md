## Summary

Keep open. Current `main` has overflow-recovery compaction and timeout guards, but the code and docs still allow a single compaction to run for up to 900 seconds by default, which matches the reported 10+ minute stall window. I did not find evidence on current `main` or in the changelog that this specific `compactionAttempts=0` hang was resolved after the later reproductions on 2026.4.15 and 2026.4.21.
