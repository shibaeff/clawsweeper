## Summary

Keep open. Current main still classifies bare Feishu `oc_...` outbound targets as `direct` unless the caller uses an explicit `chat:`/`group:` prefix, while inbound Feishu webhooks classify real group traffic from `chat_type === "group"`. That means the reported group/direct split is still plausible on main for proactive sends that use a bare `oc_...` target.

## Evidence

- **Outbound Feishu session routing still defaults bare targets to direct:** `resolveFeishuOutboundSessionRoute` only marks the target as group when the raw target starts with `group:`, `chat:`, or `channel:`. Otherwise `isGroup` stays false unless the ID looks like `ou_`/`on_`, so a bare `oc_...` target produces `peer.kind = "direct"`, `chatType = "direct"`, and a direct session key.
  - file: [extensions/feishu/src/session-route.ts:8](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/feishu/src/session-route.ts#L8)
  - command: `nl -ba extensions/feishu/src/session-route.ts | sed -n '1,120p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Inbound Feishu handling still routes real group messages through group session logic:** Inbound processing derives `isGroup` from the webhook message `chatType` and, when it is `group`, resolves group session state via `resolveFeishuGroupSession(...)`. So inbound group traffic and outbound bare-target traffic still use different classification inputs.
  - file: [extensions/feishu/src/bot.ts:302](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/feishu/src/bot.ts#L302)

- **Outbound session entries persist whatever chatType the route returned:** `ensureOutboundSessionEntry` writes `route.chatType` into the stored message context. If Feishu outbound routing returns `direct`, that direct classification is what gets recorded for the session metadata.
  - file: [src/infra/outbound/outbound-session.ts:135](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/infra/outbound/outbound-session.ts#L135)
