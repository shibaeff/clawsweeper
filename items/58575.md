## Summary

Current `main` already gives isolated cron `payload.model` precedence over stored session overrides and prevents the agent primary from being silently re-injected as a fallback. The relevant runtime code is covered by targeted regression tests, and the same runtime/test implementation is already present in release `v2026.4.23`.

## Evidence

- **Payload model wins over stored session override:** `resolveCronModelSelection()` applies `payload.model` first and only consults `sessionEntry.modelOverride` when no payload override is present.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/model-selection.ts#L104)
  - command: `nl -ba src/cron/isolated-agent/model-selection.ts | sed -n '1,180p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Cron fallback policy no longer injects agent primary behind payload.model:** `resolveCronFallbacksOverride()` marks a cron payload model as an override, and `resolveEffectiveModelFallbacks()` returns configured fallbacks or `[]` rather than silently appending the agent primary when none are configured.
  - file: [src/cron/isolated-agent/run-fallback-policy.ts:10](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/run-fallback-policy.ts#L10)

- **Targeted regression test covers the reported behavior:** The cron isolated-agent regression test asserts that `payload.model` reaches `runWithModelFallback()` and the embedded runner, and that a plain payload model override yields `fallbacksOverride = []` instead of falling back to the agent primary.
