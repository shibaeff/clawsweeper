## Summary

Current `main` already routes Feishu health checks through a cached probe implementation, and the latest shipped release `v2026.4.22` contains the same code. The reported uncached `bot/v3/info` path is not present in the current bundled Feishu plugin.

## Close Comment

Closing from a Codex review. Current `main` already implements this: the bundled Feishu probe in `extensions/feishu/src/probe.ts:6` caches results, returns cached entries from `extensions/feishu/src/probe.ts:75`, and no longer uses the reported uncached `GET /open-apis/bot/v3/info` path. The live probe code now calls `POST /open-apis/bot/v1/openclaw_bot/ping` at `extensions/feishu/src/probe.ts:89`, and Feishu monitor startup still goes through that cached helper at `extensions/feishu/src/monitor.startup.ts:54`. Regression coverage in `extensions/feishu/src/probe.test.ts:172` and nearby cases verifies repeat probes reuse the cached result and avoid extra client calls. I also verified the same implementation is present in the shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so there is no remaining maintainer action on `main` for this report.
