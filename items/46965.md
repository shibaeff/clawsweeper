## Summary

Keep open. Current `main` still limits `suppressToolErrorWarnings` to heartbeat config, and the requested global or per-agent non-heartbeat suppression is not implemented.

## Evidence

- **Config surface is heartbeat-only:** `suppressToolErrorWarnings` is defined only inside the heartbeat block, with the comment 'Suppress tool error warning payloads during heartbeat runs.' There is no sibling `agents.defaults.suppressToolErrorWarnings` or per-agent non-heartbeat field.
  - file: [src/config/types.agent-defaults.ts:362](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/config/types.agent-defaults.ts#L362)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '355,375p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Docs still describe heartbeat-only behavior:** The user-facing config docs list `suppressToolErrorWarnings` only under `agents.defaults.heartbeat` and explicitly say it applies 'during heartbeat runs.'
  - file: [docs/gateway/config-agents.md:524](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/gateway/config-agents.md#L524)
  - command: `nl -ba docs/gateway/config-agents.md | sed -n '514,530p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Global key is still rejected:** CLI validation tests treat `agents.defaults.suppressToolErrorWarnings` as an unrecognized config path, which directly contradicts the requested feature surface.
  - file: [src/cli/config-cli.test.ts:550](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/cli/config-cli.test.ts#L550)
  - command: `nl -ba src/cli/config-cli.test.ts | sed -n '544,566p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Runtime wiring is only from heartbeat runs:** The heartbeat runner reads `heartbeat?.suppressToolErrorWarnings` and passes it as a per-run reply option; there is no analogous global/per-agent non-heartbeat wiring in this path.
  - file: [src/infra/heartbeat-runner.ts:1065](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/infra/heartbeat-runner.ts#L1065)

  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Existing global suppression is narrower than requested:** Current payload policy only lets `messages.suppressToolErrors` suppress non-mutating tool errors; mutating tool errors still surface unless `suppressToolErrorWarnings` is set on the run. Tests explicitly cover both behaviors.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:107](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/agents/pi-embedded-runner/run/payloads.ts#L107)

## Risks / Open Questions

- `messages.suppressToolErrors` partially overlaps with the request, but current tests show it does not suppress mutating tool failures, so closing as already-covered would miss the remaining gap.
