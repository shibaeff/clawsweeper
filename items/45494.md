## Summary

Keep open. Current `main` still treats Anthropic-style `api_error: Internal server error` as `timeout`, does not mark timed-out profiles for cooldown, and limits built-in transient retry/backoff to one-shot cron jobs rather than recurring cron schedules.

## Evidence

- **Current classifier still maps `api_error` internal-server messages to timeout:** The active test suite asserts that JSON `api_error` payloads such as `{"type":"error","error":{"type":"api_error","message":"Internal server error"}}` classify as `timeout`, not `overloaded` or a fast-fail category.
  - file: [src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts:1143](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts#L1143)
  - command: `nl -ba src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts | sed -n '1140,1165p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Pattern tables still treat `internal server error` as server-error/timeout path:** `ERROR_PATTERNS.serverError` includes `internal server error`, while `ERROR_PATTERNS.overloaded` does not. The classifier then maps server errors and matching `api_error` payloads to `timeout`.
  - file: [src/agents/pi-embedded-helpers/failover-matches.ts:73](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-embedded-helpers/failover-matches.ts#L73)
  - command: `nl -ba src/agents/pi-embedded-helpers/failover-matches.ts | sed -n '45,110p'; nl -ba src/agents/pi-embedded-helpers/errors.ts | sed -n '760,835p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Timeout-classified failures still do not cooldown auth profiles:** `maybeMarkAuthProfileFailure` returns early for `reason === "timeout"`, and `resolveAuthProfileFailureReason` also drops timeout reasons. That means the reported 500-path still will not mark the failing profile unavailable for quicker cross-provider failover.
  - file: [src/agents/pi-embedded-runner/run.ts:607](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-embedded-runner/run.ts#L607)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '600,670p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Recurring cron jobs still do not get the requested fast retry behavior:** `cron.retry` is documented as one-shot-only, and the timer code applies that retry branch only when `job.schedule.kind === "at"`. Recurring jobs instead compute the next run as the later of normal schedule and backoff, so they do not get the requested short-delay catch-up retry flow.
  - file: [src/cron/service/timer.ts:467](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/cron/service/timer.ts#L467)
