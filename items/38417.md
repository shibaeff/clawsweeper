## Close Comment

Closing as `clawhub` after a Codex review. This request is for `openclaw-mem0`, which is not part of the current OpenClaw core tree, and current scope docs keep optional/plugin-specific behavior in plugins maintained in their own repos (`VISION.md:54`, `docs/plugins/community.md:9`, `docs/plugins/sdk-setup.md:521`). Current main already exposes the agent-aware primitives a memory plugin needs to implement this itself: hook context includes `agentId` (`src/plugins/hook-types.ts:154`), plugin tool context includes `agentId` (`src/plugins/tool-types.ts:8`), and the memory runtime contract is agent-aware (`src/plugins/memory-state.ts:96`). Given that, `{agentId}` templating or per-agent `userId` handling belongs in the `openclaw-mem0` plugin package / ClawHub entry rather than OpenClaw core.
