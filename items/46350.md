## Summary

Keep open. Current `main` still defaults memory-host remote HTTP calls to strict guarded fetch mode, and the SSRF guard only honors `HTTP_PROXY`/`HTTPS_PROXY` when callers explicitly pass `GUARDED_FETCH_MODE.TRUSTED_ENV_PROXY`. The PR’s original `src/memory/remote-http.ts` target is stale after the memory-host-sdk refactor, but the underlying bug remains in the new helper locations.

## Evidence
