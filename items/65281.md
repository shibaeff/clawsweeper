## Summary

Keep open. Current `main` already contains backend fixes for image-only webchat turns and stale image-capability detection, but the Control UI paste handler in `ui/src/ui/views/chat.ts` is still a simple per-item `FileReader` callback path with no dedicated paste regression test. That is not strong enough evidence to conservatively close a report that specifically says paste/preview broke in the browser UI.
