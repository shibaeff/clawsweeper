## Summary

Current `main` already supports interrupting an active run, supplying corrective guidance, and continuing with the updated instruction. The gateway exposes `sessions.steer` as an interrupt-and-send path, the UI/docs expose `Stop` plus queued-message `Steer`, and queue modes include both `steer` and `interrupt`. The same behavior is present in release `v2026.4.22`.

- The exact UX proposed in the issue (for example a dedicated Ctrl+C prompt offering Continue/Correct/Abort) is not what shipped; the shipped implementation uses existing Stop, Steer, queue, and `/redirect` controls instead.

## Close Comment

Closing from a Codex review because the requested capability is already present on current `main` and was shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). OpenClaw already supports aborting an active run (`/stop` / `chat.abort`), injecting corrective guidance into a running turn via queued-message **Steer** in the Control UI, and hard interrupt-and-restart via `sessions.steer` / `/redirect`: see `src/gateway/server-methods/sessions.ts:445`, `ui/src/ui/chat/slash-command-executor.ts:861`, `ui/src/ui/chat/slash-command-executor.node.test.ts:923`, `docs/web/control-ui.md:170`, and `docs/concepts/queue.md:27`. That covers the core request to interrupt, correct, and continue with updated guidance, so there is no remaining maintainer action on this issue.
