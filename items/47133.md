## Summary

Keep open. Current `main` still ships and documents `KillMode=control-group` for the Linux systemd gateway unit, and the gateway run loop still treats any `SIGTERM` as an immediate shutdown with no startup-window guard. The proposed fix described in the issue comments is not present on `main` or in `v2026.4.22`, so there is not strong evidence that #47133 is implemented or obsolete.
