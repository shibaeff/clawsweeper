## Summary

Close the PR as already implemented on `main` by a later Telegram media-download fix. Current `main` intentionally keeps resolver-scoped explicit-proxy dispatchers for Telegram API fetches, but the reported regression is already addressed by threading Telegram transport dispatcher attempts into media downloads and allowing trusted explicit proxies to resolve target DNS during SSRF-guarded file fetches. That shipped in `v2026.4.22`.
