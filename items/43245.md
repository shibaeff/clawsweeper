## Summary

Keep open. Current `main` still allows same-provider cross-context message sends by default for bound sessions, so the reported behavior remains present and is not implemented/fixed on `main`. The docs-mismatch part of the report appears partly stale, but the code and tests still support the core issue.

## Evidence

- **Policy default still allows same-provider override:** `enforceCrossContextPolicy` computes `allowWithinProvider` as `params.cfg.tools?.message?.crossContext?.allowWithinProvider !== false` and returns early when that value is true, so bound same-provider cross-context sends are allowed unless operators explicitly set the flag to `false`.

  - command: `nl -ba src/infra/outbound/outbound-policy.ts | sed -n '1,240p'`

- **Message runner still uses that policy in the live send path:** `runMessageAction` calls `enforceCrossContextPolicy` before dispatching the send action, so the default-allow behavior in `outbound-policy.ts` is the active runtime behavior for message sends.
