## Summary

Current `main` already restores default operator scopes for shared-secret `/v1` requests, so `Authorization: Bearer <token>` no longer depends on `X-OpenClaw-Scopes` for the OpenAI-compatible HTTP surface. The fix is present on `main`, but I could not prove a released tag from this checkout, so `fixedRelease` is left null.
