## Summary

Current `main` still appears to need the narrow `openclaw node start` CLI wiring, but this PR also adds a browser-server change that conflicts with the plugin-owned node browser proxy architecture now documented and implemented on `main`. That makes the PR unsuitable for conservative cleanup closure: there is still a valid bug to fix, but this exact two-part change is not the best fix as written.

## Evidence

- **`node start` is still missing from the CLI router on `main`:** `registerNodeCli()` imports install/restart/status/stop/uninstall only, and registers `run`, `status`, `install`, `uninstall`, `stop`, and `restart` commands. There is no `start` subcommand in the current CLI router or help examples.

## Risks / Open Questions

- Closing this PR would discard a still-valid bug fix for the missing `openclaw node start` command on current `main`.
- Landing the PR as-is would deep-import browser extension internals into core node-host startup and duplicate the existing demand-driven `browser.proxy` path.
