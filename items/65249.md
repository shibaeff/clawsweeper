## Summary

This proposal is plugin-scope, not OpenClaw core scope. Current main already has baseline relevance filtering in the active memory plugin (`memory_search`, hybrid vector/BM25 ranking, configurable result caps, temporal decay, MMR, and thresholded dreaming/promotion), and the repo explicitly treats memory as an exclusive plugin slot with documented SDK seams for custom memory implementations. More opinionated AI-ranked memory selection, feedback loops, and experimentation fit a ClawHub/community memory plugin or an existing external memory plugin such as Honcho rather than core.
