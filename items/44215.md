## Summary

Current `main` does not implement this PR’s Windows PATH bootstrap behavior. `src/infra/path-env.ts` still only prepends immutable Unix dirs and appends Unix-style user/package-manager dirs; it has no `win32` branch or platform-specific delimiter handling. Recent `main` also added explicit path-hardening comments/tests that user-writable dirs must stay after system dirs, so this PR now needs redesign/rebase rather than cleanup-closing.

## Evidence

## Risks / Open Questions

- Minimal Windows PATH environments may still miss common tool locations because `main` has no Windows-specific bootstrap candidates in `src/infra/path-env.ts`.
- The PR as written prepends user-writable Windows dirs, which conflicts with the newer path-hardening model now on `main` and likely needs a safer append/order strategy.
- The PR’s `mergePath` delimiter fix is also absent on `main`, so any future Windows-path simulation test still needs platform-delimiter handling.
