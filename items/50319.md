## Summary

Issue #50319 is a valid integration request, but current main treats new messaging-platform integrations as external channel plugins distributed through ClawHub or npm, not as new core/bundled OpenClaw work.

## Evidence

## Close Comment

Closing from a Codex review because this request fits the ClawHub/community-plugin path rather than OpenClaw core. Current main says optional capability should usually ship as plugins, be hosted in their own repo, and that the bar for adding optional plugins to core is intentionally high (`VISION.md:54`). The plugin docs also explicitly cover messaging integrations as external channel plugins: `docs/plugins/building-plugins.md:11` says channel plugins should be published to ClawHub or npm instead of added to this repo, `docs/plugins/sdk-channel-plugins.md:11` documents the supported channel-plugin SDK, and `docs/plugins/community.md:9` already lists third-party channel plugins such as DingTalk, QQbot, and WeCom. Based on that current scope, MAX looks like a good community channel plugin, but not a core-maintainer issue for this repository.
