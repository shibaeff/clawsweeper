## Summary

Keep open. Current `main` still marks `minimax/MiniMax-M2.7` and `MiniMax-M2.7-highspeed` as text-only, the gateway still uses that catalog flag to treat `chat.send` attachments as non-inline offloads, and the runner still skips loading those offloaded images for models without `input: ["image"]`. The reported explicit-config workaround also still fails because provider/model merge overwrites explicit `input` with the implicit catalog value.

## Evidence

- **MiniMax provider catalog still hardcodes text-only input:** `buildMinimaxTextModel()` returns `input: ["text"]`, and `buildMinimaxCatalog()` uses it for every MiniMax catalog entry, including `MiniMax-M2.7` and `MiniMax-M2.7-highspeed`.

  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Explicit model `input` overrides are still discarded during provider merge:** `mergeProviderModels()` assigns `input: implicitModel.input` when an explicit model matches an implicit catalog row, so a user-configured `input: ["text", "image"]` does not survive merging.
  - file: [src/agents/models-config.merge.ts:100](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/models-config.merge.ts#L100)
