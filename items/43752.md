## Summary

Keep open. Current `main` still passes the gateway CLI port override into `acquireGatewayLock`, but the lock-file payload written in `src/infra/gateway-lock.ts` does not include `port`, so the PR is addressing a real, unmerged bug rather than something already fixed or obsolete.

## Evidence

- **Current main still drops `port` when writing lock payload:** `GatewayLockOptions` includes `port`, and `acquireGatewayLock` reads `opts.port`, but the payload written to disk only contains `pid`, `createdAt`, `configPath`, and optional `startTime` before `JSON.stringify(payload)`. There is no `payload.port` on current `main`.
