## Summary

Keep open. Current `main` still does not persist the effective `--port` into the gateway lock payload, so the PR addresses a real gap rather than something already implemented or obsolete.

## Evidence

- **Lock payload on main has no persisted port field:** `LockPayload` and `LockPayloadSchema` only include `pid`, `createdAt`, `configPath`, and optional `startTime`; there is no `port` field in the serialized lock payload.
  - file: [src/infra/gateway-lock.ts:18](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/infra/gateway-lock.ts#L18)

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Lock file writer still omits port:** When `acquireGatewayLock` creates the lock file, it writes JSON with `pid`, `createdAt`, `configPath`, and optional `startTime` only. No branch persists `opts.port` before `handle.writeFile(JSON.stringify(payload), "utf8")`.
  - file: [src/infra/gateway-lock.ts:276](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/infra/gateway-lock.ts#L276)
