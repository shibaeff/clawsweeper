## Summary

Current `main` already short-circuits all-profiles-in-cooldown cases into bounded fallback/probe behavior and a final cooldown message. The indefinite session retry loop described in #61622 is no longer the current behavior.

## Evidence

- **Cooldowned providers are skipped instead of retried indefinitely:** `runWithModelFallback` checks whether every auth profile for a candidate provider is already in cooldown, then records a skip and continues rather than re-running that candidate. The same block limits transient cooldown probing to at most one attempt per provider per fallback run.
