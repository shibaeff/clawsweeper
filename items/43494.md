## Summary

Keep open. Current `main` only partially addresses the request: Slack now has documented slash-command workarounds, and per-agent `reasoningDefault` exists, but the requested global `agents.defaults.reasoningDefault` option is still not implemented in schema/runtime resolution.

## Evidence

- **Global agent defaults still omit reasoningDefault:** `AgentDefaultsConfig` defines `thinkingDefault`, `verboseDefault`, and other defaults, but no `reasoningDefault`; the matching Zod schema also lacks it, so `agents.defaults.reasoningDefault` is not a valid config key on current main.
  - file: [src/config/types.agent-defaults.ts:297](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/types.agent-defaults.ts#L297)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '286,318p'; nl -ba src/config/zod-schema.agent-defaults.ts | sed -n '218,252p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Runtime still falls back straight to off:** Reasoning resolution is `inline/session override -> per-agent reasoningDefault -> "off"`. There is no global default fallback path for `agents.defaults.reasoningDefault`.
  - file: [src/auto-reply/reply/get-reply-directives.ts:426](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/auto-reply/reply/get-reply-directives.ts#L426)

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Only per-agent reasoningDefault is implemented and documented:** Current schema/help/docs expose `agents.list[].reasoningDefault` as an optional per-agent override, including tests that assert that behavior; there is no equivalent docs entry for `agents.defaults.reasoningDefault`.
  - file: [src/config/types.agents.ts:81](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/types.agents.ts#L81)

## Risks / Open Questions

- Closing this would drop a still-valid feature gap: users still cannot set reasoning visibility globally via `agents.defaults.reasoningDefault` on current main.
- The Slack side is only a workaround path that requires Slack slash-command setup; there is still no alternative text prefix like the issue proposed.
