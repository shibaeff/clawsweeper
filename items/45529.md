- There is no dedicated `--stdin` flag on current main; this closure relies on the shipped `--batch-file` path being sufficient for the reported PowerShell quoting problem.

## Close Comment

Closing this after a Codex review because current `main` already has a shell-safe file-input path for `openclaw config set`. The CLI exposes `--batch-file <path>` on `config set` (`src/cli/config-cli.ts:1611`), the parser reads and applies batch operations from disk (`src/cli/config-set-input.ts:118`), the docs show that mode (`docs/cli/config.md:150`), and regression tests cover successful and malformed file-backed updates (`src/cli/config-cli.test.ts:1333`). The changelog also records JSON/batch `config set` support in the `2026.3.22` release (`CHANGELOG.md:2272`). That gives Windows/PowerShell users a supported way to avoid quote-mangling by putting the literal JSON value in a file and applying it with `openclaw config set --batch-file ...`.
