## Summary

Current `main` does not contain this PR's fallback, so the reported gap still appears real: `resolveEffectiveToolPolicy()` still stops after explicit `agentId` or `sessionKey` resolution and never falls back to the default agent when both are absent. The existing tests cover global tool config and explicit agent selection, but not the no-`sessionKey` default-agent case this PR adds. I do not have high-confidence evidence for any auto-close reason.
