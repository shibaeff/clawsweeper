## Summary

Current `main` still appears to have the reported bug. In `src/agents/pi-tools.policy.ts`, `resolveEffectiveToolPolicy()` only resolves `agentId` from an explicit `agentId` or from `sessionKey`; it does not fall back to the configured default agent, so a config-only call still leaves `agentId` undefined and skips agent-specific tool policy resolution. The existing tests cover global tool sections and explicit-agent/session-key cases, but there is no regression test for the no-`sessionKey`, no-`agentId` path.
