## Summary

Keep open. Current `main` partially addresses the report: it now supports `audienceType: "app-url"`, verifies add-on `systemIdToken` requests, and unwraps add-on `messagePayload` into a normal Chat `MESSAGE` event. But the user-facing docs still do not document `appPrincipal`, and the add-on body translation is still limited to `chat.messagePayload` rather than the broader add-on event shapes discussed in the issue/comment workaround. That is not strong enough for a conservative cleanup close.
