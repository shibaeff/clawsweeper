## Summary

Current `main` already implements the subagent announce-routing behavior this PR is aiming for: canonical requester-key routing is preserved, cron requesters stay internal, and fallback only happens when the parent requester is actually unusable. The open PR is superseded by code and regression coverage now present on `main`.

## Evidence

- **Canonical requester key is used for routing depth/internal detection:** `runSubagentAnnounceFlow` initializes `targetRequesterSessionKey` from `params.requesterSessionKey` and computes internal-session status from that canonical key, while still treating cron session keys as internal. This avoids the display-key alias drift called out in review.
