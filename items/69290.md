## Summary

Current `main` and the shipped `v2026.4.23` release already gate the `"✅ ACP session reset in place."` reply behind `isAcpSessionKey(...)`, so a non-ACP bound Telegram session should fall through to the generic reset-hook path instead. The linked follow-up PR #69349 is test-only and matches that reading rather than identifying a production fix.

## Evidence

- **Current main ACP reply branch is explicitly gated:** `maybeHandleResetCommand` resolves a bound session key, then only enters the ACP reset/reply branch if `isAcpSessionKey(boundAcpSessionKey)` is true. The ACP success reply is inside that gated block.
  - file: [src/auto-reply/reply/commands-reset.ts:118](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/auto-reply/reply/commands-reset.ts#L118)
  - command: `nl -ba src/auto-reply/reply/commands-reset.ts | sed -n '116,145p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Non-ACP path falls through to generic reset hooks:** If the ACP gate is not satisfied, the handler skips the ACP reply path, emits the normal reset hooks, and returns `null`.
  - file: [src/auto-reply/reply/commands-reset.ts:153](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/auto-reply/reply/commands-reset.ts#L153)
  - command: `nl -ba src/auto-reply/reply/commands-reset.ts | sed -n '151,165p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Resolver can surface non-ACP session keys, so the caller-side ACP gate is the critical invariant:** `resolveBoundAcpThreadSessionKey` calls `resolveEffectiveResetTargetSessionKey(... allowNonAcpBindingSessionKey: true)`, and the reset-target resolver returns `serviceSessionKey` unchanged when that flag is set.
  - file: [src/auto-reply/reply/acp-reset-target.ts:141](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/auto-reply/reply/acp-reset-target.ts#L141)

- **ACP detection excludes ordinary agent/channel session keys:** `isAcpSessionKey` returns true only for `acp:` keys or parsed agent session keys whose `rest` starts with `acp:`.
  - file: [src/sessions/session-key-utils.ts:86](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/sessions/session-key-utils.ts#L86)
