## Summary

Current main already implements the reported silent-empty-turn failure mode: OpenClaw now retries provider turns that end with `stopReason="error"` and zero output, and if that still fails it surfaces a visible error reply instead of silently completing with `replies=0`.

## Close Comment

Closing from a Codex review because current `main` already covers the silent-empty-turn path described here. The shipped 2026.4.20 changelog notes this exact class of fix in `CHANGELOG.md:405`: OpenClaw now retries provider turns that end with `stopReason="error"` and no output. On current `main`, that retry is implemented in `src/agents/pi-embedded-runner/run.ts:2070`, and if the retry still cannot produce a reply the runner now returns a visible `⚠️ Agent couldn't generate a response. Please try again.` payload (`src/agents/pi-embedded-runner/run.ts:2119`, `src/agents/pi-embedded-runner/run/incomplete-turn.ts:213`) instead of silently finishing with no reply. Final payloads then go through the normal delivery path in `src/auto-reply/reply/dispatch-from-config.ts:1084`, so this is no longer the old `queuedFinal=false, replies=0` silent failure behavior reported against 2026.3.13. If you can still reproduce this on a current build, please open a new issue with updated logs and the exact current version.
