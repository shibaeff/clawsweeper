## Summary

Keep open. Current `main` still has the gap the PR describes: Feishu reply attachments call `sendMediaFeishu()` without forwarding agent-scoped `mediaLocalRoots`, even though `sendMediaFeishu()` now documents that local-path sends require those roots. The normal Feishu outbound path already passes `ctx.mediaLocalRoots`, so reply attachments remain inconsistent on `main`.

## Evidence

- **Reply dispatcher still omits mediaLocalRoots on main:** `sendMediaReplies()` in `extensions/feishu/src/reply-dispatcher.ts` calls `sendMediaFeishu({ cfg, to, mediaUrl, replyToMessageId, replyInThread, accountId })` and does not pass `mediaLocalRoots`.
  - file: [extensions/feishu/src/reply-dispatcher.ts:346](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/feishu/src/reply-dispatcher.ts#L346)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Feishu media send path expects mediaLocalRoots for local files:** `sendMediaFeishu()` documents that local-path sends must receive `mediaLocalRoots`, and forwards them to `loadWebMedia(..., { localRoots })`. Without that field, local reply attachments can fail validation.
  - file: [extensions/feishu/src/media.ts:573](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/feishu/src/media.ts#L573)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Other Feishu outbound flow already propagates roots:** The Feishu channel action path passes `ctx.mediaLocalRoots` into media sending, which shows the intended behavior exists elsewhere but not in reply dispatch.
  - file: [extensions/feishu/src/channel.ts:728](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/feishu/src/channel.ts#L728)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Current tests do not cover the missing propagation:** The existing reply-dispatcher attachment test only asserts `replyInThread` forwarding for media attachments; there is no assertion for `mediaLocalRoots` propagation on current `main`.
  - file: [extensions/feishu/src/reply-dispatcher.test.ts:698](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/feishu/src/reply-dispatcher.test.ts#L698)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
