## Summary

Keep open. Current `main` at `9153598e659daf5a52c479731f5b4ab8a5bf16b7` still does not implement per-agent/session isolation for cron visibility or mutations. The service list paths read from the full store with no `agentId`/`sessionKey` filter, `update`/`remove`/`enqueueRun` accept any job id, and the gateway handlers do not thread caller identity into those operations. Gateway tests also still assert global `cron.runs` visibility with `scope: "all"`.
