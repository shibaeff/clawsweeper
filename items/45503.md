## Summary

Current `main` still only supports automatic cache-TTL-based tool-result pruning/clearing, not a user- or agent-triggered manual purge of specific tool results. The issue remains a valid core feature request and should stay open.

## Evidence

- **Docs describe pruning as automatic TTL flow:** `docs/concepts/session-pruning.md:28-34` says pruning works by waiting for cache TTL expiry, then soft-trimming and hard-clearing old tool results.
  - file: [docs/concepts/session-pruning.md:28](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/concepts/session-pruning.md#L28)

  - file: [src/agents/pi-hooks/context-pruning/settings.ts:7](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-hooks/context-pruning/settings.ts#L7)
