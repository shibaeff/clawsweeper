## Summary

Keep open. Current main does not already include the PR's dedicated `automation/system-events` recipes page, and the proposed doc content has unresolved correctness and safety problems called out by review and confirmed against current code/docs.

## Evidence

- **Dedicated page is not on current main:** The Automation nav on current `main` lists `automation/index`, `automation/cron-jobs`, `automation/tasks`, `automation/taskflow`, `automation/standing-orders`, and `automation/hooks`; there is no `automation/system-events` entry, and `docs/automation/` has no `system-events.md` file. The PR's main deliverable is therefore not already implemented on `main`.
