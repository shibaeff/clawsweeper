## Summary

Keep open. Current `main` still materializes `sessions.json` with `readFileSync` + `JSON.parse`, and several session paths still read whole transcript files into memory rather than streaming or paginating. The exact reported startup/RSS numbers were not reproduced in this read-only review, but the underlying implementation concern is still present on `main`.

## Evidence

- **Session store still fully read and parsed:** `loadSessionStore()` reads the entire store file with `fs.readFileSync(storePath, "utf-8")` and immediately `JSON.parse`s it before returning a cloned in-memory object.
