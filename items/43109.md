## Summary

Current `main` does not appear to implement this PR’s alias-resolution fix. `src/hooks/llm-slug-generator.ts` still feeds the raw effective model string into `parseModelRef`, which treats alias-only values as `DEFAULT_PROVIDER/<alias>`. The alias-aware helpers already exist elsewhere on `main`, and the hook’s current tests do not cover alias resolution.

## Evidence
