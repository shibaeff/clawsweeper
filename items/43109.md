## Summary

Keep open. The underlying bug still appears to exist on current `main`: `generateSlugViaLLM` still feeds bare model strings through `parseModelRef`, which turns an alias like `analysis-model` into `DEFAULT_PROVIDER/analysis-model` instead of consulting the configured alias map. OpenClaw already has alias-aware helpers and tests for this pattern elsewhere, so the PR’s core fix is still directionally correct, but it now needs a refresh against the current hook/test shape on `main` rather than an auto-close.

## Evidence

- **Current hook still uses non-alias-aware parsing:** `generateSlugViaLLM` reads `resolveAgentEffectiveModelPrimary(...)` and immediately passes it to `parseModelRef(...)`, then uses that provider/model for the embedded PI run. There is no alias index or alias-aware resolution in this path.
