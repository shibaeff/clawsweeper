Current scope guidance is explicit: `VISION.md:54-70` says optional capability should usually ship as plugins, hosted in the author’s own repo, and that new skills should be published to ClawHub first; `VISION.md:99-101` also lists new core skills that can live on ClawHub as out of scope for core. If you want discoverability for this project, the right path is publishing/listing it as a community plugin or skill rather than keeping it open as a core OpenClaw issue.
