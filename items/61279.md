## Summary

Keep open. Current `main` and the latest release still run the image as `USER node` without pre-creating `/home/node/.openclaw` with node ownership, so the reported named-volume startup failure is not clearly fixed in the image itself. The repo only shows workarounds in the Docker setup script and docs, and the linked fix PR is still only referenced from the issue discussion.
