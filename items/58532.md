## Summary

Current `main` already handles the persisted-live-selection/failover conflict well enough to close this. The live-switch path is now gated on `liveModelSwitchPending`, fallback treats `LiveSessionModelSwitchError` as a failover signal instead of re-entering the same model, and the outer runner has a hard retry bound plus regression tests covering the infinite-loop case.

## Evidence

- **Live switch no longer blindly reapplies persisted selection:** `shouldSwitchToLiveModel()` only returns a persisted selection when `liveModelSwitchPending` is set, and clears stale pending state when the current model already matches.
  - file: [src/agents/live-model-switch.ts:124](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/live-model-switch.ts#L124)

- **Fallback now absorbs live-switch conflicts:** `runWithModelFallback()` converts `LiveSessionModelSwitchError` into a failover path and continues to the next candidate instead of rethrowing into an outer retry loop.
  - file: [src/agents/model-fallback.ts:921](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/model-fallback.ts#L921)
