## Summary

Keep open. Current `main` still lacks any JSON-syntax truncation handling, so the reported leak remains plausible, but this PR is not conservative enough to close out the item: it only patches `formatAssistantErrorText`, while errored assistant content still flows through `sanitizeUserFacingText`, and broad JSON-syntax matching can also misclassify unrelated local `JSON.parse` failures as stream truncation.

## Evidence

- **Raw fallback still exists on main:** `formatAssistantErrorText` handles invalid streaming event order, but there is still no JSON-syntax-specific branch before the final raw-string fallback, so unclassified parse errors still return to the user verbatim.

## Risks / Open Questions

- Merging the PR as-is can still leak the raw parse line through `sanitizeUserFacingText`, even if `formatAssistantErrorText` becomes friendlier.
- Broad JSON-syntax matching can misreport malformed local history or persisted signature data as provider stream truncation, sending users toward retrying instead of repairing/resetting corrupted session state.
