## Summary

Current main already provides agent memory persistence and reuse: memory files/search, default pre-compaction memory flush, and a bundled `/new`/`/reset` session-memory capture path. The exact UX is not a dedicated `/learn` slash command, but the requested behavior is implemented well enough on main and was already present by release `v2026.4.22`.

## Risks / Open Questions

- There is still no dedicated `/learn` slash command; the current UX is natural-language memory writes plus the bundled `session-memory` hook.
- The `/new`/`/reset` capture path is exposed as a bundled hook, so operators must enable that hook if they want lifecycle capture on reset/new.

## Close Comment

Closing after a Codex review because current main already covers the substance of this request. OpenClaw now has persisted agent memory and retrieval (`docs/concepts/memory.md:9`), a built-in pre-compaction memory flush enabled by default (`docs/reference/session-management-compaction.md:324`), and a bundled `session-memory` hook that saves the previous session into `memory/YYYY-MM-DD-slug.md` on `/new` and `/reset` (`docs/automation/hooks.md:152`, `src/hooks/bundled/session-memory/handler.ts:53`). Cross-session reuse is also covered by the `/new` startup-memory path (`src/gateway/server-methods/agent.test.ts:1123`). This was already present by release `v2026.4.22` at commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. The UX differs from the proposed dedicated `/learn` slash command, but the requested memory capture and reuse behavior is implemented on main well enough; if a dedicated `/learn` alias is still wanted, please open a narrower follow-up request for that specific command surface.
