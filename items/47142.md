## Summary

Current main already handles gateway self-restart conservatively: supervised installs restart through the service manager, in-process SIGUSR1 restarts exit and let the supervisor relaunch instead of leaving sibling gateways behind, and unmanaged restart refuses to proceed when multiple listeners are present.

## Evidence

## Close Comment

Closing from a Codex review because current OpenClaw already implements the requested behavior. On Linux, `openclaw gateway restart` goes through systemd (`src/daemon/systemd.ts:604`), and in-process restart handling releases the gateway lock, detects supervised environments, and exits so the supervisor restarts the gateway instead of leaving a sibling process behind (`src/cli/gateway-cli/run-loop.ts:82`, `src/infra/process-respawn.ts:20`, `src/infra/supervisor-markers.ts:24`). For unmanaged runs, restart also refuses to continue when multiple gateway listeners are present (`src/cli/daemon-cli/lifecycle.ts:124`). The same implementation is already present in the `v2026.4.22` release snapshot as well. On that basis this item looks implemented rather than still open.
