## Summary

Issue #47004 describes the 2026.3.13 Synology Chat reply path that depended on username/nickname-to-Chat-user resolution. Current `main` no longer does that by default: reply delivery stays bound to the stable webhook `user_id`, the legacy name-matching path is behind `dangerouslyAllowNameMatching`, and regression tests/docs cover that behavior. The changelog records this as shipped in release 2026.3.22.
