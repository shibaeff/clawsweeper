## Summary

Keep open. Current `main` does not implement prompt-less Slack slash commands or a `contextAware` config surface, so this is not `implemented_on_main`. The PR is also stale against the current Slack extension layout and would need a real rebase/port plus directive-safety review before a maintainer cleanup close would be justified.

## Evidence

- **Empty Slack slash prompts are still rejected on main:** `handleSlashCommand` still returns `"Message required."` when `prompt.trim()` is empty, so invoking a Slack slash command with no text is not supported on current `main`.
  - file: [extensions/slack/src/monitor/slash.ts:326](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/slack/src/monitor/slash.ts#L326)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **No `contextAware` option in resolved Slack slash config:** The resolved Slack slash config only includes `enabled`, `name`, `sessionPrefix`, and `ephemeral`; there is no `contextAware` field on current `main`.
  - file: [extensions/slack/src/monitor/commands.ts:19](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/slack/src/monitor/commands.ts#L19)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Public config/schema/docs also omit the option:** `SlackSlashCommandConfig` only defines `enabled`, `name`, `sessionPrefix`, and `ephemeral`; the Slack channel schema and docs list the same defaults, which shows the feature is not shipped or documented.
  - file: [src/config/types.slack.ts:81](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/config/types.slack.ts#L81)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **PR is stale against the current code layout:** The PR touches `src/slack/monitor/*`, but current Slack monitor code lives under `extensions/slack/src/monitor/*`; `find src -maxdepth 2 -type d -name 'slack' -o -path 'src/slack*'` returned no matching Slack subtree on this checkout.
  - command: `find src -maxdepth 2 -type d -name 'slack' -o -path 'src/slack*'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Current directive parsing would need careful handling for history-injected prompts:** Directive parsing prefers `CommandBody`/`RawBody` and otherwise falls back to prompt text. A context-aware slash implementation that injects recent channel history into those fields would need explicit hardening so historical `/think`-style tokens cannot affect the invocation.
  - file: [src/auto-reply/reply/get-reply-directives.ts:75](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/auto-reply/reply/get-reply-directives.ts#L75)
