## Summary

Current `main` already removes the unconditional session-override switch path that would pull fallback attempts back to the primary model, and it now treats `LiveSessionModelSwitchError` as failover instead of looping.

## Evidence

- **Live switch is now explicitly gated:** `shouldSwitchToLiveModel` only returns a persisted selection when `liveModelSwitchPending` is set; if the flag is absent, it returns `undefined`, so a stored session override alone no longer forces a model switch during fallback.
  - file: [src/agents/live-model-switch.ts:124](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/live-model-switch.ts#L124)
  - command: `nl -ba src/agents/live-model-switch.ts | sed -n '120,190p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Only explicit model-change flows set the pending flag:** Session override writes set `liveModelSwitchPending` only when callers pass `markLiveSwitchPending: true`, which is done by explicit `/model` and gateway session-patch flows rather than fallback rotation.
  - file: [src/sessions/model-overrides.ts:115](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/sessions/model-overrides.ts#L115)
  - command: `nl -ba src/sessions/model-overrides.ts | sed -n '1,140p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Active-run switch path uses the pending-flag gate:** The embedded runner now asks `shouldSwitchToLiveModel(...)` during an active attempt and throws `LiveSessionModelSwitchError` only from that gated path, replacing the older unconditional pre-attempt conflict behavior described in the report.
  - file: [src/agents/pi-embedded-runner/run.ts:974](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/pi-embedded-runner/run.ts#L974)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '968,998p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Fallback no longer dead-loops on switch conflicts:** `runWithModelFallback` catches `LiveSessionModelSwitchError`, converts it into a normal failover record, and continues to the next candidate instead of rethrowing and restarting the overloaded model.
  - file: [src/agents/model-fallback.ts:921](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/model-fallback.ts#L921)
  - command: `nl -ba src/agents/model-fallback.ts | sed -n '918,940p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Regression coverage matches the reported failure mode:** Tests assert that no switch happens when `liveModelSwitchPending` is false and that fallback continues past `LiveSessionModelSwitchError` rather than looping.
  - file: [src/agents/live-model-switch.test.ts:399](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/live-model-switch.test.ts#L399)
