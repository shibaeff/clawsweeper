## Summary

Keep open. Current `main` does not implement disk-backed or cross-process Discord component registry persistence, so the PR's requested behavior is still missing. The superseded-by-`#66241` comment is useful context, but without equivalent landed code on `main`, conservative cleanup should not auto-close this PR.
