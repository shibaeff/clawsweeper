## Summary

Keep open. Current `main` does not implement this PR’s core change: the skill scanner still only treats JS/TS-family files as scannable, while skill install security scanning still depends on that scanner. The reported blind spot for Python and shell skill scripts therefore remains real on `main`.

## Evidence

- **Current scanner only allows JS/TS extensions:** `SCANNABLE_EXTENSIONS` contains only `.js`, `.ts`, `.mjs`, `.cjs`, `.mts`, `.cts`, `.jsx`, `.tsx`, and `isScannable()` is a direct `path.extname()` membership check. No `.py`, `.sh`, or `.bash` support is present on current `main`.
  - file: [src/security/skill-scanner.ts:39](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/security/skill-scanner.ts#L39)
