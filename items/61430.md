## Summary

Keep open. Current `main` still does not add a `description`/`intent`/`reason` field to the core OpenClaw `exec` tool schema or docs. There is a related implementation in the bundled Codex app-server path that carries `reason` and forwards it into approval text, but that is a separate approval protocol and does not satisfy the generic `exec` tool request in this issue.
