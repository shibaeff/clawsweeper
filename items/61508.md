## Summary

Current main already sends periodic websocket keepalive traffic (`tick` every 30s), the browser client reconnects automatically after closes, and the Control UI reconnect path clears orphaned run state. The report’s specific "missing keepalive + refresh required" behavior no longer matches current main.
