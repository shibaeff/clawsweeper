## Summary

Current `main` already has built-in context compaction for long-running agent sessions, plus a pluggable compaction-provider extension point that matches the requested middleware-style compression hook.

## Close Comment

Closing as implemented on current `main` after a Codex review. OpenClaw already ships built-in session compaction that summarizes older history and keeps recent turns available (`docs/reference/session-management-compaction.md:202`), and it also exposes a pluggable compaction-provider hook for middleware-style summarizers via `registerCompactionProvider()` / `agents.defaults.compaction.provider` (`src/plugins/compaction-provider.ts:1`, `src/agents/pi-hooks/compaction-safeguard.ts:795`, `src/config/types.agent-defaults.ts:425`). The shipped changelog also records this provider-backed compaction path in `CHANGELOG.md:1071` (`2026.4.7`). If you want to integrate `context-pack`, the current extension point is a compaction provider plugin rather than a new core feature request.
