## Summary

Keep open. Current main already narrowed the skills watcher to `SKILL.md` targets, so that part of the PR is stale, but the memory watcher on current `main` still uses recursive `**` watch globs with no depth cap, so the reported FD-exhaustion problem remains plausible. The PR is also not a conservative close candidate because its proposed memory changes introduce review-identified regression risk.

That makes the PR's `depth: 2` skills hunk largely obsolete, but only for the skills side.
