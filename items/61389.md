## Summary

Keep open. Current `main` still rejects `gateway.channelMaxRestartsPerHour: 0`, and while the stale-threshold half of the report is already covered by cross-field validation, the requested behavior is not implemented end-to-end.

## Evidence

- **Schema still requires both values >= 1:** Current `main` keeps `channelStaleEventThresholdMinutes: z.number().int().min(1)` and `channelMaxRestartsPerHour: z.number().int().min(1)` in the gateway config schema.
  - file: [src/config/zod-schema.ts:747](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/zod-schema.ts#L747)

  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Stale-threshold guard already exists:** A `superRefine` check rejects `channelStaleEventThresholdMinutes` when it is below the effective health-check interval, including the default 5-minute interval when `channelHealthCheckMinutes` is omitted.
  - file: [src/config/zod-schema.ts:887](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/zod-schema.ts#L887)
  - command: `nl -ba src/config/zod-schema.ts | sed -n '742,900p'`

- **Tests cover the stale-threshold behavior:** Config tests explicitly reject `channelStaleEventThresholdMinutes: 4` both when `channelHealthCheckMinutes` is 5 and when it is omitted, and accept 5/6 with a 5-minute interval.
  - file: [src/config/config-misc.test.ts:312](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/config-misc.test.ts#L312)
