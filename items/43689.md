## Summary

Current `main` already contains explicit Slack socket disconnect/error handling and regression tests that cover the formerly dangling reconnect path, so the reported `[openclaw] Unhandled promise rejection: undefined` after a Slack Socket Mode reconnect is implemented/fixed on `main` and was already shipped in `v2026.4.22`.

## Evidence

- **Slack socket reconnect path now awaits a managed disconnect waiter:** `monitorSlackProvider` uses `startSlackSocketAndWaitForDisconnect(...)` inside the reconnect loop, then handles the returned disconnect/error payload synchronously before retrying, instead of leaving a socket error callback to leak a rejection.
  - file: [extensions/slack/src/monitor/provider.ts:441](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/slack/src/monitor/provider.ts#L441)
  - command: `nl -ba extensions/slack/src/monitor/provider.ts | sed -n '436,516p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Disconnect waiter preserves Slack error payloads:** `waitForSlackSocketDisconnect` resolves on `disconnected`, `error`, and `unable_to_socket_mode_start`, carrying the original `error` value through the reconnect path.
  - file: [extensions/slack/src/monitor/reconnect-policy.ts:45](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/slack/src/monitor/reconnect-policy.ts#L45)

## Close Comment

Closing from a Codex review because the Slack Socket Mode reconnect path is already implemented on current `main`, and the same fix surface shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The current code now routes socket disconnect/error events through a managed waiter and reconnect loop (`extensions/slack/src/monitor/provider.ts:441`, `extensions/slack/src/monitor/reconnect-policy.ts:45`, `extensions/slack/src/monitor/provider-support.ts:151`), and there are regression tests covering the formerly dangling cases, including `onStarted` throwing and `unable_to_socket_mode_start` preserving its error payload (`extensions/slack/src/monitor/provider.reconnect.test.ts:109`). That means the reported `Unhandled promise rejection: undefined` path is already addressed on supported builds. If you still hit this on `v2026.4.22` or newer, please open a fresh issue with updated logs and exact Slack channel/plugin config details.
