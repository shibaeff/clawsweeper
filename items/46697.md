## Summary

Keep open. Current `main` still implements `sessions_send` A2A delivery as a target-authored announce message sent to the visible channel/thread, not as direct posting of the full target reply. The docs and tests on `main` match that current behavior, and the PR discussion already records unresolved correctness concerns in the proposed patch.

There is no code on `main` that posts `latestReply` directly to the thread/channel first.
