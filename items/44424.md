## Summary

Keep open. Current `main` clearly supports Telegram forum `threadId` routing, but there is no strong evidence that the reported intermittent cross-topic failure is fixed. The outbound send path still only retries non-idempotent sends for proven pre-connect/rate-limit failures, and tests explicitly keep generic grammY `Network request for 'sendMessage' failed...` envelopes as hard failures. That matches the reported symptom closely enough that conservative cleanup should not close this issue.
