## Summary

Keep open. Current main has a shipped Linux child-process OOM mitigation from #70419, but the evidence only shows OOM-victim biasing and generic signal reporting, not that intermittent SIGKILLs for broad exec discovery commands are actually eliminated or reclassified. I did not find a targeted reproduction or regression proof for this exact report, so closing would be speculative.

## Evidence

- **Timeline points at a related but different fix:** The provided issue timeline cross-references PR #70419. That supports relation to the same symptom family, but the PR title and shipped changelog entry are about raising child `oom_score_adj` under Linux OOM pressure, not about eliminating intermittent SIGKILLs for broad `find`/`grep -R` exec discovery commands.
- **Shipped release contains the Linux OOM-score mitigation:** The `v2026.4.23` changelog says OpenClaw now wraps gateway-managed supervisor, PTY, MCP stdio, and browser child processes in a `/bin/sh` shim that raises the child's `oom_score_adj` on Linux. The entry explicitly references issue `#70404` and PR `#70419`, not issue `#69242`.
