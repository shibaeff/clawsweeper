## Summary

Keep open. Current `main` already documents cron scheduling and Telegram delivery, so the new docs in this PR are mostly overlap, but the substantive new work is an unmerged standalone Telethon script that the review thread identified as carrying hardcoded Telegram credentials, machine-specific paths, and backlog-loss/portability bugs.
