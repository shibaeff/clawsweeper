## Summary

Keep open. Current `main` has related strict-agentic protections for GPT-5 OpenAI/Codex runs, but it still does not implement the specific runtime guard this issue asks for: a final-reply truthfulness check that blocks or rewrites unbacked later-follow-up promises after a turn that already made some tool progress. The reported `memory_search` -> promise-style final reply path remains plausibly open.

## Evidence

- **`memory_search` is a distinct tool, not one of the narrow single-action retry-safe tools:** The tool catalog registers `memory_search` as its own tool id/label.
  - file: [src/agents/tool-catalog.ts:129](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/tool-catalog.ts#L129)
- **Single-action narrative retry bypass is narrow:** The only retry-safe single non-plan tools are `read`, `search`, `find`, `grep`, `glob`, and `ls`.
  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:72](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L72)
- **Planning-only guard still treats prior non-plan tool activity as progress:** `resolvePlanningOnlyRetryInstruction` returns `null` when there was non-plan tool activity unless it matches that narrow bypass. A turn that already did `memory_search` and then says "I'll check and come back" would not be caught here.
  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:560](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L560)
- **Existing regression test accepts tool-use-then-narration as non-retriable:** The test with prior `read`/`search` tool activity expects no planning-only retry, confirming the current guard is not a general "unbacked follow-up promise" check.
  - file: [src/agents/pi-embedded-runner/run.incomplete-turn.test.ts:560](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-embedded-runner/run.incomplete-turn.test.ts#L560)
- **No reply-pipeline follow-up-truthfulness guard found:** Targeted search found no relevant runtime code that rewrites or annotates final replies for unbacked "I'll come back later" / follow-up commitments, unlike the issue's suggested post-processing fix.
  - command: `rg -n "did not schedule a reminder|I did not schedule|come back|follow-up commitment|follow up commitment|unbacked|I'll check.*come back|send a short follow-up|Потом отпишусь|Сделаю это и вернусь" src/agents src/infra src/channels docs test`
