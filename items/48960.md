## Summary

Keep open. Current `main` still silently drops malformed skills and does not include the detailed loading diagnostics this PR proposes, and the review thread surfaced a credible test-contract issue that still needs resolution before merge.

## Evidence

- **Malformed skills still fail silently on main:** `loadSingleSkillDirectory` returns `null` when frontmatter parsing fails or when required metadata is missing, with no logging or diagnostic payload. That means the underlying silent-failure behavior this PR targets still exists on `main`.

It does not log inaccessible roots, `SKILL.md` parse failures, frontmatter failures, per-source completion counts, or an overall loading summary before returning results.
