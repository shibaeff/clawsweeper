## Summary

Keep open. Current `main` still sends memory remote embedding requests through `fetchWithSsrFGuard` without opting into trusted env proxy mode, while the guarded fetch implementation and tests explicitly preserve the default behavior of ignoring env proxy settings in strict mode.

## Evidence

- **Memory remote path still uses default guarded fetch mode:** `withRemoteHttpResponse(...)` calls `fetchWithSsrFGuard(...)` with only `url`, `init`, `policy`, and `auditContext`; it does not pass `mode: "trusted_env_proxy"` or any proxy-aware wrapper.
  - file: [packages/memory-host-sdk/src/host/remote-http.ts:22](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/packages/memory-host-sdk/src/host/remote-http.ts#L22)
  - command: `nl -ba packages/memory-host-sdk/src/host/remote-http.ts | sed -n '1,120p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Gemini memory embeddings use that memory remote helper:** The Gemini embedding provider posts both single and batch embedding requests through `withRemoteHttpResponse(...)`, so this code path applies directly to `memory_search` with Gemini embeddings.
  - file: [extensions/google/embedding-provider.ts:173](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/google/embedding-provider.ts#L173)
  - command: `nl -ba extensions/google/embedding-provider.ts | sed -n '170,215p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Guarded fetch only honors env proxy in trusted mode:** `fetchWithSsrFGuard(...)` resolves its mode, then only creates an `EnvHttpProxyAgent` when `mode === GUARDED_FETCH_MODE.TRUSTED_ENV_PROXY` and proxy env vars are present. Otherwise it continues with the strict pinned-DNS path.
  - file: [src/infra/net/fetch-guard.ts:311](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/infra/net/fetch-guard.ts#L311)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '300,390p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Tests lock in the strict-mode behavior:** The SSRF guard tests explicitly assert that env proxy is ignored by default in strict mode, and only used when trusted proxy mode is enabled.
  - file: [src/infra/net/fetch-guard.ssrf.test.ts:1007](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/infra/net/fetch-guard.ssrf.test.ts#L1007)

  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Other provider HTTP helpers already added an auto-upgrade path that memory does not use:** `fetchWithTimeoutGuarded(...)` in the shared provider HTTP helper auto-upgrades to trusted env proxy mode when proxy env vars are configured and `NO_PROXY` does not apply. The memory embedding path does not route through this helper.
