## Summary

Current `main` appears to have fixed the narrow loader-side-effect bug, but the broader stale-route report is not proven resolved. Legacy `codex/*` compatibility is still intentionally present, session/model selection still honors persisted overrides ahead of runtime fields, docs say harness choice is pinned per session until reset, and there is no in-tree `codex-proxy/*` normalization path to show the reporter's custom route is handled.
