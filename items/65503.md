## Summary

Current main contains direct fixes for the reported Codex dirty-path symptoms: cumulative Codex totals are no longer surfaced as fresh per-turn usage, native request_user_input state is cleared on resolution/abort, queued follow-up input is steered separately, and approval denial/cancel paths now resolve through the final decision flow.

## Evidence

- **Cumulative-only usage no longer inflates per-turn usage:** The Codex event projector now reads only current/last usage payloads from `thread/tokenUsage/updated`; if the app-server only sends a cumulative `total` block, OpenClaw leaves `attemptUsage` unset instead of treating that total as fresh context usage. The paired test uses a 1,000,000-token cumulative total and asserts zero fresh assistant usage.
