## Summary

Keep open. Current `main` has partial downstream handling for trailing `NO_REPLY` in cron delivery, but the broad own-line suppression described in PR #46626 is not implemented across the remaining paths. Gateway/chat-history and subagent follow-up still rely on exact-token suppression, and current tests still encode delivery of substantive text followed by `NO_REPLY` rather than treating that pattern as silent.

## Evidence

- **Silent-token helper on main is still exact-match only:** `isSilentReplyText()` matches only the exact token with surrounding whitespace, `isSilentReplyPayloadText()` only adds the JSON envelope case, and there is no `isSilentTokenOnOwnLine()` helper on current `main`. That means `text\n\nNO_REPLY` is not recognized as silent by the shared token helpers.
