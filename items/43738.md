## Summary

Keep open. Current `main` still implements only `daily`/`idle` reset modes and evaluates combined daily+idle expiry with OR semantics (`staleDaily || staleIdle`). The requested `adaptive` AND-logic mode is not present in config types, schema, docs, or tests, and the linked implementation PR is still open in the provided GitHub context.
