## Summary

The request is a valid workflow idea, but current main already exposes plugin hooks that cover the needed activity tracking, prompt injection, soft-stop, and startup/background wiring. Under the repo vision, that kind of optional operational policy belongs in a plugin/ClawHub package rather than OpenClaw core.

## Evidence
