## Close Comment

Closing from a Codex review because this looks like a better fit for ClawHub/community-plugin distribution than for OpenClaw core. Current project direction says optional capability should usually ship as plugins, plugin authors should host and maintain them in their own repository, and the bar for adding optional plugins to core is intentionally high (`VISION.md:54`, `VISION.md:57`, `VISION.md:59`). The plugin docs also explicitly say new plugins do not need to be added to this repo and should be published to ClawHub or npm instead (`docs/plugins/building-plugins.md:16`), with ClawHub called the canonical discovery surface for community plugins (`docs/plugins/community.md:9`). On current `main`, the shipped bundled-plugin set documented in `docs/tools/plugin.md:85` does not include a `guardian` plugin. Given that scope guidance, plus this PR’s size as a new 5,186-line bundled plugin, I’m closing this here and would encourage publishing it as a community plugin/ClawHub package instead.
