## Summary

Current `main` still has the reported mismatch: the TUI footer renders `?/ctx` when a session’s total token count is exactly `0`, while the status-message path can render `0/ctx`. The issue is therefore not implemented on `main` and remains actionable.

## Evidence

- **Gateway session listing drops zero totals:** `resolvePositiveNumber` only returns values greater than 0, so a fresh `totalTokens` value of `0` is discarded before `sessions.list` builds the row.
  - file: [src/gateway/session-utils.ts:223](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/session-utils.ts#L223)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '220,225p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **sessions.list converts zero totals to missing:** `totalTokens` for session rows is computed with `resolvePositiveNumber(resolveFreshSessionTotalTokens(entry)) ?? resolvePositiveNumber(transcriptUsage?.totalTokens)`, so `0` becomes `undefined` even when freshness is true.
  - file: [src/gateway/session-utils.ts:1330](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/session-utils.ts#L1330)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1292,1336p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **TUI footer renders missing totals as question mark:** The footer uses `formatTokens(sessionInfo.totalTokens ?? null, sessionInfo.contextTokens ?? null)`, and `formatTokens` prints `tokens ?/<ctx>` when total is null.
  - file: [src/tui/tui.ts:827](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/tui/tui.ts#L827)
  - command: `nl -ba src/tui/tui.ts | sed -n '816,843p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **TUI formatter behavior matches the reported symptom:** `formatTokens` explicitly formats `total == null` as `?`, which is the exact `?/200k` display from the report when the session row omits zero totals.
  - file: [src/tui/tui-formatters.ts:373](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/tui/tui-formatters.ts#L373)
  - command: `nl -ba src/tui/tui-formatters.ts | sed -n '373,385p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Status/session_status path can still show 0 correctly:** The status-message formatter only treats `null`/`undefined` as unknown; a numeric `0` is formatted as `0/<ctx>`, which explains why `/status` or `session_status` can show `0/200k` while the TUI footer shows `?/200k`.
  - file: [src/status/status-message.ts:223](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/status/status-message.ts#L223)
