## Summary

Current `main` still classifies metadata-only poll fields as poll intent on `action="send"`, and the existing tests lock that behavior in. PR #65295 appears to fix a real unfixed bug with targeted coverage, so conservative cleanup should leave it open.

## Evidence

- **Current send guard still rejects any poll creation params:** At current `main` SHA `1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19`, `runMessageAction()` throws for `action === "send"` whenever `hasPollCreationParams(params)` is true. There is no narrower `hasRealPollIntent` check on `main`.
