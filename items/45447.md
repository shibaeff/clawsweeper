## Summary

Current main already handles the empty `--all` path as a clean no-op, and the deprecated `openclaw hooks update` command forwards into that implementation. The shipped `v2026.4.22` tree also contains the fix and a regression test.

## Close Comment

Closing as implemented on main. This was a Codex review against the current tree, and the deprecated `openclaw hooks update` path now forwards into `runPluginUpdateCommand` (`src/cli/hooks-cli.ts:542`). That shared updater explicitly handles the empty `--all` case by logging `No tracked plugins or hook packs to update.` and returning cleanly instead of falling through to the usage error (`src/cli/plugins-update-command.ts:33`). There is also a regression test covering the exact empty-install-records scenario (`src/cli/plugins-cli.update.test.ts:127`). The same behavior is already present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so there is nothing left to fix on current main.
