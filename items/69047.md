## Summary

Keep open. Current `main` still exposes `ctx.channelRuntime` to external channel plugins through a loose `ChannelRuntimeSurface` with an index signature, while the concrete `reply.dispatchReplyWithBufferedBlockDispatcher` shape remains defined only on internal runtime types. The issue's core SDK typing gap is still present, and there is no public stable reply-surface type or shared confirmation-dispatch factory on `main`.

## Evidence
