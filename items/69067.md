## Summary

Current `main` already suppresses Slack non-DM verbose/tool-progress delivery, including `ChatType: "channel"`, and has regression coverage for the reported Slack-channel case. The latest tagged release still had the older group-only gate, so this looks fixed on `main` but not yet shipped in a release.

## Evidence

- **Current main suppresses Slack non-direct verbose progress/tool summaries:** `dispatchReplyFromConfig` now computes `isSlackNonDirectSurface` and gates verbose progress/tool summaries through `shouldSendVerboseProgressMessages`, which is false for Slack when `ChatType !== "direct"`.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:677](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/auto-reply/reply/dispatch-from-config.ts#L677)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Regression test covers Slack channel sessions:** A dedicated test sets `Provider: "slack"`, `Surface: "slack"`, `ChatType: "channel"`, enables verbose mode, and asserts `dispatcher.sendToolResult` is not called.
  - file: [src/auto-reply/reply/dispatch-from-config.test.ts:1498](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/auto-reply/reply/dispatch-from-config.test.ts#L1498)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Slack channel traffic still maps to ChatType=channel:** Slack inbound/session context continues to map non-DM, non-MPIM messages to `ChatType: "channel"`, so the new guard covers the exact surface described in the issue.
  - file: [extensions/slack/src/monitor/context.ts:195](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/slack/src/monitor/context.ts#L195)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Latest release still had the old group-only gate:** At release `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`), the same file still used `const shouldSendToolSummaries = ctx.ChatType !== "group" || ctx.IsForum === true;`, matching the bug report rather than the current main behavior.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:643](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/auto-reply/reply/dispatch-from-config.ts#L643)
