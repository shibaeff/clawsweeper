## Summary

Keep open. The specific root-cause text in #47229 is outdated on current `main` because OpenClaw no longer uses the Anthropic SDK streaming parser in this path, but the underlying bug still appears plausible on current `main`: Anthropic `tool_use` `input_json_delta` content is reparsed through `parseStreamingJson`, and the pinned `@mariozechner/pi-ai` implementation still falls back to `JSON.parse` for complete JSON, which loses precision for integers above `Number.MAX_SAFE_INTEGER`. I did not find an in-repo fix or a regression test covering unsafe large integers on the Anthropic transport path.
