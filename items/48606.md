## Summary

Keep open. Current `main` still lacks the macOS known-path fallback this PR proposes, so the reported Edge/LaunchServices mismatch remains plausible on current code. The PR is directionally correct but still needs the missing regression coverage called out in review.

## Evidence

- **Current macOS detector still hard-fails without fallback:** `detectDefaultChromiumExecutableMac()` accepts LaunchServices bundle IDs like `com.microsoft.edgemac`, but if `osascript`, `defaults`, or the resolved executable path fails, it immediately returns `null`. There is no `KNOWN_BUNDLE_ID_PATHS` or equivalent fallback on current `main`.
  - file: [extensions/browser/src/browser/chrome.executables.ts:185](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/extensions/browser/src/browser/chrome.executables.ts#L185)
  - command: `nl -ba extensions/browser/src/browser/chrome.executables.ts | sed -n '180,230p'`
  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)
- **Fallback search order can still pick Chrome ahead of Edge:** When default-browser detection returns `null`, `resolveBrowserExecutableForPlatform()` falls through to `findChromeExecutableMac()`, whose candidate order checks Chrome before Edge. That matches the bug described in the PR and maintainer comment.
  - file: [extensions/browser/src/browser/chrome.executables.ts:488](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/extensions/browser/src/browser/chrome.executables.ts#L488)

  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)
- **Current tests do not cover the missing fallback behavior:** `main` has a success-path test for `com.microsoft.edgemac`, but the failure-path test explicitly expects Chrome when `osascript` returns no app path. There is no regression test for `com.microsoft.edgemac` plus failed `osascript/defaults` plus an existing known Edge path returning Edge.

## Risks / Open Questions

- Closing this PR would drop a still-plausible macOS default-browser bug where OpenClaw can launch Chrome instead of the configured default Edge when LaunchServices bundle IDs and app bundle IDs differ.
- Merging as-is would still be risky without the missing regression tests for failed `osascript`/`defaults` resolution and the no-known-path case.
