## Summary

Keep open. Current `main` still implements cron delivery as a single primary target plus optional `failureDestination`; it does not ship multi-channel `additionalTargets`, and the PR discussion already identified unresolved gaps in the proposed implementation.

## Evidence

- **Current cron delivery type has no multi-target field:** `CronDelivery` on current `main` includes only `mode`, `channel`, `to`, `threadId`, `accountId`, `bestEffort`, and `failureDestination`; there is no `additionalTargets` field in the persisted/runtime contract.

  - file: [src/gateway/protocol/schema/cron.ts:180](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/gateway/protocol/schema/cron.ts#L180)
