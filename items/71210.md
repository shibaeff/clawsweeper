## Summary

Current `main` already fixes this CLI/local-embedding lookup path by declaring the built-in `local` provider in the `memory-core` manifest and resolving embedding providers through the capability fallback loader when the runtime registry is still cold. The reported failure still matches released `v2026.4.23`, but the fix is present on `main` and documented in the unreleased changelog.

## Evidence

- **Current main declares `local` in the memory-core manifest:** `extensions/memory-core/openclaw.plugin.json` now includes `contracts.memoryEmbeddingProviders: ["local"]`, which is the contract standalone CLI paths need before the full Gateway runtime has registered providers.
  - file: [extensions/memory-core/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/extensions/memory-core/openclaw.plugin.json#L4)
  - command: `nl -ba extensions/memory-core/openclaw.plugin.json | sed -n '1,12p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Capability fallback loader uses manifest contracts:** `resolveBundledCapabilityCompatPluginIds()` filters bundled plugins by `plugin.contracts.memoryEmbeddingProviders`, and `resolvePluginCapabilityProvider()` loads that plugin registry with `activate: false` when the active runtime does not already have the provider.
  - file: [src/plugins/capability-provider-runtime.ts:46](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/plugins/capability-provider-runtime.ts#L46)
  - command: `nl -ba src/plugins/capability-provider-runtime.ts | sed -n '40,90p' && nl -ba src/plugins/capability-provider-runtime.ts | sed -n '201,229p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Memory embedding lookup now falls back to capability providers:** `getMemoryEmbeddingProvider()` and `listMemoryEmbeddingProviders()` first check registered adapters, then fall back to capability-provider resolution. That is the code path that prevents `Unknown memory embedding provider: local` when CLI startup has not eagerly registered providers yet.
  - file: [src/plugins/memory-embedding-provider-runtime.ts:17](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/plugins/memory-embedding-provider-runtime.ts#L17)

- **Regression test covers cold-registry fallback:** `src/plugins/memory-embedding-provider-runtime.test.ts` includes a test named `falls back to declared capability adapters when the registry is cold`, asserting the provider is resolved through the capability path even with an empty registration store.
