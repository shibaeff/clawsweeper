## Summary

Keep open. Current `main` does not implement this behavior: malformed `SKILL.md` files are still dropped during local skill loading, and the current status/CLI surfaces still have no `parseError` field or dedicated parse-error output. The PR is also incomplete against the stated goal because the review feedback about `skills info` and `skills check` still points to real missing handling.
