## Summary

Keep open. Current `main` has mitigation for compaction loss via documented pre-compaction memory flushes and `/status`/session token counters, but the specific requested passive `context=X%` runtime signal is still not implemented in the system-prompt path. The same docs also still describe `/new`, `/reset`, and daily reset as creating a new `sessionId`, so the reset-loss half of the report is not clearly resolved. ([github.com](https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt.ts))

## Evidence

- **Runtime line still lacks a context field:** `src/agents/system-prompt.ts` still builds the Runtime section via `buildRuntimeLine(...)`, and there is no `contextPercent` symbol in that file. The runtime builder’s visible fields are the existing runtime info properties, not a usage percentage. ([github.com](https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt.ts))
  - file: `src/agents/system-prompt.ts:4000`
- **Runtime info type has no contextPercent:** `src/agents/system-prompt-params.ts` defines `RuntimeInfoInput` with `agentId`, `host`, `os`, `arch`, `node`, `model`, `defaultModel`, `shell`, `channel`, `capabilities`, `channelActions`, `repoRoot`, and `canvasRootDir`; `contextPercent` is not present. ([github.com](https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt-params.ts))
  - file: `src/agents/system-prompt-params.ts:493`
- **Embedded runner does not pass context usage into prompt params:** `src/agents/pi-embedded-runner/run/attempt.ts` calls `buildSystemPromptParams(...)` with `host`, `os`, `arch`, `node`, `model`, `defaultModel`, `shell`, `channel`, `capabilities`, and `channelActions`, and the file also has no `contextPercent` symbol. ([github.com](https://github.com/openclaw/openclaw/blob/main/src/agents/pi-embedded-runner/run/attempt.ts))
  - file: `src/agents/pi-embedded-runner/run/attempt.ts:3769`
- **Docs show a compaction mitigation, not the requested runtime visibility:** `docs/reference/session-management-compaction.md` documents session token counters as feeding `/status` and dashboards, and separately documents pre-compaction "memory flush" as implemented. That addresses some compaction loss, but it is not the requested passive runtime-line percentage. ([github.com](https://github.com/openclaw/openclaw/blob/main/docs/reference/session-management-compaction.md))
  - file: `docs/reference/session-management-compaction.md:405`
- **Reset/session-loss concern still appears live in docs:** The same session-management doc still says reset (`/new`, `/reset`) and daily reset create a new `sessionId`. I did not find corresponding documentation here for a pre-reset agentic flush or runtime percentage visibility that would make this issue obsolete. ([github.com](https://github.com/openclaw/openclaw/blob/main/docs/reference/session-management-compaction.md))
  - file: `docs/reference/session-management-compaction.md:356`
