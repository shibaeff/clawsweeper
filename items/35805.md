## Summary

The issue is asking OpenClaw core to enforce a workspace-specific file-age policy for custom governance files, but current main only treats a fixed bootstrap file set as agent context. The requested mtime-threshold alerting is useful, but it fits the existing plugin/skill/hook automation model and ClawHub scope better than OpenClaw core.

## Close Comment

Closing this as `clawhub` after a Codex review of current `main`. The requested behavior is a configurable freshness/alerting policy for custom governance files, but OpenClaw core currently only recognizes a fixed workspace bootstrap set (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, `MEMORY.md`) as agent context (`src/agents/workspace.ts:171`, `docs/concepts/agent-workspace.md:64`). The files called out here (`DAO.md`, `CONSTITUTION.md`, `AWARENESS.md`, `CLARA.md`, etc.) are not part of that core contract, and the docs explicitly steer optional/operator-specific automation into hooks, plugins, and ClawHub (`docs/automation/hooks.md:9`, `VISION.md:52`). A governance freshness checker with thresholds and channel alerts is a useful idea, but it fits a ClawHub skill/plugin or hook/cron package better than OpenClaw core.
