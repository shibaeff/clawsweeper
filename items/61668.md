## Summary

Keep open. Current `main` still routes native reply-dispatch output into outbound delivery without a shared fail-closed structured-output validation seam. The current behavior is normalization plus degradation/fallback, and the docs explicitly describe degradation instead of blocking send.

## Evidence

- **Reply dispatcher still hands normalized payloads straight to delivery:** `createReplyDispatcher` normalizes reply payloads, then immediately calls `options.deliver(normalized, { kind })`. There is no separate pre-send structured-output validator in this native handoff.
  - file: [src/auto-reply/reply/reply-dispatcher.ts:207](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/auto-reply/reply/reply-dispatcher.ts#L207)
  - command: `nl -ba src/auto-reply/reply/reply-dispatcher.ts | sed -n '200,260p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Structured presentation normalization is lossy, not fail-closed:** `normalizePresentationBlock` drops invalid/unsupported blocks, and `normalizeMessagePresentation` returns a cleaned payload when any title or valid blocks remain. That is normalization-by-pruning, not a blocking validation gate.
  - file: [src/interactive/payload.ts:203](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/interactive/payload.ts#L203)
  - command: `nl -ba src/interactive/payload.ts | sed -n '180,320p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Outbound delivery degrades presentation instead of rejecting it:** `deliverOutboundPayloads` normalizes payloads, runs `message_sending`, then `renderPresentationForDelivery` either renders natively or strips `presentation` and converts it to fallback text before send. The send path then proceeds via `sendPayload`/`sendText` without a canonical validation-failure branch for structured payloads.
  - file: [src/infra/outbound/deliver.ts:382](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/infra/outbound/deliver.ts#L382)

- **Docs define degradation as the intended behavior:** The public message-presentation docs say core normalizes presentation, falls back to conservative text when a renderer cannot handle it, and that unsupported native controls should degrade rather than fail the whole send. Only `delivery.pin.required` is documented as fail-closed.
