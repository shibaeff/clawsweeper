## Summary

Current `main` does not implement the requested per-agent `instructions` field or any equivalent config surface that appends to `SOUL.md`. The supported config still only exposes full `systemPromptOverride` replacement, while append-style behavior is available only through hooks/plugin machinery, not agent config.

## Evidence

- **Per-agent config still only exposes full override:** `AgentConfig` has `systemPromptOverride` as the prompt-related field; there is no `instructions`/append field in the typed agent config.
  - file: [src/config/types.agents.ts:76](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/config/types.agents.ts#L76)
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Runtime schema matches: no append field:** The runtime Zod schema for `agents.list[]` accepts `systemPromptOverride` but no `instructions` field, so current config parsing does not support the requested shape.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/config/zod-schema.agent-runtime.ts#L812)
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Docs still describe full replacement:** The config reference explicitly says `agents.defaults.systemPromptOverride` replaces the entire OpenClaw-assembled system prompt, including per-agent use.
  - file: [docs/gateway/configuration-reference.md:1328](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/docs/gateway/configuration-reference.md#L1328)
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Prompt assembly still short-circuits on override:** CLI prompt preparation resolves `systemPromptOverride` first and only calls `buildSystemPrompt(...)` when no override is present, so the normal SOUL/bootstrap prompt is replaced rather than extended.
  - file: [src/agents/cli-runner/prepare.ts:279](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/agents/cli-runner/prepare.ts#L279)
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Embedded runner override also replaces, not merges:** `createSystemPromptOverride()` returns a function that always yields the trimmed override string and ignores the default prompt input.
  - file: [src/agents/pi-embedded-runner/system-prompt.ts:92](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/agents/pi-embedded-runner/system-prompt.ts#L92)
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Only hook/plugin paths currently support append-style prompt mutation:** Docs show `before_prompt_build` can inject `appendSystemContext`, and `agent:bootstrap` can mutate injected bootstrap files like `SOUL.md`; these are extensibility hooks, not the requested simple per-agent config field.
  - file: [docs/concepts/agent-loop.md:94](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/docs/concepts/agent-loop.md#L94)
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)

## Risks / Open Questions

- The issue body uses the older `systemPrompt` term, but current `main` still has the same full-replacement behavior under `systemPromptOverride`, so the request remains coherent rather than obsolete.
- There are hook-based workarounds for appending prompt text, which may influence product direction, but they do not satisfy the requested built-in agent config surface.
