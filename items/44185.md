## Summary

Current `main` already honors `--repair` in non-interactive doctor flows. `createDoctorPrompter()` now auto-approves repair mode before the `nonInteractive` early return, and the released `v2026.4.22` tag contains that implementation.

## Evidence

- **Prompter now approves repair before non-interactive fallback:** `confirmDefault` first calls `shouldAutoApproveDoctorFix(repairMode)` and only then checks `repairMode.nonInteractive`, so `--repair` is no longer blocked by the non-interactive path.
