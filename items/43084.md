## Summary

Close as implemented on main. Current OpenClaw no longer re-sends Feishu typing reactions on each keepalive tick once the reaction exists, and typing cadence is already configurable globally via `typingIntervalSeconds`.

## Risks / Open Questions

- The interval knob is global/per-session or per-agent (`typingIntervalSeconds`), not a Feishu-only `channels.feishu.typingIndicatorIntervalMs` field. If maintainers require channel-specific cadence overrides, that would be a separate feature request.

## Close Comment

Closing based on a Codex review of current `main` and the latest release. The specific Feishu problem described here is already addressed: `extensions/feishu/src/reply-dispatcher.ts:141` short-circuits keepalive re-adds once a typing reaction already exists, so the repeated 3-second Feishu API refreshes from the report no longer happen. OpenClaw also already exposes typing cadence control through `typingIntervalSeconds` (`src/auto-reply/reply/get-reply.ts:231`, documented in `docs/concepts/typing-indicators.md:8` and `docs/gateway/config-agents.md:638`). This behavior is already present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so there is no remaining cleanup action on the issue as written.
