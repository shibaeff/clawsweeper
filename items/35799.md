## Summary

Keep open. Current `main` already implements deterministic heartbeat phase staggering and top-of-hour cron staggering, but the issue asks for more than that: prime-derived `every` cadence and/or explicit configurable jitter for heartbeat and cron `every` schedules. Those requested config/API surfaces are not present, and cron `every` still uses a fixed anchor-based interval with no prime/jitter mode.
