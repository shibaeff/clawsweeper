## Summary

Keep open. Current `main` still validates `channels.feishu` with a strict schema that does not include `footer` or `threadSession`, and there is no current docs/doctor migration path for those keys, so the startup failure described in the issue remains plausible. A history check against shipped `v2026.3.23` docs/schema did not corroborate the claim that those fields were supported in that release, so this looks more like an unresolved compatibility/migration question than a safe auto-close.
