## Summary

Keep open. Current main still does not provide the requested internal `agent:error` hook; the closest existing surface is the plugin lifecycle `agent_end` hook, which is a different API and only exposes a coarse `success` flag plus optional `error` string.

## Evidence

- **Internal hook docs still omit `agent:error`:** The internal hooks docs list command/session/bootstrap/message/gateway events, but no `agent:error` event.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/automation/hooks.md#L34)

- **Bundled hook README still calls agent errors a future event:** The bundled hook README says supported internal events stop at `agent:bootstrap`/`gateway:startup` and explicitly says more event types are coming soon, including agent errors.
  - file: [src/hooks/bundled/README.md:174](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/hooks/bundled/README.md#L174)
