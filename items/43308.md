## Summary

Keep open. Current `main` has a real observability foundation, but it is still partial relative to this request: diagnostics/OTel cover model usage and message-flow spans, and plugin hooks expose LLM/tool-call interception, yet there is no unified first-class tracing surface that emits tool-call/agent-execution spans through the diagnostics catalog/exporters described in the issue.

## Evidence

- **Diagnostics exist, but the built-in event catalog is narrower than the requested tracing scope:** Docs describe diagnostics as structured events for model runs and message-flow telemetry, and the catalog lists `model.usage`, webhook/message/session/queue events, but not dedicated tool-call, agent-execution, or nested per-run span events.
  - file: [docs/logging.md:176](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/docs/logging.md#L176)
