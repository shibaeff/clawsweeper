## Summary

Current main already routes main-session bundle MCP through the shared session runtime, only registers tools after successful stdio connect/list, and throws an explicit OpenClaw error when a session is absent instead of falling through to the SDK's null-transport "Not connected" path. The shipped v2026.4.23 release contains that implementation.

## Evidence

- **Main-session Pi runs use the shared session MCP runtime:** The embedded runner's main attempt path calls `getOrCreateSessionMcpRuntime(...)` and then `materializeBundleMcpToolsForRun(...)`, so main sessions use the same session-managed MCP runtime rather than a separate stale catalog path.
