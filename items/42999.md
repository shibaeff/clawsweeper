## Summary

Keep open. Current `main` still implements fixed context-window warning/block thresholds (`16_000` / `32_000`) in `src/agents/context-window-guard.ts`, and the active PI runner passes those constants through directly. The new tests on `main` also assert those exact fixed values, so the requested dynamic percentage-based budgeting is not implemented.
