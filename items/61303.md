## Summary

Current main already implements the webchat tool-result delivery path: the browser client advertises `tool-events`, the gateway broadcasts tool events to registered webchat websocket recipients even with verbose off, and the webchat UI reloads persisted history once on final so exec/cron tool output is retained. Dedicated regression tests cover both the live-event and final-reload behavior, and the same `app-gateway` logic is present in the latest release snapshot.

## Evidence
