## Summary

Current `main` still does not include the answering provider/model in the per-response usage footer, so this PR is not implemented on `main` and should remain open. The feature request still matches current docs and tests, but the branch is stale and the prior review correctly flagged model-label normalization edge cases that would need to be handled if it is revived.
