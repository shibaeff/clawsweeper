## Summary

Keep open. Current main preserves session transcripts and has adjacent recovery features, but it does not implement the issue's requested timeout-triggered temporary memory staging/recovery flow. The existing memory flush is tied to pre-compaction, the bundled session-to-memory hook only runs on `/new` or `/reset`, and transcript indexing into `memory_search` is still opt-in/experimental.
