## Summary

Current main clearly fixes the `openclaw gateway`/`openclaw gateway run` routing part of the report, but the LaunchAgent plist still does not set `StandardInPath`, and I do not have high-confidence evidence that the full self-update crash-loop report is obsolete. Conservative cleanup says keep it open.

## Evidence

- **Gateway root command now runs the foreground gateway path:** `registerGatewayCli()` wraps the top-level `gateway` command with `addGatewayRunCommand(...)`, and `addGatewayRunCommand()` attaches the startup action directly. That means `openclaw gateway` and `openclaw gateway run` both route into the same run handler on current main.
  - file: [src/cli/gateway-cli/register.ts:392](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cli/gateway-cli/register.ts#L392)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Docs match the alias behavior:** The Gateway CLI docs explicitly document `openclaw gateway` as the run command and `openclaw gateway run` as the foreground alias, which aligns with the current command registration.
  - file: [docs/cli/gateway.md:22](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/cli/gateway.md#L22)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Current production startup paths do not show stdin reads in gateway/daemon code:** A targeted search over non-test CLI/gateway/daemon sources found stdin consumers only in update/prompt/native-hook/exec-approvals code, not in `src/cli/gateway-cli/**`, `src/daemon/**`, or gateway startup code.
  - command: `rg -n "process\.stdin|readline|createInterface|for await \(const chunk of process\.stdin\)" src/cli src/index.ts src/daemon src/gateway --glob '!**/*.test.ts'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **LaunchAgent plist still omits StandardInPath:** `buildLaunchAgentPlist()` writes `ProgramArguments`, optional `WorkingDirectory`, `StandardOutPath`, and `StandardErrorPath`, but no `StandardInPath`. That specific requested change is not implemented on current main.
  - file: [src/daemon/launchd-plist.ts:89](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/daemon/launchd-plist.ts#L89)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Self-update / launchd recovery paths have since been hardened:** Current main includes a detached macOS restart helper that falls back from `kickstart` to `bootstrap`, plus launch-agent re-bootstrap recovery when an installed plist exists but the agent is not loaded.
  - file: [src/cli/update-cli/restart-helper.ts:98](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cli/update-cli/restart-helper.ts#L98)
