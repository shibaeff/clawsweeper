## Summary

Current `main` does not show a real fix for this Windows FTS5 report. OpenClaw still supports Node `>=22.14.0` in code, still creates the builtin memory FTS index directly with `node:sqlite` `fts5`, and only degrades status to `fts unavailable` if that table creation fails. The builtin-memory docs still describe keyword/hybrid search as built in with no Windows caveat. That is not strong evidence for `implemented_on_main` or `cannot_reproduce`, so this should stay open.
