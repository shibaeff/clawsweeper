## Summary

Current `main` now loads managed inbound screenshot paths into native vision image payloads, and adds regression coverage for that exact workspace-only inbound-media case. The bug still appears unreleased, so this is implemented on `main` but not yet tied to a shipped tag.

## Evidence

- **Current main loads prompt image refs through `loadWebMedia` with explicit local-root validation instead of the old workspace-only path rejection.:** `loadImageFromRef` now calls `loadWebMedia(targetPath, { maxBytes, localRoots: [workspaceDir] })` for non-sandbox `workspaceOnly` runs, which lets the shared media loader resolve permitted managed inbound files into actual image bytes.
  - file: [src/agents/pi-embedded-runner/run/images.ts:386](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/agents/pi-embedded-runner/run/images.ts#L386)

  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Shared local-media policy explicitly allows managed inbound media before root checks.:** `assertLocalMediaAllowed` returns early for `resolveInboundMediaReference(mediaPath)`, and `loadWebMedia` applies that guard before local reads. That means files under the managed inbound media store are eligible to be read as media even when they are outside the agent workspace.
