## Summary

Issue #50293 is already implemented on current `main` and in the shipped `v2026.4.22` release: the Control UI now builds selector values from each catalog entry's own `provider` + `id`, and slash-containing model refs are preserved or normalized via the catalog instead of being re-qualified under a stale provider.
