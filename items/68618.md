## Summary

Keep open. Current `main` does not implement Feishu sender alias mapping: the config schema exposes only the boolean `resolveSenderNames`, runtime sender-name resolution is API-only, and the user-facing docs/tests describe only that toggle. The report is recent, coherent, and in scope for the bundled Feishu plugin, so there is not strong evidence for any close reason.

## Evidence

- **Feishu config schema exposes only `resolveSenderNames`:** The shared Feishu config shape includes `resolveSenderNames: z.boolean().optional()` and no alias-map field; the top-level schema defaults that boolean to `true`.
  - file: [extensions/feishu/src/config-schema.ts:184](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/feishu/src/config-schema.ts#L184)
