## Summary

Current main already supports disabling Telegram group mention filtering with explicit `requireMention: false`; this PR changes the documented default for `groupPolicy: "open"`, but the capability requested in #68476 is already shipped.

## Evidence

- **Linked issue asks for a way to disable mention filtering:** Issue #68476 reports 'no way to disable mention filtering in groups' and asks for a parameter to disable mention filtering for trusted groups.
  - command: `curl -L --silent https://api.github.com/repos/openclaw/openclaw/issues/68476`
- **Core resolver already honors explicit requireMention=false:** `resolveChannelGroupRequireMention` returns the configured boolean from group or wildcard config before falling back to the default `true`, so an explicit `requireMention: false` already disables mention gating on current main.
  - file: [src/config/group-policy.ts:376](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/config/group-policy.ts#L376)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Telegram plugin uses the same supported override path:** Telegram first checks topic/group/wildcard `requireMention` overrides and then falls back to `resolveChannelGroupRequireMention`, so Telegram groups already have a shipped config path to disable mention filtering.
  - file: [extensions/telegram/src/group-policy.ts:61](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/telegram/src/group-policy.ts#L61)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)

  - file: [extensions/telegram/src/bot.create-telegram-bot.test.ts:1468](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/telegram/src/bot.create-telegram-bot.test.ts#L1468)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Docs describe open as still mention-gated unless explicitly overridden:** Current docs say `groupPolicy: "open"` bypasses group allowlists while mention-gating still applies, and the Telegram config examples include explicit `requireMention: false` overrides.
  - file: [docs/gateway/config-channels.md:32](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/docs/gateway/config-channels.md#L32)

- **Telegram status checks treat requireMention=false as a supported configuration:** The Telegram status surface warns operators about privacy-mode implications when groups are configured with `requireMention=false`, which is further evidence that disabling mention filtering is already a supported path on main.
