## Summary

Current main already contains the protections this report asks for, and the same implementation is present in the shipped `v2026.4.22` release: duplicate `exec.finished` events are deduped by `sessionKey/runId`, heartbeat preflight classifies exec completions as `exec-event` instead of falling back to generic heartbeat handling, busy session lanes are skipped to avoid re-entry, and `exec-event` turns no longer reset or retarget session state.

## Evidence
