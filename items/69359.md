## Summary

Current OpenClaw main and the latest public release already send WebChat image uploads as base64 attachment payloads and pass them to the model as inline images or local `media://` refs, not CDN URLs.

## Evidence

- **WebChat client sends base64 attachments:** `buildApiAttachments` converts each browser `dataUrl` into `{ type: "image", mimeType, content }` and `requestChatSend` sends those attachments over `chat.send`, so the bundled WebChat path is not forwarding a CDN URL.
