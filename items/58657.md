## Summary

Keep open. Current `main` still forwards full tool outputs into ACP `tool_call_update` events, documents that raw I/O/text content is streamed, and the bundled ACPX wrapper adds no local size/replay guard around the file-backed session store. I did not find strong evidence in the checked-out code or changelog that this failure mode is fixed on `main`.

## Evidence

- **ACP bridge still sends raw tool output verbatim on updates/results:** `handleAgentEvent()` forwards `partialResult` and `data.result` directly as `rawOutput`, and also derives `content` from the same values for `tool_call_update`, with no byte cap or truncation in this path.
