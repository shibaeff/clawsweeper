## Summary

Current OpenClaw no longer forwards browser-local webchat image paths into the agent path-checking flow. The Control UI converts uploads to base64, the gateway persists them under managed inbound media and rewrites transcript media fields to those saved paths, and the 2026.4.23 changelog explicitly notes the shipped WebChat attachment preservation fix for text-only models. The report was against 2026.4.15, which predates those changes.

## Evidence

- **Control UI uploads are serialized as base64 attachments, not filesystem paths:** Webchat file select/drop reads each image with `FileReader.readAsDataURL(...)`, stores `dataUrl`, then `buildApiAttachments()` strips the data URL into `{ type: "image", mimeType, content }` for `chat.send`. That means the browser sends image bytes, not a host-local `C:\...` or `/Users/...` path.
  - file: [ui/src/ui/views/chat.ts:244](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/ui/src/ui/views/chat.ts#L244)
