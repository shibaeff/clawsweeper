## Summary

The reported bug is in the external `@tencent-weixin/openclaw-weixin` plugin, not in the OpenClaw core checkout, and current `main` already exposes the needed plugin hook for this behavior.

## Evidence

- **WeChat integration is explicitly external:** The repo docs say OpenClaw connects to WeChat through Tencent's external `@tencent-weixin/openclaw-weixin` plugin, that direct chats are supported, group chats are not advertised by the plugin metadata, and the WeChat-specific runtime is owned by the external plugin rather than core.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/channels/wechat.md#L10)
  - command: `nl -ba docs/channels/wechat.md | sed -n '1,80p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Core already provides the exact extension point:** `detectTargetKind()` consults `messaging.inferTargetChatType` before falling back to generic heuristics. The reported failure mode is therefore a plugin not implementing the optional hook, not a missing core seam.
  - file: [src/infra/outbound/target-resolver.ts:144](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/infra/outbound/target-resolver.ts#L144)
  - command: `nl -ba src/infra/outbound/target-resolver.ts | sed -n '144,175p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Current tests cover plugin-side chat-type steering:** `target-resolver.test.ts` verifies that when a channel plugin returns `inferTargetChatType: () => "direct"`, resolution uses peer lookup and returns a user target. That matches the one-line plugin fix described in the issue.
  - file: [src/infra/outbound/target-resolver.test.ts:156](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/infra/outbound/target-resolver.test.ts#L156)
  - command: `nl -ba src/infra/outbound/target-resolver.test.ts | sed -n '138,185p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **No Weixin plugin source exists in this checkout:** A repository-wide file search for `weixin|wechat` only found docs and a few generic references; there is no `extensions/.../weixin` or plugin `channel.ts` to patch in `openclaw/openclaw` main.
  - command: `rg --files . | rg 'weixin|wechat'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Project vision keeps plugin ownership outside core:** `VISION.md` says optional capability should usually ship as plugins, plugins should be hosted and maintained in their own repository, and plugin discovery/promotion belongs in ClawHub rather than OpenClaw core.
