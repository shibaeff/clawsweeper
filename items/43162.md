## Summary

Keep open. Current `main` still lacks ACP transcript mirroring for same-surface deliveries, so this is not already implemented, but the PR as described is not ready for conservative cleanup because it appears to choose the transcript store from the ACP runtime agent instead of the session-key owner and would also blanket-catch pre-`runTurn` policy failures as assistant transcript entries.

## Evidence

The success path at lines 430-474 and error path at lines 475-515 only deliver replies/errors; there is no transcript append in current `main`.

## Risks / Open Questions

- Closing this PR would discard a still-real gap on current `main`: same-surface ACP replies are visible to users but are not mirrored into session transcripts by the current direct delivery path.
- Landing the PR as described would likely miss mixed `agentId`/`acpAgentId` bindings and could silently no-op transcript writes in those setups.
- Landing the PR without tightening the error-path conditions would risk polluting transcripts with gateway policy/stale-session rejection text labeled as assistant output.
