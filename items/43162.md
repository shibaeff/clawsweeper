## Summary

Keep open. Current `main` still does not mirror ACP dispatch output into session transcripts, so the reported gap looks unresolved. The proposed PR also has unresolved correctness risks raised in review and supported by current code: it can select the wrong session store for configured bindings that use `acpAgentId`, and naive mirroring from accumulated ACP text would not be aligned with confirmed user-visible delivery.

## Evidence

- **Current main still lacks ACP transcript mirroring in the ACP dispatch path:** On current `main`, `tryDispatchAcpReply` runs `acpManager.runTurn()`, flushes/projects output, and delivers success/error replies, but there is no `appendAssistantMessageToSessionTranscript()` call anywhere in this flow. The ACP-specific bug described by the PR is therefore not implemented on `main`.

## Risks / Open Questions

- Closing this PR would leave a plausible still-unfixed ACP transcript bug on current `main`.
- Landing the PR as written could mirror into the wrong agent's session store for configured bindings that use `acpAgentId`.
- Mirroring accumulated ACP text before confirmed delivery can make `sessions_history` diverge from what the user actually saw.
