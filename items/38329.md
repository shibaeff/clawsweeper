## Summary

Keep open. Current `main` has newer stream-resolution work, but it does not clearly implement this PR’s generic custom `openai-completions` auth fix. The only explicit API-key injection on `main` is in the `providerStreamFn` branch; the default `openai-completions` transport still reads `options.apiKey`/env and does not consult `authStorage`, so there is not strong evidence the reported bug is fully fixed on `main`.
