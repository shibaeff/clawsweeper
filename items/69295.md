## Summary

Keep open. Current `main` still returns only aggregate `doctor.memory.status` fields and does not expose per-agent `{dirty, files, chunks}` counts, while the memory CLI still has per-agent status output. The issue is recent and already has a linked open PR, so there is no conservative auto-close basis.

## Evidence
