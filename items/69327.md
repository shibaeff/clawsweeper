## Summary

Keep open. Current `main` has explicit workspace inheritance and sandbox recreate/config-hash logic that likely addresses the stale workspace/bootstrap half of the report, but I did not find equally strong evidence that spawned subagents now reliably receive the expected `sandbox.docker.env` state. Given the issue is only 5 days old and the remaining env-propagation question is still plausible, this is not strong enough for a conservative auto-close.
