## Summary

Current `main` does not implement a configurable dreaming diary language. The dreaming narrative system prompt is hardcoded in English, the `memory-core` manifest exposes no `dreaming.language` key, and the public docs only describe `enabled`/`frequency` (with examples using `timezone`). This issue is recent, specific, corroborated by another user, and remains actionable rather than stale or already fixed.
