## Summary

Current `main` already implements the runtime-snapshot sync needed for auth-profile rotation, so this PR is redundant. `ensureAuthProfileStoreWithoutExternalProfiles` still serves runtime snapshots first, `updateAuthProfileStoreWithLock` already funnels writes through `saveAuthProfileStore`, and `saveAuthProfileStore` now refreshes any existing runtime snapshot before later reads. The round-robin logic in `orderProfilesByMode` still sorts by `usageStats.lastUsed`, so the stale-snapshot failure described in the PR is already addressed on `main` and in release `v2026.4.22`.
