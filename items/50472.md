## Summary

Keep open. Current main already adds the requested observability pieces for uncaught exceptions/rejections and persisted stability bundles with memory samples, but I did not find strong evidence that the underlying Windows gateway self-termination is fixed. This looks like better diagnostics landed, not a proven resolution of the reported silent-exit behavior.

## Evidence

- **CLI now logs uncaught exceptions before exit:** The main CLI entry installs an `uncaughtException` handler that logs the formatted error, runs fatal hooks, restores terminal state, and exits non-zero.
  - file: [src/cli/run-main.ts:226](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/cli/run-main.ts#L226)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **CLI now logs and classifies unhandled rejections:** Unhandled rejections are globally trapped; fatal/config errors are logged and exited, while known transient network/SQLite cases are logged and suppressed.
  - file: [src/infra/unhandled-rejections.ts:340](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/infra/unhandled-rejections.ts#L340)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Gateway diagnostics heartbeat installs fatal bundle capture:** Gateway startup enables the diagnostics heartbeat, and the heartbeat installs the fatal stability hook so fatal paths can persist a stability bundle.
  - file: [src/gateway/server.impl.ts:308](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/server.impl.ts#L308)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Failure bundles and memory samples are implemented:** Failure-bundle writing is implemented in the diagnostics bundle module, and memory samples come from `process.memoryUsage()` in the diagnostics memory sampler.
  - file: [src/logging/diagnostic-stability-bundle.ts:621](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/logging/diagnostic-stability-bundle.ts#L621)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Docs describe fatal-exit bundle persistence:** The health docs now state that fatal Gateway exits, shutdown timeouts, and restart startup failures persist a stability bundle under `~/.openclaw/logs/stability/`.
  - file: [docs/gateway/health.md:29](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/docs/gateway/health.md#L29)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **No generic process-exit logging/watchdog was found in the gateway path:** A focused search found direct `process.exit(...)` calls and only a CLI `process.once("exit")` hook for debug-proxy finalization; I did not find a generic process-exit interceptor/watchdog that would prove silent exits are fully explained now.
  - command: `rg -n "process\.on\(\"exit\"|process\.once\(\"exit\"|beforeExit|process\.exit\(" src/cli src/gateway src/infra src/index.ts src/runtime.ts`
