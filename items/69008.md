## Summary

Keep open: current `main` still makes outbound agent delivery opt-in via explicit `deliver: true`, with no Telegram group/topic session-key exception. The latest shipped release still shows the same behavior, so this is not implemented on `main` and the report remains actionable.

## Evidence

- **Current gateway handler still requires explicit deliver:** `wantsDelivery` is derived from `request.deliver === true`, and final outbound delivery is computed from the same explicit flag (`deliver = request.deliver === true && resolvedChannel !== INTERNAL_MESSAGE_CHANNEL`). There is no Telegram group/session-key override in this path.
