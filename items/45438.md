## Summary

Keep open. Current `main` still contains the exact `structuredClone` session/auth-store clone paths called out in the report, and the latest shipped release `v2026.4.22` still has them as well. I found mitigation work for oversized session stores, but not a landed fix replacing these clone paths or otherwise resolving this issue family.
