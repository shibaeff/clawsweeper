## Summary

The requested per-run Telegram forum-topic behavior is a Telegram-specific orchestration workflow, not a missing core primitive on current `main`; OpenClaw already exposes Telegram topic creation and thread-routing seams, and the project vision directs optional channel-specific capability to plugins/ClawHub rather than new core cron modes.

## Evidence

- **Vision puts optional/channel-specific capability in plugins:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and plugin promotion belongs in ClawHub rather than core.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/VISION.md#L54)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Plugin docs explicitly support ClawHub delivery plus channel/tools/hooks/CLI seams:** `docs/plugins/building-plugins.md` says plugins should be published to ClawHub or npm and lists channel plugins, agent tools, plugin hooks, and CLI subcommands as supported extension points; it also documents typed `threadId` routing fields for plugin hooks.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/plugins/building-plugins.md#L16)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Telegram already exposes forum-topic creation as a first-class action:** `docs/channels/telegram.md` documents Telegram `createForumTopic` / `topic-create` as available message actions, and the runtime implementation exports `createForumTopicTelegram(...)` in the bundled Telegram plugin.
  - file: [docs/channels/telegram.md:458](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/channels/telegram.md#L458)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **CLI thread creation already remaps to Telegram topic creation:** `src/cli/program/message/register.thread.ts` routes `message thread create` through channel-specific action remapping, and `extensions/telegram/src/channel-actions.ts` remaps `thread-create` to Telegram `topic-create`.
  - file: [src/cli/program/message/register.thread.ts:7](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/program/message/register.thread.ts#L7)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Core cron delivery is generic/static, not a dynamic topic-lifecycle feature:** `src/cron/types.ts` defines cron delivery as generic `channel` / `to` / optional `threadId`, while cron docs document Telegram forum delivery via a static `chatId:topic:topicId` target. That supports fixed topic routing, but not a Telegram-specific per-run topic creation mode in core.
  - file: [src/cron/types.ts:24](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cron/types.ts#L24)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
