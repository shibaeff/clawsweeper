## Summary

Keep open. Current `main` does not contain this PR's catalog-enrichment behavior: `loadModelCatalog()` only returns Pi registry entries plus provider-plugin supplements, `modelSupportsVision()` still depends on `entry.input`, and current tests still assert that configured providers are not added unless a plugin supplies them.

## Evidence

- **`loadModelCatalog()` does not read configured provider models:** The current implementation builds the catalog from Pi-discovered registry entries, then appends only `augmentModelCatalogWithProviderPlugins()` results. There is no code here that reads `config.models.providers.*.models` or enriches existing entries with configured `input` fields.
  - file: [src/agents/model-catalog.ts:113](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/model-catalog.ts#L113)
  - command: `nl -ba src/agents/model-catalog.ts | sed -n '80,210p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Vision support still depends on catalog `input`:** `modelSupportsVision()` returns true only when the catalog entry itself includes `image` in `input`, so missing `input` metadata in the loaded catalog still disables vision support.
  - file: [src/agents/model-catalog.ts:227](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/model-catalog.ts#L227)
  - command: `nl -ba src/agents/model-catalog.ts | sed -n '224,250p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Current tests still encode the pre-PR behavior:** `model-catalog.test.ts` still expects that no unrelated configured provider model is added when provider plugins return nothing. That is the opposite of this PR's intended change to merge configured provider models beyond plugin-owned supplements.
  - file: [src/agents/model-catalog.test.ts:369](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/model-catalog.test.ts#L369)

- **Configured provider-model parsing exists elsewhere, not in `loadModelCatalog()`:** `buildConfiguredModelCatalog()` does parse `cfg.models.providers.*.models`, but it lives in model-selection code and is separate from `loadModelCatalog()`. That means current main has not implemented this PR's proposal to preserve configured image capabilities in the shared catalog loader.
  - file: [src/agents/model-selection-shared.ts:633](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/model-selection-shared.ts#L633)
