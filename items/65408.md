## Summary

Keep open. Current `main` still explicitly supports the reported shared-token to persisted device-token retry path on `AUTH_TOKEN_MISMATCH`, still persists Control UI device auth in browser storage, shows the login gate whenever `connected` is false before async bootstrap/connect completes, and I did not find a first-class Control UI auth logout flow in the app sources.

## Evidence
