## Summary

Keep open. Current `main` still forwards several gateway-facing `--timeout` values with raw `Number(...)` coercion instead of rejecting invalid input at the CLI boundary, so the PR addresses a live bug rather than an already-landed fix. The underlying paths have shifted slightly since the PR was opened (`gateway-rpc` now delegates to `gateway-rpc.runtime.ts`), so the branch likely needs a refresh, but the behavior the PR targets is still present on `main`.

## Evidence
