## Summary

Current `main` already supports configurable thread-binding idle timeouts through config, including global defaults and channel overrides, and the same implementation is present in release `v2026.4.22`.

## Close Comment

Closing from a Codex review because the requested configurability is already implemented on `main` and is present in release `v2026.4.22`. OpenClaw accepts `session.threadBindings.idleHours` in config (`src/config/zod-schema.session.ts:65`), uses that value when resolving thread-binding idle timeouts at runtime (`src/channels/thread-bindings-policy.ts:181`), and documents both the global setting and channel overrides such as `channels.discord.threadBindings.idleHours` (`src/config/schema.help.ts:1354`, `docs/gateway/configuration.md:263`, `src/config/bundled-channel-config-metadata.generated.ts:3128`). The docs also note that `0` disables idle auto-unfocus. If you are still seeing a hard 24-hour timeout on a current build, please open a new bug with the exact config block, channel/plugin, and observed behavior.
