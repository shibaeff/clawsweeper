## Summary

Keep open. Current `main` does not implement this hardening in `src/channels/logging.ts`, so there is nothing to close as already shipped. The PR also appears to need rework before merge because OpenClaw already has a shared `sanitizeForLog()` utility and the proposed new string-only helper would risk a runtime regression for JS/native plugin callers on this public plugin-SDK surface.
