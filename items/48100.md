## Summary

Current `main` does not implement an `after_agent_complete` hook or any equivalent post-generation/pre-delivery plugin seam, and the PR discussion raises real design questions around streaming and transcript state that make a conservative close inappropriate.

## Evidence

- **Hook registry on main omits the requested lifecycle hook:** `PluginHookName` and `PLUGIN_HOOK_NAMES` include `before_agent_reply` and `agent_end`, but there is no `after_agent_complete` entry on current `main`. This means the requested hook is not already implemented.

There is no documented hook that runs after generation but before delivery.

## Risks / Open Questions

- Closing this PR as implemented would be incorrect: the requested hook does not exist on current `main`, and existing hooks only cover before-run interception or post-run observation.
- Closing it as ClawHub/plugin-scope would conflict with `VISION.md`, which explicitly invites plugin-API extensions in core when a capability cannot yet be built as a plugin.
- The PR discussion identified unresolved behavior around streaming and transcript persistence; those concerns argue for maintainer review of the design, not for conservative cleanup closure.
