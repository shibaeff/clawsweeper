## Summary

Current `main` does not implement this PR's followup dedupe fix. The followup path still unconditionally removes text/media duplicates before checking whether the messaging-tool send targeted the origin conversation, so the reported cross-target suppression remains plausible.

## Evidence

- **Followup path still uses shared delivery helper:** `createFollowupRunner` passes `messagingToolSentTexts`, `messagingToolSentMediaUrls`, and `messagingToolSentTargets` into `resolveFollowupDeliveryPayloads`, so the bug now lives in that helper rather than the older inline code.
  - file: [src/auto-reply/reply/followup-runner.ts:358](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/auto-reply/reply/followup-runner.ts#L358)
