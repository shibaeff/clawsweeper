## Summary

Current `main` contains an explicit Gateway startup fix for Tailscale Serve/Funnel in containers and now treats Serve as external TLS termination with an internal plain-HTTP loopback gateway. The startup path no longer requires `gateway.tls.enabled=true` for `gateway.tailscale.mode="serve"`, and the Control UI now derives `wss://` from the HTTPS page origin instead of the internal gateway TLS setting.
