## Summary

Current `main` already fixes the webhook-route 404 caused by plugin-registry churn. The gateway now pins the startup HTTP-route registry and resolves plugin HTTP auth/dispatch against that live route registry, with regression tests covering registry swaps and route registration after the active registry changes.
