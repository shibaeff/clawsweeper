## Summary

Keep open. Current `main` still forwards unclamped `reserveTokens` into compaction summarization, the pinned upstream `@mariozechner/pi-coding-agent@0.70.2` still derives `maxTokens` as `Math.floor(0.8 * reserveTokens)`, and the latest shipped release still lacks this clamp. The PR discussion also shows the earlier test-gap feedback was addressed with focused regression tests on the branch, so this remains useful unmerged work rather than a stale duplicate.
