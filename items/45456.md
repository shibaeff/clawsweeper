## Summary

Keep open. Current `main` has session/provider usage reporting and some historical usage aggregation by channel, but it does not implement configurable per-channel or per-chat token budgets, enforcement in the inbound message pipeline, or `/status`/`session_status` reporting that matches the request.

## Evidence

- **Status surfaces are session/provider scoped:** `buildStatusText` only loads provider usage windows for the current provider and formats a single `📊 Usage:` line; it does not read channel/chat budget policy or report per-channel limits.
  - file: [src/status/status-text.ts:185](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/status/status-text.ts#L185)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Docs describe `/status` and `session_status` as session token/cost views, not channel budgets:** The token-use and usage-tracking docs say `/status`, `session_status`, and `openclaw status --usage` show session tokens and provider quota windows. They do not document per-channel or per-chat budget reporting.
  - file: [docs/reference/token-use.md:62](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/reference/token-use.md#L62)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Config schema lacks requested budget fields:** `SessionConfig` has `sendPolicy` allow/deny matching by `channel` and `chatType`, but no token-budget fields. The only top-level `rateLimit` schema in core config here is `gateway.auth.rateLimit`, which throttles failed auth attempts, not model-token spend.
  - file: [src/config/types.base.ts:103](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/config/types.base.ts#L103)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Historical usage aggregation is adjacent but not enforcement:** Gateway usage code records `channel` and `chatType` on session usage entries and aggregates totals by channel from stored session/transcript usage, but this is post-hoc reporting only. There is no budget check or skip/queue enforcement path here.
  - file: [src/gateway/server-methods/usage.ts:659](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/gateway/server-methods/usage.ts#L659)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)

## Risks / Open Questions

- There is adjacent usage-reporting infrastructure, especially `gateway` usage aggregation by channel, so closing as `implemented_on_main` would overstate what shipped.
- I could not inspect related GitHub PRs/issues via `gh` because the environment lacks `GH_TOKEN`, but the repository code/docs on `main` are sufficient to show the requested feature is not implemented.
