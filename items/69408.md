## Summary

Current `main` does not implement the requested downgrade. Warning-path loop detections still log at `warn` unconditionally, and the hook/runtime types do not carry cron-session metadata such as `sessionTarget`, session lifetime, or cron interval that would allow the proposed isolated-session special case. Docs also describe isolated cron runs as fresh per-run sessions, which matches the issue’s premise rather than disproving it.
