## Summary

Keep open. Current `main` already requests Markdown by default in `web_fetch`, but it still does not implement the proposed `tools.web.fetch.defaultHeaders` config surface in the runtime type/schema or generated config metadata, so this PR is neither implemented on `main` nor strong enough to close as obsolete.

## Evidence

- **Current web_fetch already asks for Markdown:** `runWebFetch` sends `Accept: "text/markdown, text/html;q=0.9, */*;q=0.1"` today, so the PR's Cloudflare-Markdown motivation is partly already covered on `main`.

## Risks / Open Questions

- If maintainers revive this PR, they still need full config-contract coverage, not just runtime wiring: runtime schema, generated schema/help, and any doc/UI surfaces must all agree.
- The current built-in `Accept` header weakens the original motivation, but it does not replace the broader feature request for configurable default fetch headers.
