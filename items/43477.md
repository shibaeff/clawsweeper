## Summary

Keep open. Current `main` does not implement `tools.web.fetch.defaultHeaders`, and the PR is not safely closeable because the proposed change remains unmerged and still has the schema/cache/header correctness concerns already raised in review.

## Evidence

- **Current main already covers the PR's stated Cloudflare Markdown motivation:** `web_fetch` already sends `Accept: text/markdown, text/html;q=0.9, */*;q=0.1`, and there is a dedicated test asserting that behavior. That means the narrow motivation in the PR body is partly obsolete, but it does not prove the broader `defaultHeaders` feature is implemented.
  - file: [src/agents/tools/web-fetch.ts:424](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/agents/tools/web-fetch.ts#L424)
  - command: `nl -ba src/agents/tools/web-fetch.ts | sed -n '388,430p' && nl -ba src/agents/tools/web-fetch.cf-markdown.test.ts | sed -n '51,62p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Current main still has no `defaultHeaders` config entry for web_fetch:** The strict runtime schema for `tools.web.fetch` lists `enabled`, `provider`, `maxChars`, `maxCharsCap`, `maxResponseBytes`, `timeoutSeconds`, `cacheTtlMinutes`, `maxRedirects`, `userAgent`, `readability`, `ssrfPolicy`, and `firecrawl`, but no `defaultHeaders`. The public type/help/label surfaces also omit it.
  - file: [src/config/zod-schema.agent-runtime.ts:339](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/config/zod-schema.agent-runtime.ts#L339)

  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Current cache key would be wrong for header-varying requests:** `runWebFetch` caches by URL, extract mode, max chars, and the RFC2544 policy flag only. If request headers become configurable, response-varying headers would need to participate in the cache key; otherwise different header sets can alias to the same cached payload.
  - file: [src/agents/tools/web-fetch.ts:392](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/agents/tools/web-fetch.ts#L392)
