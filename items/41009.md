## Summary

Keep open. Current `main` still enforces loopback-only Tailscale Serve/Funnel binds, but the runtime legacy migrations on `main` only normalize host aliases and seed `gateway.controlUi.allowedOrigins`; they do not rewrite legacy explicit `gateway.bind="lan"|"tailnet"|"custom"` Serve/Funnel configs to loopback before startup validation.

## Evidence

- **Serve/Funnel validation still hard-fails non-loopback binds:** `validateGatewayTailscaleBind` returns an error unless `gateway.bind` is `loopback` or `custom` with `127.0.0.1`, so legacy `lan`, `tailnet`, or non-loopback `custom` configs still validate as invalid on current `main`.
  - file: [src/config/validation.ts:588](https://github.com/openclaw/openclaw/blob/33d5ebbff7bceb782b314de14ebd9a213ba31869/src/config/validation.ts#L588)
