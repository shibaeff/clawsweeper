## Summary

Current `main` already contains a targeted fix for the oversized context warning in chat: the notice now uses `totalTokens` instead of historic `inputTokens`, explicitly constrains the warning SVG to 16x16, and has a regression test covering the large-`inputTokens`/small-`totalTokens` case described in #45442. The same implementation is present in release `v2026.4.22`.

## Evidence

- **Current context notice uses current context usage, not accumulated input history:** `getContextNoticeViewModel` reads `session.totalTokens` and `session.contextTokens`, and returns `null` when only stale/invalid totals exist. This prevents a large historic `inputTokens` count from inflating the warning state.
  - file: [ui/src/ui/chat/context-notice.ts:56](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/chat/context-notice.ts#L56)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Current warning icon is explicitly size-constrained:** The warning SVG is rendered with `width="16" height="16"`, and the chat CSS also fixes `.context-notice__icon` to `16px` by `16px`, preventing the giant triangle/viewport overflow behavior from the report screenshot.
  - file: [ui/src/ui/chat/context-notice.ts:101](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/chat/context-notice.ts#L101)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Chat view still renders the notice in the normal thread flow:** The chat surface renders the context notice inline above the composer via `renderContextNotice(...)`, so the fix applies to the `main:main` session chat UI the issue reports.
  - file: [ui/src/ui/views/chat.ts:1192](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/views/chat.ts#L1192)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)

## Close Comment

Closing from a Codex review because this looks implemented on `main` and already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The current Control UI notice uses `totalTokens` instead of historic `inputTokens` in `ui/src/ui/chat/context-notice.ts`, and both the SVG markup there and the CSS in `ui/src/styles/chat/layout.css` explicitly constrain the warning icon to 16x16. There is also a regression test in `ui/src/ui/chat/context-notice.test.ts` that covers the reported shape directly: a session with very large `inputTokens` but only `46_000` current `totalTokens` must not render the warning, and the icon size stays fixed. That matches the oversized warning-triangle bug described here, so there is not an open maintainer action left on current main.
