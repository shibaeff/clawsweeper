## Summary

Current `main` does not implement `/think <level> <message>` as a one-shot override. The core directive parser strips `/think <level>` from mixed messages, but `resolveReplyDirectives()` clears the parsed think directive when any non-directive body remains, so the per-message level is lost. Control-command detection still treats `/think ...` as a command with args, and the Control UI slash executor persists `/think` via `sessions.patch` instead of sending a one-shot message.

## Evidence

- **Mixed-message directives drop the parsed think level:** `resolveReplyDirectives()` reparses messages with remaining body text and replaces parsed directives with `clearInlineDirectives(...)` when `directiveOnlyCheck.cleaned` is still non-empty. For `/think high Write a poem`, that clears `hasThinkDirective` and `thinkLevel`, so the one-shot override is not preserved.
