## Summary

Current `main` already fixes the WebChat/Control UI image-loss path: the UI sends attachments via `chat.send`, the gateway now offloads/persists images for text-only models instead of dropping them, and the reply pipeline treats WebChat image-only turns as media-bearing. The latest tagged release still had the old drop-on-text-only behavior, so this appears fixed on `main` but not yet shipped in a release.
