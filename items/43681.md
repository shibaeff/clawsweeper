## Summary

Keep open. Current main already supports rotating among multiple same-provider auth profiles once they exist, but the OpenAI Codex OAuth login flow still derives the saved profile id from the email when one is present and offers no login-time alias/profile-id flag. That means the reported same-email multi-team case is not clearly implemented on main and would still collapse to one `openai-codex:<email>` profile unless the token lacks email or the user edits profile ids some other way.

## Risks / Open Questions

- If the relevant OpenAI OAuth tokens omit email in the multi-team case, the newer stable-subject fallback could separate some accounts automatically; current code and tests still show email wins whenever present.
- The request mixes two behaviors: distinct profile creation and automatic 429 failover. Rotation exists, but the login/profile-separation part is not strongly evidenced as fixed.
