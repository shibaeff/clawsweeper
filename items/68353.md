## Summary

Keep open. Current `main` does not implement first-class per-message author attribution on `chat.send`: the wire schema has no `author` field, the gateway handler derives sender identity from the connected client (`client.id` / `client.displayName`) rather than a per-message payload, and chat history normalization only surfaces `senderLabel`. This is a coherent core gateway/protocol request, not a ClawHub/plugin-scope item.

## Evidence
