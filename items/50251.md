## Summary

Current main already contains the runtime-store and Windows loader changes that address the reported Feishu initialization failure, and the shipped v2026.4.22 release contains the same implementation.

## Evidence

- **Feishu now uses plugin-scoped runtime store:** The Feishu runtime setter/getter is created through `createPluginRuntimeStore({ pluginId: "feishu", errorMessage: "Feishu runtime not initialized" })`, which routes Feishu through the shared plugin-id store instead of a per-module closure.
  - file: [extensions/feishu/src/runtime.ts:4](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/feishu/src/runtime.ts#L4)
