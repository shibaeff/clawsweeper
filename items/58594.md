## Summary

Current `main` already removes loader-owned internal hook registrations before each reload and includes a regression test proving the same hook does not fire multiple times after repeated reloads.

## Evidence

- **Reload path unregisters prior loader-owned hooks:** `resetLoadedInternalHooks()` drains the tracked registrations and calls `unregisterInternalHook(...)`; `loadInternalHooks()` invokes that reset before doing any new registration.
  - file: [src/hooks/loader.ts:51](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/hooks/loader.ts#L51)

## Risks / Open Questions

- `registerInternalHook()` itself is still append-only, so manual repeated registration of the same handler could still duplicate calls; the implemented fix is specifically for the loader reload path described in this issue.
