## Summary

Current main and the latest release already catch bundled `loadChannelPlugin()` failures inside `getBundledChannelPluginForRoot()` and return `undefined`, so the reported config-read crash does not match shipped behavior; the PR’s added regression test also does not exercise the claimed path because its fixture is not discoverable as a bundled plugin.

## Evidence

- **Current main already catches this loader failure:** `getBundledChannelPluginForRoot()` wraps `entry.loadChannelPlugin()`, the subsequent `plugin.id` access, and normalization in a `try/catch`; on any thrown error it logs a warning, stores `null` in the cache, and returns `undefined` instead of letting the exception escape.
  - file: [src/channels/plugins/bundled.ts:546](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/channels/plugins/bundled.ts#L546)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Shipped release has the same behavior:** Release `v2026.4.23` contains the same `try/catch` + negative-cache path in `getBundledChannelPluginForRoot()`, so the linked issue’s `Failed to read config ... TypeError` stack does not match the current shipped implementation of this path.
  - file: [src/channels/plugins/bundled.ts:481](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/channels/plugins/bundled.ts#L481)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/channels/plugins/bundled.ts | nl -ba | sed -n '465,500p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Bundled channel discovery is manifest-driven:** Bundled channel lookup first resolves manifest-backed metadata, and the metadata scanner skips directories without a loadable plugin manifest or `openclaw.extensions` entry. A bare `index.js` fixture is therefore not enough to make a bundled channel discoverable.
  - file: [src/plugins/bundled-plugin-metadata.ts:107](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/plugins/bundled-plugin-metadata.ts#L107)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Existing review already identified the regression-test gap:** The PR review discussion correctly notes that the new test only writes `index.js` and mocks `listChannelCatalogEntries`, but `getBundledChannelPluginForRoot()` resolves candidates through bundled metadata, not the catalog registry, so the test returns `undefined` before `loadChannelPlugin()` is invoked.
  - command: `curl -L -s https://api.github.com/repos/openclaw/openclaw/pulls/comments`
