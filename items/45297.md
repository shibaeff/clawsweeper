## Summary

Keep open. Current `main` only supports Signal group mention gating by rewriting Signal mention placeholders into `@uuid`/`@phone` text and then running regex-based `mentionPatterns`. It does not resolve explicit Signal `dataMessage.mentions` against the bot account UUID the way the issue requests, and the docs still state that Signal lacks native mention support.

## Evidence

- **Signal group gating still uses regex matches, not explicit bot-UUID mention detection:** The inbound Signal handler computes `wasMentioned` via `matchesMentionPatterns(messageText, mentionRegexes)` and passes `hasAnyMention: false` / `implicitMentionKinds: []` into `resolveInboundMentionDecision`. There is no direct comparison between `dataMessage.mentions[*].uuid` and the bot account UUID.
  - file: [extensions/signal/src/monitor/event-handler.ts:665](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/signal/src/monitor/event-handler.ts#L665)

  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Signal mention metadata is only rewritten into synthetic text tokens:** `renderSignalMentions` replaces object-replacement placeholders with `@${uuid}` or `@${number}`. That is text normalization for downstream regex matching, not native mention resolution against the current bot identity.
  - file: [extensions/signal/src/monitor/mentions.ts:28](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/signal/src/monitor/mentions.ts#L28)

That does not line up with Signal's rewritten `@uuid` / `@phone` tokens, so native Signal mentions are not treated like channels with first-class explicit-mention support.

## Risks / Open Questions

- Signal mention handling is partially improved already, so the remaining work is narrower than the issue body suggests: explicit native-mention resolution against the bot identity, plus docs/tests alignment.
- A user can partially work around this today by configuring `mentionPatterns` to match rewritten `@uuid` or `@phone` tokens, but that is not the same behavior as native mention gating on other channels.
