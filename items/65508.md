## Summary

Current `main` does not implement this PR’s core fix on the outbound `sanitizeUserFacingText` path. The shared tool-XML stripper exists, but it is only wired into `sanitizeAssistantVisibleText`; user-facing reply normalization still calls `sanitizeUserFacingText`, which only strips `<final>` and internal runtime context. The PR discussion also shows substantial reviewed work and green CI on the PR head, so this is not a safe conservative close.
