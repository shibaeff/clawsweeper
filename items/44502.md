## Summary

Current `main` does not show the reported Discord hardening. The preflight path still detects mentions from the full `baseText`, has no logic for "other configured Discord agent" targeting, and only drops unrelated user/role mentions behind `ignoreOtherMentions`. The issue remains plausible and is not strongly closable.

## Evidence

- **Mention detection still scans full message text:** `mentionText` is the full `baseText`, and `resolveDiscordMentionState()` passes that into `matchesMentionWithExplicit()`, which tests the whole normalized text against mention regexes. There is no attention-window / first-paragraph truncation in current main.

- **Explicit mention matcher has no quote-window scoping:** `matchesMentionWithExplicit()` normalizes the supplied text and runs regexes over the entire string; if explicit mentions are available it still falls back to whole-text regex matching. That matches the report's concern about later quoted/copied mentions still satisfying gating.
