## Summary

Current `main` still does not implement the Markdown-image exclusion. The active TUI submit path, bash alias routing, and ACP command-bypass logic all still treat any non-lone leading `!` as a bash command candidate, so `![image](url)` would still be misrouted. This PR should stay open; it addresses a real gap, although it likely needs a rebase onto the current file layout.
