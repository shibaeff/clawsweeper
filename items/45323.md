## Summary

Current main does not implement Slack-style `@agent` mention autocomplete in the Control UI chat composer. The composer is still a plain textarea with slash-command autocomplete only; the gateway exposes `agents.list` but no `agents.directory`; and current Control UI docs do not describe mention chips or mention rendering. This is a live feature request, not an implemented or obsolete item.

## Evidence

- **Composer is still a plain textarea:** `renderChat` binds the draft directly to a `<textarea>` with ordinary key/input/paste handlers. There is no rich-text or `contenteditable` composer path that could host Slack-style mention chips.
  - file: [ui/src/ui/views/chat.ts:1230](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/views/chat.ts#L1230)

- **Autocomplete logic is slash-command only:** The only composer autocomplete UI is `slash-menu`, and keyboard handling covers slash-command navigation and selection. No `@mention` trigger, agent picker, or mention insertion flow exists here.
  - file: [ui/src/ui/views/chat.ts:727](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/views/chat.ts#L727)
