## Summary

Keep open. Current `main` has partial follow-along for ACP background runs, including streamed progress snippets and a best-effort "may be waiting for input" notice, but it still does not implement the requested attach/recover workflow. The ACP bridge docs explicitly mark terminal methods unsupported, ACP sessions are still documented as non-interactive with no TTY, and the `/acp` command surface has no attach/resume/input command for an active interactive process.

## Evidence
