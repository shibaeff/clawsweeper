## Summary

Current `main` still leaves `openclaw channels logs` on the old path-resolution behavior, while the rolling-log fallback now lives in shared logging code used by Gateway and `openclaw logs`. The underlying bug described by PR #43043 still appears real, but the PR is stale against current `main` because that fallback logic has since been centralized elsewhere.

## Evidence
