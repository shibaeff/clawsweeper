## Summary

Current `main` and the shipped `v2026.4.23` UI already solve the narrow-screen navigation-access problem by replacing the old <=1100px horizontal/top navigation pattern with a slide-over drawer and vertical nav. This PR’s mouse drag-scroll implementation is therefore superseded, and the prior review also found an edge-case bug in the proposed drag handler.

## Evidence

- **Responsive drawer shipped on narrow screens:** The <=1100px layout switches the left nav to a fixed slide-over drawer controlled by `shell--nav-drawer-open`; the same code is present in the `v2026.4.23` release, so the narrow-screen nav access change is already shipped.
