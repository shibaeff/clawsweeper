## Summary

Keep open. Current `main` already normalizes Telegram reply payloads through the outbound payload plan, so part of the PR's intent landed elsewhere, but the reply-tag-plus-`MEDIA:` parsing gap described in the PR still exists, and Telegram delivery still does not thread a `currentMessageId` through this path.

## Evidence
