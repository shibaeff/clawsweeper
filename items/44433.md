## Summary

Keep open. Current `main` still builds gateway service PATH from `src/daemon/service-env.ts`, and that code does not add `~/.nix-profile/bin` or parse `NIX_PROFILES` on either macOS or Linux, so the PR's requested behavior is not implemented on `main`.

## Evidence

  - file: [src/commands/daemon-install-helpers.ts:288](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/commands/daemon-install-helpers.ts#L288)
  - command: `nl -ba src/commands/daemon-install-helpers.ts | sed -n '260,340p'`

  - file: [src/daemon/service-env.ts:89](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/service-env.ts#L89)
