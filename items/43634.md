## Summary

Current `main` still shows the export-html regression this PR is trying to fix. The real `template.html` no longer contains the `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` placeholders that `commands-export-session.ts` replaces, and `.gitattributes` does not protect the file. This is not implemented on `main`, so the PR should remain open.

## Risks / Open Questions

- `/export-session` HTML output on current `main` can ship with unreplaced JS placeholder content, breaking the exported page.
- The current test setup masks the regression because it does not exercise the checked-in `src/auto-reply/reply/export-html/template.html` file.
- Without a `.gitattributes` guard or equivalent protection, future formatting/normalization can re-break this template even after a fix lands.
