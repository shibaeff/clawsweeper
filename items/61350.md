## Summary

Keep open. Current `main` still only strips XML-style reasoning tags on both the streaming and final-delivery paths, and there is no code or test coverage for the malformed Gemini `...94>thought CRITICAL INSTRUCTION` variant described in #61350.

## Evidence

- **Streaming filter still only matches literal tag forms:** `THINKING_TAG_SCAN_RE` only recognizes `<think>`, `<thinking>`, `<thought>`, and `<antthinking>` forms with angle brackets. It would not match a `...94>thought ...` preamble.
  - file: [src/agents/pi-embedded-subscribe.ts:38](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/pi-embedded-subscribe.ts#L38)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '34,40p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Shared sanitizer has the same limitation:** The shared reasoning-tag sanitizer only looks for `<...>` / `</...>` tag patterns, so the malformed Gemini variant is not covered there either.
  - file: [src/shared/text/reasoning-tags.ts:5](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/shared/text/reasoning-tags.ts#L5)
