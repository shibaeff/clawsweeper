## Summary

Current main already contains a targeted fix for this regression: config validation no longer treats the inferred default memory slot (`memory-core`) as a required explicit plugin reference, and the changelog says this shipped in 2026.3.22.

## Evidence

- **Release note matches the reported failure:** The 2026.3.22 changelog explicitly says gateway/config validation stopped failing with `plugins.slots.memory: plugin not found: memory-core` when `memory-core` was only inferred.
  - file: [CHANGELOG.md:2433](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/CHANGELOG.md#L2433)
- **Current validator only blocks explicit memory-slot config:** Validation now checks `hasExplicitMemorySlot` before raising `plugins.slots.memory` missing-plugin errors, so the implicit default slot no longer blocks startup or update verification.
  - file: [src/config/validation.ts:1084](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/config/validation.ts#L1084)
- **Regression test covers the exact implicit-slot case:** The test `does not fail validation for the implicit default memory slot when plugins config is explicit` verifies that validation succeeds even when bundled plugins are unavailable and only the inferred default memory slot would have been used.
  - file: [src/config/config.plugin-validation.test.ts:304](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/config/config.plugin-validation.test.ts#L304)
- **Default memory plugin is still `memory-core`:** Current docs still describe `memory-core` as the default active memory plugin, which matches the reported config shape and shows the fix was not a rename/removal of the plugin id.
  - file: [docs/cli/memory.md:13](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/cli/memory.md#L13)
