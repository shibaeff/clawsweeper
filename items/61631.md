## Summary

Current OpenClaw already supports the requested pre-delivery enforcement via plugin hooks: `message_sending` can rewrite or cancel outbound content before channel delivery, and that path is present in the released code. The exact API shipped as a plugin hook rather than `gateway.nodes.denyMessages` or an internal shell hook, but the core capability requested in the issue is implemented.

## Evidence

- **Plugin hook docs define pre-delivery cancellation:** The plugin hook reference says message hooks are for channel-level delivery policy; `message_sending` can rewrite outbound `content` or return `{ cancel: true }`, and `cancel: true` is terminal.

## Risks / Open Questions

- The shipped surface is a plugin hook (`message_sending` / `before_dispatch`), not the exact `gateway.nodes.denyMessages` or shell-command `preDeliveryHook` shape proposed in the issue.
- Public docs for the hook landed on `main` after the current release tag, so some users may only discover the feature through the plugin docs site or code examples.
