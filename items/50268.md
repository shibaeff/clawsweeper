## Summary

Current `main` still appears to overwrite `agents.defaults.model.primary` when `openclaw configure` or `openclaw onboard --auth-choice ...` adds a provider. The preservation logic exists only for `setDefaultModel=false`, but both user-facing flows in this report still pass `setDefaultModel: true`.

## Evidence

- **`configure` still forces default-model replacement:** `src/commands/configure.gateway-auth.ts:149` calls `applyAuthChoice(...)` with `setDefaultModel: true`, then goes straight to the allowlist picker. There is no guard or confirmation before the primary model is replaced.
  - file: [src/commands/configure.gateway-auth.ts:149](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/commands/configure.gateway-auth.ts#L149)
