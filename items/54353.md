## Summary

Keep open. Current `main` still routes inbound Discord messages through `extensions/discord/src/monitor/message-handler.preflight.ts`, and that file still contains the reported allowlist-bypass computation (`channelConfig?.allowed !== false`) for unconfigured channels. The PR discussion's substantive review comment about the live path remaining unfixed matches the repository state today. The separate review comment about missing helper modules is now stale because those monitor helper files exist on `main`, but that does not change the main conclusion: the requested behavior is not implemented on `main`, so this PR should not be conservatively closed as already fixed or obsolete.
