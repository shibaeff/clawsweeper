## Summary

Keep open. Current `main` does not contain this Windows restart-wrapper change, so the requested behavior is not implemented on `main`. The PR also still has unresolved correctness/regression concerns in review: it drops existing `cmd.exe` argument escaping for `%`/`!` in script paths and still depends on `wscript.exe` without a fallback if Windows Script Host is unavailable.
