## Summary

Current `main` already implements the reported behavior: unmanaged `openclaw gateway restart` sends `SIGUSR1`, unmanaged stop sends `SIGTERM`, and the launchd-supervised restart path exits cleanly for supervisor relaunch instead of treating restart as a stop. The same implementation is present in the latest release tag `v2026.4.22`.
