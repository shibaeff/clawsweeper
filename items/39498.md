## Summary

Keep open. Current `main` does not implement the proposed fix, and the PR's shared-normalizer change would likely regress valid non-header secrets and secret-presence checks.

## Evidence

- **Current main still uses trim-only secret normalization:** `hasConfiguredSecretInput` and `normalizeSecretInputString` on `main` still only trim strings; they do not call the Latin-1/ByteString-safe helper proposed in the PR, so the requested behavior is not already implemented.
  - file: [src/config/types.secrets.ts:132](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/config/types.secrets.ts#L132)
  - command: `nl -ba src/config/types.secrets.ts | sed -n '128,150p'`
  - sha: [07cf1dd65c93](https://github.com/openclaw/openclaw/commit/07cf1dd65c937c75cf48a1645ea580404f5056b4)
- **ByteString failure is still reproducible at the platform boundary:** A direct Node `Headers` construction with U+2502 still throws `Cannot convert argument to a ByteString... value of 9474`, matching the PR/issue description.
  - command: `node -e "try { new Headers({authorization: 'Bearer \u2502sk-test'}); console.log('ok'); } catch (error) { console.log(String(error)); process.exitCode = 0; }"`
- **Current main still has a header-bound callsite using the trim-only helper:** Comfy resolves `config.apiKey` through `normalizeSecretInputString` and then places that value directly into the `X-API-Key` header, so the ByteString-risk path still exists on `main`.
  - file: [extensions/comfy/workflow-runtime.ts:167](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/extensions/comfy/workflow-runtime.ts#L167)

- **The proposed PR change would affect non-header secret flows:** Matrix resolves configured `password`/`accessToken` values through `normalizeResolvedSecretInputString`; if `normalizeSecretInputString` were changed to the Latin-1-stripping helper, valid non-Latin1 passwords used in JSON-body auth would be mutated before use. This matches the PR review concern and is supported by the current callsite.
