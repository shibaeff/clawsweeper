## Summary

Current `main` still has the gap this PR is trying to close. OpenClaw only strips aborted assistant messages for the special `sessions_yield` abort path; the normal abort path records a prompt error but does not remove an aborted assistant tail from the live session transcript or session file. I found replay-time sanitizers for aborted turns, but those are applied when rebuilding history for later prompts, not when the abort happens, so they do not prove the stale-content bug is already fixed on `main`.
