## Summary

Keep open. Current `main` does not show a first-class OpenClaw workspace directory-listing tool that is documented, cataloged, and policy-managed alongside `read`/`write`/`edit`. There are hints of upstream Pi built-ins like `ls`, but that is not strong enough evidence that this request is already implemented on OpenClaw `main`.

## Evidence

  - file: [docs/gateway/config-tools.md:34](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/docs/gateway/config-tools.md#L34)

## Risks / Open Questions

- Embedded Pi runtimes may already expose an upstream `ls` builtin, so part of the gap may be documentation/product-surface rather than raw capability.
- There is an ambiguity between upstream Pi built-ins and OpenClaw first-class tools; closing this as implemented would be too aggressive without stronger proof in current `main` docs/catalog/tests.
