## Summary

Current OpenClaw already buckets dreaming session-corpus output by each message timestamp instead of the transcript file mtime, and current tests cover the cross-day/older-message case directly. The latest shipped release also contains that timestamp-based logic. The later issue comments about `cron service unavailable` describe a separate cron reconciliation path, which current tests cover independently.

## Evidence

- **Current main uses per-message timestamps for session-corpus day bucketing:** In session ingestion, the corpus day is derived from `entry.messageTimestampsMs[index]` and only falls back to file mtime when a message timestamp is missing. That is the behavior needed for sessions that cross midnight.
