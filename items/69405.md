## Summary

The reported stale-busy-state path is already fixed in the Control UI: current `main` clears `chatRunId` on terminal chat events, the send guard only blocks while that flag is set, and the same behavior was already shipped in `v2026.4.23` with regression coverage.

## Evidence

- **Terminal chat events clear the active run on current main:** `handleChatEvent` nulls `state.chatRunId` when chat state becomes `final`, `aborted`, or `error`, which directly addresses the reported stale-busy-state theory.
