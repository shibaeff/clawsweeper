## Summary

Keep open. Current `main` still documents the static Browserbase `cdpUrl` flow and the browser integration still treats Browserbase as a plain remote CDP endpoint; there is no code for creating Browserbase sessions via REST or consuming an ephemeral `connectUrl`.

## Evidence

- **Docs still recommend static Browserbase websocket URL:** The Browserbase docs example still tells users to configure `cdpUrl: "wss://connect.browserbase.com?apiKey=<BROWSERBASE_API_KEY>"`.
  - file: [docs/tools/browser.md:325](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/tools/browser.md#L325)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Docs still claim no manual session creation is needed:** The same page still says Browserbase auto-creates a browser session on WebSocket connect, which matches the issue's reported stale guidance.
  - file: [docs/tools/browser.md:338](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/tools/browser.md#L338)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Runtime only discovers `/json/version` or reuses the configured websocket URL:** `getChromeWebSocketUrl()` only does `/json/version` discovery and otherwise returns the original `cdpUrl` for bare `ws/wss` inputs; there is no provider-specific REST session creation path here.
  - file: [extensions/browser/src/browser/chrome.ts:221](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/browser/src/browser/chrome.ts#L221)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Browser connection path uses the configured endpoint directly:** `connectBrowser()` asks `getChromeWebSocketUrl()` for an endpoint and then calls `chromium.connectOverCDP()` on that URL. It does not call any Browserbase session API before connecting.
  - file: [extensions/browser/src/browser/pw-session.ts:447](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/browser/src/browser/pw-session.ts#L447)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Repo search found no Browserbase session-API implementation:** A repo-wide search found Browserbase references in docs, config normalization, and rate-limit messaging, but no `/v1/sessions`, `connectUrl`, or `api.browserbase` handling in the browser integration.
  - command: `rg -n "Browserbase|browserbase|connectUrl|api\.browserbase|/v1/sessions" extensions/browser/src docs/tools/browser.md src/plugin-sdk src/config -g '!**/dist/**'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
