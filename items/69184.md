## Summary

Keep open. Current `main` does not implement a `download` action on the existing `feishu_drive` tool, and the request is a coherent feature gap in a bundled Feishu plugin surface rather than something already shipped, obsolete, or too unclear to act on.

## Evidence

- **Tool schema has no download action:** `FeishuDriveSchema` enumerates `list`, `info`, `create_folder`, `move`, `delete`, `list_comments`, `list_comment_replies`, `add_comment`, and `reply_comment`; there is no `download` variant.
  - file: [extensions/feishu/src/drive-schema.ts:22](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/feishu/src/drive-schema.ts#L22)

  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Runtime dispatch omits download handling:** The registered `feishu_drive` tool description lists the same actions, and its `switch (p.action)` handles only those cases. There is no branch that downloads Drive media.
  - file: [extensions/feishu/src/drive.ts:758](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/feishu/src/drive.ts#L758)
