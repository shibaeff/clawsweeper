## Summary

Keep open. Current `main` still hard-caps non-loopback `gateway probe` targets to 1500ms, and the tests explicitly lock that behavior in. The CLI still presents `--timeout` as the overall probe budget and reports that overall value in JSON/text output without surfacing the smaller per-target remote budget, so the false-negative report remains plausible and current rather than obsolete.

## Evidence

- **Current main still clamps non-loopback probe targets to 1500ms:** `resolveProbeBudgetMs()` returns `Math.min(1500, overallMs)` for any target that is not loopback; `runGatewayStatusProbePass()` passes that derived value into `probeGateway()` for each target.
