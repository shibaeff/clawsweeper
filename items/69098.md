## Summary

Current `main` has a strict-provider replay repair that matches this OpenRouter failure class: OpenRouter models run on the `openai-completions` transport, replay sanitization now repairs/moves missing tool results and drops aborted/error assistant turns before replay, and new unit/live regression coverage proves the repaired transcript shape is accepted by real models. This appears fixed on `main` but not yet in a tagged release.

## Evidence

- **OpenRouter uses the strict OpenAI-compatible transport path:** The bundled OpenRouter plugin resolves runtime chat models with `api: "openai-completions"`, so they flow through the strict OpenAI-compatible replay policy rather than a provider-specific custom path.
