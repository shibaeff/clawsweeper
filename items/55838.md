## Summary

Keep open. Current `main` still only backs off `sendChatAction` 401s and rethrows other failures, so the requested transient-network behavior is not implemented. The PR is useful, but the final patch still has an unresolved correctness bug: transient failures remain counted across intervening non-transient errors, which can falsely trigger backoff/suspension.
