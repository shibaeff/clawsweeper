## Summary

Current `main` still does not infer a cron job `agentId` from the workspace or routed agent context when `openclaw cron add` is called without `--agent`. The CLI only forwards an explicit `--agent`, isolated cron runs still fall back to the default agent when `job.agentId` is missing, and the current test suite covers explicit `--agent` but not workspace inference. This PR is addressing a real gap and should remain open for maintainer review rather than conservative auto-close.

## Evidence

- **Cron add CLI still drops agent context unless `--agent` is passed:** `register.cron-add.ts` normalizes `opts.agent` and otherwise leaves `agentId` as `undefined`; there is no workspace-based inference on current `main`.
  - file: [src/cli/cron-cli/register.cron-add.ts:127](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/cron-cli/register.cron-add.ts#L127)
  - command: `nl -ba src/cli/cron-cli/register.cron-add.ts | sed -n '118,232p'`
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Cron agent execution still falls back to default agent when job has no agentId:** `prepareCronRunContext` picks `input.agentId`, then `input.job.agentId`, and otherwise falls back to `resolveDefaultAgentId(input.cfg)`, so an isolated cron job created without `agentId` will execute under the default agent.
  - file: [src/cron/isolated-agent/run.ts:428](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cron/isolated-agent/run.ts#L428)

  - file: [src/cli/cron-cli.test.ts:460](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cli/cron-cli.test.ts#L460)
