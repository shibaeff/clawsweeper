## Summary

Keep open. Current `main` includes the narrow late-compaction repair from #45493, but it does not implement the broader "single authoritative post-timeout compaction reconciliation across session state" requested here. The late-success path only reconciles `compactionCount`; token snapshots, session-id rotation, lifecycle hooks, memory-flush bookkeeping, and other compaction-derived state still update through separate code paths.

## Evidence

- **Shipped fix is explicitly narrow:** The changelog entry for #45493 says the shipped change only reconciles `sessions.json.compactionCount` after late embedded auto-compaction success.
  - file: [CHANGELOG.md:1993](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/CHANGELOG.md#L1993)
  - command: `nl -ba CHANGELOG.md | sed -n '1988,1998p'`

  - file: [src/agents/pi-embedded-subscribe.handlers.compaction.ts:42](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-embedded-subscribe.handlers.compaction.ts#L42)
