## Summary

Current `main` still appears to have the Feishu reaction-suffix bug path, and this PR is not safely closable. OpenClaw still generates synthetic Feishu reaction IDs with `:reaction:...` suffixes, then routes those IDs back into multiple Feishu API calls without normalization. The proposed fix remains relevant, but it likely needs a refresh because current `main` has additional reply paths such as streaming cards that also pass raw `replyToMessageId` values.
