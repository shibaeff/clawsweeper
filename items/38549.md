## Close Comment

Thanks for the submission. Closing after a Codex review because this now belongs on ClawHub rather than as a docs-only change in `openclaw/openclaw`.

Current `main` explicitly says ClawHub is the canonical discovery surface for community plugins and that contributors should not open docs-only PRs just to add their plugin here (`docs/plugins/community.md:14` and `docs/plugins/community.md:147`). The broader project direction in `VISION.md:57` also keeps optional plugins in their own repos and points discovery to ClawHub.

If there is a source-doc issue to fix in OpenClaw itself, a focused docs PR is still welcome. For plugin discoverability, the current maintainer path is to publish and maintain it through ClawHub.
