## Summary

Current `main` does not downgrade Gemini CLI `gemini-3.1-*` selections to `gemini-3-*` on the outgoing CLI command. The cited template IDs in `extensions/google/provider-models.ts` are used for template lookup only; the resolved runtime model keeps the user-selected `3.1` ID, and the CLI runner passes that exact ID to `--model`.

## Evidence

- **Gemini CLI provider does not normalize model IDs:** `google-gemini-cli` resolves dynamic models through `resolveGoogleGeminiForwardCompatModel` and does not wire `normalizeGoogleModelId`; that normalization is only registered on the direct `google` provider.
  - file: [extensions/google/gemini-cli-provider.ts:123](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/google/gemini-cli-provider.ts#L123)
  - command: `nl -ba extensions/google/gemini-cli-provider.ts | sed -n '1,240p'; nl -ba extensions/google/provider-registration.ts | sed -n '1,140p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Template clone preserves selected 3.1 model ID:** `resolveGoogleGeminiForwardCompatModel` passes the trimmed requested model ID into the clone helper, and `cloneFirstTemplateModel` overwrites the cloned template's `id` and `name` with that requested ID. The `gemini-3-pro-preview` / `gemini-3-flash-preview` entries are lookup templates, not the emitted runtime model ID.
  - file: [src/plugins/provider-model-helpers.ts:30](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/plugins/provider-model-helpers.ts#L30)
  - command: `nl -ba extensions/google/provider-models.ts | sed -n '118,200p'; nl -ba src/plugins/provider-model-helpers.ts | sed -n '1,220p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Regression scenario is covered by tests:** The Google provider tests explicitly cover the Gemini CLI case where the template source is `gemini-3-pro-preview` but the resolved runtime model remains `id: gemini-3.1-pro-preview`.
  - file: [extensions/google/provider-models.test.ts:91](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/google/provider-models.test.ts#L91)
  - command: `nl -ba extensions/google/provider-models.test.ts | sed -n '88,120p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **CLI runner passes the resolved model ID through unchanged:** `buildCliArgs` appends `params.modelId` directly after `--model`, and the corresponding test asserts the final argv contains `--model gemini-3.1-pro-preview`.
  - file: [src/agents/cli-runner/helpers.ts:366](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/cli-runner/helpers.ts#L366)

  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
