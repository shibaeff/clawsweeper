## Summary

Keep open. Current `main` still exposes `server_error` as a cron retry option in config/docs, but the runtime does not honor it end-to-end: cron retry detection only regex-matches raw `5xx` text, failover classification maps `server_error` payloads to `timeout` instead of a distinct reason, and the cron protocol/schema still cannot surface `lastErrorReason: "server_error"`. This PR addresses a real gap on current `main`, not an already-implemented fix.
