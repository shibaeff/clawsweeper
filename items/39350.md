## Summary

Keep open. Current `main` still only suppresses exact control tokens in the final chat payload, while lead-fragment suppression is limited to delta/flush paths. That means the PR’s reported partial-`NO_REPLY` final-message leak is still plausible on `main`, and the PR discussion also surfaced a real regression risk for legitimate uppercase `"NO"` replies, so there is not strong evidence to close this PR as implemented, obsolete, or incoherent.

## Evidence
