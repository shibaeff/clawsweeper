## Summary

Keep open. The report is fresh and concrete, and current `main` still shows the same core shape: `openclaw agent` waits for a final Gateway `agent` response with a client-side timeout, then blindly falls back to embedded mode on any Gateway error, while embedded session writes still use exclusive `.jsonl.lock` files with a 10s default contention timeout. I did not find a main-branch fix or doc update that would justify conservative closure.

## Evidence

- **CLI still waits for final `agent` response and falls back to embedded on any Gateway error:** `agentViaGatewayCommand` computes a wrapper timeout as `max(10000, (timeoutSeconds + 30) * 1000)`, calls `callGateway(..., expectFinal: true)`, and `agentCliCommand` catches any error and immediately reruns the same request via embedded mode. With `--timeout 30`, that wrapper timeout is 60000ms.
  - file: [src/commands/agent-via-gateway.ts:115](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/commands/agent-via-gateway.ts#L115)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Gateway `agent` is still a two-phase response on main:** The server sends an initial `{status:"accepted", runId}` response, then later sends a second response with `{status:"ok", summary:"completed", result}` on the same request id. This is the path the CLI is waiting on when it uses `expectFinal: true`.
  - file: [src/gateway/server-methods/agent.ts:274](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/gateway/server-methods/agent.ts#L274)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Embedded/session writes still use exclusive per-session lock files with 10s default timeout:** `acquireSessionWriteLock()` creates `<session>.jsonl.lock` with `fs.open(..., "wx")`, retries until timeout, and then throws `SessionWriteLockTimeoutError`. The default timeout is 10000ms when none is supplied.
  - file: [src/agents/session-write-lock.ts:476](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/session-write-lock.ts#L476)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Public docs still describe fallback but do not document daemon/embedded contention or per-invocation state-dir isolation for `openclaw agent`:** The docs say Gateway mode falls back to embedded when the Gateway request fails, but they do not describe the limitation reported here. The CLI surface for `openclaw agent` exposes `--local` and `--timeout`, but no dedicated state-dir isolation flag on this command.
  - file: [docs/cli/agent.md:53](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/docs/cli/agent.md#L53)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Main-vs-release delta does not show an obvious fix for this report:** Diffing latest release `a979721` to current `main` over the touched files shows changes to JSON stderr routing, MCP cleanup, session-lock error typing, and gateway abort bookkeeping, but not a change to the CLI fallback pattern or a documented fix for daemon/embedded contention.
  - command: `git diff --stat a9797214338ba31c52c796adbb75afb16e0684a9..c070509b7f8fdbe479e5d829643782c272e30e83 -- src/gateway/server-methods/agent.ts src/commands/agent-via-gateway.ts src/agents/session-write-lock.ts docs/cli/agent.md`
