## Summary

Current main, and the shipped v2026.4.23 release, already evaluate exec allowlist entries by resolved executable path on non-Windows, so additional command arguments should not cause a path entry like `/usr/bin/find` to miss.

## Evidence

- **Path-only allowlist semantics in shipped code:** `matchAllowlist()` returns the first resolved-path match on non-Windows and only consults `argPattern` on Windows. Extra argv does not invalidate a POSIX path match.
  - file: [src/infra/exec-command-resolution.ts:318](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/exec-command-resolution.ts#L318)
