Current `main` does not contain this PR's `keyed(selectedAgent.id, ...)` remount or an equivalent reset path. The overview panel still reuses the same Lit subtree while rendering native `<select>` and `<input>` controls for model settings, and the existing tests do not cover switching agents while the overview panel stays active. On the available evidence, this remains a plausible unfixed UI bug rather than a cleanup candidate.
