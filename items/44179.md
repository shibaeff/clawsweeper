## Summary

Current `main` still leaves the TUI/main chat output path unsanitized for model special tokens. The canonical stripping logic exists, but `resolveFinalAssistantText` and the assistant render path in `src/tui/tui-formatters.ts` still return text after only `sanitizeRenderableText`, which does not call `stripModelSpecialTokens`. This issue is not implemented on `main` at `aa21d4ea7e9aab034acbf0b13f8c3575057c5c54`.
