## Summary

Keep open. Current `main` still emits the embedded-runner `llm_output` hook even when no assistant message exists, which contradicts the documented contract and the CLI runner's current behavior. The PR's core guard looks directionally correct, but the same PR also bundles an unrelated `chat-content` change that review comments flagged as behaviorally unsafe, so there is not strong evidence to auto-close it.
