## Summary

Current `main` already supports cross-agent `sessions_spawn` for custom agent IDs, but the allowlist lives under `agents.*.subagents.allowAgents`, not `tools.agentToAgent.allow`. The reported configuration targets different tools.

## Evidence

- **Spawn code uses subagent allowlists for `agentId`:** `sessions_spawn` resolves cross-agent targeting from `resolveAgentConfig(cfg, requesterAgentId)?.subagents?.allowAgents` or `cfg.agents?.defaults?.subagents?.allowAgents`, and only returns `allowed: none` when those lists are empty.
