## Summary

Keep open. Current `main` moved PDF fallback extraction into the bundled `document-extract` plugin, but that extractor still calls PDF.js without `standardFontDataUrl`, so this PR's bug fix is not implemented on `main` and still needs a rebase onto the new plugin-owned path.

## Evidence

- **Current main still omits the PDF.js font asset option:** The bundled PDF extractor defines `getDocument` with only `{ data, disableWorker? }` and calls `pdfJsModule.getDocument({ data: new Uint8Array(request.buffer), disableWorker: true })`, with no `standardFontDataUrl` or equivalent runtime asset resolution.
