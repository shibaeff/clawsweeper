## Summary

Current `main` already preserves real Ollama Cloud credentials instead of falling back to the synthetic `ollama-local` marker, and the shipped `v2026.4.23` release contains the same implementation plus regression coverage for explicit remote/cloud configs and Authorization header precedence.

## Evidence

- **Discovery now prefers real/env/explicit Ollama credentials:** `resolveOllamaDiscoveryApiKey` returns `OLLAMA_API_KEY`, then explicit provider `apiKey`, then resolved runtime key, and only falls back to `ollama-local` last. This directly addresses the report that the real key was ignored.
