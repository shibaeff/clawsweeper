## Summary

Keep open. Current `main` still prefers the rate-limit path for `429`/quota-style provider errors before billing handling, so the core masking complaint is not implemented away. Some secondary symptoms look partially stale on current `main` (`/model` is now handled locally through `sessions.patch`, and session-memory slug generation falls back instead of showing an obvious retry loop), but the main issue is not strong enough to close.
