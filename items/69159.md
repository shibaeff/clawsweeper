## Summary

Current `main` only partially overlaps with this PR. OpenClaw now has a typed session-lock timeout error and a separate fix that stops those lock waits from being misclassified as model failover, but `runEmbeddedAttempt` still does not forward the run `--timeout` into `acquireSessionWriteLock`, and the public agent-harness SDK surface still does not re-export the timeout error. This PR remains relevant, though it likely needs rebasing onto the newer `SessionWriteLockTimeoutError` API.

## Risks / Open Questions

- `main` renamed the surfaced error to `SessionWriteLockTimeoutError`, so this PR's API shape is already behind current code and likely conflicts semantically.
- Even after PR #68700, long-running `--session-id` turns can still hit the helper's default 10s lock wait and fail early because `params.timeoutMs` is not yet forwarded from `runEmbeddedAttempt`.
