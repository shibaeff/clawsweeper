## Summary

Keep open. Current `main` still does not implement the PR's core behavior: sandboxed skill prompts are rendered from host absolute paths (optionally `~/`-compacted), not workspace-relative paths like `skills/.../SKILL.md` that match the sandbox mount at `/workspace`.

## Evidence

- **Current path compaction is home-only:** `compactSkillPaths` on `main` only calls `compactHomePath(...)`; it does not accept `workspaceDir` or rewrite workspace-local skills to relative paths.
  - file: [src/agents/skills/workspace.ts:68](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/skills/workspace.ts#L68)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '35,95p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Skill entries keep host filesystem paths:** Discovered skills are stored with `filePath: skillMd`, where `skillMd` is the absolute host path under the resolved workspace root.
  - file: [src/agents/skills/workspace.ts:494](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/skills/workspace.ts#L494)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '360,560p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Prompt renderer emits `skill.filePath` verbatim:** The available-skills prompt writes `<location>${skill.filePath}</location>` directly, so without workspace-relative compaction the model sees the host path.
  - file: [src/agents/skills/skill-contract.ts:59](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/skills/skill-contract.ts#L59)

- **Sandboxed runs still build prompts from the host-side sandbox workspace:** Embedded runs call `resolveSkillsPromptForRun(...)` with `workspaceDir: effectiveWorkspace`, and for read-only sandboxes `effectiveWorkspace` is `sandbox.workspaceDir` on the host side, while the container workdir remains `/workspace`. That means host paths in `<location>` do not match the in-container path the model can read.
