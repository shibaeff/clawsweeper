## Summary

Keep open. Current `main` does not implement the proposed Feishu `mentionForwardMode` seam or its docs/tests, and the existing Feishu bot path still auto-injects/passes mention targets unconditionally. The PR remains a live bug fix rather than cleanup-close material.

## Evidence

- **Current main lacks a config seam for disabling mention forwarding:** `FeishuGroupSchema` and the shared Feishu config shape expose `requireMention` and related settings, but no `mentionForwardMode` or equivalent field exists on current `main`. That means users still have no supported config path to disable forwarding behavior.
