## Summary

Current main, and the shipped `v2026.4.23` tag I checked, no longer keep the prior active transcript attached to a rolled-over daily/reset session. The runtime now captures the previous session entry for explicit and stale daily resets, mints a fresh `sessionId`, resolves/persists the transcript for that new session, and archives the old transcript instead of continuing to use it.

## Evidence

- **Daily/stale reset path captures previous session and creates a fresh session id:** `initSessionState()` explicitly snapshots the prior entry for both explicit resets and stale daily resets, then generates a new `sessionId` when the old entry is not fresh.
  - file: [src/auto-reply/reply/session.ts:455](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/auto-reply/reply/session.ts#L455)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Rollover resolves/persists the new session transcript and archives the old one:** After rollover, `initSessionState()` calls `resolveAndPersistSessionFile(...)` for the new session entry, clears stale token/context counters for the new session, then archives the previous transcript via `archiveSessionTranscriptsDetailed(...)`. That means the old transcript is not kept as the active session transcript across days.
  - file: [src/auto-reply/reply/session.ts:720](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/auto-reply/reply/session.ts#L720)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Session-file resolver only reuses a transcript path when the session id is unchanged:** `resolveAndPersistSessionFile()` computes `shouldReusePersistedSessionFile` from `baseEntry.sessionId === sessionId`. When the session id changes, it resolves/persists a new transcript path instead of reusing the old one.
  - file: [src/config/sessions/session-file.ts:21](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/config/sessions/session-file.ts#L21)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Regression test covers transcript-path rotation on session rollover:** `resolveAndPersistSessionFile` has a test that changes the `sessionId` on the same `sessionKey` and asserts the resulting `sessionFile` is a different path and gets persisted.
  - file: [src/config/sessions/sessions.test.ts:400](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/config/sessions/sessions.test.ts#L400)
