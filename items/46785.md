## Summary

Current main does not expose a `sessions_spawn.maxAnnounceChars` parameter, but the reported Telegram failure mode is obsolete: sub-agent completion announcements are routed back through normal agent delivery, and Telegram outbound sends chunk long HTML text to 4000-character messages instead of silently dropping it.

## Close Comment

Closing after a Codex review. On current `main`, the failure mode described here no longer matches the Telegram completion-delivery path: sub-agent completion announcements are routed back through the normal `agent` delivery flow (`src/agents/subagent-announce-delivery.ts:517`), the Telegram adapter sends both text and payload messages through `sendMessageTelegram` in HTML mode (`extensions/telegram/src/outbound-adapter.ts:63` and `extensions/telegram/src/outbound-adapter.ts:207`), and `sendMessageTelegram` chunks long HTML text into 4000-character sends (`extensions/telegram/src/send.ts:751`, covered by `extensions/telegram/src/send.test.ts:1774`). I do not see a current-main path where an over-4096-character Telegram completion message is silently dropped for the reason described here, so I’m closing this as cannot reproduce rather than adding `sessions_spawn.maxAnnounceChars`. If you still hit this on current `main`, please open a new issue with the exact channel/plugin path and payload shape that bypassed this chunking flow.
