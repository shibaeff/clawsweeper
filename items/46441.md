## Summary

Keep open. Current `main` already supports plugin-level `before_tool_call` blocking and plugin approvals, and the linked CRE plugin is a real workaround/example, but there is still no standardized `GuardrailProvider` contract or `guardrails` config on `main`. The issue is also active and has a linked open PR (#64868), so it is not a conservative auto-close candidate.

## Evidence

That cuts against closing this as pure ClawHub/plugin-only scope.
