## Summary

Current `main` still contains the unguarded LM Studio `normalizeConfig` hook, and the surrounding core/runtime code makes the reported foreign-provider `/v1` mutation plausible. The PR is small, aligned with existing hook patterns, and adds the missing regression coverage, so there is not strong evidence to auto-close it.
