## Summary

Do not close. Current `main` mitigates some cron-failure/stuck-marker cases and documents `cron status`/`cron runs`, but it does not implement the PR’s proposed doctor-side cron reliability diagnostics. The underlying request is still plausibly useful, and the evidence is not strong enough for an `implemented_on_main` close.
