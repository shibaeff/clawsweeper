## Summary

Keep open. Current `main` and the latest shipped release still return plain `filename="..."` values verbatim in the Feishu download path, so the UTF-8-from-Latin-1 recovery proposed in PR #48578 is not implemented on `main`. The existing test coverage only exercises an ASCII filename case, and a simple reproduction of the current parser leaves mojibake unchanged. This remains a plausible, scoped bugfix rather than a cleanup close.
