## Summary

Keep open. Current `main` does not implement this PR's partial-mode coalescing approach: Telegram still rotates/materializes the answer preview on assistant-message boundaries, the test suite explicitly expects `forceNewMessage()` in `streamMode: "partial"`, and the proposed `revive()` API is absent. That matches the maintainer's March 8 triage comment to hold the PR for rebase/alignment and unresolved boundary regressions.
