## Summary

Keep open. Current `main` still drops per-session model/provider overrides on implicit daily freshness rollovers, so this PR appears to address a real unfixed bug.

## Evidence

- **Implicit daily rollover still happens with `resetTriggered=false`:** Current tests cover the stale daily reset path and assert it creates a new session while leaving `resetTriggered` false (`isNewSession === true`, `resetTriggered === false`). That is the exact path described in the PR.
  - file: [src/auto-reply/reply/session.test.ts:2271](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/auto-reply/reply/session.test.ts#L2271)
  - command: `nl -ba src/auto-reply/reply/session.test.ts | sed -n '2268,2316p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Model/provider carry-over is still gated on explicit reset only:** In the new-session branch, preservation of model/provider selection only happens inside `if (resetTriggered && entry)`, so implicit stale/daily rollovers do not preserve those fields.
  - file: [src/auto-reply/reply/session.ts:496](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/auto-reply/reply/session.ts#L496)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '448,610p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Stale rollover cannot fall back to the old entry:** `baseEntry` is only set when the existing session is reused. On a stale daily rollover, `baseEntry` becomes `undefined`, and the final `sessionEntry.modelOverride` / `providerOverride` fields therefore depend entirely on the persisted values that were never set in the implicit-reset path.
  - file: [src/auto-reply/reply/session.ts:529](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/auto-reply/reply/session.ts#L529)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '525,610p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Current helper is explicitly scoped to `/new` and `/reset`:** `resolveResetPreservedSelection` documents that it decides which overrides survive a `/new` or `/reset`, matching the code path above and confirming there is no separate preservation mechanism for daily freshness rollovers.
  - file: [src/config/sessions/reset-preserved-selection.ts:14](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/config/sessions/reset-preserved-selection.ts#L14)
