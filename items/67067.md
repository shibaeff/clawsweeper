## Summary

The request is a valid idea, but current main positions this as plugin/automation scope rather than an OpenClaw core or bundled active-memory change. Active Memory is intentionally a bounded pre-reply recall plugin, while scheduled reminders and proactive behavior are already exposed through cron, heartbeat, and plugin hook surfaces that are intended for ClawHub/community plugins.
