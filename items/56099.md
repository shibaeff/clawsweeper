## Summary

Current `main` still hard-cancels compaction when `modelRegistry.getApiKeyAndHeaders` is missing, so the compaction bug described in PR #56099 is not implemented on `main`. The PR also carries an unrelated channel-loader change that looks largely superseded on current `main`, and the posted review correctly flags a preserve-history risk in the proposed auth fallback, so this is not a conservative auto-close candidate.
