## Summary

Keep open. Current `main` does not already contain the requested fixes, and the PR as described is not merge-ready: it would address a real export-template bug, but at least two of its proposed fixes introduce or preserve correctness problems.

## Evidence

A read-only reproduction with `node -e` against current files reported `true true true` for leftover `JS`, `MARKED_JS`, and `HIGHLIGHT_JS` formatted placeholders after the existing replacement chain.
  - file: [src/auto-reply/reply/commands-export-session.ts:91](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/auto-reply/reply/commands-export-session.ts#L91)
  - command: `node -e 'const fs=require("fs"); const template=fs.readFileSync("src/auto-reply/reply/export-html/template.html","utf8"); const out=template.replace("{{CSS}}","").replace("{{JS}}","MAIN_JS").replace("{{SESSION_DATA}}","DATA").replace("{{MARKED_JS}}","MARKED").replace("{{HIGHLIGHT_JS}}","HL"); console.log(/\{\s*\{\s*JS\s*;?\s*\}\s*\}/.test(out), /\{\s*\{\s*MARKED_JS\s*;?\s*\}\s*\}/.test(out), /\{\s*\{\s*HIGHLIGHT_JS\s*;?\s*\}\s*\}/.test(out));'`

The PR diff in the review context changes `resolveApiKeyForProvider` to return only `{ source, mode: "oauth" }` for google-vertex ADC, so those paths would throw `No API key resolved...` instead of working.
  - file: [src/agents/model-auth-runtime-shared.ts:28](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/model-auth-runtime-shared.ts#L28)

That means legitimate hardening rules such as denying `camera.snap` would be warned as "unknown", so this part of the PR is misleading as written.
