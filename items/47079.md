## Summary

The reported Mistral `max_completion_tokens` incompatibility is already fixed on current `main` and documented as shipped in v2026.3.28. Current code resolves Mistral models to `max_tokens`, wires that compat through the Mistral provider plugin, and includes targeted regression coverage for both direct Mistral and native-host custom routes.
