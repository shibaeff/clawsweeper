## Summary

Current `main` already handles this class of LM Studio/custom-provider vision routing. Explicit `agents.defaults.imageModel` is now returned before any fallback lookup, and config-defined providers with image-capable models are auto-registered for media-understanding, with regression coverage for custom providers.

## Close Comment

Closing as implemented on current `main` after a Codex review. The reported fallback path is no longer how image model resolution works: `agents.defaults.imageModel` is returned directly in `src/media-understanding/runner.ts:597`, config-defined providers with image-capable models are auto-registered in `src/media-understanding/provider-registry.ts:41`, and the image tool now resolves provider-scoped vision models from config in `src/agents/tools/image-tool.helpers.ts:137`. There is regression coverage for custom image-capable providers in `src/media-understanding/provider-registry.test.ts:88` and `src/agents/tools/image-tool.test.ts:682`. I could not tie this to a released tag yet, so if you are on `v2026.4.22` or older, please update to a newer build from `main`; if it still reproduces there, open a new issue with the exact `openclaw.json` provider block and model metadata.
