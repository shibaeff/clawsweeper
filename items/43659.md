## Summary

Keep open. Current `main` still uses one shared `github-copilot.token.json` cache entry with no GitHub-token/profile fingerprint, so rotating to a different `github-copilot` auth profile can still reuse the previous profile's exchanged Copilot token instead of forcing a fresh exchange.

## Evidence

- **Single shared cache path on main:** `resolveCopilotTokenCachePath()` still returns one fixed file, `credentials/github-copilot.token.json`, for all Copilot token exchanges.
  - file: [src/agents/github-copilot-token.ts:18](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/github-copilot-token.ts#L18)
  - command: `nl -ba src/agents/github-copilot-token.ts | sed -n '1,240p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Cache reuse ignores profile identity:** `resolveCopilotApiToken()` reuses any still-usable cached token based only on `token` and `expiresAt`; there is no fingerprint or profile check before returning the cached token.
  - file: [src/agents/github-copilot-token.ts:122](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/github-copilot-token.ts#L122)
