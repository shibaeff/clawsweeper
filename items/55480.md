## Summary

Keep open. Current `main` still drops Unicode-only Discord channel names from session group labels, but this PR is not safe to close or merge as-is because its `GroupSpace` changes would regress documented and tested guild-ID policy resolution.

## Evidence

- **Current main still strips non-ASCII channel names to an empty slug:** `normalizeDiscordSlug()` lowercases, removes `#`, then replaces every non-`[a-z0-9]` run with `-` and trims dashes. A pure non-ASCII channel name therefore normalizes to `""`.
  - file: [extensions/discord/src/monitor/allow-list.ts:90](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/discord/src/monitor/allow-list.ts#L90)
  - command: `nl -ba extensions/discord/src/monitor/allow-list.ts | sed -n '88,104p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Discord preflight still uses the ASCII-only slug for display:** `displayChannelSlug` is still assigned directly from `normalizeDiscordSlug(displayChannelName)` with no Unicode-preserving fallback on current `main`.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:764](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/discord/src/monitor/message-handler.preflight.ts#L764)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '752,790p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Missing displayChannelSlug still leads to guild-only fallback display names:** Inbound session metadata uses `GroupChannel` only when `displayChannelSlug` is non-empty; otherwise the session display name builder falls back to `space`/ID and prefixes bare group labels with `g-`, which matches the PR's reported symptom.
  - file: [src/config/sessions/metadata.ts:144](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/sessions/metadata.ts#L144)

The resolver checks the raw `groupSpace` key before normalized slug matching, and tests cover ID-keyed guild configs for `requireMention` and tool policy resolution.

- **Prior PR review already identified the same regression risk:** The existing Codex review comment on PR #55480 called out that changing `GroupSpace` to prefer `guildSlug` would break ID-keyed Discord guild policy lookups when configs omit `slug`. That matches the current docs/tests contract on `main`.

## Risks / Open Questions

- Unicode-only Discord channel names still appear to lose their channel label in session display names on current `main`.
- Merging PR #55480 as written would risk silently changing per-guild `requireMention` and tool-policy behavior for valid ID-keyed Discord configs.
