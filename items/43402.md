## Summary

Current `main` already preserves Telegram topic/thread context for skills/tools via the channel threading tool-context path, and the fix is documented as shipped in `v2026.4.22`.

## Evidence

## Close Comment

Closing as implemented on current `main`. Codex review: Telegram topic/thread context is now carried through the channel threading tool-context path instead of needing a separate `TELEGRAM_MESSAGE_THREAD_ID` env var. `extensions/telegram/src/threading-tool-context.ts:9` resolves the thread id from `MessageThreadId` or a `:topic:` target, `extensions/telegram/src/channel.ts:992` registers that for Telegram, and `src/auto-reply/reply/agent-runner-utils.ts:123` feeds it into native-command/tool runs so thread-aware skills/tools receive `currentThreadTs`. The checked-in coverage in `extensions/telegram/src/threading-tool-context.test.ts:24` and `src/auto-reply/reply/agent-runner-utils.test.ts:205` asserts that Telegram topic context is preserved. This also appears in the shipped changelog for `v2026.4.22` (`CHANGELOG.md`, entry for `#52217`). If you still need the exact legacy env var names for an external runner, please open a fresh issue against the current release with a minimal repro and the specific contract you need.
