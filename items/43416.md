## Summary

Keep open. Current `main` still does not implement a one-click copy control for assistant messages, so this is not `implemented_on_main`, and the PR still needs maintainer judgment because its proposed copy path appears to use raw assistant payload text rather than the rendered visible text when traces are hidden.

## Evidence

- **Current main lacks the requested copy affordance:** `ChatMessageBody` on `main` renders assistant/user text, attachments, and tool cards, then finishes with bubble styling. There is no copy button, no message-level `Button`, and no clipboard call in the assistant message rendering path. A repo-wide search only found pasteboard usage in composer/settings flows, not assistant message bubbles.
