## Summary

Keep open. Current `main` still enforces `tools.fs.workspaceOnly` as a workspace-root guard for `read`, and the skills prompt/snapshot path flow still preserves original managed/bundled skill locations rather than granting extra read access or rewriting them into workspace-local paths. A recent `main` change added sandbox skill copying, but that is only a partial mitigation and does not fully resolve the reported behavior.
