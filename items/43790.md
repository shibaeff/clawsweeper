## Summary

Current `main` does not implement this PR's explicit-agent message routing change. The embedded runner still computes an explicit agent id for tool runs, but `createOpenClawTools()` does not thread that id into `createMessageTool()`, and `createMessageTool()` still derives routing from `agentSessionKey` alone. The PR discussion also shows the author addressed the earlier fallback concern on the PR branch, so this remains plausible useful work rather than something conservative cleanup should close.
