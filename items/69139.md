## Summary

Keep open. Current `main` still rejects root-level `paperclip` metadata on `agent` requests, so this PR addresses a real gateway bug that is not yet implemented on `main`.

## Evidence

- **`AgentParamsSchema` on current `main` does not accept `paperclip`:** At `7f6452897e25543c9263b67aabf140379d1c0b0c`, `src/gateway/protocol/schema/agent.ts` defines `AgentParamsSchema` with named fields through `label` and then closes with `{ additionalProperties: false }`. There is no optional `paperclip` field, so a root-level `paperclip` object is still invalid.
  - file: [src/gateway/protocol/schema/agent.ts:131](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/gateway/protocol/schema/agent.ts#L131)
