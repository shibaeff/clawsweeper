## Summary

Keep open: current `main` still lacks the gateway-mode TUI subscription/event handling needed to receive transcript updates from other channels in real time.

## Evidence

- **TUI event loop ignores transcript events:** `client.onEvent` only routes `chat`, `chat.side_result`, and `agent` events. There is no handler for `session.message` or `sessions.changed`, so external transcript appends are not consumed by the TUI.
  - file: [src/tui/tui.ts:1034](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/tui/tui.ts#L1034)
  - command: `nl -ba src/tui/tui.ts | sed -n '1030,1065p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **TUI gateway backend has no subscribe call:** `GatewayChatClient` exposes `sendChat`, `abortChat`, `loadHistory`, `listSessions`, `listAgents`, `patchSession`, `resetSession`, `getGatewayStatus`, and `listModels`, but no wrapper for `sessions.subscribe` or `sessions.messages.subscribe`.
  - file: [src/tui/gateway-chat.ts:172](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/tui/gateway-chat.ts#L172)
  - command: `nl -ba src/tui/gateway-chat.ts | sed -n '150,230p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Gateway requires explicit transcript subscription:** The gateway implements `sessions.messages.subscribe` / `sessions.messages.unsubscribe` as the mechanism that registers per-session transcript listeners.
  - file: [src/gateway/server-methods/sessions.ts:608](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/server-methods/sessions.ts#L608)
  - command: `nl -ba src/gateway/server-methods/sessions.ts | sed -n '600,652p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Transcript appends are emitted as session.message:** Transcript updates are broadcast as `session.message` to session subscribers (plus general session subscribers), not as `chat` events.
  - file: [src/gateway/server-session-events.ts:83](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/server-session-events.ts#L83)
