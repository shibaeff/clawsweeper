## Summary

Keep open. Current `main` still keys voice-call sessions by normalized caller phone number and exposes no `sessionScope` or equivalent `per-call` option, so the requested behavior is not implemented.

## Evidence

- **Session storage is still phone-scoped:** `generateVoiceResponse()` strips non-digits from `from` and sets `sessionKey = `voice:${normalizedPhone}``, then loads/saves the shared session entry under that key. That will reuse one session across multiple calls from the same number.
  - file: [extensions/voice-call/src/response-generator.ts:188](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/voice-call/src/response-generator.ts#L188)
  - command: `nl -ba extensions/voice-call/src/response-generator.ts | sed -n '180,230p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **No config field exists for per-call vs per-phone scope:** `VoiceCallConfigSchema` defines the voice-call plugin config surface and does not include `sessionScope` or any similar selector. `normalizeVoiceCallConfig()` only merges the existing fields, so there is no hidden compatibility path for this option either.
  - file: [extensions/voice-call/src/config.ts:280](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/voice-call/src/config.ts#L280)

  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Current docs still expose no such option:** The voice-call plugin documentation lists the supported config keys under `plugins.entries.voice-call.config`; the published example includes provider, webhook, outbound, and streaming settings, but nothing for session scoping.
  - file: [docs/plugins/voice-call.md:54](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/plugins/voice-call.md#L54)

- **No regression coverage for session-scope behavior:** `response-generator` tests currently cover spoken-output parsing and prompt shaping only; there is no test asserting separate sessions per call ID or a configurable session scope, which matches the missing implementation.
