## Summary

Keep open. Current `main` still appears to leak unresolved `responsePrefix` template variables on early abort replies: the fast-abort path returns before any model-selection callback can populate prefix context, and both dispatcher and route-reply normalization preserve unresolved placeholders when context is missing.

## Evidence

- **Fast-abort short-circuits before normal reply execution:** `dispatchReplyFromConfig` handles fast aborts immediately and sends a bare abort payload via `routeReplyToOriginating(...)` or `dispatcher.sendFinalReply(...)`, with no extra prefix-context seeding in that branch.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:612](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/auto-reply/reply/dispatch-from-config.ts#L612)
  - command: `nl -ba src/auto-reply/reply/dispatch-from-config.ts | sed -n '600,640p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Reply prefix context only gets model/thinking data from `onModelSelected`:** `createReplyPrefixContext` initializes only `identityName`; `{provider}`, `{model}`, and `{thinkingLevel}` are filled later only when `onModelSelected(...)` runs.
  - file: [src/channels/reply-prefix.ts:31](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/channels/reply-prefix.ts#L31)
  - command: `nl -ba src/channels/reply-prefix.ts | sed -n '24,52p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Normalizer leaves raw template placeholders intact when context is missing:** `normalizeReplyPayload` uses the raw `responsePrefix` when `responsePrefixContext` is absent, and the tests explicitly assert unresolved placeholders like `[{model}]` are preserved rather than removed or replaced.
  - file: [src/auto-reply/reply/normalize-reply.ts:112](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/auto-reply/reply/normalize-reply.ts#L112)
  - command: `nl -ba src/auto-reply/reply/normalize-reply.ts | sed -n '108,130p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Regression expectation still only checks plain abort text:** The current fast-abort test only asserts that `dispatchReplyFromConfig` sends `{ text: "⚙️ Agent was aborted." }`; it does not cover response-prefix interpolation on that path.
  - file: [src/auto-reply/reply/dispatch-from-config.test.ts:1651](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/auto-reply/reply/dispatch-from-config.test.ts#L1651)
  - command: `nl -ba src/auto-reply/reply/dispatch-from-config.test.ts | sed -n '1640,1695p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Route-reply path also omits prefix context:** When abort replies are routed to the originating channel, `routeReply` resolves `responsePrefix` and calls `normalizeReplyPayload(...)` without any `responsePrefixContext`, so template placeholders remain unresolved there too.
  - file: [src/auto-reply/reply/route-reply.ts:112](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/auto-reply/reply/route-reply.ts#L112)
