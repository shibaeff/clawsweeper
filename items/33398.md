## Summary

Current main already covers the underlying request for Telegram forum topics: topics have isolated session keys, topic configs can inject topic-specific prompt context via `systemPrompt`, and topics can route to a dedicated agent via `agentId` for fully separate workspace/memory/bootstrap behavior.

## Close Comment

Closing from a Codex review because current `main` already covers the requested Telegram topic-specific context behavior. The runtime also exposes `MessageThreadId`, `IsForum`, and `TopicName` in inbound context (`extensions/telegram/src/bot-message-context.session.ts:322`), and the docs describe isolated topic session keys plus per-topic agent routing in `docs/channels/telegram.md:257` and `docs/channels/telegram.md:492`. The latest release `v2026.4.22` already contains these implementation paths.
