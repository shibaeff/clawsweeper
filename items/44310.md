## Summary

Current `main` does not implement per-model request timeout support for multi-model routing. OpenClaw exposes only global agent/LLM timeout knobs, and the per-model `agents.defaults.models.<provider>/<model>` schema still accepts only alias/params/streaming metadata.

## Evidence

- **Per-model config schema has no timeout field:** `AgentModelEntryConfig` only defines `alias`, `params`, and `streaming`, and `agents.defaults.models` is typed as `Record<string, AgentModelEntryConfig>`.
  - file: [src/config/types.agent-defaults.ts:31](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/config/types.agent-defaults.ts#L31)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '24,40p'; nl -ba src/config/types.agent-defaults.ts | sed -n '195,210p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Validation schema still rejects any extra per-model timeout key:** `AgentDefaultsSchema` defines each `agents.defaults.models.*` entry as a strict object with only `alias`, `params`, and `streaming`. There is no `requestTimeoutMs`, `requestTimeoutSeconds`, or similar key.
  - file: [src/config/zod-schema.agent-defaults.ts:52](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/config/zod-schema.agent-defaults.ts#L52)

## Risks / Open Questions

- `agents.defaults.llm.idleTimeoutSeconds` is a useful global workaround for slow providers, but it still cannot express different timeout budgets for different models in the same fallback chain.
- Provider-specific values could be tunneled through `agents.defaults.models.<id>.params`, but that is not a documented or generic OpenClaw routing contract and does not satisfy this feature request conservatively.
