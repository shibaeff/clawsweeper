## Summary

Current main already accepts image-only WebChat sends and persists uploaded images into transcript/media metadata, matching the reported "image appears briefly then disappears" behavior from 2026.3.7. The shipped changelog for 2026.4.20 also calls out a WebChat image fix for empty-turn gating.

## Close Comment

Closing from a Codex review because current `main` already implements this path. `src/gateway/server.chat.gateway-server-chat.test.ts:461` covers an image-only `chat.send`, `src/gateway/server-methods/chat.ts:741` and `src/gateway/server-methods/chat.ts:2489` persist uploaded images into transcript `MediaPath`/`MediaPaths` and emit the user turn immediately, and `src/gateway/server-methods/chat.directive-tags.test.ts:1720` verifies those saved media paths are present on the user transcript update. The shipped changelog also includes the matching WebChat image fix in `CHANGELOG.md:365` for version `2026.4.20`. If this still reproduces on a current release, please open a new issue with the exact version, whether the image was sent with or without text, and gateway logs.
