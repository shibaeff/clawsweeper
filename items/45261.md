## Summary

This request is a workflow-specific review-staging/provenance feature, and current main positions that kind of optional agent capability as plugin/skill territory rather than OpenClaw core.

## Risks / Open Questions

- If maintainers later want a repo-wide temporary-artifact ledger for many workflows, this issue may need to be reconsidered as a broader core design rather than a review-flow-specific feature.
- A follow-up docs/plugin pattern for review bundle provenance could still be useful even if core does not take this on.

## Close Comment

Thanks for the write-up. This Codex review suggests closing this as `clawhub` rather than tracking it in OpenClaw core.

Current `main` points optional workflow-specific behavior to plugins/skills: `VISION.md:54-60` says core stays lean and optional capability should usually ship as plugins, and `VISION.md:68-70` plus `VISION.md:101` says new skills should go to ClawHub first rather than core by default. The existing cleanup surface is also much narrower than this proposal: `docs/cli/sessions.md:69-71` limits cleanup to session stores/transcripts, and `src/infra/tmp-openclaw-dir.ts:5-168` only provides a hardened temp directory path, not a first-class temp-artifact registry with provenance/promotion semantics. There is also repo guidance to avoid Codex-specific cleanup unless a generic lifecycle gap exists (`docs/plan/codex-context-engine-harness.md:471-473`).

Given that scope, a ClawHub skill/plugin is the better fit for review-staged bundles: it can stage non-canonical review material, record provenance/canonical-source metadata, and provide promotion/cleanup commands without adding a new core temp-artifact taxonomy.
