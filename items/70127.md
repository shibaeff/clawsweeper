## Summary

Keep open. The bug is still present on current `main`: the MS Teams `search` action still calls `searchMessagesMSTeams()`, and that function still builds `.../messages?$search=...` requests for chat/channel message listing. Microsoft’s Graph docs for chat message listing only support `$top`, `$orderby`, and `$filter` on that endpoint, while Teams message search is documented against the signed-in user’s search surface rather than an app-only chat-message listing flow. ([learn.microsoft.com](https://learn.microsoft.com/hi-in/graph/api/chat-list-messages?view=graph-rest-beta&utm_source=openai))
