## Summary

Current main already supports cross-platform Docker builds through Docker's built-in `DOCKER_DEFAULT_PLATFORM` handling, because `scripts/docker/setup.sh` delegates directly to `docker build` without stripping Docker CLI environment variables. The requested `OPENCLAW_DOCKER_PLATFORM` alias would be a convenience layer, not a missing capability in current OpenClaw.

## Evidence

- **Docker build wrapper delegates to plain docker build:** `run_docker_build()` is implemented as `DOCKER_BUILDKIT=1 docker build "$@"`, so ambient Docker CLI environment variables still apply to the build command.
  - file: [scripts/docker/setup.sh:27](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/scripts/docker/setup.sh#L27)

  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Local gateway image build uses that wrapper unchanged:** The main image path calls `run_docker_build` with build args, tag, Dockerfile, and context only; there is no environment scrub or alternate builder path that would block Docker's own platform selection behavior.
  - file: [scripts/docker/setup.sh:474](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/scripts/docker/setup.sh#L474)

  - file: [scripts/docker/setup.sh:542](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/scripts/docker/setup.sh#L542)

- **Current OpenClaw Docker docs route users through setup.sh:** The public Docker install flow is `./scripts/docker/setup.sh`, and the env-var table lists only OpenClaw-specific knobs; nothing in the documented flow overrides or disables Docker's standard CLI env handling.
  - file: [docs/install/docker.md:29](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/docs/install/docker.md#L29)

- **Upstream Docker already provides the platform override:** Docker's official CLI reference documents `DOCKER_DEFAULT_PLATFORM` as the default platform for commands that accept `--platform`, and Docker's build reference documents `docker build --platform ...` for target selection. That covers the reported cross-architecture use case without an OpenClaw-specific alias.
  - command: `Reviewed official Docker docs: docs.docker.com/reference/cli/docker/ and docs.docker.com/engine/reference/commandline/build/`

## Risks / Open Questions

- Discoverability is still weak: OpenClaw's Docker docs do not currently mention `DOCKER_DEFAULT_PLATFORM`, so users may assume the workflow is unsupported.
- An OpenClaw-specific alias could still be a reasonable UX/docs enhancement later, but it is not required to achieve cross-platform builds on current main.
