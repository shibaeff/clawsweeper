## Summary

Keep open. Current `main` still has Anthropic prompt caching, but it does not implement the requested Anthropic-native server compaction knobs/transport handling or the mixed `1h` stable-prefix plus `5m` trailing-turn TTL behavior described in #65287.

## Evidence

- **Anthropic stream wrapper has no native compaction path:** `wrapAnthropicProviderStream()` only composes beta-header, service-tier, and fast-mode wrappers; there is no Anthropic compaction wrapper or request-param handling for `anthropicServerCompaction`, `anthropicCompactThreshold`, `anthropicCompactPauseAfter`, or `anthropicCompactInstructions`.
  - file: [extensions/anthropic/stream-wrappers.ts:191](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/anthropic/stream-wrappers.ts#L191)
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Requested Anthropic compaction symbols are absent on main:** Repository search for `anthropicServerCompaction|anthropicCompactThreshold|anthropicCompactPauseAfter|anthropicCompactInstructions|compact_20260112|context_management.edits` returned no matches in `extensions/`, `src/`, `docs/`, `test/`, or `packages/`.
  - command: `rg -n "anthropicServerCompaction|anthropicCompactThreshold|anthropicCompactPauseAfter|anthropicCompactInstructions|compact_20260112|context_management\.edits" extensions/anthropic src docs test packages`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Anthropic cache policy is one retention mode, not mixed stable/user TTLs:** `resolveCacheRetention()` resolves a single `cacheRetention` value (`none|short|long`). `applyAnthropicPayloadPolicyToParams()` then applies the same resolved cache-control object to cached system blocks and the trailing user block. With `cacheRetention: "long"`, tests assert `ttl: "1h"` on both the system prefix and the trailing user/tool_result content, not `1h` for stable prefix plus `5m` for conversation content.
  - file: [src/agents/anthropic-payload-policy.ts:52](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/anthropic-payload-policy.ts#L52)

- **Docs still expose Anthropic cache retention only, while OpenAI has explicit server compaction docs:** Anthropic docs document only `cacheRetention` values (`short`, `long`, `none`) and per-agent overrides. OpenAI docs separately document server-side compaction with `responsesServerCompaction` and `responsesCompactThreshold`, matching the issue's claimed precedent but not Anthropic parity.
