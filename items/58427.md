## Summary

Current `main` already contains the plugin-runtime changes that address the reported intermittent `api.runtime.subagent` gateway-request error: gateway plugin loads are now marked gateway-bindable, `api.runtime.subagent` late-binds to the gateway runtime instead of permanently capturing the unavailable fallback, and read-only plugin status loads are non-activating/non-cached to avoid command-registry drift. The same implementation is present in release `v2026.4.23`.

## Evidence

- **Late-binding subagent runtime on current main:** `createPluginRuntime()` now builds `subagent` through `createLateBindingSubagent(...)`, which resolves to the process-global gateway subagent when `allowGatewaySubagentBinding` is enabled instead of permanently holding the unavailable fallback.

  - file: [src/gateway/server-plugins.ts:487](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/gateway/server-plugins.ts#L487)
