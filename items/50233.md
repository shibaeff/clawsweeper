## Summary

Keep open. Current `main` does not implement the PR's core `gateway.trustedNetworks` config/end-to-end forwarding, and the model-selection test change does not match current behavior. Parts of the PR are already superseded on `main` by different code paths, but not enough to close this as implemented.

## Evidence

- **`gateway.trustedNetworks` is not present in the current config contract:** `GatewayConfig` still defines `trustedProxies` and then jumps to `allowRealIpFallback`; there is no `trustedNetworks` field on `main`. The strict Zod schema also accepts `trustedProxies` but not `trustedNetworks`.
