## Summary

Current `main` already covers the broad capability gap behind this issue: WhatsApp inbound audio is transcribed before agent handling, WhatsApp outbound can send audio as a voice note, and ElevenLabs is a bundled speech provider. But the specific feature request is not fully implemented as written: the repo does not have the referenced `saas/` layer, and the current user controls are generic `/tts` prefs, not WhatsApp-specific `voice response on|off|female|male|neutral` commands or phone-keyed Redis preferences. That makes auto-closing too aggressive.
