## Summary

Keep open. Current `main` still matches the report: when `cron.webhook` is unset, doctor only warns and leaves `notify` in place, but it will still print `Cron store normalized...` if any other legacy cron fields were rewritten in the same pass.

## Evidence

- **No-webhook branch leaves legacy notify unresolved:** `migrateLegacyNotifyFallback` pushes a warning and `continue`s when `cron.webhook` is unset; it does not delete `raw.notify` in that branch.
  - file: [src/commands/doctor-cron.ts:117](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/commands/doctor-cron.ts#L117)

  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Normalization message can still be emitted with unresolved notify:** `changed` is true if any other legacy normalization mutated the job (`normalized.mutated`), and that is enough to save the store and print `Cron store normalized...` even when `notifyMigration.warnings` is non-empty.
  - file: [src/commands/doctor-cron.ts:185](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/commands/doctor-cron.ts#L185)
