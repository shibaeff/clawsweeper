## Summary

Keep open. Current `main` still turns oversized or otherwise unresolved local avatars into `404` on `/avatar/{agentId}`, which matches the reporter’s confirmed reproduction path, and the docs still do not explain the local avatar size limit. I did not find strong evidence that the reported behavior is implemented/fixed on `main`.

## Evidence

- **Reporter confirmed the failing case was a large local file:** In the issue thread, the reporter said their local avatar was about 8 MB and started working only after reducing it below 2 MB, which directly matches the current size gate.
  - command: `GitHub context provided in prompt (issue comment by LuanBSPinheiro on 2026-03-07)`
