## Summary

Keep open. Current `main` does not implement Android `location.enabledMode: always` or background location fetches: the Android node UI only exposes `Off`/`While Using`, persisted `always` is downgraded to `whileUsing`, background `location.get` still hard-fails with `LOCATION_BACKGROUND_UNAVAILABLE`, and the manifest/service wiring for background location is absent.

## Evidence

- **Location mode enum excludes always:** `LocationMode` only defines `Off` and `WhileUsing`. Its parser explicitly converts stored `"always"` to `WhileUsing`, so current main treats `always` as unsupported rather than implementing it.
  - file: [apps/android/app/src/main/java/ai/openclaw/app/LocationMode.kt:3](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/apps/android/app/src/main/java/ai/openclaw/app/LocationMode.kt#L3)
  - command: `nl -ba apps/android/app/src/main/java/ai/openclaw/app/LocationMode.kt | sed -n '1,80p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Android settings expose only two location modes:** The Android settings UI renders radio rows for `Off` and `While Using` plus the separate `Precise Location` switch. There is no `Always` option in the current app settings surface.
  - file: [apps/android/app/src/main/java/ai/openclaw/app/ui/SettingsSheet.kt:1125](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/apps/android/app/src/main/java/ai/openclaw/app/ui/SettingsSheet.kt#L1125)
