## Summary

Current main does not implement a cron-style retention control for completed subagent `mode: "run"` sessions in general. The only cleanup automation is tied to `cleanup: "delete"`; default run-mode spawns still default to `cleanup: "keep"` and therefore remain in `sessions.json` until broader session maintenance prunes them.

## Evidence

- **`sessions_spawn` still defaults subagent cleanup to keep:** The tool normalizes omitted `cleanup` to `"keep"`, so a plain `mode: "run"` spawn does not opt into deletion or archival by default.
  - file: [src/agents/tools/sessions-spawn-tool.ts:192](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/tools/sessions-spawn-tool.ts#L192)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '188,206p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Run-mode spawns inherit the same default:** `spawnSubagentDirect()` sets `cleanup` to `"keep"` for run-mode spawns unless the caller explicitly passes `cleanup: "delete"`.
  - file: [src/agents/subagent-spawn.ts:584](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/subagent-spawn.ts#L584)

- **Archive timer is only assigned for delete-cleanup runs:** `registerSubagentRun()` leaves `archiveAtMs` undefined when `spawnMode === "session"` or `cleanup === "keep"`; only delete-cleanup runs get an auto-archive deadline from `archiveAfterMinutes`.
  - file: [src/agents/subagent-registry-run-manager.ts:302](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/subagent-registry-run-manager.ts#L302)
  - command: `nl -ba src/agents/subagent-registry-run-manager.ts | sed -n '292,310p'`

- **Tests explicitly cover keep-mode run sessions not being archived:** The archive e2e test asserts that a keep-mode run subagent has `archiveAtMs === undefined`, while a delete-mode run gets `archiveAtMs` and is swept later.
  - file: [src/agents/subagent-registry.archive.e2e.test.ts:86](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/subagent-registry.archive.e2e.test.ts#L86)

## Risks / Open Questions

- `docs/tools/subagents.md` says sub-agent sessions are automatically archived after `archiveAfterMinutes`, but the code and tests show that this does not apply to run-mode sessions using the default `cleanup: "keep"` path.
- Because the requested feature is specifically a global retention policy for completed run-mode subagents, closing as implemented would hide a still-open behavioral gap in current main.
