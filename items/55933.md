## Summary

Keep open. Current `main` does not implement the PR's cache-mount change, and the underlying condition the PR describes still exists in the checked-out code: the root Dockerfile still uses `node-linker=hoisted` and still runs `pnpm prune --prod` in the `runtime-assets` stage without a pnpm store cache mount. I did not find a superseding implementation on `main` that would justify closing this PR as already handled.
