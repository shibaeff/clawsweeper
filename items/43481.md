## Summary

Keep open. Current `main` strips `opts.images` and `imageOrder` on fallback retries, but the original prompt body is still preserved on retry and the embedded runner re-detects and reloads prompt image references, including offloaded `[media attached: ...]` refs. I also found no provider-level trust-policy config for agent fallback attachments/images, so the requested trust-boundary control is not implemented on `main`.
