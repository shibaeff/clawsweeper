## Summary

Keep open. Current `main` still removes only the cron job definition; it does not clean `sessions.json` or `cron/runs/<jobId>.jsonl`. Gateway session listing still hides only `:run:` alias keys, while the cron reaper explicitly keeps the base `agent:<agentId>:cron:<jobId>` session row, so the reported ghost-session behavior remains plausible on `main`. The PR also appears to have incorporated the key review fixes on its branch, so it still contains useful work rather than being obviously obsolete.
