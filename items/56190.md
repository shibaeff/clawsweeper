## Summary

Keep open. Current `main` still only supports plain `${VAR}` and `$${VAR}` config env tokens; the proposed `${VAR:-default}` fallback syntax is not implemented in code, restore logic, or tests, so this PR is still unlanded feature work rather than something already fixed or obsolete.

## Evidence

- **Main substitution code has no `:-default` support:** `parseEnvTokenAt` only recognizes `$${VAR}` and `${VAR}` by slicing up to the next `}` and validating the whole inner text as an env var name. `substituteString` then either substitutes from `env`, preserves the original `${VAR}` via `onMissing`, or throws `MissingEnvVarError`. There is no branch for `${VAR:-default}` on current `main`.
