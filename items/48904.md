## Summary

Keep open. Current `main` does not implement config-driven `xhigh` opt-in for custom models: the compat type/schema still omit `supportsXHighThinking`, the thinking helpers still derive support from provider/runtime profile only, and the native `/think` choices plus ACP session presentation still use config-unaware `listThinkingLevels(provider, model)` calls. The PR discussion also contains a concrete repro from a real custom provider on `main`, so this remains a live gap rather than a stale or already-shipped change.
