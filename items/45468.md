## Summary

Current `main` no longer has the reported failure mode. The gateway tick watchdog now refuses to close the socket while there are pending requests, pending requests have explicit per-request timeouts, and the close handler still flushes any remaining pending requests. The same behavior is already present in the shipped `v2026.4.22` release tag.

## Evidence

- **Close handler flushes pending requests:** `ws.on("close")` calls `flushPendingErrors(...)`, rejecting and clearing all pending requests on socket close.
  - file: [src/gateway/client.ts:332](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/gateway/client.ts#L332)
