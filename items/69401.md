## Summary

Keep open. Current `main` still treats inbound channel metadata as untrusted blocks, but the actual inbound message body is appended to the prompt as ordinary user text without a dedicated untrusted wrapper. Dreaming/session ingestion also still consumes redacted session transcripts, so this request is neither implemented on `main` nor obsolete.

## Evidence

  - file: [src/auto-reply/reply/inbound-meta.ts:135](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/auto-reply/reply/inbound-meta.ts#L135)
