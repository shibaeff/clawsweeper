## Summary

Current `main` still lacks the Control UI history decoding/rendering that this PR adds, so this is not an `implemented_on_main` cleanup candidate. The PR also has follow-up evidence in its discussion that the initial review concerns were addressed, and the related PR #70525 only shows overlap, not that this work is already shipped.

## Evidence

- **Current UI still ignores image attachments in history:** `ChatMessageBody.inlineAttachments` only includes content types `file` and `attachment`, and `AttachmentRow` renders a paperclip row with filename text only. There is no image preview/rendering path on current `main` for `image` history content.
