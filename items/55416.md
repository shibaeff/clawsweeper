## Summary

Keep open. Current `main` still only converts native Ollama `message.tool_calls` into tool executions, so the Qwen inline-call gap is not implemented elsewhere. This PR also is not a safe cleanup close: the review discussion already identified unresolved P1/P2 risks, and the active Ollama stream code has since moved into `extensions/ollama`, so the patch would need a fresh port/re-review rather than auto-closure.
