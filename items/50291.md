## Summary

Keep open. Current `main` partially addressed the tracing request by adding `trace` to agent/tool hook contexts and passing `runId` in agent hook context, but the issue's core correlation gaps still exist: agent contexts still lack `messageId`/`senderId`, `before_agent_start` and `agent_end` events still do not carry `runId`, and outbound message hooks still do not expose `sessionKey`/`runId`/`messageId` for cross-hook correlation.
