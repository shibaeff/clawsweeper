## Summary

Keep open. Current `main` does not implement the gateway-startup cache bypass described by PR #68567, and I found no equivalent shipped fix on `main`. The PR discussion identified an initial no-op bug, but the later PR patchset shown in the provided timeline/file list appears to add the missing `cache` plumbing that still is absent from `main`, so this remains live maintainer work rather than a close candidate.

## Evidence

- **Gateway bootstrap on main has no cache option:** `GatewayPluginBootstrapParams` in `src/gateway/server-plugin-bootstrap.ts` has no `cache` field, and `prepareGatewayPluginLoad()` calls `loadGatewayPlugins()` without any cache override. That means gateway startup on current `main` still uses the default loader behavior.
  - file: [src/gateway/server-plugin-bootstrap.ts:25](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/gateway/server-plugin-bootstrap.ts#L25)
  - command: `nl -ba src/gateway/server-plugin-bootstrap.ts | sed -n '1,180p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Gateway plugin wrapper on main still drops cache control:** `loadGatewayPlugins()` in `src/gateway/server-plugins.ts` does not declare a `cache` parameter and its `loadOpenClawPlugins({...})` call omits `cache`, so there is no way for gateway startup/reload on current `main` to force a fresh plugin registry load through this path.
  - file: [src/gateway/server-plugins.ts:434](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/gateway/server-plugins.ts#L434)
  - command: `nl -ba src/gateway/server-plugins.ts | sed -n '400,520p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Core loader only bypasses reuse when cache is explicitly false:** `PluginLoadOptions` exposes `cache?: boolean`, and `loadOpenClawPlugins()` computes `const cacheEnabled = options.cache !== false;`. Without the missing gateway plumbing above, current `main` continues to reuse cached plugin state on the startup path.
  - file: [src/plugins/loader.ts:145](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/plugins/loader.ts#L145)
  - command: `nl -ba src/plugins/loader.ts | sed -n '140,220p' && nl -ba src/plugins/loader.ts | sed -n '1940,2010p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **PR discussion shows branch work still not present on main:** The provided review comments correctly flagged the original PR as a no-op because `cache: false` was not threaded through `loadGatewayPlugins()`. The later PR file list in the provided GitHub context shows follow-up edits adding `cache?: boolean` in both `src/gateway/server-plugin-bootstrap.ts` and `src/gateway/server-plugins.ts`, which matches the gap still visible on current `main`.
