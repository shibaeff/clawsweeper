## Summary

Current `main` already implements the underlying fix in a different UI flow: tool messages are collapsed behind an inline disclosure and expand in place, with sidebar opening only from an explicit button inside expanded content. The PR's target (`renderToolCardSidebar`) is not part of the active chat render path on `main`, so this PR is obsolete.
