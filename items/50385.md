## Summary

Current main’s exec and system.run paths preserve internal argument spacing. I could not find a present-day code path that would collapse `hot 10` into `hot10`, and the relevant files are unchanged from the latest release to HEAD.

## Evidence

- **Exec tool contract is a single literal command string:** The exec tool still accepts one `command: string`, so OpenClaw is not rebuilding shell argv from separate model-side argument fields in this path.
  - file: [src/agents/bash-tools.schemas.ts:3](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/bash-tools.schemas.ts#L3)
- **Exec path only trims outer whitespace:** Current exec handling does `const rawCommand = command.trim()` and then analyzes/splits that string; it does not collapse internal spaces between arguments.
  - file: [src/agents/bash-tools.exec.ts:1284](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/bash-tools.exec.ts#L1284)
- **Node-host wrapper forwards the exact command text:** Node execution wraps the original command string directly as `/bin/sh -lc <command>` on POSIX or `cmd.exe /d /s /c <command>` on Windows, preserving interior spaces.
  - file: [src/infra/node-shell.ts:3](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/infra/node-shell.ts#L3)
- **system.run renders and validates argv with preserved spacing:** `formatExecCommand()` joins argv with spaces, and `validateSystemRunCommandConsistency()` rejects mismatches between `rawCommand` and the rendered command text instead of silently collapsing boundaries.
  - file: [src/infra/system-run-command.ts:41](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/infra/system-run-command.ts#L41)
- **No drift since latest release in the relevant exec whitespace paths:** A diff from `v2026.4.22` to `HEAD` for `src/agents/bash-tools.exec.ts`, `src/infra/node-shell.ts`, and `src/infra/system-run-command.ts` is empty.
  - command: `git diff --unified=0 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5..HEAD -- src/agents/bash-tools.exec.ts src/infra/node-shell.ts src/infra/system-run-command.ts`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
