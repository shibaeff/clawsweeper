## Summary

Current main already clears billing-disable state on successful use and on expired disable windows, documents `auth-state.json`/`models status` instead of the reported `provider_status.json` path, and has regression coverage for billing-vs-transient classification. The reported stale duplicate-alert path appears to be obsolete on main.
