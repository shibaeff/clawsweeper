## Summary

Current main already implements the requested WhatsApp reconnect replay protection: inbound messages are deduplicated by account/chat/message id before enqueue, stale reconnect-history append events are skipped, and regression tests cover both behaviors. The changelog records this replay-dedupe sweep as shipped in v2026.4.12.
