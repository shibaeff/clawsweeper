## Summary

Keep open. Current `main` still has the same-model-only thinking preservation behavior described in the PR, so this is not already implemented. But the proposed unconditional cross-model preservation is not a safe auto-close candidate either: the OpenAI Responses transport currently `JSON.parse`s every replayed `thinkingSignature`, so preserving Anthropic-native thinking blocks across provider boundaries would likely break OpenAI-bound replay unless the fix is gated by target-provider compatibility and expanded tests.

## Evidence

- **Current main still preserves Anthropic thinking only on same-model replay:** `transformTransportMessages` keeps redacted thinking blocks only when `isSameModel`, keeps signed thinking blocks only when `isSameModel && block.thinkingSignature`, and otherwise downgrades non-empty thinking to text. That means the PR's claimed gap is still present on `main` and has not been implemented already.
