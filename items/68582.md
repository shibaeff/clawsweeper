## Summary

Current `main` still matches the reported WebChat/CLI race. The Control UI reloads `chat.history` after any final chat event, `server-chat` finalization only broadcasts the buffered assistant reply without appending it to the session transcript, and `chat.history` only reads persisted transcript data (with a Claude-CLI-only import path). I did not find a current-main persistence path for `openai-codex` or `google-gemini-cli` assistant replies before that reload, so this should stay open.
