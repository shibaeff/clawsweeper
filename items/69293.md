## Summary

Keep open: current `main` still omits Commander positional arguments from generated zsh leaf-command completions, so `openclaw wiki ingest`'s required path argument is not modeled for filename completion.

## Evidence

- **Leaf zsh completion only emits options:** `generateZshArgs()` serializes `cmd.options` only, and leaf-command completion functions call `_arguments -C` with just that output. There is no handling of Commander positional arguments or `_files` for leaf commands.
  - file: [src/cli/completion-cli.ts:187](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cli/completion-cli.ts#L187)
  - command: `nl -ba src/cli/completion-cli.ts | sed -n '187,205p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Leaf command handler has no positional fallback:** For commands without subcommands, the generated zsh function is just `_arguments -C \
    ${generateZshArgs(cmd)}`. That leaves `wiki ingest` with option completion only.
  - file: [src/cli/completion-cli.ts:257](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cli/completion-cli.ts#L257)
  - command: `nl -ba src/cli/completion-cli.ts | sed -n '233,262p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **`wiki ingest` requires a positional path:** The memory-wiki CLI registers `wiki ingest` with `.argument("<path>", "Local file path to ingest")`, so the command surface does have a required positional file input that zsh completion should expose.
  - file: [extensions/memory-wiki/src/cli.ts:715](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/extensions/memory-wiki/src/cli.ts#L715)
  - command: `nl -ba extensions/memory-wiki/src/cli.ts | sed -n '708,724p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Coverage does not guard positional zsh completion:** The current completion test only asserts nested subcommands and options for zsh generation; there is no test for positional-argument completion such as a file path on `wiki ingest`.
  - file: [src/cli/completion-cli.test.ts:25](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cli/completion-cli.test.ts#L25)

  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
