## Summary

Keep open: current `main` still runs POSIX exec shells with plain `-c`, so the reported startup-file env override path is not implemented on `main`, and the proposed shell-arg regression coverage is also absent there.

## Evidence

- **Current main still uses plain `-c` for POSIX shells:** `getShellConfig()` returns `['-c']` for fish->bash fallback, fish->sh fallback, direct `envShell`, and bare `sh`/`bash` fallback. There is no shell-specific suppression for zsh/bash/fish on `main`.
  - file: [src/agents/shell-utils.ts:71](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/agents/shell-utils.ts#L71)

  - sha: [07cf1dd65c93](https://github.com/openclaw/openclaw/commit/07cf1dd65c937c75cf48a1645ea580404f5056b4)
- **The affected helper is still on the live exec path:** Host exec builds `childArgv` from `getShellConfig()` and passes the inherited runtime env into that spawned shell, so shell startup files would still run before the command. PTY mode also reuses the same helper.
  - file: [src/agents/bash-tools.exec-runtime.ts:669](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/agents/bash-tools.exec-runtime.ts#L669)
