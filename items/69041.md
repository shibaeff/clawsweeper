## Summary

Keep open. The report hits official OpenClaw web/install surfaces, current main does not contain a Jio/India-specific fix or workaround, and my non-Jio checks only prove the endpoints are reachable from another network, not that the India/Jio DNS/reset problem is obsolete.

## Evidence

- **Installer scripts are an official product surface:** The installer docs explicitly say OpenClaw ships installer scripts served from `openclaw.ai`, so failure to resolve `openclaw.ai/install.sh` affects a supported install path rather than an unrelated external mirror.
  - file: [docs/install/installer.md:10](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/docs/install/installer.md#L10)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Quick-start still depends on openclaw.ai/install.sh:** The main install docs on current `main` still tell users to run `curl -fsSL https://openclaw.ai/install.sh | bash` as the default macOS/Linux/WSL install flow.
  - file: [docs/install/index.md:23](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/docs/install/index.md#L23)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Repo docs already acknowledge ISP/security-network blocking as a real docs-access failure mode:** The first-run FAQ documents that some Comcast/Xfinity connections incorrectly block `docs.openclaw.ai` and points users to a workaround. That makes this Jio/India access report coherent, not incoherent or obviously user error. There is no adjacent Jio/India-specific remediation on current `main`.
  - file: [docs/help/faq-first-run.md:245](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/docs/help/faq-first-run.md#L245)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Current endpoints are reachable from the review environment, but that does not reproduce the reported network:** From this review environment, `curl -I --max-time 15 https://openclaw.ai/install.sh` returned `HTTP/2 200`, and a resolver lookup returned addresses for both `openclaw.ai` and `docs.openclaw.ai`. That narrows the issue to region/ISP/CDN behavior rather than a dead hostname, but it does not disprove the user’s India/Jio-specific report.
