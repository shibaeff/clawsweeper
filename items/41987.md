## Summary

Current `main` still ignores `agents.defaults.imageMaxDimensionPx` for browser screenshots, and this PR is written against the old `src/agents/tools` layout rather than the current `extensions/browser` plugin architecture. The bug is real, but the patch needs to be ported before it can land.

## Evidence

- **Current screenshot path omits image sanitization:** The browser plugin screenshot branch returns `imageResultFromFile({ label, path, details })` without any `imageSanitization` option, so configured `imageMaxDimensionPx` is not applied on current `main`.
  - file: [extensions/browser/src/browser-tool.ts:755](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/extensions/browser/src/browser-tool.ts#L755)
  - command: `nl -ba extensions/browser/src/browser-tool.ts | sed -n '716,760p'`
