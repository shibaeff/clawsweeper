## Summary

Current `main` already routes Telegram polling through an explicit transport/fetch layer that honors proxy settings, including `HTTP_PROXY`/`HTTPS_PROXY`, and has regression coverage for env-proxy polling behavior. The reported silent polling-init failure is no longer consistent with the code on `main`.

## Evidence
