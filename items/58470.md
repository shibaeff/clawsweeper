## Summary

Current `main` still persists session lifecycle `status` into `sessions.json`, but gateway startup only cleans stale session lock files and runs orphan-key migration. I did not find a startup reconciliation path that clears lockless `status: "running"` entries, so there is not strong evidence to close this issue.
