## Summary

Keep open. Current `main` still contains the broken export HTML placeholders in `src/auto-reply/reply/export-html/template.html`, while the export code still does exact `"{{...}}"` string replacement. The build copy step ships that template verbatim, and the latest release tag `v2026.4.22` contains the same broken template, so the issue is not implemented on main and not obsolete.

## Evidence

- **Broken template on current main:** `src/auto-reply/reply/export-html/template.html` no longer contains literal `{{MARKED_JS}}`, `{{HIGHLIGHT_JS}}`, and `{{JS}}` placeholders. Instead it has formatted blocks like `{ { MARKED_JS; } }` inside `<script>` tags, which will not match the export replacement code.
