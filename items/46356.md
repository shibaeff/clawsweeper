## Summary

Current `main` has adjacent fixes for preserving external session routes and for Discord/WhatsApp proxy handling, but it does not already implement this PR's core webchat bidirectional mirroring behavior or the proposed `channels.whatsapp.proxy` config surface. The PR discussion also still carries unresolved correctness concerns in the new mirror/proxy code paths, so there is not strong evidence to close it conservatively.

## Evidence

That is the opposite of this PR's headline behavior ('Replies sent from webchat/control UI now mirror back to routed external chats').
