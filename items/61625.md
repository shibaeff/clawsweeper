## Summary

Keep open. Current `main` does not implement `voiceCall.agent`, and the PR as described is still incomplete: voice-call runs would change path/session resolution but still omit `agentId` when calling `runEmbeddedPiAgent`, leaving agent-scoped fallback/execution-contract behavior on the default agent.
