## Risks / Open Questions

- The implementation is narrower than the issue's example `dangerouslyAllowPrivateNetwork` proposal: it only opts into the RFC2544 fake-IP range instead of broad private-network access. That matches the reported TUN/fake-IP problem and preserves the default SSRF posture.

## Close Comment

Closing as implemented on `main` after Codex review. OpenClaw already exposes `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` for fake-IP/TUN proxy environments: see `src/config/types.tools.ts:561`, runtime wiring in `src/agents/tools/web-fetch.ts:390` and `src/agents/tools/web-fetch.ts:651`, and regression coverage in `src/agents/tools/web-fetch.ssrf.test.ts:159`. The changelog also records this as shipped in `v2026.4.10` at `CHANGELOG.md:931`. This is a narrower opt-in than broad `dangerouslyAllowPrivateNetwork`, but it directly addresses the reported Clash/Mihomo TUN fake-IP case without weakening the default SSRF protections.
