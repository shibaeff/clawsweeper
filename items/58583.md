## Summary

Current `main` does not implement size-based log rotation. The gateway logger still hard-caps a single file at `maxFileBytes`, emits one warning, and suppresses all further writes; docs and tests both describe and lock in that behavior, so issue #58583 remains a valid open cleanup item.

## Evidence

- **Logger still suppresses at cap:** `buildLogger()` checks `nextBytes > settings.maxFileBytes`, writes a single warning, then `return`s without rotating or reopening another file.
