## Summary

Current `main` does not implement the requested tool-policy audit logging, and the PR discussion surfaced a plausible correctness problem in the proposed sandbox logging path. This should stay open for maintainer review rather than be auto-closed.

## Evidence

- **Current main lacks pipeline audit logging:** `applyToolPolicyPipeline` still expands the policy and filters tools without any INFO/audit log or removed-tool attribution after each step.
  - file: [src/agents/tool-policy-pipeline.ts:156](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/tool-policy-pipeline.ts#L156)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current main sandbox path formats user guidance but does not log provenance:** `formatSandboxToolPolicyBlockedMessage` builds the blocked-tool message and fix-it text, but there is no structured audit log for which rule/source blocked the tool.
  - file: [src/agents/sandbox/runtime-status.ts:128](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/sandbox/runtime-status.ts#L128)

  - file: [docs/cli/sandbox.md:28](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/cli/sandbox.md#L28)
