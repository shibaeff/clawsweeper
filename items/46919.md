## Summary

Current `main` has partial related knobs (`gateway.auth.mode: "none"`, Tailscale Serve identity auth, and `gateway.controlUi.dangerouslyDisableDeviceAuth`), but it does not implement the requested direct Tailnet-IP/no-token/no-HTTPS config path. Non-loopback binds still require shared auth or trusted-proxy, and the docs still describe Tailnet bind as token-authenticated.

## Evidence

- **Non-loopback gateway binds still require auth:** Runtime config rejects any non-loopback bind without shared token/password auth or trusted-proxy auth, which blocks the requested `bind: "tailnet"` + tokenless config path.
  - file: [src/gateway/server-runtime-config.ts:141](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/gateway/server-runtime-config.ts#L141)

- **Docs still require token for direct Tailnet bind:** The Tailnet docs explicitly show direct `bind: "tailnet"` using `auth: { mode: "token", token: "your-token" }`, while tokenless auth is only documented for Tailscale Serve on loopback.
  - file: [docs/gateway/tailscale.md:63](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/gateway/tailscale.md#L63)

- **`auth.mode: "none"` is limited to trusted local loopback:** The configuration reference says non-loopback binds require gateway auth and describes `gateway.auth.mode: "none"` as loopback-only/private-ingress, not a general Tailnet/IP setting.
  - file: [docs/gateway/configuration-reference.md:306](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/gateway/configuration-reference.md#L306)
