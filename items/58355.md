## Summary

Keep open. Current `main` still enforces allowlist-based session model switching, the relevant code paths are unchanged from `v2026.4.23`, and the report does not include enough config/repro detail to prove this was only expected allowlist behavior rather than a real Ollama UX bug.

## Risks / Open Questions

- The report omits the actual reproduction path and config state. If the user typed a newly pulled Ollama ref while `agents.defaults.models` was already populated, the rejection is expected; if the UI picker surfaced that ref anyway, that would still be a real bug.
- `docs/providers/ollama.md:153-181` says newly pulled Ollama models are automatically discovered and available to use, which is in tension with the allowlist-only switching behavior documented in `docs/concepts/models.md`. That mismatch makes the report plausible enough that closing would be premature.
