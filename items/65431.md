## Summary

Current `main` contains a partial MiniMax mitigation, but not enough high-confidence evidence to auto-close this issue. The image tool now auto-pairs a MiniMax primary model to `MiniMax-VL-01`, and MiniMax onboarding/tests now treat `MiniMax-M2.7` as text-only for image-tool purposes. But the exact reported path still looks ambiguous: an explicit `agents.defaults.imageModel` override bypasses that auto-pairing, the runtime still throws `Model does not support images` when the chosen registry model lacks image input, and current docs contradict each other about whether explicit `models.providers.minimax` entries should make M2.7 image-capable. With that mixed contract, conservative cleanup should keep the issue open.
