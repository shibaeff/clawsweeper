## Summary

Keep open. Current `main` adds per-run timeout mitigation, but it does not implement the requested consecutive-timeout circuit breaker across later heartbeats/messages. There is no `maxConsecutiveTimeouts` config, no persisted degraded-session breaker state, and the heartbeat scheduler continues to queue future interval runs unless heartbeats are manually disabled.

## Evidence

- **No breaker config on main:** Searches for the proposed config name and related wording returned no matches in `src/` or `docs/`, so there is no shipped `agents.defaults.maxConsecutiveTimeouts`-style setting.
  - command: `rg -n "maxConsecutiveTimeouts|consecutive timeout" src docs packages`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Mitigation is per-run only:** `run.ts` hard-caps timeout-triggered compaction attempts at 2 inside a single embedded run, then falls through to failover/error handling. This is not a cross-run/session breaker.
  - file: [src/agents/pi-embedded-runner/run.ts:544](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-embedded-runner/run.ts#L544)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Retry exhaustion only returns an error for that run:** When retry limits exhaust, OpenClaw returns an error payload with `error.kind = "retry_limit"` and `livenessState = "blocked"`, but this result is local to the run; there is no persisted session flag that disables future retries.
  - file: [src/agents/pi-embedded-runner/run/retry-limit.ts:27](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/pi-embedded-runner/run/retry-limit.ts#L27)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Heartbeat scheduling continues automatically:** The heartbeat runner keeps scheduling interval wakes with `requestHeartbeatNow({ reason: "interval" ... })` and only skips runs when heartbeats are globally/manual disabled or the interval is off. There is no automatic disable-on-consecutive-timeouts path.
  - file: [src/infra/heartbeat-runner.ts:1390](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/infra/heartbeat-runner.ts#L1390)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Docs expose workaround-style mitigation, not a breaker:** Docs describe `heartbeat.isolatedSession` as an optional way to avoid sending full conversation history each heartbeat and reduce token cost, which helps with thrash, but they do not describe any consecutive-timeout breaker, degraded session state, or notification hook for this case.
  - file: [docs/gateway/config-agents.md:507](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/gateway/config-agents.md#L507)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
