## Summary

Current `main` covers most of the symptom cluster through different fixes, but this PR is not implemented one-for-one: the chat controller still handles `state: "error"` by setting `lastError` only, and gateway WS connections still fall through to the generic handshake-timeout path instead of fast-rejecting legacy query-auth URLs. Because substantive PR behavior is still absent on `main`, this is not a strong `implemented_on_main` close.
