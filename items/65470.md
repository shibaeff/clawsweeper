## Summary

Current `main` no longer funnels all `sessions_send` work through one shared `nested` lane. `sessions_send` and its A2A follow-up paths now scope nested work to `nested:<sessionKey>`, so the queue's default `maxConcurrent: 1` applies per target session rather than globally. The same behavior is already present in the shipped `v2026.4.23` release.

## Evidence
