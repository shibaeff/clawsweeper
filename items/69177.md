## Summary

Current `main` does not contain the PR's one-line change, but the proposed fix is not strongly proven to change behavior: `session_status` already passes the full `sessionEntry`, and downstream status rendering already reads `sessionEntry.contextTokens` when computing the context line. The PR may still target a narrower edge case, but the available evidence is not strong enough to auto-close or to claim the bug is already fixed.
