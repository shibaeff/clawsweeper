## Summary

Current `main` does not already implement this PR, and the PR’s proposed precedence change is not safe to auto-close around: `deriveSessionTitle()` still ignores `entry.label`/`origin.label` on `main`, so the labeled-session heartbeat bug remains plausible, but the PR review correctly points out that putting `origin.label` ahead of `firstUserMessage` would regress unlabeled inbound session titles because the TUI prefers `derivedTitle` over `displayName`. This needs maintainer revision/judgment, not conservative cleanup closure.

## Evidence

- **Current main still derives titles from transcript before any session/origin label:** `deriveSessionTitle()` on `main` checks `displayName`, then `subject`, then `firstUserMessage`, then `sessionId`; it does not consider `entry.label` or `entry.origin?.label`. That means the behavior described in the PR is not already implemented on `main`.
