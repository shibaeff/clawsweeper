## Summary

Current `main` already covers the reported update failure mode: package self-updates resume post-update work from the freshly installed entrypoint, `openclaw update` restarts the gateway by default unless `--no-restart` is used, and the macOS restart helper now retries through `launchctl enable/bootstrap` before a second `kickstart` while preserving failures in the restart log. The changelog shows the stale-hashed-chunk respawn fix shipped in `v2026.4.12` and the macOS update-restart helper hardening shipped in `v2026.4.18`, so this issue is implemented on current `main` and in released builds.

## Evidence
