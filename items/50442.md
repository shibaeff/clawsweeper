## Summary

Keep open. Current `main` still writes the backup archive to `outputPath.<uuid>.tmp` in the destination directory and only removes it from normal in-process cleanup paths. There is no general CLI `SIGTERM`/timeout shutdown handling around `openclaw backup create`, and existing tests only cover ordinary failure/race cases, not external termination. The latest release tag available locally (`v2026.4.22`) shows the same backup-create implementation.

## Evidence

## Risks / Open Questions

- External timeout or signal termination can still strand large `*.tmp` archives in the output directory and exhaust disk space.
- Because the failure mode is not covered by tests, closing this as fixed would be unsafe.
