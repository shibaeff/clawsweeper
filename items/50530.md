## Summary

Current `main` still preserves single newlines when rendering Telegram outbound text, and there is no Telegram channel setting implementing the proposed collapse-before-send behavior. The issue remains actionable rather than already fixed or obsolete.

## Evidence

- **Shared Markdown IR keeps soft line breaks:** The Markdown renderer used by Telegram appends `"\n"` for both `softbreak` and `hardbreak`, so wrapped prose/list lines are preserved instead of collapsed to spaces.
  - file: [src/markdown/ir.ts:634](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/markdown/ir.ts#L634)
  - command: `nl -ba src/markdown/ir.ts | sed -n '628,640p'`
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Telegram formatter applies Markdown IR directly:** `markdownToTelegramHtml()` calls `markdownToIR()` and immediately renders the result; there is no normalization pass that collapses mid-paragraph newlines before producing Telegram HTML.
  - file: [extensions/telegram/src/format.ts:75](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/telegram/src/format.ts#L75)
  - command: `nl -ba extensions/telegram/src/format.ts | sed -n '65,115p'`
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Telegram send path uses that formatter for outbound text:** `sendTelegramText()` formats outbound Markdown with `markdownToTelegramHtml(text)`, and the outbound adapter uses `markdownToTelegramHtmlChunks` for chunked delivery, so current sends inherit the preserved single-newline behavior.
  - file: [extensions/telegram/src/bot/delivery.send.ts:125](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/telegram/src/bot/delivery.send.ts#L125)
  - command: `nl -ba extensions/telegram/src/bot/delivery.send.ts | sed -n '100,150p'`
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Existing tests assert newline preservation:** Telegram formatting tests currently expect multiline blockquotes to render with an embedded newline (`<blockquote>first\nsecond</blockquote>`), which matches the present preserve-newlines behavior rather than a collapse pass.
  - file: [extensions/telegram/src/format.test.ts:43](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/telegram/src/format.test.ts#L43)
  - command: `nl -ba extensions/telegram/src/format.test.ts | sed -n '1,70p'`
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **No Telegram collapse-newlines option exists:** Searches for `collapseNewlines` / `collapse newlines` found no Telegram feature toggle. The documented Telegram text controls cover chunking (`chunkMode`) and draft break preference, not collapsing mid-paragraph newlines before send.
  - file: [docs/channels/telegram.md:692](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/docs/channels/telegram.md#L692)
