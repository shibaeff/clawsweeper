## Summary

Keep open. Current `main` still does not implement PR #45270's stale-token fix: `buildStatusMessage` in `src/status/status-message.ts` synthesizes `totalTokens` from `inputTokens + outputTokens` without checking `totalTokensFresh`, then renders that value in the `Context:` line. The repo already has a helper that treats stale totals as unknown, but this status-message path is not using it.
