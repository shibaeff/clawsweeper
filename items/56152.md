## Summary

Keep open. Current `main` still uses the pre-PR `realpath`/fallback check, so the reported `/tmp` vs `/private/tmp` false negative is not implemented there, and the proposed PR has unresolved review comments identifying a likely symlink-escape security regression if merged as written.

## Evidence

- **Current main still uses single canonical-path comparison:** `assertLocalMediaAllowed` on `main` resolves the media path with `fs.realpath(mediaPath)` and falls back to `path.resolve(mediaPath)`, then compares that single value against each root's single resolved value. There is no variant-matching logic from PR #56152 on current `main`, so the proposed behavior is not implemented yet.
