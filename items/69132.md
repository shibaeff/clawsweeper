## Summary

Keep open. Current `main` still routes Ollama web search to `/api/experimental/web_search`, the corresponding test still asserts that exact endpoint, the docs still tell users the provider uses that experimental path, and the latest release `v2026.4.23` shipped the same implementation. The issue discussion also already links two still-open fix PRs (`#69178`, `#69222`), so this is not implemented on `main` and is not a conservative close.
