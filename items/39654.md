## Summary

Current `main` does not contain the PR's flat-array coercion. `commands.allowFrom` still reuses the record-only `ElevatedAllowFromSchema`, the public type is still `Record<string, Array<string | number>>`, and runtime authorization still expects provider-keyed or `"*"` entries. The docs/examples on `main` now consistently show the record form, so the documentation-mismatch part of the PR rationale appears fixed separately, but the requested config compatibility change itself is not implemented.
