## Summary

Keep open. Current `main` does not contain this PR's core streaming fix: the shared reasoning-tag sanitizer already handles attributed `<final>` tags, but the live streaming state machine in `src/agents/pi-embedded-subscribe.ts` still uses the older plain-tag regex, so this is not implemented on `main`. Some PR hunks look stale because message-end sanitization has since moved through shared helpers, so the patch likely needs rebasing/narrowing rather than closure.
