## Summary

Keep open. Current `main` still lacks the supervisor/exec changes this PR proposes, and I did not find strong evidence that the reported background-output race is already fixed or obsolete.

## Evidence

- **Supervisor path on main still matches the pre-fix logic:** `createProcessSupervisor()` still resolves `waitPromise` immediately after `adapter.wait()`, then disposes/finalizes without any extra event-loop drain or finalize-before-yield handling. That is the exact area this PR changes.
  - file: [src/process/supervisor/supervisor.ts:201](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/process/supervisor/supervisor.ts#L201)
  - command: `nl -ba src/process/supervisor/supervisor.ts | sed -n '30,250p'`
  - sha: [07cf1dd65c93](https://github.com/openclaw/openclaw/commit/07cf1dd65c937c75cf48a1645ea580404f5056b4)
- **Related cancel/finalize race guards are also absent on main:** `cancel()` still writes `registry.updateState(...terminationReason)` directly, and `setForcedReason()` still only guards on `forcedReason`, not `settled`. The PR discussion shows these were later review fixes, but they are not present on current `main`.
  - file: [src/process/supervisor/supervisor.ts:45](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/process/supervisor/supervisor.ts#L45)
  - command: `nl -ba src/process/supervisor/supervisor.ts | sed -n '30,250p'`
  - sha: [07cf1dd65c93](https://github.com/openclaw/openclaw/commit/07cf1dd65c937c75cf48a1645ea580404f5056b4)
- **Child adapter still settles on `close` with no post-close drain on main:** The child adapter records `stdout`/`stderr` drain state for Windows, but on the common path it settles as soon as the child `close` event fires and `wait()` returns that result directly. There is no equivalent of the PR's proposed drain-before-return behavior.
  - file: [src/process/supervisor/adapters/child.ts:247](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/process/supervisor/adapters/child.ts#L247)
  - command: `nl -ba src/process/supervisor/adapters/child.ts | sed -n '210,275p'`
  - sha: [07cf1dd65c93](https://github.com/openclaw/openclaw/commit/07cf1dd65c937c75cf48a1645ea580404f5056b4)
- **Exec runtime still snapshots finished output at `managedRun.wait()` resolution:** When the managed run resolves, exec immediately builds the outcome from `session.aggregated.trim()` and then calls `markExited()`. If output callbacks can arrive after `wait()`, finished-session aggregation would still be vulnerable on current `main`.
  - file: [src/agents/bash-tools.exec-runtime.ts:770](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/agents/bash-tools.exec-runtime.ts#L770)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '730,810p'`
