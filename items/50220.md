## Summary

Keep open. Current `main` still contains the same strict `resolveCanonicalApprovalCwdSync` logic, and the latest release `v2026.4.22` carries the same code path. I found no macOS `/Users` firmlink or launchd-specific normalization/fallback, and the only adjacent shipped fix was for omitted remote-node `workdir`, not explicit `/Users/...` requests.
