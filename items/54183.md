## Summary

Current `main` has the newer write-ahead outbound queue and WhatsApp reconnect-drain path, but it does not implement this PR's separate configurable WhatsApp outbound retry surface. The WhatsApp config type on `main` still has no `retry` field, and the current outbound send path calls the send functions directly without a retry wrapper. That matches the 2026-04-25 maintainer timeline note that this PR is 'adjacent rather than identical' to the shipped queue work and still needs a fresh review rather than cleanup-closing.
