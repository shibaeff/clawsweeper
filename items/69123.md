## Summary

Current `main` still rejects partial `agents.defaults.cliBackends.<backendId>` overrides at config-schema level, even though backend resolution already merges overrides with a base backend and revalidates `command` after merge. This PR addresses a real gap, but the submitted patch appears incomplete because the generated config schema artifact still requires `command`, so the item should stay open for follow-up rather than be auto-closed.
