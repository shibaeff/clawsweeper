## Summary

Keep open. Current `main` still has the observability gap from the report: the preemptive check logs only truncation/compaction branches, not the `route: "fits"` path. `cache-ttl` pruning still removes tool-result content before prompt submission, so the reported masking theory is still plausible. There is a recent unreleased partial change for context-engine windowing (`unwindowedMessages`), but it does not close out the logging request or clearly prove the large-context/cache-ttl failure mode is fixed.
