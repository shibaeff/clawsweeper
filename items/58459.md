## Summary

Current `main` already handles isolated cron `payload.model` overrides without tripping a stale live-session switch: cron model selection prefers `payload.model`, persists that selection before the run, suppresses a switch when the current model already matches persisted state, and retries genuine `LiveSessionModelSwitchError` cases. The same relevant files are unchanged between `v2026.4.23` and current `main`, so the fix is already shipped.

## Evidence

  - file: [src/cron/isolated-agent/run.ts:640](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/run.ts#L640)
