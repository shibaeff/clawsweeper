## Summary

The reported macOS LaunchAgent restart failure is already fixed on current `main` and was shipped after the reporter's `2026.3.8` build. Current code keeps the LaunchAgent registered during explicit restarts, uses a detached launchd handoff when the restart request comes from inside the managed process tree, and falls back to bootstrap+kickstart if launchd no longer has the service loaded.

## Evidence

- **Release note for the fix:** `2026.3.11` changelog explicitly says: 'Gateway/macOS launchd restarts: keep the LaunchAgent registered during explicit restarts, hand off self-restarts through a detached launchd helper, and recover config/hot reload restart paths without unloading the service.' That release is newer than the reporter's `2026.3.8` environment.
  - file: [CHANGELOG.md:2714](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/CHANGELOG.md#L2714)
  - command: `nl -ba CHANGELOG.md | sed -n '2681,2718p'`
- **Current launchd restart implementation:** `restartLaunchAgent()` now schedules a detached handoff when invoked from inside the managed LaunchAgent, otherwise does `launchctl enable` + `launchctl kickstart -k`, and falls back to `bootstrap` + retry instead of leaving the service unloaded.
  - file: [src/daemon/launchd.ts:625](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/launchd.ts#L625)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '620,690p'`
- **Detached handoff avoids self-termination race:** The helper script waits for the caller PID to exit, then runs `launchctl enable` and `launchctl kickstart -k`, with bootstrap fallback. This directly addresses the self-restart race that could stop the service without bringing it back.
  - file: [src/daemon/launchd-restart-handoff.ts:78](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/launchd-restart-handoff.ts#L78)
  - command: `nl -ba src/daemon/launchd-restart-handoff.ts | sed -n '78,148p'`
- **Regression coverage for managed restart path:** Tests cover the normal LaunchAgent restart path (`kickstart` with no `bootout`) and the fallback path when launchd cannot find the service.
  - file: [src/daemon/launchd.test.ts:649](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/launchd.test.ts#L649)
  - command: `nl -ba src/daemon/launchd.test.ts | sed -n '649,717p'`
- **Current CLI semantics:** Docs now describe `openclaw gateway restart` as restarting the background service, while unmanaged listener fallback uses SIGUSR1 separately. That matches current daemon lifecycle code rather than the older broken behavior from `2026.3.8`.
