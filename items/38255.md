## Summary

Keep open. Current `main` still hardcodes Telegram preview throttle/initial-preview behavior instead of exposing `channels.telegram.streamThrottleMs` and `channels.telegram.minInitialChars`, so the request is not implemented. The PR discussion also has an unresolved review concern: the proposed `streamThrottleMs` surface is only lower-bounded, while the shared draft loop passes it directly to `setTimeout`, and a large value overflows to `1ms` in Node.
