## Summary

Current `main` does not implement this Telegram transient-failure cooldown. The shared `sendChatAction` handler still only tracks 401s, still rethrows transient failures immediately, and its tests still lack transient cooldown coverage. Given the PR discussion and the current code paths, this remains a plausible, scoped bug fix rather than a cleanup close.

## Evidence

- **Current handler is still 401-only:** `createTelegramSendChatActionHandler` only maintains `consecutive401Failures`/`suspended`, only backs off when the 401 counter is nonzero, and rethrows all non-401 errors. There is no transient cooldown state or 429/5xx/network classification in the handler on `main`.
