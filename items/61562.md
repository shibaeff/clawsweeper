## Summary

Current `main` does not implement an optional intent/description field on the `exec` tool or in exec-approval requests. The request is coherent and still maps to an unmet core capability: plugin hooks can require approval, but they only receive existing tool params, and internal hooks do not intercept exec tool calls at all.

## Evidence

- **`exec` tool schema still only exposes command-oriented params:** The current `execSchema` includes `command`, `workdir`, `env`, `yieldMs`, `background`, `timeout`, `pty`, `elevated`, `host`, `security`, `ask`, and `node`; there is no optional `description` field.

The related `SystemRunApprovalPlan` also only adds `commandPreview`, not an approval description.
