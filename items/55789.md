## Summary

Keep open. Current `main` still accepts explicit `x-openclaw-session-key` overrides unchanged on the OpenAI-compatible and OpenResponses HTTP paths, and the public docs still describe that header as explicit session routing. This PR is therefore not implemented on `main`; it remains a real behavior change that needs maintainer judgment rather than conservative auto-close.
