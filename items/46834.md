## Summary

Keep open. Current `main` still turns off inline child delivery for `streamTo="parent"` ACP spawns, and the parent relay only writes internal system events instead of routing a final visible reply back to non-threaded direct chats. The requested behavior is not implemented on `main`, and there is still an unresolved whitespace-delta edge in the relay path reviewers called out.

## Evidence

- **`streamTo="parent"` disables child inline delivery:** `resolveAcpSpawnBootstrapDeliveryPlan()` sets `useInlineDelivery` only when there is a delivery target, `effectiveStreamToParent` is false, and spawn mode is `session`; the ACP dispatch then passes `deliver: deliveryPlan.useInlineDelivery` to the child gateway call. That means `streamTo="parent"` explicitly suppresses direct child delivery and relies on the parent relay path.
