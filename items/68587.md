## Summary

Keep open. Current `main` still runs `openclaw mcp serve` over stdio without the explicit stdout-cleaning guard used by OpenClaw’s other stdio MCP servers, and the existing MCP channel smoke does not exercise a raw `tools/list` request/response path. The channel bridge does register tools through the SDK `McpServer.tool(...)` API, so the report looks more like an unresolved stdio/protocol-output problem than an already-fixed intentional `notifications/tools/list` implementation.

## Evidence

- **Channel MCP stdio path still lacks stderr routing:** `serveOpenClawChannelMcp()` creates the stdio transport directly and never calls `routeLogsToStderr()`, so the `openclaw mcp serve` path still has no explicit guard keeping stdout protocol-only.
  - file: [src/mcp/channel-server.ts:65](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/mcp/channel-server.ts#L65)
  - command: `nl -ba src/mcp/channel-server.ts | sed -n '1,140p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Other OpenClaw stdio MCP servers added the missing guard:** The newer shared stdio MCP helper explicitly comments that MCP stdio requires stdout to stay protocol-only and calls `routeLogsToStderr()` before connecting the transport.
  - file: [src/mcp/tools-stdio-server.ts:24](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/mcp/tools-stdio-server.ts#L24)
  - command: `nl -ba src/mcp/tools-stdio-server.ts | sed -n '1,60p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **CLI startup only reroutes logs for JSON-output commands:** Startup presentation only calls `routeLogsToStderr()` when `suppressDoctorStdout` is true, and startup policy sets that flag from `jsonOutputMode` only. `mcp serve` is a plain text command and is not marked specially.
  - file: [src/cli/command-execution-startup.ts:29](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/cli/command-execution-startup.ts#L29)

- **Channel tools are registered through SDK tool APIs, not a custom tools notification path:** `registerChannelMcpTools()` uses `server.tool(...)` on `McpServer`, which suggests `tools/list` should come from the SDK’s normal request/response handling rather than an intentional custom `notifications/tools/list` implementation in OpenClaw code.
