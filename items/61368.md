## Summary

Keep open. Current `main` does not implement the requested full workspace file-management surface: gateway methods stop at `agents.files.{list,get,set}`, handlers only allow the fixed bootstrap/MEMORY files, the protocol has no path-based workspace operations, and the Control UI is still scoped to editing core instruction files. The issue is also recent and already cross-referenced to open PR #62417, so conservative cleanup should not close it.

## Evidence

- **Gateway method surface stops at agents.files:** The server method registry on `main` includes `agents.files.list`, `agents.files.get`, and `agents.files.set`; there are no `agents.workspace.*` methods registered.
  - file: [src/gateway/server-methods-list.ts:63](https://github.com/openclaw/openclaw/blob/ed7ea75fc0eafc101754de104103408ba13debc7/src/gateway/server-methods-list.ts#L63)

  - sha: [ed7ea75fc0ea](https://github.com/openclaw/openclaw/commit/ed7ea75fc0eafc101754de104103408ba13debc7)
- **Handler hard-limits access to bootstrap files:** `agents` server methods build an `ALLOWED_FILE_NAMES` set from the bootstrap files plus `MEMORY.md`, and reject any other requested file name as `unsupported file` before touching the workspace.
  - file: [src/gateway/server-methods/agents.ts:99](https://github.com/openclaw/openclaw/blob/ed7ea75fc0eafc101754de104103408ba13debc7/src/gateway/server-methods/agents.ts#L99)
