## Summary

Keep open. Current `main` still decides DM-vs-space using only `space.type`, does not model Google Chat's replacement `spaceType` / `singleUserBotDm` fields, and still defaults Google Chat group handling to fail-closed allowlist behavior. The latest release `v2026.4.23` also contains that same logic, so the report remains actionable rather than obsolete.

## Evidence

- **Current monitor logic only reads `space.type`:** `processMessageWithPipeline` computes `spaceType` from `space.type` and sets `isGroup = spaceType !== "DM"`, with no fallback to `space.spaceType` or `space.singleUserBotDm`.
  - file: [extensions/googlechat/src/monitor.ts:127](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/googlechat/src/monitor.ts#L127)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Google Chat event types do not include the newer space fields:** `GoogleChatSpace` only declares `name`, `displayName`, and `type`. A repo search found no `spaceType` or `singleUserBotDm` handling anywhere under the Google Chat plugin.
  - file: [extensions/googlechat/src/types.ts:1](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/googlechat/src/types.ts#L1)
  - command: `rg -n "spaceType|singleUserBotDm" extensions/googlechat/src`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Group-path drops match the reported silent-ignore behavior:** When `isGroup` is true, inbound access returns early for disabled, empty-allowlist, or not-allowlisted spaces, logging drop reasons instead of creating a session.
  - file: [extensions/googlechat/src/monitor-access.ts:221](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/googlechat/src/monitor-access.ts#L221)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Google Chat still defaults groups to fail-closed:** The Google Chat account schema defaults `groupPolicy` to `"allowlist"`, which makes a misdetected space/group event easy to drop silently unless explicitly configured otherwise.
  - file: [src/config/zod-schema.providers-core.ts:813](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/config/zod-schema.providers-core.ts#L813)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Docs and latest release still present conflicting state:** Docs say Google Chat is "ready for DMs + spaces", but release `v2026.4.23` still ships the same `space.type`-only check in `extensions/googlechat/src/monitor.ts:127-128`.
  - file: [docs/channels/googlechat.md:8](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/channels/googlechat.md#L8)
