## Summary

Current `main` does not implement the PR's `<notts>` / TTS-preprocessing feature set. The shipped TTS path still documents and gates only on `[[tts:...]]` directives, there is no `<notts>`/`<tts>` preprocessing in the runtime parser, the related config knobs are absent, and the helper/doc files introduced by the PR are not present on `main`. This is not strong enough for any close reason; keep the PR open if maintainers still want the feature, ideally split from the unrelated commits and reworked against current code.
