## Summary

Current `main` already contains bundled-MCP lifecycle cleanup for session/run teardown plus idle-runtime reaping, which is sufficient to address the reported duplicate MCP process accumulation on reconnect.

## Evidence

- **Session-scoped MCP runtimes are now tracked and retireable by session key:** The session runtime manager caches bundle-MCP runtimes by `sessionId`, records `sessionKey -> sessionId`, and exposes `retireSessionMcpRuntimeForSessionKey(...)`, so a later run for the same logical session can retire the prior runtime instead of leaving it orphaned.
  - file: [src/agents/pi-bundle-mcp-runtime.ts:462](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/agents/pi-bundle-mcp-runtime.ts#L462)
  - command: `nl -ba src/agents/pi-bundle-mcp-runtime.ts | sed -n '430,660p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Gateway session cleanup now disposes bundled MCP runtimes:** Gateway session reset/delete cleanup calls `retireSessionMcpRuntime(...)` after the embedded run ends, so session rollover no longer leaves bundle-MCP child processes behind.
  - file: [src/gateway/session-reset-service.ts:225](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/gateway/session-reset-service.ts#L225)
  - command: `nl -ba src/gateway/session-reset-service.ts | sed -n '194,245p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **One-shot embedded runs retire bundle-MCP at run end:** Embedded runner teardown now retires bundle-MCP first by `sessionKey` and then by `sessionId` fallback when `cleanupBundleMcpOnRunEnd` is enabled, covering one-shot runs that previously could leave MCP children alive after completion.
  - file: [src/agents/pi-embedded-runner/run.ts:2302](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/agents/pi-embedded-runner/run.ts#L2302)
