## Summary

Current `main` already covers the reported cron delivery paths: main-session cron runs now wake heartbeat delivery with `target: "last"`, and isolated `delivery.channel="last"` resolves against stored session routing or fails closed with explicit diagnostics instead of blindly sending an invalid request.

## Evidence

## Risks / Open Questions

- `lastDeliveryStatus` for main-session cron jobs is still not the delivery signal; delivery happens through heartbeat routing, so older expectations around `not-requested` can still be misleading when inspecting state alone.
