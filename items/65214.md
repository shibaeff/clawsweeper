## Summary

Current main already addresses the issue’s two reported subproblems: plugin install/config now uses the canonical manifest id instead of the npm package name, and TTS provider resolution now fallback-loads bundled speech-provider owners from manifest contracts when no active provider was startup-loaded. The gateway startup plugin count still excludes speech-only plugins by design, so that log line is no longer evidence that ElevenLabs TTS is unavailable.
