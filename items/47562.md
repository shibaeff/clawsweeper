## Summary

Keep open. Current `main` still does not implement Telegram custom emoji reactions inbound or outbound, and the action-layer assumptions called out in PR review are still present in the equivalent runtime path on `main`.

## Evidence

- **Outbound Telegram reactions are still emoji-only:** `reactMessageTelegram()` trims `emoji` and builds the request as either `[]` or a single `{ type: "emoji", emoji: ... }` reaction. There is no `customEmojiId` option and no `custom_emoji` branch on current `main`.

That matches the review concern for custom-emoji-only requests.
