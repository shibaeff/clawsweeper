## Summary

Keep open. Current `main` appears to have already addressed the registry-swap half of the report by resolving plugin channels from pinned runtime state, but it still hard-rejects unknown `channel`/`replyChannel` values in the gateway agent handler with no `deliver=false` + `lane="subagent"` bypass. The PR therefore is not fully implemented on `main`, and the submitted branch also had a build-breaking duplicate import called out in review.

## Evidence

- **Current main still strictly rejects unknown channels in `agent`:** The gateway handler normalizes `request.channel`/`request.replyChannel` and immediately returns `invalid agent params: unknown channel: ...` when the value is not a known gateway channel. There is no subagent-specific exemption in this block.
