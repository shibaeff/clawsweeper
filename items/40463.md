## Summary

Keep open. Current `main` already contains parts of this PR's intent, but not the full fix set. In particular, the current URL builder still lacks the proposed `/chats/...` fallback for channel-thread IDs and still prefers `conversationId` over `channelData.chatId` for chat URLs, so this is not `implemented_on_main`. I also do not have a safe `cannot_reproduce` path for the reported cross-tenant Teams cases in a read-only review.
