## Summary

Current `main` already implements the reported Telegram `/model` propagation fix: session model changes are persisted immediately, future/queued turns resolve from the stored override, Telegram session-state reads prefer that override, and stale runtime model metadata is cleared so status/current-model surfaces stop reporting the old model.
