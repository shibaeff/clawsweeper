## Close Comment

Closing this after a Codex review. The current main policy is that community plugins should live outside `openclaw/openclaw`: `VISION.md:54-61` says optional capability should usually ship as plugins and that plugin authors should host and maintain them in their own repository, and `docs/plugins/community.md:9-22` plus `docs/plugins/community.md:129-165` says community plugins are third-party packages published on ClawHub or npm and installed with `openclaw plugins install`. This PR is explicitly a community plugin, and current main already exposes the public hook surfaces it needs (`before_tool_call`, `after_tool_call`, `subagent_ended`; see `docs/concepts/agent-loop.md:91-100`, `src/plugins/hook-types.ts:72-81`, and `src/agents/harness/hook-helpers.ts:8-50`). The right path here is ClawHub/npm plus a public repo, not merging a new `community/plugins/spawn-interceptor/` tree into core.
