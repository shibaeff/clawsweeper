## Summary

Keep open. Current main still rejects `messages.groupChat.historyLimit: 0` in the core Zod schema and generated base schema, while docs and changelog say `0` disables group history. The PR also bundles unrelated `extensions/memory-core` changes, and review discussion already flagged a plausible regression there, so this is not a conservative auto-close candidate.
