## Summary

Keep open. Current `main` still uses insertion-order eviction in the cron cache, so the reported FIFO-vs-LRU bug remains plausible, and this PR still contains useful unmerged work rather than an already-shipped or obsolete change.

## Evidence

- **Current cache logic is still FIFO-style:** On cache hit, `resolveCachedCron()` returns immediately without deleting/re-setting the entry. When full, it evicts `cronEvalCache.keys().next().value`, which is the oldest insertion-order key in a `Map`, not the least recently used entry.
