## Summary

Current main already implements the requested behavior: `security=full` + `ask=off` no longer has a separate heuristic obfuscation approval gate. The only remaining extra approval path is the explicit opt-in `tools.exec.strictInlineEval` hardening mode, and the same behavior is already present in the shipped `v2026.4.23` tag.
