## Summary

Keep open. Current `main` still hardcodes the 5 MB inbound media cap and exposes no `media.maxBytes` config, so the underlying request remains valid. The submitted PR also appears to have unresolved correctness issues in review, so it should not be auto-closed as implemented or obsolete.
