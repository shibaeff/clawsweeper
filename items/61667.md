## Summary

Current `main` still falls back to exact-command durable approvals for interpreter-style execs, so `allow-always` does not generalize across dynamic `python ...`/similar commands. The secondary claim that `exec-approvals.json` is wiped on restart is not proven by the current code review, but the primary bug remains open on `main`.

## Evidence

- **Interpreter commands are specially classified:** `python`, `python3`, and similar runtimes are explicitly treated as interpreter-style executables in the inline-eval/interpreter guard list used by exec approval persistence.
  - file: [src/infra/exec-inline-eval.ts:35](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/infra/exec-inline-eval.ts#L35)
  - command: `nl -ba src/infra/exec-inline-eval.ts | sed -n '35,40p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Allow-always skips interpreter-like executables:** `collectAllowAlwaysPatterns()` returns without persisting an allowlist pattern when the resolved executable is interpreter-like, unless strict inline-eval mode is enabled for an actual inline eval case. A normal `python script.py ...` command therefore produces no reusable executable rule.
  - file: [src/infra/exec-approvals-allowlist.ts:965](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/infra/exec-approvals-allowlist.ts#L965)
  - command: `nl -ba src/infra/exec-approvals-allowlist.ts | sed -n '965,977p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Gateway exec falls back to durable exact-command approval:** When an `allow-always` approval yields zero persisted patterns, the gateway exec path calls `addDurableCommandApproval(...)` instead of storing an executable- or pattern-level rule.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:324](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/bash-tools.exec-host-gateway.ts#L324)
  - command: `nl -ba src/agents/bash-tools.exec-host-gateway.ts | sed -n '324,338p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Durable approval is stored as an exact command hash:** `addDurableCommandApproval()` stores `=command:<sha256-prefix>` derived from the full command text, and `hasExactCommandDurableExecApproval()` checks that exact command-derived pattern.
  - file: [src/infra/exec-approvals.ts:810](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/infra/exec-approvals.ts#L810)

  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Current tests assert the same limitation:** Main-branch tests explicitly expect interpreter-like executables to persist no allow-always patterns.
