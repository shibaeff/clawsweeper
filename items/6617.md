## Summary

Keep open. Current `main` still does not implement GitHub-style inline markdown images in Control UI agent replies. The chat renderer uses `toSanitizedMarkdownHtml(...)` for assistant markdown, and that markdown renderer only emits `<img>` for base64 `data:image/...` sources; normal `https://...` markdown images are intentionally flattened to alt text instead.

## Evidence

- **Assistant replies still flow through markdown sanitizer:** Control UI assistant chat messages render markdown via `toSanitizedMarkdownHtml(markdown)`, so markdown image behavior is defined by the shared renderer.
  - file: [ui/src/ui/chat/grouped-render.ts:1476](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/ui/src/ui/chat/grouped-render.ts#L1476)

  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Renderer only allows base64 data-image sources:** `md.renderer.rules.image` returns alt text unless `src` matches `^data:image/...;base64,`; only then does it emit `<img class="markdown-inline-image" ...>`.
