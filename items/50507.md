## Summary

Keep open. Current `main` at `9b7f6250f43bbc1816cdaf26e8c07116dfa4975f` still does not implement outbound WhatsApp native location sending, and the core/runtime seams this PR adds are still absent.

## Evidence

- **WhatsApp action discovery still excludes location:** `describeWhatsAppMessageActions()` only adds `react` and `poll`, and the channel plugin only declares `supportsAction`/`resolveExecutionMode` for `react`. A configured message tool on current `main` will not expose a WhatsApp `location` action.
