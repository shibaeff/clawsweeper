## Summary

Keep open. Current `main` still preserves the reported failure shape: blank tool-call names can survive normalization, get fallback ids like `call_auto_1`, and later persist as `toolName: "unknown"` instead of being replaced with a safe sentinel or blocked before dispatch.

## Evidence

- **Blank names are still preserved with fallback ids:** `src/agents/pi-embedded-runner/run/attempt.test.ts:1157-1171` explicitly asserts that when both tool name and id are blank, the blank name stays blank and the id becomes `call_auto_1`. That matches the issue’s reported malformed-tool-call shape rather than proving a fix.
  - file: [src/agents/pi-embedded-runner/run/attempt.test.ts:1157](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/agents/pi-embedded-runner/run/attempt.test.ts#L1157)

  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Dispatch path still accepts blank tool names:** `src/agents/pi-embedded-subscribe.handlers.tools.ts:575-576` and `:791-792` pass `evt.toolName` through `normalizeToolName(...)`. In this codepath, blank strings normalize to blank, not to a sentinel, so start/end handling can still run with an empty tool name.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:575](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/agents/pi-embedded-subscribe.handlers.tools.ts#L575)
