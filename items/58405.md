## Summary

Keep open. Current `main` still does not implement a per-request `skills` override for `POST /v1/responses`, so this PR is not obsolete.

## Evidence

- **`/v1/responses` schema still rejects unknown fields like `skills`:** `CreateResponseBodySchema` enumerates supported fields through `truncation` and then ends with `.strict()`. There is no `skills` property, so the requested API extension is not present on current `main`. See `src/gateway/open-responses.schema.ts:193`.
  - file: [src/gateway/open-responses.schema.ts:193](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/gateway/open-responses.schema.ts#L193)
  - command: `nl -ba src/gateway/open-responses.schema.ts | sed -n '180,230p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **OpenResponses ingress has no path to pass a skills override:** `runResponsesAgentCommand` forwards `message`, `images`, `clientTools`, `extraSystemPrompt`, `model`, `streamParams`, `sessionKey`, `runId`, `messageChannel`, and auth flags into `agentCommandFromIngress`; there is no `skillsOverride` field in this path. See `src/gateway/openresponses-http.ts:400` and `src/agents/command/types.ts:70`.
  - file: [src/gateway/openresponses-http.ts:400](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/gateway/openresponses-http.ts#L400)

  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Agent execution still derives skills only from agent config:** Current session/run logic calls `resolveAgentSkillsFilter(cfg, sessionAgentId)` and uses that result to build or refresh the workspace skills snapshot. There is no branch for an ephemeral per-request override. See `src/agents/agent-command.ts:574`.
  - file: [src/agents/agent-command.ts:574](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/agent-command.ts#L574)

There is no per-request `skills` section or request-field entry on current `main`. See `docs/gateway/openresponses-http-api.md:69`.
