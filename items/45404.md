## Summary

Current `main` clearly treats `openclaw` as the canonical managed browser profile, but I did not find any doctor, migration, or repair path that detects or archives leftover `~/.openclaw/browser/clawd/...` browser state. The issue still describes a real cleanup gap rather than something already implemented.
