## Summary

Keep open. Current `main` does not implement native task-based model routing within a single conversation. The shipped config and runtime only support a primary model, ordered fallbacks, per-agent overrides, and a few specialized alternate model slots such as `imageModel`/`pdfModel`/media-generation models. The request is coherent and still within OpenClaw core scope rather than a ClawHub-only skill.

## Evidence

- **Model selection docs:** The user-facing docs describe model choice as primary model, then ordered fallbacks, then provider-auth failover. They separately call out `imageModel`, `pdfModel`, and per-agent overrides, but no task/classification-based routing within one session.
  - file: [docs/concepts/models.md:16](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/docs/concepts/models.md#L16)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Config surface:** The public config docs show `agents.defaults.model` as either a string or `{ primary, fallbacks }`, plus dedicated slots for `imageModel`, `imageGenerationModel`, `musicGenerationModel`, `videoGenerationModel`, and `pdfModel`. There is no documented `routing`, keyword, or complexity classifier config under model selection.
  - file: [docs/gateway/config-agents.md:336](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/docs/gateway/config-agents.md#L336)

  - file: [src/config/zod-schema.agent-model.ts:3](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/config/zod-schema.agent-model.ts#L3)
