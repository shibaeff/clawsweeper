## Summary

Keep open. Current `main` does not contain this PR’s `wsConfig` change, and the pinned Feishu SDK contract on `main` does not expose a `wsConfig` constructor option. The underlying SDK already manages heartbeat/reconnect state from server-provided ping metadata, so this PR may be addressing a real problem but the proposed implementation needs maintainer review rather than conservative auto-close.
