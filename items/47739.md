## Summary

Keep open. Current `main` still flushes buffered assistant text before tool-start events with no colon-ending suppression, so this PR is neither implemented on `main` nor obsolete. The existing tests cover the generic pre-tool flush behavior but do not cover the colon-ending narration case this PR targets.

- **Current main still flushes all buffered pre-tool text:** `flushBufferedChatDeltaIfNeeded` on `main` only suppresses empty/silent/heartbeat text, then broadcasts the buffered delta. There is no `:`-ending or `suppressIncompleteNarration` guard in the helper.
