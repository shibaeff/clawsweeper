## Summary

The proposed hot/warm/cold memory tiering is better scoped as memory-plugin or ClawHub work, not an OpenClaw core item. Current main already assigns recall/promotion/runtime ownership to the active memory plugin and ships related plugin surfaces for dreaming, long-term memory, and wiki/dashboard-style memory views.
