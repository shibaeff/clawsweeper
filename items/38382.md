## Summary

Current `main` does not implement recursive scanning of symlinked external directories for startup/project-context injection. Bootstrap injection is still limited to a fixed allowlist of workspace files, and the only extra-file path explicitly keeps matches inside the workspace root via boundary/realpath checks. This is a plausible feature request, not a cleanup close.

## Risks / Open Questions

- Following symlinked directories out of the workspace would cut across the current workspace-boundary and hardlink/symlink safety model.
- Recursive corpus injection also raises prompt-size and prompt-injection concerns beyond the current fixed bootstrap-file design.
- There is an extension path for custom context assembly (`docs/concepts/context-engine.md`), so product/scope triage may still decide this belongs in a plugin rather than core; that is not strong enough for an auto-close here.
