## Summary

Close the PR as obsolete. Current `main` no longer uses the mobile `.nav` layout this patch targets; the Control UI now renders a separate off-canvas `.shell-nav` drawer on small screens, and the burger toggle drives `navDrawerOpen` rather than the old collapsed-nav path. Against current `main`, the reported leftover mobile strip/sidebar behavior is not reproducible from the code and existing browser coverage.

## Evidence

- **Mobile nav architecture changed on current main:** The Control UI shell now renders a dedicated `.shell-nav` wrapper with a `.sidebar`/`.sidebar-nav` inside it. The topbar burger button toggles `navDrawerOpen`, not `navCollapsed`, so the mobile open/close path is different from the PR description.

This supersedes the older 'top horizontal strip' behavior described in the PR body.
