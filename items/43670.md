## Summary

Keep open. Current `main` already covers much of the requested failover classification, but it does not fully subsume this PR: raw network failures are timeout-classified in the shared failover helpers, yet the before-reply retry path in `runAgentTurnWithFallback()` is still HTTP-only and subagent announce delivery still omits `fetch failed` from transient retry matching.
