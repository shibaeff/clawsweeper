## Summary

Current `main` already supports the requested per-agent bundled-skill filtering via `agents.defaults.skills` and `agents.list[].skills`. The implementation loads bundled skills into the merged skill set, then applies the effective per-agent skill filter to that full set, so bundled skills are filtered per agent without needing a separate `agents.list[].allowBundled` key.

## Risks / Open Questions

- The implementation uses the existing per-agent `skills` allowlist rather than adding a separate `agents.list[].allowBundled` override, so the exact config shape requested is different even though the user-facing behavior is already available.

## Close Comment

Closing as implemented on `main` after a Codex review. OpenClaw already supports per-agent filtering of bundled skills through `agents.defaults.skills` and `agents.list[].skills`: the loader merges bundled skills into the full skill set first (`src/agents/skills/workspace.ts:544`), then applies the effective per-agent skill filter to that merged set (`src/agents/skills/agent-filter.ts:24`, `src/agents/skills/workspace.ts:756`). The config type and docs also describe `agents.list[].skills` as the final per-agent visible skill set (`src/config/types.agents.ts:89`, `docs/tools/skills.md:44`). This behavior is present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
