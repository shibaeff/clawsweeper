## Summary

Keep open. Current `main` still relies on prompt-level "ask first" guidance while host exec defaults remain `security: full` and `ask: off` on gateway/node, and the public exec docs explicitly say no-approval host exec is the default with no extra heuristic prefilter in that mode. I did not find evidence on current `main` that this specific destructive-misinterpretation path has been closed out conservatively enough to auto-close the issue.

## Evidence
