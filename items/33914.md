## Summary

Keep open. Current `main` partially supersedes the PR by already carrying `sessionKey` on message-hook contexts, but the core request is still not implemented: `PluginHookAgentContext` on `main` does not expose `sourceProvider`, sender identity, `groupId`, or `spawnedBy`, and those fields are still not propagated consistently through queued runs, compaction hooks, and outbound hook paths. The discussion also includes real third-party plugin demand (`openclaw-chat-only-trigger`), so this remains a useful maintainer-reviewed feature PR rather than cleanup-close material.

## Risks / Open Questions

- The branch is partially outdated because `main` independently picked up some `sessionKey` plumbing, so any eventual merge would need a careful rebase/cherry-pick rather than a blind land.

- This is a public plugin-sdk surface change, so maintainers would need contract/docs/baseline review before deciding whether and how to land it.
