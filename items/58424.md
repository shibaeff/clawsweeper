## Summary

Current `main` already implements the requested interrupt path for main-session exec/tool execution: chat runs get an `AbortController`, `/stop` aborts that controller, every tool call checks the abort signal before execution, and the `exec` tool kills the active foreground process on abort. Current `main` also has loop-detection guardrails for repeated no-progress tool calls and configurable exec timeouts. The implementing code is present on `main` in commit `a6aa151653ab478e9aa23e95c6446c41adc4fce1`, which postdates the issue and is newer than the latest listed release `v2026.4.23`, so there is no confirmed shipped release tag yet.
