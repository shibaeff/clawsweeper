## Summary

Current main already retries recovered auth profiles after cooldown expiry on the next run and records typed failover reasons in structured logs, so the reported behavior no longer matches the shipped code.

## Evidence

- **Per-run auth order resolution:** Each embedded run recomputes auth profile order before selecting credentials, so recovered profiles are reconsidered on subsequent requests instead of staying pinned forever to the last fallback.
  - file: [src/agents/pi-embedded-runner/run.ts:397](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/pi-embedded-runner/run.ts#L397)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Expired cooldowns are cleared before ordering:** resolveAuthProfileOrder clears expired cooldown state and then keeps available profiles ahead of cooldowned ones, including explicit auth.order paths.
  - file: [src/agents/auth-profiles/order.ts:77](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/auth-profiles/order.ts#L77)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Recovered profiles get a fresh retry window:** clearExpiredCooldowns removes expired cooldown/disabled windows and resets error counters so a recovered profile is retried normally instead of remaining effectively stuck behind escalated backoff state.
  - file: [src/agents/auth-profiles/usage.ts:428](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/auth-profiles/usage.ts#L428)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Regression coverage:** Integration tests assert that expired cooldown profiles return to the available list in both round-robin and explicit-order cases, and that stale penalty state is reset.
  - file: [src/agents/auth-profiles.cooldown-auto-expiry.test.ts:25](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/auth-profiles.cooldown-auto-expiry.test.ts#L25)

## Risks / Open Questions

- This is not a background timer that promotes profiles with no new traffic; recovery happens when auth order is resolved for the next run/request. If the request required an out-of-band periodic probe while idle, that narrower behavior is still not present.

## Close Comment

Codex review: closing as implemented on current main. OpenClaw now clears expired auth-profile cooldowns before every auth-order resolution (`src/agents/auth-profiles/order.ts:77`, `src/agents/auth-profiles/usage.ts:428`), and each embedded run re-resolves profile order before selecting credentials (`src/agents/pi-embedded-runner/run.ts:397`). That means a higher-priority profile is retried once its cooldown expires instead of staying pinned to the fallback indefinitely. The behavior is covered by `src/agents/auth-profiles.cooldown-auto-expiry.test.ts:25`. Failure reasons are also differentiated in structured logs via `failoverReason` / `profileFailureReason` and auth-profile state `reason` fields (`src/agents/pi-embedded-runner/run/failover-observation.ts:30`, `src/agents/auth-profiles/state-observation.ts:8`). This implementation is present in the `v2026.4.22` release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`.
