## Summary

Current `main` already contains an always-on unknown-tool loop guard for embedded agent streams. It rewrites repeated unavailable tool calls into plain assistant text after the configured threshold, which directly addresses the reported `Tool weather not found` runaway retry loop. The change is on `main` but not in a tagged release yet.

## Risks / Open Questions

- I did not run a live LM Studio / Gemma / Qwen reproduction in this read-only review, so confidence comes from current `main` source and targeted tests rather than an end-to-end provider check.
- The default unknown-tool guard threshold is 10 attempts, so users may still see several repeated failures before the loop is cut off.
- The fix is not in the latest tagged release yet, so users on released builds may still hit the original behavior until the next release ships.
