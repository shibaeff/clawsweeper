## Summary

Keep open. Current `main` still tells the top-level requester that a completed subagent task is "ready for user delivery" and to "send that user-facing update now," and the proposed replacement wording / verification guardrail is not present. Nested subagent-to-parent orchestration is handled more carefully, but the main-agent completion path described in the issue is still encoded and covered by tests.
