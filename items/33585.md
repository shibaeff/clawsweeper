## Summary

The reported default `workspace/memory/` watcher bug is already fixed on current `main` and is present in the latest shipped release: memory-core now watches the `memory` directory directly instead of the broken `memory/**/*.md` glob, with regression coverage and changelog evidence.

## Evidence

- **Current main watches `memory/` directly:** `ensureWatcher()` seeds chokidar with `path.join(this.workspaceDir, "memory")` and uses an `ignored` callback to filter non-markdown churn, so the literal `memory/**/*.md` default-watch path from the report is no longer used for the workspace memory directory.

- `memorySearch.extraPaths` directories still add `**/*.md` and multimodal glob watch entries in `extensions/memory-core/src/memory/manager-sync-ops.ts:383-389`, so there may still be related chokidar-v5 risk outside the specific default `workspace/memory/` path reported in this issue.
