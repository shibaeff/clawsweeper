## Summary

Current main does not count `sessions.json` terminal entries toward `acp.maxConcurrentSessions`; it counts only live in-memory ACP runtime handles, and the reported `failed`/rehydration behavior does not match the shipped code or tests.

## Evidence

- **Concurrent limit is enforced from live runtime cache:** `enforceConcurrentSessionLimit()` checks `this.runtimeCache.size()` and throws from that in-memory count; it does not read `sessions.json` when enforcing `acp.maxConcurrentSessions`.
  - file: [src/acp/control-plane/manager.core.ts:1644](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/acp/control-plane/manager.core.ts#L1644)
  - command: `nl -ba src/acp/control-plane/manager.core.ts | sed -n '1643,1659p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Persisted ACP session state has no `failed` terminal enum:** The persisted ACP metadata schema only allows `state: "idle" | "running" | "error"`; there is no `failed` ACP session state in the current `sessions.json` contract.
  - file: [src/config/sessions/types.ts:49](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/sessions/types.ts#L49)
  - command: `nl -ba src/config/sessions/types.ts | sed -n '44,52p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Runtime cache is only populated after successful runtime ensure/init:** The manager calls `setCachedRuntimeState()` only after successful `initializeSession()` persistence and after successful `ensureRuntimeHandle()`, so stale store entries are not rehydrated into the live concurrent-session count by themselves.
  - file: [src/acp/control-plane/manager.core.ts:409](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/acp/control-plane/manager.core.ts#L409)
  - command: `nl -ba src/acp/control-plane/manager.core.ts | sed -n '348,411p;1510,1542p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Close and tolerated backend-failure paths evict cached handles:** `closeSession()` clears the cached runtime handle on fast reset, successful close, and tolerated backend-unavailable / stale-exit recovery paths, explicitly preventing those handles from continuing to count against `maxConcurrentSessions`.
  - file: [src/acp/control-plane/manager.core.ts:1281](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/acp/control-plane/manager.core.ts#L1281)
  - command: `nl -ba src/acp/control-plane/manager.core.ts | sed -n '1233,1335p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Tests document that a failed persistent turn is still an active session until closed/evicted:** The manager test intentionally keeps a timed-out persistent ACP session counted as active (`activeSessions === 1`) and rejects opening a second session at the concurrency cap, showing that turn failure is not treated as automatic session death.
  - file: [src/acp/control-plane/manager.test.ts:678](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/acp/control-plane/manager.test.ts#L678)
  - command: `nl -ba src/acp/control-plane/manager.test.ts | sed -n '521,585p;667,685p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Stored error-state ACP sessions are reinitialized, not revived as live handles:** Persistent-binding coverage expects a stored ACP session in `error` state to be closed and reinitialized, which is inconsistent with `sessions.json` itself being the source of live concurrent-session accounting.
  - file: [src/acp/persistent-bindings.test.ts:930](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/acp/persistent-bindings.test.ts#L930)
  - command: `nl -ba src/acp/persistent-bindings.test.ts | sed -n '930,945p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
