## Summary

Keep open. Current `main` still does not support `memorySearch.remote.proxy`, and the built-in remote memory embedding path still uses strict guarded fetch behavior that explicitly does not honor `HTTP_PROXY`/`HTTPS_PROXY` unless trusted env-proxy mode is opted in.

## Evidence

- **Memory remote HTTP path does not enable env-proxy mode:** `withRemoteHttpResponse()` in `packages/memory-host-sdk/src/host/remote-http.ts` calls `fetchWithSsrFGuard(...)` directly and passes only `url`, `init`, `policy`, and `auditContext`; it does not use `withTrustedEnvProxyGuardedFetchMode(...)`.
  - file: [packages/memory-host-sdk/src/host/remote-http.ts:22](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/packages/memory-host-sdk/src/host/remote-http.ts#L22)
  - command: `nl -ba packages/memory-host-sdk/src/host/remote-http.ts | sed -n '1,40p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Default guarded fetch mode is strict:** `resolveGuardedFetchMode()` in `src/infra/net/fetch-guard.ts` falls back to `GUARDED_FETCH_MODE.STRICT` when no explicit mode is provided. That is the path used by the memory remote helper above.
  - file: [src/infra/net/fetch-guard.ts:113](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/infra/net/fetch-guard.ts#L113)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '101,121p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Tests confirm strict mode ignores HTTP_PROXY/HTTPS_PROXY:** `src/infra/net/fetch-guard.ssrf.test.ts` has an explicit regression test: `ignores env proxy by default to preserve DNS-pinned destination binding`. A separate test immediately below shows env proxy routing happens only when trusted proxy mode is explicitly enabled.
  - file: [src/infra/net/fetch-guard.ssrf.test.ts:1007](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/infra/net/fetch-guard.ssrf.test.ts#L1007)
  - command: `nl -ba src/infra/net/fetch-guard.ssrf.test.ts | sed -n '1007,1018p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Config schema does not allow memorySearch.remote.proxy:** `MemorySearchSchema` defines `memorySearch.remote` with only `baseUrl`, `apiKey`, `headers`, and `batch`, and the object is `.strict()`. A `proxy` key is unsupported on current `main` and would be rejected as an unknown config key rather than treated as a supported field.
  - file: [src/config/zod-schema.agent-runtime.ts:662](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/config/zod-schema.agent-runtime.ts#L662)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '662,679p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Public docs still expose no proxy setting for memory search:** `docs/reference/memory-config.md` documents the remote endpoint config with only `remote.baseUrl`, `remote.apiKey`, and `remote.headers`. There is no documented memory-search proxy option or env-proxy guidance here.
  - file: [docs/reference/memory-config.md:79](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/reference/memory-config.md#L79)
