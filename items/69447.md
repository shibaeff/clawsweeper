## Summary

Keep open. Current `main` still hard-codes web chat attachments as images in the UI and Gateway, and the linked PR is still only cross-referenced, not merged.

## Evidence

- **UI picker and drag/drop still reject non-image files:** The Control UI attachment allowlist is `image/*`, and both file-select and drag/drop handlers skip anything whose MIME type does not start with `image/`.
  - file: [ui/src/ui/chat/attachment-support.ts:1](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/ui/src/ui/chat/attachment-support.ts#L1)
  - command: `nl -ba ui/src/ui/chat/attachment-support.ts | sed -n '1,120p'; nl -ba ui/src/ui/views/chat.ts | sed -n '244,305p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **UI request serialization still forces attachments to image type:** `buildApiAttachments` serializes every outgoing attachment as `{ type: "image", mimeType, content }`, so even if the picker were broadened the web client would still label all uploads as images.
  - file: [ui/src/ui/controllers/chat.ts:349](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/ui/src/ui/controllers/chat.ts#L349)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '335,380p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Gateway parser still drops non-image payloads:** `parseMessageWithAttachments` warns and drops detected non-image content, rejects unknown non-image MIME when sniffing fails, and only appends inline `{ type: "image" }` blocks for accepted attachments.
  - file: [src/gateway/chat-attachments.ts:293](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/gateway/chat-attachments.ts#L293)
  - command: `nl -ba src/gateway/chat-attachments.ts | sed -n '293,472p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Current tests codify image-only behavior:** Gateway tests explicitly expect non-image MIME rejection and non-image payload dropping, which confirms the current mainline behavior is still intentionally image-only rather than already fixed.
  - file: [src/gateway/chat-attachments.test.ts:39](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/gateway/chat-attachments.test.ts#L39)

  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
