## Summary

Keep open. Current `main` includes partial mitigation for this report, but not the full request: QA child processes are explicitly killed and temp roots are removed when `gateway.stop()` runs, and live transport lanes can omit `qa-channel`; however the `qa suite` command path still has no `SIGINT`/`SIGTERM` trapping like the interruptible server path, so interrupted runs are not guaranteed to reach cleanup, and restart-settle still depends on a fixed sleep before probing readiness.

## Evidence

- **Interrupted suite path lacks signal cleanup:** `runInterruptibleServer` installs `SIGINT`/`SIGTERM` handlers and awaits `server.stop()`, but `runQaSuiteCommand` goes straight into `runQaSuiteFromRuntimeWithInfraRetry(...)` with no equivalent signal handling. That leaves the issue's "interrupted QA runs" case without a guaranteed path to `gateway.stop()` cleanup.
