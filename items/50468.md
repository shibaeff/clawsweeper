## Summary

Keep open. Current `main` still builds the skills watcher from glob targets like `<root>/SKILL.md` and `<root>/*/SKILL.md` in `src/agents/skills/refresh.ts`, while the repo is on `chokidar` `^5.0.0`; the latest release tag `v2026.4.22` contains the same code, and the unit test still asserts those glob targets. I found no evidence in the checked-out history that this issue was fixed on `main`.
