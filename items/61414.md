## Summary

Current `main` already honors explicit cron `delivery.channel` for isolated `agentTurn` jobs. The fallback to `"last"` now only applies when no delivery object is configured, and the latest release tag available in this checkout (`v2026.4.23`) contains the same delivery-plan implementation.

## Evidence

- **Current delivery-plan preserves explicit channel:** `resolveCronDeliveryPlan()` reads `delivery.channel`, defaults to `"last"` only when the job has a delivery object without an explicit channel, and only uses the implicit isolated-agent fallback in the no-delivery branch.
  - file: [src/cron/delivery-plan.ts:29](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/cron/delivery-plan.ts#L29)
  - command: `nl -ba src/cron/delivery-plan.ts | sed -n '1,120p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Isolated cron runner passes resolved plan channel through:** The isolated cron path calls `resolveCronDeliveryPlan(job)` and forwards `deliveryPlan.channel` into `resolveDeliveryTarget(...)`, so an explicit channel is used during target resolution instead of being overwritten to `last`.
  - file: [src/cron/isolated-agent/run.ts:304](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/cron/isolated-agent/run.ts#L304)
