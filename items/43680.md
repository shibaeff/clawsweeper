## Summary

Current `main` already honors `agents.list[].model` over `agents.defaults.model`, preserves per-agent fallback overrides, and includes regression coverage plus docs for that behavior. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Per-agent primary wins during default-model resolution:** `resolveDefaultModelForAgent(...)` reads `resolveAgentEffectiveModelPrimary(...)` for the selected agent, then rewrites `agents.defaults.model.primary` before calling `resolveConfiguredModelRef(...)`. That means per-agent model config is used for runtime resolution instead of the global default.
  - file: [src/agents/model-selection.ts:195](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/model-selection.ts#L195)

- **Agent scope explicitly prefers agent config before defaults:** `resolveAgentEffectiveModelPrimary(...)` returns the explicit `agents.list[].model` value first and only falls back to `agents.defaults.model` when the agent has no override. `resolveAgentModelFallbacksOverride(...)` also treats an explicit empty fallback array as a real override.
  - file: [src/agents/agent-scope.ts:101](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/agent-scope.ts#L101)

## Close Comment

Closing from a Codex review because current OpenClaw already implements this behavior. `src/agents/agent-scope.ts:101` resolves `agents.list[].model` before `agents.defaults.model`, `src/agents/model-selection.ts:195` feeds that per-agent primary into the runtime model resolver, and `src/agents/agent-scope.test.ts:157` covers explicit per-agent primary/fallback behavior including `fallbacks: []`. The same implementation is present in the shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit this on a newer build, please open a fresh issue with the exact version, the agent id being invoked, and the status/session output showing the resolved model so maintainers can compare it against the current code path.
