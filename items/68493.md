## Summary

Current `main` has several mitigations around SIGUSR1 restarts, lock release, stale-lock recovery, and `EADDRINUSE` retry, but I could not get to high-confidence closure for this Windows hot-reload report. There is no clear fixing commit or linked PR for #68493, the related supervisor-restart issue #65668 is still open, and this review could not perform a real Windows reproduction. The item remains plausible enough that conservative cleanup should leave it open.

## Evidence

  - file: [src/cli/gateway-cli/run-loop.ts:82](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/cli/gateway-cli/run-loop.ts#L82)
