## Summary

Keep open. Current `main` only covers the upload-layer part of this idea; it does not thread `filename` through the generic send/reply payload path into Feishu outbound sends, so this PR is not implemented on `main`. The PR also remains a poor conservative-maintainer cleanup candidate because it bundles the Feishu work with many unrelated workflow/docs/local-script/skill changes, and prior review flagged real regressions in the PR version of `extensions/feishu/src/media.ts`.
