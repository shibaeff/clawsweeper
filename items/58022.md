## Summary

Keep open. Current `main` does not implement the proposed auto-reload behavior, and the PR's remaining approach is still too broad: reloading on every WebSocket close code `1000` would fire for generic disconnect paths, while current stale-auth handling on `main` uses explicit auth-change signals instead.
