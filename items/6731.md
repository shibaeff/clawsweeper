## Summary

Keep open. Current main has new opt-in exec-policy presets and approvals tooling, but it still documents and implements gateway/node host exec as `security="full"` + `ask="off"` by default, with no extra dangerous-op heuristic gate in that mode. That means the issue's core request for a built-in safe/unsafe hard gate is only partially addressed, not implemented strongly enough to close.

## Evidence
