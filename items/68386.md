## Summary

Keep open. Current `main` still routes async exec wake handling through the isolated `:heartbeat` session when `heartbeat.isolatedSession` is enabled, so the issue's core session-routing complaint is not implemented away. Recent fixes on `main` improve exec-completion prompt content/classification, but they do not move the completion turn back to the originating main session.

## Evidence

- **Exec completion wakes the originating session key:** Background exec completion enqueues a system event on the exec run's `sessionKey` and requests a targeted heartbeat wake with `reason: "exec-event"` for that same session key.
  - file: [src/agents/bash-tools.exec-runtime.ts:279](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/agents/bash-tools.exec-runtime.ts#L279)

  - file: [src/infra/heartbeat-runner.ts:865](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/infra/heartbeat-runner.ts#L865)
