## Summary

Keep open. Current `main` still runs Active Memory synchronously in `before_prompt_build`, waits up to `timeoutMs` before failing open, and the active-memory implementation/docs are unchanged from the latest shipped release. The issue discussion also includes a newer beta confirmation, so there is not strong evidence for an auto-close reason.
