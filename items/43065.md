## Summary

Keep open: current `main` still resolves a raw heartbeat `responsePrefix` and normalizes heartbeat replies without any template-resolution context, so the PR's fix is not implemented on `main` despite later cross-references to replacement PRs.

## Evidence

- **Heartbeat runner still uses raw prefix for heartbeat replies:** `runHeartbeatOnce()` reads `responsePrefix` directly from `resolveEffectiveMessagesConfig(...)` and later passes that raw string into `normalizeHeartbeatReply(...)`; there is no `createReplyPrefixContext()` or post-LLM `resolveResponsePrefixTemplate()` step in the heartbeat path.
  - file: [src/infra/heartbeat-runner.ts:834](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/infra/heartbeat-runner.ts#L834)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '828,1115p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Heartbeat reply options never forward model-selection context:** The heartbeat `replyOpts` only include heartbeat flags/timeouts/light-context, and the `getReplyFromConfig(...)` call does not pass `onModelSelected`, so the heartbeat path cannot populate model/provider values for prefix interpolation.
  - file: [src/infra/heartbeat-runner.ts:1068](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/infra/heartbeat-runner.ts#L1068)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '1060,1080p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Template interpolation requires explicit prefix context:** `normalizeReplyPayload()` only interpolates template variables when `responsePrefixContext` is provided; otherwise it uses the prefix string as-is. That matches the bug described in the PR.
  - file: [src/auto-reply/reply/normalize-reply.ts:112](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/auto-reply/reply/normalize-reply.ts#L112)
  - command: `nl -ba src/auto-reply/reply/normalize-reply.ts | sed -n '100,130p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **The reusable prefix-context helper exists elsewhere on main:** `createReplyPrefixContext()` already provides both `responsePrefixContextProvider` and `onModelSelected`, and the agent runner invokes `onModelSelected` once the actual provider/model is known. The missing piece is wiring that helper into heartbeat replies, which is exactly what this PR proposes.
  - file: [src/channels/reply-prefix.ts:24](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/channels/reply-prefix.ts#L24)
