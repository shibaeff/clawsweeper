## Summary

Keep open. Current `main` still leaves `Refresh chat data` clickable during active turns, and that path still reloads chat history in a way that clears streaming state. The linked follow-up PR (#66395) proposes the missing guard, but its change is not present in `main` at `1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19`.

## Evidence

- **Refresh button is still enabled during active turns on current main:** `ui/src/ui/app-render.helpers.ts:240` only disables the button for `chatLoading` or disconnected state. It does not include the existing busy-state guards used elsewhere (`chatSending`, `chatRunId`, `chatStream`).
  - file: [ui/src/ui/app-render.helpers.ts:240](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/app-render.helpers.ts#L240)
  - command: `nl -ba ui/src/ui/app-render.helpers.ts | sed -n '236,251p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Clicking the button still triggers a full chat refresh path:** The same handler in `ui/src/ui/app-render.helpers.ts:241-250` calls `refreshChat(...)`, and `ui/src/ui/app-chat.ts:510-522` unconditionally runs `loadChatHistory(...)` plus other refresh work. There is no active-run guard in `refreshChat`.
  - file: [ui/src/ui/app-chat.ts:510](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/app-chat.ts#L510)

- **History reload still clears streaming state:** `ui/src/ui/controllers/chat.ts:182-186` explicitly resets tool-stream state and clears `chatStream` during `loadChatHistory(...)`, which is consistent with the reported mid-turn disruption.
  - file: [ui/src/ui/controllers/chat.ts:182](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/controllers/chat.ts#L182)

Manual refresh still bypasses that safeguard.
  - file: [ui/src/ui/app-gateway.ts:496](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/app-gateway.ts#L496)

Current `main` still has the older condition, so the fix is not landed yet.
