## Summary

Close as `clawhub`. The proposed eval framework expands core `openclaw skills` into author-only testing/benchmarking tooling, but current `main` keeps core skills CLI limited to discovery/install/inspection, routes creator workflows to ClawHub or optional plugins, and already keeps skill-authoring helpers inside `skills/skill-creator` itself.

## Close Comment

Closing from a Codex review. Current `main` keeps `openclaw skills` focused on discovery/install/inspection (`src/cli/skills-cli.ts:52`, `docs/cli/skills.md:2`), routes skill publish/sync workflows to ClawHub and the separate `clawhub` CLI (`docs/tools/skills.md:109`), and already keeps skill-authoring helpers inside the bundled `skill-creator` skill itself (`skills/skill-creator/SKILL.md:203`, `skills/skill-creator/scripts/quick_validate.py:67`, `skills/skill-creator/scripts/test_quick_validate.py:1`, `skills/skill-creator/scripts/test_package_skill.py:1`). `VISION.md:52` also says core should stay lean and optional capability should usually ship as plugins / ClawHub. Because this PR adds new core `openclaw skills test|bench|analyze-trigger` authoring commands for an optional skill-eval workflow, it fits better as ClawHub/plugin-side tooling or an extension of the existing `skill-creator` scripts than as OpenClaw core.
