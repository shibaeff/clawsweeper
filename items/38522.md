## Summary

Current main has partial fan-out/fan-in primitives (`sessions_spawn`, `sessions_yield`, descendant completion aggregation), but it does not implement the requested first-class barrier API: there is no `sessions_await`/barrier tool, no batched cohort spawn request, no `awaitMode` (`all`/`any`/`minComplete`) parameter, and combined child results are delivered as unstructured announce text rather than a single structured tool response.

## Evidence

- **No barrier tool in current tool catalog:** The sessions tool catalog on main includes `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status`, but no `sessions_await` or barrier-style tool.
  - file: [src/agents/tool-catalog.ts:169](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/tool-catalog.ts#L169)

- **Current documented orchestration model is spawn-now, wait-via-yield later:** Docs say `sessions_spawn` is always non-blocking and returns immediately with a `runId`/`childSessionKey`; waiting is done by ending the turn with `sessions_yield` so results arrive as a later follow-up event.
