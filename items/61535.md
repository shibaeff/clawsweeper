## Summary

Current `main` no longer drops text for captioned `MEDIA:` replies when block streaming is off. The direct-send fallback now applies only to media-only blocks, the final payload path keeps buffered text+media replies, and outbound delivery sends the text as the media caption.

## Evidence

- **Current block-reply logic keeps captioned media buffered:** `createBlockReplyDeliveryHandler` now only direct-sends when `blockHasMedia && !blockPayload.text`; captioned media no longer goes through the old `text: undefined` path when block streaming is disabled.

  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Regression test covers captioned media with block streaming off:** The current test asserts that a text+media block reply does not call `onBlockReply` directly and does not add a direct-send dedupe key, which prevents the text from being discarded before final delivery.
  - file: [src/auto-reply/reply/reply-delivery.test.ts:16](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/auto-reply/reply/reply-delivery.test.ts#L16)
