## Summary

Keep open. Current `main` does not contain the proposed privacy fix, and the PR still points at a real gap: `append` upserts reach access control before the history filter, while access control has no append-specific suppression. At the same time, the submitted patch changes currently tested behavior by skipping read receipts for stale `append` messages, so there is not strong evidence to auto-close this PR as obsolete or already handled.

## Evidence

That means append messages still reach the access-control path first.
