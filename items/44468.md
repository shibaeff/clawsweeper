## Summary

Current `main` does not implement this request. `agents.defaults.models` still only accepts `alias`, `params`, and `streaming`, runtime code still only consumes alias/params from that map, and provider-level override validation still requires `baseUrl` plus model `name` for explicit provider entries.

## Evidence

- **Agent defaults schema is still strict:** `agents.defaults.models` entries are a strict object with only `alias`, `params`, and `streaming`; there is no `contextWindow`, `maxTokens`, or passthrough support.
