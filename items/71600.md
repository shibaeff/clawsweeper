## Summary

Current `main` already contains a targeted fix for this exact Windows `ENAMETOOLONG` path: the bundled `claude-cli` backend now passes the system prompt by file (`--append-system-prompt-file`) instead of inline argv, and the runner writes that prompt to a temp file for fresh Claude sessions. The reported release `v2026.4.23` predates the fix, so this is fixed on `main` but not yet in that tagged release.
