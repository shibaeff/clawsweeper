## Summary

Keep open. Current `main` already offers `gateway.reload.mode=hot` or `off` as a workaround to avoid automatic restarts from direct config-file edits, but it does not implement the requested approval-gated/manual-approval restart flow, queued restart state, or admin notification path. The feature request is still distinct from current behavior.

## Evidence

- **Docs say direct file edits are watched and `hybrid` auto-restarts critical changes:** The hot-reload docs explicitly say the gateway watches `~/.openclaw/openclaw.json`, and in `hybrid` mode it 'Automatically restarts for critical ones'; `hot` only logs that restart is needed and leaves handling to the operator; `off` disables watching until a manual restart.
  - file: [docs/gateway/configuration.md:502](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/docs/gateway/configuration.md#L502)
  - command: `nl -ba docs/gateway/configuration.md | sed -n '500,555p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **`plugins.*` changes are classified as restart-required:** Reload planning marks `plugins` and `gateway` config prefixes as `restart`, so a change like `plugins.allow` is still treated as requiring a gateway restart.
  - file: [src/gateway/config-reload-plan.ts:111](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/gateway/config-reload-plan.ts#L111)

- **Current reloader only supports auto-restart, hot-ignore, or off:** When a restart-required change is detected, the reloader either returns early for `off`, warns and ignores for `hot`, or queues a restart path. There is no approval queue, approval RPC/CLI, or notification hook in this control flow.
  - file: [src/gateway/config-reload.ts:286](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/gateway/config-reload.ts#L286)

- **Restart handler defers until idle, then restarts without a human approval step:** The reload handler computes active work, defers until the gateway is idle, then logs and proceeds to restart. The code has deferral/cooldown behavior, but no explicit human-approval gate or pending-restart acknowledgment mechanism.
