## Summary

Keep open. Current `main` still appears to have the Discord `thread-reply` bug this PR targets: the guild-admin entrypoint only reads `media`, not `path`/`filePath`, and it does not pass the `mediaLocalRoots` options through on the `thread-reply` path. The PR discussion also shows the author addressed the review feedback about `mediaLocalRoots` passthrough in the latest force-push.

## Evidence

- **Current main still drops `path`/`filePath` for `thread-reply`:** In the `thread-reply` branch, current `main` reads only `readStringParam(actionParams, "media", { trim: false })` and forwards that as `mediaUrl`; there is no fallback to `path` or `filePath` here.
