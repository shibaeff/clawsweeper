## Summary

Keep open. Current `main` still does not implement this Discord `thread-reply` alias fix: `send` accepts `media`, `path`, and `filePath`, but `thread-reply` still reads only `media`, so the reported attachment drop remains plausible on `main`.

## Evidence

- **`send` and `thread-reply` still diverge on `main`:** `extensions/discord/src/actions/handle-action.ts` reads `media ?? path ?? filePath` for `send`, but `extensions/discord/src/actions/handle-action.guild-admin.ts` still reads only `media` in the `thread-reply` branch.
  - file: [extensions/discord/src/actions/handle-action.guild-admin.ts:364](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/discord/src/actions/handle-action.guild-admin.ts#L364)
  - command: `nl -ba extensions/discord/src/actions/handle-action.guild-admin.ts | sed -n '360,390p'; nl -ba extensions/discord/src/actions/handle-action.ts | sed -n '76,95p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Runtime path only forwards `mediaUrl`:** The Discord messaging runtime for `threadReply` reads only `mediaUrl` and passes that to `sendMessageDiscord`, so if `thread-reply` never maps `path` or `filePath` into `mediaUrl`, attachments will not be forwarded.
  - file: [extensions/discord/src/actions/runtime.messaging.ts:527](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/discord/src/actions/runtime.messaging.ts#L527)

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **No current regression coverage for the alias behavior:** Current Discord channel action tests on `main` only show `threadReply` target extraction/delegation coverage; there is no test covering `thread-reply` attachment aliases like `path` or `filePath`.
  - file: [extensions/discord/src/channel-actions.test.ts:125](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/discord/src/channel-actions.test.ts#L125)
