## Summary

Current `main` already changed Control UI/chat-history image handling so assistant/generated images are persisted as managed Gateway media references instead of replaying raw base64 image blocks through history. That removes the historical-image payload path that `agents/tool-images` would resize on `session:history`, and there is explicit regression coverage for the history response shape.

## Evidence

- **Assistant image replies are converted to managed blocks before transcript/history persistence:** `buildAssistantDisplayContentFromReplyPayloads` creates managed image blocks from reply media URLs, and the final chat-send path persists that content into the assistant transcript entry, then attaches the managed image records to the message.
  - file: [src/gateway/server-methods/chat.ts:301](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/gateway/server-methods/chat.ts#L301)
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Managed image blocks are URL refs, not inline base64:** `createManagedOutgoingImageBlocks` stores image data in managed records/files and returns chat-history blocks with `/api/chat/media/outgoing/.../full` URLs; `attachManagedOutgoingImagesToMessage` upgrades them to `history` retention tied to the transcript message.
  - file: [src/gateway/managed-image-attachments.ts:548](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/gateway/managed-image-attachments.ts#L548)
