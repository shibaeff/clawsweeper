## Summary

Keep open. Current `main` does not implement a `compat.clearThinking` option or emit `clear_thinking` on Z.AI `openai-completions` requests. The issue body's claim about current `enable_thinking` behavior appears stale, but the requested capability is still absent.

## Evidence

- **Compat reader only recognizes existing fixed keys:** `getCompat()` reads `supportsStore`, `supportsDeveloperRole`, `supportsReasoningEffort`, `maxTokensField`, `requiresToolResultName`, `requiresAssistantAfterToolResult`, `requiresThinkingAsText`, `thinkingFormat`, `openRouterRouting`, `vercelGatewayRouting`, `supportsStrictMode`, `requiresStringContent`, and `visibleReasoningDetailTypes`. There is no `clearThinking` field.
  - file: [src/agents/openai-transport-stream.ts:1529](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/openai-transport-stream.ts#L1529)
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **OpenAI completions payload builder never emits `clear_thinking`:** `buildOpenAICompletionsParams()` constructs the request body with `model`, `messages`, `stream`, `stream_options`, optional `store`, token limits, `temperature`, `tools`, `tool_choice`, `reasoning`, and `reasoning_effort`. It has no branch that writes `clear_thinking`.
  - file: [src/agents/openai-transport-stream.ts:1705](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/openai-transport-stream.ts#L1705)
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Z.AI compat is detected, but only as a special thinking format with reasoning-effort disabled:** The OpenAI-compat defaults mark Z.AI as `thinkingFormat: "zai"` and set `supportsReasoningEffort` false. There is no companion compat flag for preserved-thinking or `clear_thinking`.
  - file: [src/agents/openai-completions-compat.ts:72](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/openai-completions-compat.ts#L72)
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **No implementation or docs mention `clearThinking` or `clear_thinking`:** Repository search returned no matches for either token under `src`, `extensions`, or `docs`. That indicates the requested config surface and wire parameter are not present on `main`.
  - command: `rg -n "clearThinking|clear_thinking" src extensions docs`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **The issue body's `enable_thinking` description is stale relative to current main:** Current source only references `enable_thinking` in live-test error matchers, not in request-building code. That means the reported current snippet no longer matches `main`, but it does not prove the feature request was implemented.
  - file: [src/agents/models.profiles.live.test.ts:298](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/agents/models.profiles.live.test.ts#L298)
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)

  - file: [docs/gateway/config-tools.md:419](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/docs/gateway/config-tools.md#L419)
