## Summary

Keep open. Current `main` still has a single shared `agents.*.memorySearch.remote` config, passes that same remote block into fallback-provider activation, and does not expose a separate fallback model/remote object. The reported Gemini primary + OpenAI-compatible fallback use case is therefore not implemented as requested.

## Evidence

- **Config surface still only has shared remote + string fallback:** `src/config/types.tools.ts:363` defines `memorySearch.provider`, a single `memorySearch.remote` object, and `memorySearch.fallback` as a string. There is no `fallback.remote`, `fallbackModel`, or object-shaped fallback config.
