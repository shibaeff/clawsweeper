## Summary

Current `main` and the shipped `v2026.4.23` tag already handle the TUI silent/empty-final path by either falling back to streamed assistant text or reloading transcript-backed history, with regression tests covering the same behavior.

## Evidence

- **Empty final triggers history reload:** The TUI chat handler explicitly treats a `final` chat event with no `message` as a recovery path: it reloads history for the run, drops the transient assistant bubble, and finalizes the run state.
  - file: [src/tui/tui-event-handlers.ts:336](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/tui/tui-event-handlers.ts#L336)

- **Deferred reload is preserved across overlapping runs:** Regression tests cover both reloading history when a local run ends without a displayable final message and deferring that reload until a newer active run finishes.
  - file: [src/tui/tui-event-handlers.test.ts:622](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/tui/tui-event-handlers.test.ts#L622)
