## Summary

Keep open. Current `main` has backend session lifecycle fields and a manual `activeMinutes` cutoff, but the dashboard Sessions UI still renders a flat table with no status/completed/archive badge or column, and the only built-in filter is a user-entered recency threshold. The issue’s exact archived-transcript explanation may be partly outdated because `sessions.list` is store-backed and skips cron-run keys, but the requested dashboard distinction is not implemented.
