## Summary

Keep open. Current `main` at `aa21d4ea7e9aab034acbf0b13f8c3575057c5c54` still has the pre-PR cron fallback behavior: it gathers up to four descendant replies and concatenates them, and the regression test still asserts joined multi-descendant output. One unrelated piece from the PR (`src/shared/global-singleton.ts` type cast) is already on `main`, but the stated cron fix is not implemented.
