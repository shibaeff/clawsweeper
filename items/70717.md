## Summary

Current `main` has a targeted mitigation for startup-time bundled channel setup loading, but I did not find high-confidence proof that #70717's reported macOS gateway RSS regression is fully resolved. The code now skips legacy state-migration setup loads for disabled channels and avoids staging runtime deps during setup discovery, yet there is no direct RSS benchmark, reproduction result, or reporter confirmation showing the 700MB+ cold-start baseline and climb are fixed on `main`.
