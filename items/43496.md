## Summary

Current `main` has partial ACP control support, but not the agent-tool/session-action support requested here. ACP sessions can be controlled through `/acp` chat commands and `sessions_send` is documented for ACP follow-ups, yet the `subagents` tool remains native-subagent-only, there is no ACP session action tool in the agent tool registry, and `sessions_send` still just forwards raw gateway errors instead of adding ACP-specific completed/dead-session handling.

## Evidence

- **`subagents` is still subagent-only:** The tool schema and implementation only support `list`, `kill`, and `steer` for controlled subagent runs, with user-facing text explicitly describing spawned sub-agents only.
  - file: [src/agents/tools/subagents-tool.ts:23](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/tools/subagents-tool.ts#L23)
  - command: `nl -ba src/agents/tools/subagents-tool.ts | sed -n '1,260p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **No ACP action tool is registered for agents:** The core tool registry wires up `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, and `subagents`, but there is no ACP-specific session-control tool exposed to agent/tool callers.
  - file: [src/agents/openclaw-tools.ts:273](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/openclaw-tools.ts#L273)
