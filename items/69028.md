## Summary

Keep open. Current `main` still appears to have the reported silent-failure path: Feishu inbound media uses `mediaMaxMb` (default 30 MB), `saveMediaBuffer()` throws when the attachment exceeds that limit, and `resolveFeishuMediaList()` catches that error and only logs it before returning no media. The Feishu bot then continues building the agent request from an empty media list, so there is no user-facing failure notice and no explicit signal to the agent that the attachment was rejected.
