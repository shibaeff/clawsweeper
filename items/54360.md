## Summary

Current `main` and the latest shipped release both still carry `user-invocable: true` in `skills/gh-issues/SKILL.md`, while the runtime, tests, and public docs all treat `user-invocable` as a supported optional frontmatter key. The incompatibility described in PR #54360 does not reproduce from the checked-out codebase, so this PR is not fixing a current OpenClaw behavior regression.
