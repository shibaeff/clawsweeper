## Summary

Keep open. Current `main` added a lightweight local docs inventory (`pnpm docs:list`), but it does not implement the requested published `docs_map.md` with per-page H2/H3/H4 hierarchy, and this request belongs to repo docs infrastructure rather than ClawHub/plugin scope.

## Evidence

- **Current docs discovery feature is a local list command:** `package.json` wires `docs:list` to `node scripts/docs-list.js`. The script walks `docs/**/*.md` and prints only each page path, front-matter `summary`, and `read_when` hints; it does not parse document headings or generate a `docs_map.md` artifact.
  - file: [package.json:1354](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/package.json#L1354)
  - command: `nl -ba package.json | sed -n '1346,1362p'; nl -ba scripts/docs-list.js | sed -n '1,173p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Implementation gap vs requested artifact:** `scripts/docs-list.js` extracts front matter in `extractMetadata()` and emits console output from the markdown file list. There is no logic for collecting H2/H3/H4 headings or writing a sibling docs map file.
  - file: [scripts/docs-list.js:78](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/scripts/docs-list.js#L78)
  - command: `nl -ba scripts/docs-list.js | sed -n '78,173p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **No docs-map or llms map artifact present in the repo docs tree:** A direct filename search under `docs/` found no `docs_map.md`, `docs-map.md`, `llms.txt`, or similar sibling artifact that would satisfy the request.
  - command: `find docs -maxdepth 2 \( -name '*llms*' -o -name '*docs_map*' -o -name '*docs-map*' \) -print | sort`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **This is docs-site infrastructure, not plugin/ClawHub scope:** `docs/docs.json` is the Mintlify site config for `docs.openclaw.ai`, and `VISION.md` reserves ClawHub for plugin discovery/promotion. A generated docs navigation artifact for the official docs site fits repo docs tooling, not a third-party plugin close reason.
  - file: [docs/docs.json:1](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/docs/docs.json#L1)
