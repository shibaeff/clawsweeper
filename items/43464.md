## Summary

Current `main` already restores Telegram inbound media IPv4 fallback by threading Telegram transport fallback attempts into SSRF-guarded file downloads, and the changelog records that fix in the `2026.3.13` release.

## Evidence

- **Telegram media path now passes fallback dispatcher attempts into guarded media fetches:** `downloadAndSaveTelegramFile()` sends both `transport.sourceFetch` and `transport.dispatcherAttempts` into `fetchRemoteMedia()`, so inbound file downloads reuse the Telegram transport fallback plan instead of a single pinned-dispatcher attempt.
  - file: [extensions/telegram/src/bot/delivery.resolve-media.ts:235](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/telegram/src/bot/delivery.resolve-media.ts#L235)
  - command: `nl -ba extensions/telegram/src/bot/delivery.resolve-media.ts | sed -n '232,245p'`
- **Guarded media fetch retries across dispatcher attempts:** `fetchRemoteMedia()` builds an `attempts` list from `dispatcherAttempts` and retries `fetchWithSsrFGuard()` across those attempts until one succeeds or the retry policy stops.
  - file: [src/media/fetch.ts:127](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/media/fetch.ts#L127)
  - command: `nl -ba src/media/fetch.ts | sed -n '127,180p'`
- **Telegram transport exports IPv4 fallback policies for media downloads:** `resolveTelegramTransport()` creates fallback policies with `autoSelectFamily: false`, `dnsResultOrder: "ipv4first"`, and `forceIpv4: true`, then exports them through `dispatcherAttempts` for reuse by media fetches.
  - file: [extensions/telegram/src/fetch.ts:602](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/telegram/src/fetch.ts#L602)
  - command: `nl -ba extensions/telegram/src/fetch.ts | sed -n '599,617p'`
- **Regression coverage matches the reported failure mode:** Tests assert that Telegram media downloads export fallback dispatcher attempts, including IPv4-only attempts with `autoSelectFamily: false`, and that media download resolution passes those attempts into `fetchRemoteMedia()`.
  - file: [extensions/telegram/src/fetch.test.ts:478](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/telegram/src/fetch.test.ts#L478)
  - command: `nl -ba extensions/telegram/src/fetch.test.ts | sed -n '478,528p'; nl -ba extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts | sed -n '332,378p'`
