## Summary

Keep open. Current `main` still does not make the default workspace root follow `OPENCLAW_STATE_DIR`; the workspace resolver remains home-based, while state/config paths already honor `OPENCLAW_STATE_DIR`. The request is coherent and still relevant, and the PR discussion also surfaced a concrete correctness issue in the submitted patch (`~` expansion), so this is not a safe cleanup close.

## Evidence

There is no `OPENCLAW_STATE_DIR` branch in this function on current `main`.
