## Summary

Keep open. Current `main` already has the compact context-notice styling, but it does not implement the PR's core behavior change: `/usage` still computes context percentage from accumulated `inputTokens`, and the chat notice explicitly suppresses stale rows instead of falling back to `inputTokens` when `totalTokensFresh === false`.

## Evidence

- **`/usage` still uses accumulated input tokens:** `executeUsage()` computes `pct` from `input / ctx`, not from fresh `totalTokens` with a stale fallback.
