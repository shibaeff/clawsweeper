## Summary

Keep open. Current `main` still routes `ACTION_ASSIST` launches into Chat, and the docs describe that as the present behavior. The report is recent, concrete, and actionable, so none of the conservative auto-close reasons apply.

## Evidence

- **Main routing still hard-codes Chat:** `handleAssistantLaunch` always sets `_requestedHomeDestination` to `HomeDestination.Chat` and only manages chat draft / auto-send state. There is no branch that routes `source == "assist"` with `prompt == null` to Voice.
