## Summary

Keep open. Current `main` does not contain the browser session-persistence feature from PR #45371, and the managed `openclaw` browser still uses the same basic shutdown path the PR was trying to address. Docs also still position the `user` existing-session profile as the path when logged-in sessions matter, which is consistent with this feature not having landed on `main`.
