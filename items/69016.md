## Summary

Current `main` still routes CLI-backend sessions around the context-engine lifecycle. The CLI path goes straight to `runCliAgent()`, the CLI runner only fires generic hook events, transcript persistence just appends user/assistant messages, and docs only claim full context-engine lifecycle support for embedded Pi and the Codex harness. The linked `lossless-claw` issues strengthen that this is an unresolved core/plugin integration gap, not a stale report.

## Evidence

- **CLI backends are documented as a conservative non-ACP path:** The docs describe CLI backends as a "text-only fallback", a "safety net rather than a primary path", and explicitly say "CLI backends are not ACP," which matches a reduced lifecycle surface compared with embedded harnesses.
