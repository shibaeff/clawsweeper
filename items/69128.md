## Summary

Current `main` already contains the relevant Control UI/Gateway logic for this report: reconnect `hello` snapshots re-apply session defaults on the active socket, user-selected non-alias sessions are preserved instead of being overwritten on reconnect, `models.list` is global/config-backed rather than session-scoped, and the chat model picker now keeps the current/default model visible even when the active session row is temporarily missing. The fixing commit is on `main` after the latest tagged release, so this looks implemented but not yet shipped in `v2026.4.23`.

## Evidence
