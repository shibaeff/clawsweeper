## Summary

Keep open. Current `main` does not implement this PR's `reuseSession` feature, and the repo still documents and tests the opposite behavior for isolated cron jobs.

## Evidence

- **Runtime still forces fresh isolated cron sessions:** `runCronIsolatedAgentTurn` still passes `forceNew: input.job.sessionTarget === "isolated"` into `resolveCronSession`, so isolated cron runs always mint a fresh session on current `main`.
