## Summary

Keep open. Current `main` still intentionally creates a fresh session for isolated cron runs, and the public cron types/schema/CLI/docs do not expose the proposed `reuseSession` opt-in. There is a supported workaround via persistent `current` or `session:<id>` targets, but that is not the same behavior this PR adds.
