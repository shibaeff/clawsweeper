## Summary

Keep open. Current `main` still does not expose a trusted inbound voice/message-kind flag to the agent context or `openclaw.inbound_meta.v2`, and the remaining audio hint can disappear once transcription is present.

## Evidence

- **Inbound meta prompt has no message-kind field:** `src/auto-reply/reply/inbound-meta.ts:153` builds `openclaw.inbound_meta.v2` with `account_id`, `channel`, `provider`, `surface`, `chat_type`, and optional `response_format`, but no `is_voice`, `message_type`, or equivalent field.
  - file: [src/auto-reply/reply/inbound-meta.ts:153](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/auto-reply/reply/inbound-meta.ts#L153)
  - command: `nl -ba src/auto-reply/reply/inbound-meta.ts | sed -n '135,176p'`
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Telegram inbound context payload omits voice/message-kind metadata:** `extensions/telegram/src/bot-message-context.session.ts:322` finalizes the inbound context with fields like `Body`, `BodyForAgent`, `MediaType`, `MediaTypes`, `Sticker`, `WasMentioned`, `MessageThreadId`, and routing fields, but no voice-specific or message-type field.
  - file: [extensions/telegram/src/bot-message-context.session.ts:322](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/extensions/telegram/src/bot-message-context.session.ts#L322)
  - command: `nl -ba extensions/telegram/src/bot-message-context.session.ts | sed -n '320,385p'`
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Base inbound context type has no slot for voice/message-kind:** `src/auto-reply/templating.ts:31` defines `MsgContext`. It includes media fields (`MediaType`, `MediaTypes`, `Transcript`) and routing/thread metadata, but no `is_voice`, `message_type`, or similar trusted field that channels could populate.
  - file: [src/auto-reply/templating.ts:31](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/auto-reply/templating.ts#L31)

That means the agent may not reliably infer 'voice' from prompt text after STT.
  - file: [src/auto-reply/media-note.ts:126](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/auto-reply/media-note.ts#L126)
