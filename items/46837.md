## Summary

Keep open. Current `main` still does not warn in `openclaw doctor` when `agents.defaults.memorySearch.extraPaths` points at missing paths, so this is not implemented on `main`. The PR also has unresolved review feedback around `~` handling: runtime memory path normalization is workspace-relative for any non-absolute path, so the proposed doctor check needs to match that behavior before it is ready to land.

## Evidence

In the current file, the only `resolveUserPath` usage is for validating a local model path, not memory search extra paths.
