## Summary

Keep open. Current `main` still dereferences malformed `{ type: "text" }` tool-result blocks in both the tool-result char estimator and context-pruning paths, so this PR’s defensive guards are not implemented on `main`. The PR body’s original repro path (`tool.execute()` returning `undefined`) is partly outdated because the adapter now normalizes that case, but malformed persisted/history blocks can still survive sanitization and reach the crashing code paths.
