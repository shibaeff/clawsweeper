## Summary

Keep open. Current `main` still omits `mediaLocalRoots` in Mattermost `handleAction`, so the PR addresses a real gap that is not already implemented on `main`.

## Evidence

- **Mattermost handleAction still drops mediaLocalRoots:** `extensions/mattermost/src/channel.ts` destructures `handleAction` as `{ action, params, cfg, accountId }` and calls `sendMessageMattermost(...)` without a `mediaLocalRoots` option.
  - file: [extensions/mattermost/src/channel.ts:113](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/mattermost/src/channel.ts#L113)
  - command: `nl -ba extensions/mattermost/src/channel.ts | sed -n '100,223p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Mattermost runtime upload path only honors explicit mediaLocalRoots:** `sendMessageMattermost` passes `opts.mediaLocalRoots` into `loadOutboundMediaFromUrl(...)`, so omitted roots in `handleAction` mean the runtime falls back to default local-root behavior.
  - file: [extensions/mattermost/src/mattermost/send.ts:428](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/mattermost/src/mattermost/send.ts#L428)
  - command: `nl -ba extensions/mattermost/src/mattermost/send.ts | sed -n '420,445p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Default local-root path checking rejects workspace-* dirs when roots are omitted:** `assertLocalMediaAllowed` explicitly rejects paths under sibling `workspace-*` directories when `localRoots` is undefined, which matches the PR's reported failure mode for agent workspace uploads.
  - file: [src/media/local-media-access.ts:52](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/media/local-media-access.ts#L52)
