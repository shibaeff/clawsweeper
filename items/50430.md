## Summary

Current `main` no longer blocks gateway readiness on QMD startup. Gateway startup now logs `ready (...)` immediately after the listener binds, while sidecars run asynchronously and QMD memory initialization is launched via `setImmediate` without being awaited. The startup sequencing fix is also called out in the `2026.4.12` changelog.

The gateway startup path now logs `ready (...)` during post-attach startup (`src/gateway/server-startup-log.ts:32`) before sidecars are allowed to finish, and the QMD memory startup is kicked off asynchronously via `setImmediate` without being awaited (`src/gateway/server-startup-post-attach.ts:318-330`, `src/gateway/server-startup-post-attach.ts:471-521`). The changelog records the shipped startup sequencing fix in `v2026.4.12` (`CHANGELOG.md:743-771`). If you still hit a startup stall on a newer build, please open a fresh issue with current logs, config, and exact version details.
