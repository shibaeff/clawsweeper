## Summary

Keep open. Current `main` still does not ship Arabic Control UI locale support: `ui/src/i18n/lib/types.ts` excludes `"ar"`, `ui/src/i18n/lib/registry.ts` does not register or auto-resolve Arabic, there is no `ui/src/i18n/locales/ar.ts`, the English language picker map has no Arabic label, and `ui/src/i18n/lib/translate.ts` still has no locale-driven RTL document direction handling. This is therefore not implemented on `main`, and the PR should not be conservatively closed.
