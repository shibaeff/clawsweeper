## Summary

Keep open. Current `main` still does not implement WhatsApp inbound reaction forwarding, so this PR remains substantive work rather than something already shipped or obsolete.

## Evidence

- **Inbound message type still lacks reaction metadata:** `WebInboundMessage` on `main` does not define `reactionEmoji` or `reactionMessageId`; the type ends after media/context fields and batching flags.
  - file: [extensions/whatsapp/src/inbound/types.ts:54](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/whatsapp/src/inbound/types.ts#L54)
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Inbox monitor only subscribes to message upserts and connection updates:** The WhatsApp inbox monitor registers listeners for `messages.upsert` and `connection.update` only. There is no `messages.reaction` listener or reaction handler on current `main`.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:699](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/whatsapp/src/inbound/monitor.ts#L699)
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Constructed inbound payload has no reaction fields:** The queued `WebInboundMessage` built in the inbound path includes body, sender, reply, media, and dedupe metadata, but no reaction-specific payload. Current `main` therefore cannot forward reaction events to `onMessage`.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:564](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/whatsapp/src/inbound/monitor.ts#L564)
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **WhatsApp reactions are supported outbound, not inbound:** The send API already has `sendReaction(chatJid, messageId, emoji, fromMe, participant?)`, which shows reaction support exists for outbound actions while the inbound forwarding requested by this PR is still missing.
  - file: [extensions/whatsapp/src/inbound/send-api.ts:109](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/whatsapp/src/inbound/send-api.ts#L109)
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **No matching inbound reaction docs/tests found on main:** Searches for `messages.reaction`, `reactionEmoji`, and `reactionMessageId` in WhatsApp tests/docs returned no matches, which is consistent with the feature not being present on current `main`.
  - command: `rg -n "reaction.*dedupe|reactionEmoji|reactionMessageId|messages\.reaction" extensions/whatsapp/src/*.test.ts extensions/whatsapp/src/**/*.test.ts docs/channels/whatsapp.md`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
