## Summary

Keep open. Current `main` still forces Docker onboarding with no documented skip parameter, and the issue is coherent: the Docker docs already describe persistent mounted config that survives container replacement, which matches the reporter's use case.

## Evidence

- **Setup script still unconditionally runs onboarding:** `scripts/docker/setup.sh` prints the onboarding banner and then always executes `run_prestart_cli onboard --mode local --no-install-daemon`. There is no conditional branch or skip flag around that call.
