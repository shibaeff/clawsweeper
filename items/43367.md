## Summary

Keep open. Current main has partial mitigations for parts of this report, but not strong evidence that end-to-end multi-agent orchestration is reliably fixed. The CLI `agents add` path now uses snapshot/hash-checked config replacement, OAuth refresh contention has explicit cross-agent coverage, and detached CLI runs are tracked via the task ledger. But session-lock timeouts are still an active error path, `openclaw agent` still falls back from gateway to embedded on failure, detached-work cleanup is still documented as best-effort, and the gateway `agents.create` path still writes config without conflict handling.

- The issue bundles multiple failures together. Some sub-problems are clearly improved on main, but the remaining lock/fallback/detached-work behavior is still enough that closing the whole issue would be premature.
- `gh` inspection was unavailable in this environment because `GH_TOKEN` was not set, so this review relied on checked-out code/docs rather than live PR state.
