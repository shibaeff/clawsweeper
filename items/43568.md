## Summary

Keep open. Current `main` supports proxy overrides only at the model-provider request layer, not on auth profiles. Auth-profile config/types do not define `proxyUrl`, and auth resolution returns only credential material, so same-provider multi-account proxy isolation is not implemented on `main`.

## Evidence

- **Auth-profile schema has no proxy field:** `AuthProfileCredential` only models API-key, token, and OAuth credential fields; there is no `proxyUrl` or other transport field on stored auth profiles.
  - file: [src/agents/auth-profiles/types.ts:18](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/auth-profiles/types.ts#L18)

- **Config-side auth profile metadata has no proxy field:** `AuthProfileConfig` only allows `provider`, `mode`, `email`, and `displayName`, so `auth.profiles` cannot declare per-profile proxy routing either.
  - file: [src/config/types.auth.ts:1](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/types.auth.ts#L1)
