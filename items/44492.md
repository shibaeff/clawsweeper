## Close Comment

Closing from Codex review. The report was against `2026.3.11` / `2026.3.23-2`, but current `main` already contains the Telegram topic-routing fixes for this failure class. The shipped changelog explicitly covers restoring Telegram forum-topic ACP follow-up routing (`CHANGELOG.md:1514`), pinning delayed exec completions to the originating Telegram topic even when session routing drifts (`CHANGELOG.md:988`), and stopping heartbeat/exec-event turns from poisoning shared-session routing metadata (`CHANGELOG.md:700` and `CHANGELOG.md:702`). Current tests also lock this behavior in: `src/agents/subagent-announce.format.e2e.test.ts:1322` verifies forum-topic completion delivery keeps the topic thread, `src/agents/bash-tools.test.ts:772` preserves exec-event topic delivery context, and `src/infra/heartbeat-runner.ghost-reminder.test.ts:536` keeps exec-event replies pinned to the original topic after route drift. Based on that evidence, this looks implemented on main rather than an open maintainer task.
