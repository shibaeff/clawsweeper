## Summary

Keep open. Current `main` does not implement the proposed outbound-session prewarm, and the PR as submitted appears incomplete: it would advance first-turn session state too early and would not honor per-channel skill filters on the prewarm path.

## Evidence

- **Current main has no outbound prewarm step:** `ensureOutboundSessionEntry` only records session metadata, then returns. There is no dynamic import of `session-updates.runtime`, no `prewarmMirroredSession`, and no follow-up snapshot warming on outbound send.
  - file: [src/infra/outbound/outbound-session.ts:135](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/infra/outbound/outbound-session.ts#L135)
  - command: `nl -ba src/infra/outbound/outbound-session.ts | sed -n '135,220p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **First-turn behavior is keyed off `systemSent`:** Reply execution computes `isFirstTurnInSession` from `isNewSession || !currentSystemSent`, and uses that to decide first-turn-only behavior such as group intro injection. Separately, `ensureSkillSnapshot` sets `systemSent: true` when called with `isFirstTurnInSession: true`. Prewarming via that path before the first callback would therefore consume first-turn state early.
  - file: [src/auto-reply/reply/get-reply-run.ts:298](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/auto-reply/reply/get-reply-run.ts#L298)
  - command: `nl -ba src/auto-reply/reply/get-reply-run.ts | sed -n '288,570p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Snapshot generation is filter-sensitive on main:** `ensureSkillSnapshot` accepts `skillFilter` and refreshes snapshots when the stored filter does not match. The normal reply path passes `opts?.skillFilter` into `ensureSkillSnapshot`, so any prewarm path that omits the filter will not match filtered-agent behavior and may still force a rebuild on first callback.
  - file: [src/auto-reply/reply/session-updates.ts:96](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/auto-reply/reply/session-updates.ts#L96)
  - command: `nl -ba src/auto-reply/reply/session-updates.ts | sed -n '96,220p'; nl -ba src/auto-reply/reply/get-reply-run.ts | sed -n '540,560p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Main test coverage does not cover the proposed regression path:** The current outbound-session test only verifies that metadata is persisted into the owning session store. There is no test on `main` covering proactive outbound send followed by first callback prewarming.
  - file: [src/infra/outbound/outbound-session.test.ts:412](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/infra/outbound/outbound-session.test.ts#L412)
  - command: `nl -ba src/infra/outbound/outbound-session.test.ts | sed -n '412,490p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **PR review already identified substantive correctness gaps:** The PR discussion includes review comments flagging two concrete problems: prewarm would mark first-turn state as consumed before the first inbound callback, and the prewarm path does not pass `skillFilter`, defeating the intended cold-start avoidance for filtered agents. Those concerns are consistent with the current mainline code paths above.
