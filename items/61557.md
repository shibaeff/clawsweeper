## Summary

Close as ClawHub-scope: OpenClaw core already exposes `before_install` for gateway-controlled skill/plugin install policy, while ClawHub skill download/install and external installer vetting are documented as separate ClawHub-owned flows.

## Evidence

- **Vision puts skill distribution and security review in ClawHub, not core:** `VISION.md` says optional capability should usually ship as plugins, bundle-style plugins package skills, and plugin discovery/provenance/security review live in ClawHub; new skills should be published through ClawHub first.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/VISION.md#L54)
  - command: `nl -ba VISION.md | sed -n '50,90p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **ClawHub docs define skill install as a separate registry flow:** The public ClawHub docs describe `openclaw skills install` and the separate `clawhub` CLI as ClawHub skill download/install flows, including that the standalone `clawhub` CLI installs into `./skills` and OpenClaw picks those up later.
  - file: [docs/tools/clawhub.md:10](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/tools/clawhub.md#L10)
  - command: `nl -ba docs/tools/clawhub.md | sed -n '8,120p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **OpenClaw already has an install-policy hook for installs it owns:** `docs/plugins/hooks.md` documents plugin hook `before_install` as the lifecycle hook that inspects skill or plugin install scans and can block the install after the built-in scan.
  - file: [docs/plugins/hooks.md:103](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/plugins/hooks.md#L103)
  - command: `nl -ba docs/plugins/hooks.md | sed -n '96,116p'; nl -ba docs/plugins/hooks.md | sed -n '206,216p'`

- **Gateway-backed skill installs already call `before_install` with skill metadata:** `scanSkillInstallSourceRuntime` runs `runBeforeInstallHook` for `targetType: "skill"` and `requestKind: "skill-install"`, passing the skill install id/spec before allowing the install path to continue.
  - file: [src/plugins/install-security-scan.runtime.ts:875](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/plugins/install-security-scan.runtime.ts#L875)

- **Tests prove existing hook blocks gateway-managed skill installs:** `src/agents/skills-install.test.ts` verifies `before_install` receives `targetType: "skill"`, can add findings, and can block the install with an enterprise policy reason.
  - file: [src/agents/skills-install.test.ts:206](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/skills-install.test.ts#L206)
