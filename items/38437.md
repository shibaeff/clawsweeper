## Summary

Current `main` still treats a bundled plugin and a distinct global/user-installed plugin with the same id as a real duplicate, logs a warning for it, and by default gives bundled precedence unless the global install is explicitly tracked. That does not match this issue's requested behavior of quietly loading only the user copy by default.

## Evidence

- **Docs define bundled > global precedence and say duplicate drops are still logged:** The plugin manifest docs say duplicate ids keep only the highest-precedence manifest, with precedence `config-selected`, then `bundled`, then `global install`, then `workspace`, and explicitly note that duplicate drops are logged for Doctor/startup diagnostics.
  - file: [docs/plugins/manifest.md:644](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/plugins/manifest.md#L644)
