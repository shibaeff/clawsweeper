## Summary

Current `main` does not implement configurable provider-level retry for model API calls. The provider config type/schema/docs still omit `models.providers.*.retry`, the transport path still calls provider APIs directly without a retry runner, and the existing retry-policy surface on `main` is channel-focused. This PR is therefore not implemented on `main`, and the open PR should stay open rather than be conservatively closed.
