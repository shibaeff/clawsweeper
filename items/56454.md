## Summary

Keep open. Current `main` still hardcodes a 30s timeout in the shared `image` tool and the media-understanding runtime aborts on that deadline, so the bug this PR targets is not implemented on `main`. At the same time, OpenClaw already has a `tools.media.image.timeoutSeconds` config surface for media understanding, so whether this fix should merge as-is or be retargeted onto the existing timeout surface needs maintainer judgment rather than conservative cleanup closure.
