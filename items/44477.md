## Summary

Current `main` already supports the requested outcome: multiple credentials per provider with automatic rotation on rate-limit failures. OpenClaw implements this through auth profiles plus multi-key env support, not the exact single-JSON-array shape proposed in the issue.

## Close Comment

Closing after Codex review: current `main` already implements this capability, although the interface is auth-profile based rather than a single JSON array field. Multiple provider credentials can coexist in `auth-profiles.json` and are ordered/rotated via `src/agents/auth-profiles/order.ts:66` and `src/agents/model-auth.ts:497`. For API-key env setups, `src/agents/live-auth-keys.ts:29` collects `<PROVIDER>_API_KEYS`, `<PROVIDER>_API_KEY`, and numbered `<PROVIDER>_API_KEY_*` entries, and `src/agents/api-key-rotation.ts:40` retries the next key on rate-limit failures. The user-facing behavior is also documented in `docs/concepts/model-providers.md:43`, `docs/concepts/model-failover.md:57`, and `docs/gateway/authentication.md:146`. This shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
