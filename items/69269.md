## Summary

Keep open. Current `main` still has the two invariant gaps described in the issue: reset hooks do not pass `messageProvider`, and compaction accounting only refreshes cached totals when `tokensAfter > 0`, so zero-token compactions can leave stale `totalTokens`/cost state behind. The discussion also points to an intended coordinated fix, and that work does not appear landed on `main` yet.

There is no normalized provider in the reset hook context.
