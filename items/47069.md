## Summary

Keep open. Current `main` still routes any non-empty `/model` argument through `sessions.patch`, so the reported Web UI bug remains plausible. The proposed PR direction is also not strong enough for conservative cleanup because its unconditional `status|list|info` reservation conflicts with the current free-form `/model <provider/model or id>` contract and matches the regression concern already raised in PR review.

## Evidence
