## Summary

Keep open. Current `main` still routes non-empty `/model` arguments through `sessions.patch`, so `/model status` is still plausibly broken in the Web UI. The PR addresses a real inconsistency with the agent-side `/model status` and legacy `/model list` UX, but its current reserved-word approach would also block selecting any real model whose id is exactly `status`, `list`, or `info`, which conflicts with the existing `model id (provider/model or id)` contract.

## Evidence

## Risks / Open Questions
