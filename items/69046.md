## Summary

Keep open. Current `main` still lacks any caller-principal field on `ChannelOutboundContext`, and the outbound delivery path only uses requester identity for media-access policy before constructing the adapter context. In contrast, the shared `message` action path has an explicit trusted requester identity seam and enforcement hook, which supports the report that raw outbound sends still do not carry equivalent authz binding. The issue body is also concrete and actionable, including a third-party plugin workaround (`@skavinski/openclaw-zoho`) and a minimal repro.
