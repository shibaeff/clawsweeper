## Summary

Current `main` does not have `openclaw configure` write `session.dmScope`; that defaulting now happens in onboarding, and the stale main-route follow-up described in the issue is also handled on current `main`.

## Evidence

- **`openclaw configure` path does not touch `session.dmScope`:** The configure wizard clones the existing config, optionally sets `gateway.mode` to `local`, and then only updates the selected sections (`workspace`, `model`, `web`, `gateway`, `channels`, `plugins`, `skills`, `daemon`, `health`). There is no `session.dmScope` write in this flow.
  - file: [src/commands/configure.wizard.ts:469](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/commands/configure.wizard.ts#L469)
