## Summary

Current `main` already contains Google/Gemini-specific tool-schema normalization and OpenAI-compatible replay/transport safeguards that cover the reported `google/gemini-2.5-flash` 400 path for sub-agents, and that implementation was already shipped in `v2026.4.23`.

## Evidence

- **Sub-agent runs pass the active model provider into tool creation:** The embedded sub-agent path calls `createOpenClawCodingTools` with `modelProvider: params.model.provider`, so a spawned `google/gemini-2.5-flash` child gets Google-specific tool normalization rather than a generic inventory.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:607](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/pi-embedded-runner/run/attempt.ts#L607)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Tool schemas are normalized before they reach the model transport:** `createOpenClawCodingTools` always normalizes tool JSON Schemas before handing them to pi-ai, explicitly noting Gemini-specific keyword stripping via the model-provider-aware normalization path.
  - file: [src/agents/pi-tools.ts:697](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/pi-tools.ts#L697)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Google tool inventory regression coverage exists:** There is an explicit regression test that a Google tool inventory contains no `GEMINI_UNSUPPORTED_SCHEMA_KEYWORDS`, which matches the discussion’s suspicion that the failure was schema-compat related rather than an API-key/config problem.
  - file: [src/agents/pi-tools.create-openclaw-coding-tools.test.ts:425](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/pi-tools.create-openclaw-coding-tools.test.ts#L425)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Sessions tool schemas are shaped for Gemini compatibility:** Current tests assert the sessions tool family uses Gemini-compatible schema shapes such as `number` instead of `integer`, which is directly relevant to sub-agent flows that expose OpenClaw session tools.
  - file: [src/agents/openclaw-tools.sessions.test.ts:171](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/openclaw-tools.sessions.test.ts#L171)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **OpenAI-compatible replay fallback now applies Gemini safeguards:** For `openai-completions`, replay fallback enables strict tool-call-id sanitization and Gemini turn validation; Google routes also get thought-signature sanitization and assistant-first ordering repair to avoid malformed Gemini payloads.
  - file: [src/agents/transcript-policy.ts:80](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/transcript-policy.ts#L80)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **The exact Google OpenAI-compatible route is covered by transport tests:** Current transport tests exercise a Google `openai-completions` model at `https://generativelanguage.googleapis.com/v1beta/openai` and verify Gemini `thought_signature` handling on that route.
  - file: [src/agents/openai-transport-stream.test.ts:1736](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/openai-transport-stream.test.ts#L1736)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **This implementation was already shipped in the latest release:** The same Google OpenAI-compatible transport coverage is present in tag `v2026.4.23`, so this was not just added after the current release.
  - command: `git grep -n "Gemini thought_signature round-trip on OpenAI-compatible completions" v2026.4.23 -- src/agents/openai-transport-stream.test.ts`
