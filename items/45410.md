## Summary

Keep open. Current `main` already supports model-specific matching for tool policy through exact `provider/model` keys under `tools.byProvider`, and it now has a generic async `requireApproval` hook path for tool calls. But the reported request is not fully implemented as a declarative config feature: there is still no `tools.byModel`, no `tools.byProvider.requireApproval`, and no `escalateToModel` field in the tool policy config/schema.

## Risks / Open Questions

- The issue title mixes two scopes: model-targeted selection is already present via `tools.byProvider` exact `provider/model` keys, but declarative approval semantics are still missing.
- The new hook-based approval path may reduce urgency, but current config/docs still do not give end users a built-in `tools.byModel` or `requireApproval` policy surface.
