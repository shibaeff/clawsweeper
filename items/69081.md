## Summary

Current `main` does not show a safe fix for this TUI desync report. The watchdog is still only re-armed from chat delta handling, not from tool events, and the reconnect path still reloads history without clearing `activeChatRunId` or watchdog state. That leaves the reported desync behavior plausible on current code, so this item should stay open.

## Evidence

- **Watchdog still delta-only:** `handleChatEvent` sets `streaming` and calls `armStreamingWatchdog(evt.runId)` on chat `delta`, but `handleAgentEvent` processes `stream === "tool"` without any watchdog re-arm. That matches the issue's Bug 1 root-cause sketch rather than disproving it.
