## Summary

Keep open. Current `main` added pre-load repair for malformed JSONL session files, but it still does not repair or rewrite orphaned `tool_result` / `tool_use` pairing back to disk on session load. The pairing fix remains an in-memory replay sanitization step, so the issue's requested self-healing-on-load behavior is not implemented on `main`.

## Evidence

- **Pre-load session repair only drops malformed JSON lines:** `repairSessionFileIfNeeded` reads the JSONL file, counts lines that fail `JSON.parse`, and rewrites the file only when malformed lines were dropped. It does not inspect or repair message-level `tool_result` / `tool_use` pairing.
  - file: [src/agents/session-file-repair.ts:41](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/session-file-repair.ts#L41)
  - command: `nl -ba src/agents/session-file-repair.ts | sed -n '1,140p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Load path calls only the malformed-line repair before opening the session:** The embedded runner invokes `repairSessionFileIfNeeded` before loading the session manager. There is no additional load-time rewrite step here that applies `sanitizeToolUseResultPairing` to the persisted transcript.
