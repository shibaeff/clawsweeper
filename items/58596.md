## Summary

The issue asks for a team-specific GitHub posting header and related resource-flow behavior. Current `main` treats agent identity as workspace/delegate configuration and GitHub operations as a skill/plugin surface, so this belongs in a ClawHub-published GitHub skill/plugin rather than OpenClaw core.
