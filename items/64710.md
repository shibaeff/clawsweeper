## Summary

Keep open. Current `main` includes a fix for a different timeout path (#70831 / guarded Undici body-timeout propagation), but this issue is specifically about the 120-second `LLM idle timeout` path. The idle-timeout watchdog and user-facing idle-timeout error are still present on `main`, and a maintainer comment on April 25, 2026 explicitly said this should stay open unless a retest on a build containing #70831 proves the issue is gone.
