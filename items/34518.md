## Summary

Current main already ships a different, documented solution for `delivery.mode="none"`: isolated cron still resolves the chat target so the agent can use the shared `message` tool, while runner fallback delivery stays disabled unless `deliveryRequested` is true. This PR's change would regress the current contract rather than fix a still-open bug on main.
