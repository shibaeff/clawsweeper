## Summary

Keep open. Current `main` still does not implement the PR’s raw-stdout/tool-result handling: cron retry is still driven by assistant payload text, and cron output recovery still ignores successful `toolResult` transcript text.

## Evidence

- **Interim-ack retry still has no raw-stdout/tool-result escape hatch:** `executeCronRun` still derives `interimText` from `resolveCronPayloadOutcome(...)` and re-prompts when `isLikelyInterimCronMessage(interimText)` is true. There is no special-case for raw-stdout jobs or for runs that already produced a successful `toolResult`.
  - file: [src/cron/isolated-agent/run-executor.ts:343](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/isolated-agent/run-executor.ts#L343)

- **Cron output recovery still ignores transcript tool results:** `resolveCronPayloadOutcome` still computes `outputText` only from assistant payload text or `finalAssistantVisibleText`; it does not read the session transcript or recover the latest successful `toolResult` text.
