## Summary

Keep open. Current `main` still lacks the TUI reconnect UX from this PR, and the macOS remote-probe safety fix discussed in review is still not present, so the branch is not superseded by shipped code.

## Evidence

- **Generic TUI disconnects still fall back to idle:** `resolveGatewayDisconnectState()` on current `main` still returns `activityStatus: "idle"` for non-pairing disconnects. That means the PR's reconnect hint/status change is not implemented in core.
