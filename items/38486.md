## Summary

Current main already supports disabling ambient Ollama auto-discovery with `plugins.entries.ollama.config.discovery.enabled: false` and still honors the legacy `models.ollamaDiscovery.enabled: false` alias. The discovery path returns `null` before probing/building the provider when disabled, and this shipped in `v2026.4.22`.

## Close Comment

Closing from a Codex review: this request is already implemented on current `main` and was shipped in `v2026.4.22`.

You can disable ambient Ollama discovery with `plugins.entries.ollama.config.discovery.enabled: false`; the bundled plugin declares that config in `extensions/ollama/openclaw.plugin.json:27`, and the discovery path short-circuits before probing/building Ollama in `extensions/ollama/src/discovery-shared.ts:105`. There is also a regression test in `extensions/ollama/index.test.ts:184` that verifies discovery returns `null` and `buildOllamaProvider` is not called when disabled. The release note for this shipped behavior is in `CHANGELOG.md:207`.

There is also a legacy-compatible alias at `models.ollamaDiscovery.enabled` still read by the runtime.
