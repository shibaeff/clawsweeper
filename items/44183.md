## Summary

Current `main` already contains a Discord mention-metadata hydration fallback that covers the reported `no-mention` behavior when thread messages include mention syntax but arrive with empty mention arrays, and it has a regression test for that exact case.

I could not verify a released tag containing this from the local checkout, so `fixedRelease` is left `null`, but the main-branch proof commit here is `4f4d2ef1dffd27fc7486850185af16429061606d`. If this still reproduces on a build newer than current `main`, please open a fresh issue with the raw Discord payload and logs.
