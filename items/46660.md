## Summary

Keep open. Current `main` still does not implement an `agent_to_agent_turn` hook or equivalent A2A-turn visibility surface, and the stated use case matches OpenClaw's documented direction of extending the plugin API when plugins cannot express a capability yet. No allowed close reason applies.

## Evidence

- **A2A flow still only exposes the final announce send:** In current `main`, `runSessionsSendA2AFlow` performs the ping-pong loop internally and only calls `callGateway({ method: "send" })` after that loop for the final announce. There is no hook emission inside the intermediate-turn loop, so the PR's requested behavior is not already implemented.
