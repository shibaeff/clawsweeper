## Summary

Current `main` does not contain the proposed README section, so this PR is not `implemented_on_main`. It is also not a clean fit for the other close buckets. However, current docs now establish a policy tension: README still points users to ClawHub as the skills registry, and the new community-plugin docs say ClawHub is the canonical discovery surface and explicitly discourage docs-only discoverability PRs. That makes this a maintainer/product decision, not a conservative auto-close.
