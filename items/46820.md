## Summary

Current `main` has already moved `openclaw models list` off the old side-effecting discovery path: it no longer syncs `models.json` during listing, uses read-only auth/model discovery, and has regression tests covering the exact slow-path concerns from this report.

## Evidence

- **`models list` no longer forces `models.json` sync:** Current `modelsListCommand` loads config, then either uses a configured-entry registry path for default listings or a narrower registry path for `--all`; the old unconditional `ensureOpenClawModelsJson(...)` call is gone from `main`.
  - file: [src/commands/models/list.list-command.ts:50](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/commands/models/list.list-command.ts#L50)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
- **Default listing is narrowed to configured entries:** For non-`--all` invocations, current code uses `loadConfiguredListModelRegistry(...)`, which resolves only configured entries instead of enumerating the full registry.
  - file: [src/commands/models/list.list-command.ts:80](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/commands/models/list.list-command.ts#L80)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
- **Auth/model discovery is now read-only:** Both registry loaders call discovery with `{ readOnly: true }`, and `discoverAuthStorage` skips legacy `auth.json` scrubbing/writes in read-only mode. That directly removes write/sync side effects from `models list` discovery.
  - file: [src/commands/models/list.registry.ts:111](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/commands/models/list.registry.ts#L111)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
- **Regression coverage matches the reported failure mode:** Tests now assert that `models list` runs without auth.json sync, does not enumerate all registry models by default, and does not persist `models.json` as a side effect.
  - file: [src/commands/models.list.e2e.test.ts:299](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/commands/models.list.e2e.test.ts#L299)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
