## Summary

Current `main` already avoids the deprecated `spawn(..., { shell: true })` path in the shared Windows exec helper used by Gateway-adjacent startup flows, and the changelog shows that fix shipped in `v2026.2.14`.

## Evidence

- **Shared Windows exec path no longer enables shell spawning:** `resolveChildProcessInvocation()` rewrites Windows `.cmd`/`.bat` launches to `cmd.exe /d /s /c ...`, and `shouldSpawnWithShell()` is hard-coded to return `false`, so the deprecated Node 24 `shell: true` + argv path is not used by `runCommandWithTimeout()` on current `main`.
