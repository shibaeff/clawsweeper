## Summary

Current main has targeted CJK/trigram handling and tests, but that support is still explicitly tied to the opt-in `trigram` FTS tokenizer while the documented default remains `unicode61`. Given the report is about inconsistent Chinese keyword hits on a stock 2026.3.8 setup, and this review was read-only with no live reproduction allowed, the evidence is not strong enough to conclude the bug is already fixed or obsolete.

## Risks / Open Questions
