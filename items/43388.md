## Summary

Current `main` has only partial overlap with PR #43388. Feishu already sends `.opus/.ogg` as `msg_type: "audio"`, but the hardening this PR proposes is not present on `main`: there is no mp3/m4a/wav/aac/flac/webm-to-ogg/opus transcode path, no duration probing passed into Feishu uploads, and no stripping of internal `[[tts]]` tags in Feishu reply dispatch or shared outbound normalization.
