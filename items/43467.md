## Summary

Keep open. Current `main` still has the reported cosine-similarity bug in the relocated memory-host SDK code, and a read-only reproduction on `main` returns inflated scores for different-length vectors. The PR is stale in file paths, not in substance.

## Evidence

- **Current implementation still truncates norms to shared length:** `cosineSimilarity` computes `len = Math.min(a.length, b.length)` and only accumulates `normA` and `normB` inside that shared-length loop; there are no follow-up loops for remaining dimensions.
  - file: [packages/memory-host-sdk/src/host/internal.ts:492](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/packages/memory-host-sdk/src/host/internal.ts#L492)
  - command: `nl -ba packages/memory-host-sdk/src/host/internal.ts | sed -n '492,510p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Bug still affects live search fallback:** When sqlite-vec is unavailable, memory search scores each candidate with `cosineSimilarity(params.queryVec, chunk.embedding)`, so the buggy helper is still on the current execution path.
  - file: [extensions/memory-core/src/memory/manager-search.ts:216](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/extensions/memory-core/src/memory/manager-search.ts#L216)
  - command: `nl -ba extensions/memory-core/src/memory/manager-search.ts | sed -n '210,220p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Dimension mismatches are only guarded on the sqlite-vec table path:** The vector-table path tracks one embedding dimension and drops/recreates the vec table on dimension changes, which protects sqlite-vec storage only. The fallback path above does not enforce equal lengths before calling `cosineSimilarity`.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:258](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/extensions/memory-core/src/memory/manager-sync-ops.ts#L258)
  - command: `nl -ba extensions/memory-core/src/memory/manager-sync-ops.ts | sed -n '258,272p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Direct reproduction on current main logic:** A read-only Node reproduction of the current function returns `1` for `cosineSimilarity([1, 1], [1])` and `1` for `cosineSimilarity([1, 0, 0, 1], [1, 0])`, matching the reported inflated-score bug instead of the mathematically correct reduced similarity.
  - command: `node -e 'function cosineSimilarity(a,b){if(a.length===0||b.length===0)return 0; const len=Math.min(a.length,b.length); let dot=0,normA=0,normB=0; for(let i=0;i<len;i+=1){const av=a[i]??0; const bv=b[i]??0; dot+=av*bv; normA+=av*av; normB+=bv*bv;} if(normA===0||normB===0)return 0; return dot/(Math.sqrt(normA)*Math.sqrt(normB));} console.log(cosineSimilarity([1,1],[1])); console.log(cosineSimilarity([1,0,0,1],[1,0]));'`
