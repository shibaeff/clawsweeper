## Summary

Keep open. Current `main` still has the reported cosine-similarity bug: `cosineSimilarity()` only accumulates norms across `Math.min(a.length, b.length)`, so extra dimensions in the longer vector are ignored. The fallback memory search path still calls this function, and there is no matching regression test on `main`.

## Evidence

- **Current implementation still truncates both norms to shared length:** `cosineSimilarity()` computes `len = Math.min(a.length, b.length)` and only updates `normA`/`normB` inside that shared-length loop before returning the score. There are no follow-up loops for the remaining dimensions.

## Risks / Open Questions

- The impact is concentrated in the non-sqlite-vec fallback path, so users with working vector search may see this less often.
- `model` filtering and full reindex-on-model-change reduce some mixed-dimension scenarios, but current fallback scoring does not validate or normalize embedding lengths before comparison.
