## Summary

Keep open. Current `main` still lacks the proposed runtime-provider fast path: `resolvePluginProviders()` goes straight through `resolveRuntimePluginRegistry(loadState.loadOptions)`, and the loader’s compatibility key still includes `workspaceDir`, so distinct worker/session workspaces do not reuse the active gateway registry. The PR discussion also shows the original safety concern was addressed by switching to `resolveCompatibleRuntimePluginRegistry()`, so this remains plausible, reviewable work rather than something already implemented or obsolete.

## Evidence

A different workspace therefore misses compatibility reuse.
