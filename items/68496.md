## Summary

Current `main` does not implement first-class Hermes ACP harness support. The bundled ACPX router and alias docs enumerate supported harness ids/commands and omit `hermes`, ACP thread detail hints only special-case Codex/Kimi, and the only existing path for Hermes-like setups is the generic `agents.<id>.command` override. This is a concrete, recent feature request against shipped bundled ACPX behavior, so there is not strong evidence for any close reason.

## Evidence

- **Bundled ACP router omits Hermes from direct name-to-agent mapping:** The built-in ACP router maps named harnesses to fixed `agentId` values (`pi`, `openclaw`, `claude`, `codex`, `copilot`, `cursor`, `droid`, `opencode`, `gemini`, `iflow`, `kilocode`, `kimi`, `kiro`, `qwen`) and says these defaults match current ACPX built-in aliases. `hermes` is not listed.
