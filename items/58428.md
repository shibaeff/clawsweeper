## Summary

Current `main` no longer exhibits the reported fallback to `agent:main:main` when a heartbeat is requested with a non-main subagent session key. The heartbeat path now derives the target agent from `opts.sessionKey`, and then intentionally falls back only to that agent's main session rather than routing to the subagent session itself.

## Evidence

- **Heartbeat runner derives agent from forced session key:** `runHeartbeatOnce` parses `opts.sessionKey` and uses its `agentId` when no explicit `agentId` was supplied, so a key like `agent:codingxia:...` resolves under agent `codingxia` instead of defaulting to `main`.
  - file: [src/infra/heartbeat-runner.ts:738](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/infra/heartbeat-runner.ts#L738)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '728,790p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Subagent heartbeat routing is intentionally guarded:** `resolveHeartbeatSession` explicitly rejects forced subagent session keys and falls back to the resolved agent's main session key. That means the current design is 'use the owning agent main session', not 'route heartbeats into subagent sessions'.
  - file: [src/infra/heartbeat-runner.ts:273](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/infra/heartbeat-runner.ts#L273)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '240,380p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Regression test codifies the subagent fallback behavior:** There is a dedicated test asserting that forcing a subagent heartbeat session key falls back to the main session and does not send the heartbeat through the subagent session.
  - file: [src/infra/heartbeat-runner.subagent-session-guard.test.ts:12](https://github.com/openclaw/openclaw/blob/3de44fe593057b508d76f65abb36d58e0936ef1d/src/infra/heartbeat-runner.subagent-session-guard.test.ts#L12)
  - command: `nl -ba src/infra/heartbeat-runner.subagent-session-guard.test.ts | sed -n '1,140p'`
  - sha: [3de44fe59305](https://github.com/openclaw/openclaw/commit/3de44fe593057b508d76f65abb36d58e0936ef1d)
- **Main release already carries the same logic:** The current latest release tag contains the same `forcedSessionAgentId` derivation and subagent-session guard, so this obsolete routing path is not just fixed on unreleased `main`.
  - file: [src/infra/heartbeat-runner.ts:738](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/infra/heartbeat-runner.ts#L738)

## Risks / Open Questions

- Current behavior still intentionally does not route heartbeats into subagent sessions themselves; if that capability is desired, it would need a separate design/feature discussion rather than keeping this bug report open as written.
