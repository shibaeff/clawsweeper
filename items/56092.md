## Summary

Keep open. Current `main` at `95b7a85f0607784d8e33b4ce7e42617174196490` still routes wake-triggered heartbeats through `forcedSessionKey` before `heartbeat.session`, so the reported session-routing bug is not implemented on `main`. The PR's supporting queue-migration machinery and dedicated regression test are also absent from `main`.

## Evidence

- **Wake path still passes the originating session into preflight:** `runHeartbeatOnce()` forwards `opts.sessionKey` as `forcedSessionKey` into `resolveHeartbeatPreflight()`, so a system-event wake still brings the originating session key into session resolution.
  - file: [src/infra/heartbeat-runner.ts:763](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/heartbeat-runner.ts#L763)

That is the opposite of the PR's claimed fix, so configured heartbeat routing still loses when a wake supplies a source session key.
