## Summary

Keep open. Current `main` still drops per-plugin config before plugin tool factories run, so the PR’s core bug report remains valid. The repo also carries explicit fallback/workaround patterns for plugin config access, which is further evidence that this path is not implemented on `main` yet.

## Evidence

- **Tool factory context still has no `pluginConfig` field:** `OpenClawPluginToolContext` on current `main` includes `config`, `runtimeConfig`, and other runtime fields, but not `pluginConfig`.
  - file: [src/plugins/tool-types.ts:8](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/tool-types.ts#L8)

  - file: [src/plugins/registry-types.ts:55](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/registry-types.ts#L55)
