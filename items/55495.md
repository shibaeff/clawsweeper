## Summary

Keep open. Current `main` still leaves the skills snapshot version state at `0` on process start, and the refresh check still skips persisted snapshots whose stored version is also `0`. The bug described in PR #55495 is therefore not implemented on `main`; the proposed fix still appears relevant, but the patch needs rebasing because the logic now lives in `src/agents/skills/refresh-state.ts` rather than `src/agents/skills/refresh.ts`.

## Evidence

- **Global version still starts at 0 on main:** `src/agents/skills/refresh-state.ts` initializes `globalVersion = 0`, so a gateway restart still resets the in-memory baseline to zero.
  - file: [src/agents/skills/refresh-state.ts:9](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/skills/refresh-state.ts#L9)

  - file: [src/agents/skills/refresh-state.ts:65](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/skills/refresh-state.ts#L65)
