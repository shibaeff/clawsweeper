## Summary

Current `main` has diverged from this PR’s restart approach, but it does not implement the PR’s launchd EIO mitigation in the remaining install/repair bootstrap paths. That is not strong enough evidence for `implemented_on_main`, and I cannot conservatively close it on any other allowed basis.

## Evidence

- **Normal restart path changed on main:** `restartLaunchAgent` is now kickstart-first: it enables the service, runs `launchctl kickstart -k`, and only falls back to bootstrap if the service is not loaded. The PR’s bootout/wait/escalate/bootstrap flow is therefore not already present on `main`, and the branch would need a rebase/rethink rather than auto-close.
  - file: [src/daemon/launchd.ts:659](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/daemon/launchd.ts#L659)
