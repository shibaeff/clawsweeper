## Summary

The report targets a custom local context-engine plugin (`db-context-engine`) that is not part of OpenClaw main. Current main treats context engines as plugin-owned extensions, and the supported SDK already passes session identity and assembled message history into the plugin lifecycle, so a plugin fallback that depends on a private `session_messages.agent_id` column is outside the core contract.

## Evidence

- **Context engines are plugin-installed surfaces:** The docs describe context engines as plugins installed separately and selected via `plugins.slots.contextEngine`; plugin-specific behavior/config belongs to the plugin.
  - file: [docs/concepts/context-engine.md:28](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/concepts/context-engine.md#L28)
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **OpenClaw does not list a bundled `db-context-engine`:** The plugin docs enumerate official/core plugins and bundled plugin categories; there is no shipped `db-context-engine` in current main.
  - file: [docs/tools/plugin.md:85](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/tools/plugin.md#L85)

## Close Comment

Closing as `clawhub` after a Codex review. The report is about a custom local context-engine plugin (`~/.openclaw/plugins/db-context-engine/index.js`), not a shipped OpenClaw component. Current main documents context engines as plugin-owned extensions (`docs/concepts/context-engine.md:28`) and keeps optional capability in plugins rather than core (`VISION.md:54`). The supported SDK already passes `sessionId`, `sessionKey`, `sessionFile`, and `messages` into context-engine hooks (`src/context-engine/types.ts:166`, `src/agents/pi-embedded-runner/run/attempt.ts:1637`), and I could not find any core `db-context-engine` implementation or `session_messages.agent_id` schema contract on `main`. This should be tracked in the plugin's own repo / ClawHub package instead of OpenClaw core.
