## Summary

Keep open. Current `main` does not implement this feature, and the PR discussion raises unresolved correctness risks around freeing a lane slot before the timed-out task is actually canceled. The review concern is material because restart drain logic keys off `activeTaskIds`, and the proposed timeout path would change that accounting.
