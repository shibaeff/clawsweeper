## Summary

Keep open. Current `main` still sends MiniMax VLM requests with plain `fetch()` against a host derived from `apiHost` / `MINIMAX_API_HOST` / `modelBaseUrl`, so the SSRF concern is not implemented away on `main`. The PR should not be auto-closed, but it is also stale as written because its `withStrictGuardedFetchMode(...)` approach conflicts with the later proxy-required-environment fix now present on `main`.

## Evidence

There is no `fetchWithSsrFGuard(...)` or hostname/IP validation on this path on current `main`.
