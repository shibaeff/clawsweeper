## Summary

Current main does not appear to implement the requested native PowerShell smoke coverage for contributor commands. `pnpm check:docs` runs only in the Linux `check-docs` job, `build:strict-smoke` coverage is Linux-only via the `check` shard, and the Windows CI lane is explicitly scoped to Windows-specific tests run from a bash default shell rather than a native PowerShell contributor-command smoke.

## Evidence

- **Docs check is Linux-only:** The `check-docs` job runs `pnpm check:docs` in `.github/workflows/ci.yml:1772-1773`; that job is outside the Windows lane and is not configured for PowerShell execution.
  - file: [.github/workflows/ci.yml:1772](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/.github/workflows/ci.yml#L1772)
  - command: `nl -ba .github/workflows/ci.yml | sed -n '1760,1774p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Strict smoke is Linux-only:** The main `check` matrix defines `check-strict-smoke` at `.github/workflows/ci.yml:1385-1387` and executes `pnpm build:plugin-sdk:strict-smoke` in `.github/workflows/ci.yml:1470-1472`, all on Ubuntu runners rather than Windows PowerShell.
  - file: [.github/workflows/ci.yml:1385](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/.github/workflows/ci.yml#L1385)
  - command: `nl -ba .github/workflows/ci.yml | sed -n '1368,1474p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)

  - file: [.github/workflows/ci.yml:1818](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/.github/workflows/ci.yml#L1818)

## Risks / Open Questions

- Closing this as already implemented would drop a still-valid CI gap: current main has Windows-native tests, but not a native PowerShell smoke lane for contributor commands like `pnpm check:docs` or `pnpm build:strict-smoke`.
- The current documented CI design deliberately keeps Windows coverage narrow, so the requested expansion still appears to be unshipped rather than obsolete.
