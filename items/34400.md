## Summary

Keep open. Current `main` does recurse through `memory/` when collecting files and watches nested paths, so the original single-depth indexing claim is outdated. But the user-facing contract still says `memory/*.md`, nested dated notes are still not recognized by temporal decay, and search-triggered sync is still fire-and-forget. That is not strong enough to close as implemented or cannot-reproduce.

## Risks / Open Questions

- Closing as `implemented_on_main` would discard remaining work to align docs/tool prompts with actual recursive behavior.
- Nested dated files under subdirectories still appear to get different ranking semantics than top-level dated files because temporal decay only recognizes top-level paths.
- The search path is recursive, but the current async `onSearch` sync path means freshness bugs after file changes are still plausible and not clearly closed out by current `main`.
