## Summary

Keep open. Current `main` still closes Feishu webhook servers with plain `server.close()` in both the shared stop helper and the abort-driven webhook cleanup, and PR #48329 only changes `monitor.state.ts`, so it does not fix the gateway restart path called out in review.
