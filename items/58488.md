## Summary

Current `main` no longer shows the old Claude-specific `_meta.claudeCode` forwarding path in the OpenClaw tree, and explicit ACP `model` overrides now flow through a generic runtime config option. But I could not find strong evidence that an agent's configured default model (`model.primary`) is forwarded into ACP session initialization for the reported kilocode case. Since the issue's concrete repro is about agent-config defaults, not only explicit `/acp model` or `sessions_spawn.model`, the evidence is not strong enough to close conservatively.

## Evidence

- **Generic ACP model option exists:** ACP runtime options include a generic `model` field, described as an ACP runtime config option rather than a Claude-specific metadata namespace.
  - file: [src/config/sessions/types.ts:59](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/config/sessions/types.ts#L59)
- **Generic model option maps to runtime config key:** OpenClaw maps the ACP runtime `model` option to the generic config-option pair `model=<value>`.
  - file: [src/acp/control-plane/runtime-options.ts:301](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/acp/control-plane/runtime-options.ts#L301)
- **Runtime applies generic config options before turns:** Before an ACP turn runs, OpenClaw applies runtime config options through `session/set_config_option`; this is the active model-forwarding seam I found on `main`.
  - file: [src/acp/control-plane/manager.runtime-controls.ts:86](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/acp/control-plane/manager.runtime-controls.ts#L86)
- **Explicit ACP spawn model override is forwarded:** ACP spawn passes `params.model` into `initializeSession` as `runtimeOptions: { model: ... }`, and there is a regression test covering that explicit override path.
  - file: [src/agents/acp-spawn.ts:853](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/acp-spawn.ts#L853)
- **Docs only claim explicit override behavior:** The ACP docs say the optional `model` field is honored for `runtime: "acp"` so the child uses the requested model; I did not find comparable evidence for automatic forwarding from agent `model.primary`.
  - file: [docs/tools/acp-agents.md:330](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/tools/acp-agents.md#L330)
- **Old Claude-specific path not present in current tree:** A repo search for `claudeCode`, `_meta: { claudeCode`, and `buildClaudeCodeOptionsMeta` under ACP/OpenClaw ACPX sources returned no matches.
  - command: `rg -n "claudeCode|_meta: \{ claudeCode|buildClaudeCodeOptionsMeta" src/acp extensions/acpx/src docs/tools/acp-agents.md docs/tools/acp-agents-setup.md -S`

## Risks / Open Questions

- The issue's repro is specifically about agent-configured default models on ACP sessions; the code I found clearly covers explicit ACP model overrides, but not that default-model path.
- Closing now would assume the explicit generic runtime option fully replaces the reported failing path for kilocode, which I could not verify from current `main` without stronger evidence.
