## Summary

Current `main` already has a native post-compaction workspace refresh path: compaction retries rebuild bootstrap context, and follow-up turns inject a post-compaction `AGENTS.md` refresh that reruns startup guidance by default. The implementation is present in the shipped `v2026.4.23` tree.

## Evidence

- **Native post-compaction refresh prompt exists:** `readPostCompactionContext()` reads workspace `AGENTS.md`, extracts the default `Session Startup` and `Red Lines` sections, falls back to legacy `Every Session` / `Safety` headings, substitutes runtime date placeholders, and returns a `[Post-compaction context refresh]` prompt that tells the agent to rerun its startup sequence before replying.
  - file: [src/auto-reply/reply/post-compaction-context.ts:71](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/auto-reply/reply/post-compaction-context.ts#L71)
  - command: `git show a9797214:src/auto-reply/reply/post-compaction-context.ts | nl -ba | sed -n '71,163p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Refresh is queued automatically after compaction:** After auto-compaction, `agent-runner.ts` calls `readPostCompactionContext(...)` and enqueues the returned text as a system event for the next turn, so the refresh is native runtime behavior rather than a manual workaround.
  - file: [src/auto-reply/reply/agent-runner.ts:1576](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/auto-reply/reply/agent-runner.ts#L1576)
