## Summary

Keep open. Current `main` still uses the default agent primary model for session-memory slug generation on the shared `temp:slug-generator` embedded run lane. The only evident follow-up on `main` is configurable timeout handling; there is still no dedicated lightweight slug model, no fast heuristic cutoff before the embedded run completes, and no debouncing of concurrent slug requests.

## Evidence

- **Primary model still drives slug generation:** `generateSlugViaLLM()` resolves `modelRef` via `resolveAgentEffectiveModelPrimary(params.cfg, agentId)` and passes that provider/model into `runEmbeddedPiAgent(...)`. The run still uses `sessionKey: "temp:slug-generator"`, so it remains a shared lane. See `src/hooks/llm-slug-generator.ts:58` and `src/hooks/llm-slug-generator.ts:65`.
