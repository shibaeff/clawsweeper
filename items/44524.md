## Summary

Current `main` already covers the reported Discord empty-`mentionedUsers` case by hydrating inbound messages from Discord REST when the raw message text contains mention syntax but the event payload lacks mention metadata, and there is a regression test for that exact scenario.

## Evidence

## Risks / Open Questions

- The current fix relies on REST hydration rather than a direct content-only explicit-mention fallback. If Discord REST also omits mentions or `client.rest` is unavailable, that narrower edge case would still depend on other mention detection paths.
- I could not inspect GitHub PR metadata from `gh` because `GH_TOKEN` is not configured in this environment, so the exact introducing commit was not recoverable from local history.
