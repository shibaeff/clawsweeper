## Summary

Keep open. Current main still exposes `buffer`/base64 on the `message` tool schema, but the generic `send` path and Telegram send action only consume `media`/`mediaUrl`. Buffer hydration is explicitly limited to `sendAttachment`/`upload-file`, so the reported Telegram `message.send` buffer-upload gap is still plausible and is not implemented on `main`.

## Evidence

- **Message tool still advertises buffer/base64 for send:** The shared send schema still includes `buffer` with the description `Base64 payload for attachments (optionally a data: URL)` alongside `media`, so reporters are led to expect buffer-backed sends to work through `message` action `send`.
