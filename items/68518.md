## Summary

Keep open. Current `main` does not yet implement the UI-side system-event filter, and PR #68518 as written is not conservative enough: its broad `System:` / `System (untrusted):` prefix matcher would also hide legitimate user-authored content.

## Evidence

- **Current main still lacks the claimed UI filter:** On current `main`, `shouldHideHistoryMessage` only drops assistant `NO_REPLY` messages and the synthetic transcript-repair tool result. There is no system-event history filter in the shipped UI path.
