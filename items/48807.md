## Summary

Keep open. Current `main` still performs health-monitor restarts without any active-run drain, still timestamps cooldown from the pre-restart `now`, and still resolves timing once at monitor startup, so this PR's fix is not already implemented on `main`. The PR discussion is useful evidence that the author addressed several review comments on the branch, but there is no corresponding implementation in the checked-out `main` code.
