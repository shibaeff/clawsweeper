## Summary

Keep open. Current `main` still writes mirrored transcript header `cwd` from `process.cwd()` instead of the target agent workspace, so the request is not implemented on `main`. The PR also needs rework because its proposed `loadConfig()` call would run on the post-send mirror path and can turn transcript bookkeeping failures into outbound delivery failures.

## Risks / Open Questions

- Merging the PR in its current form risks retries or failed queue bookkeeping after a message was already sent, because transcript mirroring is awaited on the outbound success path.
