## Summary

Keep open. Current `main` still lacks user-facing `channels.feishu.mediaLocalRoots` support, so this is not implemented already, but the PR is also incomplete: the normal outbound path already injects agent-scoped `mediaLocalRoots`, which would shadow the proposed config fallback in regular sends.

## Evidence

- **Feishu config schema on main has no mediaLocalRoots field:** `FeishuSharedConfigShape` includes `mediaMaxMb` and then moves directly to `httpTimeoutMs`; there is no `mediaLocalRoots` entry in the current Feishu schema.
  - file: [extensions/feishu/src/config-schema.ts:174](https://github.com/openclaw/openclaw/blob/33d5ebbff7bceb782b314de14ebd9a213ba31869/extensions/feishu/src/config-schema.ts#L174)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '150,230p'`
  - sha: [33d5ebbff7bc](https://github.com/openclaw/openclaw/commit/33d5ebbff7bceb782b314de14ebd9a213ba31869)
- **Feishu media send path has no config fallback on main:** `sendMediaFeishu` only accepts `mediaLocalRoots?: readonly string[]` and forwards that param directly to `loadWebMedia`; it does not read `account.config?.mediaLocalRoots` on current `main`.
  - file: [extensions/feishu/src/media.ts:576](https://github.com/openclaw/openclaw/blob/33d5ebbff7bceb782b314de14ebd9a213ba31869/extensions/feishu/src/media.ts#L576)
  - command: `nl -ba extensions/feishu/src/media.ts | sed -n '570,640p'`
  - sha: [33d5ebbff7bc](https://github.com/openclaw/openclaw/commit/33d5ebbff7bceb782b314de14ebd9a213ba31869)
- **Regular outbound sends already inject mediaLocalRoots:** The shared outbound service resolves agent-scoped media access and passes `mediaLocalRoots: mediaAccess.localRoots` into channel message actions. The paired test asserts that dispatch receives `mediaLocalRoots: ["/tmp/agent-roots"]`. That matches the PR review concern that a `mediaLocalRoots ?? account.config?.mediaLocalRoots` fallback would be shadowed in the normal send flow.
  - file: [src/infra/outbound/outbound-send-service.ts:79](https://github.com/openclaw/openclaw/blob/33d5ebbff7bceb782b314de14ebd9a213ba31869/src/infra/outbound/outbound-send-service.ts#L79)

  - sha: [33d5ebbff7bc](https://github.com/openclaw/openclaw/commit/33d5ebbff7bceb782b314de14ebd9a213ba31869)
- **Agent-scoped media access expands localRoots by default:** `resolveAgentScopedOutboundMediaAccess` populates `localRoots` from agent-scoped roots or source-parent expansion when host reads are allowed, so the common outbound pipeline usually has explicit local roots before Feishu code runs.

## Risks / Open Questions

- Closing this PR would drop a still-real Feishu config gap that is not present on current `main` or in current docs.
- Merging the PR as-is would likely leave users thinking `channels.feishu.mediaLocalRoots` works in normal outbound sends when the injected agent-scoped roots would still win.
- The new fallback path is security-sensitive local-file access and the PR discussion correctly notes it lacks direct regression tests.
