## Summary

Current `main` does not show the requested exec-approval denylist feature. The exec approvals docs still define only `security: deny | allowlist | full`, show only `allowlist` in the persisted schema, and explicitly say allowlist matching is path-only, with command arguments left to the operator. That means the requested `denylist` mode and `argsMatch`-style argument matching are not implemented on current `main`. ([github.com](https://github.com/openclaw/openclaw/blob/main/docs/tools/exec-approvals.md))

## Evidence

- **Security modes still exclude denylist:** `docs/tools/exec-approvals.md` on current `main` lists `exec.security` values as only `deny`, `allowlist`, and `full`, and `askFallback` as only `deny`, `allowlist`, and `full`. No `denylist` mode is documented. ([github.com](https://github.com/openclaw/openclaw/blob/main/docs/tools/exec-approvals.md))
