## Summary

Keep open. Current `main` has some generic plugin SecretRef runtime plumbing, but the ACPX config surface requested by this PR is still not implemented end-to-end: ACPX MCP env vars are still validated as plain strings in both the runtime schema and the shipped plugin manifest, so explicit SecretRef objects are not accepted on current `main`.

## Evidence
