## Summary

Keep open. Current `main` has partial groundwork for MCP env SecretRef handling, but the end-to-end feature in PR #46825 is not implemented on `main`: the shipped acpx config schemas still only accept string env values, and the acpx runtime types still model MCP env entries as plain strings.

## Evidence

- **Runtime acpx schema still rejects SecretRef env values:** `McpServerConfig.env` is still `Record<string, string>` and `McpServerConfigSchema.env` still validates each env value as `z.string(...)`, so SecretRef objects are not accepted by the runtime config parser on current `main`.
  - file: [extensions/acpx/src/config-schema.ts:13](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/acpx/src/config-schema.ts#L13)
