## Summary

Keep open. Current `main` at `aa21d4ea7e9aab034acbf0b13f8c3575057c5c54` does not implement the PR’s session-memory cleanup, and the PR also includes additional scheduled-reset hook behavior that is not present on `main` and still needs maintainer judgment.

## Evidence

That means channel-injected prefixes would still be persisted on current `main`.

## Risks / Open Questions

- Closing this PR as implemented would be incorrect: the current session-memory transcript path still persists raw inbound user text without metadata stripping.
- The PR is broader than its title: it changes reset-hook dispatch semantics for scheduled rollovers, and current `main` does not already contain that behavior.
- The proposed Feishu `[message_id: ...]` stripping intersects with an existing invariant that user text should not be discarded just because it resembles a metadata envelope.
