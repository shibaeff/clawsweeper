## Summary

Current main already solves the underlying request: the chat header uses the session selector itself as the header, and that selector now resolves explicit session labels/display names into friendly visible labels instead of raw session keys. The PR's separate `.page-title` implementation is no longer needed.

## Risks / Open Questions

- The shipped implementation differs from this PR's shape: it uses the selector's resolved visible label instead of adding a separate `.page-title`. If maintainers still want extra prominence, that would be a new UX enhancement rather than an unimplemented fix.

## Close Comment

Closing from a Codex review because current `main` already implements the underlying behavior. The chat header now renders through `ui/src/ui/app-render.ts:1417`, which uses `renderChatSessionSelect(state)`, and that selector resolves the active session to a friendly label instead of the raw key (`ui/src/ui/chat/session-controls.ts:30`, `ui/src/ui/chat/session-controls.ts:390`, `ui/src/ui/chat/session-controls.ts:598`). The shipped test at `ui/src/ui/app-render.helpers.node.test.ts:409` also asserts that a labeled subagent session renders `Subagent: cron-config-check` and not the raw session key. This is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so the separate `.page-title` approach in this PR is redundant now.
