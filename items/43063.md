## Summary

Current `main` does not implement this PR’s requested `openclaw models status --json` billing surface. The JSON output still stops at `auth.providers` / `auth.oauth`, provider-usage types remain window-only and exclude `openrouter`, and the bundled OpenRouter plugin does not yet provide a usage snapshot hook. The work still looks useful, but it needs a plugin-owned rewrite rather than a conservative close.
