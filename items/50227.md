## Summary

Current main already implements the reported `write`-tool tilde-path fix for host edit/write operations, and the change shipped in `v2026.4.22` with regression coverage.

## Evidence

- **Shipped fix in release notes:** `CHANGELOG.md` for release `2026.4.22` explicitly says host edit/write `~/...` paths now resolve against the OS home directory so these operations stop failing when `OPENCLAW_HOME` differs.
  - file: [CHANGELOG.md:589](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/CHANGELOG.md#L589)

- **Current implementation expands tilde before write/mkdir:** `createHostWorkspaceWriteTool` now routes non-workspace host writes through `expandTildeToOsHome()`, which uses `resolveOsHomeDir()` before `mkdir`/`writeFile`. The paired host edit path uses the same resolver.
  - file: [src/agents/pi-tools.read.ts:746](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/pi-tools.read.ts#L746)

- **Regression coverage for write and OPENCLAW_HOME mismatch:** Tests cover `writeFile` and `mkdir` with `~`, and explicitly verify the tool ignores `OPENCLAW_HOME` and writes to the OS home target instead.
  - file: [src/agents/pi-tools.read.host-tilde-expansion.test.ts:99](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/pi-tools.read.host-tilde-expansion.test.ts#L99)

## Close Comment
