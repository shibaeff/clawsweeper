## Summary

Keep open. Current `main` still uses startup-captured config for Slack and Discord native/slash command routing, while route resolution continues to read `cfg.session.dmScope`. Telegram has since been refactored to reload config per invocation, but the PR's Slack and Discord fixes are still not present on `main`.

## Evidence
