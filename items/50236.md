## Summary

Current `main` already fixes the reported crash path: unhandled transient network rejections, including undici `TypeError("fetch failed")`, are classified as non-fatal and no longer exit the gateway process. The handler is installed in both main entrypaths, and regression tests assert that `fetch failed` unhandled rejections do not call `process.exit`. This appears fixed on `main` by commit `f353a61bab3d1157c706fb71df93aa86911d7a16`, but it does not appear in the latest tagged release `v2026.4.22`, so the fix is not yet tied to a shipped release tag from this checkout.

## Risks / Open Questions

- The fix is present on current `main` but does not appear in the latest tagged release `v2026.4.22`, so stable users may still hit the old behavior until a newer release ships.
- OpenClaw's followup queue is still process-local (`src/auto-reply/reply/queue/state.ts:23`), so unrelated crashes/restarts can still lose queued followups even though the specific unhandled `fetch failed` crash path is fixed.
