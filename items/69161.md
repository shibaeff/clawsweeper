## Summary

Keep open. Current `main` still rejects cron-context interactive exec approvals at the tool layer, but I did not find a run-level fast-exit that stops the cron after the first denial. The current code instead records the denial as a tool error, suppresses non-timeout exec details for cron payloads, leaves loop blocking optional/high-threshold, and still disables the default LLM idle timeout for cron runs.

## Evidence

- **Cron no-route denial still exists on main:** The gateway exec host still throws a hard error for cron/headless no-approval-route cases via `buildHeadlessExecApprovalDeniedMessage(...)`, and the current regression test still expects `Cron runs cannot wait for interactive exec approval`.
