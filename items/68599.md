## Summary

Keep open. Current `main` does not implement the proposed `[context-overflow-hint]` log or the matching compaction troubleshooting docs, so this is not `implemented_on_main`. The PR is also not a safe cleanup close because the existing review comment identifies a real correctness gap: the proposed `overflowCompactionAttempts > 0` gate is based on a run-scoped counter that is never reset, so it can mislabel a later independent overflow in the same run as an immediate post-compaction retry.

## Evidence

- **Run-scoped counter on main:** `overflowCompactionAttempts` is declared once before the main `while (true)` run loop and is not reset anywhere else in `run.ts`, so any future branch keyed only on `> 0` would be cumulative across the whole run rather than limited to the immediate retry after compaction.
