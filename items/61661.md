## Summary

Current `main` still reloads Control UI from persisted browser settings, stores arbitrary per-gateway `sessionKey` / `lastActiveSessionKey` values unchanged, and only normalizes aliases like `main` during gateway hello. The reported routed-session restore behavior remains plausible and is also reflected in existing UI tests, so there is not strong evidence to close this item.

## Evidence

- **Reload path starts from persisted settings:** `OpenClawApp` initializes `settings` by calling `loadSettings()` during app construction, so browser reload starts from whatever session selection was saved locally.
  - file: [ui/src/ui/app.ts:141](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/ui/src/ui/app.ts#L141)

  - file: [ui/src/ui/storage.ts:126](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/ui/src/ui/storage.ts#L126)
