## Summary

Keep open. Current `main` does not implement generic agent attestation headers for outbound external requests. OpenClaw attribution is still limited to native provider traffic, while `web_fetch` still uses a Chrome-like User-Agent. The related duplicate issue #11137 and PR #11124 were closed without merge, so #7338 remains the canonical open tracker.

## Risks / Open Questions

- The repo has partial attribution machinery for model-provider traffic, so maintainers may eventually choose to solve part of this via provider/plugin seams rather than a generic core attestation layer.
- The issue bundles several different designs (static headers, JWT attestation, per-skill policy), so any future implementation may narrow the scope rather than deliver the full request as written.
