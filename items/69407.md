## Summary

Current `main` already adds bounded browser-control request timeouts on both the browser client and gateway dispatch paths, with regression coverage and docs/changelog updates. The symptom in #69407 is fixed on `main`, but it appears unreleased after `v2026.4.23`.

## Evidence

- **Browser client now bounds hung browser-control calls:** `fetchBrowserJson` and `fetchHttpJson` add `AbortController` timers for both loopback HTTP requests and local-dispatch calls, and the local path races dispatch against an abort promise instead of waiting indefinitely.
  - file: [extensions/browser/src/browser/client-fetch.ts:169](https://github.com/openclaw/openclaw/blob/8e18b3cc20d6c8a5c761315b94303885dd7d2773/extensions/browser/src/browser/client-fetch.ts#L169)

- **Gateway browser.request dispatch is also timeout-wrapped:** The local `browser.request` handler now wraps dispatcher calls in `withTimeout(..., timeoutMs, "browser request")`, returning an error instead of holding the request forever.
  - file: [extensions/browser/src/gateway/browser-request.ts:252](https://github.com/openclaw/openclaw/blob/8e18b3cc20d6c8a5c761315b94303885dd7d2773/extensions/browser/src/gateway/browser-request.ts#L252)
