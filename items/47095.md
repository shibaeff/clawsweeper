## Summary

Keep open. Current `main` still appears to ignore Discord per-channel `systemPrompt` overrides for voice transcript runs: the voice path resolves channel config for access control, but the transcript run only sends `message/sessionKey/agentId/...` to `agentCommandFromIngress` and does not pass `GroupSystemPrompt` or `extraSystemPrompt`, while the text-message path does build `GroupSystemPrompt` from the same channel config and the reply pipeline appends it into the effective system prompt.

## Evidence

- **Voice ingress resolves channel config only for allowlist/access decisions:** `authorizeDiscordVoiceIngress` loads `channelConfig` via `resolveDiscordChannelConfigWithFallback`, then uses it for `enabled`, allowlist, and member access checks. There is no prompt assembly or return path for `systemPrompt`.
  - file: [extensions/discord/src/voice/access.ts:48](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/discord/src/voice/access.ts#L48)
  - command: `nl -ba extensions/discord/src/voice/access.ts | sed -n '40,90p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Discord voice transcript run does not inject channel prompt:** After transcription, `processSegment` formats the transcript text and calls `agentCommandFromIngress` with `message`, `sessionKey`, `agentId`, `messageChannel`, `senderIsOwner`, `allowModelOverride`, and `deliver`. No `extraSystemPrompt` or equivalent prompt augmentation is passed.
  - file: [extensions/discord/src/voice/manager.ts:764](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/discord/src/voice/manager.ts#L764)

  - file: [extensions/discord/src/monitor/message-handler.process.ts:303](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/extensions/discord/src/monitor/message-handler.process.ts#L303)
