## Summary

Keep open: current `main` still has no shipped path that returns the winning fallback auth profile from embedded runs and persists it back to the session override, so the PR’s core fix is not implemented on `main` and the reported behavior remains plausible.

## Evidence

- **Attempt execution still forwards only the pre-run session override:** `runAgentAttempt` derives `sessionAuthProfileId` from `params.sessionEntry?.authProfileOverride` and passes the resulting `authProfileId` into `runEmbeddedPiAgent`; there is no post-success writeback in this function.
  - file: [src/agents/command/attempt-execution.ts:286](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/command/attempt-execution.ts#L286)
  - command: `nl -ba src/agents/command/attempt-execution.ts | sed -n '286,296p;417,447p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Embedded run result type cannot report a final winning profile:** `EmbeddedPiRunResult` on current `main` contains payload/meta fields but no `finalAuthProfileId` or equivalent field, so callers have no typed way to persist the actual fallback winner after a successful run.
  - file: [src/agents/pi-embedded-runner/types.ts:136](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-runner/types.ts#L136)

  - file: [src/agents/pi-embedded-runner/run.ts:2196](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/agents/pi-embedded-runner/run.ts#L2196)
