## Summary

Keep open. Current `main` already has partial equivalents for some ideas in #58398, but it does not fully implement the broader Claude-style multi-layer compaction architecture the issue asks for.

## Evidence

- **Microcompact-style pruning already exists, but as separate context pruning:** `agents.defaults.contextPruning` is documented as pruning old tool results from in-memory context only, with soft-trim plus hard-clear using the placeholder `[Old tool result content cleared]`. The implementation is explicitly described as `"microcompact"-style` and hard-clears old tool results when thresholds are exceeded.
