## Summary

Keep open. Current `main` still drops Telegram direct-thread context during cron delivery inference, so this PR is not implemented on `main` or in the latest released code I inspected.

  - file: [src/agents/tools/cron-tool.ts:361](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/tools/cron-tool.ts#L361)
