## Summary

Keep open. Current `main` does not include the PR's explicit `$include` NUL-byte/length hardening, docs, or tests, and the proposed patch still has unresolved correctness/review issues around byte-based limits and boundary messaging/tests.

## Evidence

  - file: [docs/gateway/configuration-reference.md:1193](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/docs/gateway/configuration-reference.md#L1193)

## Risks / Open Questions

- Current main still depends on generic Node/fs failures for simple NUL-byte paths and OS `ENAMETOOLONG` behavior for overlong paths, so the error surface is less explicit and less deterministic than the PR intends.
- The proposed PR should not be treated as ready-to-land as written because its limit check uses UTF-16 code units rather than bytes, and the review-noted boundary wording/test gaps remain unresolved.
