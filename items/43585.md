## Summary

Keep open. Current `main` does not contain the PR's include-path hardening: `resolvePath` still performs only path resolution, root-boundary checks, and symlink validation, while docs and tests also lack the proposed null-byte and path-length rules. There is not strong evidence for any allowed close reason.

## Evidence

- **Current include resolution lacks NUL/length guards:** `resolvePath` resolves and normalizes the path, then checks only config-directory escape and symlink traversal. There is no `MAX_INCLUDE_PATH_LENGTH`, null-byte rejection, or resolved-path length cap on `main`.
  - file: [src/config/includes.ts:190](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/includes.ts#L190)
  - command: `nl -ba src/config/includes.ts | sed -n '160,245p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Tests do not cover the PR behavior:** Current tests still treat a null-byte path as a generic throw case and do not assert a specific `ConfigIncludeError` message or any path-length boundary behavior.
  - file: [src/config/includes.test.ts:579](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/includes.test.ts#L579)
  - command: `nl -ba src/config/includes.test.ts | sed -n '585,660p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Reference docs do not document null-byte or 4096-character rules:** The configuration reference documents only the config-directory boundary and generic include errors; it does not mention null-byte rejection or maximum include-path length.
  - file: [docs/gateway/configuration-reference.md:1116](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/gateway/configuration-reference.md#L1116)

- **User docs likewise omit the proposed hardening:** The gateway configuration guide says relative paths are resolved from the including file, but it does not document invalid path format, null bytes, or a path-length ceiling.
  - file: [docs/gateway/configuration.md:480](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/gateway/configuration.md#L480)
