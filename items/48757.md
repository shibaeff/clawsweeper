## Summary

Keep open. Current `main` still resolves `google-vertex` ADC auth to a non-secret marker and then stores that value into runtime auth in both embedded-runner paths, so the requested guard is not implemented on `main`. The PR is also stale against the current code shape because embedded-run auth now lives in `src/agents/pi-embedded-runner/run/auth-controller.ts` rather than the old `run.ts` path from the PR.
