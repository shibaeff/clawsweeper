## Summary

Current main already covers this report: transcript appends now deduplicate duplicate assistant delivery-mirror rows in the session JSONL, and the control UI defers `session.message` history reloads until the active run ends so the temporary duplicate-render window is avoided. The same transcript and UI safeguards are present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Close Comment

Closing from a Codex review because current main already implements the requested fix. In `src/config/sessions/transcript.ts:194`, OpenClaw now reuses an existing assistant transcript entry when the same delivery-mirror idempotency key appears again, and it also reuses the latest equivalent assistant message instead of appending a second `.jsonl` row; `src/config/sessions/transcript.test.ts:124` covers both cases and asserts the transcript stays at one assistant row. The temporary duplicate-render side is also guarded in `ui/src/ui/app-gateway.ts:474`, which defers `session.message` history reloads until the active run ends, with coverage in `ui/src/ui/app-gateway.node.test.ts:669`. Those safeguards are present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this looks fixed on current OpenClaw builds newer than `2026.3.11`.
