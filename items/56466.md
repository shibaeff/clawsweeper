## Summary

Keep open. Current `main` still lacks wildcard handling for plugin API ranges, so the PR addresses a real unfixed bug rather than something already implemented or obsolete.

## Evidence

- **Current comparator logic does not special-case wildcard tokens:** `satisfiesComparator()` only treats empty tokens as a match and then handles `^` or standard comparator operators. There is no branch for `*`, `x`, or `X`, so those tokens fall through to generic semver parsing.
  - file: [src/infra/clawhub.ts:302](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/clawhub.ts#L302)
  - command: `nl -ba src/infra/clawhub.ts | sed -n '260,360p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Wildcard tokens cannot parse as comparable semver on main:** `parseComparableSemver()` accepts only numeric semver/calver forms matching `^v?[0-9]+\.[0-9]+\.[0-9]+...$`. A bare `*`/`x`/`X` does not match, so `compareSemver()` returns `null` and `satisfiesComparator()` returns `false`.
  - file: [src/infra/semver-compare.ts:19](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/semver-compare.ts#L19)
  - command: `nl -ba src/infra/semver-compare.ts | sed -n '1,70p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Install compatibility checks still rely on this path:** Plugin installation still rejects incompatible plugin API ranges through `satisfiesPluginApiRange(runtimeVersion, compatibility.pluginApiRange)`, so the missing wildcard support remains user-visible in the install flow.
  - file: [src/plugins/clawhub.ts:688](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/clawhub.ts#L688)
  - command: `nl -ba src/plugins/clawhub.ts | sed -n '680,700p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Main test coverage still omits wildcard regression cases:** The existing `checks plugin api ranges without semver dependency` test covers caret, comparator, calver, and invalid-version cases, but not `*`, `x`, or `X`, matching the unfixed implementation gap described in the PR.
  - file: [src/infra/clawhub.test.ts:80](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/infra/clawhub.test.ts#L80)
