## Summary

Keep open. Current `main` still clears a failed channel-config save error during the subsequent refresh path, so this PR is not implemented on `main` and still addresses a real UX bug.

## Evidence

- **Current save handler lacks error preservation:** `handleChannelConfigSave` on current `main` still does `saveConfig` -> `loadConfig` -> `loadChannels` with no capture/restore of the save error.
  - file: [ui/src/ui/app-channels.ts:39](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/app-channels.ts#L39)
  - command: `nl -ba ui/src/ui/app-channels.ts | sed -n '1,220p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Config reload clears the save error:** `loadConfig` unconditionally resets `state.lastError = null` before requesting `config.get`, so any error set by `saveConfig` is lost on the immediate reload path.
  - file: [ui/src/ui/controllers/config.ts:40](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/controllers/config.ts#L40)
  - command: `nl -ba ui/src/ui/controllers/config.ts | sed -n '1,260p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Channel refresh also clears channel error state:** `loadChannels` unconditionally sets `state.channelsError = null` before fetching channel status, so the channels page only keeps an error if the refresh itself fails.
  - file: [ui/src/ui/controllers/channels.ts:10](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/controllers/channels.ts#L10)
  - command: `nl -ba ui/src/ui/controllers/channels.ts | sed -n '1,220p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Channels UI renders `channelsError`, not config `lastError`:** The channels tab passes `state.channelsError` into `renderChannels`, and that view renders `props.lastError` from that channel-specific field. A config save failure therefore disappears after the refresh sequence on current `main`.
  - file: [ui/src/ui/app-render.ts:1601](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/ui/src/ui/app-render.ts#L1601)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '1588,1638p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **PR discussion matches current code gap:** The review discussion already identified the needed fix shape: preserve the save error after `saveConfig`, then restore it after `loadConfig`/`loadChannels`. Current `main` still does not contain that behavior, and the PR author notes the PR is a small fix with green CI and maintainer LGTM.
  - command: `Provided PR context for #56186 (comments, review comments, timeline)`
