## Summary

Current `main` already supports Telegram DM-topic-specific agent routing via per-DM `direct.<chatId>.topics.<message_thread_id>.agentId` config, with distinct DM thread session keys and regression coverage. The requested capability shipped before this issue and the report's observed `agent:main`-only behavior is obsolete on current `main`.

## Risks / Open Questions

- The exact internal approach proposed in the issue (`peer.id` suffix / `parentPeer` for DM threads) is not what current main uses; the close rationale is based on the shipped user-facing capability for DM-thread-specific agent routing.
- Docs still describe persistent ACP topic bindings as group-forum-topic scoped, so this close should be read as resolving DM-thread agent routing, not necessarily every binding-layer variant discussed in the issue body.

## Close Comment

Closing from a Codex review because current `main` already implements the requested DM-thread agent routing. OpenClaw now has per-DM Telegram config at `channels.telegram.direct.<chatId>.topics.<message_thread_id>` (`src/config/types.telegram.ts:125`, `src/config/types.telegram.ts:291`), the Telegram runtime resolves that DM topic config (`extensions/telegram/src/bot-core.ts:544`), and message routing passes `topicConfig?.agentId` into the route resolver (`extensions/telegram/src/bot-message-context.ts:211`). There is explicit regression coverage for a DM topic routing to a non-main agent (`extensions/telegram/src/bot-message-context.topic-agentid.test.ts:120`) and for DM thread session isolation (`extensions/telegram/src/bot-message-context.dm-threads.test.ts:67`). This capability was shipped in `v2026.3.7` per `CHANGELOG.md:2920`, and the DM-topic session support itself landed in `v2026.3.1` per `CHANGELOG.md:3504`, so the report's observed "all DM threads route to agent:main" behavior is obsolete on current `main`.
Codex evidence: Reviewed against main [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad). Fix evidence: release [v2026.3.7](https://github.com/openclaw/openclaw/releases/tag/v2026.3.7).
