## Summary

Keep open. Current `main` still does not implement the Grok web-search fixes described in PR #43609, and the PR also mixes in unrelated `status` changes that had review-flagged regression risk.

## Evidence

- **Grok still uses the global 30s default timeout:** `executeXaiWebSearchProviderTool` still resolves Grok timeout from `DEFAULT_TIMEOUT_SECONDS`, and that constant is still `30` on current `main`. This means the PR's provider-specific 60s default is not present.

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Null-guard fix is not on main:** `extractXaiWebSearchContent` still dereferences `output.type`, `block.type`, and `annotation.type` without null/object guards. The PR's defensive checks for malformed/null xAI response entries are not present.
  - file: [extensions/xai/src/responses-tool-shared.ts:23](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/xai/src/responses-tool-shared.ts#L23)
