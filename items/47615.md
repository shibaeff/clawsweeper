## Summary

Current `main` already routes Feishu WebSocket proxy handling through the shared `proxy-agent` path, which in this repo is wired to honor ambient proxy env vars including `NO_PROXY`. The PR’s custom `NO_PROXY` matcher is no longer needed, and the review discussion identified correctness issues plus an unrelated prompt-surface change.

## Evidence

- **Feishu WS uses shared ambient proxy helper on main:** `createFeishuWSClient()` now gets its WebSocket agent from `getWsProxyAgent()`, and that helper delegates to the shared ambient proxy resolver instead of building a Feishu-specific `HttpsProxyAgent` path.
