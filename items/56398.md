## Summary

Keep open. Current `main` still fast-fails two-phase exec approvals when there is no connected approval client, no turn-source approval route, and no forwarded delivery, so this PR addresses a real unresolved bug on the gateway exec approval path.

## Evidence

- **Current main still expires pending approvals before checking `twoPhase`:** `handlePendingApprovalRequest(...)` on `main` expires any pending approval with `"no-approval-route"` when there are no approval clients, no turn-source route, and no delivery. The `twoPhase` acceptance branch only runs after that fast-fail, so two-phase exec approvals still get denied in this case.

There is no corresponding exec-approval test on `main` covering the two-phase pending case described in PR #56398.
