## Summary

Current `main` already contains an unreleased Control UI assistant-avatar fix that preserves avatar metadata across bootstrap, identity reloads, and avatar refresh/render paths, which matches the reported "shows briefly, then falls back to placeholder" behavior. The fix landed after `v2026.4.23`, so the report is valid for that release but already implemented on `main`.
