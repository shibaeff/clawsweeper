## Summary

Keep open. Current `main` does not contain the session-run cancel seam or the new `sessions_send` delegation hook, so this is not already implemented. The PR discussion also surfaces unresolved correctness risks in the proposed public SDK/runtime shape, especially around premature handler consumption and late cancel-handler registration races.

## Risks / Open Questions

- If `emitSessionRunCancel` remains public on the plugin SDK subpath, plugin code can trigger core-owned cancel fan-out early and silently drop the later real abort notification for that run target.
- If cancellation is only emitted once and not persisted as terminal state, delegated runtimes that register `onSessionRunCancel` slightly after the owning run aborts can miss cancellation entirely.
- This PR expands the public plugin SDK and hook contract, so it needs maintainer review on the boundary shape, not cleanup closure.
