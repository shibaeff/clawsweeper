## Summary

Keep open. Current `main` still lacks multiple substantive parts of PR #45290, so this is not safely closable as implemented or obsolete.

## Evidence

- **Plain-text leakage cleanup is not on main:** `sanitizeForPlainText()` still only converts/strips HTML-like markup. The PR’s LongCat/tool-call/reply-tag leakage stripping is absent, and the current tests only cover HTML/plain-text cases.
  - file: [src/infra/outbound/sanitize-text.ts:22](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/infra/outbound/sanitize-text.ts#L22)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Mirror-session outbound channel fallback is absent:** `sendMessage()` still resolves the required channel from only `cfg` and an explicit `channel`. It does not use `extractDeliveryInfo(...).deliveryContext.channel` as a fallback for mirrored sessions, which was a core part of this PR.
  - file: [src/infra/outbound/message.ts:159](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/infra/outbound/message.ts#L159)

- **Argus session metadata/protocol changes are not present:** `SessionEntry` on `main` has no `argus` field, and the gateway `sessions.patch` schema has no Argus patch parameters. The PR’s lock/watchdog/session-protection work therefore has not landed.
  - file: [src/config/sessions/types.ts:254](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/config/sessions/types.ts#L254)

In `deliver.ts`, best-effort partial failures still go straight to `failDelivery()` without shrinking the queued payload set first.
  - file: [src/infra/outbound/deliver.ts:705](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/infra/outbound/deliver.ts#L705)
