## Summary

Keep open. Current `main` still uses Telegram's proxy-aware transport for Bot API calls and inbound file downloads, but the outbound remote-media send/reply paths do not thread that transport into `loadWebMedia`, so the behavior described by PR #47387 is not implemented on `main` yet.

## Evidence

- **Telegram API client transport is proxy-aware, but outbound media send is not:** `resolveTelegramClientOptions` builds a proxy-aware `fetch` for grammY from `account.config.proxy`, but `sendMessageTelegram` later calls `loadWebMedia` with only `maxBytes`, `mediaLocalRoots`, `mediaReadFile`, and `optimizeImages`. No proxy/fetch transport fields are forwarded into outbound media loading.
