## Summary

Do not close. Current main clearly fixes the `openclaw skills info <name>` lookup mismatch, but I could not prove the broader agent-availability bug is resolved. The agent-side filtering path still matches configured skill allowlists against `entry.skill.name` only, so there is not strong evidence that the reported `ready in list / unavailable to agent` case is fully fixed on `main`.

## Evidence
