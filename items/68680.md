## Summary

Keep open. Current `main` still finalizes `child.on("exit")` with `error: null` whenever `code === null`, so signal-killed subprocesses are still surfaced as silent undefined exits rather than structured signal errors. The PR is recent, has addressed the review concerns in-thread, and is not implemented on `main` yet.

## Evidence

- **Current main still drops exit signals in node-host exec path:** `runCommand` on current `main` listens for `exit` with only the `code` parameter and calls `finalize(code === null ? undefined : code, null)`. A SIGKILL/SIGTERM exit therefore still produces `exitCode: undefined` and `error: null` instead of a structured signal error.
