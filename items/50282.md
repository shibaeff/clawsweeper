## Summary

Current `main` already has public plugin hooks for this workflow: `message_sending` can rewrite or suppress each outbound payload immediately before delivery, and `before_agent_reply` can short-circuit a turn with a synthetic/silent reply before the LLM runs.

## Risks / Open Questions

- `message_sending` only rewrites text/cancels delivery; it is not a full arbitrary payload transformer for non-text fields.

## Close Comment

Closing from a Codex review because current `main` already covers this request through existing public hooks. `message_sending` is the outbound pre-delivery hook: plugins receive the outgoing content plus routing metadata and can replace the text or cancel the send before the channel delivery happens (`src/plugins/hook-message.types.ts:45`, `src/plugins/hooks.ts:732`, `src/infra/outbound/deliver.ts:567`, `src/infra/outbound/deliver.ts:834`). For earlier interception, `before_agent_reply` can also short-circuit the turn with a synthetic or silent reply before the LLM runs (`src/auto-reply/reply/get-reply.ts:567`; shipped per `CHANGELOG.md:1548`). Given that these hooks are already present on `main`, this issue does not need a new core `before_reply_send` hook.
