## Summary

Keep open. Current `main` still builds the webchat session dropdown only from persisted session rows plus the active session, so agents with no saved session are still omitted. The proposed PR also has an unresolved correctness issue for non-default main session keys: `resolveSidebarChatSessionKey()` can return a full canonical session key, so concatenating it as a suffix would synthesize malformed agent session keys.

## Evidence
