## Summary

`openai-responses` client support already exists on current `main` and was shipped in release `v2026.4.22`, including config/schema support, transport routing, request/response conversion, and docs for self-hosted `/v1` endpoints.

## Close Comment

Closing from a Codex review: this is already implemented on current `main`, and the shipped `v2026.4.22` release already contains it. `src/config/types.models.ts:9` and `src/config/schema.base.generated.ts:1388` already allow `models.providers.*.api: "openai-responses"`; `src/agents/provider-transport-stream.ts:13` and `src/agents/provider-transport-stream.ts:79` route that adapter to the shared Responses client transport; and `src/agents/openai-transport-stream.ts:197` plus `src/agents/openai-transport-stream.ts:655` convert OpenClaw messages/tool results into Responses-format payloads and send them with `client.responses.create(...)`. The docs also already cover self-hosted `/v1` backends using `api: "openai-responses"` in `docs/gateway/local-models.md:14` and `docs/gateway/local-models.md:114`. Since the requested client-side `/v1/responses` support is present and released, I’m closing this as implemented.
