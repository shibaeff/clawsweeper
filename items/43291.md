## Summary

Current `main` does not contain the PR’s fix. The CLI-backend prompt builder still ignores `sessionKey`, so cron/subagent CLI sessions still fall through to the default `full` system prompt and can receive the `## Heartbeats` / `HEARTBEAT_OK` instructions. This remains a plausible, narrowly-scoped bugfix PR rather than cleanup-close material.

## Evidence

- **CLI backend prompt builder still has no session-aware mode:** `buildSystemPrompt` in the CLI runner accepts no `sessionKey` and calls `buildAgentSystemPrompt` without any `promptMode`, so it cannot suppress heartbeat sections for cron/subagent sessions.
