## Summary

Keep open. Current `main` does not contain the PR's fix: the CLI runner still builds its system prompt without any `sessionKey`/`promptMode` plumbing, so `buildAgentSystemPrompt` falls back to `"full"` mode. The embedded runner still has its own `resolvePromptModeForSession()` helper for cron/subagent sessions, so the parity gap described in PR #43291 remains on current `main`. A later `main` change (`d29eaeafc1fc1ca86f0ccb5c6b2b3d3668502132`) only added the `before_agent_reply` cron hook seam; it did not implement the heartbeat-section suppression or shared prompt-mode resolution from this PR.
