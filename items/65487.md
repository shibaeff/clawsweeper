## Summary

Current `main` does not implement the requested context-window display in the per-reply `/usage` footer. The footer formatter still emits only input/output, optional cache, optional estimated cost, and optional session key. Context usage exists elsewhere (`/status` and the TUI footer), so this remains an open feature request rather than an already-shipped fix.

## Evidence

- **Per-reply usage footer formatter lacks context fields:** `formatResponseUsageLine()` accepts only `input`, `output`, `cacheRead`, and `cacheWrite`, then returns `Usage: … in / … out …`; there is no context-window parameter or formatting path.
  - file: [src/auto-reply/reply/agent-runner-usage-line.ts:9](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/auto-reply/reply/agent-runner-usage-line.ts#L9)
  - command: `nl -ba src/auto-reply/reply/agent-runner-usage-line.ts | sed -n '1,120p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Reply pipeline only appends usage line plus session suffix:** `runReplyAgent()` builds the footer from `formatResponseUsageLine(...)` and, in `full` mode, only appends `· session \`<sessionKey>\``. No context data is added before appending to the reply payload.
