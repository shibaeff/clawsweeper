## Summary

Current `main` still provides isolated hierarchical subagents plus basic per-session usage/memory primitives, but it does not implement this RFC’s four proposed layers: learned capability profiling, shared blackboard coordination, layered shared-memory visibility, or token-budget governance. This remains an open feature-direction discussion rather than something already implemented or safely closable.

## Evidence

- **Subagents are still isolated by default:** The user-facing subagent docs describe spawned agents as separate sessions that announce results back to the requester. They explicitly say subagents are isolated by default, each child has its own context/token usage, and `context: "fork"` is only an optional transcript copy for a single spawn.
  - file: [docs/tools/subagents.md:10](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/docs/tools/subagents.md#L10)
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Only `isolated` or same-agent transcript fork exists:** `prepareSubagentSessionContext()` supports only `isolated` and `fork`. `fork` is restricted to the same target agent as the requester and copies the parent transcript into the child; there is no shared workspace, peer-to-peer channel, or blackboard-style coordination primitive here.
  - file: [src/agents/subagent-spawn.ts:248](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/agents/subagent-spawn.ts#L248)
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Capability handling is depth/role only:** `resolveSubagentCapabilities()` derives only depth, role, control scope, and booleans like `canSpawn`. There is no learned performance profile, success-rate history, task typing, or cost-aware routing signal in this capability layer.
  - file: [src/agents/subagent-capabilities.ts:123](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/agents/subagent-capabilities.ts#L123)
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Run records do not store profiling/governance data from the RFC:** `SubagentRunRecord` stores task text, session keys, runtime timestamps, outcome, and announce/cleanup fields, but no declared/proven skills, historical match scores, recursive child token totals, or dispatch-quality metrics.
  - file: [src/agents/subagent-registry.types.ts:6](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/agents/subagent-registry.types.ts#L6)
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Usage accounting exists, but no token-governance config:** Session entries persist `inputTokens`, `outputTokens`, `totalTokens`, and `estimatedCostUsd`, but subagent config only defines allowlists, concurrency/depth limits, archival, model/thinking defaults, and timeouts. There are no `globalDailyTokenLimit`, `perTaskTokenBudget`, `budgetExceededAction`, or similar governance settings on current `main`.
  - file: [src/config/types.agent-defaults.ts:388](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/config/types.agent-defaults.ts#L388)
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Memory plugin has no layered visibility model:** The bundled LanceDB memory plugin exposes a single memory store with `{id, text, vector, importance, category, createdAt}` entries, and its config only covers embedding/db/autocapture/autorecall settings. There are no private/team/global visibility tiers or sharing metadata.
  - file: [extensions/memory-lancedb/config.ts:5](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/extensions/memory-lancedb/config.ts#L5)
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **No dedicated blackboard/teleconference/orchestrator implementation paths found:** A path scan for dedicated `octo`, `blackboard`, `teleconference`, or `meeting` implementation paths in `src`, `docs`, and `extensions` returned no matches on the checked-out `main`.
  - command: `rg --files src docs extensions | rg '(^|/)(octo|blackboard|teleconference|meeting)([./-]|$)'`
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)

- Some related orchestration experiments may exist in linked issues/PRs or forks, but there is no evidence in the checked-out `main` that this RFC has already landed well enough to close.
