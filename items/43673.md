## Summary

Current `main` has incremental building blocks for multi-agent and tool-policy hardening, but it does not implement the issue’s requested first-class org/team deployment surface. The repo documents manual per-agent config, sender-scoped tool restrictions, and multi-user DM isolation; it does not expose a `workspace init` scaffolding flow, a declarative `agents.yaml`-style deployment manifest, or an official first-class org/team deployment pattern.

## Evidence

- **No workspace scaffolding CLI found:** Repo-wide search for `openclaw workspace init` / `workspace init` only found the existing one-time `openclaw setup` docs. `docs/cli/setup.md:9`-`45` documents `openclaw setup` with `--workspace`, `--wizard`, and remote flags, but nothing for team templates or multi-agent scaffolding.
  - file: [docs/cli/setup.md:9](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/cli/setup.md#L9)

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **No declarative deployment manifest surface found:** Searches for `agents.yaml`, `agents.yml`, and `deployment manifest` returned no matches across `docs`, `src`, `scripts`, or `README.md`, so there is no visible first-class manifest format for org/team deployments on current `main`.
  - command: `rg -n "agents\.ya?ml|deployment manifest|declarative deployment manifest" docs src scripts README.md`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
