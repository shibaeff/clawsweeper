## Summary

Keep open. Current `main` does not implement either half of the request: the WhatsApp plugin does not expose a `channels resolve` resolver surface, and OpenClaw does not expose a message-status/delivery-ACK command for WhatsApp.

## Evidence

- **`channels resolve` only works for plugins that provide a resolver:** The command explicitly requires `plugin.resolver.resolveTargets` and throws `Channel <name> does not support resolve.` when that adapter is missing.
  - file: [src/commands/channels/resolve.ts:130](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/commands/channels/resolve.ts#L130)
  - command: `nl -ba src/commands/channels/resolve.ts | sed -n '100,170p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **WhatsApp plugin has no resolver adapter:** The WhatsApp channel plugin defines `messaging`, `directory`, and `actions`, but no `resolver` block. A repo search for `resolveTargets:` under `extensions/whatsapp` returned no matches.
  - file: [extensions/whatsapp/src/channel.ts:113](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/whatsapp/src/channel.ts#L113)
  - command: `nl -ba extensions/whatsapp/src/channel.ts | sed -n '110,160p'; rg -n "resolveTargets:" extensions/whatsapp`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **No message-status action or CLI surface exists:** The global message action registry contains no `status` action, and a repo search for `message status` / `status --channel whatsapp` in command and docs paths returned no matches.
  - file: [src/channels/plugins/message-action-names.ts:1](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/channels/plugins/message-action-names.ts#L1)
  - command: `nl -ba src/channels/plugins/message-action-names.ts | sed -n '1,120p'; rg -n "\bmessage status\b|status --channel whatsapp" src/commands docs/channels/whatsapp.md docs/cli/channels.md`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **WhatsApp send path returns only send metadata, not delivery/read ACKs:** `sendMessageWhatsApp()` returns `{ messageId, toJid }`. The inbound monitor remembers outbound message IDs only to suppress self-echoes, not to persist or expose delivery/read receipt state.
  - file: [extensions/whatsapp/src/send.ts:55](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/whatsapp/src/send.ts#L55)
