## Summary

Keep open. Current `main` still treats provider usage as a standard part of `/status` and `session_status`, and the per-agent config schema has no `status.hideUsage` or similar suppression hook, so the requested behavior is not implemented and remains a coherent core feature request.

## Evidence

- **Per-agent config has no status-card suppression field:** `AgentConfig` exposes per-agent knobs such as model, thinking, sandbox, params, tools, and runtime, but no `status`, `hideUsage`, or section-visibility setting.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/config/types.agents.ts#L70)

- **Runtime schema likewise lacks any per-agent status visibility option:** `AgentEntrySchema` validates the agent config surface and includes no `status` object or usage-hiding toggle, so the JSON proposed in the issue would currently be rejected as unknown config.
  - file: [src/config/zod-schema.agent-runtime.ts:812](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/config/zod-schema.agent-runtime.ts#L812)
