## Summary

Keep open. Current `main` does not contain this fix: the equivalent memory-path and temporal-decay checks are still case-sensitive, and current tests still do not cover case-variant `memory/` directory prefixes.

## Risks / Open Questions

- The original PR targeted older paths (`src/memory/*`); the fix now needs rebasing onto the current equivalents in `src/memory-host-sdk/host/*` and `extensions/memory-core/src/memory/*`.
- This may not affect every built-in indexing path on case-insensitive filesystems, but the current code still rejects case-variant `memory/` prefixes in direct path checks and temporal-decay classification, so closing would be premature.
