## Summary

Keep open. Current `main` still takes `openclaw browser --help` through the primary-command plugin registration path, which uses the full plugin loader, and the browser plugin's supposed CLI-metadata path still imports `runtime-api.ts`, which re-exports the heavy browser runtime barrel. The issue timeline also points to related PRs #65460 and #66640 as still open, so there is not strong evidence of a landed fix on `main`.

## Evidence

For `openclaw browser --help`, that means the browser plugin is loaded during command registration rather than handled by the root-help fast path.
