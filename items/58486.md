## Summary

Current `main` already scopes `openrouter/anthropic/*` models to the `openrouter` provider, resolves credentials by `model.provider`, filters provider-mismatched auth profiles out of selection, and has embedded-runner coverage for ignoring mismatched pinned profiles. The reported Claude MAX OAuth leak path does not match the current implementation.

## Evidence

- **OpenRouter model stays on OpenRouter provider:** Regression coverage asserts `openrouter/anthropic/claude-sonnet-4-6` resolves as provider `openrouter` with model `anthropic/claude-sonnet-4-6`, so auth lookup is anchored to OpenRouter rather than Anthropic.
  - file: [src/agents/model-selection.test.ts:277](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/model-selection.test.ts#L277)

  - file: [src/agents/model-auth.ts:723](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/model-auth.ts#L723)
