## Summary

Current `main` already routes Discord DMs by direct-user identity instead of the channel id, and it also normalizes phantom Discord channel session keys back to direct-chat keys, so the reported DM-to-guild-session collision is implemented away on `main`.

## Evidence

- **DM routing uses direct peer identity:** The Discord preflight path resolves DM vs guild from fetched `ChannelType`, then builds the route with `kind: "direct"` and `directUserId: author.id`, so DM routing is keyed by the sender rather than the Discord channel id.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:447](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/extensions/discord/src/monitor/message-handler.preflight.ts#L447)
  - command: `nl -ba extensions/discord/src/monitor/message-handler.preflight.ts | sed -n '438,680p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Direct-chat identity wins over channel targets:** For current conversation identity, `chatType: "direct"` first resolves the sender target from `from` and returns `user:<id>` before considering `originatingTo` or fallback channel targets.
  - file: [extensions/discord/src/conversation-identity.ts:42](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/extensions/discord/src/conversation-identity.ts#L42)
  - command: `nl -ba extensions/discord/src/conversation-identity.ts | sed -n '1,120p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Explicit session keys are rewritten away from phantom channel ids:** If a Discord message is marked direct, `normalizeExplicitDiscordSessionKey` rewrites legacy `discord:dm:*` keys and also rewrites `discord:channel:<id>` to `discord:direct:<id>` when the sender matches, which is the exact guild-id/default-channel-id collision shape from the report.
  - file: [extensions/discord/src/session-key-normalization.ts:23](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/extensions/discord/src/session-key-normalization.ts#L23)

- **Regression tests cover the phantom channel-key case:** Tests assert that a direct-chat context rewrites `discord:channel:123456` to `discord:direct:123456`, and the core explicit-session-key normalizer dispatches Discord keys through that provider-specific rewrite.
  - file: [extensions/discord/src/session-key-normalization.test.ts:25](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/extensions/discord/src/session-key-normalization.test.ts#L25)
