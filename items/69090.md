## Summary

Keep open. Current `main` does not already implement this PR’s core changes: `openclaw security ...` is still not exempt from eager warmup, and `runSecurityAudit` still falls back to loading the configured-channel runtime registry instead of using the proposed bundled read-only fast path. Some adjacent audit workspace/read-only scope hardening has landed separately, but not the behavior this PR is trying to upstream.
