## Summary

PR #46614 is obsolete against current `main`. The file it edits is now only a wrapper, the real Docker setup logic lives in `scripts/docker/setup.sh`, and current code explicitly rejects whitespace in `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` before compose YAML is written. The specific claim that unquoted mounts with spaces inherently produce invalid YAML also does not hold in a direct YAML parse check, so this PR does not match current behavior and should close rather than stay open in its current form.
