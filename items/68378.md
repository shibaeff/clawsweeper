## Summary

Keep open. Current `main` does not show a strong fix for this report; the gateway agent handler still merges persisted session delivery metadata ahead of the active request, which matches the maintainer diagnosis in draft PR #69010 and leaves the reported stale cross-channel routing path plausible.

## Evidence

- **Maintainer diagnosis in issue discussion:** The Apr 19 maintainer comment says draft PR #69010 fixes this by making current request route fields win over persisted session `deliveryContext`, because stale session fields were authoritative in the failing Discord->Feishu shape. I did not find that behavior reflected in current `main`.
- **Gateway agent handler still prefers stored delivery context:** `agentHandlers.agent` builds `requestDeliveryHint`, then calls `mergeDeliveryContext(deliveryFields.deliveryContext, requestDeliveryHint)`. That passes persisted session delivery data as the primary value and the active request as fallback before writing `deliveryContext`, `lastChannel`, `lastTo`, and related fields back to the session entry.
  - file: [src/gateway/server-methods/agent.ts:661](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/gateway/server-methods/agent.ts#L661)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '661,698p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Merge helper keeps primary channel authoritative on conflicts:** `mergeDeliveryContext(primary, fallback)` keeps `primary.channel` when both channels exist and conflict, and in that case also keeps only the primary route fields (`to`, `accountId`, `threadId`). With the current call site, a stale persisted Feishu context stays authoritative over a Discord request hint.
  - file: [src/utils/delivery-context.shared.ts:103](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/utils/delivery-context.shared.ts#L103)

It therefore does not clearly subsume the issue discussion's stale `deliveryContext` precedence diagnosis.
