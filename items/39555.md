## Summary

Keep open. Current `main` still uses a single full-query FTS lookup in provider-backed hybrid memory search, and the FTS query builder still joins tokens with `AND`, so the reported multi-word cross-chunk zero-hit behavior remains plausible. The PR’s intent still matches a real gap, but the branch is stale against the moved `extensions/memory-core` implementation and the review thread includes an unresolved fan-out/perf concern.
