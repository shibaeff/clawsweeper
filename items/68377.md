## Summary

Keep open. Current `main` still does not preserve existing proxy env on reinstall, and the PR still has a substantive lowercase-vs-uppercase proxy precedence problem already raised in review.

## Evidence

- **Current reinstall path drops preserved proxy vars from the existing service env:** `mergeInstallInvocationEnv()` only carries forward existing service env keys that are not flagged as dangerous overrides. That means a forced reinstall started from a shell without proxy vars will not reuse the prior LaunchAgent proxy settings.
  - file: [src/cli/daemon-cli/install.ts:49](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/cli/daemon-cli/install.ts#L49)
  - command: `nl -ba src/cli/daemon-cli/install.ts | sed -n '27,62p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Proxy vars are explicitly classified as blocked override keys:** The current security contract marks `HTTP_PROXY`, `HTTPS_PROXY`, `ALL_PROXY`, and `NO_PROXY` variants as dangerous override keys, so the generic env-preservation path intentionally excludes them.
  - file: [src/infra/host-env-security.test.ts:672](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/infra/host-env-security.test.ts#L672)
  - command: `nl -ba src/infra/host-env-security.test.ts | sed -n '672,678p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Gateway install-plan preservation also filters proxy vars out:** `collectPreservedExistingServiceEnvVars()` applies the same dangerous-override filter before copying existing service env into the next install plan, so current `main` still lacks the requested proxy-specific preservation behavior.
  - file: [src/commands/daemon-install-helpers.ts:203](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/commands/daemon-install-helpers.ts#L203)
  - command: `nl -ba src/commands/daemon-install-helpers.ts | sed -n '178,255p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **The review comment about stale lowercase proxy keys is backed by current runtime behavior:** `resolveEnvHttpProxyUrl()` prefers lowercase `http_proxy`/`https_proxy` over uppercase forms. If an existing plist keeps stale lowercase keys while a reinstall only provides uppercase overrides, traffic can still route through the stale proxy.
