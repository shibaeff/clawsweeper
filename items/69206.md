## Summary

Current main still advertises `dock_*` commands, but Telegram native handling only forwards `/dock_discord` into the normal command/chat pipeline and there is no dock command handler in that pipeline, so the reported fall-through remains plausible on current code.

## Evidence

- **Dock commands are still generated and documented as first-class commands:** The command registry still creates `dock:<plugin>` entries with native name `dock_<plugin>` and text aliases such as `/dock-discord` and `/dock_discord`; the slash-command docs still list `/dock-discord` and `/dock_telegram` as generated dock commands.
