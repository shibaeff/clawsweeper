## Summary

Keep open. Current `main` does not implement this PR’s interactive custom-provider metadata flow. The custom-provider wizard still ends after base URL/auth/model/alias setup, and custom model entries still rely on baked-in default metadata rather than prompting for `contextWindow`, `maxTokens`, reasoning, inputs, or pricing.

## Evidence

- **No metadata hook in configure flow:** `promptAuthConfig()` handles `custom-api-key` by calling `promptCustomApiConfig()` and then breaking out of the loop. There is no follow-up call to any metadata prompt; the only post-loop prompt is the non-custom model allowlist branch.
