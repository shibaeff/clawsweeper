## Summary

Keep open. Current `main` still does not provide an interactive metadata step after custom-provider setup in `openclaw configure --section model`; the code path stops after custom auth setup, and the docs still describe manual `models.providers` / `config set` edits for `contextWindow`, `maxTokens`, and related metadata. The PR is therefore not implemented on `main`, and the linked review/timeline context points to a still-live UX gap rather than an obsolete or incoherent change.
