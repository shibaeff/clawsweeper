## Summary

Keep open. Current `main` still appears susceptible to the reported rollover bug: the dated log path changes in `getLogger()`, but long-lived child/subsystem loggers keep a previously created transport/logger instance and there is no refresh path for them after midnight.

## Evidence

- **Daily path is recomputed, and only the cached base logger is rebuilt:** `resolveSettings()` defaults `file` to `defaultRollingPathForToday()`. `settingsChanged()` compares the resolved file path, and `getLogger()` replaces only `loggingState.cachedLogger` when that path changes.
  - file: [src/logging/logger.ts:127](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/logging/logger.ts#L127)
  - command: `nl -ba src/logging/logger.ts | sed -n '120,245p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **File transport closes over the resolved file path:** `buildLogger()` attaches a transport that writes with `appendLogLine(settings.file, payload)`. That closure uses the `settings.file` value from logger construction time, not a path resolved at write time.
  - file: [src/logging/logger.ts:176](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/logging/logger.ts#L176)
  - command: `nl -ba src/logging/logger.ts | sed -n '153,210p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Subsystem loggers cache a child logger and never refresh it:** `createSubsystemLogger()` initializes `fileLogger` once via `getChildLogger(...)` and then reuses that cached child on later writes. There is no invalidation when `getLogger()` rebuilds the base logger for a new dated file.
  - file: [src/logging/subsystem.ts:349](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/logging/subsystem.ts#L349)
  - command: `nl -ba src/logging/subsystem.ts | sed -n '340,420p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **No obvious regression coverage for rollover refresh:** Current logging tests include child min-level behavior, but the repo search did not surface a test for midnight rollover or refreshing stale child loggers after the rolling filename changes.
  - file: [src/logging/level-filter.test.ts:100](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/logging/level-filter.test.ts#L100)
