## Summary

Keep open. Current `main` still intentionally sends user-visible tool-failure warnings for mutating tools such as `write` and `message`, including recoverable validation-style errors, so the reported UX problem is not implemented/fixed.

## Evidence

- **Mutating tool errors are always surfaced before global suppression is considered:** `resolveToolErrorWarningPolicy()` returns `showWarning: true` for any mutating tool error, and only checks `messages.suppressToolErrors` afterward. That means `write`/`message` validation failures still produce outbound warning payloads.

## Risks / Open Questions

- Closing now would dismiss an active UX bug: recoverable validation failures for mutating tools still reach user chat surfaces on current `main`.
- There is also a docs/runtime mismatch around `messages.suppressToolErrors`, so users may believe warnings are suppressed when they are not.
