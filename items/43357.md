## Summary

Keep open. Current `main` and the latest release still use ASCII-only Discord mention parsing, so the requested Unicode-handle behavior is not implemented. The PR addresses a real gap, and the discussion shows useful follow-up fixes, but the last review thread still points to an unresolved wrapped-URL regression, so there is no strong basis for conservative cleanup closure.
