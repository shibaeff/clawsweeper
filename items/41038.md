## Summary

Keep open. Current `main` still does not surface `stdout` for successful structured tool results in the Swift chat UI, so this PR is neither already implemented nor obsolete from the checked-out code.

## Evidence

- **Current formatter still ignores stdout:** `renderDictionary` reads `status`, `error/reason`, and `message/result/detail`, then returns `messageText` or `status`. There is no `stdout` extraction or priority path on current `main`, so an exec-style payload with both `message` and `stdout` would still show the generic message instead of command output.
