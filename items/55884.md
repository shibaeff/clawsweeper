## Summary

Keep open. Current `main` still emits reset hooks without any same-session dedupe, while the PR as submitted has unresolved review findings about both a TOCTOU race and overbroad process-wide suppression, so this is neither implemented on `main` nor safely closable.

## Evidence

- **Current main still marks `resetHookTriggered` only after awaiting the hook:** `emitResetCommandHooks` creates the hook event and awaits `triggerInternalHook(...)` before setting `params.command.resetHookTriggered = true`, so there is no built-in dedupe for same-session re-entry inside the hook emitter itself.
