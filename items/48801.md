## Summary

Keep open. Current `main` still does not accept the gateway's local `1008` closes for `connect failed`, `device identity required`, or `pairing required` during restart health checks, so the reported timeout path remains plausible. The proposed PR also has unresolved review concerns: its `normalized.includes("pair")` match is too broad, and it would treat empty-reason `1008` closes as healthy without proof that is safe.

## Risks / Open Questions

- The PR's proposed `normalized.includes("pair")` check would match unrelated strings such as `repair` or `keypair`, which could create false healthy restart classifications.
- The PR's proposed `!normalized` branch would treat any empty-reason `1008` close as healthy, which could hide non-gateway or unrelated policy-violation listeners on the port.
- `gh` inspection was unavailable in this checkout because `GH_TOKEN` is not configured, so the review relies on the provided PR discussion plus local code on `main`.
