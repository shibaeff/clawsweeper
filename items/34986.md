## Summary

Keep open. Current `main` has heartbeat cost controls such as `lightContext`, `isolatedSession`, active-hours gating, and a built-in `no-tasks-due` short-circuit for task-based `HEARTBEAT.md` heartbeats, but it does not implement the requested configurable pre-flight script/command. There is no `agents.defaults.heartbeat.preFlightCommand` config on `main`, no schema/docs for it, and `HEARTBEAT_OK` short-circuiting still occurs only after the model run.

## Evidence

It includes `every`, `activeHours`, `model`, `session`, `target`, `directPolicy`, `to`, `accountId`, `prompt`, `includeSystemPromptSection`, `ackMaxChars`, `suppressToolErrorWarnings`, `timeoutSeconds`, `lightContext`, `isolatedSession`, and `includeReasoning`. There is no `preFlightCommand` field.
