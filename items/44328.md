## Summary

Current `main` already fixes the WebChat image-loss regression for text-only session models by offloading inbound images to saved media refs, persisting them into the transcript, and rendering those transcript media paths back in the Control UI. The latest tagged release `v2026.4.22` predates that fix, so this is implemented on `main` but not yet in a shipped release tag I could confirm.
