## Summary

Keep open. Current `main` does not provide strong evidence that the LAN/plain-HTTP WebChat path from #46997 is fixed: the browser still skips device identity outside secure contexts, the gateway still clears scopes for device-less token/password sessions by default, and the relevant tests only preserve no-device Control UI scopes for localhost insecure auth. The HTTP default-scope change on `main` is real, but it does not prove the WebSocket WebChat regression is resolved.

## Evidence
