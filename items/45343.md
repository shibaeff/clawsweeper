## Summary

Keep open. I did not find strong evidence that `main` or the latest release fixed this Windows `sharp` regression. Current code still depends on runtime `sharp` imports for image handling, the Windows installer still performs a plain global `npm.cmd install` without the documented `SHARP_IGNORE_GLOBAL_LIBVIPS=1` safeguard, and the global updater still has an npm fallback that retries with `--omit=optional` while verifying only packaged dist files rather than native deps like `sharp`.
