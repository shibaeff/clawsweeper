## Summary

Keep open. Current `main` still does not make queued command-lane work abort-aware, so the reported stale-queued-run behavior remains plausible even though isolated cron runs are now remapped from the `cron` lane to `nested`.

## Evidence

- **Queue enqueue path still has no abort-aware removal:** `enqueueCommandInLane` only accepts `warnAfterMs` and `onWait`, pushes directly into `state.queue`, and relies on `drainLane` to `shift()` entries later. There is no `AbortSignal` option and no code path that removes a queued entry before dequeue.
  - file: [src/process/command-queue.ts:242](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/process/command-queue.ts#L242)
  - command: `nl -ba src/process/command-queue.ts | sed -n '120,220p' && nl -ba src/process/command-queue.ts | sed -n '242,268p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Embedded runner still enqueues without abort propagation:** `runEmbeddedPiAgent` builds `enqueueGlobal`/`enqueueSession` wrappers that call `enqueueCommandInLane(..., task, opts)` without any abort signal. It only calls `throwIfAborted()` inside the queued task body, which means cancellation is checked after dequeue, not while waiting in the queue.
  - file: [src/agents/pi-embedded-runner/run.ts:247](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/pi-embedded-runner/run.ts#L247)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '232,305p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Cron isolated runs were rerouted, but not fixed via abort-aware queueing:** Current cron isolated execution passes `lane: resolveNestedAgentLane(params.lane)` into `runEmbeddedPiAgent`, and `resolveNestedAgentLane` maps missing/`cron` lanes to `nested`. The regression path therefore moved off the `cron` lane, but the underlying non-abort-aware shared queue behavior remains.
  - file: [src/cron/isolated-agent/run-executor.ts:160](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/isolated-agent/run-executor.ts#L160)

  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **Tests cover lane remap, not queued abort removal:** The current regression coverage verifies that cron embedded runs use `nested`, but there is no matching `command-queue` test for aborting a still-queued entry before dequeue.
  - file: [src/cron/isolated-agent.lane.test.ts:85](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cron/isolated-agent.lane.test.ts#L85)
