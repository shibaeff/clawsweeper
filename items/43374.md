## Summary

Keep open. Current main has new LLM transport timeout hardening, but I could not prove this issue is fixed, and one of the reporter's concrete pressure points still exists in code. The strongest code evidence cuts against closing: fallback attempts get a fresh per-attempt timer, so the reported shared fallback-budget theory is not supported by current main, yet Telegram polling concurrency is still coupled to `maxConcurrent`, and I could not reproduce or verify the end-to-end Telegram multi-agent timeout report from this read-only checkout.

## Evidence

- **Fallback attempts use a fresh timeout per attempt:** `timeoutMs` is resolved once in `agent-command.ts`, then passed into each `runWithModelFallback` attempt. The embedded attempt creates its own abort timer from `params.timeoutMs`, which means current main does not show a single shared fallback-chain deadline.
