## Summary

Current `main` already handles transient Discord gateway metadata failures without crashing the process: the Discord plugin catches 5xx/plain-text `/gateway/bot` responses, falls back to the default gateway URL, and the global unhandled-rejection handler also treats the wrapped Discord transport failure shape as non-fatal. The same implementation is present in tagged release `v2026.4.22`.
