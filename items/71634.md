## Summary

Keep open. Current `main` still has the non-OpenRouter failover gap described in #71620: transport code throws the provider-agnostic message `An unknown error occurred`, but failover classification on `main` only treats that phrase as a timeout when the provider matches Anthropic. The draft PR is narrowly scoped to that gap, adds the missing regression coverage/docs, and the earlier `PR_DESCRIPTION.md` review concern appears resolved in the current diff.

## Evidence

That means a `google` provider will not classify this wrapper text as `timeout` on current `main`.
