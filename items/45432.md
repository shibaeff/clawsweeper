## Summary

Keep open. Current `main` does not show an obvious repo-side path that injects a UTF-8 BOM during `write`, but the edit path also does not strip a leading `\uFEFF`, so a BOM-prefixed file could still plausibly trigger the reported exact-match failure. From this Linux checkout, without a Windows repro or raw-byte proof on current `main`, the evidence is not strong enough for a conservative close.
