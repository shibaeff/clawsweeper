## Summary

Keep open. Current `main` still lacks the PR's core outbound-context threading, so the multi-channel exec-approval forwarding bug remains plausible and the new regression coverage is not present.

## Evidence

- **Forwarder still drops target-derived tool context:** `deliverToTargets(...)` forwards `channel`, `to`, `accountId`, `threadId`, and `payloads`, but it does not attach any `toolContext` derived from the resolved target before calling outbound delivery.
  - file: [src/infra/exec-approval-forwarder.ts:358](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/infra/exec-approval-forwarder.ts#L358)
  - command: `nl -ba src/infra/exec-approval-forwarder.ts | sed -n '339,366p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Outbound delivery path cannot carry tool context:** Current `deliver.ts` still defines `ChannelHandlerParams` without `toolContext`, builds the outbound context without it, and constructs the channel handler from params that therefore cannot pass `currentChannelId/currentChannelProvider` into channel adapters.
  - file: [src/infra/outbound/deliver.ts:131](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/infra/outbound/deliver.ts#L131)
  - command: `nl -ba src/infra/outbound/deliver.ts | sed -n '131,147p'; nl -ba src/infra/outbound/deliver.ts | sed -n '307,327p'; nl -ba src/infra/outbound/deliver.ts | sed -n '871,887p'`
  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)
- **Adapter context type still has no toolContext field:** `ChannelOutboundContext` on `main` does not expose `toolContext`, so even if the forwarder wanted to preserve the resolved target context, the outbound adapter contract still cannot receive it.
  - file: [src/channels/plugins/outbound.types.ts:16](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/channels/plugins/outbound.types.ts#L16)
