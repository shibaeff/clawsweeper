## Summary

Keep open. Current `main` still routes `pnpm ui:build` through `scripts/ui.js`, which resolves an absolute `pnpm` shim path and, on Windows, runs `.cmd` launchers via `spawn(..., { shell: true })`. That is the same execution pattern that matches the reported `DEP0190` warning and `'C:\Program' is not recognized` failure when the shim lives under `C:\Program Files\...`. I did not find a current-main change that switches `scripts/ui.js` to the repo's safer quoted `cmd.exe /c` wrapper flow, and the latest shipped release snapshot still contains the same logic.
