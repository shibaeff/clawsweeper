## Summary

Current `main` already maps CLI `--target`/`target` into the plugin-facing `to` field for send actions, and QQ Bot exposes target normalization for raw OpenIDs and `qqbot:c2c:` forms. The same target resolution path is also wired into agent delivery. This implementation is present in the latest release tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

Closing as implemented. Codex review of current `main` found that the CLI message path now forwards `--target` into the shared message-action normalization layer, which converts `target` into the plugin-facing `to` field for send actions (`src/commands/message.ts:68`, `src/infra/outbound/message-action-normalization.ts:24`, `src/infra/outbound/channel-target.ts:15`). The normalization behavior is covered by tests in `src/infra/outbound/message-action-normalization.test.ts:23`. This implementation is already present in the latest stable release, `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
