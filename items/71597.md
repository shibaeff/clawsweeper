## Summary

Current `main` already contains post-`v2026.4.23` fixes for OpenAI-compatible local backends that match this report: proxy-like local endpoints now default `tool_choice` to `auto`, and streamed `finish_reason: "tool_call"` is accepted as tool use. The latest release does not include those transport fixes yet, so this looks fixed on `main` but unreleased.

## Evidence

- **Current main defaults tool_choice for proxy-like local completions endpoints:** `buildOpenAICompletionsParams` now sets `tool_choice = "auto"` when tools are present on explicit proxy-like `openai-completions` routes, which covers local OpenAI-compatible servers.
  - file: [src/agents/openai-transport-stream.ts:1765](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/openai-transport-stream.ts#L1765)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Current main accepts singular tool_call finish reasons:** `mapStopReason` now treats `finish_reason: "tool_call"` the same as `tool_calls`, preventing local backends that emit the singular form from being surfaced as provider errors.
  - file: [src/agents/openai-transport-stream.ts:1835](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/agents/openai-transport-stream.ts#L1835)

Blame attributes those assertions to commit `443b837bd521b0485c87bb7359f370a4ccec10fa` on 2026-04-25, after the `v2026.4.23` release.

  - sha: [443b837bd521](https://github.com/openclaw/openclaw/commit/443b837bd521b0485c87bb7359f370a4ccec10fa)
