## Summary

Keep open. Current `main` does not implement this PR's Telegram DM ACP-binding behavior: the binding compiler/matcher still accepts only topic-shaped conversations tied to negative group chat ids, while Telegram direct-message routing still supplies plain positive chat ids. The public Telegram docs also still describe persistent ACP bindings as forum-topic-only.

## Evidence

- **Configured Telegram ACP bindings still reject DM chat ids on main:** `normalizeTelegramAcpConversationId` parses only topic conversations and returns `null` unless the parsed `chatId` starts with `-`, so a plain direct-chat id like `123456789` is still rejected during binding compilation.
