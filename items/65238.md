## Summary

Current `main` already fixes the reported regression by stripping persisted base64 audio payloads from chat-history messages before the oversized-message placeholder check, and the fix was shipped in `v2026.4.23`. This matches the issue comment diagnosing TTS audio transcript blocks as the trigger.

## Evidence

- **Base64 audio is redacted during chat-history sanitization:** `sanitizeChatHistoryContentBlock` removes `source.data` from `type: "audio"` blocks, replaces it with `omitted`/`bytes`, and returns the sanitized block instead of leaving multi-hundred-KB base64 in history payloads.
  - file: [src/gateway/server-methods/chat.ts:954](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/gateway/server-methods/chat.ts#L954)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '900,965p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Sanitization happens before oversized placeholder replacement:** `chat.history` builds `normalized` from `sanitizeChatHistoryMessages(...)` first, then runs `replaceOversizedChatHistoryMessages(...)`, so persisted audio base64 is stripped before the 128 KB placeholder logic runs.
  - file: [src/gateway/server-methods/chat.ts:1908](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/gateway/server-methods/chat.ts#L1908)

- **Regression test covers the reported root cause:** `server-methods.test.ts` has an explicit test, `redacts base64 audio content blocks from chat history`, asserting that chat history keeps the text block and replaces inline audio base64 with `omitted: true` plus byte metadata.
  - file: [src/gateway/server-methods/server-methods.test.ts:266](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/gateway/server-methods/server-methods.test.ts#L266)
