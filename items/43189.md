## Summary

Keep open. Current `main` still routes heartbeat-triggered reply runs through the live session transcript file, so this PR is not redundant and the underlying transcript-pollution bug remains plausible on both CLI and embedded paths.

## Evidence

- **Current main still uses live transcript for CLI heartbeat runs:** `runAgentTurnWithFallback()` sets `trigger: params.isHeartbeat ? "heartbeat" : "user"` but still passes `sessionFile: params.followupRun.run.sessionFile` directly into `runCliAgent()` with no heartbeat-specific clone or alternate path.
