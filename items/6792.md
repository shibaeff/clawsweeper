## Summary

Keep open. Current `main` does not implement manifest-level `configPatch` support for `openclaw plugins install`. The manifest type/loader never reads a `configPatch` field, install persistence only enables the plugin and records install metadata, and current docs/tests describe and assert that narrower behavior.

## Evidence

- **Manifest contract has no manifest-level config patch field:** `PluginManifest` enumerates supported top-level manifest fields through `configContracts` and `channelConfigs`; there is no `configPatch` field in the manifest type.
  - file: [src/plugins/manifest.ts:157](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/plugins/manifest.ts#L157)
  - command: `nl -ba src/plugins/manifest.ts | sed -n '130,260p'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Manifest loader ignores unknown install-patch data:** `loadPluginManifest()` reads specific `raw.*` keys and returns a manifest object built from those keys only. It does not read or return `raw.configPatch`, so a manifest-supplied install patch would currently be ignored.
  - file: [src/plugins/manifest.ts:859](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/plugins/manifest.ts#L859)
  - command: `nl -ba src/plugins/manifest.ts | sed -n '809,936p'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Plugin install persistence only enables and records install metadata:** `persistPluginInstall()` adds the plugin to the allowlist if needed, enables it, records `plugins.installs`, applies slot selection, and writes the config. There is no manifest patch merge, validation, diff output, or uninstall tracking for install-time config changes.
  - file: [src/cli/plugins-install-persist.ts:29](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/cli/plugins-install-persist.ts#L29)
  - command: `nl -ba src/cli/plugins-install-persist.ts | sed -n '1,140p'`
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)
- **Current docs still describe manifest as pre-runtime metadata, not install-time config mutation:** The manifest docs say `openclaw.plugin.json` is cheap metadata read before plugin runtime loads and explicitly say not to use it for install metadata. The CLI docs describe `plugins install`/`uninstall` in terms of plugin entries, install records, and load paths, with no manifest config-patch behavior documented.
  - file: [docs/plugins/manifest.md:39](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/docs/plugins/manifest.md#L39)

- **Current install tests assert only enablement plus install record persistence:** CLI install tests build an `installedCfg` containing enabled plugin state plus install metadata and assert `writeConfigFile(installedCfg)`, with no expectation of extra config writes or patch-report output.
  - file: [src/cli/plugins-cli.install.test.ts:347](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/cli/plugins-cli.install.test.ts#L347)
