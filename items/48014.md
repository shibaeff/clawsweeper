## Summary

Keep open. Current `main` ships Microsoft Teams passwordless support via `federated` auth, but it does not implement this PR's `defaultAzureCredential` auth type or the simplified Azure CLI-style/local-dev flow described in the PR. The PR is also based on an older Teams integration layer, so it looks stale against current `main`, but not safely closeable as already implemented.

## Risks / Open Questions

- The PR targets the older `@microsoft/agents-hosting` integration described in its patch, while current `main` uses `@microsoft/teams.apps` and `@microsoft/teams.api`; any merge would require a substantive rebase/port, not a trivial revive.
- Closing as `implemented_on_main` would overstate the match: current `main` covers passwordless Teams auth generally, but not the specific `defaultAzureCredential` operator experience or Azure CLI/local-dev path this PR proposes.
