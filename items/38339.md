## Summary

Close as out of scope for OpenClaw core. Current main keeps MCP on the external bridge/registry path (`mcporter` plus OpenClaw-managed `mcp.servers`) and does not define any core `preferClientsHub` or `clientsHub.path` policy surface.

## Close Comment

Closing this as a Codex review because it is out of scope for OpenClaw core on current `main`. The current MCP surface is intentionally limited to OpenClaw-managed `mcp.servers` definitions (`src/config/types.mcp.ts:23`, `src/config/zod-schema.ts:230`, `docs/cli/mcp.md:356`), and `VISION.md:72` plus `VISION.md:105` now explicitly says MCP stays on the external `mcporter` bridge path rather than adding first-class MCP runtime/policy in core. A hub-first MCP client router still looks useful, but the right home is the external MCP bridge/hub layer rather than OpenClaw core.
