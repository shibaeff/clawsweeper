## Summary

Keep open. Current `main` does not include the PR change, and the repo evidence does not support an auto-close: `memory-core` still lacks `enabledByDefault`, while the existing config/slot logic already treats `memory-core` as the default memory slot. That mismatch means the proposed one-line fix may be useful, but it is not already implemented and it is not safe to dismiss as obsolete from current code alone.
