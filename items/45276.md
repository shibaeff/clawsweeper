## Summary

Current `main` does not include this fix. `isLocalGatewayAddress()` still compares tailnet IPv6 values by raw lowercased strings, so equivalent IPv6 forms such as expanded vs compressed addresses can still fail to match.

## Evidence

- **IPv6 tailnet comparison still uses raw lowercased strings on main:** `src/gateway/net.ts:221`-`225` still does `normalizeLowercaseStringOrEmpty(ip) === normalizeLowercaseStringOrEmpty(tailnetIPv6)` instead of comparing normalized IP values. The IPv4 branch at `src/gateway/net.ts:217`-`219` also still lacks the PR's `normalizeIp(tailnetIPv4)` change.
  - file: [src/gateway/net.ts:221](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/gateway/net.ts#L221)
