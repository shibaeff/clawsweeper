## Summary

Keep open. Current `main` still only detects `[[tts:...]]` directives and `[[tts:text]]...[[/tts:text]]` blocks, so plain `[[tts]]...[[/tts]]` shorthand is not implemented and would remain visible text. Docs and prompt hints also still describe only the long-form syntax, so this item is neither fixed nor obsolete.

## Evidence

- **Parser early-exit still ignores plain tags:** `parseTtsDirectives` returns immediately unless the text matches `[[tts:`. Plain `[[tts]]...[[/tts]]` has no colon, so it never reaches any stripping or TTS-text extraction logic.
  - file: [src/tts/directives.ts:65](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/tts/directives.ts#L65)
  - command: `nl -ba src/tts/directives.ts | sed -n '56,86p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Only `[[tts:text]]` blocks are recognized:** The only block-form regex on `main` is `\[\[tts:text\]\]...\[\[/tts:text\]\]`; there is no alias for plain `[[tts]]...[[/tts]]`.
  - file: [src/tts/directives.ts:79](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/tts/directives.ts#L79)
