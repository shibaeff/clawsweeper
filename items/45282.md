## Summary

Keep open. Current main still lacks the core behavior this PR is trying to add: session stats are only refreshed after reset-style runs, not after every normal chat turn, and the session-list API/UI still has no pinning path for the active session during that refresh. The branch is also not a conservative cleanup as-is because it mixes the UI fix with unrelated provider, docs, and cron changes.

## Evidence

- **Normal turns still do not trigger a sessions refresh:** `refreshSessions` is derived from `isChatResetCommand(message)`, so ordinary chat sends do not add their run id to `refreshSessionsAfterChat`. That means the post-turn refresh path is still limited to reset/new-style commands on current main.
