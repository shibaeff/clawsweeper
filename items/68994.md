## Summary

Keep open. Current `main` still preserves stale cross-root absolute `sessionFile` paths, so the reported Docker/systemd `EACCES` failure remains plausible, and the PR itself has an unresolved Node fallback syntax bug noted in review.

## Evidence

- **Current main keeps stale cross-root session paths unchanged:** `resolvePathWithinSessionsDir()` still returns `resolveStructuralSessionFallbackPath()` unchanged for cross-root absolute paths. The adjacent test still expects a stored `/original/state/.../agents/bot2/sessions/sess-1.jsonl` path to remain unchanged when `OPENCLAW_STATE_DIR` is `/different/state`, which matches the bug report rather than fixing it.
  - file: [src/config/sessions/paths.ts:219](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/config/sessions/paths.ts#L219)
  - command: `nl -ba src/config/sessions/paths.ts | sed -n '180,280p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Test suite documents the unfixed behavior on main:** The current test `resolves cross-agent paths when OPENCLAW_STATE_DIR differs from stored paths` still asserts `expect(sessionFile).toBe(bot2Session)`, so main explicitly preserves the stale absolute path instead of rebasing it.
  - file: [src/config/sessions.test.ts:618](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/config/sessions.test.ts#L618)

If `sessionFile` still points at a stale `/home/node/...` root while the runtime is using another HOME, this is the path that can trigger the reported `EACCES`.
  - file: [src/config/sessions/transcript.ts:25](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/config/sessions/transcript.ts#L25)

- **Current Docker setup has no stale-path normalization step:** `scripts/docker/setup.sh` currently goes from the ownership fix straight into onboarding and gateway config sync; there is no normalization pass for persisted absolute paths before onboarding on `main`.
  - file: [scripts/docker/setup.sh:503](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/scripts/docker/setup.sh#L503)

Reproducing the same construct with `node -` fails with `SyntaxError: Illegal return statement`, which would be fatal under the script's `set -euo pipefail` if `python3` is unavailable and Node is used.
