## Summary

Current `main` already implements WebChat/Control UI clipboard-image handling end to end: pasted images are captured in the chat composer, preserved through send, encoded into `chat.send.attachments`, and covered by gateway attachment tests. I could not determine a released tag that definitively shipped this yet from the shallow checkout, so this looks fixed on `main` with no reliable shipped release identified.
