## Summary

Keep open. Current `main` still builds the final agent response payloads from raw `assistantTexts` and `attempt.lastAssistant`, while `before_message_write` hooks can mutate or block the persisted transcript. The proposed PR has a plausible core fix, but it also has an unresolved review note about returning a prior-turn assistant when the latest user write is blocked, so there is not strong evidence for an auto-close.

## Evidence

- **Current main still builds final payloads from raw attempt fields:** `runEmbeddedPiAgent()` passes `attempt.assistantTexts` and `attempt.lastAssistant` straight into `buildEmbeddedRunPayloads()`; it does not switch to a post-hook transcript-derived assistant when `before_message_write` hooks are present.
