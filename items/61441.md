## Summary

Current `main` has only a partial version of this fix. The lightweight bootstrap mode/run-kind is honored in the embedded attempt path, but the command system-prompt path and compaction runtime context still drop those fields, so the PR’s core behavior is not fully implemented on `main` yet.

## Evidence

- **Bootstrap selection is mode/run-kind sensitive on main:** `resolveBootstrapContextForRun` still accepts `contextMode` and `runKind`, and the adjacent tests prove lightweight heartbeat and lightweight cron choose different bootstrap files. That means callers must forward those fields correctly for behavior to match the requested mode.
  - file: [src/agents/bootstrap-files.ts:255](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/bootstrap-files.ts#L255)
  - command: `nl -ba src/agents/bootstrap-files.ts | sed -n '240,360p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Main still has a regression test proving the lightweight modes differ:** `bootstrap-files.test.ts` expects heartbeat-only files in lightweight heartbeat mode and an empty bootstrap set in lightweight cron mode, confirming the omitted fields are behaviorally significant rather than cosmetic.
  - file: [src/agents/bootstrap-files.test.ts:134](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/bootstrap-files.test.ts#L134)
  - command: `nl -ba src/agents/bootstrap-files.test.ts | sed -n '120,165p'`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Commands system-prompt path still drops the bootstrap options:** `resolveCommandsSystemPromptBundle()` currently calls `resolveBootstrapContextForRun()` with only `workspaceDir`, `config`, `sessionKey`, and `sessionId`. It does not pass `params.opts?.bootstrapContextMode` or derive/pass a run kind, so this PR's command-path fix is not present on `main`.
  - file: [src/auto-reply/reply/commands-system-prompt.ts:39](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/auto-reply/reply/commands-system-prompt.ts#L39)

- **Compaction runtime context still cannot carry bootstrap mode/run-kind:** `EmbeddedCompactionRuntimeContext` and `buildEmbeddedCompactionRuntimeContext()` do not define or populate `bootstrapContextMode` / `bootstrapContextRunKind`. The timeout/overflow compaction paths build runtime context through this helper, so those compaction paths still lose the lightweight bootstrap settings on `main`.
