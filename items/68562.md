## Summary

Keep open. Current `main` still has the reported behavior: `generateAndAppendDreamNarrative()` falls back on `RequestScopedSubagentRuntimeError`, returns early when `runId` is `null`, then still unconditionally calls `subagent.deleteSession()` in `finally`, which can throw the same request-scoped error outside a gateway request. The runtime contract and current tests both support the report rather than disproving it.

## Evidence

- **Fallback path returns `null`, but cleanup still runs unconditionally:** `startNarrativeRunOrFallback()` catches `RequestScopedSubagentRuntimeError`, writes the fallback narrative, logs the request-scoped warning, and returns `null`. In `generateAndAppendDreamNarrative()`, the outer body returns early on `!runId`, but the `finally` block still always calls `await params.subagent.deleteSession({ sessionKey })` and warns on any cleanup error without filtering the request-scoped case.
