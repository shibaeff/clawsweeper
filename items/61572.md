## Summary

Keep open. Current `main` does not already implement the PR's stricter provider/target separation or compact Discord read behavior, and the review discussion identified a concrete regression risk against current `main`: ACP tool-message edits still send `to`, so the PR's blanket legacy-target rejection would break an existing internal caller unless that path is updated too.

That means the PR's stated removal of legacy aliases/current-context fallback is not already present on `main`.
