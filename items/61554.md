## Summary

Current `main` already contains the gateway handshake fixes that address this report: the pre-auth/connect-challenge timeout is now 10s by default, the timeout is configurable, and the CLI gateway path yields an event-loop turn before starting the WebSocket handshake. The changelog shows this shipped in `v2026.3.22`, after the reporter's `2026.3.13` version.
