## Summary

Keep open. Current `main` still does not implement the existing-config tool-profile preservation from this PR, and the model-auth preflight is not present as proposed. The Spark-specific runtime problem is partly overtaken because `gpt-5.3-codex-spark` is now treated as a removed model, but that makes this PR stale in implementation details, not safely closable as already implemented.

"coding"`, so rerunning local setup on an existing config with no explicit profile still changes effective tool policy from implicit `full` to explicit `coding`.
