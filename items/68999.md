## Summary

Current `main` already routes `session_status` through the same transcript-backed usage fallback as `/status`, with tests covering non-zero context recovery from sparse session metadata. The report matches older behavior on `2026.4.15`, but the inspected main branch now backfills context/token usage instead of staying at `0%`.

## Evidence
