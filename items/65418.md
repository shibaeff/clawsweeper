## Summary

Keep open. Current `main` still mutates signed Anthropic thinking text in the transport layer, which directly contradicts the repo's own immutable-replay policy/tests for Anthropic thinking blocks. I did not find a shipped fix or an active runtime recovery path wired into production code.

## Evidence

- **Signed Anthropic thinking replay is still sanitized before send:** The Anthropic transport still applies `sanitizeTransportPayloadText` to `block.thinking` even when replaying a signed `type: "thinking"` block with `signature: block.thinkingSignature`. That is the exact mutation the report describes.
  - file: [src/agents/anthropic-transport-stream.ts:337](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/anthropic-transport-stream.ts#L337)

  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
