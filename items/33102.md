## Summary

Keep open. Current `main` still has no config-backed default for `openclaw tui --deliver`; the CLI flag remains hard-coded default-off, the TUI runtime falls back to `false` when the flag is omitted, and the typed config surface does not expose any `tui` or equivalent deliver-default setting.

## Evidence

- **CLI still defines `--deliver` as default false:** `openclaw tui` still registers `--deliver` with Commander default `false`, so omitting the flag does not consult config.
  - file: [src/cli/tui-cli.ts:19](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/cli/tui-cli.ts#L19)
  - command: `nl -ba src/cli/tui-cli.ts | sed -n '10,25p'`
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **TUI runtime still falls back to false:** `runTui()` computes `deliverDefault` as `opts.deliver ?? false`; there is no config lookup on this path.
  - file: [src/tui/tui.ts:315](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/tui/tui.ts#L315)
