## Summary

Keep open. Current `main` still defaults `tools.exec.notifyOnExitEmptySuccess` to opt-in only, so empty-output successful background execs still do not notify/wake by default. The PR’s main behavior change is not implemented on `main`, although its proposed 50ms wake coalesce is partly stale because `main` already uses `coalesceMs: 0`.
