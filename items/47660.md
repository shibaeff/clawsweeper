## Summary

Keep open. This PR is not conservative cleanup against current `main`: it removes shipped Docker build contracts (`OPENCLAW_EXTENSIONS`, `OPENCLAW_VARIANT`, digest pinning, staged extension dependency baking) that current docs, setup flow, smoke coverage, and release workflows still rely on, and the existing review discussion already identified a concrete regression around selective extension dependency baking.

## Evidence

- **Current Dockerfile still defines the removed contracts:** `main` still documents and implements opt-in extension dependency baking and runtime variant selection: `OPENCLAW_EXTENSIONS`/`OPENCLAW_VARIANT` are declared, `ext-deps` extracts selected extension manifests, and the runtime stage still uses `FROM base-${OPENCLAW_VARIANT}` with pinned base-image digests. That means the PR is removing active behavior, not dead complexity.
