## Summary

Keep open. Current `main` still does not implement the PR’s ACP spawn-history fix: the spawn path only ensures a transcript path exists, then dispatches the ACP task; the actual user prompt is only appended later during ACP turn persistence, so immediate `sessions_history` can still be empty. The linked discussion reporting empty ACP histories is still consistent with current code.

## Evidence
