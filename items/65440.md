## Summary

Keep open. Current main does not implement Android-side HTTP relay, and the shipped remote-node/browser routing surfaces are narrower workarounds rather than this feature request.

## Evidence

- **Android node command inventory lacks HTTP relay:** The Android app exposes canvas, camera, sms, location, device, notifications, system, photos, contacts, calendar, motion, and call-log commands only; there is no `http.request` or similar node command to receive gateway HTTP traffic.
