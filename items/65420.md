## Summary

Current `main` already contains unreleased iMessage self-chat/echo-loop hardening that no longer relies only on `message.is_from_me`, and the new regression coverage matches the reported DM echo-loop failure mode.

## Evidence

- **Inbound processing no longer treats self-chat solely as `is_from_me`:** Current `main` distinguishes true self-chat from ambiguous DM rows using `destination_caller_id`, remembers ambiguous self-thread reflections, and only drops ordinary outbound rows as `from me` when the self-chat conditions are not met.
  - file: [extensions/imessage/src/monitor/inbound-processing.ts:210](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/imessage/src/monitor/inbound-processing.ts#L210)
  - command: `nl -ba extensions/imessage/src/monitor/inbound-processing.ts | sed -n '210,258p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Reflected DM replies are suppressed via self-chat and echo caches:** After access checks, inbound processing still drops self-chat reflected duplicates and scoped echo-cache matches even when the message is not handled by the initial `is_from_me` branch.
  - file: [extensions/imessage/src/monitor/inbound-processing.ts:334](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/imessage/src/monitor/inbound-processing.ts#L334)
  - command: `nl -ba extensions/imessage/src/monitor/inbound-processing.ts | sed -n '334,367p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Outbound iMessage replies populate the echo cache immediately after send:** `deliverReplies` remembers each sent text/media reply in `sentMessageCache` right after `sendMessageIMessage(...)`, which is the fallback path that prevents reflected outbound replies from re-entering as fresh inbound messages.
  - file: [extensions/imessage/src/monitor/deliver.ts:57](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/imessage/src/monitor/deliver.ts#L57)

- **Regression tests cover the self-chat/echo cases behind this report:** The self-chat regression suite verifies `is_from_me=true` self-chat handling, ambiguous/missing `destination_caller_id`, and reflected duplicate suppression for self-chat DM flows.
  - file: [extensions/imessage/src/monitor/self-chat-dedupe.test.ts:341](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/imessage/src/monitor/self-chat-dedupe.test.ts#L341)
  - command: `nl -ba extensions/imessage/src/monitor/self-chat-dedupe.test.ts | sed -n '341,677p'`
