## Summary

Current `main` already repairs an installed-but-unloaded macOS Gateway LaunchAgent on both `openclaw gateway start` and `openclaw gateway restart`, so the reported "fails until re-install" behavior is implemented on `main`. The newer code explicitly re-bootstraps launchd from the existing plist instead of requiring `gateway install` again, and that path is covered by unit and integration tests.
