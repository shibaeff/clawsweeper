## Summary

Current main still has the duplication path the issue describes. OpenClaw injects workspace bootstrap files into Project Context on every turn, bare `/new` and `/reset` still instruct the agent to run Session Startup and read required files, and the shipped personal-assistant AGENTS template still tells the agent to reread `SOUL.md`, `USER.md`, and `MEMORY.md` before responding. There is newer template guidance to use runtime-provided startup context first, but it is not the active default behavior everywhere and does not enforce deduplication.
