## Summary

Current `main` already suppresses the reported false provenance warnings for allowlisted plugins, includes regression coverage for both direct allowlist and runtime-source-snapshot cases, and the fix was shipped in `v2026.4.23`.

## Evidence

- **Loader skips provenance warning for allowlisted plugin ids:** `warnAboutUntrackedLoadedPlugins(...)` returns early when `allowSet.has(plugin.id)`, so a loaded non-bundled plugin in `plugins.allow` no longer emits the `loaded without install/load-path provenance` warning.
