## Summary

Keep open. Current `main` has backend support for sanitized/truncated approval text and separate command previews, but the Control UI prompt path still ignores that metadata, renders the raw `request.command`, and always shows an `Always allow` button even when the server may reject that decision. This is not strong evidence that issue #58687 is fixed on `main`.
