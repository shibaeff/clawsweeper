## Summary

Keep open. Current `main` appears to have a partial mitigation for the user-message/session reload race, but the CLI assistant-final path still looks unresolved: CLI runs emit assistant+lifecycle events, `server-chat` still broadcasts the final message without persisting it to the local transcript, WebChat still reloads history on final, and non-Claude CLI history still falls back to local transcript only.

## Evidence

- **CLI runs still finish through agent events, not a gateway transcript append path:** For CLI providers, `runAgentTurnWithFallback` calls `runCliAgent`, emits an `assistant` event with the final text, then emits a lifecycle `end` event with `startedAt`/`endedAt`. I did not find a local transcript append in this path.
