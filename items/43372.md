## Summary

Current `main` already carries the tool-call lifecycle handling this report describes: the embedded runner surfaces pending tool calls, the OpenResponses gateway preserves them as `function_call` items instead of finalizing the run away, and tests cover the `previous_response_id` + `function_call_output` continuation path.
