## Summary

Current `main` still intentionally prepends Feishu inbound metadata like `[message_id: ...]` and sender identifiers into `BodyForAgent`, including reply/quote context. That behavior is covered by tests and documented in the changelog, so there is not strong evidence that issue #61621 has been fixed or become obsolete.

## Evidence

- **Feishu inbound body still injects raw metadata tags:** `buildFeishuAgentBody` still formats quoted replies as `[Replying to: ...]`, prefixes the speaker, and then prepends `[message_id: ...]` on its own line.
  - file: [extensions/feishu/src/bot.ts:155](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/feishu/src/bot.ts#L155)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Injected metadata is still passed through as agent-facing body on dispatch:** The Feishu dispatch path still sends `messageBody` as `BodyForAgent`, so the inline metadata survives into the agent turn on current `main`.
  - file: [extensions/feishu/src/bot.ts:983](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/feishu/src/bot.ts#L983)

  - file: [extensions/feishu/src/bot.test.ts:1636](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/feishu/src/bot.test.ts#L1636)
