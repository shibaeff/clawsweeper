## Summary

Keep open. Current `main` still validates/setup-applies channel input incrementally and writes config immediately; it does not expose the deferred candidate-config validation hooks or CLI/setup input surface this PR proposes. The PR discussion also identified an account-id propagation risk that current `main` still structurally shares when `resolveAccountId` depends on later-collected input.
