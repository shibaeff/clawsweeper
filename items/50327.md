## Summary

Current main already avoids provider-usage-only auto-compaction. It estimates prompt/context size from transcript content before prompt submission, triggers preemptive compaction or tool-result truncation from that estimate, and `/status` now requests transcript-usage fallback when provider usage is missing. The shipped `v2026.4.22` release tag also contains this implementation.

## Evidence

## Close Comment

Closing from a Codex review because current OpenClaw already no longer depends on provider-reported `usage` to trigger auto-compaction. `src/agents/pi-embedded-runner/run/preemptive-compaction.ts:19` estimates prompt tokens from transcript/system/user content, and `src/agents/pi-embedded-runner/run/attempt.ts:2276` uses that estimate to trigger precheck compaction or tool-result truncation before the provider returns anything. `/status` also now enables transcript-usage fallback in `src/agents/tools/session-status-tool.ts:598`, covered by `src/agents/openclaw-tools.session-status.test.ts:411`. I also checked the shipped `v2026.4.22` release commit (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), and it already contains this implementation, so this issue is implemented on main and shipped.
