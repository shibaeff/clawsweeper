## Summary

Keep open. Current `main` does not implement quoted/replied Feishu media download: the handler builds `mediaPayload` before fetching the parent message, and fetched Feishu messages do not retain raw JSON content needed to extract `image_key`/`file_key`. The existing tests only cover text-only quoted messages.

## Evidence

- **Quoted media cannot be added to agent payload on current main:** `handleFeishuMessage` resolves inbound media from the live message, immediately calls `buildAgentMediaPayload(mediaList)`, and only afterward fetches `ctx.parentId` via `getMessageFeishu`. There is no later rebuild of `mediaPayload`, so quoted/replied media is never attached to the finalized inbound context.
  - file: [extensions/feishu/src/bot.ts:723](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/extensions/feishu/src/bot.ts#L723)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '712,772p'`
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Fetched quoted messages lose the raw Feishu media keys:** `parseFeishuMessageItem` reads `item.body?.content` into `rawContent`, but only returns parsed `content` and `contentType`. The raw JSON string is discarded, so callers cannot recover `image_key` / `file_key` for a quoted media message.
