## Summary

Keep open: current `main` still omits `agentSessionKey` when `extensions/memory-wiki` calls the shared memory search path, while the memory search contract and shipped CLI/QMD path both rely on `sessionKey` for scope-aware searches. The PR is still addressing a live bug on `main` rather than obsolete work.

## Evidence

- **Current main still drops the session key in memory-wiki shared search:** `searchMemoryWiki()` forwards `agentSessionKey` into `getActiveMemorySearchManager(...)` but then calls `sharedMemoryManager.search(params.query, { maxResults })` without `sessionKey`, so the shared backend does not receive the session context on current `main`.
