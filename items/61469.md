## Summary

Current `main` still couples `stream:"thinking"` WS emission to `reasoningMode: "stream"` plus `onReasoningStream`, so the PR’s core bug claim is not implemented on `main`. The PR looks directionally useful, but the existing review discussion surfaced real edge cases around `message_end` coverage and suppressed-output turns, so this is not strong close evidence.

## Evidence

- **Current main still gates reasoning emission behind stream mode and callback:** `emitReasoningStream()` returns early when `state.streamReasoning` is false or `params.onReasoningStream` is absent, before broadcasting the `stream: "thinking"` agent event. That means WS clients do not get live thinking updates unless the session is already in explicit reasoning-stream mode with a callback.
