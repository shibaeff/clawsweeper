## Summary

Current `main` still shows the pre-fix behavior that this PR targets. The exec tool update path still calls `onUpdate` without handling a returned rejected promise, and memory-core still uses millisecond-based narrative session keys, has no session-corpus file-size cap, and writes the short-term recall store without an entry cap. This PR remains actionable and should not be auto-closed.
