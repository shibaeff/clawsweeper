## Summary

Keep open. Current `main` still has the reported gap: `models.list` loads the full gateway catalog and only applies `agents.defaults.models` allowlist filtering, while docs still define `models.mode: "replace"` as using only configured providers. The Control UI fetches its dropdown from `models.list`, and there is no current test covering `models.mode` filtering on that RPC path.

## Evidence

- **Gateway handler still returns full catalog fallback:** `models.list` loads `context.loadGatewayModelCatalog()`, builds an allowlist from `buildAllowedModelSet(...)`, then returns `allowedCatalog` only when that allowlist is non-empty; otherwise it falls back to the full `catalog`. There is no `models.mode` branch here.
