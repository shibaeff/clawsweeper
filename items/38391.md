## Summary

Keep open. Current `main` does not implement account-level Slack `slashCommand.nativeNames`; Slack native naming is still limited to the bundled plugin's fixed provider override (`status` -> `agentstatus`). The PR is stale against the moved `extensions/slack` layout, but the requested behavior is not already implemented on `main`, so there is no strong close reason.
