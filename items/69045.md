## Summary

Current main already documents and implements the third-party channel-plugin path this issue asks about: channel plugins should use the public Plugin SDK, ship an `openclaw.plugin.json` manifest, and be published/discovered through ClawHub or npm rather than through OpenClaw core issue tracking.

## Evidence

- **Vision keeps optional integrations out of core:** Repo vision says optional capability should usually ship as plugins, with npm/local plugin distribution as the preferred path, and that plugin discovery/promotion belongs in ClawHub rather than OpenClaw core.

  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/docs/plugins/building-plugins.md#L16)
