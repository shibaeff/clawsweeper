## Summary

The requested brainstorm behavior is not implemented on `main`, but the request fits OpenClaw's documented plugin/ClawHub boundary rather than core. `hooks.internal.entries.brainstorm` is only a config selector for discovered hooks; it does not create a built-in brainstorm implementation. Current docs and code point toward shipping this as a community plugin or hook package that uses message/subagent plugin hooks for mention targeting and multi-round orchestration.

## Evidence

- **Vision pushes optional capability to plugins/ClawHub, not core:** OpenClaw's vision says core should stay lean, optional capability should usually ship as plugins, and plugin promotion belongs in ClawHub with a high bar for adding optional plugins to core.
  - file: [VISION.md:52](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/VISION.md#L52)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Plugin getting-started guide says external plugins should be published outside this repo:** The plugin guide says new capabilities should be built as plugins, published to ClawHub or npm, and installed with `openclaw plugins install`; it explicitly says the plugin does not need to be added to the OpenClaw repository.
  - file: [docs/plugins/building-plugins.md:11](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/plugins/building-plugins.md#L11)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Plugin hooks already cover this routing/orchestration surface:** The plugin hooks reference documents hooks for inbound message claiming, message rewrite/cancel, reply dispatch, and subagent routing/completion delivery, which are the extension points needed for @mention targeting and multi-round brainstorm coordination.
  - file: [docs/plugins/hooks.md:10](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/plugins/hooks.md#L10)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Internal hook config only filters discovered hooks; it does not define built-ins:** `hasConfiguredInternalHooks` and `resolveConfiguredInternalHookNames` treat `hooks.internal.entries.*` as enabled names, and `loadInternalHooks` only loads discovered hook entries whose names match that configured set. A config key like `brainstorm` does not create an implementation by itself.
  - file: [src/hooks/loader.ts:90](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/hooks/loader.ts#L90)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Bundled hook docs list only four shipped internal hooks:** The hooks CLI docs show the bundled/discovered example list as `boot-md`, `bootstrap-extra-files`, `command-logger`, and `session-memory`; there is no built-in `brainstorm` hook in the documented shipped set.
  - file: [docs/cli/hooks.md:26](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/cli/hooks.md#L26)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Repo search found no brainstorm plugin or hook on main:** A repo search for `\bbrainstorm\b` across `src`, `extensions`, `docs`, `README.md`, and `VISION.md` returned no matches, and `find src/hooks/bundled -maxdepth 2 -name HOOK.md` listed only the four bundled hooks above.
  - command: `rg -n "\bbrainstorm\b" src extensions docs README.md VISION.md -g '!**/*.svg' ; find src/hooks/bundled -maxdepth 2 -name HOOK.md | sort`
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Current mention gating is group-level trigger policy, not per-participant brainstorm moderation:** Group message docs describe `requireMention` as group-level reply triggering. Feishu docs likewise describe `channels.feishu.groups.<chat_id>.requireMention` as a per-group @mention requirement, not selective per-agent group-discussion routing.
  - file: [docs/channels/groups.md:24](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/channels/groups.md#L24)

## Risks / Open Questions

- The current `hooks.internal.entries.<name>` UX can mislead users into thinking arbitrary names correspond to built-in hooks; a separate docs/validation cleanup could still be worthwhile.
- Closing as ClawHub leaves the underlying user need unresolved unless someone publishes and maintains the brainstorm plugin.
