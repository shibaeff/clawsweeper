## Summary

Current `main` does not honor `OPENCLAW_WRAPPER` or expose a `--wrapper` install flag. Gateway service install still rebuilds LaunchAgent `ProgramArguments` from resolved runtime + CLI entrypoint, docs list only `--port/--runtime/--token/--force/--json`, and doctor compares entrypoints without any wrapper-specific signal. The provided issue timeline also shows an open follow-up PR (#69425), so this is active, not obsolete.

## Evidence

There is no branch for `OPENCLAW_WRAPPER` or any wrapper path override.
