## Summary

Current `main` already contains a post-release fix that prevents assistant commentary / `<think>` content from being shown as user-visible output. The fix is present in commit `de07739e4056bc21424908ae5ebd030fd8371b91` on `main`, but it does not appear to be in the latest tagged release `v2026.4.23`, so this is fixed on `main` and unreleased rather than still open.

## Evidence

- **Current main prefers only final-answer assistant text:** `extractAssistantVisibleText()` now returns `final_answer` text when present and refuses to fall back to commentary-only assistant text.
  - file: [src/shared/chat-message-content.ts:182](https://github.com/openclaw/openclaw/blob/de07739e4056bc21424908ae5ebd030fd8371b91/src/shared/chat-message-content.ts#L182)
  - command: `nl -ba src/shared/chat-message-content.ts | sed -n '182,188p'`
  - sha: [de07739e4056](https://github.com/openclaw/openclaw/commit/de07739e4056bc21424908ae5ebd030fd8371b91)
- **Streaming path suppresses output unless a `<final>` block is seen when enforcement is enabled:** `stripBlockTags()` strips `<think>` content and, in strict final-tag mode, returns an empty string until a `<final>` start tag is observed, explicitly to prevent leaking thinking-out-loud text.
  - file: [src/agents/pi-embedded-subscribe.ts:517](https://github.com/openclaw/openclaw/blob/de07739e4056bc21424908ae5ebd030fd8371b91/src/agents/pi-embedded-subscribe.ts#L517)
  - command: `nl -ba src/agents/pi-embedded-subscribe.ts | sed -n '517,593p'`
  - sha: [de07739e4056](https://github.com/openclaw/openclaw/commit/de07739e4056bc21424908ae5ebd030fd8371b91)
- **UI assistant text extraction suppresses commentary-only messages:** Control UI message extraction uses the shared assistant-visible extractor, and assistant-role text is sanitized through shared scaffolding stripping before render/export/search paths consume it.
  - file: [ui/src/ui/chat/message-extract.ts:20](https://github.com/openclaw/openclaw/blob/de07739e4056bc21424908ae5ebd030fd8371b91/ui/src/ui/chat/message-extract.ts#L20)
  - command: `nl -ba ui/src/ui/chat/message-extract.ts | sed -n '20,28p'`
  - sha: [de07739e4056](https://github.com/openclaw/openclaw/commit/de07739e4056bc21424908ae5ebd030fd8371b91)
- **UI sanitization strips `<think>` and related internal scaffolding:** `stripThinkingTags()` delegates to `stripAssistantInternalScaffolding()`, which is the shared sanitizer used for assistant-visible text.
  - file: [ui/src/ui/format.ts:83](https://github.com/openclaw/openclaw/blob/de07739e4056bc21424908ae5ebd030fd8371b91/ui/src/ui/format.ts#L83)
  - command: `nl -ba ui/src/ui/format.ts | sed -n '83,85p'`
  - sha: [de07739e4056](https://github.com/openclaw/openclaw/commit/de07739e4056bc21424908ae5ebd030fd8371b91)
- **Regression tests cover the reported leak shape:** Tests now assert that commentary-only assistant text is not rendered, `<final>` enforcement suppresses output without a start tag, and `<think>` / `<final>` segments are stripped from visible text.
  - file: [src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.filters-final-suppresses-output-without-start-tag.test.ts:12](https://github.com/openclaw/openclaw/blob/de07739e4056bc21424908ae5ebd030fd8371b91/src/agents/pi-embedded-subscribe.subscribe-embedded-pi-session.filters-final-suppresses-output-without-start-tag.test.ts#L12)
