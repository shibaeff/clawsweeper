## Summary

PR #43433 is a docs-only discoverability change for a third-party plugin, but current main explicitly routes community plugin discovery to ClawHub instead of accepting source-doc listing PRs. The requested listing is therefore out of scope for OpenClaw core/docs maintenance and should be handled through ClawHub publishing/listing.

## Close Comment

Thanks for the PR. Closing after Codex review because this is now out of scope for OpenClaw source docs rather than missing implementation.

Current `main` explicitly treats ClawHub as the canonical discovery surface for community plugins: `docs/plugins/community.md:14-16` says not to open docs-only PRs just to add a plugin for discoverability, and `docs/plugins/community.md:147-153` repeats that docs PRs should only be used for actual source-doc changes. The broader project direction in `VISION.md:54-61` also keeps optional plugin capability in external plugin repos rather than growing core/docs curation.

Since this PR only adds a third-party plugin entry to `docs/plugins/community.md`, the right path is to publish/list the Memori plugin on ClawHub instead of maintaining a source-doc listing PR here.
