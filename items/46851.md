## Summary

Keep open. Current main has adjacent liveness features like typing indicators, preview tool-progress edits, and lifecycle status reactions, but I found no implementation of the requested configurable fixed-interval per-turn heartbeat with start/running/done-failed text updates, per-channel overrides, or templates.

## Evidence

- **Streaming docs describe event-driven previews, not fixed-interval heartbeats:** Channel UX is documented as block streaming plus preview-message edits. The same doc says tool-progress updates are short status lines shown while tools run, following the active preview mode; it does not define any strict every-N-seconds cadence.
  - file: [docs/concepts/streaming.md:12](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/concepts/streaming.md#L12)

- **Per-channel config is limited to preview tool-progress toggles:** Telegram, Discord, and Slack docs expose `streaming.preview.toolProgress` for reusing the preview message during tool activity. That is channel-specific preview behavior, not a global/per-channel fixed-interval heartbeat lifecycle with templates.
  - file: [docs/channels/telegram.md:274](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/channels/telegram.md#L274)
