## Summary

Keep open. Current `main` does not implement this change: `browser screenshot` still always routes through `imageResultFromFile(...)`, which reads the saved file and embeds image data in the tool result. The proposed `embed` parameter is not present on `main`, and a path-only default would also need extra follow-up because current media extraction relies on `details.media` or image content, not a bare JSON `{path}` result.
