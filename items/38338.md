## Summary

Current `main` does not implement this fix. Runtime DM routing still falls back to `"main"` in multiple live paths, while onboarding still defaults new configs to `"per-channel-peer"`, so the reported mismatch remains real. The PR should stay open, though its patch is stale against current code because one outbound fallback now lives in `src/infra/outbound/base-session-key.ts` instead of the file touched in the PR.
