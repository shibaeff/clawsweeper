## Summary

Keep open. Current `main` does not implement this PR’s requested Tailscale pairing bypass, and the docs explicitly state the opposite behavior: Control UI still requires one-time pairing even on the same Tailnet with `gateway.auth.allowTailscale: true`. That rules out `implemented_on_main`, and none of the other allowed close reasons fit.
