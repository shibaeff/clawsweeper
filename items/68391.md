## Summary

Keep open. Current `main` still accepts `models --agent <id>` at the parent command, but `models set` and `models set-image` do not read agent scope and still write the global defaults under `agents.defaults.*`. The underlying per-agent `agents.list[].model` config surface exists, so the reported behavior is still a real gap rather than obsolete confusion.
