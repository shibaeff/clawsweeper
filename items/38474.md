## Summary

Keep open. Current `main` does not implement Bailian TTS/STT, and the codebase has moved to plugin-manifest speech/media provider registries, so this PR's goal is still valid but its implementation targets stale internals rather than the current extension-based architecture.

## Evidence

- **TTS is registry-driven on main:** Gateway TTS provider selection/status now resolves providers from the registered speech-provider registry (`canonicalizeSpeechProviderId`, `getSpeechProvider`, `listSpeechProviders`) instead of a hard-coded provider enum, so the feature would need to land as a registered speech provider.
