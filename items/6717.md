## Summary

Current main already implements agent-driven natural-language model switching via the `session_status(model=...)` tool path, and the same behavior is present in released tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **System prompt teaches model-use switching:** The agent system prompt explicitly describes `session_status` as the tool for model-use questions and says it supports an optional per-session model override.
  - file: [src/agents/system-prompt.ts:507](https://github.com/openclaw/openclaw/blob/cc295fb8c9a3e775549838a23fff7dbfb72081bf/src/agents/system-prompt.ts#L507)
  - sha: [cc295fb8c9a3](https://github.com/openclaw/openclaw/commit/cc295fb8c9a3e775549838a23fff7dbfb72081bf)

## Risks / Open Questions

- The shipped behavior is per-session rather than true per-message/per-turn routing, so that narrower 'ideally' scope would still be a separate enhancement.

## Close Comment

Closing from a ClawSweeper/Codex review. Current `main` already implements this well enough through the agent-facing `session_status(model=...)` path: the system prompt tells the agent to use `session_status` for model-use questions (`src/agents/system-prompt.ts:507`), the tool contract explicitly exposes a per-session model override (`src/agents/tool-description-presets.ts:44`), and the tool persists the override with `liveModelSwitchPending` so the runtime can switch cleanly (`src/agents/tools/session-status-tool.ts:473`, `src/agents/agent-command.ts:942`). This is covered by regression tests (`src/agents/openclaw-tools.session-status.test.ts:778`) and is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Per-message granularity is still a possible future enhancement, but the core request here is already implemented on main.
