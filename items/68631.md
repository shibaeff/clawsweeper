## Summary

Keep open. Current `main` already supports per-agent `tools.allow`/`tools.deny` and per-agent filesystem scoping via `tools.fs.workspaceOnly`, so the first half of the request is partly obsolete. But the requested fine-grained `tools.path.allow/deny` glob controls and command-specific `exec` deny semantics are not implemented in the current schema or enforcement code, so this is still a real feature gap rather than something safe to auto-close.
