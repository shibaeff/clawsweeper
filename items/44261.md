## Summary

Keep open. Current `main` still does not implement automatic Control UI/webchat mirroring back to the last external channel. The browser chat controller sends `chat.send` with `deliver: false`, the gateway explicitly blocks webchat from inheriting external delivery routes by default, and reply-routing tests assert that internal webchat turns do not route externally unless delivery is made explicit. That means the requested behavior is not implemented on `main`, even if related routing-preservation bugs were fixed.
