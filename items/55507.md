## Summary

Keep open. Current `main` still accepts `memorySearch.outputDimensionality` in config and forwards it into provider selection, but the active OpenAI embedding path drops that value before the remote `/embeddings` request is built, so the PR’s requested behavior is not implemented on `main` yet.

## Evidence

- **Config still accepts the setting:** The runtime schema still accepts `agents.defaults.memorySearch.outputDimensionality` as a positive integer, so the setting remains part of the public config surface.
  - file: [src/config/zod-schema.agent-runtime.ts:682](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/config/zod-schema.agent-runtime.ts#L682)

- **Gateway passes the field into provider creation:** `handleOpenAiEmbeddingsHttpRequest` forwards either request `dimensions` or `memorySearch.outputDimensionality` into `createConfiguredEmbeddingProvider`, so current main still intends to carry this option through the embeddings stack.
  - file: [src/gateway/embeddings-http.ts:303](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/embeddings-http.ts#L303)
