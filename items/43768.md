## Summary

Current `main` already implements the reported subagent-model behavior: explicit `sessions_spawn.model` wins, `agents.defaults.subagents.model` is used as the default, the child session is patched with that selection, and the resolved runtime model is persisted before the child run starts. Regression tests on `main` cover both the model-selection plan and the pre-run child-session model persistence.

## Evidence

- **Subagent model selection order:** `resolveSubagentSpawnModelSelection(...)` now resolves subagent spawn models in the expected order: explicit override first, then configured subagent default, then agent/global default.
  - file: [src/agents/model-selection.ts:247](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/model-selection.ts#L247)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Spawn plan includes resolved child model:** `resolveSubagentModelAndThinkingPlan(...)` always carries the resolved child model into `initialSessionPatch.model`, so native subagent spawns pass the chosen model into the child session patch path.
  - file: [src/agents/subagent-spawn-plan.ts:35](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/subagent-spawn-plan.ts#L35)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
