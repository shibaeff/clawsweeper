## Summary

Keep open. Current `main` still renders assistant bubble action buttons without ever adding the `has-copy` class that reserves right-side padding, so the reported text-overlap bug remains plausible and appears unfixed in both `main` and the latest release `v2026.4.23`.

## Evidence

- **Bubble markup still omits `has-copy`:** `ui/src/ui/chat/grouped-render.ts:1379-1418` builds `bubbleClasses` from `"chat-bubble"`, optional `"streaming"`, and `"fade-in"` only. `hasActions` is computed at `1414`, but the returned bubble at `1417` never adds a class tied to those actions.
  - file: [ui/src/ui/chat/grouped-render.ts:1379](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/ui/src/ui/chat/grouped-render.ts#L1379)
  - command: `nl -ba ui/src/ui/chat/grouped-render.ts | sed -n '1360,1455p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **CSS padding exists only behind unreachable class:** `ui/src/styles/chat/grouped.css:199-223` defines `.chat-bubble.has-copy { padding-right: 70px; }`, while `.chat-bubble-actions` is absolutely positioned at the top-right. Without the `has-copy` class, message text can occupy the same space as the action buttons.
  - file: [ui/src/styles/chat/grouped.css:199](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/ui/src/styles/chat/grouped.css#L199)
  - command: `nl -ba ui/src/styles/chat/grouped.css | sed -n '182,226p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Latest release still contains the same mismatch:** The tagged release `v2026.4.23` shows the same `bubbleClasses` construction in `ui/src/ui/chat/grouped-render.ts:1379-1418` and the same `.chat-bubble.has-copy` rule in `ui/src/styles/chat/grouped.css:199-223`, so this was not already fixed in the latest shipped version.
  - file: [ui/src/ui/chat/grouped-render.ts:1379](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/ui/src/ui/chat/grouped-render.ts#L1379)
  - command: `git show v2026.4.23:ui/src/ui/chat/grouped-render.ts | nl -ba | sed -n '1370,1428p'; git show v2026.4.23:ui/src/styles/chat/grouped.css | nl -ba | sed -n '190,224p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **No test currently guards this class hookup:** A repo search finds `has-copy` only in the stylesheet, not in renderer tests or runtime code outside the CSS selector, so there is no evidence that a regression test covers this layout contract.
  - file: [ui/src/styles/chat/grouped.css:199](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/ui/src/styles/chat/grouped.css#L199)
  - command: `rg -n "has-copy|classList\.(add|toggle)\(['\"]has-copy|chat-bubble-actions" ui/src -S`
