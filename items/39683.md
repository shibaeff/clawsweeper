## Summary

Keep open. Current `main` does not already implement this create/update-time Feishu/Lark rejection, so `implemented_on_main` does not fit. But the PR's blanket `delivery.to` check no longer matches current cron delivery architecture on `main`: core validation is intentionally generic, explicit channel jobs can inherit a prior session route instead of requiring `to`, plugin-owned target resolution now handles missing/default targets, and the existing review already identified a regression where legacy jobs without `delivery.to` could no longer be disabled. This looks like a real cleanup idea that now needs a different implementation, not an auto-close.
