## Summary

Keep open. Current main makes Feishu DM and group conversations separate session buckets by design, so the cross-session-memory part of the report is not clearly a current bug. But I did not find strong evidence that the reported Feishu multi-agent group-delivery loss is fixed on main, and the current Feishu broadcast path still treats only one agent as the visible replying agent while other agents run as silent observers.

## Evidence
