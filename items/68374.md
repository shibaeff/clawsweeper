## Summary

Keep open. Current `main` still documents OpenResponses `reasoning` as ignored, the `/v1/responses` SSE implementation only emits assistant text/content-part events on this path, and reasoning visibility remains documented as a separate channel-style `Reasoning:` message with `stream` limited to Telegram. That is strong evidence the requested HTTP reasoning exposure is not implemented on `main`.
