## Summary

Current `main` has partial overlap with the request: OpenClaw already implements a silent pre-compaction memory flush, compaction lifecycle hooks, brief start/end notices, and checkpoint snapshots/branching. But I did not find the requested configurable pre-warning events or any bounded compaction deferral mechanism/tool, so the issue is not implemented well enough to close.

## Evidence

- **Docs explicitly mark pre-compaction memory flush as implemented:** The compaction reference says OpenClaw already uses a pre-threshold memory flush before auto-compaction, with configurable `softThresholdTokens`, `prompt`, and `systemPrompt`, and notes that it runs once per compaction cycle.
  - file: [docs/reference/session-management-compaction.md:324](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/reference/session-management-compaction.md#L324)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Runtime runs a silent pre-compaction handoff-like turn:** `runMemoryFlushIfNeeded()` computes a soft threshold from context window minus reserve floor and `softThresholdTokens`, then launches a silent embedded agent run with `trigger: "memory"`, a dedicated `memoryFlushWritePath`, and compaction event tracking. This covers part of the proposed handoff/persist-state behavior.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:535](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/auto-reply/reply/agent-runner-memory.ts#L535)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Compaction lifecycle hooks already exist:** OpenClaw emits `session:compact:before` / `session:compact:after` internal hook events and also runs plugin `before_compaction` / `after_compaction` hooks with token/message metrics.
  - file: [src/agents/pi-embedded-runner/compaction-hooks.ts:185](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/pi-embedded-runner/compaction-hooks.ts#L185)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **User notices are only start/end/incomplete notices:** `notifyUser` only sends `🧹 Compacting context...`, `🧹 Compaction complete`, or `🧹 Compaction incomplete` once compaction is already in progress or done; it is not a configurable early warning threshold system.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:639](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/auto-reply/reply/agent-runner-execution.ts#L639)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **No compaction-specific warning-threshold or deferral API found:** A repo search for `compaction_defer`, `preCompactionTurn`, `warningThreshold`, and `criticalThreshold` in compaction-related source/docs found only the unrelated tool-loop detection thresholds. I did not find a compaction deferral tool/policy or configurable pre-warning event surface on current `main`.
