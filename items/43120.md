## Summary

Keep open. Current `main` does have the isolated-cron skill plumbing: cron builds a workspace skill snapshot for isolated runs, embedded runs load workspace skill entries, and minimal prompt mode still includes the skills catalog. But the reported Windows behavior is still plausible because skill eligibility depends on `PATH` and daemon/service environment, while `openclaw skills list` can be run from a different interactive shell environment. I do not have strong enough evidence to say the report is already fixed or obsolete.
