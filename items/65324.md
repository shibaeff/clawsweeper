## Summary

Current `main` does not contain this WSL2 `DISPLAY` fallback, and the PR as proposed still has a credible regression risk called out in review: it would overwrite an existing user `DISPLAY` value while adding tests that do not exercise the real Chrome launch path. Conservative cleanup should leave this PR open for maintainer judgment rather than auto-close it.
