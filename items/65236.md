## Summary

Current `main` already defaults `minimax-portal` to the Anthropic Messages transport, and the runtime maps that API family to `.../v1/messages` rather than `/responses`.

## Evidence

- **MiniMax portal catalog defaults to Anthropic Messages:** `buildMinimaxPortalProvider()` returns `api: "anthropic-messages"` with `authHeader: true`, so the bundled portal provider no longer defaults to OpenAI Responses routing.
  - file: [extensions/minimax/provider-catalog.ts:79](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/minimax/provider-catalog.ts#L79)

- **OAuth config patch persists the same transport:** The MiniMax OAuth flow writes `models.providers.minimax-portal.api = "anthropic-messages"` and `authHeader: true` into the config patch returned from auth setup.
  - file: [extensions/minimax/provider-registration.ts:147](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/minimax/provider-registration.ts#L147)
