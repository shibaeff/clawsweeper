## Summary

Current `main` now treats committed messaging delivery as a replay-invalid side effect, short-circuits the incomplete-turn retry/warning path when delivery or a tool error already happened, and blocks the remaining silent-error retry whenever `hadPotentialSideEffects` is true. Regression tests on `main` cover the exact committed-delivery-plus-tool-error scenario described in #69448. The latest tagged release `v2026.4.23` predates these changes, so there is no shipped release tag to cite yet.

## Evidence
