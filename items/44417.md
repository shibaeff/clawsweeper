## Summary

Current `main` does not implement the reported fix. The Linux systemd helper still prefers `systemctl --machine <SUDO_USER>@ --user` whenever `SUDO_USER` is a non-root user, before trying plain `systemctl --user`, and the current tests explicitly lock in that behavior. That matches the issue report rather than showing it resolved on `main`.

## Evidence

- **Non-root SUDO_USER still forces machine scope first:** `execSystemctlUser` reads `env.SUDO_USER` and, when it is set to a non-root user, immediately returns `execSystemctl([..."--machine", `${user}@`, "--user", ...args])` before any direct `--user` call.
  - file: [src/daemon/systemd.ts:374](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/systemd.ts#L374)
  - command: `nl -ba src/daemon/systemd.ts | sed -n '330,410p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Gateway availability/install path uses that helper:** Both `isSystemdUserServiceAvailable` and `assertSystemdAvailable` call `execSystemctlUser(env, ["status"])`, so `openclaw gateway install` and related lifecycle commands still inherit the same sudo/machine-scope behavior.
  - file: [src/daemon/systemd.ts:400](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/systemd.ts#L400)

  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Tests enforce no direct fallback under sudo:** Current tests assert that when `SUDO_USER` is set, OpenClaw should use machine scope and should not fall back to direct `systemctl --user` if machine scope fails.
  - file: [src/daemon/systemd.test.ts:189](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/systemd.test.ts#L189)
  - command: `nl -ba src/daemon/systemd.test.ts | sed -n '188,205p'`

  - file: [src/daemon/systemd.test.ts:844](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/systemd.test.ts#L844)
