## Summary

Current `main` already implements the reported behavior: `apply_patch` is a shipped core tool in the `coding` profile, the bundled-tool policy pass explicitly suppresses false unknown-entry warnings for valid core tools, and regression tests cover both cases. The same implementation is present in release `v2026.4.22`.

## Evidence

- **`apply_patch` is a core `coding`-profile tool:** `src/agents/tool-catalog.ts` defines `apply_patch` as a built-in filesystem tool with `profiles: ["coding"]`, so it is not a plugin-only entry.
  - file: [src/agents/tool-catalog.ts:76](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/tool-catalog.ts#L76)
