## Summary

Keep open. Current `main` has partial session-lock hardening, but it still queues same-session callers, does not expose a `SESSION_BUSY`/machine-readable busy contract, and still flattens downstream lock failures to generic gateway `UNAVAILABLE` errors. That is not strong enough to say issue #69112 is implemented on `main`.

## Evidence

- **Agent loop docs still describe serialized same-session execution:** The current docs say runs are serialized per session key and transcript writes are protected by a session write lock. That matches the reported contention surface rather than a shipped busy/retry contract.
  - file: [docs/concepts/agent-loop.md:46](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/docs/concepts/agent-loop.md#L46)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **ACP manager still routes turns through a per-session actor:** `AcpSessionManager.runTurn()` enters `withSessionActor(...)` for the session key, so concurrent callers are still serialized instead of getting an immediate busy response.
  - file: [src/acp/control-plane/manager.core.ts:699](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/acp/control-plane/manager.core.ts#L699)
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Concurrency test proves queueing, not busy signaling:** The current test starts `r1`, then `r2`, asserts the runtime only has one in-flight turn until the first releases, and only then allows the second turn to run. That is explicit queueing behavior on `main`.
  - file: [src/acp/control-plane/manager.test.ts:410](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/acp/control-plane/manager.test.ts#L410)

- **Gateway still collapses downstream errors to generic UNAVAILABLE:** When the background agent run fails, the gateway wraps it as `errorShape(ErrorCodes.UNAVAILABLE, String(err))` and returns only a generic error plus summary text. That does not preserve a machine-readable busy/session-lock reason for CLI callers.
  - file: [src/gateway/server-methods/agent.ts:291](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/gateway/server-methods/agent.ts#L291)

That matches the linked follow-up direction in the issue timeline rather than a completed fix on `main`.
  - file: [src/config/sessions/store.ts:620](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/config/sessions/store.ts#L620)
