## Summary

Current Slack plugin code already preserves inbound Slack thread replies and sends replies back into the same thread by passing `thread_ts` through the reply pipeline. The same implementation is present in the `v2026.4.23` release tree, so the report no longer matches current shipped behavior.

## Evidence
