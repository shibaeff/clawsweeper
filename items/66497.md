## Summary

Keep open. Current `main` still strips inline directive tags in the websocket chat path without trimming, so the whitespace mismatch described in #66497 remains plausible and there is no evidence in the checked-out code that the fix shipped.

## Evidence

- **Gateway delta path still lacks trim:** `emitChatDelta` still uses `stripInlineDirectiveTagsForDisplay(...).text` for both `text` and `delta` before merge, with no `.trim()` on either value.
  - file: [src/gateway/server-chat.ts:691](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/gateway/server-chat.ts#L691)
