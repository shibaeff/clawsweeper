## Summary

Keep open. Current `main` still handles TUI `Ctrl+C` as local clear/warn/exit only, without checking for an active run or telling the user to use `Esc` or `/abort`, so the reported confusion is not implemented away on `main`.

## Evidence

- **Ctrl+C handler still shows generic clear/exit messages:** `handleCtrlC()` sets only `cleared input; press ctrl+c again to exit` or `press ctrl+c again to exit`. It does not inspect active-run state or mention `Esc`/`/abort`.
  - file: [src/tui/tui.ts:972](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/tui/tui.ts#L972)
  - command: `nl -ba src/tui/tui.ts | sed -n '972,991p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Ctrl+C decision helper has no active-run branch:** `resolveCtrlCAction()` only returns `clear`, `warn`, or `exit` from `hasInput`, `now`, and `lastCtrlCAt`; there is no parameter or branch for `activeChatRunId` or abort guidance.
  - file: [src/tui/tui.ts:261](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/tui/tui.ts#L261)
