## Summary

Keep open. Current `main` still intentionally gives explicit CLI gateway calls the full legacy operator scope bundle, which matches the reported repeated scope-upgrade behavior for gateway-backed commands. Telegram native approvals still start a separate `operator.approvals` gateway client, surface `pairing required` before hello, and the bootstrap retries/logs on failure, so the persistent pending/log-noise half of the report also remains plausible. One reproduction example in the issue body is outdated on current `main`: `openclaw config get` is a local config-file read now, not a gateway RPC.
