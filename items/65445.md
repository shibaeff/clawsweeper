## Summary

Keep open. Current `main` does not implement the requested default session-maintenance garbage collection for orphaned archive artifacts. Normal maintenance deletes stale `.deleted.*` archives and can prune `.reset.*` via `resetArchiveRetention`, but generic orphan/artifact cleanup including `.bak.*` only happens under the optional disk-budget path, and the existing integration test explicitly shows old `.bak` artifacts surviving a normal maintenance pass.

## Evidence

- **Normal maintenance only cleans deleted/reset archives:** `saveSessionStoreUnlocked()` archives removed session transcripts as `deleted` artifacts, then calls `cleanupArchivedSessionTranscripts()` only for `reason: "deleted"` and `reason: "reset"`. There is no unconditional `bak` cleanup in the regular maintenance path.

  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Current integration test proves old .bak artifacts remain after maintenance:** The integration test seeds an old `.jsonl.deleted.*`, a recent `.jsonl.deleted.*`, and an old `.jsonl.bak.*`; after `saveSessionStore(storePath, store)`, the old deleted archive is removed but the `.bak` archive is still expected to exist.
  - file: [src/config/sessions/store.pruning.integration.test.ts:151](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/config/sessions/store.pruning.integration.test.ts#L151)

- **Artifact cleanup for orphaned/bak files is gated behind optional disk budget:** `enforceSessionDiskBudget()` returns immediately unless `maxDiskBytes` and `highWaterBytes` are configured. Only in that budget-enforcement path does it scan the sessions directory and remove archive artifacts or unreferenced primary transcripts.

There is no `garbageCollect`, `garbageCollectRetention`, or recycle-path option on current `main`.
