## Summary

Keep open. Current `main` still writes `delivery-mirror` assistant transcript entries and still serves them through `chat.history`; the current sanitizer only drops commentary/suppressed-control assistant messages, not `delivery-mirror` rows. The PR is stale against the refactored gateway history code, but the underlying bug is not implemented on `main`. The existing PR discussion only raised a cosmetic JSDoc issue and missing test coverage, not a reason to close it.
