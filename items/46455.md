## Summary

Keep open. Current `main` only shipped the narrower zero-value poll-default fix, not this PR's broader send-path cleanup: `src/poll-params.ts` still treats non-zero numeric poll params and boolean `true` values as poll intent, `src/infra/outbound/message-action-runner.ts` still rejects `action="send"` when those params are present, and `src/infra/outbound/message-action-runner.send-validation.test.ts` explicitly locks in rejection for `pollDurationSeconds: "60"` plus `pollPublic: "true"`. The `components` half of the PR is also stale against current `main`, because the shared message tool now expects `presentation` and asserts `components` is absent, so this branch needs a rebase/rescope rather than an auto-close.
