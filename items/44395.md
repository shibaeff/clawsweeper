## Summary

Keep open. Current `main` still implements memory chunking as line-by-line token/character accumulation with only `tokens` and `overlap` settings, and there is no heading-aware/section-aware chunking option in code, config, or tests. Related PRs that attempted this (`#45545`, `#51913`) were closed unmerged.

## Evidence

- **Current chunker is still line-based only:** `chunkMarkdown` in `src/memory-host-sdk/host/internal.ts` splits `content` by newline, accumulates segments until `maxChars`, and flushes only on size overflow. There is no heading or section boundary logic.
