## Summary

Current `main` still shows a single static `doctor` update note and does not wire the existing per-step update spinner into `maybeOfferUpdateBeforeDoctor`, so this PR addresses a real gap that is not yet implemented on `main`.

## Evidence

- **Doctor path still uses static note only:** `maybeOfferUpdateBeforeDoctor` prints `Running update (fetch/rebase/build/ui:build/doctor)…` and calls `runGatewayUpdate` without a `progress` controller or `try/finally` cleanup. That matches the PR's reported gap.
  - file: [src/commands/doctor-update.ts:55](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/commands/doctor-update.ts#L55)

- **Existing spinner helper is already available on main:** `createUpdateProgress` exists in the update CLI and is designed for exactly this step-level progress reporting, including a `stop()` cleanup hook.
  - file: [src/cli/update-cli/progress.ts:99](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/src/cli/update-cli/progress.ts#L99)
