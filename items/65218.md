## Summary

Current `main` contains a targeted fix for this small-context embedded-heartbeat overflow path: it reapplies configured Pi compaction overrides after settings reload, honors `reserveTokens`/`reserveTokensFloor` from config, and clamps precheck reserve against the actual context budget. The repo also carries regression coverage for the exact `reserveTokensFloor: 0` small-model case and records the fix in the `v2026.4.15` changelog.

## Evidence

- **Config override now honors explicit reserveTokens and reserveTokensFloor with context-aware cap:** `applyPiCompactionSettingsFromConfig()` reads configured `reserveTokens`/`reserveTokensFloor`, caps the floor against the resolved context budget, and computes the target reserve from the configured value instead of leaving Pi's default 16384 in place.

- **Embedded runner reapplies compaction settings after Pi reload:** The runner now immediately reapplies OpenClaw compaction overrides after `DefaultResourceLoader.reload()`, with an inline comment explaining that reload can drop the earlier overrides. That directly addresses the reported runtime mismatch between config and the live reserve value.

- **Precheck no longer lets oversized reserve starve the whole prompt budget:** `shouldPreemptivelyCompactBeforePrompt()` now computes `effectiveReserveTokens` as the minimum of requested reserve and the maximum reserve that still leaves the required prompt budget, instead of effectively treating a 16K reserve on a 16K model as a one-token prompt budget.

That is the exact class of failure reported here.

- **Regression test covers clamped precheck behavior on 16K-class contexts:** A dedicated precheck test now asserts that a too-large reserve on a small context window is clamped so the prompt still fits, with `effectiveReserveTokens` and `promptBudgetBeforeReserve` both reduced to a safe value instead of immediately overflowing.

That also answers the reporter's follow-up asking whether `4.14` contained the fix: the repo records it under `2026.4.15`, not `2026.4.14`.
