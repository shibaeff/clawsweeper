## Summary

Keep open. Current `main` already absorbed part of this batch (Codex-error sanitization and safer subagent completion-route handling), but it does not include the Discord `Unknown Channel` permanent-error rule or a `chat.history` delivery-mirror filter, and the PR still has two unresolved review regressions called out in discussion. The plugin duplicate-warning change also conflicts with current tested `main` behavior.

## Risks / Open Questions

- Merging the PR as-is risks reintroducing the review-noted completion-routing regression for completion flows with an internal requester origin plus an explicit external completion target.
- The `chat.history` change as proposed would still risk returning fewer visible messages than requested whenever filtered delivery-mirror entries occupy the tail, because slicing happens first.
- The plugin-registry part would remove a currently tested duplicate-warning diagnostic on `main`, so the batch PR is not a clean merge candidate even aside from the missing fixes.
