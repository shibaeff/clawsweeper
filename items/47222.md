## Summary

Keep open. Current `main` does not implement this fix: the generated systemd unit still uses `KillMode=control-group`, the unit test still asserts that value, and the gateway run loop still converts every `SIGTERM` directly into shutdown with no startup-specific guard. The PR also still needs maintainer judgment because its proposed `KillMode` change would not update already-installed systemd units automatically, so even a merge would leave existing Linux users on the old behavior until reinstall.
