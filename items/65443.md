## Summary

Current `main` already fixes the built-in Telegram TTS delivery path: successful `tts` results are carried as structured queued media with `audioAsVoice`, the legacy `Generated audio reply.\nMEDIA:/tmp/reply.opus` case has an explicit regression test, and pending voice media is flushed before agent-end completion. The fix is on `main` but appears newer than the latest stable release `v2026.4.23`.

## Evidence
