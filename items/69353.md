## Summary

Keep open. Current `main` still hard-gates `context1m` to `provider === "anthropic"`, current tests intentionally preserve the non-Anthropic exclusion, and the linked PR #69426 is still only a cross-referenced open candidate rather than a merged fix.

## Evidence

- **Core resolver still blocks proxied providers:** `isAnthropic1MModel()` returns `false` unless the provider is exactly `anthropic`, and `resolveContextTokensForModel()` only returns 1M when that helper passes. A `github-copilot` Anthropic model with `params.context1m = true` still falls through to the normal context window on current `main`.
  - file: [src/agents/context.ts:377](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/agents/context.ts#L377)

- **Current tests encode the exclusion:** `src/agents/context.test.ts` still expects `provider: "github-copilot", model: "claude-opus-4.7"` to stay at `128_000` rather than 1M, which confirms the non-Anthropic-provider gate is still the intended current behavior on `main`.
  - file: [src/agents/context.test.ts:314](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/agents/context.test.ts#L314)
