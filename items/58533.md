## Summary

Current `main` already honors isolated cron `payload.model` ahead of stored session overrides, forwards the resolved model into execution, and ships that behavior in `v2026.4.23`. The thundering-herd root cause described in the issue does not match the current codepath.

## Evidence

- **Payload model wins over stored session override:** `resolveCronModelSelection()` resolves `payload.model` first and only consults `sessionEntry.modelOverride` when no payload model is present.

## Risks / Open Questions

- The issue body also asked for broader scheduler burst jitter. Current shipped behavior documents automatic staggering for recurring top-of-hour cron expressions plus explicit per-job `staggerMs`, not a blanket spread for every same-tick cron burst.
