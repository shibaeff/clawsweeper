## Summary

Current main already freezes workspace bootstrap file contents per session via a bootstrap snapshot cache, invalidates that snapshot on session rollover, and the same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Per-session bootstrap snapshot cache:** `src/agents/bootstrap-cache.ts:5` caches `WorkspaceBootstrapFile[]` by `sessionKey`; `src/agents/bootstrap-cache.ts:14` loads from disk once, and `src/agents/bootstrap-cache.ts:15` stores the snapshot for reuse.
  - file: [src/agents/bootstrap-cache.ts:5](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/bootstrap-cache.ts#L5)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Normal runs use the cached snapshot:** `src/agents/bootstrap-files.ts:229` routes bootstrap loading through `getOrLoadBootstrapFiles(...)` whenever a `sessionKey` exists, so later turns reuse the same in-memory snapshot instead of re-reading workspace files.
  - file: [src/agents/bootstrap-files.ts:229](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/bootstrap-files.ts#L229)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Workspace files are read eagerly into content-bearing records:** `src/agents/workspace.ts:477` loads workspace bootstrap files, and `src/agents/workspace.ts:518` onward reads file contents into `WorkspaceBootstrapFile.content`, which is what the cache retains for the session.
  - file: [src/agents/workspace.ts:477](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/workspace.ts#L477)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Snapshot invalidates on session rollover:** `src/auto-reply/reply/session.ts:491` calls `clearBootstrapSnapshotOnSessionRollover(...)` before starting a new session, so a new session gets a fresh bootstrap snapshot.
  - file: [src/auto-reply/reply/session.ts:491](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/auto-reply/reply/session.ts#L491)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)

  - file: [src/agents/bootstrap-cache.test.ts:41](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/bootstrap-cache.test.ts#L41)

## Close Comment

Codex review: current main already implements the requested session-stable workspace bootstrap behavior. `src/agents/bootstrap-cache.ts:5` caches bootstrap files per `sessionKey`, `src/agents/bootstrap-files.ts:229` uses that cache for normal runs instead of re-reading the workspace every turn, and `src/auto-reply/reply/session.ts:491` clears the snapshot on session rollover so the next session reloads from disk. This is covered by `src/agents/bootstrap-cache.test.ts:41` and `src/auto-reply/reply/session.test.ts:1268`, and the same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Closing as already implemented. If you still want a separate user-facing `bootstrapFreezeMode` toggle rather than the current built-in session snapshot behavior, please open a focused follow-up for that config/API request.
