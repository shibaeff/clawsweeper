## Summary

Keep open. Current `main` still runs Active Memory synchronously in `before_prompt_build`, awaits the sub-agent before the main reply, and preserves a default 15s timeout. The behavior is documented and tested as blocking, and the shipped `v2026.4.23` tag matches current `main` on the relevant files, so there is no evidence this performance issue is already fixed.

## Evidence

- **Reply path still blocks on Active Memory:** The plugin registers `before_prompt_build`, builds the recall query, then `await`s `maybeResolveActiveRecall(...)` before returning any prompt prefix. That means reply generation waits for recall to finish or time out.
