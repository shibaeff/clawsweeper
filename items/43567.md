## Summary

Keep open. Current main does not implement the requested distinct Control UI treatment for system/runtime messages. System-role messages are still rendered through the normal grouped chat bubble path, only labeled via the footer, while the TUI already renders them as separate dim system lines.

## Evidence

- **Control UI maps system messages to generic "other" chat groups:** `renderMessageGroup` preserves `system` as the sender label, but assigns any non-user/non-assistant/non-tool role to `roleClass = "other"`, so system messages do not get a dedicated system container/class.
  - file: [ui/src/ui/chat/grouped-render.ts:303](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/ui/src/ui/chat/grouped-render.ts#L303)

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Control UI still renders system messages with the standard chat bubble:** Per-message rendering always uses `chat-bubble` for non-tool messages; there is no system-specific bubble/block branch.
  - file: [ui/src/ui/chat/grouped-render.ts:1379](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/ui/src/ui/chat/grouped-render.ts#L1379)
