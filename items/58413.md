## Summary

Current `main` does not implement automatic or templated per-agent provider request headers for Anthropic-style outbound requests. Anthropic attribution remains explicitly disabled by default, transport turn-state hooks do not receive `agentId`, and provider request header config only accepts literal strings. There is a manual per-agent `models.json` workaround for static headers, but it does not satisfy the issue's requested automatic `X-Agent-ID`/`{{agentId}}` behavior.

## Risks / Open Questions

- There is a partial workaround through per-agent `models.json` provider headers, so maintainers may want to respond with that guidance if they keep the issue open.
- Implementing this cleanly likely needs a new agent-aware transport context or a documented templating contract; current plugin/provider hooks do not expose enough request identity to add it externally.
