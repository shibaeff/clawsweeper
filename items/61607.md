## Summary

Keep open. Current `main` still implements mention gating only for group chats, while direct messages are governed by `dmPolicy`/allowlists without a separate trigger gate. There is no `dmRequireMention` config surface or runtime path on `main`, and the linked PR has not landed into `main` yet.

## Evidence

- **WhatsApp runtime only gates groups before processing:** The inbound handler runs `applyGroupGating(...)` only inside the `msg.chatType === "group"` branch. The DM branch just normalizes sender E.164 and then falls through to `processForRoute(...)`, so authorized DMs continue straight to normal processing.
  - file: [extensions/whatsapp/src/auto-reply/monitor/on-message.ts:102](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/whatsapp/src/auto-reply/monitor/on-message.ts#L102)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Mention gating implementation is explicitly group-only:** `applyGroupGating` resolves `requireMention`, computes mention/reply-based gating, and skips unmentioned traffic only for group conversations. There is no analogous DM gating function or DM mention decision in this code path.
  - file: [extensions/whatsapp/src/auto-reply/monitor/group-gating.ts:98](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/whatsapp/src/auto-reply/monitor/group-gating.ts#L98)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **WhatsApp config types expose DM access but not DM trigger gating:** The shared WhatsApp config includes `dmPolicy`, `allowFrom`, `groupPolicy`, and per-group `requireMention`. The per-direct config only exposes `systemPrompt`; there is no `dmRequireMention` field or equivalent DM trigger-gating type.
  - file: [src/config/types.whatsapp.ts:25](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/types.whatsapp.ts#L25)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **WhatsApp schema validates DM access policy only:** The schema defines `dmPolicy`, `allowFrom`, `groups`, and `direct`, with `requireMention` only on group entries and `systemPrompt` only on direct entries. No schema key or validation exists for `dmRequireMention`.
  - file: [src/config/zod-schema.providers-whatsapp.ts:21](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/config/zod-schema.providers-whatsapp.ts#L21)

They also note that `direct["*"]` only applies after a DM is already admitted by `dmPolicy` plus allowlists/pairing.
  - file: [docs/channels/whatsapp.md:159](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/channels/whatsapp.md#L159)
