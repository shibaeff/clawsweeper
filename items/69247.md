## Summary

Keep open. Current `main` still flags any `startedAt < createdAt` or `endedAt < startedAt` with no tolerance, and current task creation still stamps `createdAt` from a fresh `Date.now()` even when a caller already has `startedAt`. That leaves the reported false-positive path plausible on `main`. This PR also has an unresolved review concern that its proposed `1000ms` tolerance is much broader than the stated jitter problem, so it is not strong evidence for conservative closure.
