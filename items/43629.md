## Summary

Close as implemented on main. Current OpenClaw already documents and implements the exact configuration this issue describes: Telegram has a dedicated `channels.telegram.proxy`, Telegram API timeouts are configurable with `channels.telegram.timeoutSeconds`, and env proxy/`NO_PROXY` behavior is explicitly handled in the network stack. The reporter also confirmed on March 12, 2026 that the configuration solved the problem.
