## Summary

Keep open. Current `main` does not implement Discord `accountId` -> matching agent fallback, and the stale PR still has unresolved correctness issues called out in review, so there is not strong evidence for any auto-close reason.

## Evidence

- **Current router still falls straight from explicit bindings to default:** `resolveAgentRoute` only checks explicit binding tiers (`binding.peer`, `binding.guild`, `binding.team`, `binding.account`, `binding.channel`) and then returns `choose(resolveDefaultAgentId(input.cfg), "default")`; there is no Discord-only implicit account->agent fallback on `main`.
  - file: [src/routing/resolve-route.ts:777](https://github.com/openclaw/openclaw/blob/33d5ebbff7bceb782b314de14ebd9a213ba31869/src/routing/resolve-route.ts#L777)
