## Summary

Keep open. Current `main` still lacks the PR’s core fallback-origin tracking, still persists the last runtime model into session state, and still resolves later runs/display surfaces from those stored runtime fields with no guard to retry the configured primary model.

## Evidence

- **Session entries still have no fallback-origin flag on main:** `SessionEntry` includes `modelProvider` and `model`, but no `modelIsFromFallback`-style field. `setSessionRuntimeModel()` still only writes `provider` and `model`, so current main has no way to distinguish an explicit runtime model from one chosen by fallback.
  - file: [src/config/sessions/types.ts:224](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/config/sessions/types.ts#L224)
  - command: `nl -ba src/config/sessions/types.ts | sed -n '220,380p'`
  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)
- **Agent session persistence still stores the model that actually ran:** `updateSessionStoreAfterAgentRun()` derives `modelUsed`/`providerUsed` from `result.meta.agentMeta` or `fallbackModel`/`fallbackProvider`, then calls `setSessionRuntimeModel(next, { provider: providerUsed, model: modelUsed })` with no fallback marker. That means a fallback-selected model is still persisted as the session runtime model on main.
  - file: [src/agents/command/session-store.ts:69](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/command/session-store.ts#L69)
