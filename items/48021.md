## Summary

Keep open. Current `main` still returns and renders the raw bootstrap-file `name`, so hook-injected bootstrap entries with a valid `path` but missing/empty `name` can still surface as `undefined` in `/context` output. The PR remains relevant, and its review discussion already addressed the Windows-path basename detail.

## Evidence

- **Current implementation still uses raw `file.name`:** `buildBootstrapInjectionStats` on `main` computes `pathValue` and truncation stats, but returns `name: file.name` and `path: pathValue || file.name` with no basename fallback. That matches the bug described in the PR rather than the proposed fix.
  - file: [src/agents/bootstrap-budget.ts:145](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/bootstrap-budget.ts#L145)
  - command: `nl -ba src/agents/bootstrap-budget.ts | sed -n '120,220p'`
  - sha: [151befb90b90](https://github.com/openclaw/openclaw/commit/151befb90b903f52dce9f4951991d39b9df8b358)
- **`/context` output renders the stored name directly:** The context report formats each workspace file line as `- ${f.name}: ...`, so an undefined runtime `name` leaks straight into user-visible `/context list` and `/context detail` output.
  - file: [src/auto-reply/reply/commands-context-report.ts:124](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/auto-reply/reply/commands-context-report.ts#L124)

- **Hook contract allows runtime mutation of bootstrap files:** Docs and hook types expose `context.bootstrapFiles` as a mutable array during `agent:bootstrap`. That means runtime hook code can still append malformed or partially-specified entries even if TypeScript types nominally require `name`.
  - file: [docs/automation/hooks.md:123](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/docs/automation/hooks.md#L123)

- **Sanitization currently guards `path`, not `name`:** `sanitizeBootstrapFiles` only drops entries with missing/invalid `path` and otherwise preserves the object. There is no comparable normalization or fallback for `name`, so a valid-path/missing-name hook result still reaches `buildBootstrapInjectionStats`.
