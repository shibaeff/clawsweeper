## Summary

Keep open. Current `main` and the latest release still key webchat heartbeat suppression off `showOk`, which conflicts with the documented `showAlerts` alert-delivery contract, so this is not implemented on `main`. But the proposed PR also does not look safely closeable as-is: switching the gate to `showAlerts` alone would hide OK-only heartbeat acknowledgments in webchat when `showOk=true` and `showAlerts=false`, so the discussion points to a real bug but not a confidently resolved fix yet.
