## Summary

Keep open. Current main already treats explicit HTTP 500 as failover-eligible, but the broader bug is not clearly fixed: the plain OpenAI/Codex server-error wording reported on 2026-04-20 still has no matcher in the current failover classifiers, so a status-less error string can still bypass fallback.

## Evidence

- **HTTP 500 is already mapped to timeout on main:** `classifyFailoverClassificationFromHttpStatus` returns `timeout` for `500`, `502`, and `504`. This addresses the original narrow code-gap claim but not every server-error surface.
  - file: [src/agents/pi-embedded-helpers/errors.ts:673](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/agents/pi-embedded-helpers/errors.ts#L673)
