## Summary

Keep open. Current `main` has cron failure backoff and alerting after runs fail, but it does not verify provider reachability before loading/scheduling cron jobs or before dispatching an isolated agent turn.

## Evidence

- **Cron startup loads and arms jobs without provider checks:** `start()` clears stale markers, loads persisted jobs, runs missed jobs, and arms the timer. There is no provider/network reachability gate in this path.
  - file: [src/cron/service/ops.ts:87](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/service/ops.ts#L87)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '1,120p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Cron store recompute is schedule-only:** `ensureLoaded()` reloads jobs and calls `recomputeNextRuns(state)`; it normalizes job shape but does not inspect model provider health or endpoint reachability.
  - file: [src/cron/service/store.ts:19](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/service/store.ts#L19)
  - command: `nl -ba src/cron/service/store.ts | sed -n '1,120p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Model selection validates config, not provider availability:** `resolveCronModelSelection()` resolves configured/allowed provider+model refs from catalog and overrides, then returns `{ ok: true, provider, model }`. No reachability probe or health check is performed.
  - file: [src/cron/isolated-agent/model-selection.ts:45](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/model-selection.ts#L45)

  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Cron execution dispatches directly into model run path:** `createCronPromptExecutor()` calls `runWithModelFallback(...)` and then `runCliAgent(...)` or `runEmbeddedPiAgent(...)` for the chosen provider/model. The first provider contact happens during execution, not as a preflight scheduling gate.
  - file: [src/cron/isolated-agent/run-executor.ts:109](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/run-executor.ts#L109)

This reduces retry storms, but it still requires the job to fail first and does not disable scheduling while a provider is offline.
