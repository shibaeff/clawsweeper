## Summary

Current main routes Codex harness compaction through context-engine maintenance and post-compaction side effects, and the newer transcript-derived token freshness logic removes the stale `totalTokens` loop described in #69300.

## Evidence

- **Preflight no longer blocks later memory-flush evaluation:** The main reply flow runs `runPreflightCompactionIfNeeded(...)` and then immediately runs `runMemoryFlushIfNeeded(...)`, so harness compaction no longer short-circuits the later memory-flush pass in the same cycle.

- **Transcript-derived prompt snapshot replaces stale totals:** Preflight compaction is now gated off stale/unknown totals, and the memory-flush path derives prompt tokens from the transcript and persists `totalTokensFresh=true` when it can recover a fresh snapshot.
