## Summary

Keep open. Current `main` exposes generic plugin hooks around dispatch and tool calls, but it still does not define any skill-specific lifecycle hook or event. Skills remain core-owned prompt/tool routing behavior, so this is a coherent core feature request rather than a ClawHub-only idea or an already-implemented capability.

## Evidence

- **Skills are instructions, not a separate lifecycle object:** `docs/tools/skills.md` defines skills as `SKILL.md` instruction folders that teach the agent how to use tools, which explains why current core has no distinct skill runtime event model.
  - file: [docs/tools/skills.md:9](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/tools/skills.md#L9)
  - command: `nl -ba docs/tools/skills.md | sed -n '1,30p'`
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Skill commands still run through core dispatch paths:** User-invocable skills either bypass the model and dispatch directly to a tool, or they are rewritten into model-facing prompt text. Direct tool dispatch passes `skillName`, but there is no separate skill hook fired here.
  - file: [src/auto-reply/reply/get-reply-inline-actions.ts:245](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/auto-reply/reply/get-reply-inline-actions.ts#L245)

- **No skill lifecycle hook exists in the typed hook surface:** `PluginHookName` includes generic hooks like `before_dispatch`, `before_tool_call`, and `after_tool_call`, but there is no `skill:*` hook name on current `main`.
  - file: [src/plugins/hook-types.ts:57](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/plugins/hook-types.ts#L57)

  - file: [src/auto-reply/reply/dispatch-from-config.ts:682](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/auto-reply/reply/dispatch-from-config.ts#L682)

## Risks / Open Questions

- Maintainership could decide the existing generic hooks are sufficient for some slash-command skill use cases, but current `main` still lacks the dedicated skill-level lifecycle requested in the issue.
- Prompt-routed skills do not have a discrete success/failure event today, so closing as implemented would overstate what the current hook surface actually provides.
