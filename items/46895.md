## Summary

Current `main` still intercepts `/model ...` locally in the web UI and `executeModel` only treats an empty argument as the info path, so `/model status` is still routed to `sessions.patch` instead of the existing status/listing UX described elsewhere in the codebase. The PR should stay open because the underlying bug is still present on `main`, and the review discussion shows unresolved correctness concerns about the proposed keyword-reservation approach for real model ids named `status` or `list`.

## Evidence
