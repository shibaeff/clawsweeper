## Summary

Keep open. Current `main` still mishandles `/model status` and `/model list` in the web UI: the UI intercepts `/model` locally, but `executeModel` only treats empty args as info mode and otherwise sends `sessions.patch`. That conflicts with current docs and the agent-side command path, which both define `/model list` and `/model status` as info views rather than model-selection writes.

## Evidence

- **UI intercepts `/model` before agent handling:** `app-chat.ts` routes local slash commands through `dispatchSlashCommand`, so `/model ...` never reaches the agent-side directive handler once marked local.
