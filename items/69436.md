## Summary

Current `main` already provides a public runtime seam for this: plugins can observe same-turn tool outcomes via `after_tool_call` and intercept final outbound delivery via `message_sending` / `reply_dispatch`, with `sessionKey`/`runId` available for correlation. That satisfies the requested enforceable pre-send extension point on `main`, even though it is not a built-in YAML `responseGuards` policy.
