## Summary

Keep open. Current `main` does not show strong evidence that this regression is fixed: gateway session listing still only returns stored session entries, and the chat dropdown only force-adds the currently selected session key, not a missing `main` session. If the active key is a heartbeat session, `main` can still be absent from the dropdown.

## Evidence

- **Gateway list still does not synthesize a main session:** `listSessionsFromStore` starts from `Object.entries(store)` and filters/sorts those rows; it does not add a synthetic `main`/`agent:main:main` row when missing from the store.
