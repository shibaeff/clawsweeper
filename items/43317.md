## Summary

Keep open. Current `main` already has a different pluggable sandbox-backend architecture and a shipped core Docker backend, but it does not include a gVisor backend or the PR’s sandbox hardening/browser helper surface, so the PR is only partially superseded rather than implemented on `main`.

## Evidence

- **Pluggable sandbox backend registry already exists on main:** `src/agents/sandbox/backend.ts:36-85` already provides `registerSandboxBackend`, lookup helpers, and built-in `docker`/`ssh` registrations, which covers the PR’s proposed pluggable interface at a different seam.
  - file: [src/agents/sandbox/backend.ts:36](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/sandbox/backend.ts#L36)

- **Plugin SDK already exposes backend registration for external backends:** `src/plugin-sdk/sandbox.ts:24-43` exports `registerSandboxBackend` and related sandbox backend types from the public SDK, so new sandbox runtimes already have a supported extension point on `main`.
  - file: [src/plugin-sdk/sandbox.ts:24](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/plugin-sdk/sandbox.ts#L24)

- **Runtime creation already resolves arbitrary registered backends:** `src/agents/sandbox/context.ts:155-172` obtains a backend via `requireSandboxBackendFactory(...)` and records it in the registry, and `src/agents/sandbox.resolveSandboxContext.test.ts:89-125` verifies a registered non-docker backend works end-to-end.
  - file: [src/agents/sandbox/context.ts:155](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/sandbox/context.ts#L155)

## Risks / Open Questions

- Closing as `implemented_on_main` would be incorrect because `main` still lacks any gVisor sandbox backend and does not contain the PR’s `ExecBrowserHelper`/hardening modules.
- If this PR is revived, it likely needs a substantial rebase onto the current backend registry/plugin-SDK architecture rather than merging its parallel `ISandboxProvider` abstraction directly.
