## Summary

Current `main` has partial bundle-LSP support for embedded Pi, but it does not implement the requested real-time in-editor diagnostics dashboard, live diagnostic feed, or richer editor-native troubleshooting UX. The ACP docs still describe editor integration as a bridge rather than a full editor runtime, so this remains a plausible feature request that needs product/scope judgment rather than conservative auto-closure.

## Evidence

- **ACP bridge is explicitly not a full editor runtime:** The public ACP docs say `openclaw acp` is a Gateway-backed bridge, not a full ACP-native editor runtime, and the compatibility matrix still marks several richer editor/runtime surfaces as partial or unsupported.
