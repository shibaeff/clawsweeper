## Summary

Current `main` already addresses this report well enough: assistant `MEDIA:` replies are now explicitly documented as secondary to the `message` tool, sandbox reply-media normalization accepts OpenClaw-managed outbound media instead of dropping it as a sandbox escape, blocked media no longer degrades silently, and Telegram still routes resolved media through native upload APIs rather than plain text sends. The relevant fix landed after `v2026.4.23`, so this is fixed on `main` but not in the latest tagged release.

## Evidence
