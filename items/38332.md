## Summary

Keep open. Current `main` does not implement the PR's requested managed-browser `browser.userDataDir` config. The managed launcher still resolves its profile data dir to `~/.openclaw/browser/<profile>/user-data`, while the only `userDataDir` now in schema/docs is `browser.profiles.*.userDataDir` for `existing-session` Chrome MCP attach profiles, which is explicitly attach-only and still lacks managed-browser features.

## Evidence

- **No top-level browser.userDataDir in current config surface:** `BrowserConfig` includes `attachOnly`, `cdpPortRangeStart`, and `profiles`, but no top-level `userDataDir`. The only `userDataDir` field is per-profile on `BrowserProfileConfig`.
  - file: [src/config/types.browser.ts:1](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/config/types.browser.ts#L1)
