## Summary

Keep open: current `main` still only recognizes `${VAR}` env refs in `parseEnvTemplateSecretRef`, and `collectAuthStoreSecrets` still records a resolved ref and then warns on any non-empty auth-profile value anyway, so the reported false positive remains live. The supplied PR context also shows a follow-up commit that addresses the review concern about preserving explicit plaintext findings.
