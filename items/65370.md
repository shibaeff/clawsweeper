## Summary

Keep open. Current `main` still does not implement this PR’s behavior: the chat UI reads per-message cost only from top-level `m.cost`, not `m.usage.cost`; the gateway sanitizer keeps only `cost.total`; and there is no current savings badge path on `main`. The PR discussion also does not show a reason to close it conservatively: the only review finding in the provided timeline was a cosmetic grouped-message aggregation edge case, not a superseding fix or scope rejection.
