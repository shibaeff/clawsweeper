## Summary

Keep open. Current main still treats heartbeat-isolated sessions as their own UI session labels instead of preserving the base session label, so the requested cleanup does not appear implemented.

## Evidence

- **Chat header shows the selected session option label directly:** The control-panel chat header uses the resolved session option label for the active session; there is no heartbeat-specific suppression at render time.
  - file: [ui/src/ui/chat/session-controls.ts:30](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/ui/src/ui/chat/session-controls.ts#L30)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '24,40p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Only exact main keys map to 'Main Session':** Session-name parsing special-cases only `main` and `agent:main:main`. A heartbeat-isolated key like `agent:main:main:heartbeat` is not normalized back to the main-session label.
  - file: [ui/src/ui/chat/session-controls.ts:348](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/ui/src/ui/chat/session-controls.ts#L348)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '344,388p'`

- **Fallback session labels preserve the raw rest/key:** When a session row has no better label/displayName, the UI falls back to `rest` or the raw key. For a heartbeat synthetic session this would preserve the heartbeat-specific identifier rather than the base session label.
  - file: [ui/src/ui/chat/session-controls.ts:603](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/ui/src/ui/chat/session-controls.ts#L603)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '598,616p'`

  - file: [src/config/sessions/types.ts:122](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/config/sessions/types.ts#L122)
