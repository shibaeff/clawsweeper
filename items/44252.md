## Summary

Current `main` already implements the reported behavior well enough: embedded agent runs are executed through the shared model-fallback loop, overloaded embedded failures are escalated into fallback/error paths instead of silently dying, and embedded lifecycle now emits a user-visible error when a run ends in error. The changelog shows the user-visible overload/429 fix shipped in `v2026.3.28`.

## Evidence
