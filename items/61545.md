## Summary

Current `main` already supersedes this PR with a different implementation: captioned media-bearing block replies stay buffered into the final payload when block streaming is disabled, direct-send dedupe still exists for the remaining direct-send cases, and the cron `NO_REPLY` cleanup concern from review is covered in both code and tests.

## Evidence
