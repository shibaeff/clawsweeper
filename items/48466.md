## Summary

Keep open. Current `main` still stops Telegram streaming previews on HTML parse failures in `draft-stream.ts`, while other Telegram delivery paths already recover by retrying plain text. The PR is addressing a real gap and has meaningful review/follow-up work, so conservative cleanup should not close it.
