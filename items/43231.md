## Summary

Current `main` only caches and exposes the current forum topic's name alongside its numeric thread id; it does not provide a first-class topic directory, name-to-id lookup, or `topics`/list action that would let agents route to another Telegram forum topic by human-readable name. The issue is not implemented well enough to close.

## Evidence

- **Only id->name cache exists:** `extensions/telegram/src/topic-name-cache.ts:155` updates cache entries keyed by `chatId + threadId`, and `extensions/telegram/src/topic-name-cache.ts:185` only exposes `getTopicName(chatId, threadId)`. There is no reverse lookup by topic title or API to enumerate topics.

## Risks / Open Questions

- Telegram may not offer a direct server-side topic-name lookup API, so a complete fix may require an OpenClaw-maintained topic catalog/cache plus an agent/tool surface to query it.
- Current partial mitigation (cached `TopicName` for the active thread) can still leave cross-topic routing ambiguous when the target topic has not recently emitted a service message or when multiple topics have similar names.
