## Summary

Keep open. Current `main` already supports `includeLastMessage` in the gateway/TUI path, but the web Control UI sessions list still does not request or render last-message previews or inline timestamps, so this PR is not implemented on `main` and is not obsolete.

## Risks / Open Questions

- Closing this PR would discard a still-missing Control UI feature; there is no equivalent implementation on current `main` for the web sessions list.
- The PR as described still appears to need maintainer review on bounded `includeLastMessage` usage because the protocol explicitly documents per-session transcript reads, and Control UI currently allows `limit=0` (all sessions).
- The submitted PR also bundled an unrelated `package.json` override per the review context, so even if kept open it likely needs cleanup before merge.
