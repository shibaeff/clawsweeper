## Summary

Current `main` already has the runtime SSRF pieces for Nextcloud Talk private-host access, but it does not fully implement this PR’s user-facing setup/docs flow. The request is only partially superseded, so there is not strong enough evidence to close it as implemented.

## Risks / Open Questions

- Closing this as `implemented_on_main` would overstate the current state: runtime support exists, but the PR’s setup/onboarding UX is still absent on `main`.
- The current shipped surface uses `network.dangerouslyAllowPrivateNetwork` plus legacy migration, not the exact flat `allowPrivateNetwork` config shape described in the PR.
- Docs and setup remain inconsistent with the runtime capability, so the PR may still contain useful work even if parts of it are now obsolete.
