## Summary

Keep open. Current `main` does not implement a general config surface to reroute gateway/internal warning messages to a dedicated channel. The config schema only exposes suppression knobs (`messages.suppressToolErrors`, heartbeat-only `suppressToolErrorWarnings`) plus a cron-specific failure destination, and at least one internal warning path still resolves delivery from the active session’s existing channel context.

## Evidence
