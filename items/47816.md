## Summary

Keep open. The PR contains useful work and later commits addressed some early review feedback, but the current branch still has blocking problems: the documented `curl | bash` path hard-fails, the docs and wrapper promise host WebChat without reconciling OpenClaw's loopback-by-default gateway contract, the new page is not wired into Mintlify navigation, and `cmd_status` still `source`s a user-controlled password file.

## Evidence

- **Documented one-line install path still fails:** The doc tells users to run `curl -sSL https://raw.githubusercontent.com/openclaw/openclaw/main/scripts/virtualbox/install.sh | bash`, but the final PR script only succeeds when it can resolve a local checked-out repo and otherwise exits with `openclaw-vm must be run from the openclaw repository`. A quick shell reproduction also shows `BASH_SOURCE[0]` is empty when Bash reads from stdin, so the advertised pipe-to-bash flow cannot satisfy the script's local-path check.
