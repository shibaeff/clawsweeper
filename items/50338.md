## Summary

Keep open. Current `main` already appears to carry the inbound Telegram `message_id` through the generic reply-threading pipeline, so the PR's stated root cause does not line up with the code on `main`; however, I do not have strong enough proof to close it because there is no Telegram-specific regression test on `main` that exercises the exact implicit `replyToMode: "all"` path this PR targets, and the PR branch adds narrowly-scoped coverage plus follow-up edge-case fixes.
