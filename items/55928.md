## Summary

Current `main` does not implement the PR’s HTTP guard. The `/sessions/:key/history` handler still authorizes `chat.history` and then immediately canonicalizes/loads the target session without blocking internal `subagent`, `cron`, or `acp` namespaces, so the proposed fix is not stale or already shipped.
