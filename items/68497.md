## Summary

Keep open. Current `main` has partially changed the reported behavior by injecting `window.openclawSendUserAction` into canvas pages, so the old `Bridge: missing` symptom is no longer a good close signal. But the requested browser fallback is still not implemented: the helper only posts to native iOS/Android bridges, the canvas WebSocket is used only for live reload, and there is no HTTP `/__openclaw__/canvas/action` route on current `main`.

## Evidence
