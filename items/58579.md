## Summary

Current `main` already performs an automatic post-update gateway service refresh and restart for auto-updates, including a macOS LaunchAgent re-enable/bootstrap/kickstart path, so the requested behavior is implemented on `main` even though the latest tagged release predates it.

## Evidence

- **Auto-update reuses update command:** Background auto-update runs `openclaw update --yes --channel <...> --json`, so the normal update pipeline is the path used for automatic updates.
  - file: [src/infra/update-startup.ts:237](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/infra/update-startup.ts#L237)
  - command: `nl -ba src/infra/update-startup.ts | sed -n '232,295p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Update command refreshes loaded service:** When restart is enabled and the gateway service is loaded, the updater sets `refreshGatewayServiceEnv = true`, then calls `refreshGatewayServiceEnv`, which runs `gateway install --force` from the updated install root.
  - file: [src/cli/update-cli/update-command.ts:256](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/cli/update-cli/update-command.ts#L256)

  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Update command auto-restarts service:** The updater automatically prepares restart work for a loaded service and executes a detached restart helper or daemon restart without requiring a manual follow-up command.
  - file: [src/cli/update-cli/update-command.ts:1395](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/cli/update-cli/update-command.ts#L1395)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '1388,1432p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **macOS restart helper repairs LaunchAgent:** On macOS the detached post-update helper runs `launchctl enable`, falls back to `launchctl bootstrap`, and then `launchctl kickstart -k` for the gateway LaunchAgent, matching the requested reinstall/restart behavior after updates.
  - file: [src/cli/update-cli/restart-helper.ts:98](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/cli/update-cli/restart-helper.ts#L98)
  - command: `nl -ba src/cli/update-cli/restart-helper.ts | sed -n '98,138p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Tests cover refresh and macOS launchd restart:** Current tests assert that updates refresh service env when already installed and that the macOS restart helper generates a launchd restart script with enable/bootstrap/kickstart behavior.
  - file: [src/cli/update-cli.test.ts:1432](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/cli/update-cli.test.ts#L1432)
