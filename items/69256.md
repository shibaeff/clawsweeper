## Summary

Keep open. Current `main` still deletes isolated direct-delivery cron sessions unconditionally, so the underlying bug remains plausible, but this PR's proposed early-return guard is not sufficient because `cleanupDirectCronSessionIfNeeded()` is only reached during delivery-time paths and has no later retry once descendants finish.

## Evidence

- **Current main still deletes direct-delivery sessions without descendant check:** `src/cron/isolated-agent/delivery-dispatch.ts:465` defines `cleanupDirectCronSessionIfNeeded()` with only `deleteAfterRun` / `directCronSessionDeleted` guards, then immediately calls `sessions.delete`; there is no `countActiveDescendantRuns()` check in current `main`. The direct-delivery wrapper at `src/cron/isolated-agent/delivery-dispatch.ts:667` always runs that cleanup in `finally`.
