## Summary

Keep open. Current `main` still uses the pre-PR version-refresh logic for skills snapshots, so the stale-snapshot gap this PR targets is not clearly eliminated on `main`. Managed skills are already part of snapshot construction, which points to cache invalidation/refresh behavior as the real issue rather than source loading.

## Evidence

- **Version-0 snapshots still do not self-refresh:** `shouldRefreshSnapshotForVersion` returns `false` when both cached and current snapshot versions are `0`, so an existing version-0 snapshot is treated as fresh unless something else bumps the version.

There is no current-main special case for 'first turn with version 0'.
