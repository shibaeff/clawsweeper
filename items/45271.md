## Summary

Keep open. Current `main` has a phase-aware suppression path for commentary text, but it is scoped to messages that carry explicit assistant phase metadata. The reported repro uses Ollama, and the Ollama transport still builds assistant messages with both `text` and `toolCall` blocks plus `stopReason: "toolUse"`, without adding phase metadata. I did not find a generic `stripTextBeforeToolCalls`-style delivery guard for phaseless providers, so the specific Ollama regression is not strongly shown as fixed on `main`.
