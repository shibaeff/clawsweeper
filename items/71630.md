## Summary

Keep open. Current main does not implement the requested composer-adjacent runtime status chip. The Web UI toolbar in `ui/src/ui/views/chat.ts:1134` renders attach/voice/talk/token-count controls on the left and send/export/new-session controls on the right, but no session run-state indicator. The only existing chat status UI is the transient compaction/fallback toast pair from `ui/src/ui/chat/status-indicators.ts`, rendered above the input in `ui/src/ui/views/chat.ts:1078`; those are not the proposed `In progress` / `Interrupted` / `Done` session activity states. Reconnect handling in `ui/src/ui/app-gateway.ts:322` also clears orphaned run state after disconnect without surfacing a composer status chip. The report is new, concrete, and actionable, so there is no conservative basis to close it.
