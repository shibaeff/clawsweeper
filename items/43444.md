## Summary

Current main already provides a supported extension seam for arbitrary bootstrap/context-file injection through `agent:bootstrap` hooks, and project direction is to keep optional capability in plugins/hook packs rather than add new core config for it.

- Discoverability is the main downside: users looking only at agent-default config may miss that hooks/plugins already provide this extension path.

## Close Comment

Closing as `clawhub` after Codex review. Current `main` already exposes a supported extension seam for this exact class of customization: `agent:bootstrap` hooks can mutate `context.bootstrapFiles`, and current tests explicitly cover adding an arbitrary `EXTRA.md` file (`src/agents/bootstrap-files.test.ts:21`, `src/agents/bootstrap-hooks.test.ts:25`). Hook packs are installable as plugins (`docs/automation/hooks.md:123`, `docs/automation/hooks.md:142`), and the project vision is to keep optional capability in plugins rather than expand core config surface (`VISION.md:52`). The bundled `bootstrap-extra-files` hook is intentionally limited to canonical bootstrap basenames (`src/hooks/bundled/bootstrap-extra-files/handler.ts:42`, `docs/automation/hooks.md:192`), so a configurable arbitrary-file injector fits better as a ClawHub/OpenClaw plugin or hook pack than a new built-in `agents.defaults.*` setting.
