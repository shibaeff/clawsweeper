## Summary

Current `main` already handles live-session model switches during fallback: the runner persists fallback-selected session overrides before retrying, and the fallback loop now treats `LiveSessionModelSwitchError` as a failover signal instead of aborting the chain. The regression is covered by focused tests, but it does not appear to be in the latest tagged release yet.

## Evidence

- **Fallback-selected model is persisted before retry:** `persistFallbackCandidateSelection(...)` writes the fallback provider/model/auth override into the active session entry before the retry starts, which closes the stale-session race described in the issue.
