## Summary

Keep open. Current `main` at `41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00` still scopes Control UI session tokens and persisted settings by the exact parsed gateway host, so `ws://127.0.0.1:18789`, `ws://localhost:18789`, and `ws://[::1]:18789` remain distinct scopes. The PR’s behavior is not already implemented on `main`, and the reported footgun is still plausible from the current code.
