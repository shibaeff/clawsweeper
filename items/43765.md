## Summary

Keep open. Current `main` does not already implement the PR well enough: Feishu still does not publish websocket health into channel status, and exec/process sessions still treat ordinary non-zero exits as completed in key paths. Some heartbeat-event consumption logic exists on `main`, but the PR's core runtime-recovery changes are not fully present.
