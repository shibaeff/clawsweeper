## Summary

Keep open. Current `main` still uses the misleading `no bootstrap files` / `bootstrap file(s) present` wording on the regular `openclaw status --deep` report path, even though repo docs and workspace tests treat `BOOTSTRAP.md` absence as the normal completed state. A separate `status --all` formatter was updated to `N bootstrapping`, but that does not cover the reported command path.

## Evidence

- **Regular status report still renders the misleading wording:** `buildStatusAgentsValue()` formats `bootstrapPendingCount` as `"no bootstrap files"` when zero and `"N bootstrap file(s) present"` when nonzero. `statusCommand()` builds the normal text report through `buildStatusCommandReportData()` -> `buildStatusCommandOverviewRows()` -> `buildStatusAgentsValue()`, which is the `openclaw status --deep` path described in the issue.
