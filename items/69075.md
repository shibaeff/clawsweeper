## Summary

Keep open. Current `main` still rejects extra fields on `models.providers.*.models[]`, so the requested provider-config surface is not implemented yet. There is a documented per-model workaround through `agents.defaults.models["provider/model"].params.extra_body`, and OpenRouter already preserves an explicit `reasoning.max_tokens` payload, but that is a different config path than the issue requests. The issue also has an active cross-referenced PR for the exact feature.
