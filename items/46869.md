## Summary

Keep open. Current `main` still performs the scheduled-task restart inline in `restartScheduledTask`, so this PR's Windows handoff change is not already implemented and there is not enough evidence to conservatively close it for any other reason.

## Evidence

- **Current main still ends the task before inline restart:** `restartScheduledTask()` calls `execSchtasks(["/End", "/TN", taskName])`, then cleanup, then `runScheduledTaskOrThrow(...)`, which itself runs `schtasks /Run` inline. That matches the PR's reported code path rather than the proposed detached handoff.
