## Summary

Keep open. Current `main` does not show the PR's live-tail fix as implemented: the TUI still only listens for `chat`/`chat.side_result`/`agent`, `ChatEvent` still excludes a `user-message` state, and `chat.send` on `main` emits eager transcript updates rather than a new chat event that the TUI would render immediately.
