## Summary

The issue asks for a broad core "agent orchestration + integrations hub". Current main's documented scope pushes optional integrations, skills, and marketplace-style discovery to plugins/ClawHub, and VISION.md explicitly rejects heavy orchestration layers and first-class core MCP/runtime expansion. Main already covers the core-side pieces that do belong in OpenClaw: multi-agent routing, ClawHub/plugin install surfaces, and UI/CLI model-provider switching.

## Close Comment

Thanks for the idea. Closing this as a Codex maintainer review because the remaining request fits ClawHub/plugin scope rather than OpenClaw core.

Current main already has the core pieces that do belong here: multi-agent routing (`docs/concepts/multi-agent.md:8`), plugin marketplace/install flows with ClawHub-first discovery (`docs/cli/plugins.md:38`, `docs/plugins/community.md:9`), skills install/manage from ClawHub (`docs/cli/skills.md:12`), and model/provider switching surfaces in CLI/UI (`docs/cli/models.md:21`, `docs/web/control-ui.md:116`, `ui/src/ui/chat-model-ref.test.ts:25`).

For the rest, `VISION.md` is the scope anchor on current main: optional capability should usually ship as plugins (`VISION.md:54`), new skills should go to ClawHub first (`VISION.md:68`), and OpenClaw is explicitly not taking heavy orchestration layers or first-class core MCP/runtime expansion right now (`VISION.md:99`). If someone wants to build an integrations hub or orchestration framework, the project direction is to do that as a plugin/skill/marketplace contribution rather than expand core.
