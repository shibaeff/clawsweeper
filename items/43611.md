## Summary

Keep open. Current `main` only handles Windows code-page decoding in `src/node-host/invoke.ts`; the `exec` and supervisor child-process paths still decode stdout/stderr with plain `Buffer.toString()`, `scripts/ui.js` still uses `shell: true` for Windows shim commands, and the docs still describe garbled Chinese output on native Windows as an active problem. The PR also has unresolved review concerns about Windows shell argument safety and decoder flushing, so there is not strong evidence to close it conservatively.
