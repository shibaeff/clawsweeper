## Summary

Keep open. Current `main` still encodes the reported behavior: when elevated access is allowed and no per-call `elevated` flag is provided, the session default falls back to `"on"`, which makes plain `exec(...)` calls count as elevated and route off-sandbox to `gateway` unless the selected target is `node`. Docs and tests on `main` still describe and assert that behavior, so there is no strong basis to close as fixed or obsolete.
