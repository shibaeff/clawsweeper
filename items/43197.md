## Summary

The request is coherent, but current OpenClaw scope guidance says optional channel integrations like this should be shipped and maintained as third-party plugins, not added to the core/bundled repo.

## Close Comment

Closing this as a ClawHub/plugin-scope request after a Codex review. The current project direction is to keep optional capability in plugins, with authors hosting and maintaining them in their own repositories rather than adding them to OpenClaw core (`VISION.md:54`). The plugin docs say new channel plugins do not need to be added to this repo and should be published to ClawHub or npm (`docs/plugins/building-plugins.md:16`), and the community plugins page already treats third-party channel integrations as the canonical path (`docs/plugins/community.md:9`). Pilot Protocol looks like a specialized external channel integration around its own daemon/CLI bridge, so it fits that third-party plugin path rather than a bundled/core OpenClaw feature. If you publish it externally, a follow-up PR adding it to the community plugins page would fit the documented workflow.
