## Summary

Current `main` already provides the liveness/progress feedback this issue asks for on both channels: Telegram has live preview edits, typing, optional tool-progress preview, and lifecycle status reactions; Feishu has real-time streaming cards plus typing reactions. Raw tool stdout is still not forwarded verbatim, but the renamed issue's broader 'real-time progress feedback' request is already implemented well enough to close.

## Risks / Open Questions

- Telegram's verbose tool-progress preview is opt-in (`streaming.preview.toolProgress`); the default liveness path is preview edits/typing/status reactions rather than chatty per-tool echo.
- OpenClaw still does not forward raw tool stdout or true token-delta streams directly into channel messages; this close depends on the issue's renamed broader goal of visible progress/liveness feedback.
