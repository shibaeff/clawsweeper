## Summary

Current main no longer treats the standard `amazon-bedrock` provider as an Anthropic Messages stream path. It now documents and auto-discovers `amazon-bedrock` as native `bedrock-converse-stream` with AWS SDK auth, while Anthropic Messages + Claude Opus 4.7 + bearer-auth streaming is handled by the separate bundled `amazon-bedrock-mantle` provider shipped in `v2026.4.23`. Main also clears stale fallback-pinned session model state on `/reset` and `/new`.

## Evidence

- **Standard Bedrock provider is native Converse on main:** The public Bedrock docs describe `amazon-bedrock` as the native Bedrock Converse provider, with API `bedrock-converse-stream`, and show explicit config using `auth: "aws-sdk"`. The same docs also state that `AWS_BEARER_TOKEN_BEDROCK` is only an env marker and that runtime model calls still use the AWS SDK default credential chain.
