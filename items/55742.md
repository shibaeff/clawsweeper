## Summary

Current `main` does not implement the execute-context injection described by PR #55742. Plugin tool factories still receive `OpenClawPluginToolContext`, but resolved tools are returned directly without any wrapper that forwards agent/session identity into `tool.execute(...)`, and the public docs still describe plugin tools with the existing two-argument execute shape. The unrelated `getApiKeyAndHeaders` compaction work appears to have landed elsewhere on `main`, but the requested plugin execute-context seam itself has not, so there is not strong evidence to close this PR.
