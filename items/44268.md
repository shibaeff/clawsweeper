## Summary

Current `main` already supports Telegram forum-topic routing for outbound sends. The CLI exposes `--thread-id`, Telegram targets parse `:topic:<id>` / numeric topic suffixes, outbound Telegram sends pass `message_thread_id`, and the Telegram threading adapter auto-preserves the current topic for same-chat tool sends. The docs and tests on `main` match that behavior, so the workaround issue is obsolete.
