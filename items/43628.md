## Summary

Current `main` now treats launchd-managed gateway restarts as supervisor-owned restarts instead of detached respawns, which addresses the duplicate-process path described in #43628.

## Evidence

- **Launchd-managed restarts no longer detached-respawn a second process:** `restartGatewayProcessWithFreshPid()` detects launchd markers and returns `mode: "supervised"`; the launchd branch explicitly avoids the detached handoff and lets the supervisor relaunch the process.
  - file: [src/infra/process-respawn.ts:30](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/infra/process-respawn.ts#L30)
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Gateway run loop exits cleanly for supervised restarts:** After server close, the run loop releases the gateway lock, logs a full-process restart, and exits with code 0 for `spawned` or `supervised` restart modes; for launchd it adds a brief exit delay and then exits instead of doing an in-process restart.
  - file: [src/cli/gateway-cli/run-loop.ts:82](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/cli/gateway-cli/run-loop.ts#L82)
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Regression coverage for launchd marker path:** Tests assert that launchd markers on macOS produce `mode: "supervised"` and do not call the old restart helper/kickstart path.
  - file: [src/infra/process-respawn.test.ts:88](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/infra/process-respawn.test.ts#L88)
