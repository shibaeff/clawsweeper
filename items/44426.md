## Summary

Keep open. Current `main` still turns a timed-out `doctor.memory.status` gateway call into a failed memory probe, and the doctor memory-search check still emits a warning for an explicit local provider whenever that probe is `checked` but not `ready`. `openclaw memory status --deep` remains a separate direct probe path, so the reported false-warning discrepancy is still plausible. The timeout default is now 8000ms instead of the reported 3000ms, which may reduce frequency, but I did not find evidence on `main` that the warning is retried, downgraded, or suppressed when runtime memory status is otherwise healthy.
