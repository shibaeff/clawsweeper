## Summary

Keep open. Current `main` does not implement this fix, and the PR still targets a real ACP identity-reconcile behavior. At the same time, the existing review comments raise a plausible regression: marking ensure-derived identities as `resolved` would change `mergeSessionIdentity()` precedence and can let stale `ensureSession()` IDs overwrite fresher persisted IDs when status refresh is skipped or fails. That needs maintainer judgment rather than conservative auto-close.
