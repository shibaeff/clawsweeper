## Summary

Keep open. Current `main` still implements remote MCP auth as static HTTP headers only; there is no `auth` config field, no transport `authProvider` wiring, and no OAuth callback/refresh flow in the MCP client path. The issue is recent and its timeline already links an open follow-up PR (#62134), so conservative cleanup should not close it.

## Evidence
