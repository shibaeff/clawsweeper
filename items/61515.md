## Summary

Keep open. Current `main` does not contain the PR's `dist-runtime` skill-path remap or the separate watch-path handling, and the bundled-plugin runtime still prefers `dist-runtime/extensions` when a paired `dist/extensions` tree exists. That means the cleanup proposed here is not implemented on `main`, and I do not have strong evidence that the report is obsolete.

## Evidence

- **Skill loading still validates only the raw resolved candidate under `record.rootDir`:** `resolvePluginSkillDirs()` on current `main` still does `path.resolve(record.rootDir, trimmed)` and immediately applies `isPathInsideWithRealpath(record.rootDir, candidate, { requireRealpath: true })`. There is no `dist-runtime` -> `dist` remap helper and no alternate load/watch path split in this file.
  - file: [src/agents/skills/plugin-skills.ts:105](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/skills/plugin-skills.ts#L105)

- **Watcher still uses load dirs and still ignores `/dist/`:** `refresh.ts` still imports `resolvePluginSkillDirs()` directly, pushes those paths into the watch set, and keeps `/dist/` in `DEFAULT_SKILLS_WATCH_IGNORED`. The PR's `resolvePluginSkillWatchDirs()` follow-up is not present on `main`.
  - file: [src/agents/skills/refresh.ts:41](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/agents/skills/refresh.ts#L41)
