## Summary

Keep open. Current `main` has partial TUI theme support, but it does not implement this PR’s requested 5 built-in themes, `--theme` CLI flag, or runtime `/theme` switching, so there is no strong basis to close it as already implemented or obsolete.

## Evidence

- **Current theme system is only light/dark:** `src/tui/theme/theme.ts` exports exactly two palettes (`darkPalette`, `lightPalette`) and computes a single `lightMode` boolean from `OPENCLAW_THEME`/`COLORFGBG`; there is no `setTheme`, theme registry, or multi-theme runtime switching API on current `main`.
  - file: [src/tui/theme/theme.ts:49](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/tui/theme/theme.ts#L49)
  - command: `nl -ba src/tui/theme/theme.ts | sed -n '49,232p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **TUI CLI has no theme flag:** `registerTuiCli` defines options like `--local`, `--url`, `--deliver`, `--message`, `--timeout-ms`, and `--history-limit`, then passes them to `runTui`. There is no `--theme <name>` option on current `main`.
  - file: [src/cli/tui-cli.ts:10](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/cli/tui-cli.ts#L10)

  - file: [src/tui/commands.ts:57](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/tui/commands.ts#L57)
