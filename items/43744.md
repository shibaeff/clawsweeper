## Summary

Current main makes ClawHub the canonical discovery surface for community plugins and explicitly rejects docs-only discoverability PRs. This PR only adds a community-plugin listing for OpenViking, and its proposed install flow does not match the required `openclaw plugins install <package-name>` path.

## Evidence

- **Community plugin docs now reject docs-only discoverability PRs:** `docs/plugins/community.md` says community plugins are published on ClawHub or npm, calls ClawHub the canonical discovery surface, and explicitly says not to open docs-only PRs just to add a plugin for discoverability.

  - file: [docs/plugins/community.md:18](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/docs/plugins/community.md#L18)
