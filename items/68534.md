## Summary

Keep open. Current `main` still lacks the cooldown guard this PR is trying to add, so it is not `implemented_on_main` or obsolete, and the PR still appears materially useful. At the same time, the latest review thread leaves a real contract question unresolved: the revised PR switches to fixed 6h/7d cooldowns, while current code/docs/tests say the unified `dreaming.frequency` cadence applies across phases. That is not strong enough evidence to auto-close a non-maintainer PR.
