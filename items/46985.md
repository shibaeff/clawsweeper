## Summary

Keep open. The PR’s core Web UI stream-deduping fix is not on current `main`: current code still stores the full cumulative assistant stream when a tool starts and later renders the full live stream again, so the duplicated-prefix behavior around tool interruptions remains plausible. The PR discussion shows the author addressed the review finding inside the PR branch, but that implementation has not landed on `main` and the branch also mixes in unrelated audit/inbound-metadata/plugin-registry changes that would need maintainer judgment.
