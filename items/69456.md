## Summary

Keep open. Current `main` still leaves the Windows CI lane intentionally narrow and does not cover the Lobster regression path described in the issue. `package.json` still excludes `extensions/lobster/src/lobster-runner.test.ts` from `test:windows:ci`, the workflow comment says the Windows lane is limited to process/path wrappers, the Lobster runner still imports `@clawdbot/lobster/core` directly with no visible packaged-install fallback in this file, and the pnpm runner tests still only cover the Windows `pnpm.cmd` fallback when `npm_execpath` is unavailable rather than a direct `pnpm.exe` case.
