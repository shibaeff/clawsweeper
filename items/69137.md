## Summary

Current `main` still rejects a root-level `paperclip` field on Gateway `agent` requests, and the latest release `v2026.4.23` still carries the same strict schema. The reporter's comment shows a viable local workaround, but there is no evidence in OpenClaw `main` that the compatibility gap has been fixed yet.

## Evidence

- **Current main still forbids extra root fields on `agent` requests:** `AgentParamsSchema` enumerates allowed keys, does not include `paperclip`, and ends with `{ additionalProperties: false }`, so a root-level `paperclip` object is still invalid on current `main`.
