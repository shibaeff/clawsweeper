## Summary

Current `main` still matches the report. `agent.wait` returns `status: "timeout"` when no terminal snapshot arrives before the deadline, and `sessions_send` forwards that as a bare timeout result without any delivery-state field or accepted/pending distinction. I found no evidence on `main` that this issue has been implemented or made obsolete.

## Evidence

- **Gateway timeout means wait deadline expired, not confirmed delivery failure:** `agent.wait` responds with `{ runId, status: "timeout" }` when no snapshot is available by the deadline.
  - file: [src/gateway/server-methods/agent.ts:1097](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/gateway/server-methods/agent.ts#L1097)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '1060,1125p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **sessions_send still collapses that state to bare timeout:** After `waitForAgentRunAndReadUpdatedAssistantReply`, the tool returns only `status: "timeout"`, `error`, and `sessionKey`; it does not include delivery metadata or a distinct delivered-but-wait-timed-out status.
  - file: [src/agents/tools/sessions-send-tool.ts:377](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/tools/sessions-send-tool.ts#L377)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '375,420p'`

- **Run-wait helper preserves timeout/pending from agent.wait:** `waitForAgentRun` maps gateway `status: "timeout"` directly, and `waitForAgentRunAndReadUpdatedAssistantReply` returns any non-`ok` wait result unchanged. `sessions_send` only special-cases `timeout` and `error`, so there is still no richer reply-timeout state.
  - file: [src/agents/run-wait.ts:118](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/agents/run-wait.ts#L118)
