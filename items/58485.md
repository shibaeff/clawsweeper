## Summary

Current `main` already honors isolated cron `payload.model` overrides. The isolated cron model-selection path resolves `payload.model` before stored session overrides and agent defaults, the runner consumes that resolved model, regression tests cover the exact precedence, and the latest release tag `v2026.4.23` contains the same implementation.

## Evidence

- **Isolated cron model selection prefers payload.model:** `resolveCronModelSelection` reads `params.payload.model`, validates/resolves it, and only falls back to stored session overrides when no per-job model override is present.
