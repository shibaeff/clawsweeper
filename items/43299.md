## Summary

Keep open. Current `main` has lower-level per-run provider/model override plumbing, but the `openclaw agent` CLI still does not expose a `--model` flag or pass a model override through either the gateway or embedded path, so the PR's requested user-facing behavior is not implemented on `main` yet.

## Evidence

- **CLI flag missing on current main:** `openclaw agent` registers `--thinking`, `--verbose`, `--channel`, `--reply-*`, `--local`, `--deliver`, `--json`, and `--timeout`, but no `--model` option.
  - file: [src/cli/program/register.agent.ts:27](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/cli/program/register.agent.ts#L27)
  - command: `nl -ba src/cli/program/register.agent.ts | sed -n '18,55p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **CLI adapter does not carry model overrides:** `AgentCliOpts` has no `model` field, the gateway request params omit `model`, and the embedded fallback only forwards `agentId`, `replyAccountId`, and cleanup state. Current CLI code cannot send the requested per-turn override.
  - file: [src/commands/agent-via-gateway.ts:36](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/commands/agent-via-gateway.ts#L36)
