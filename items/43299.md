## Summary

Keep open. Current `main` has internal per-run provider/model override plumbing, but the user-facing `openclaw agent --model <id>` CLI surface from PR #43299 is still not implemented on `main`.

## Evidence

- **`openclaw agent` CLI still has no `--model` flag:** `registerAgentCommands()` defines `openclaw agent` options, and the command includes `--message`, `--to`, `--session-id`, `--agent`, `--thinking`, `--verbose`, `--channel`, `--reply-*`, `--local`, `--deliver`, `--json`, and `--timeout`, but no `--model` option.
  - file: [src/cli/program/register.agent.ts:25](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/cli/program/register.agent.ts#L25)
  - command: `nl -ba src/cli/program/register.agent.ts | sed -n '1,90p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **CLI transport layer does not accept or forward a model override:** `AgentCliOpts` in the gateway-facing CLI wrapper has no `model` field, the gateway RPC params omit `model`, and the local fallback path also does not map any CLI model option into the agent command.
  - file: [src/commands/agent-via-gateway.ts:35](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/commands/agent-via-gateway.ts#L35)
  - command: `nl -ba src/commands/agent-via-gateway.ts | sed -n '1,240p'`
  - sha: [41c5ffc5d5f4](https://github.com/openclaw/openclaw/commit/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00)
- **User docs for `openclaw agent` still omit `--model`:** The `docs/cli/agent.md` options and examples list `--thinking`, `--verbose`, `--channel`, `--reply-*`, `--local`, `--deliver`, `--timeout`, and `--json`, but no `--model` flag or per-turn model override example.
  - file: [docs/cli/agent.md:23](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/docs/cli/agent.md#L23)
  - command: `nl -ba docs/cli/agent.md | sed -n '1,90p'`

- **Main already has lower-level model-override support, so the PR is only partially obsolete:** The core agent path already supports per-run `provider`/`model` overrides (`opts.provider` / `opts.model`) with allowlist validation and non-persistence semantics, and the gateway protocol already exposes `provider` and `model` fields. That means some underlying plumbing landed on `main`, but not the specific CLI flag requested by this PR.
