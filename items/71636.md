## Summary

Keep open. The PR targets a real gap for Ollama autodiscovery, but the proposed one-line Zod change is incomplete: current main still requires `models` in the generated base config schema and in the exported `ModelProviderConfig` type, so merging this as-is would leave schema/tooling drift and a public type mismatch.

## Evidence

- **Current Zod schema still requires provider models:** `ModelProviderSchema` on current `main` declares `models: z.array(ModelDefinitionSchema)` with no `.optional()`. The underlying bug is still present on `main`, so this is not already implemented.
