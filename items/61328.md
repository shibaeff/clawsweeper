## Summary

Current main already has the failure-class-aware fallback foundation this issue asks for: explicit failover reason classes, reason-specific fallback policy, configured model-selection/fallback routing, structured fallback telemetry, and regression coverage. The same behavior is present in the shipped v2026.4.23 tree, so this is not just an unreleased main-only implementation.
