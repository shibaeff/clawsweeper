## Summary

Current main already prevents the reported duplicate root-memory behavior in both bootstrap prompt loading and memory indexing/search, and the changelog says this shipped in v2026.3.12.

## Evidence

- **Bootstrap only accepts canonical root memory:** `loadWorkspaceBootstrapFiles` skips the memory bootstrap entry unless an exact `MEMORY.md` directory entry exists, using `exactWorkspaceEntryExists` rather than probing both casings.
  - file: [src/agents/workspace.ts:520](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/workspace.ts#L520)
  - command: `nl -ba src/agents/workspace.ts | sed -n '500,540p'`
- **Exact-case workspace entry check:** `exactWorkspaceEntryExists` reads the directory entries and requires an exact filename match, so a case-insensitive mount alias does not count as a second logical root-memory file.
  - file: [src/memory/root-memory-files.ts:24](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/memory/root-memory-files.ts#L24)
  - command: `nl -ba src/memory/root-memory-files.ts | sed -n '1,80p'`
- **Memory indexing/search prefers canonical root memory:** `listMemoryFiles` resolves only the canonical root file via `resolveCanonicalRootMemoryFile` and skips legacy/auxiliary root-memory paths under the workspace, preventing duplicate root-memory enumeration.
  - file: [src/memory-host-sdk/host/internal.ts:132](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/memory-host-sdk/host/internal.ts#L132)
  - command: `nl -ba src/memory-host-sdk/host/internal.ts | sed -n '115,185p'`
- **Regression coverage and shipped release note:** Tests cover ignoring lowercase-only root `memory.md` during bootstrap and memory indexing, and the changelog for 2026.3.12 explicitly says OpenClaw now loads at most one root memory bootstrap file and fixes duplicate memory injection on case-insensitive Docker mounts.
  - file: [CHANGELOG.md:2584](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/CHANGELOG.md#L2584)
  - command: `nl -ba CHANGELOG.md | sed -n '2560,2605p'`
