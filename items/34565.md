## Summary

Keep open. Current `main` still manufactures synthetic Feishu reaction message IDs with a `:reaction:<emoji>:<uuid>` suffix and still forwards `ctx.messageId` directly into API-facing reply/typing paths without stripping that suffix. The PR is also stale against current `main`: it targets the main dispatcher path, but current `extensions/feishu/src/bot.ts` now has an additional ACP-init error reply path that also uses the raw message ID.
