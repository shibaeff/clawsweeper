## Summary

Keep open. Current `main` and the latest release still synthesize Feishu reaction message IDs with a `:reaction:<emoji>:<uuid>` suffix and route those IDs unchanged into reply/typing API calls. That means the underlying bug described in #34528 is still present, so this PR is not stale cleanup material. The proposed change looks directionally correct, but it likely still needs maintainer review because there is another Feishu reply path in `bot.ts` that also builds `replyTargetMessageId` without stripping the suffix.
