## Summary

Keep open. Current `main` still hardcodes bootstrap file ordering and only treats `HEARTBEAT.md` as dynamic, and there is no config surface for `bootstrapDynamicFiles` or `bootstrapFileOrder`. The existing hook/docs surfaces are partial workarounds for adding or mutating bootstrap files, but they do not implement the requested cache-boundary classification control.
