## Summary

Keep open. Current `main` has some Feishu conversation-binding plumbing, but it does not fully implement this PR’s user-facing scope: the docs still describe `/focus`/`/unfocus` thread-binding as Discord-only, Feishu docs only document ACP `/acp spawn ... --thread here`, and the Feishu reply dispatcher still intentionally drops block payloads instead of delivering them when streaming cards are inactive.
