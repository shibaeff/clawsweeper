## Summary

Keep open. Current `main` still uses only Ollama's experimental web-search endpoint, so the compatibility bug is not implemented on `main`. This PR is also not safe to auto-land/auto-close: its cloud fallback reuses the same bearer header for local and `ollama.com` requests, and related open PR #69222 appears to supersede it with safer auth scoping plus broader tests/docs.

## Evidence

- **Current main is still experimental-only:** `extensions/ollama/src/web-search-provider.ts` on `main` defines only `OLLAMA_WEB_SEARCH_PATH = "/api/experimental/web_search"` and `runOllamaWebSearch()` posts directly to that single path. There is no local `/api/web_search` fallback and no cloud retry on `main`.
