## Summary

Keep open. Current `main` does not contain the PR's bundled-layout fallback change: `src/agents/workspace-templates.ts` still has a single `../../docs/reference/templates` module-relative fallback and still checks `cwd/docs/...` before that fallback. The new `resolveOpenClawPackageRoot` logic and current tests improve the normal dist-rooted package-root case, but they do not prove the reported `packageRoot === null` bundled-layout failure is fixed.

## Evidence

- **Single fallback still on main:** `resolveWorkspaceTemplateDir` still defines one module-relative fallback with `../../docs/reference/templates` and builds candidates as `[packageRoot/docs, cwd/docs, FALLBACK_TEMPLATE_DIR]`. The PR's proposed extra bundled `../docs` fallback and reordered candidate list are not present on current `main`.
