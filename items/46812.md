## Summary

Current `main` has partial overlap: the Control UI can already list and edit allowlisted workspace bootstrap files, including `USER.md` and `MEMORY.md`, and `USER.md` content is injected every session. But there is no dedicated pinned-facts panel/sidebar, no structured key-value editing UX, and no evidence of a persistent Control UI widget specifically for user-defined facts. That is not strong enough to close as already implemented.

## Evidence

- **Control UI has a generic Files editor, not a pinned-facts panel:** The Agents UI renders a `Core Files` editor with tabs, a textarea, and save/reset actions for editing agent workspace files.
  - file: [ui/src/ui/views/agents-panels-status-files.ts:373](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/ui/src/ui/views/agents-panels-status-files.ts#L373)
- **Gateway exposes editable workspace files including USER.md and MEMORY.md:** The allowlist includes bootstrap files plus `MEMORY.md`, and the file listing appends `DEFAULT_MEMORY_FILENAME` after the bootstrap set that already contains `DEFAULT_USER_FILENAME`.
  - file: [src/gateway/server-methods/agents.ts:58](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/gateway/server-methods/agents.ts#L58)
- **Control UI/Gateway contract already includes agent file management:** The protocol docs say `agents.files.list`, `agents.files.get`, and `agents.files.set` manage the bootstrap workspace files exposed for an agent.
  - file: [docs/gateway/protocol.md:333](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/gateway/protocol.md#L333)
- **Persistent context already comes from workspace files:** OpenClaw creates `USER.md` in the workspace by default, and `MEMORY.md` is loaded for normal sessions when present.
  - file: [docs/start/openclaw.md:75](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/start/openclaw.md#L75)
- **Docs distinguish always-injected files from optional memory behavior:** The standing-orders docs say the workspace bootstrap injects `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and `MEMORY.md`; Discord docs further clarify that if context must be present every session/channel, `AGENTS.md` or `USER.md` is the stable place for it.
  - file: [docs/automation/standing-orders.md:43](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/docs/automation/standing-orders.md#L43)

- This review is based on static inspection only; I did not run the Control UI interactively.
