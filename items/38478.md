## Summary

Keep open. Current `main` partially addresses the request by preloading recent `memory/*.md` files on bare `/new` and `/reset`, and by running an automatic pre-compaction memory flush, but it does not implement automatic persistence on arbitrary session end. The bundled `session-memory` hook still only fires for `/new` and `/reset`, so the core gap described in the issue remains real enough that closing would be premature.
