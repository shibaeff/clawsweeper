## Summary

Current `main` does not show a remaining code path that writes heartbeat settings under `gateway`. Heartbeat is defined under `agents.defaults.heartbeat`, `gateway` is a strict schema without a `heartbeat` field, and the config editor/`config.patch` pipeline mutates explicit paths then re-validates before persisting.

## Evidence

- **Heartbeat schema lives under agents defaults:** `heartbeat` is defined on agent defaults, not on `gateway`.
  - file: [src/config/zod-schema.agent-defaults.ts:251](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/zod-schema.agent-defaults.ts#L251)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Gateway schema has no heartbeat property:** The `gateway` object is strict and lists its allowed fields; there is no `heartbeat` key in this schema block.
  - file: [src/config/zod-schema.ts:668](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/config/zod-schema.ts#L668)
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)

  - file: [ui/src/ui/controllers/config.ts:203](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/ui/src/ui/controllers/config.ts#L203)

## Close Comment

Closing after a Codex review of current `main`. I could not find a remaining OpenClaw code path that writes heartbeat settings under `gateway`. On current `main`, heartbeat is defined under `agents.defaults.heartbeat` (`src/config/zod-schema.agent-defaults.ts:251`), while `gateway` is a strict schema without any `heartbeat` field (`src/config/zod-schema.ts:668`). The Control UI/config pipeline updates explicit config paths and then re-validates before persisting (`ui/src/ui/controllers/config.ts:203`, `ui/src/ui/controllers/config/form-utils.ts:20`, `src/gateway/server-methods/config.ts:471`), so a generated `gateway.heartbeat` write does not match current behavior. I also checked current docs/tests, which consistently point to `agents.defaults.heartbeat` (`docs/cli/config.md:39`, `docs/gateway/heartbeat.md:118`, `src/config/config-misc.test.ts:648`). If this still reproduces on a newer build, please open a fresh issue with the exact surface that wrote the config and the before/after JSON so maintainers can pin the remaining writer.
