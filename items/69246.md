## Summary

Current `main` already wires chat-send lifecycle failures through to Control UI terminal error handling, which matches the reported infinite-loading symptom. The issue was filed against `2026.4.15`; the relevant fixes are present on current `main` and documented in the unreleased changelog, but they do not appear to have shipped in `v2026.4.23` yet.

## Evidence

- **Chat runs are now registered for lifecycle error delivery:** `chat.send` adds the run to the chat-run registry before execution, so later lifecycle failures have a client-visible chat run link instead of being dropped.
  - file: [src/gateway/server-methods/chat.ts:2252](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/gateway/server-methods/chat.ts#L2252)
