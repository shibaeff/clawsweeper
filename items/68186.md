## Summary

Keep open. Current `main` does not include this hardening: malformed `screen.snapshot` params still fall back to defaults, snapshot responses are still emitted as inline base64 without a payload bound, validation/capture errors are still collapsed to generic `.unavailable`, and current tests only cover the happy path.

## Evidence

- **Malformed params still fail open on main:** `handleScreenSnapshotInvoke` still decodes with `try? ... ?? MacNodeScreenSnapshotParams()` and then immediately invokes `snapshotScreen`, so malformed `paramsJSON` still falls back to default params instead of returning `INVALID_REQUEST`. The same method base64-encodes the snapshot payload with no size guard before returning it.
