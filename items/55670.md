## Summary

Keep open. Current `main` already covers the Discord stale-socket/OOM path, the subagent fallback prompt loss, and the auth runtime-snapshot refresh, but it does not include the PR's string-safe `validateAnthropicTurns` follow-up. `openai-completions` replay still enables `validateAnthropicTurns` while persisted user turns are stored as plain strings and `mergeConsecutiveUserTurns` still concatenates only array-form content, so this PR still contains non-duplicated fix value.
