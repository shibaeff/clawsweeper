## Summary

Current `main` does not implement the requested `file_shared` forwarding path. The Slack monitor still only registers `message` and `app_mention` for inbound message flow, there is no `file_shared` listener or generic Slack-event plugin hook, and the existing `file_share` handling only applies when Slack already delivers the upload as a `message` subtype.

## Evidence

- **Slack monitor wiring has no file_shared registration:** The Slack monitor registers message, reaction, member, channel, pin, and interaction handlers only. There is no separate `file_shared` event registration path in the monitor bootstrap.
  - file: [extensions/slack/src/monitor/events.ts:18](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/slack/src/monitor/events.ts#L18)

- **Inbound message events only subscribe to message and app_mention:** `registerSlackMessageEvents` calls `ctx.app.event("message", ...)` and `ctx.app.event("app_mention", ...)` only. There is no `ctx.app.event("file_shared", ...)` or generic passthrough listener.
  - file: [extensions/slack/src/monitor/events/messages.ts:215](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/slack/src/monitor/events/messages.ts#L215)
