## Summary

Keep open. Current `main` still distinguishes only core tool IDs, not core tool-group IDs, when composing unknown-allowlist warnings, so the reported `group:memory` false warning remains plausible on `main`. The PR also has unresolved review evidence of a test regression in the dedupe-cache test, so this is not a strong close candidate.

## Evidence

- **`group:memory` is a shipped core group:** Core tool groups are generated as `group:${tool.sectionId}` for every core tool section, and the memory section plus `memory_search`/`memory_get` core tools are present, so `group:memory` is a valid built-in group.
  - file: [src/agents/tool-catalog.ts:39](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/src/agents/tool-catalog.ts#L39)

- **Allowlist analysis recognizes tool groups as core entries:** `analyzeAllowlistByToolType` expands allowlist entries through `expandToolGroups([entry])` and treats the entry as core when any expanded tool is in the core tool set.
