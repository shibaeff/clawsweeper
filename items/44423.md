## Summary

Keep open. Current `main` has related Telegram troubleshooting and polling-stall guidance, but it does not contain this PR's added host-level recovery watchdog guidance or the systemd binary-path note, so the PR is not implemented on `main`. The proposed `openclaw health --json` watchdog advice also still needs maintainer review because current raw health snapshots include session-path/recent-session data and per-channel probe data that the repo treats as needing sanitization before sharing.
