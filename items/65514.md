## Summary

Close as cannot_reproduce. Current main still forwards the MiniMax context budget into compaction, has a defensive fallback when that budget is missing, and the issue discussion itself shifted the likely root cause to CJK/tokenizer accounting rather than 'undefined tokenBudget' or 'silent compaction skip.'

## Evidence

- **Budget forwarded into embedded run:** Current main passes `contextTokenBudget: ctxInfo.tokens` into the embedded attempt, so the reported 'tokenBudget arrives undefined at the LCM compaction trigger point' does not match the present code path.
  - file: [src/agents/pi-embedded-runner/run.ts:867](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/pi-embedded-runner/run.ts#L867)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '840,890p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **MiniMax context window is defined:** MiniMax model definitions still declare `DEFAULT_MINIMAX_CONTEXT_WINDOW = 204800`, which is the value the thread discussion identified as feeding `ctxInfo.tokens` for MiniMax.
  - file: [extensions/minimax/model-definitions.ts:9](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/minimax/model-definitions.ts#L9)
  - command: `nl -ba extensions/minimax/model-definitions.ts | sed -n '1,120p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Preemptive compaction has missing-budget fallback:** Before pre-prompt compaction, current main computes `const contextTokenBudget = params.contextTokenBudget ?? DEFAULT_CONTEXT_TOKENS;`, so even an absent forwarded budget no longer means the compaction precheck is silently skipped.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2450](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/pi-embedded-runner/run/attempt.ts#L2450)

- **Current overflow handling retries compaction:** The current runner has explicit coverage for assistant-side context overflow errors triggering compaction retry, and only returns a bounded user-facing overflow payload after retries are exhausted.
  - file: [src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts:416](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts#L416)

- **Issue discussion contradicted the original report:** The April 14 thread was useful: one contributor traced the same code path and argued `contextTokenBudget` was already populated, and the reporter's follow-up said compaction does trigger and LCM summaries are functional around 150k-180k in CJK-heavy sessions. That contradicts the original claims that compaction was silently skipped and summaries stayed empty.
