## Summary

Keep open. Current `main` at `0bef73d151bd0a8d547b28bf16c3aa2e21078e0f` still suppresses direct-webchat conversation metadata only; it still emits the separate `Sender (untrusted metadata)` block whenever sender data resolves to a label, so the PR’s behavior is not already implemented on `main`. The matching regression test from the PR is also absent.
