## Summary

Keep open. Current `main` still emits a `Sender (untrusted metadata)` block for direct `webchat` chats, and the TUI/gateway path still sets that sender identity from the transport client (`openclaw-tui` / `gateway-client`), so this PR appears to fix a real, unmerged prompt-shaping bug.
