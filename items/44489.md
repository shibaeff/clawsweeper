## Summary

Keep open. Current `main` at `aa21d4ea7e9aab034acbf0b13f8c3575057c5c54` does not implement the proposed adaptive session reset mode: core reset types still only allow `daily | idle`, freshness logic still uses the existing daily/idle OR behavior, runtime config validation rejects any third mode, hook end reasons do not include `adaptive`, and docs still describe only daily/idle reset behavior.
