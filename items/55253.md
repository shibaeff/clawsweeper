## Summary

Keep open. Current `main` has channel-level restart mitigations and fail-open plugin hook logging, but it does not implement the core request: no generic plugin circuit breaker/auto-disable path, no session auto-restart tied to typing TTL or stuck-session warnings, and no generic timeout wrapper for normal plugin tool calls.

## Evidence

- **Plugin hook failures are fail-open, not circuit-broken:** `HookFailurePolicy` defaults to fail-open, and hook runner paths catch/log plugin hook errors instead of disabling the plugin or tripping a failure budget. That covers `before_prompt_build`/`agent_end` style hooks directly implicated by the report.
  - file: [src/plugins/hooks.ts:135](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/hooks.ts#L135)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '240,660p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Plugin service startup failures are only logged:** `startPluginServices()` logs service start failures and continues. There is no auto-disable, quarantine, or config mutation path here for a repeatedly failing plugin service.
  - file: [src/plugins/services.ts:47](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugins/services.ts#L47)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Typing TTL does not restart sessions:** The typing TTL safety path only logs and stops the typing indicator. Separately, diagnostic heartbeat only emits stuck-session warnings after `diagnostics.stuckSessionWarnMs`; it does not reset or restart the session.
  - file: [src/channels/typing.ts:50](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/channels/typing.ts#L50)

A hard timeout exists in `src/gateway/voiceclaw-realtime/tool-runtime.ts`, but that is specific to the VoiceClaw realtime path, not the general agent/plugin tool path requested in the issue.
