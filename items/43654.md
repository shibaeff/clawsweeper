## Summary

Keep open. Current `main` does not show a code-level fix for this Linux managed-browser startup failure. The browser client still hard-times out `start` after 15s and wraps it with the same generic browser-unavailable guidance, while Linux executable detection still accepts `/usr/bin/chromium-browser`. The docs now explicitly describe Ubuntu `chromium-browser` as a snap wrapper that can break OpenClaw-managed startup, which matches the issue report rather than proving the bug is fixed.
