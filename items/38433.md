## Summary

Keep open. Current `main` supports non-blocking subagent spawn, `sessions_yield`, per-child completion wakeups, and optional batching of queued announces, but it does not expose a first-class wait-for-all/barrier primitive or grouped child-completion contract that would satisfy this feature request's deterministic fan-in requirement.

## Evidence

- **Session tool surface lacks a barrier API:** The published session-tool docs list `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status`. `sessions_yield` is documented only as ending the current turn so follow-up sub-agent results arrive next, and `sessions_spawn` is documented as always non-blocking; there is no documented `sessions_await`, `sessions_wait_for_children`, `sessions_spawn_many`, or equivalent wait-for-all primitive.
  - file: [docs/concepts/session-tool.md:15](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/concepts/session-tool.md#L15)
  - command: `nl -ba docs/concepts/session-tool.md | sed -n '1,140p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Runtime tool catalog matches the docs:** The coding tool catalog registers `sessions_spawn`, `sessions_yield`, `subagents`, and `session_status` in this area, with no separate await/barrier tool present on current `main`.
  - file: [src/agents/tool-catalog.ts:168](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/tool-catalog.ts#L168)
  - command: `nl -ba src/agents/tool-catalog.ts | sed -n '168,196p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Completion events are modeled per child, not per group:** `AgentTaskCompletionInternalEvent` carries a single `childSessionKey`/`childSessionId` plus one result payload. There is no grouped completion event shape with sibling sets, pending counts, or `allComplete` state.
  - file: [src/agents/internal-events.ts:12](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/internal-events.ts#L12)
  - command: `nl -ba src/agents/internal-events.ts | sed -n '1,80p'`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Announce path wakes the parent once per child completion:** Subagent announce builds a single `task_completion` internal event and delivers it back to the requester. The direct delivery path either wakes the active requester session with that one trigger message or performs an `agent` call with that one event payload.
  - file: [src/agents/subagent-announce.ts:502](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/subagent-announce.ts#L502)

  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Existing batching is opportunistic queue collection, not a deterministic barrier:** When announce queue mode is `collect`, queued announce prompts can be merged into one follow-up message while the agent is busy, but this is delivery batching after the fact. It does not declare required siblings, persist a barrier key, or guarantee a continuation exactly when the last prerequisite child finishes.
