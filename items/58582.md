## Summary

Current main no longer matches the reported broken Anthropic CLI onboarding path and now routes away from unresolved configured primary model refs instead of continuing with them.

## Evidence

- **Anthropic CLI onboarding now defaults to Opus 4.7:** The bundled Claude CLI default model is `claude-cli/claude-opus-4-7`, the Anthropic CLI migration uses that as its fallback default, and the onboarding/auth flow writes `result.defaultModel` into `agents.defaults.model.primary`.
  - file: [extensions/anthropic/cli-migration.ts:171](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/extensions/anthropic/cli-migration.ts#L171)
  - command: `nl -ba extensions/anthropic/cli-constants.ts | sed -n '1,10p'; nl -ba extensions/anthropic/cli-migration.ts | sed -n '160,186p'; nl -ba extensions/anthropic/register.runtime.ts | sed -n '458,470p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **The reported Claude CLI Sonnet ref is covered as resolvable:** Regression coverage expects `resolveModelWithRegistry({ provider: "claude-cli", modelId: "claude-sonnet-4-6" ... })` to resolve successfully via the Anthropic forward-compat path, which contradicts the report that this ref is inherently unresolvable on current main.
  - file: [src/agents/pi-embedded-runner/model.forward-compat.test.ts:139](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-embedded-runner/model.forward-compat.test.ts#L139)
  - command: `nl -ba src/agents/pi-embedded-runner/model.forward-compat.test.ts | sed -n '120,158p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Unresolvable configured primary models no longer stay active:** If `agents.defaults.model.primary` cannot be resolved, `resolveConfiguredModelRef` logs a warning and falls back to the default/configured provider model instead of returning the unresolved ref for runtime use.
  - file: [src/agents/model-selection-shared.ts:425](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/model-selection-shared.ts#L425)
  - command: `nl -ba src/agents/model-selection-shared.ts | sed -n '378,449p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Startup path now prewarms and retries model resolution once:** The embedded runner retries one transient provider-runtime miss during startup before surfacing `Unknown model`, and the changelog records this as the gateway-startup fix for first-message unknown-model failures.
  - file: [src/agents/pi-embedded-runner/model.ts:846](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/pi-embedded-runner/model.ts#L846)
  - command: `nl -ba src/agents/pi-embedded-runner/model.ts | sed -n '846,907p'; nl -ba src/agents/pi-embedded-runner/model.startup-retry.test.ts | sed -n '71,112p'; nl -ba CHANGELOG.md | sed -n '2328,2331p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)

## Risks / Open Questions

- Docs and some QA/live-test text still mention `claude-cli/claude-sonnet-4-6` as a default (`docs/help/testing-live.md:117`), which could confuse users even though the bundled onboarding/runtime default is now `claude-cli/claude-opus-4-7`.
