## Summary

Current `main` already handles the reported behavior: workspace-installed skills are loaded with their real source, the status path reports that source through the gateway/CLI, and a regression test on `main` and the latest release explicitly covers the bundled-name override case. This PR’s behavior change is already satisfied on shipped code.

## Evidence

That means the reported mislabel does not occur for normal workspace-loaded skills on current `main`.
