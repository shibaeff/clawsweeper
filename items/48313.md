## Summary

Keep open. Current `main` still lacks the requested inbound Feishu interactive-card parsing on the bot/event path, so this is not implemented on `main`, and the PR itself has unresolved correctness issues called out in review.

## Evidence

- **Inbound Feishu messages still route through `bot-content.ts`:** `parseFeishuMessageEvent()` imports `parseMessageContent` from `./bot-content.js` and uses it for inbound event content parsing, so the feature would need to exist there to be shipped on `main`.
