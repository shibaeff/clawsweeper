## Summary

Keep open. Current `main` has explicit browser download interception helpers, but it does not implement automatic saving for ordinary in-browser JavaScript/blob-triggered downloads from the managed `openclaw` profile. The reported bug still matches the current design.

## Evidence

- **Managed browser launch has no download-save configuration:** `buildOpenClawChromeLaunchArgs` sets CDP/user-data/headless/sandbox flags only. There is no launch arg or download behavior setup for a default downloads directory or automatic blob download persistence.
  - file: [extensions/browser/src/browser/chrome.ts:104](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/browser/src/browser/chrome.ts#L104)
  - command: `nl -ba extensions/browser/src/browser/chrome.ts | sed -n '80,140p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Profile decoration writes color/name prefs, not download prefs:** The managed profile setup mutates `Local State` and `Default/Preferences` for profile naming/color only. It does not write Chrome download preferences such as a default download directory or prompt suppression.
  - file: [extensions/browser/src/browser/chrome.profile-decoration.ts:108](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/browser/src/browser/chrome.profile-decoration.ts#L108)

- **Current download support is explicit interception endpoints only:** Browser download handling on `main` is exposed through `/wait/download` and `/download`, which call Playwright helpers only when the caller explicitly invokes those routes. That is not the same as ordinary user clicks inside the managed browser automatically saving files.
  - file: [extensions/browser/src/browser/routes/agent.act.download.ts:23](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/browser/src/browser/routes/agent.act.download.ts#L23)

- **CLI/docs still describe manual download commands, not automatic browser-profile downloads:** The documented browser download flow is `openclaw browser waitfordownload` or `openclaw browser download <ref> ...`. I did not find any current doc/config surface promising that clicking a site download button in the managed browser will persist the file automatically to disk.
  - file: [docs/cli/browser.md:161](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/cli/browser.md#L161)
