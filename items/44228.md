## Summary

Keep open. Current `main` still does not normalize Windows drive-letter media paths in reply-media dedupe, so the reported behavior is not implemented on `aa21d4ea7e9aab034acbf0b13f8c3575057c5c54`. The PR also mixes in unrelated config/protocol changes that are already present on `main`, so it still needs maintainer cleanup rather than closure.
