## Summary

Keep open. Current `main` does not persist the Discord component/modal registry across gateway restarts: the registry is stored only in process-global maps, the tests cover duplicate module instances in one process rather than restart persistence, and the commenter-cited replacement PR #66241 is still open and unmerged.

## Evidence

- **Current main stores registry only in process-global maps:** `components-registry.ts` initializes `componentEntries` and `modalEntries` via `resolveGlobalMap(...)` and all register/resolve paths operate only on those in-memory maps. There is no state-dir path, JSON store, or disk reload logic in current `main`.
  - file: [extensions/discord/src/components-registry.ts:11](https://github.com/openclaw/openclaw/blob/151befb90b903f52dce9f4951991d39b9df8b358/extensions/discord/src/components-registry.ts#L11)

- **Current tests do not verify restart persistence:** The registry tests cover normal register/consume behavior and sharing state across duplicate module instances loaded in the same process. They do not cover persistence to disk or recovery after a gateway restart.
