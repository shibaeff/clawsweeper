## Summary

Current `main` already serializes Telegram turns per chat/topic and adds regression coverage for late `message_start` / partial-ordering cases, so the reported same-session streaming race is implemented on `main`.

## Evidence

  - file: [extensions/telegram/src/sequential-key.ts:51](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/extensions/telegram/src/sequential-key.ts#L51)
