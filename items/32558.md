## Summary

Keep open. Current `main` does not implement upload-session fallback, and while the PR's original `>4MB` motivation is weakened by current Microsoft Graph docs, `channels.msteams.mediaMaxMb` remains configurable and the PR still proposes real resumable-upload behavior beyond what ships today. That is not strong enough evidence for a conservative auto-close.

## Evidence

- **OneDrive upload on main is still a single `PUT ...:/content` call:** `uploadToOneDrive` on current `main` has no size threshold or upload-session branch; it directly `PUT`s to `/me/drive/root:/...:/content`.

## Risks / Open Questions

- The PR summary assumes Graph simple upload is limited at 4MB, but current Microsoft docs say 250MB; if this lands later, it should be re-scoped against the current dependency contract rather than the older premise.
- `channels.msteams.mediaMaxMb` is configurable, so there is still a plausible gap for installations that raise the cap above the default. That makes an auto-close too aggressive without a clearer maintainer decision on desired support limits.
