## Summary

Current `main` does not implement `VALUE.md` as a first-class workspace context file, and the existing bootstrap machinery explicitly recognizes only the current built-in filenames. The report is coherent, recent, and still actionable, so there is not strong evidence to close it.

## Evidence

- **Bootstrap filename set excludes VALUE.md:** The workspace bootstrap filename constants, union type, and runtime validation set include `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and `MEMORY.md`, with no `VALUE.md`.
  - file: [src/agents/workspace.ts:30](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/workspace.ts#L30)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Workspace seeding/loading has no VALUE.md path:** Workspace setup seeds only the existing templates, and bootstrap loading enumerates only those same files when building injected workspace context.
  - file: [src/agents/workspace.ts:364](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/workspace.ts#L364)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Docs list current injected bootstrap files only:** The system-prompt docs enumerate the injected workspace bootstrap files and do not mention `VALUE.md`.
  - file: [docs/concepts/system-prompt.md:99](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/concepts/system-prompt.md#L99)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)

  - file: [docs/concepts/agent-workspace.md:62](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/concepts/agent-workspace.md#L62)
