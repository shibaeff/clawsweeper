## Summary

Current `main` no longer uses the heuristic host-exec obfuscation gate that produced the misleading approval-unavailable message, and the change shipped in `v2026.4.5`. The reporter’s own follow-up on `2026.4.5` shows a different direct preflight error, which is consistent with this specific bug being fixed.

## Evidence

- **Release note:** The `2026.4.5` changelog explicitly says host exec removed heuristic command-obfuscation gating, so the old `command too long` approval-mismatch path is no longer part of host exec.
  - file: [CHANGELOG.md:1341](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L1341)

- **Exec docs:** Current user docs state that in `security=full` plus `ask=off` mode there is no extra heuristic command-obfuscation prefilter or script-preflight rejection layer.
  - file: [docs/tools/exec.md:102](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/tools/exec.md#L102)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)

  - file: [docs/tools/exec-approvals.md:133](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/tools/exec-approvals.md#L133)
