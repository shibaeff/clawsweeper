## Summary

Current `main` still omits the implicit `default` account when a channel has named `accounts`, while Slack and Discord runtime resolution can inherit base credentials into those named accounts. That means the PR still targets a real behavior gap, and its review thread also shows unresolved edge-case questions, so this is not a safe cleanup close.

## Evidence

- **Current helper still omits implicit default:** On `main`, `listAccountIds()` only returns configured account IDs, with `default` used only as the empty-list fallback. There is no current logic that adds `default` when base channel tokens exist alongside named accounts.
