## Summary

Keep open. Current `main` does not implement this restart path, and the PR as proposed is not safe: the launchd self-restart branch still only schedules a detached handoff, while authorized in-process SIGUSR1 restarts must go through the restart authorization path that this PR bypasses.

## Evidence

- **Current main still lacks the claimed fix:** In the launchd self-restart branch, `restartLaunchAgent` only schedules `scheduleDetachedLaunchdRestartHandoff({ mode: "kickstart", waitForPid: process.pid })` and returns `{ outcome: "scheduled" }`. There is no `emitGatewayRestart()` call and no SIGUSR1 emission in this code path.
  - file: [src/daemon/launchd.ts:638](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/launchd.ts#L638)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '625,690p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Authorized SIGUSR1 restarts require restart authorization:** `emitGatewayRestart()` increments restart-cycle state and calls `authorizeGatewaySigusr1Restart()` before emitting SIGUSR1. That authorization is part of the contract for in-process restart delivery.
  - file: [src/infra/restart.ts:120](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/infra/restart.ts#L120)
  - command: `nl -ba src/infra/restart.ts | sed -n '115,180p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Unauthorized SIGUSR1 is ignored by the gateway run loop:** The gateway SIGUSR1 handler calls `consumeGatewaySigusr1RestartAuthorization()`. When authorization is absent and external restarts are not allowed, it logs `SIGUSR1 restart ignored` and returns without restarting.
  - file: [src/cli/gateway-cli/run-loop.ts:230](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/cli/gateway-cli/run-loop.ts#L230)
  - command: `nl -ba src/cli/gateway-cli/run-loop.ts | sed -n '226,252p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Existing tests only prove detached handoff scheduling:** The launchd unit test for this branch asserts only that the detached helper is scheduled with `waitForPid: process.pid`; it does not cover an authorized SIGUSR1 restart through the gateway run loop.
  - file: [src/daemon/launchd.test.ts:753](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/daemon/launchd.test.ts#L753)
