## Summary

Keep open. Current `main` still has a streaming-only gap for Anthropic-style reasoning tags: the live subscribe path strips `<think>`, `<thinking>`, `<thought>`, and `<antthinking>`, but not `<antml:thinking>`. Shared final-text sanitization does handle `antml:` tags, which matches the report/comment that leakage happens during streaming rather than only in final message cleanup.

## Evidence

- **Streaming strip regex omits `antml:` tags:** `subscribeEmbeddedPiSession` uses `THINKING_TAG_SCAN_RE = /<\s*(\/?)\s*(?:think(?:ing)?|thought|antthinking)\s*>/gi;`, so the streaming block stripper does not recognize `<antml:thinking>...</antml:thinking>`.
  - file: [src/agents/pi-embedded-subscribe.ts:38](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/pi-embedded-subscribe.ts#L38)

  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Streaming reasoning extraction helpers also omit `antml:` tags:** The tagged-text helpers used by streaming reasoning/block promotion only match `think`, `thinking`, `thought`, and `antthinking`, not `antml:thinking`, so both partial reasoning extraction and write-time promotion miss the namespaced Anthropic form.
  - file: [src/agents/pi-embedded-utils.ts:192](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/pi-embedded-utils.ts#L192)
