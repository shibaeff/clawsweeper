## Summary

Current `main` already applies cron `payload.model` at runtime for isolated `agentTurn` jobs, forwards that selection into execution, and documents/tests the precedence, so the reported behavior is implemented on `main` rather than still open.

## Evidence

- **Runtime model selection reads `payload.model`:** `resolveCronModelSelection()` explicitly reads `agentTurn` payload `model`, validates it, and promotes it over the default/session selection.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/model-selection.ts#L104)
  - command: `sed -n '1,260p' src/cron/isolated-agent/model-selection.ts`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Prepared cron run uses resolved provider/model before execution:** `run.ts` calls `resolveCronModelSelection()` and assigns the returned provider/model into the live execution context before the run starts.
  - file: [src/cron/isolated-agent/run.ts:518](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/run.ts#L518)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '500,640p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Executor forwards the selected model into fallback runner:** `createCronPromptExecutor()` passes `params.liveSelection.provider` and `params.liveSelection.model` into `runWithModelFallback()`, so execution uses the resolved cron payload override rather than agent defaults.
  - file: [src/cron/isolated-agent/run-executor.ts:110](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/run-executor.ts#L110)
  - command: `nl -ba src/cron/isolated-agent/run-executor.ts | sed -n '1,180p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Regression tests cover the exact runtime-forwarding bug:** The regression suite asserts that a cron payload model override reaches `runWithModelFallback()` and the embedded agent runner as `google/gemini-2.0-flash`, not the agent default.
  - file: [src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts:108](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts#L108)
  - command: `nl -ba src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts | sed -n '69,240p'`

- **Docs now describe per-job `--model` precedence:** CLI/docs state that isolated cron model precedence is Gmail hook override, then per-job `--model`, then stored cron-session override, then agent/default selection.
