## Summary

Keep open. Current `main` still passes an empty tool list through the `codex-cli` runner when building both the system prompt and `systemPromptReport`, while the bundled `codex-cli` backend still enables bundle MCP injection. The report bug described in #65404 appears to remain present on `main` and in the latest release tag provided.

## Evidence

- **CLI runner still drops tools before reporting:** `prepareCliRunContext()` builds the CLI system prompt with `tools: []` and then calls `buildSystemPromptReport()` with `tools: []` again, so the CLI path cannot populate `systemPromptReport.tools.entries` or `schemaChars` from injected MCP tools.
  - file: [src/agents/cli-runner/prepare.ts:284](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/src/agents/cli-runner/prepare.ts#L284)
  - command: `nl -ba src/agents/cli-runner/prepare.ts | sed -n '280,379p'`
  - sha: [30aa7e0d4d22](https://github.com/openclaw/openclaw/commit/30aa7e0d4d225650b58c2fba79e71bffa5627b08)
- **Report builder only reflects provided tool array:** `buildSystemPromptReport()` derives `tools.entries` by mapping the passed `tools` array and sums `schemaChars` from those entries. If callers pass `[]`, the report necessarily returns empty entries and zero schema chars.
  - file: [src/agents/system-prompt-report.ts:39](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/src/agents/system-prompt-report.ts#L39)
  - command: `nl -ba src/agents/system-prompt-report.ts | sed -n '34,130p'`

- **Bundled codex-cli still enables MCP bridge:** The bundled OpenAI CLI backend still declares `bundleMcp: true`, `bundleMcpMode: "codex-config-overrides"`, and `defaultMcpProbe: true`, so `codex-cli` is still expected to receive MCP tools through the loopback bridge.
  - file: [extensions/openai/cli-backend.ts:9](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/extensions/openai/cli-backend.ts#L9)
