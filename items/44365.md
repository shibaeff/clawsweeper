## Summary

Keep open. Current `main` still auto-switches Linux npm installs to `~/.npm-global` in `scripts/install.sh`, and the user-facing update docs still present manual `npm i -g openclaw@latest` without a warning about the sudo/user-prefix split. `openclaw update` has newer owning-prefix logic, but that does not fix the reported manual `sudo npm i -g` path.

## Evidence

- **Installer still rewrites npm prefix to ~/.npm-global on Linux:** `fix_npm_permissions()` calls `npm config set prefix "$HOME/.npm-global"`, appends the PATH export to shell rc files, and is still invoked in the default npm install flow before `install_openclaw`.
