## Summary

Keep open. Current `main` still aborts the shared Telegram client fetch on polling restart/shutdown, so the PR's reported regression is not implemented on `main`. The shipped tests and docs still describe the restart path that can trigger this, and the main code does not contain the PR's `getUpdates`-only scoping. The review comment about `apiThrottler` swallowing abort signals is not a strong reason to close this PR: the pinned `@grammyjs/transformer-throttler` version forwards `signal` through to `prev(...)` upstream.

## Evidence
