## Summary

Keep open. Current `main` now has an `openclaw docs` CLI, but its implementation still shells out to `mcporter` against `https://docs.openclaw.ai/mcp.SearchOpenClaw`, and a direct invocation from this checkout fails with `MCP error -32602: Tool SearchOpenClaw not found`. I do not have high-confidence evidence that the reported macOS docs failure is resolved on current `main`.

## Evidence

- **Current implementation still depends on the live docs MCP URL:** `docsSearchCommand` hardcodes `https://docs.openclaw.ai/mcp.SearchOpenClaw` and invokes it through `mcporter call ... --args <payload> --output text` for every non-empty query.
  - file: [src/commands/docs.ts:8](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/commands/docs.ts#L8)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **CLI surface is documented as a live docs search:** The user-facing docs say `openclaw docs` searches the live docs index and accepts query terms passed through as one search request.
  - file: [docs/cli/docs.md:8](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/docs/cli/docs.md#L8)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Direct tool invocation still fails from this checkout:** Running the same selector style used by the implementation returned `MCP error -32602: Tool SearchOpenClaw not found`, so I could not confirm working search behavior on current `main`.
  - command: `npx -y mcporter call https://docs.openclaw.ai/mcp.SearchOpenClaw --args '{"query":"gateway token secretref"}' --output text`
- **No dedicated regression test covers this command path:** A repository search for `docsSearchCommand`, `registerDocsCli`, and `SearchOpenClaw` found the implementation and registration, but no command-specific tests validating a successful docs search path.
  - command: `rg -n "docsSearchCommand|registerDocsCli|SearchOpenClaw" test src`
