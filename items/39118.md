## Summary

Keep open. Current `main` does not contain this PR's protections, and the review discussion raises credible unresolved regressions in the proposed implementation: it would likely hide custom-named extra gateways, miss system-scope duplicates during `--deep` scans, and fail to suppress mismatch-only `ExecStart` rewrites because that issue is computed later in the doctor flow.

## Evidence

- **Deep scan still covers user and system services:** `findExtraGatewayServices` scans user services first and, with `--deep`, also walks `/etc/systemd/system`, `/usr/lib/systemd/system`, and `/lib/systemd/system`. `maybeScanExtraGatewayServices` then reports every returned service. The public docs also say `--deep` scans for extra launchd/systemd/schtasks installs and warns when multiple gateway-like services are detected.

## Risks / Open Questions

- The underlying bug report still looks plausible on current `main`: companion services can still be surfaced as extra gateway-like services, and doctor still repairs entrypoint mismatches without a running-service exception.
- If this PR is revived, it needs scope-aware active checks and detection rules that preserve custom-labeled extra gateways on Linux and macOS.
