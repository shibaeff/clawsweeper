## Summary

Current `main` does not appear to handle Discord inbound message edits. The Discord monitor still registers only `MessageCreate`, reaction, thread-update, and optional presence listeners, and the inbound handler types are still anchored to `MessageCreateListener`, so the feature request remains actionable rather than already fixed or stale-obsolete.

## Evidence

- **Monitor listener registration has no edit listener:** `registerDiscordMonitorListeners` registers `DiscordMessageListener`, reaction add/remove, thread update, and optional presence listeners only. There is no `messageUpdate`/edit listener registration in this startup path.
  - file: [extensions/discord/src/monitor/provider.startup.ts:253](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/extensions/discord/src/monitor/provider.startup.ts#L253)

  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Inbound Discord message listener is create-only:** The listener module imports `MessageCreateListener`, defines `DiscordMessageEvent` from `MessageCreateListener["handle"]`, and `DiscordMessageListener` extends `MessageCreateListener`. That keeps the inbound pipeline typed and wired around create events, not edit events.
  - file: [extensions/discord/src/monitor/listeners.ts:4](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/extensions/discord/src/monitor/listeners.ts#L4)
