## Close Comment

Closing from a Codex review: current `main` already sanitizes this class of provider error before it reaches user-visible chat. The outbound reply path runs `sanitizeUserFacingText` in `src/auto-reply/reply/normalize-reply.ts:98`, that sanitizer rewrites `Codex error: {...}` payloads via `formatRawAssistantErrorForUi` in `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts:374` and `:417`, and the formatter emits `LLM error <type>: <message>` instead of raw JSON in `src/shared/assistant-error-format.ts:206`. There are explicit regression tests for the reported Codex `server_error` payload shape in `src/agents/pi-embedded-helpers.sanitizeuserfacingtext.test.ts:103` and `src/agents/pi-embedded-helpers.formatassistanterrortext.test.ts:97`. The same implementation is already present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this issue appears fixed on `main` and in release.
