## Summary

Keep open. Current `main` still does not implement a WhatsApp `channels.whatsapp.silentErrorReplies` setting or schema parity with Telegram, so the specific request in #68457 is not implemented. There is a new unreleased core change on `main` that treats exact `NO_REPLY` as a deliberate silent reply, which may mitigate the reported symptom for that one case, but it does not add the requested WhatsApp config surface and does not prove broader error-reply parity.
