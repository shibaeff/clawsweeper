## Summary

Keep open. Current `main` does not contain the PR's core lock-loss protection: the session lock watchdog still force-releases locks without notifying the holder, `acquireSessionWriteLock()` has no `onForceRelease` hook, `runEmbeddedAttempt()` still only acquires the lock and does not install any session write guard, and the proposed new guard files/tests are absent. One review concern about teardown leaking the session lock is already addressed separately by `cleanupEmbeddedAttemptResources()`, but the concurrent-write-after-lock-loss fix itself is not implemented on `main`.
