## Summary

Keep open. I did not find strong evidence that `main` fully fixes #45364. Current `main` still ships `node-llama-cpp` as an optional peer rather than an installed dependency, still lazy-imports it at runtime, and still tells users that reinstalling OpenClaw should install it. That guidance conflicts with the package metadata and leaves the Linux/npm global-update regression plausible rather than clearly resolved.

## Evidence

- **Package metadata still does not install node-llama-cpp with OpenClaw:** `package.json` declares `node-llama-cpp` under `peerDependencies` with `peerDependenciesMeta.optional=true`, not under `dependencies` or `optionalDependencies`. A normal `npm i -g openclaw@latest` therefore does not itself guarantee that `node-llama-cpp` is installed alongside OpenClaw.
