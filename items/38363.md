## Summary

Keep open. Current `main` does not implement a configurable chat send shortcut: the chat composer still hardcodes Enter-to-send behavior, persisted UI settings do not include a `chatEnterSends` flag, and the chat controls do not expose a shortcut toggle. This PR is still net-new work rather than something already shipped or obsolete.

## Evidence

- **Composer still hardcodes Enter-to-send:** `renderChat` still shows `Message ... (Enter to send)` and the keydown handler sends only on `Enter` without `Shift`, so the requested configurable shortcut is not present on current `main`.
