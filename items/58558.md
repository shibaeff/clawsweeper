## Summary

Current `main` still treats `tool_result_persist` and `before_message_write` as sync-only hooks, explicitly ignores Promise-returning handlers, and documents `tool_result_persist` as synchronous. The requested async support is not implemented on `main`, so this issue should stay open.

## Evidence

- **Public hook types are still synchronous:** `PluginHooks` defines `tool_result_persist` and `before_message_write` without any `Promise` return in `src/plugins/hook-types.ts:723` and `src/plugins/hook-types.ts:727`.
  - file: [src/plugins/hook-types.ts:723](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/plugins/hook-types.ts#L723)

## Risks / Open Questions

- Adding async support would change a hot transcript-persistence path and likely requires coordinated updates across the hook runner, session guard, SDK types, docs, and compatibility tests.
- `before_message_write` has the same sync-only design, but the current issue is still actionable even if maintainers choose to scope a fix to `tool_result_persist` first.
