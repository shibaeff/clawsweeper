## Summary

Keep open. Current main appears to have fixed the specific gateway-auth interception theory for `/googlechat`, but the reported 401 remains plausibly reproducible because `app-url` webhook auth now requires `channels.googlechat.appPrincipal` to match the JWT `sub`, and that requirement is still not surfaced in the Google Chat docs or UI.

## Evidence

- **Google Chat webhook is now registered as plugin-managed auth:** The Google Chat webhook target registers `/googlechat` with `auth: "plugin"`, so the route is intended to bypass gateway bearer auth and let the plugin verify Google tokens itself.
  - file: [extensions/googlechat/src/monitor-routing.ts:27](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/extensions/googlechat/src/monitor-routing.ts#L27)
  - command: `nl -ba extensions/googlechat/src/monitor-routing.ts | sed -n '1,55p'`
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **Gateway tests cover bearer-header traffic to /googlechat without gateway interception:** Current tests explicitly assert that `/googlechat` returns 200 both without auth and with `Authorization: Bearer test-token`, while `/api/channels/...` remains gateway-protected. That does not match the April comment claiming the gateway consumed the bearer header before the plugin ran.
  - file: [src/gateway/server.plugin-http-auth.test.ts:541](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/gateway/server.plugin-http-auth.test.ts#L541)
  - command: `nl -ba src/gateway/server.plugin-http-auth.test.ts | sed -n '541,592p'`
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **App-url auth still hard-fails without add-on principal binding:** For `audienceType === "app-url"`, add-on issuer tokens are rejected unless `expectedAddOnPrincipal` is configured and matches `payload.sub`; otherwise the verifier returns `missing add-on principal binding` or `unexpected add-on principal` and the webhook path answers 401.
  - file: [extensions/googlechat/src/auth.ts:169](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/extensions/googlechat/src/auth.ts#L169)
  - command: `nl -ba extensions/googlechat/src/auth.ts | sed -n '151,181p'`
  - sha: [9153598e659d](https://github.com/openclaw/openclaw/commit/9153598e659daf5a52c479731f5b4ab8a5bf16b7)
- **User-facing docs and UI still omit appPrincipal guidance:** The Google Chat docs describe only `audienceType` and `audience` in setup/config/troubleshooting, and the UI status card surfaces only the audience fields. A repo search found `appPrincipal` only in schema/types metadata, not in the Google Chat docs or UI.
  - file: [docs/channels/googlechat.md:139](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/docs/channels/googlechat.md#L139)
