## Summary

Keep open. Current `main` still treats `agents.defaults.models` entries as strict `alias`/`params`/`streaming` objects and `applyModelDefaults()` does not merge arbitrary passthrough fields into provider model definitions, so the PR's requested behavior is not already implemented on `main` and is still a live code change rather than conservative cleanup.

## Evidence
