## Summary

Keep open. Current main shows account-scoped Mattermost send, pairing, and session-routing paths, but the issue thread does not establish a reproducible core bug or a shipped fix. The reporter later said deleting a corrupted `mattermost-pairing.json` resolved the mis-posting, yet current docs define that file as pending DM pairing state rather than outbound sender-routing state, so the root cause is still unclear and not strong enough for `cannot_reproduce` or `implemented_on_main`.
