## Summary

Keep open. Current `main` partially hardens GPT-5.4/Codex context persistence, but the issue is not fully obsolete: read-only status/default-session paths still fall back to `200000`, and other persistence paths still use sync-only context resolution. The report's expectation of `1050000` everywhere is not the current contract, though: `main` treats native `contextWindow` and runtime `contextTokens` as separate values for Codex GPT-5.4.

## Evidence

- **Codex GPT-5.4 contract is split, not 1M everywhere:** The provider defines GPT-5.4 with native `contextWindow = 1_050_000` and default runtime `contextTokens = 272_000`, and the provider test asserts that exact split. That means the reporter's 'full 1M everywhere' expectation does not match current provider behavior.
