## Summary

Current `main` still rejects explicit WhatsApp direct-message targets unless they are in `allowFrom`, so PR #48387 is not implemented on `main` and remains a live behavior change rather than a stale cleanup candidate.

## Evidence

- **Current resolver still gates direct targets on allowFrom for all non-group sends:** `resolveWhatsAppOutboundTarget` returns success for group JIDs, wildcard, empty allowlist, or allowlist match, and otherwise errors. The `mode` parameter is accepted but never consulted, so explicit targets are still gated the same as implicit ones.
