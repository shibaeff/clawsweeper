## Summary

Keep open. Current `main` still does not expose the PR's live queue diagnostics surface: there is no `diagnostics.queue` Gateway RPC, no `openclaw gateway diagnostics queue` CLI command, and no queue snapshot API in `src/process/command-queue.ts`. Existing diagnostics on `main` are the bounded stability recorder and support export, which overlap operationally but do not provide the requested first-class live lane/session backlog snapshot.
