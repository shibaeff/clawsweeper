## Summary

Keep open. Current `main` still lacks the main safeguards requested here: `openclaw mcp set` persists config without startup validation, the bundle MCP runtime has no per-server backoff/circuit-breaker state, and the generated/documented systemd units still omit default `MemoryHigh`/`MemoryMax`/`TasksMax`/`OOMScoreAdjust` guardrails. Current code does contain cleanup/OOM mitigations, so parts of the incident narrative are partly mitigated, but not enough to close conservatively.
