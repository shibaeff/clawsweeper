## Summary

Current `main` already ships Feishu document creation and related wiki placement as supported bundled-plugin capabilities: the Feishu plugin registers doc/wiki/drive tools, `feishu_doc` implements `action: "create"` with optional folder placement and requester-permission handling, `feishu_wiki` implements wiki-node creation, bundled skill docs document the workflow, and tests cover the create path and guardrails.

## Evidence

- **Bundled Feishu plugin exposes doc/wiki/drive tools:** The bundled Feishu entry registers `registerFeishuDocTools`, `registerFeishuWikiTools`, and `registerFeishuDriveTools` from `registerFull`, so these workflows are part of the shipped plugin surface.
  - file: [extensions/feishu/index.ts:73](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/index.ts#L73)
- **Document creation is part of the public tool schema:** `feishu_doc` includes `action: "create"` with `title`, optional `folder_token`, and `grant_to_requester` in the tool parameter schema.
  - file: [extensions/feishu/src/doc-schema.ts:42](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/src/doc-schema.ts#L42)
- **Document creation is implemented in runtime:** `createDoc()` calls `client.docx.document.create`, supports optional folder placement, and returns the created document id and URL while handling requester permission grants.
  - file: [extensions/feishu/src/docx.ts:907](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/src/docx.ts#L907)
- **Knowledge-base/wiki creation is implemented:** `feishu_wiki` handles `action: "create"` by creating a wiki node, which covers the issue's requested wiki or knowledge-base placement path.
  - file: [extensions/feishu/src/wiki.ts:203](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/src/wiki.ts#L203)
- **Bundled skill docs document the create workflow:** The Feishu doc skill docs include a dedicated 'Create Document' section, including optional `folder_token` usage and permission guidance.
  - file: [extensions/feishu/skills/feishu-doc/SKILL.md:43](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/skills/feishu-doc/SKILL.md#L43)
- **Tests cover create behavior and safety guardrails:** Tests assert successful create, trusted-requester permission grants, skip behavior when requester identity is unavailable, and opt-out behavior via `grant_to_requester: false`.
  - file: [extensions/feishu/src/docx.test.ts:377](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/extensions/feishu/src/docx.test.ts#L377)
- **Latest release tag contains the same implementation:** The `v2026.4.22` tag contains the same `feishu_doc` create implementation and schema, so this is not only on unreleased `main`.
  - command: `git show v2026.4.22:extensions/feishu/src/docx.ts | sed -n '907,960p'; git show v2026.4.22:extensions/feishu/src/doc-schema.ts | sed -n '35,55p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- `docs/channels/feishu.md` is still mostly channel-setup documentation and does not directly walk through the doc/wiki tool flows, so capability discovery still leans on bundled skill docs.
- `extensions/feishu/skills/feishu-doc/SKILL.md` still shows `owner_open_id` in examples even though the current schema/runtime use trusted requester context plus `grant_to_requester`, so that example text appears stale.

## Close Comment

Closing from a Codex review because current `main` already implements this capability well enough. The bundled Feishu plugin registers `feishu_doc`, `feishu_wiki`, and `feishu_drive` as shipped plugin features (`extensions/feishu/index.ts:73`), `feishu_doc` supports `action: "create"` with optional folder placement (`extensions/feishu/src/doc-schema.ts:42`, `extensions/feishu/src/docx.ts:907`), `feishu_wiki` supports wiki-node creation for knowledge-base placement (`extensions/feishu/src/wiki.ts:203`), and the bundled skill docs document the create flow and permissions (`extensions/feishu/skills/feishu-doc/SKILL.md:43`). The create path is also covered by tests (`extensions/feishu/src/docx.test.ts:377`) and is present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
