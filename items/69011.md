## Summary

Keep open. Current `main` still contains the same bonjour watchdog/recreate logic the report describes, and the bundled tests explicitly codify restart-loop behavior for services stuck in `announcing` or churning between `probing` and `announcing`. I did not find evidence on `main` or in the latest shipped release that the underlying ciao self-probe/state-transition problem has been fixed; docs only document disable/off switches as workarounds.
