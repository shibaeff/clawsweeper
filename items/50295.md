## Summary

Current `main` already removes the separate obfuscation/preflight approval override in YOLO host-exec mode, so the reported `security=full` + `ask=off` behavior is already addressed.

## Evidence

- **Approval policy no longer includes an obfuscation override:** `requiresExecApproval` now depends only on `ask`, `security`, `analysisOk`, `allowlistSatisfied`, and durable approval state. There is no obfuscation flag or command-length heuristic in this gate.
  - file: [src/infra/exec-approvals.ts:772](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/infra/exec-approvals.ts#L772)

- **Gateway host exec uses policy-only gating plus allowlist/heredoc/strict-inline-eval checks:** The gateway host `requiresAsk` path is `requiresExecApproval(...) || requiresAllowlistPlanApproval || requiresHeredocApproval || requiresInlineEvalApproval`; there is no obfuscation-detection branch.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:173](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/bash-tools.exec-host-gateway.ts#L173)

## Close Comment

Closing this as implemented after a Codex review. Current `main` no longer has the separate obfuscation/preflight approval override described here: approval is now driven by the configured exec policy in `src/infra/exec-approvals.ts:772`, the gateway host path has no obfuscation branch in `src/agents/bash-tools.exec-host-gateway.ts:173`, the node host path likewise has none in `src/agents/bash-tools.exec-host-node.ts:194`, and YOLO host mode explicitly skips script preflight in `src/agents/bash-tools.exec.ts:1046`. There is also regression coverage for YOLO/preflight/heredoc behavior in `src/agents/bash-tools.exec.script-preflight.test.ts:358`, and the shipped docs for `v2026.4.22` state that `security=full` + `ask=off` does not add a heuristic command-obfuscation or script-preflight gate. Because the reported behavior is already fixed on `main` and shipped in `v2026.4.22`, there is nothing left to track in this issue.
