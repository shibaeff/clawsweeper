## Summary

Keep open. Current `main` does not show a fix for this report. The code/docs only preserve partial progress for sub-agent timeouts, while terminal error paths intentionally suppress cached child output and fall back to `(no output)`. I also did not find evidence of a current retry/checkpoint path for a large `write` tool call failing with `Network connection lost`.
