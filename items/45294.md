## Summary

The reported silent-startup/runtime-only failure is already addressed on current `main` and in the shipped `v2026.4.22` release: gateway startup now prewarms an explicit primary model and logs a startup warning when model resolution misses, so an invalid Google model ref no longer waits silently until first inference.

## Close Comment

Closing from a Codex review. Current `main` and the shipped `v2026.4.22` release no longer match the reported behavior where an invalid Google model id starts cleanly and only fails on first inference. Gateway startup now prewarms an explicit `agents.defaults.model.primary` and logs `startup model warmup failed for <provider>/<model>` when model resolution misses (`src/gateway/server-startup-post-attach.ts:64`, invoked at `src/gateway/server-startup-post-attach.ts:250`), and that warning path is covered by `src/gateway/server-startup.test.ts:241`. Google model normalization still does not accept `gemini-3.0-flash` (`extensions/google/model-id.ts:3`), so that bad ref now surfaces during startup warmup instead of remaining silent until runtime. If anyone still sees a clean startup with no warning on `v2026.4.22` or newer, please open a new issue with the exact config and startup logs.
