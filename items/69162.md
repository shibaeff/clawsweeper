## Summary

Current main already uses the same cron execution path and cron-triggered exec-approval semantics for manual `openclaw cron run` and scheduled fires. The reported dry-run/scheduled-fire divergence does not match the shipped code path in `v2026.4.23`.

## Evidence

- **Gateway manual run entrypoint:** `cron.run` does not execute a separate dry-run engine; it calls `context.cron.enqueueRun(jobId, mode)`.
  - file: [src/gateway/server-methods/cron.ts:357](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/gateway/server-methods/cron.ts#L357)
  - command: `nl -ba src/gateway/server-methods/cron.ts | sed -n '357,389p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Manual runs use shared core executor:** Manual cron runs go through `finishPreparedManualRun()`, which calls `executeJobCoreWithTimeout(state, executionJob)` before applying normal job results.
  - file: [src/cron/service/ops.ts:594](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cron/service/ops.ts#L594)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '594,731p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Scheduled fires use the same core executor:** Scheduled due jobs call `executeJob()`, which also routes into `executeJobCoreWithTimeout(state, job)`. `executeJobCore()` then handles both main-session and isolated cron jobs.
  - file: [src/cron/service/timer.ts:1133](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cron/service/timer.ts#L1133)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1128,1375p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Isolated cron turns always carry `trigger: "cron"`:** The isolated cron executor launches both CLI and embedded agent runs with `trigger: "cron"`, so exec approvals see the same cron context regardless of whether the run was scheduled or manually started.
  - file: [src/cron/isolated-agent/run-executor.ts:127](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/cron/isolated-agent/run-executor.ts#L127)
  - command: `nl -ba src/cron/isolated-agent/run-executor.ts | sed -n '90,190p'`
  - sha: [7f6452897e25](https://github.com/openclaw/openclaw/commit/7f6452897e25543c9263b67aabf140379d1c0b0c)
- **Cron exec approvals are explicitly headless:** The exec approval helper treats `trigger === "cron"` as headless and builds the exact denial text `exec denied: Cron runs cannot wait for interactive exec approval.`
  - file: [src/agents/bash-tools.exec-host-shared.ts:370](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/bash-tools.exec-host-shared.ts#L370)

  - file: [src/agents/bash-tools.exec.approval-id.test.ts:1455](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/src/agents/bash-tools.exec.approval-id.test.ts#L1455)
