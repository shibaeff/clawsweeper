## Summary

Current `main` already prevents persisted live-session selection from overriding failover unless an explicit `liveModelSwitchPending` flag is set, and it normalizes `LiveSessionModelSwitchError` into failover handling so fallback can continue instead of looping on the overloaded model.

## Evidence

- **Live switch gated behind explicit pending flag:** `shouldSwitchToLiveModel()` now returns `undefined` unless the persisted session entry has `liveModelSwitchPending`, so persisted model selection is not reapplied on ordinary failover retries.
  - file: [src/agents/live-model-switch.ts:164](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/live-model-switch.ts#L164)

- **Active run only throws model-switch error for explicit pending switch:** The embedded runner consults `shouldSwitchToLiveModel()` and only throws `LiveSessionModelSwitchError` when that explicit pending switch exists and the attempt can safely restart.
  - file: [src/agents/pi-embedded-runner/run.ts:974](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/pi-embedded-runner/run.ts#L974)
