## Summary

Keep open. Current `main` already supports plugin/bundle-provided LSP defaults and diagnostics, but I did not find implementation for the issue’s requested zero-config behavior: scanning project files, discovering installed servers on `PATH`, recommending or auto-installing missing servers, or auto-configuring them from project heuristics. The proposal is coherent and still asks for net-new behavior beyond current `main`.
