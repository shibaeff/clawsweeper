## Summary

Keep open. Current `main` does not implement the PR's Discord preview-boundary option or its draft-stream hardening, and the change as proposed is now stale against the current nested `channels.discord.streaming.*` config shape and `extensions/discord/**` layout.

## Evidence

- **Discord preview rotation is still hardcoded to block mode:** `shouldSplitPreviewMessages` is set from `discordStreamMode === "block"`, and both assistant-boundary hooks only call `forceNewMessage()` behind that check. There is no separate opt-in flag.
  - file: [extensions/discord/src/monitor/message-handler.process.ts:620](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/discord/src/monitor/message-handler.process.ts#L620)

- **No resolver exists for a boundary-split option:** The Discord preview resolver only parses stream mode (`off|partial|block`) from `streaming`/`streamMode`; there is no `previewSplitOnAssistantBoundary` or equivalent resolver on `main`.
  - file: [extensions/discord/src/preview-streaming.ts:7](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/discord/src/preview-streaming.ts#L7)
