## Summary

Current `main` already supersedes this PR. Bare model IDs are now resolved through `resolveAllowedModelRefFromAliasIndex`, which infers a unique provider from configured models before validation, and current UI model-picker code sends provider-qualified refs to `sessions.patch`, removing the old failure path this PR targeted.
