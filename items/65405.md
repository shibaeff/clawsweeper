## Summary

Keep open. Current `main` does not implement filtering the dashboard `/model` catalog to `models.providers`, and there is no existing `models.list` toggle/param for a configured-only view. The current behavior only narrows the catalog when `agents.defaults.models` is used as an allowlist.

## Evidence

- **Gateway `models.list` has no configured-only switch:** The handler always loads the full gateway catalog, then applies `buildAllowedModelSet(...)`, and returns that result or the full catalog. There is no branch for `models.providers` filtering or a UI toggle.
  - file: [src/gateway/server-methods/models.ts:12](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/gateway/server-methods/models.ts#L12)
  - command: `nl -ba src/gateway/server-methods/models.ts | sed -n '1,80p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Allowlist logic keys off `agents.defaults.models`, not `models.providers`:** `buildAllowedModelSetWithFallbacks(...)` derives `rawAllowlist` from `cfg.agents?.defaults?.models ?? {}` and sets `allowAny` when that map is empty, which leaves the catalog unfiltered.
  - file: [src/agents/model-selection-shared.ts:510](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/agents/model-selection-shared.ts#L510)
  - command: `nl -ba src/agents/model-selection-shared.ts | sed -n '494,585p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Protocol exposes no filter parameter:** `ModelsListParamsSchema` is an empty object with `additionalProperties: false`, so the RPC surface currently has no `configuredOnly` or similar option for the dashboard to request.
  - file: [src/gateway/protocol/schema/agents-models-skills.ts:181](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/src/gateway/protocol/schema/agents-models-skills.ts#L181)
  - command: `nl -ba src/gateway/protocol/schema/agents-models-skills.ts | sed -n '172,188p'`
  - sha: [1bdf5307d99d](https://github.com/openclaw/openclaw/commit/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19)
- **Dashboard UI just consumes `models.list`:** The Control UI model loader requests `models.list` with `{}` and returns whatever comes back; there is no client-side filtering to configured providers/models.
  - file: [ui/src/ui/controllers/models.ts:10](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/ui/src/ui/controllers/models.ts#L10)
