## Summary

The reported silent-null behavior does not match current main: memory-core now surfaces backend failures as structured disabled/unavailable tool results, and the plugin tool loader logs when a tool factory returns null.

## Close Comment

Closing from a Codex review because the reported behavior does not reproduce on current `main`. In today’s code, `memory-core` only returns `null` from tool creation when memory tools are intentionally unavailable for the current config/session (`extensions/memory-core/src/tools.shared.ts:43`), while actual memory backend failures are surfaced as structured disabled results in `extensions/memory-core/src/tools.ts:207`, `extensions/memory-core/src/tools.ts:315`, and `extensions/memory-core/src/tools.ts:387`. The plugin tool loader also logs when any tool factory returns `null` (`src/plugins/tools.ts:141`), so that path is no longer silent. Current tests cover these failure cases in `extensions/memory-core/src/tools.test.ts:12` and `extensions/memory-core/src/tools.citations.test.ts:128`. If you still have a current repro for a silent failure, please open a new issue with the exact config/session path and logs.
