## Summary

Current main already supports Anthropic prompt caching through configurable `cacheRetention`, and Anthropic transport enables cache-control markers automatically rather than leaving them hardcoded off. The same behavior is present in the shipped `v2026.4.23` release.

## Evidence

- **Anthropic transport always enables cache-control markers:** Current main passes configured `cacheRetention` into `resolveAnthropicPayloadPolicy(...)` and sets `enableCacheControl: true`, so Anthropic cache markers are enabled whenever Anthropic transport is used.
