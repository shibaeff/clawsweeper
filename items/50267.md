## Summary

Current `main` does not show this image-tool attribution fix as implemented. The image media-understanding path still calls `complete()` without transport/request-policy headers, while current attribution logic is intentionally applied later through shared request-policy/transport helpers. The PR is also stale against `main`: it targets a removed path and uses `resolveProviderAttributionHeaders(...)`, which bypasses the newer native-endpoint gating now used elsewhere.
