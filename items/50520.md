## Summary

Keep open. Current `main` still sends outbound payload text through `parseReplyDirectives()` without stripping echoed inbound metadata first, so the reported leak is not implemented on `main` and not obsolete. There is also still a real review question on the proposed approach: `stripInboundMetadata()` removes matching metadata blocks anywhere in the text, while `stripLeadingInboundMetadata()` exists for leading/suffix cleanup only.
