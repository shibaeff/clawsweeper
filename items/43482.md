## Summary

Current `main` already routes fallback retries through persisted session history instead of blindly replaying the original prompt when prior attempt state exists, and this behavior is covered by tests and present in the latest release tag inspected.

## Evidence

- **Fallback retry checks persisted history before rerun:** The fallback runner passes `sessionHasHistory` into each retry attempt, using `!isNewSession || sessionFileHasContent(sessionFile)` so retries on sessions with flushed transcript state are treated as continuation-style retries.
  - file: [src/agents/agent-command.ts:877](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/agent-command.ts#L877)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '874,910p'`
- **Retry prompt switches to continuation mode when history exists:** `resolveFallbackRetryPrompt` keeps the original task but prepends retry context only when `isFallbackRetry` and `sessionHasHistory` are both true, explicitly replacing the older blind 'continue where you left off' behavior.
  - file: [src/agents/command/attempt-execution.helpers.ts:104](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/command/attempt-execution.helpers.ts#L104)
  - command: `nl -ba src/agents/command/attempt-execution.helpers.ts | sed -n '100,125p'`
- **Transcript-history detection is tested against flushed assistant state:** Tests verify the detector stays false for user-only transcripts and flips true once assistant content is present, including large transcript cases, which is the proxy current `main` uses to decide whether fallback has resumable prior state.
  - file: [src/agents/command/attempt-execution.test.ts:107](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/command/attempt-execution.test.ts#L107)
  - command: `nl -ba src/agents/command/attempt-execution.test.ts | sed -n '82,170p'`
- **Fallback retry behavior is explicitly tested:** Tests cover the retry prompt behavior for both `sessionHasHistory: true` and `false`, asserting continuation-style retries only when persisted history exists.
  - file: [src/agents/command/attempt-execution.test.ts:24](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/command/attempt-execution.test.ts#L24)
  - command: `nl -ba src/agents/command/attempt-execution.test.ts | sed -n '20,80p'`
- **Docs describe persisted fallback retry state and tagged release contains the same implementation:** The failover docs state that fallback retry state is persisted before retry start, and the same `agent-command` / `attempt-execution.helpers` logic is present in tag `v2026.4.22`, so this is not only on current `main` but also shipped in the latest inspected release.
  - file: [docs/concepts/model-failover.md:301](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/concepts/model-failover.md#L301)
