## Summary

Keep open. Current `main` still drops image blocks from ACP tool-call results, so this PR targets a real gap rather than an obsolete or already-shipped change. The PR discussion also shows follow-up review on validation, so auto-closing would be premature.

## Evidence

- **Current main still text-only in tool-call content extraction:** `extractToolCallContent()` on current `main` only appends blocks when `entry?.type === "text"`; image blocks are ignored, and only text/message/error fallback is emitted afterward. That means image-only tool results are still dropped on `main`. See `src/acp/event-mapper.ts:344`.
  - file: [src/acp/event-mapper.ts:344](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/acp/event-mapper.ts#L344)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **ACP translator depends on this mapper for outbound tool updates:** ACP tool update/result events pass `partialResult` and `data.result` through `extractToolCallContent()`, so the text-only mapper directly controls what ACP clients receive for tool-call outputs. See `src/acp/translator.ts:892` and `src/acp/translator.ts:909`.
