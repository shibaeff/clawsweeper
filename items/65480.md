## Summary

Keep open. Current `main` still defaults QMD collection creation to `--glob` and only falls back to `--mask` when QMD emits an explicit unsupported-option error. There is no code path or test covering the silent-ignore behavior described in the issue, and the shipped logic on `main` matches the latest release for the affected manager code.

## Evidence

- **Current manager still prefers `--glob` first:** `QmdMemoryManager` initializes `collectionPatternFlag` to `"--glob"`, and `addCollection()` builds candidate flags from that preference before invoking `qmd collection add`.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts:336](https://github.com/openclaw/openclaw/blob/1bdf5307d99d9c1edcbe099d7c2fc3947a9b0a19/extensions/memory-core/src/memory/qmd-manager.ts#L336)

- **Fallback only happens on explicit unsupported-option errors:** `addCollection()` retries with the legacy flag only when `isUnsupportedQmdOptionError(err)` matches known error strings like `unknown flag`, `unknown option`, or `unexpected argument`; a silent accept/ignore case would not trigger fallback.
