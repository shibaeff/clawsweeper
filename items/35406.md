## Summary

Keep open. Current `main` still routes `read` through the upstream `@mariozechner/pi-coding-agent` read tool without any non-image binary-document guard. The pinned upstream implementation detects images only, then decodes every other file as UTF-8 text, so `.docx`, `.pptx`, `.xlsx`, and `.pdf` are still liable to return garbled/token-wasting output. OpenClaw now has a separate `pdf` tool, but that does not change `read` behavior and does not cover the broader office-document case from the issue.

## Evidence

- **OpenClaw wrapper adds paging/image sanitation, not binary-doc rejection:** `createOpenClawReadTool` only adds adaptive paging and image-result sanitation; `createSandboxReadOperations` returns a MIME only when it starts with `image/`, so non-image binaries are not intercepted here.
