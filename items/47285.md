## Summary

Keep open. Current `main` still lacks native Azure request shaping for `memory-lancedb`, so this is not implemented already; but the PR as proposed is also not clearly merge-ready because repo-wide Azure handling now covers more host families and config surfaces than this single-file patch does.

## Evidence

- **Current main still lacks Azure-specific embedding transport:** `memory-lancedb` still builds the embeddings client as `new OpenAI({ apiKey, baseURL: baseUrl })`, and the plugin register path only passes `model`, `dimensions`, `apiKey`, and `baseUrl` into that client. There is no Azure-specific `api-key` header or `api-version` query handling on `main` for this plugin.
  - file: [extensions/memory-lancedb/index.ts:176](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/extensions/memory-lancedb/index.ts#L176)

  - sha: [0bef73d151bd](https://github.com/openclaw/openclaw/commit/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f)

That matches the PR review concern that a hardcoded Azure API version would have no user-configurable override path.
