## Summary

Current `main` already separates isolated-cron session keys, prompt text, workspace resolution, and Claude MCP config generation, so the reported path-construction regression is implemented/fixed on `main` by commit `da36c1967f95effefd08439c51140156c9ad9ab8`.

## Evidence

- **Cron run context resolves workspace independently from session/prompt text:** `prepareCronRunContext` builds `baseSessionKey`/`agentSessionKey` first, then resolves `workspaceDirRaw` via `resolveAgentWorkspaceDir(...)` and `ensureAgentWorkspace(...)`; the prompt text is only used later for `commandBody`.
  - file: [src/cron/isolated-agent/run.ts:459](https://github.com/openclaw/openclaw/blob/da36c1967f95effefd08439c51140156c9ad9ab8/src/cron/isolated-agent/run.ts#L459)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '455,520p'`
  - sha: [da36c1967f95](https://github.com/openclaw/openclaw/commit/da36c1967f95effefd08439c51140156c9ad9ab8)
- **CLI fallback passes stable session key and resolved workspace as separate fields:** The isolated-cron CLI fallback calls `runCliAgent` with `sessionKey: params.agentSessionKey`, `workspaceDir: params.workspaceDir`, and `prompt: promptText` as distinct values, so the prompt is not reused as a filesystem path.
  - file: [src/cron/isolated-agent/run-executor.ts:123](https://github.com/openclaw/openclaw/blob/da36c1967f95effefd08439c51140156c9ad9ab8/src/cron/isolated-agent/run-executor.ts#L123)
  - command: `nl -ba src/cron/isolated-agent/run-executor.ts | sed -n '120,146p'`
  - sha: [da36c1967f95](https://github.com/openclaw/openclaw/commit/da36c1967f95effefd08439c51140156c9ad9ab8)
- **Workspace fallback no longer derives from arbitrary session/prompt content:** `resolveRunWorkspaceDir` returns the provided workspace string when valid, otherwise falls back to `resolveAgentWorkspaceDir(params.config ?? {}, agentId, env)`; it does not concatenate the session key or prompt into a path.
  - file: [src/agents/workspace-run.ts:74](https://github.com/openclaw/openclaw/blob/da36c1967f95effefd08439c51140156c9ad9ab8/src/agents/workspace-run.ts#L74)
  - command: `nl -ba src/agents/workspace-run.ts | sed -n '74,110p'`
  - sha: [da36c1967f95](https://github.com/openclaw/openclaw/commit/da36c1967f95effefd08439c51140156c9ad9ab8)
- **Claude MCP config file is written to a temp directory, not built from session text:** Bundle MCP preparation creates `openclaw-cli-mcp-*` under the OS temp dir, writes `mcp.json` there, and injects that temp file into CLI args via `--mcp-config`.
  - file: [src/agents/cli-runner/bundle-mcp.ts:322](https://github.com/openclaw/openclaw/blob/da36c1967f95effefd08439c51140156c9ad9ab8/src/agents/cli-runner/bundle-mcp.ts#L322)
  - command: `nl -ba src/agents/cli-runner/bundle-mcp.ts | sed -n '322,375p'`
  - sha: [da36c1967f95](https://github.com/openclaw/openclaw/commit/da36c1967f95effefd08439c51140156c9ad9ab8)
- **Regression coverage for per-agent cron workspace/session routing:** The session-identity test asserts cron runs use the agent workspace, agent session key, and agent session store paths, which directly covers the separation that this bug report says regressed.
  - file: [src/cron/isolated-agent.session-identity.test.ts:62](https://github.com/openclaw/openclaw/blob/da36c1967f95effefd08439c51140156c9ad9ab8/src/cron/isolated-agent.session-identity.test.ts#L62)

  - file: [src/agents/workspace-run.test.ts:20](https://github.com/openclaw/openclaw/blob/da36c1967f95effefd08439c51140156c9ad9ab8/src/agents/workspace-run.test.ts#L20)
