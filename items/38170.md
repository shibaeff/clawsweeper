## Summary

Current `main` does not implement this PR’s Telegram `download-file` action, and it still does not embed `file_id` as a placeholder attribute. `main` only carries a partial fallback that appends `[file_id:...]` text when media was not downloaded. The proposed approach also has unresolved compatibility risks against current `main` around inbound-audio placeholder matching and Telegram proxy-aware fetch handling, so this is not a conservative auto-close.
