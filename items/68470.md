## Summary

Keep open. Current `main` still routes prompt-token compaction through `derivePromptTokens`, and `normalizeUsage` still does not recognize a MiniMax-specific `prompt_cache_hit_tokens` alias. The linked fix PRs in the issue timeline are still open, so there is not strong evidence that this bug is already implemented on `main`.

## Evidence

- **Current usage normalization still lacks a MiniMax cache-hit alias:** On `main` (`b84e57fca38deacfa9834a82f7e270b32562ae1b`), `normalizeUsage` recognizes `cache_read_input_tokens`, `cached_tokens`, `input_tokens_details.cached_tokens`, and `prompt_tokens_details.cached_tokens`, but there is no `prompt_cache_hit_tokens` handling in the accepted usage shape or normalization branch.
  - file: [src/agents/usage.ts:18](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/agents/usage.ts#L18)

- **Prompt-token accounting still sums cache read/write into compaction input:** `derivePromptTokens` still computes `input + cacheRead + cacheWrite`.
