## Summary

Keep open. Current `main` does not implement the PR's zero-byte retry, and the Edge TTS implementation now lives in the Microsoft plugin rather than the old `src/tts/tts-core.ts` path from the PR discussion. Blank-input rejection exists at some public entry points, but the provider helper itself still has no empty-text guard or retry, so this is not strong close evidence.

## Evidence

- **Current main still throws on first zero-byte Edge TTS output:** `edgeTTS` in the Microsoft plugin constructs `EdgeTTS`, calls `ttsPromise` once, stats the file once, and throws immediately if size is `0`. There is no retry path on current `main`.
  - file: [extensions/microsoft/tts.ts:32](https://github.com/openclaw/openclaw/blob/0bef73d151bd0a8d547b28bf16c3aa2e21078e0f/extensions/microsoft/tts.ts#L32)
