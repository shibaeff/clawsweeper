## Summary

Keep open. Current `main` at `28fc03c386e14db502429bbe7e062cbe90439ca1` does not implement Deepgram as a bundled speech/TTS provider. The Deepgram plugin is still transcription-only, and the current TTS surfaces and docs do not show Deepgram support.

## Evidence

- **Deepgram plugin registers no speech provider:** The bundled Deepgram plugin only registers media-understanding and realtime-transcription capabilities; there is no `api.registerSpeechProvider(...)` call.
  - file: [extensions/deepgram/index.ts:5](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/deepgram/index.ts#L5)
  - command: `nl -ba extensions/deepgram/index.ts | sed -n '1,80p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Deepgram manifest has no speech provider contract:** `extensions/deepgram/openclaw.plugin.json` declares only `mediaUnderstandingProviders` and `realtimeTranscriptionProviders`; there is no `speechProviders` entry.
  - file: [extensions/deepgram/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/extensions/deepgram/openclaw.plugin.json#L7)
  - command: `nl -ba extensions/deepgram/openclaw.plugin.json | sed -n '1,80p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **TTS command surface is driven by registered speech providers:** `/tts provider` lists providers from `listSpeechProviders(params.cfg)`, and provider resolution comes from the plugin capability registry keyed on `speechProviders`. Without a registered Deepgram speech provider, `/tts provider deepgram` cannot work on current main.
