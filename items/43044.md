## Summary

Keep open. Current main still treats missing `totalTokens` as intentionally unknown, and this PR would change that unknown state into `0`, which conflicts with the existing freshness contract rather than fixing it cleanly. The underlying gap is that gateway/session APIs already expose `totalTokensFresh`, but the TUI types/state do not carry that field.

## Evidence

- **TUI currently preserves unknown totals:** `formatTokens()` on current `main` returns `tokens ?` when both values are missing and uses `?` for a missing total even when context is known, so the PR is changing established unknown-state semantics rather than matching an existing main implementation.

The colocated test explicitly expects `totalTokens` to stay `undefined` and `totalTokensFresh` to be `false` when usage exists but no fresh snapshot is available.
