## Summary

Keep open. Current main still renders `tokens ?/ctx` when `totalTokens` is missing, but this PR's `0/ctx` change would be semantically wrong: OpenClaw deliberately uses missing `totalTokens` to mean stale/unknown context usage, not zero, and the TUI currently drops the `totalTokensFresh` signal that would be needed for a correct fix.

## Evidence

- **Current TUI formatter still treats missing total as unknown:** `formatTokens` returns `tokens ?` when both values are missing and uses `?` for the numerator whenever `total` is null, including the `?/ctx` case this PR changes.
  - file: [src/tui/tui-formatters.ts:373](https://github.com/openclaw/openclaw/blob/41c5ffc5d5f4dc0f1df6f2c9a7efd8ce6a627f00/src/tui/tui-formatters.ts#L373)

- **Main explicitly models missing totals as stale/unknown, not zero:** `persistSessionUsageUpdate` sets `totalTokens` to `undefined` unless there is a fresh context snapshot, and comments state that missing snapshot means context utilization is stale/unknown; `totalTokensFresh` is set from whether a numeric total exists.
