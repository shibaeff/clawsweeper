## Summary

Current `main` does not implement per-row delete/archive actions in the Control UI Sessions view. The Sessions table still exposes delete only through bulk selection, and the backend/API surface exposes `sessions.delete` but no separate archive operation or archived-session listing/filter that would satisfy the requested behavior.

## Evidence

- **Sessions UI still uses bulk-delete bar only:** The Sessions view renders `Delete` only inside the bulk bar gated on `selectedKeys.size > 0`. The table header ends at `Reasoning`; there is no per-row actions column for delete/archive.

  - file: [ui/src/ui/views/sessions.ts:475](https://github.com/openclaw/openclaw/blob/7f6452897e25543c9263b67aabf140379d1c0b0c/ui/src/ui/views/sessions.ts#L475)
