## Summary

Current `main` already contains a post-release fix for this regression: `session_status` now persists per-session model overrides, the explicit-session path reuses those stored overrides on the next turn, and fallback persistence no longer clobbers user-selected overrides.

## Evidence

- **Docs promise immediate persistence:** The user-facing docs still define `/model` as an immediate per-session change and describe pending live-switch behavior when a run is active, which matches the intended behavior in the report.
  - file: [docs/concepts/models.md:144](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/docs/concepts/models.md#L144)

- **session_status now writes the override into the session store:** Current main resolves the requested model, applies it to a copied session entry, writes that entry back with `updateSessionStore`, and sets `changedModel=true`; this is the persistence path the report said was missing.
  - file: [src/agents/tools/session-status-tool.ts:473](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/agents/tools/session-status-tool.ts#L473)

- **Override application clears stale runtime model metadata and marks pending live switch:** `applyModelOverrideToSessionEntry` stores `providerOverride`/`modelOverride`, clears stale `model`/`modelProvider` runtime fields, and marks `liveModelSwitchPending` when requested, preventing the exact stale-row shape described in the issue.
  - file: [src/sessions/model-overrides.ts:24](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/sessions/model-overrides.ts#L24)

- **Next explicit-session turn reads stored overrides:** The explicit-session command path checks the session entry for stored overrides and, when allowed, uses them as the provider/model for the next run instead of reverting to defaults.
  - file: [src/agents/agent-command.ts:702](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/src/agents/agent-command.ts#L702)

Regression tests cover both legacy and explicit user override cases.
