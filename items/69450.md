## Summary

Current main does not appear to load every agent's session store contents for a targeted chat send, but the hot path still performs all-agent session-store discovery before narrowing to the requested agent. That leaves a real O(number of agents) lookup cost in the per-message path, so there is not strong evidence to close this as fixed or non-reproducible.

## Evidence

- **Inbound message path still does session lookup:** `chat.send` immediately calls `loadSessionEntry(rawSessionKey)` for the requested session before processing the message.
