## Summary

Keep open. Current `main` still does not implement the behavior this PR targets: the shared Volc coding catalog is still text-only, and both `volcengine-plan` and `byteplus-plan` still consume that shared catalog at runtime. The PR is also stale against the current file layout, and the exact upstream image-capability matrix now appears mixed rather than uniformly image-capable across all listed models, so this needs maintainer rework rather than cleanup-close. ([docs.byteplus.com](https://docs.byteplus.com/en/docs/ModelArk/2277824))
