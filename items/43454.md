## Summary

Keep open. Current `main` still does not expose `subagent` lifecycle or turn/tool-threshold events through workspace/internal hooks. The issue is recent, specific, and plausible. There is adjacent plugin-hook support, but not the requested workspace-hook surface.

## Evidence

- **Internal hook type union excludes subagent:** `InternalHookEventType` still only allows `command | session | agent | gateway | message`; there is no `subagent` event type in the workspace/internal hook contract.
  - file: [src/hooks/internal-hook-types.ts:1](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/hooks/internal-hook-types.ts#L1)
  - command: `nl -ba src/hooks/internal-hook-types.ts | sed -n '1,220p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Subagent completion only emits plugin hook:** `emitSubagentEndedHookOnce()` calls the global plugin hook runner for `subagent_ended` and persists the marker, but it does not call `createInternalHookEvent()` or `triggerInternalHook()` to bridge that event into workspace/internal hooks.
  - file: [src/agents/subagent-registry-completion.ts:66](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/src/agents/subagent-registry-completion.ts#L66)
  - command: `nl -ba src/agents/subagent-registry-completion.ts | sed -n '1,220p'`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **Workspace hook docs do not list requested events:** The current internal-hook docs list command/session/bootstrap/gateway/message events only. They do not document `subagent:ended`, an agent turn-complete event, or a tool-call-threshold event.
  - file: [docs/automation/hooks.md:34](https://github.com/openclaw/openclaw/blob/28fc03c386e14db502429bbe7e062cbe90439ca1/docs/automation/hooks.md#L34)

  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)
- **No turn-complete or tool-threshold hook implementation found:** A repo search found no `turnComplete`, `toolCallThreshold`, `onTurnComplete`, `onSubagentComplete`, or `onToolCallThreshold` implementation on current `main`. The only nearby match is existing `toolCallCount` usage in subagent-output summarization, not an internal hook surface.
  - command: `rg -n "InternalHookEventType|subagent_ended|toolCallThreshold|turnComplete|triggerInternalHook|createInternalHookEvent|onTurnComplete|onSubagentComplete|onToolCallThreshold" src docs`
  - sha: [28fc03c386e1](https://github.com/openclaw/openclaw/commit/28fc03c386e14db502429bbe7e062cbe90439ca1)

That is relevant context, but it does not satisfy the issue's request to make those lifecycle points available to workspace/internal hooks.
