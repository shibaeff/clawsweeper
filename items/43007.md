## Risks / Open Questions

- The implementation learns topic names opportunistically from Telegram forum service metadata and cached state, not from a live `getForumTopicInfo` call; a topic name may remain unknown until OpenClaw has seen the relevant service metadata or prior cached entry.

## Close Comment

Closing as implemented on `main` after Codex review. Telegram inbound handling now learns forum topic names from Telegram forum service metadata and `reply_to_message.forum_topic_created` in `extensions/telegram/src/bot-message-context.ts:157`, caches them in `extensions/telegram/src/topic-name-cache.ts`, and exposes `TopicName` in inbound session/prompt metadata at `extensions/telegram/src/bot-message-context.session.ts:382`. There is regression coverage in `extensions/telegram/src/bot-message-context.dm-threads.test.ts:179`, and the changelog shipped this in the 2026.4.14 release (`CHANGELOG.md:653`, PR #65973). This uses cached service-message learning rather than a direct `getForumTopicInfo` call, but it does address the requested human-readable inbound topic metadata.
