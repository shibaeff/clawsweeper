## Summary

Vapi support is out of scope for OpenClaw core on current `main`. The repo direction is to keep optional/commercial integrations in plugins, phone calls already live in the installable `voice-call` plugin, and the docs already point to a community Vapi bridge rather than a core channel.

## Evidence

- **Vision keeps optional integrations out of core:** `VISION.md` says optional capability should usually ship as plugins, the bar for adding optional plugins to core is intentionally high, and commercial service integrations or wrapper channels are not in scope for core by default.

## Close Comment

Closing from a ClawSweeper/Codex review. On current `main`, this is a better fit for the plugin/ClawHub ecosystem than for OpenClaw core: `VISION.md:54` says optional capability should usually ship as plugins, `VISION.md:101` explicitly calls out commercial service integrations and wrapper channels as out of scope for core by default, and `docs/plugins/sdk-channel-plugins.md:11` documents new messaging-platform integrations as channel plugins. Phone-call support already exists through the installable `voice-call` plugin (`docs/plugins/voice-call.md:9`, `docs/channels/index.md:33`), and the repo now points users to a community Vapi bridge in `docs/start/showcase.md:282`. If someone wants to build and maintain Vapi support, the intended path is an external plugin/bridge rather than adding a new core channel.
