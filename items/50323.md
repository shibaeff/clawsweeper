## Summary

Issue #50323 appears fixed on `main` and in the shipped release `v2026.4.22`: plugin auto-enable now resolves configured third-party channel ids through the manifest registry and persists the owning plugin id, with a regression test covering the `pluginId != channelId` case that matches the reported `wecom` warning.

## Close Comment

Closing after Codex review. Current `main` already handles this case by resolving configured third-party channel ids through the plugin manifest registry before persisting `plugins.entries`, instead of writing the raw channel id (`src/config/plugin-auto-enable.shared.ts:222`, `src/config/plugin-auto-enable.shared.ts:234`, `src/config/plugin-auto-enable.shared.ts:462`). There is also a regression test for the exact `pluginId != channelId` scenario, asserting that auto-enable writes the manifest plugin id and not the channel key (`src/config/plugin-auto-enable.channels.test.ts:184`). This fix is already present in the shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so the warning reported here is treated as already implemented on current OpenClaw.
