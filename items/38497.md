## Summary

Current `main` does separate user message bodies from command parsing and documents that skills metadata belongs in system-prompt space, but the runtime still supports `before_prompt_build.prependContext` that prepends text ahead of the user prompt. Because the reported `'Skills store policy'` block does not appear in the repo and the issue lacks a concrete current-main reproduction, there is not strong evidence that the bug is fully fixed or obsolete.

That helps command parsing, but it does not prove prepended prompt blocks cannot still distract the model.
