## Summary

Keep open. Current `main` still depends on the external `gog` binary for Gmail hooks, still resolves Gog credentials from per-user config locations, and still documents headless/cron paths as skipping interactive OAuth/keychain repair. I found no strong evidence in code, docs, or changelog that intermittent `gog` Gmail auth failures under macOS cron/non-interactive sessions were fixed or made obsolete on `main`. Conservatively, this is not closeable yet.

## Evidence

- **Gmail integration still shells out to external gog:** `startGmailWatch` and `spawnGogServe` invoke `gog` directly with inherited process context; there is no OpenClaw-side auth resolution or retry logic for Gog credentials in this path.
