## Summary

Keep open. Current `main` still intentionally routes main-session cron system events through a specialized cron prompt that does not instruct the agent to read `HEARTBEAT.md`, tests lock that behavior in, and the current docs/templates do not explain this gap or workaround. The PR remains relevant documentation work rather than something already implemented or obsolete.
