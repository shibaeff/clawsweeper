## Summary

Keep open. Current `main` still has the reported behavior: isolated cron delivery waits on active descendant runs using the full cron timeout budget and suppresses parent delivery while descendants remain active, without gating either path on `delivery.bestEffort`.

## Evidence

- **Active descendants still trigger full follow-up wait on current main:** `dispatchCronDelivery` calls `waitForDescendantSubagentSummary` whenever `activeSubagentRuns > 0 || expectedSubagentFollowup` and passes `timeoutMs: params.timeoutMs`. There is no `params.deliveryBestEffort` check on this branch.
  - file: [src/cron/isolated-agent/delivery-dispatch.ts:709](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/cron/isolated-agent/delivery-dispatch.ts#L709)
  - command: `nl -ba src/cron/isolated-agent/delivery-dispatch.ts | sed -n '684,752p'`
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Wait helper uses the full cron timeout budget:** `waitForDescendantSubagentSummary` derives its deadline from `Date.now() + Math.max(timings.waitMinMs, Math.floor(params.timeoutMs))`, so the delivery path can block until the cron timeout expires.
  - file: [src/cron/isolated-agent/subagent-followup.ts:83](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/cron/isolated-agent/subagent-followup.ts#L83)

- **Active descendants still suppress delivery even for best-effort jobs:** After the wait, if descendants are still active, `dispatchCronDelivery` returns early with `status: "ok"` and `deliveryAttempted = true`, without any `deliveryBestEffort` exception. That matches the issue’s reported suppression path.
