## Close Comment

Codex review: closing this as implemented on current `main`. OpenClaw now already has the core pieces this request asks for: durable Task Flow orchestration with managed/mirrored modes and revision tracking (`docs/automation/taskflow.md:10`), detached background task tracking for ACP/subagents/cron/CLI (`docs/automation/tasks.md:12`), and a plugin runtime seam to create managed flows and child tasks from trusted plugin/tool context (`docs/plugins/sdk-runtime.md:143`, `src/tasks/task-flow-registry.ts:355`, `src/tasks/task-executor.ts:476`). The project direction is also to keep this kind of specialized workflow logic in plugins/skills, with new skills going to ClawHub first rather than adding more core special cases (`VISION.md:54`, `VISION.md:68`, `docs/tools/skills.md:109`). This shipped in `v2026.4.22`; the release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` includes the Task Flow docs and source, and the changelog entries are in `CHANGELOG.md:1530-1532`.
