## Summary

Current main already contains targeted fixes for the reported local gateway CLI handshake/probe failure: active local loopback probes now honor the caller timeout, the shared pre-auth handshake timeout is 10s with a runtime override, and gateway calls yield to the event loop before opening the WebSocket to avoid delayed `connect.challenge` handling.

## Evidence

## Close Comment

Closing this as implemented on main after a Codex review. The current codebase already carries the fixes for the local CLI/gateway timeout path reported here: `src/commands/gateway-status/helpers.ts:132` now lets active local loopback probes honor the caller timeout instead of the old short cap, `src/gateway/handshake-timeouts.ts:1` raises the shared pre-auth handshake timeout to 10 seconds and supports `OPENCLAW_HANDSHAKE_TIMEOUT_MS`, and `src/gateway/call.ts:479` yields before opening the WebSocket so `connect.challenge` is not delayed by CLI event-loop starvation. If you still see similar behavior on a current release, please open a fresh issue with the exact current version plus updated `gateway.bind`, auth mode, and gateway logs, since that would likely be a different regression.
