## Summary

Current main already guards model resolution so missing provider APIs no longer flow through as `undefined`, which is the condition described in the report. The same guardrails are present in the shipped `v2026.4.22` release, and current docs now document the supported GLM/Z.AI path instead of the older `openclaw-cn`/`zhipu` setup from the report.
