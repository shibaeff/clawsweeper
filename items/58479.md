## Summary

Current `main` already implements the approval-consumption path directly in the gateway approval manager: approval requests are registered before the client sees `approval-pending`, `exec.approval.resolve` resolves the waiting decision before any forwarding/UI follow-up work, and resolved approvals remain available briefly for late `waitDecision` callers. The same backend approval files are present in release `v2026.4.23`, so this implementation is already shipped.

## Evidence
