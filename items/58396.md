## Summary

Keep open. Current `main` does not implement `injectSystemNote()` / `injectContext()` or any equivalent prompt-mutation return path on `before_dispatch`; prompt enrichment is still modeled through `before_prompt_build`, so this remains an unresolved feature/design request rather than something already fixed or obsolete.

## Evidence

- **`before_dispatch` contract is claim-only:** `PluginHookBeforeDispatchContext` only carries routing identifiers, and `PluginHookBeforeDispatchResult` only supports `{ handled, text }`; there is no prompt/session mutation field on this hook.
  - file: [src/plugins/hook-types.ts:257](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/plugins/hook-types.ts#L257)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **`before_dispatch` runtime only checks handled reply flow:** The dispatch pipeline runs `before_dispatch` before model dispatch and only branches on `beforeDispatchResult?.handled` plus optional reply `text`; it does not forward any injected context into the agent run.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:682](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/auto-reply/reply/dispatch-from-config.ts#L682)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Prompt mutation lives on `before_prompt_build`:** `runBeforePromptBuild` is documented and wired as the phase that lets plugins inject `prependContext`, `systemPrompt`, `prependSystemContext`, and `appendSystemContext` before prompt submission.
  - file: [src/plugins/hooks.ts:530](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/plugins/hooks.ts#L530)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Docs explicitly prefer prompt mutation in `before_prompt_build`:** The agent-loop docs say `before_prompt_build` is the hook that injects prompt/system context before prompt submission.
  - file: [docs/concepts/agent-loop.md:93](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/docs/concepts/agent-loop.md#L93)
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **No requested API exists on current main:** A repo search for `injectSystemNote` / `injectContext` found no matches in `src/plugin-sdk`, `src/plugins`, or docs, so the requested surface is not already present under another obvious name.
  - command: `rg -n "injectSystemNote|injectContext" src/plugin-sdk src/plugins docs/plugins`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)

## Risks / Open Questions

- There may be unofficial plugin-side workarounds using shared in-memory state across hooks, but there is no documented or typed first-class API on `before_dispatch` that resolves this issue directly.
