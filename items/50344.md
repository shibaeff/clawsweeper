## Summary

Current `main` already fixes this Control UI model-switch bug. The picker now uses provider-qualified option values from the model catalog and forwards the selected value unchanged to `sessions.patch`, so switching from one provider to another no longer reuses the current session provider prefix. The fix was shipped in `v2026.4.2`.

## Evidence

- **Picker options carry the target provider:** `buildChatModelOptionFromLookup()` constructs each Control UI model option as `provider/model` via `buildQualifiedChatModelValue(entry.id, provider)`, so cross-provider picks are fully qualified before selection.
  - file: [ui/src/ui/chat-model-ref.ts:266](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/ui/src/ui/chat-model-ref.ts#L266)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Control UI sends the selected value unchanged:** `switchChatModel()` forwards `nextModel` directly in the `sessions.patch` request. Since the `<select>` value comes from the qualified option list, the patch now sends the target provider prefix rather than the current session provider.
  - file: [ui/src/ui/chat/session-controls.ts:251](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/ui/src/ui/chat/session-controls.ts#L251)
