## Summary

Keep open. Current main has a typed Gateway WebSocket protocol and codegen pipeline, but it does not implement the requested Swagger/OpenAPI surface or ship a checked-in OpenAPI schema for programmatic client generation.

## Evidence

- **Public programmatic API is the Gateway WebSocket protocol:** docs/gateway/protocol.md says the Gateway WS protocol is the single control plane for OpenClaw and that the full gateway API surface is defined by TypeBox schemas, not an OpenAPI document.
  - file: [docs/gateway/protocol.md:12](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/docs/gateway/protocol.md#L12)

  - file: [docs/concepts/typebox.md:12](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/docs/concepts/typebox.md#L12)
