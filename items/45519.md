## Summary

Keep open. Current `main` does not implement collapsible assistant messages in the Control UI chat. Assistant responses still render inline in full, with only an `Open in canvas` action; collapse state exists for tool rows/tool cards, not normal assistant chat bubbles.

## Evidence

- **Assistant messages still render full inline markdown:** `renderGroupedMessage` renders assistant `markdown` directly into `.chat-text` with no length threshold, collapsed state, or show-more toggle. The only adjacent assistant action is `renderExpandButton`, which opens the message in the sidebar/canvas.
  - file: [ui/src/ui/chat/grouped-render.ts:1295](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/chat/grouped-render.ts#L1295)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Expansion state is scoped to tool rows/cards only:** `syncToolCardExpansionState` tracks disclosure ids for tool cards and `toolmsg:*` entries only. There is no parallel state for collapsing regular assistant messages.
  - file: [ui/src/ui/chat/tool-expansion-state.ts:24](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/chat/tool-expansion-state.ts#L24)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Existing tests cover tool-message collapsing, not assistant-message collapsing:** `grouped-render.test.ts` asserts that tool-message content hides/shows based on `isToolMessageExpanded`, reinforcing that the current collapse behavior is limited to tool rows.
  - file: [ui/src/ui/chat/grouped-render.test.ts:546](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/ui/chat/grouped-render.test.ts#L546)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Chat text styles do not apply inline truncation/collapse:** `.chat-text` defines typography and wrapping, and code blocks get horizontal scrolling, but there is no max-height, clamp, or collapsed presentation for long assistant messages.
  - file: [ui/src/styles/chat/text.css:21](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/ui/src/styles/chat/text.css#L21)
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
