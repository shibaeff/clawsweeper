## Summary

Keep open. Current `main` already avoids the original silent-breakage symptom by warning and falling back to agent defaults at cron execution time, but the proposed CLI-side warning is not implemented on `main` and the PR discussion identifies real correctness gaps for gateway-targeted and normalized model resolution. That is not strong evidence for an auto-close.
