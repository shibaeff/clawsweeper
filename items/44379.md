## Summary

Keep open. Current `main` partially overlaps with this PR by already recognizing `context_window_exceeded` stop reasons as context overflow, but it does not implement the PR’s main behavior: a bounded retry that suppresses prompt-build hook context after compaction/truncation fail. The overflow path on `main` still gives up immediately after those existing recoveries, so there is not strong evidence to close this as already implemented or obsolete.
