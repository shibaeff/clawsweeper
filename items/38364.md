## Summary

Current `main` does not appear to implement the requested Control UI reply-metadata path. The browser chat client still sends `chat.send` with only `sessionKey`, `message`, `deliver`, `idempotencyKey`, and `attachments`, and the gateway `ChatSendParamsSchema` still has no structured reply/parent fields. The UI also only parses and renders reply metadata on displayed messages, which looks display-only rather than outbound session plumbing. There is also still a `(no output)` fallback in subagent completion announce construction, so the no-output notice concern is not clearly eliminated across all completion paths.

## Evidence

- **Gateway chat.send schema has no reply metadata fields:** `ChatSendParamsSchema` accepts `sessionKey`, `message`, `thinking`, `deliver`, origin fields, attachments, timeout, provenance, and `idempotencyKey`, but no `replyToId`, parent id, or reply summary field.
