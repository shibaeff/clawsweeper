## Summary

Keep open. Current `main` still does not carry Google Chat thread metadata as a first-class outbound/queue field, and the Google Chat inbound path does not populate `MessageThreadId`, so the follow-up queue cannot reliably restore thread routing. The issue discussion also includes multiple recent repro confirmations that match the current code.

## Evidence

- **Outbound payload contract still lacks thread metadata:** `OutboundReplyPayload` only exposes `text`, `mediaUrls`, `mediaUrl`, `sensitiveMedia`, and `replyToId`. `normalizeOutboundReplyPayload` likewise only reads `replyToId`, so there is still no first-class Google Chat thread field to preserve across streamed/chunked sends.
  - file: [src/plugin-sdk/reply-payload.ts:10](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/plugin-sdk/reply-payload.ts#L10)
  - command: `nl -ba src/plugin-sdk/reply-payload.ts | sed -n '1,80p'`
  - sha: [9577de2da7df](https://github.com/openclaw/openclaw/commit/9577de2da7dfb6234cf30e3b879940067804e26e)
- **Google Chat inbound path does not populate MessageThreadId for queued follow-ups:** The Google Chat monitor records `ReplyToId`/`ReplyToIdFull` from `message.thread?.name` and sets `OriginatingChannel`/`OriginatingTo`, but it does not set `MessageThreadId`. The follow-up runner later sources `originatingThreadId` from `ctx.MessageThreadId`, so queued Google Chat follow-ups still have no thread id to restore on drain.
  - file: [extensions/googlechat/src/monitor.ts:207](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/extensions/googlechat/src/monitor.ts#L207)

- **Block reply coalescing is only replyToId-aware:** `createBlockReplyPayloadKey` keys on trimmed text, media list, and `replyToId` only, and `replyToConflict` in the coalescer only checks `replyToId` mismatches. There is still no separate thread identity in the coalescing path.
  - file: [src/auto-reply/reply/block-reply-pipeline.ts:40](https://github.com/openclaw/openclaw/blob/9577de2da7dfb6234cf30e3b879940067804e26e/src/auto-reply/reply/block-reply-pipeline.ts#L40)
