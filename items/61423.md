## Summary

Current `main` still does not implement this PR's actual behavior. OpenClaw now shows a visible `New session` button in Control UI, but that button only sends `/new`, which resets the current `sessionKey` in place. This PR's proposed flow is materially different: create and select a distinct named session via `sessions.create`, so it is not redundant on current `main`.
