## Summary

Keep open. Current `main` does not implement the PR’s requested behavior: Google thinking payload sanitization still branches on model-ID families (`gemini-2.5-*` budget models vs `gemini-3*` thinkingLevel models) rather than `model.reasoning`, so the proposed `gemini-2.5-flash` negative-budget fix is not present. The PR likely needs a rebase because the logic moved into shared provider-stream wrappers, but the underlying gap remains.
