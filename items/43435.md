## Summary

Current `main` already routes Mattermost media finals through the normal delivery path instead of dropping them during draft-preview finalization, and the shipped `v2026.4.22` changelog explicitly calls out this preview-streaming fix for Mattermost.

## Evidence

- **Shipped release note:** `CHANGELOG.md` under `## 2026.4.22` says: `Channels/preview streaming: centralize draft-preview finalization so Slack, Discord, Mattermost, and Matrix no longer flush temporary preview messages for media/error finals, and preserve first-reply threading for normal fallback delivery.`
  - file: [CHANGELOG.md:266](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/CHANGELOG.md#L266)

  - file: [extensions/mattermost/src/mattermost/monitor.ts:307](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/mattermost/src/mattermost/monitor.ts#L307)

## Close Comment

Closing this as implemented on `main` after Codex review. Current Mattermost preview-finalization code explicitly treats media finals as non-finalizable preview edits and falls back to normal delivery instead of skipping them (`extensions/mattermost/src/mattermost/monitor.ts:307`, `src/channels/draft-preview-finalizer.ts:34`). The regression is also covered by a test that asserts a media final still calls normal delivery (`extensions/mattermost/src/mattermost/monitor.test.ts:300`). This behavior shipped in `v2026.4.22`; the changelog for that release calls out the preview-streaming fix for Mattermost and the other preview-based channels (`CHANGELOG.md:266`, release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
