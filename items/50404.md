## Summary

Keep open. Current main has partial session administration, but it does not implement the requested ChatGPT-style chat-history sidebar or true “new chat creates a separate saved session” flow. The `+` button still routes to `/new`, which resets the current session context rather than creating a distinct session entry, so the core request remains unmet.

## Evidence

- **New-session button still triggers `/new` on the active chat:** The chat toolbar `+` button is wired to `onNewSession`, and the chat app binds that to `state.handleSendChat("/new", { restoreDraft: true })` rather than creating/selecting a new session row.
  - file: [ui/src/ui/app-render.ts:2271](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/ui/src/ui/app-render.ts#L2271)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **`/new` path is a reset flow, not a create-new-session flow:** Slash command dispatch handles `new` by sending `/new` to the current session; the protocol schema for `sessions.reset` only accepts an existing `key` plus optional `reason: "new" | "reset"`, which is reset semantics, not separate session creation.
  - file: [src/gateway/protocol/schema/sessions.ts:175](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/protocol/schema/sessions.ts#L175)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Chat sidebar is for tool/detail content, not session history:** The only chat sidebar rendered in the chat view is `renderMarkdownSidebar(...)` behind `sidebarOpen`, used for tool/output detail panes. There is no session-list sidebar in chat.
  - file: [ui/src/ui/views/chat.ts:1090](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/ui/src/ui/views/chat.ts#L1090)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Session management exists as a separate Sessions page, not a chat sidebar:** Navigation exposes a dedicated `/sessions` tab, and that page provides search, rename via `sessions.patch`, delete via `sessions.delete`, and click-through back to chat. This is partial coverage, but it is materially different from the requested in-chat sidebar workflow.
  - file: [ui/src/ui/navigation.ts:53](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/ui/src/ui/navigation.ts#L53)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **No Control UI usage of `sessions.create`:** A repo search found no `sessions.create` call under `ui/src` or `docs/web/control-ui.md`, which matches the observed behavior that Control UI does not expose a true create-new-chat/session action.
  - command: `rg -n "sessions\.create" ui/src docs/web/control-ui.md`
