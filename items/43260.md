## Summary

Keep open. Current `main` does not implement a `model` field in `SKILL.md` frontmatter or any automatic per-skill model routing path. Skills only expose load-time metadata plus invocation flags, while model overrides today are available through plugin hooks or trusted plugin subagent APIs, not plain skills.

## Evidence

- **Skill frontmatter parser only resolves existing metadata and invocation flags:** `resolveOpenClawMetadata()` reads `always`, `emoji`, `homepage`, `skillKey`, `primaryEnv`, `os`, `requires`, and `install`; `resolveSkillInvocationPolicy()` only reads `user-invocable` and `disable-model-invocation`. There is no `model` or preferred-model field handling.
