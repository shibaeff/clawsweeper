## Summary

Current main already uses a newer macOS control-plane path for menubar health and Channels state: the app resolves the local gateway endpoint via `GatewayEndpointStore` with IPv4 loopback defaults (`127.0.0.1`), drives the General tab/menu status from live health snapshots, and populates the Channels tab from `channels.status`. That implementation shipped in `v2026.4.23`, and the reviewed files are effectively unchanged on current main versus that release.

## Evidence

- **Local macOS gateway endpoint now resolves through explicit IPv4 loopback:** `GatewayEndpointStore` builds the local control URL from a resolved host, and the default/auto/custom fallbacks all use `127.0.0.1` rather than `localhost`, which removes the IPv4/IPv6 ambiguity described in the report.
