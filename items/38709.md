## Summary

Keep open. Current `main` and the latest release still use the older view-local WebChat input-history behavior, so this PR's state-layer hardening is not already implemented; the review thread also documents concrete regressions and a maintainer follow-up, which makes auto-closing inappropriate.

## Evidence

- **Current main still uses the old local-only history helper:** `ui/src/ui/chat/input-history.ts` is still the simple `InputHistory` class with `push/up/down/reset` and a fixed `MAX = 50`. It has no session-aware snapshot state, loading guard, render-window alignment, or caret/draft restoration logic from PR #38709.
