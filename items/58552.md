## Summary

Keep open. Current `main` does not already contain these Apple Silicon changes, and the PR head still has multiple blocking regressions and broken additions, so none of the allowed close reasons are supported.

## Evidence

- **Not implemented on main:** Current `main` has no `src/utils/platform.ts`, `src/utils/thread-pool.ts`, `src/utils/hybrid-runtime*.ts`, or `src/utils/tensor-ops.ts`; the Apple Silicon feature set from this PR is not already present, so `implemented_on_main` does not fit.
  - command: `rg --files src/utils | rg 'platform|thread-pool|hybrid-runtime|tensor-ops'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **`runExec` buffer regression:** On current `main`, `runExec` only forwards `opts.maxBuffer` when explicitly set, preserving Node's larger implicit default. The PR changes that to `opts.maxBuffer ?? OPTIMAL_BUFFER_SIZE`, while the new helper defines only 64 KiB on non-ARM64 and 128 KiB on ARM64, which would lower existing capacity for object-form callers.
  - file: [src/process/exec.ts:141](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/process/exec.ts#L141)
  - command: `nl -ba src/process/exec.ts | sed -n '130,150p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **Unix process-tree kill regression:** Current `main` always checks and SIGKILLs the Unix process group after the grace period. In the PR, `useProcessGroup = isAppleSilicon !== false` gates that path, so non-Apple-Silicon Unix hosts fall back to killing only the parent PID and can leak descendants.
  - file: [src/process/kill-tree.ts:61](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/process/kill-tree.ts#L61)
  - command: `nl -ba src/process/kill-tree.ts | sed -n '46,78p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)
- **User shell preference override:** Current `main` respects `process.env.SHELL` unless it is a known non-interactive shell. The PR inserts an Apple Silicon zsh fast-path before that lookup, so a macOS ARM64 user with `$SHELL=/bin/bash` would be silently switched to zsh.
  - file: [src/agents/shell-utils.ts:67](https://github.com/openclaw/openclaw/blob/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41/src/agents/shell-utils.ts#L67)
  - command: `nl -ba src/agents/shell-utils.ts | sed -n '54,87p'`
  - sha: [11ad1919edc7](https://github.com/openclaw/openclaw/commit/11ad1919edc7278f9d3e36c87c56bd3cb52c6e41)

  - file: [src/utils/platform.test.ts:50](https://github.com/openclaw/openclaw/blob/7193619b09299ccfae6af7c9eeb111f7da7c1ea8/src/utils/platform.test.ts#L50)
