## Summary

Keep open. Current `main` still does not honor `openclaw node run --node-id` as the node identifier used by Gateway node registration and `nodes list/describe`. The CLI and docs still advertise an override, but the node host only forwards that value as `client.instanceId`; Gateway registration still keys the node by `connect.device.id` (device UUID) or `connect.client.id` instead.

## Evidence

- **CLI still promises `--node-id` overrides the node id:** The foreground `openclaw node run` command still exposes `--node-id <id>` with the description `Override node id (clears pairing token)`.
  - file: [src/cli/node-cli/register.ts:47](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/cli/node-cli/register.ts#L47)

- **Public docs still describe `--node-id` as a node id override:** The user-facing CLI docs repeat the same promise for `openclaw node run`, so the current documented behavior still matches the issue report's expectation.
  - file: [docs/cli/node.md:63](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/cli/node.md#L63)

  - file: [src/node-host/runner.ts:183](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/node-host/runner.ts#L183)

  - file: [src/gateway/client.ts:495](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/gateway/client.ts#L495)
