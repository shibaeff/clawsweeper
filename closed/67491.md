## Summary

Current `main` already fetches URL images into buffers before MiniMax image-understanding, then converts those buffers to base64 data URLs for `/v1/coding_plan/vlm`; regression tests cover both steps.

## Evidence

- **Image tool fetches remote URLs before model dispatch:** The current image tool classifies `http(s)` inputs, loads them through `loadWebMedia(...)`, stores the resulting `buffer`/`mimeType`, and passes those loaded images into `runImagePrompt(...)` instead of forwarding the original URL string.
