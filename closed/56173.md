## Summary

The reported `/pair approve` authorization bypass is already fixed on current `main` and in the latest shipped release `v2026.4.22`: the command path now derives internal gateway auth state from `ctx.gatewayClientScopes`, forwards `approvalCallerScopes` into approval, and `approveDevicePairing()` fails closed when operator scopes are requested without caller scopes.

## Evidence
