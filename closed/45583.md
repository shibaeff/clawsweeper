## Summary

Current `main` already handles this case through the unified plugins updater: `openclaw hooks update` is now a deprecated alias that forwards to `openclaw plugins update`, and the shared update path treats `--all` with zero tracked installs as a clean no-op instead of a usage error.

## Evidence

- **Hooks alias now forwards to plugins updater:** `openclaw hooks update` logs a deprecation warning and delegates directly to `runPluginUpdateCommand({ id, opts })`. That means the old hook-specific empty-target branch from this PR no longer exists on `main`.
