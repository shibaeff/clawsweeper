## Summary

Current main already implements the reported behavior: HTTP 401/403 provider failures are classified as auth failover reasons, HTTP 404 provider failures are classified as model_not_found, the embedded runner marks auth-profile failure before failover escalation, and the outer model-fallback loop continues to the next candidate on FailoverError. The same relevant logic is present in release v2026.4.22, with regression tests covering 401 auth fallback and 404 fallback classification.
