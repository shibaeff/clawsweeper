## Summary

Current `main` already implements the reported channel-registry pinning fix and documents it as shipped in release `v2026.4.5`, with regression tests covering startup pinning and later registry churn.

## Evidence

- **Runtime now tracks a pinned channel registry:** `setActivePluginRegistry()` updates a dedicated `state.channel` surface, and `pinActivePluginChannelRegistry()` / `releasePinnedPluginChannelRegistry()` / `requireActivePluginChannelRegistry()` exist for channel lookups.
  - file: [src/plugins/runtime.ts:76](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/plugins/runtime.ts#L76)
  - command: `nl -ba src/plugins/runtime.ts | sed -n '1,260p'`
- **Channel lookup cache now keys on pinned registry identity and version:** Loaded channel plugin resolution reads from `getActivePluginChannelRegistryFromState()` and invalidates when either the registry object or its version changes.
  - file: [src/channels/plugins/registry-loaded.ts:57](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/channels/plugins/registry-loaded.ts#L57)
  - command: `nl -ba src/channels/plugins/registry-loaded.ts | sed -n '1,260p'`
- **Gateway startup pins the startup channel registry:** Gateway runtime initialization calls `pinActivePluginChannelRegistry(params.pluginRegistry)` before serving requests, preventing later plugin-registry churn from evicting channel registrations.
  - file: [src/gateway/server-runtime-state.ts:108](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/server-runtime-state.ts#L108)
  - command: `nl -ba src/gateway/server-runtime-state.ts | sed -n '90,140p'`
- **Regression tests cover the reported failure mode:** Tests assert that a pinned startup registry survives later `setActivePluginRegistry()` calls and that channel helpers keep resolving the original configured channel after a swap.
  - file: [src/plugins/runtime.channel-pin.test.ts:99](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/plugins/runtime.channel-pin.test.ts#L99)
  - command: `nl -ba src/plugins/runtime.channel-pin.test.ts | sed -n '1,240p'`
- **Release notes say this shipped:** The changelog entry under `2026.4.5` says: `Gateway/channels: pin the initial startup channel registry before later plugin-registry churn so configured channels stay visible ... after runtime plugin loads.`
