## Summary

PR #40400 targets an older dedicated durable-memory write/upsert tool path that is no longer the current `main` architecture. Current `main` exposes only `memory_search` and `memory_get`, routes daily durable writes through the append-only memory-flush path, and handles legacy root-memory repair through `openclaw doctor`, so this PR no longer maps cleanly onto shipped behavior.
