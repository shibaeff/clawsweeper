## Summary

Current `main` already hardens the Discord gateway shutdown/reconnect path that this report describes: the Discord gateway plugin now defaults to `reconnect.maxAttempts = 50`, and teardown/dispose paths keep gateway error handling attached so late `Max reconnect attempts ...` errors are suppressed instead of crashing the process.

## Evidence

- **Reconnect attempts are no longer zero:** The Discord gateway plugin now constructs Carbon with `reconnect: { maxAttempts: 50 }`, which directly contradicts the reported `maxReconnectAttempts = 0` crash path.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:378](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/discord/src/monitor/gateway-plugin.ts#L378)
