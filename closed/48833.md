## Summary

Current `main` already covers the reported user need. Instead of hardcoding `os.homedir()` and `~/Desktop` into the default media roots, OpenClaw now expands outbound media local roots to the parent directory of the concrete host-local media path being sent, including `~`-prefixed paths, while keeping the existing fs-policy and sender/group-policy guards.

## Evidence

- **Current media root expansion handles `~` and absolute host-local paths:** `resolveLocalMediaPath()` accepts `file://`, `~`, and absolute host paths, and `appendLocalMediaParentRoots()` adds the parent directory of the requested local media file to the allowed roots. `getAgentScopedMediaLocalRootsForSources()` applies that expansion when fs policy allows it.
  - file: [src/media/local-roots.ts:74](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/media/local-roots.ts#L74)
