## Summary

Current `main` contains a new heartbeat wake/retry path plus subagent announce delivery changes that address the reported 9-11 minute post-announce stall: busy lanes now return `requests-in-flight`, the wake layer retries after 1s instead of waiting for the next interval, repeated skips do not push the next scheduled interval out, and subagent completion delivery now distinguishes active vs dormant requester sessions.

## Evidence

- **Wake layer retries busy heartbeat wakes quickly:** `requestHeartbeatNow` pending wakes are retried after `DEFAULT_RETRY_MS` when the handler returns `requests-in-flight`, instead of waiting for the next interval.
