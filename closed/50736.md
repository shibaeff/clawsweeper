## Summary

Current `main` already addresses the reported failure mode well enough: `sessions_spawn` model overrides are validated through `sessions.patch`, failed child startup is surfaced as `status: "error"` with cleanup instead of a silent accepted no-op, and the subagent registry/announce path has current regression coverage for completion delivery.

## Evidence

- **Model override validation now happens before subagent start:** `sessions.patch` resolves the requested model through `resolveAllowedModelRef(...)` before applying the session override, so aliases/invalid model refs are normalized or rejected up front instead of silently leaking through to a broken child run.
