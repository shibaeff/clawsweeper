## Summary

Current `main` already supports custom providers using the OpenAI Responses transport via `models.providers.<id>.api: "openai-responses"`, and that support was already shipped in `v2026.4.22`.

## Close Comment

Closing as implemented on `main` after a Codex review. OpenClaw already supports custom providers using the OpenAI Responses transport through `models.providers.<id>.api: "openai-responses"`: the config type and schema allow it (`src/config/types.models.ts:9`, `src/config/schema.base.generated.ts:1388`), the runtime routes `openai-responses` through the Responses transport (`src/agents/provider-transport-stream.ts:13`), and the docs ship a custom-provider example using `api: "openai-responses"` (`docs/gateway/configuration-examples.md:344`). I also verified that this support is present in the `v2026.4.22` release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. If the remaining ask is for a separate `openai-codex-responses` label in one specific onboarding surface, that would need a narrower follow-up issue, but the underlying custom-provider `/v1/responses` support requested here is already implemented.
