## Summary

Current `main` already contains targeted fixes for the reported `sessions_send` timeout/routing failure: the Gateway client no longer closes the websocket just because ticks stop while an RPC like `agent.wait` is still pending, and the session/announce path now preserves delivery routing metadata for follow-up sends.

## Evidence

- **Gateway client no longer kills pending waits on missed ticks:** The tick watchdog now returns early while any RPC is still pending, so a long-running `agent.wait` is allowed to use its own timeout instead of forcing a websocket close.

- **Regression test for pending-request watchdog behavior:** The watchdog test explicitly asserts that missing ticks do not close the socket while a request is pending, and only trigger `tick timeout` after the pending entry is cleared.
  - file: [src/gateway/client.watchdog.test.ts:148](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/gateway/client.watchdog.test.ts#L148)
