## Summary

Current main does not support the issue's stated root cause. Discord DMs require explicit pairing first, guild channels require explicit allowlisting and are mention-gated by default, inbound Discord messages are processed by the inbound worker rather than cron jobs, and current tests cover the mentioned guild-message path. This makes the report obsolete/mismatched against current main rather than an actionable maintainer bug on today's code.

## Evidence

- **Discord setup docs require DM pairing before Discord chat works:** The current Discord setup guide says to DM the bot, receive a pairing code, approve it, and only then chat via Discord DM.
  - file: [docs/channels/discord.md:150](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/channels/discord.md#L150)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Guild replies are a separate explicit setup step:** The docs describe guild/server use as a separate 'Recommended: Set up a guild workspace' step, requiring `groupPolicy: "allowlist"`, guild configuration, and optional `requireMention: false` if always-on replies are desired.
  - file: [docs/channels/discord.md:181](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/channels/discord.md#L181)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Pairing is DM-only and does not authorize groups:** The pairing docs state unknown DM senders are not processed until approved, and explicitly note that DM pairing does not grant group authorization.
  - file: [docs/channels/pairing.md:20](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/channels/pairing.md#L20)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Guild traffic defaults to allowlist mode:** Discord config tests assert that `groupPolicy` defaults to `allowlist`, so a bare onboarding/token setup is not expected to reply in arbitrary guild channels.
  - file: [extensions/discord/src/config-schema.test.ts:67](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/discord/src/config-schema.test.ts#L67)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Current main intentionally skips unmentioned guild messages when mention gating applies:** The preflight handler logs `discord: skipping guild message` with reason `no-mention` and returns early when `requireMention` is active and the message does not satisfy mention detection.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:995](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/discord/src/monitor/message-handler.preflight.ts#L995)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Explicitly mentioned guild messages are covered by a regression test:** There is a test that a guild message with an explicit bot mention is allowed even when `requireMention: true` and a configured binding route is present.
  - file: [extensions/discord/src/monitor/message-handler.preflight.test.ts:1163](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/discord/src/monitor/message-handler.preflight.test.ts#L1163)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Discord sessions are created automatically on inbound processing, not via cron jobs:** The message handler queues inbound work through `createDiscordInboundWorker`, and `processDiscordMessage` persists the session with `recordInboundSession`, contradicting the issue's assumption that a cron job or pre-spawned Discord handler session is required.
  - file: [extensions/discord/src/monitor/message-handler.ts:79](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/discord/src/monitor/message-handler.ts#L79)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
