## Summary

Keep open. Current `main` still defaults heartbeat-prompt injection on every embedded run except `cron`, while channel and CLI entrypoints pass `trigger: "user"`. That matches the PR’s reported bug and there is no evidence in the checked-out `main` that the fix or its regression test has already landed.

## Evidence

- **Current main still defaults heartbeat prompt on non-cron triggers:** `DEFAULT_EMBEDDED_RUN_TRIGGER_POLICY.injectHeartbeatPrompt` is `true`, and the only explicit override is `cron: { injectHeartbeatPrompt: false }`, so `user`, `heartbeat`, `manual`, `memory`, `overflow`, and `undefined` all fall through to injection.
