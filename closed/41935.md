## Close Comment

Closing this as better suited for [ClawHub](https://clawhub.ai/)/community plugin work after Codex review.

This PR is a useful device-specific integration, but current OpenClaw policy routes new optional tool/plugin capabilities to external plugins distributed through ClawHub or npm rather than bundling them into the main repo.

What I checked:

- **Vision steers optional capability out of core:** [`VISION.md`](https://github.com/openclaw/openclaw/blob/main/VISION.md) says optional capability should usually ship as plugins, the preferred path is npm/local plugin distribution, authors should host and maintain plugins in their own repo, and the bar for adding optional plugins to core is intentionally high. ([`VISION.md:54`](https://github.com/openclaw/openclaw/blob/7ac35b4f6900/VISION.md#L54), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))
- **Plugin docs explicitly say not to add new plugins to this repo:** [`docs/plugins/building-plugins.md`](https://docs.openclaw.ai/plugins/building-plugins) says new channels/providers/tools/capabilities do not need to be added to the OpenClaw repository and should be published to ClawHub or npm for `openclaw plugins install`. ([`docs/plugins/building-plugins.md:11`](https://docs.openclaw.ai/plugins/building-plugins), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))
- **Community plugin policy makes ClawHub the canonical path:** [`docs/plugins/community.md`](https://docs.openclaw.ai/plugins/community) describes third-party plugins for new tools/capabilities, says ClawHub is the canonical discovery surface, and tells authors to publish there instead of opening repo/docs PRs for discoverability. ([`docs/plugins/community.md:9`](https://docs.openclaw.ai/plugins/community), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))
- **Current main supports external plugin install flows:** [`docs/cli/plugins.md`](https://docs.openclaw.ai/cli/plugins) documents `openclaw plugins install <package>` with ClawHub-first resolution, which is the intended ship path for optional integrations like this one. ([`docs/cli/plugins.md:54`](https://docs.openclaw.ai/cli/plugins), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))
- **Feature is not already a bundled main capability:** Current `main` has no `extensions/insta360` files and no `./plugin-sdk/insta360` export in `package.json`; this remains an unmerged proposal to add a new bundled plugin surface. ([`package.json:54`](https://github.com/openclaw/openclaw/blob/7ac35b4f6900/package.json#L54), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))

So I’m closing this as a scope-fit item for the plugin/community path rather than keeping it open as an OpenClaw core request.
