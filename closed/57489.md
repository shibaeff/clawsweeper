## Summary

Current `main` no longer applies live-session reconciliation on every retry boundary for plain channel overrides. The runner now gates live switches behind `liveModelSwitchPending`, which is documented and tested as an explicit user-driven `/model` or `sessions.patch` signal rather than a generic fallback/default-model reconciliation path.

## Evidence

- **Live switch now gated by explicit pending flag:** `shouldSwitchToLiveModel` returns early unless the session entry has `liveModelSwitchPending`; without that flag, no live-session model switch is requested, so a `channels.modelByChannel` start model is not compared against the agent default on each retry.

- **Pending flag is only for explicit user-driven model changes:** The session type comment says `liveModelSwitchPending` is set on explicit user-driven session model changes and that system-initiated fallbacks never set it, matching the reported regression's missing distinction.
