## Summary

Current `main` and the shipped `v2026.4.23` release already flatten chat-channel `outbound.attachedResults` into a top-level `ChannelOutboundAdapter` before `deliverOutboundPayloads` runs, so the issue's reported Telegram root cause does not match the current implementation.

## Evidence

  - file: [extensions/telegram/src/channel.ts:585](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/telegram/src/channel.ts#L585)
