## Summary

Current main already treats Feishu DM traffic as direct-message context before shared chat-type normalization, so this PR's core `p2p -> direct` change is redundant on current OpenClaw. The added payload regression also does not hit a live Feishu path on current main because `buildEmbeddedRunPayloads` does not accept `messageProvider` or `chatType` inputs.

## Evidence

- **Feishu ingress already converts non-group chats to direct:** Current main computes `isGroup` only from `ctx.chatType === "group"`, then writes inbound context as `ChatType: isGroup ? "group" : "direct"`. That means Feishu `p2p` and `private` events already reach core as `direct`.
