## Summary

Current `main` already uses account-scoped Feishu configuration and top-level routing bindings; the reported `channels.feishu.bots[].bind` / channel-local binding shape does not match the current schema or docs, and current routing code matches Feishu bindings by active `accountId`.

## Evidence

- **Current Feishu schema is strict and uses `defaultAccount` + `accounts`:** `FeishuConfigSchema` defines multi-account Feishu config with `defaultAccount` and `accounts`, then marks the object `.strict()`. There is no `bots` array or channel-local `bindings` field in the current schema.
  - file: [extensions/feishu/src/config-schema.ts:208](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/feishu/src/config-schema.ts#L208)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Current user docs show multi-account Feishu config:** The current Feishu docs document multi-bot/account setup under `channels.feishu.accounts` and `defaultAccount`, not `channels.feishu.bots`.
  - file: [docs/channels/feishu.md:204](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/channels/feishu.md#L204)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Current user docs route Feishu traffic via top-level `bindings[]`:** The Feishu docs' multi-agent routing section uses top-level `bindings[]` with `match.channel: "feishu"` and peer matching, which is the supported routing surface on current `main`.
  - file: [docs/channels/feishu.md:329](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/docs/channels/feishu.md#L329)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Feishu inbound routing passes the active account into binding resolution:** `handleFeishuMessage` resolves the route with `account.accountId` and passes that same `accountId` into configured-binding resolution for the current Feishu conversation.
  - file: [extensions/feishu/src/bot.ts:590](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/feishu/src/bot.ts#L590)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Configured binding matching prefers exact `match.accountId`:** The configured-binding matcher normalizes the conversation account ID and gives exact `match.accountId` matches higher priority than wildcard/default matches.
  - file: [src/channels/plugins/configured-binding-match.ts:17](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/channels/plugins/configured-binding-match.ts#L17)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Top-level binding source of truth:** Configured routing bindings are read from `cfg.bindings`, confirming that current routing rules live at the top level rather than under `channels.feishu`.
  - file: [src/config/bindings.ts:18](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/config/bindings.ts#L18)
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
