## Summary

Issue #51330 is already implemented on current `main`. OpenClaw now cleans stale session `.jsonl.lock` files during gateway startup, exposes a built-in operator repair path via `openclaw doctor --fix`, and has targeted stale-lock cleanup coverage in the session lock tests.
