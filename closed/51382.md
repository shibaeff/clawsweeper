## Summary

Current main does not support the reported cron-count mismatch path. The Control UI’s cron jobs count is driven from the current `cron.list` page result, and the gateway resolves the explicit `enabled` filter before `includeDisabled`, so the default `enabled: "all"` request counts and displays the same job set.

## Evidence

- **Default cron filter is "all":** The Control UI initializes `cronJobsEnabledFilter` to `"all"`, so the first cron jobs load asks for all jobs, not enabled-only jobs.
  - file: [ui/src/ui/app.ts:423](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/ui/src/ui/app.ts#L423)
  - command: `nl -ba ui/src/ui/app.ts | sed -n '414,428p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **UI total comes from current cron.list page metadata:** `loadCronJobsPage()` requests `cron.list`, replaces `state.cronJobs` with the returned page, normalizes `res.total`, and sets `cronJobsTotal` from that page metadata. With a `jobs.length` of 3 and `total` of 3, this path cannot display 4.
  - file: [ui/src/ui/controllers/cron.ts:295](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/ui/src/ui/controllers/cron.ts#L295)
  - command: `nl -ba ui/src/ui/controllers/cron.ts | sed -n '280,325p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Gateway honors explicit enabled filter first:** `resolveEnabledFilter()` returns `opts.enabled` when it is `"all"`, `"enabled"`, or `"disabled"`; it only falls back to `includeDisabled` when `enabled` is absent.
  - file: [src/cron/service/ops.ts:169](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cron/service/ops.ts#L169)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '150,220p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Server total matches filtered job set:** `listPage()` filters the source jobs by the resolved enabled filter, then sets `total` to `sorted.length`. For `enabled: "all"`, the total is the all-jobs count for the same list being returned.
  - file: [src/cron/service/ops.ts:208](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/cron/service/ops.ts#L208)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '208,245p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Rendered list uses the same returned jobs and shows disabled rows explicitly:** The cron jobs view renders enabled/disabled state from each returned job row, so the visible list is not silently dropping disabled jobs after the default `"all"` load.
  - file: [ui/src/ui/views/cron.ts:1495](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/ui/src/ui/views/cron.ts#L1495)
  - command: `nl -ba ui/src/ui/views/cron.ts | sed -n '1488,1510p'`
