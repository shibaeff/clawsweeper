## Summary

Keep open. Current `main` still hardcodes insecure Control UI link schemes (`http://`/`ws://`) in the shared dashboard link builder and its callers, while adjacent gateway probe/connection code already switches to TLS-aware `wss://` URLs when `gateway.tls.enabled=true`. The PR is addressing a live inconsistency on `main`, not something already implemented or obsolete.
