## Summary

Current `main` already delivers the underlying browser voice workflow through Control UI Talk mode, with microphone access enabled on the normal gateway HTTP surface and chat/session binding handled through the existing authenticated session flow. This PR’s separate `/voice-connect` UI and websocket bridge do not appear on `main`; the feature was effectively superseded by a different implementation.
