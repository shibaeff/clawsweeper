## Summary

Current main does not show a core bug here: default per-user location belongs in injected `USER.md`, and custom weather-skill instructions belong in workspace `skills/weather/SKILL.md`, which already overrides the bundled skill. The bundled weather skill still uses `London` only in example commands, which explains the reported confusion.

## Risks / Open Questions

- The bundled weather skill still uses `London` in examples, so users may keep interpreting that sample text as a real default location even though the supported configuration path is `USER.md` or a workspace skill override.

## Close Comment

Codex review: I checked current `main`, and this does not look like an OpenClaw core bug.

`USER.md` is the supported place for stable per-user profile/location context, and OpenClaw injects it into the session prompt (`docs/concepts/agent.md:34-36`). If you want to customize the weather skill itself, workspace `skills/weather/SKILL.md` already has higher precedence than the bundled copy (`docs/concepts/agent.md:59-65`, `src/agents/skills/workspace.ts:534-597`).

The bundled weather skill still shows `London` only in example `wttr.in/...` commands (`skills/weather/SKILL.md:49-121`), so editing that example text is not how default location is configured.

Based on that, I’m closing this as not reproducible on current `main`. If you can still reproduce it on `v2026.4.22` or newer, please open a new issue with the exact file you edited plus the prompt/session transcript showing the wrong location.
