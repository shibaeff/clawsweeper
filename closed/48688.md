## Summary

Current main already injects local-time context for cron/heartbeat runs by resolving the configured user timezone with host-timezone fallback and appending `Current time: <local> (<tz>) / <UTC>` to the prompt path used for scheduled healthcheck/security-audit alerts.

## Evidence

- **Timezone resolution uses configured user timezone, then host timezone fallback:** `resolveUserTimezone` returns `agents.defaults.userTimezone` when valid, otherwise `Intl.DateTimeFormat().resolvedOptions().timeZone`, and only falls back to `UTC` if neither is available.

  - file: [src/agents/current-time.ts:23](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/current-time.ts#L23)
