## Summary

Current `main` already provides programmatic session termination/cleanup through the Gateway API: `sessions.abort` stops active work and `sessions.delete` removes the session, archives transcripts, and emits lifecycle cleanup. The exact CLI spelling requested in the issue does not appear to exist, but the requested API capability is implemented on `main`.
