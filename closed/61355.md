## Summary

Current `main` now keeps `api.registerService()` registrations in the plugin registry and explicitly calls `await service.start(...)` during gateway sidecar startup. A follow-up change on April 24, 2026 also made gateway startup wait for sidecars by default, so the reported "service.start() not invoked after plugin registration" gap is implemented on `main` but has not shipped in a tagged release yet.

## Evidence

- **Plugin service registrations are retained:** `registerService` stores plugin-owned services in `registry.services`, which is the source consumed at gateway startup.
  - file: [src/plugins/registry.ts:1092](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/plugins/registry.ts#L1092)
