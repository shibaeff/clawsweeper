## Summary

Current `main` already supports the requested WhatsApp group observe-only behavior through existing primitives: WhatsApp inbound now carries sender metadata into `message_received`, and `session.sendPolicy: "deny"` suppresses every outbound reply path while still letting inbound processing and hooks run. This shipped in `v2026.4.23` as the changelog entry for observer-style setups.

## Risks / Open Questions

- The exact proposed `observe` or `neverReply` config key was not added; the equivalent behavior is expressed through existing `requireMention`/activation plus `session.sendPolicy` settings.
- Scoping is session-policy based rather than a dedicated WhatsApp-group-only UX, so users still need to target the relevant WhatsApp group session key or channel/chatType rule.
