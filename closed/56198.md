## Summary

Current main already suppresses the duplicate ACP final text fallback once block delivery is considered visible, and Telegram is explicitly marked as a channel where ACP block replies count as visible user delivery. The repo also now has regression tests covering the routed and direct Telegram cases that assert no duplicate final reply is sent.

## Evidence

- **Final fallback is gated on visible delivery state:** `finalizeAcpTurnOutput` only sends a final text fallback when no visible text was delivered or visible delivery failed, which prevents a second final text after a visible block reply.
