## Summary

Current `main` already contains the Control UI fix that defers same-session history reloads until the active run ends, preserving optimistic user messages, and the changelog/appcast show it shipped in `v2026.4.15`.

## Evidence

- **Current Control UI code defers the reload that caused the optimistic-message race:** `handleSessionMessageGatewayEvent` skips `loadChatHistory` while `host.chatRunId` is active and stores a deferred reload instead. The inline comment explicitly says reloading mid-run races with the optimistic user-message update and delays the user message card from appearing.
