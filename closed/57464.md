## Summary

Current main already solves the reported startup problem by scoping gateway boot to the resolved startup plugin set, honoring `plugins.allow`, and skipping imports outside that scope before module load.

## Evidence

- **Gateway startup now honors allowlist/configured channels:** `canStartConfiguredChannelPlugin` rejects plugins outside `plugins.allow` unless a bundled channel is explicitly configured, and `resolveGatewayStartupPluginIds` builds the startup set from configured channels plus manifest metadata.
