## Summary

Current `main` does not contain the reported system-`crontab` / `.pending_fetch` trigger mechanism. OpenClaw scheduling is Gateway-owned cron persisted under `~/.openclaw/cron/jobs.json`, heartbeat is a periodic prompt over optional `HEARTBEAT.md`, and the issue body's `taobao-daily-data` paths point to an external workspace skill outside this repo.

## Evidence

- **Gateway cron is the supported scheduler:** Docs for current main state that cron runs inside the Gateway process, persists jobs at `~/.openclaw/cron/jobs.json`, and isolated cron teardown best-effort closes tracked browser tabs/processes.
