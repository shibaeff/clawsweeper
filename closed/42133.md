## Summary

Close as `clawhub`: current OpenClaw scope guidance prefers optional integrations like MQTT ingress to ship as external plugins on ClawHub/npm, not as new bundled core plugins. Main already provides a bundled `webhooks` plugin for trusted external automation, so this transport-specific integration is better maintained out of tree, with any truly missing generic seams proposed separately.

## Close Comment

Closing this Codex review as `clawhub`. Current OpenClaw direction is to keep optional capabilities out of core when they can ship as plugins: `VISION.md:54`-`61` says optional capability should usually ship as plugins maintained in their own repository, `docs/plugins/building-plugins.md:16`-`19` says new plugins do not need to be added to this repo, and `docs/plugins/community.md:9`-`22` points authors to ClawHub/npm as the normal distribution path. Main also already ships the bundled `webhooks` plugin for trusted external automation (`docs/plugins/webhooks.md:11`-`16`, shipped in `CHANGELOG.md:1061`). Given that, an MQTT ingress integration fits better as an external ClawHub/npm plugin than as a new bundled OpenClaw core plugin. If there is a missing generic SDK or ingress seam needed to support that external plugin cleanly, please open a smaller focused PR just for that seam.
