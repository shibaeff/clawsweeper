## Summary

Current `main` already wires slow-model request timeouts through configurable OpenClaw timeout settings instead of leaving them stuck behind an unconfigurable 15s idle watchdog. The fix is documented, covered by regression tests, and recorded in the changelog under the `2026.4.9` release.

## Evidence

- **Current timeout resolution inherits agent timeout:** `resolveLlmIdleTimeoutMs` now falls back to `agents.defaults.timeoutSeconds` when no explicit `agents.defaults.llm.idleTimeoutSeconds` is set, which addresses the reported case where a slow local provider should honor configured timeout values instead of timing out at 15s.
