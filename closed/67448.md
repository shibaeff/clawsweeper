## Summary

Current `main` does not support the PR's core bug claim: `WizardPrompter.text()` is typed to return `Promise<string>`, and both shipped prompter implementations coerce nullish input to a string or throw cancellation instead of returning `undefined`. The `apply_patch` warning part is also a behavior change against current tested policy, not a reproduced regression. The PR additionally has an unresolved optional-field-clearing regression already called out in review.
