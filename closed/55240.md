## Summary

Current `main` already handles Discord slash commands by deferring the interaction immediately, routing `/status` through a direct follow-up reply path, and covering the expired-interaction case with tests. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
