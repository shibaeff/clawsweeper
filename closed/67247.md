## Summary

Close as implemented: current `main` and the shipped `v2026.4.23` tag already cap and trim Telegram native-command menus to fit Telegram’s limits, then retry `BOT_COMMANDS_TOO_MUCH` with a smaller command set instead of leaving the menu empty. That matches the payload-budget failure mode described in the issue and in the reporter’s follow-up workaround.

## Evidence

- **Shipped menu builder enforces Telegram text budget:** The shipped `v2026.4.23` Telegram menu builder uses a conservative 5700-character total-text budget and trims descriptions before dropping commands, directly addressing the report's 'too many commands / too much text' failure mode.
