## Summary

Current `main` already implements the requested behavior for `openclaw plugins install` include-backed config writes through the newer `replaceConfigFile()` mutation path, with docs and regression tests, so PR #41816 is redundant.

## Evidence

- **Plugin install persistence uses include-aware replace path:** `persistPluginInstall()` and `persistHookPackInstall()` now persist through `replaceConfigFile(...)` instead of a direct `writeConfigFile(...)` call, so install writes flow through the centralized config mutation logic.
  - file: [src/cli/plugins-install-persist.ts:29](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/cli/plugins-install-persist.ts#L29)

- **Current writer preserves single-file top-level includes:** `replaceConfigFile()` detects when exactly one top-level section changed, resolves a single-file `$include` target for that section, writes the included file directly, and only falls back to the root writer when the write is not isolated.
  - file: [src/config/mutate.ts:56](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/mutate.ts#L56)

## Close Comment

Closing this as implemented after Codex review.

Current `main` already implements the requested behavior through a broader config-mutation path: plugin install/update writes now go through `replaceConfigFile`, which writes isolated single-section mutations directly into single-file `$include` targets instead of flattening `openclaw.json`. The checked-in regression test, docs, and changelog all confirm the fix, so this PR is obsolete on `main`.
