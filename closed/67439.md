## Summary

Current `main` already contains bounded Gateway WebSocket teardown and cron read-path responsiveness changes that match the reported 15-20s slow-exit behavior, and the same implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **CLI now waits for bounded Gateway client teardown:** `stopGatewayClient` awaits `client.stopAndWait({ timeoutMs: 1_000 })` and only falls back to `client.stop()` on failure, replacing an unbounded success-path teardown.
  - file: [src/gateway/call.ts:95](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/call.ts#L95)
