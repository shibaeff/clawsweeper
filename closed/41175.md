## Summary

Current `main` already addresses this Discord attachment wedge path: attachment fetches now have a 120s total timeout, failed downloads fall back instead of blocking the turn, remote fallback URLs are not re-injected as native images, and the prompt image loader clamps images to the 6MB image budget with image optimization. The same implementation is already present in release `v2026.4.22`.

## Evidence

- **Discord attachment downloads have a bounded total timeout:** `extensions/discord/src/monitor/timeouts.ts:5-6` defines `DISCORD_ATTACHMENT_IDLE_TIMEOUT_MS = 60_000` and `DISCORD_ATTACHMENT_TOTAL_TIMEOUT_MS = 120_000`. `extensions/discord/src/monitor/message-handler.process.ts:177-185` passes those limits into inbound media resolution for Discord messages.

- **Timed-out attachment fetches fall back instead of wedging the inbound worker:** `extensions/discord/src/monitor/message-utils.ts:323-360` races each attachment fetch against `totalTimeoutMs` and aborts on timeout. `extensions/discord/src/monitor/message-utils.ts:383-415` catches attachment fetch failures and pushes a fallback attachment entry instead of failing the whole inbound message path.
  - file: [extensions/discord/src/monitor/message-utils.ts:323](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/extensions/discord/src/monitor/message-utils.ts#L323)

That preserves the text turn instead of silently dropping it.
