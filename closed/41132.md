## Summary

Current main already declares GLM text-vs-image capabilities and strips unsupported image payloads during prompt assembly and transport replay, so the reported "one incompatible image poisons the session until /reset" path no longer matches the shipped code path for text-only models like `zai/glm-5`.

## Evidence

- **GLM capability declaration exists:** The bundled Z.AI catalog marks `glm-5.1`, `glm-5`, and `glm-5-turbo` as `input: ["text"]`, while the vision variant `glm-5v-turbo` is `input: ["text", "image"]`.
