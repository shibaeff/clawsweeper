## Summary

Current `main` already handles missing `qa/scenarios/index.md` in npm distributions and keeps completion-cache generation alive when the QA subcli cannot register. The changelog records this QA/packaging fix as shipped in `v2026.4.11`.

## Evidence

- **QA scenario pack is now optional in npm distributions:** `readQaScenarioPack()` returns an empty fallback pack when `qa/scenarios/index.md` is absent, with an inline note that this prevents completion-cache and other packaged CLI consumers from crashing when the QA scenario directory is not shipped.
  - file: [extensions/qa-lab/src/scenario-catalog.ts:331](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/qa-lab/src/scenario-catalog.ts#L331)
  - command: `nl -ba extensions/qa-lab/src/scenario-catalog.ts | sed -n '331,346p'`
  - sha: [86f8c826e20a](https://github.com/openclaw/openclaw/commit/86f8c826e20a01d1bba67c0ba6e73182fbf0455f)
- **Completion cache generation now skips failing subcommand registration:** `registerSubcommandsForCompletion()` wraps `registerSubCliByName()` in `try/catch` and logs a skip warning instead of aborting the whole `completion --write-state` flow when a subcommand fails to register.
  - file: [src/cli/completion-cli.ts:55](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/completion-cli.ts#L55)

  - file: [src/cli/completion-cli.write-state.test.ts:82](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/cli/completion-cli.write-state.test.ts#L82)
