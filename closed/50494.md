## Summary

Current `main` already implements the requested behavior through shared media-understanding plumbing rather than a Telegram-specific `inbound.autoAnalyzeImages` flag. Telegram preserves inbound image file paths for the agent context, image auto-resolution now prefers `agents.defaults.imageModel`, native-vision reply models receive the original image directly, and prompt plumbing keeps the real attachment alongside any generated image description.
