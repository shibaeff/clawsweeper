## Summary

Current `main` already routes gateway-host `exec` through a login-shell PATH merge, which addresses the reported Telegram-vs-local PATH drift for CLI-backed commands like `clawhub`.

## Evidence

- **Gateway exec now merges login-shell PATH:** `createExecTool` applies `getShellPathFromLoginShell(...)` and `applyShellPath(...)` for non-sandbox `host === "gateway"` runs when no explicit `env.PATH` override is provided, so host exec no longer depends only on the daemon's minimal launchd PATH.
