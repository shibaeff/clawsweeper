## Summary

Current `main` already has dedicated read-only CLI fast paths for `status`, `health`, `channels list/status`, and `gateway status`, with plugin preload disabled for those routes. Gateway `health`/`status` now serve summaries from read-only channel metadata instead of broad plugin activation, and `cron list` is covered by the same `loadPlugins: "never"` startup policy. The proving implementation is on `main` in commit `30aa7e0d4d225650b58c2fba79e71bffa5627b08`, but it is newer than the latest tagged release `v2026.4.23`, so no shipped release tag could be confirmed yet.

## Risks / Open Questions

- The fix is on current `main`, but the latest tagged release `v2026.4.23` predates it, so users on released builds may still see the older behavior until a newer release ships.
- This closes the reported core fast-path bug; a third-party plugin can still make other discovery/full-activation paths slow if its top-level `register(api)` or discovery code does heavyweight work.
