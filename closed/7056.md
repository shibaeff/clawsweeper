## Summary

Current OpenClaw already supports the requested containment model in core: tool execution can be forced into sandboxes for every session, using Docker locally or SSH/OpenShell for VM-style remote runtimes. The latest release also ships and documents this, so the feature request is already implemented well enough to close.

## Risks / Open Questions

- The gateway process itself still stays on the host by design; the implemented containment applies to tool execution, not to moving the entire control plane into a container or VM.
- Strict container/VM-only operation is opt-in configuration (`agents.defaults.sandbox.mode: "all"`) and can still be bypassed if an operator explicitly enables elevated host escape paths.
