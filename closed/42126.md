## Summary

Close as `clawhub`. The proposed bundled `openclaw-workflow` plugin is a heavy orchestration layer that duplicates current main’s supported orchestration stack (`Task Flow` + `Lobster`) and conflicts with the current project scope: optional capability should usually ship as an external plugin, maintained outside the core repo.

## Evidence

- **Vision: optional capability stays out of core by default:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, the preferred path is npm/local extension loading, plugins should be hosted in their own repository, and the bar for adding optional plugins to core is intentionally high.

## Risks / Open Questions

- Task Flow + Lobster are not a drop-in replacement for this PR's exact YAML schema or output-gate UX; this close decision is a scope/ownership call, not a claim of byte-for-byte feature equivalence.
- If maintainers explicitly want a new bundled first-party orchestration plugin despite the current VISION guardrails, this decision would need to be revisited.
