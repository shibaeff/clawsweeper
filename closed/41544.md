## Summary

Keep open. Current `main` has partial overlap: it supports streamed usage for specific built-in local provider IDs and for explicit `compat.supportsUsageInStreaming: true`, but it still intentionally leaves generic custom/local OpenAI-compatible endpoints disabled by default. That means this PR's address-based auto-detection for arbitrary local inference endpoints is not implemented on `main`.

## Close Comment

Closing this as implemented after Codex review.

Current `main` already preserves and records streaming usage for OpenAI-compatible streaming calls, so this PR's fix is obsolete on the present codepath.

What I checked:

- **Streaming requests always ask for usage now:** `buildOpenAICompletionsParams()` unconditionally sets `stream_options: { include_usage: true }` for streamed OpenAI-completions requests, rather than depending on a compat flag. ([`src/agents/openai-transport-stream.ts:1686`](https://github.com/openclaw/openclaw/blob/a53fea39050b/src/agents/openai-transport-stream.ts#L1686), [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050b))
- **Streaming usage is always parsed into output:** The stream reader assigns `chunk.usage` or per-choice `usage` directly onto `output.usage`, and `parseTransportChunkUsage()` converts prompt/completion tokens into persisted totals. ([`src/agents/openai-transport-stream.ts:1266`](https://github.com/openclaw/openclaw/blob/a53fea39050b/src/agents/openai-transport-stream.ts#L1266), [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050b))
- **Usage parsing implementation is present on main:** `parseTransportChunkUsage()` computes input/output/cache/total tokens from streamed usage chunks on current `main`. ([`src/agents/openai-transport-stream.ts:1741`](https://github.com/openclaw/openclaw/blob/a53fea39050b/src/agents/openai-transport-stream.ts#L1741), [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050b))
- **Tests cover the new runtime behavior:** Transport tests assert `include_usage` for explicit custom providers and for known local backends like `llama-cpp`, which is the behavior this PR aimed to recover. ([`src/agents/openai-transport-stream.test.ts:1404`](https://github.com/openclaw/openclaw/blob/a53fea39050b/src/agents/openai-transport-stream.test.ts#L1404), [a53fea39050b](https://github.com/openclaw/openclaw/commit/a53fea39050b))

So I’m closing this as already implemented rather than keeping a duplicate issue open.
