## Summary

Current main keeps hook-triggered agent turns on the resolved session key, persists their transcript path back into the session store, and reopens that transcript on follow-up turns. The reported "bootstrap-context:full discards hook turns" path does not match the current code or docs.

## Evidence

- **Hook dispatch uses provided session key:** Gateway hook agent requests are dispatched into the isolated agent runner with the resolved session key; there is no separate bootstrap-only handoff that drops the conversation state.
