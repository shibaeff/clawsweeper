## Summary

Current `main` already guards the reported failure mode: replay/compaction re-runs tool-use/tool-result pairing repair after history truncation, the transcript repair layer inserts missing tool results or drops orphaned ones, fallback classification treats `tool_use`/invalid-request signatures as format errors rather than overload, and each new turn still starts from the configured default model unless the user explicitly stored a model override.

## Evidence
