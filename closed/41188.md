## Summary

Current `main` already covers this request with the existing verbose-level contract plus the chat tool-card visibility toggle. `agents.defaults.verboseDefault` supports `off | on | full`, where docs define `off` as hiding tool chatter by default, `on` as summary-style tool notices, and `full` as forwarding tool outputs. The Control UI also has a persisted toggle that hides tool calls/results from chat history and live tool messages.

## Evidence

- **Global config already exposes the requested behavior class:** Agent defaults include `verboseDefault?: "off" | "on" | "full"`, which is the config-level switch for hiding, summarizing, or fully showing tool output.
  - file: [src/config/types.agent-defaults.ts:299](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/config/types.agent-defaults.ts#L299)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '296,304p'`
- **Docs map the three modes to hide/summary/full behavior:** The docs say `/verbose` levels are `on` (minimal), `full`, and `off` (default). They further state that `on` emits metadata-only tool summaries and `full` forwards tool outputs after completion.
  - file: [docs/tools/thinking.md:73](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/docs/tools/thinking.md#L73)
