## Summary

The reported `/v1`-append mismatch does not match current `main`: the image path and the text path both pass the configured provider `baseUrl` through to the OpenAI client, and the pinned OpenAI SDK joins overridden `baseURL` values with `/chat/completions` rather than inserting an extra `/v1`.

## Evidence

- **Image path uses resolved model directly for completion:** `describeImagesWithModelInternal` registers provider stream hooks, then calls `complete(model, context, ...)` with the resolved model object; there is no OpenClaw-side `/v1` rewrite in this path.
  - file: [src/media-understanding/image.ts:302](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/media-understanding/image.ts#L302)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Text path also uses configured baseURL directly:** `createOpenAICompletionsClient` builds `clientConfig.baseURL` from `model.baseUrl` and then calls `client.chat.completions.create(...)`, so the text loop is using the same overridden-baseURL OpenAI client style rather than a separate hard-coded `/v1` builder.
  - file: [src/agents/openai-transport-stream.ts:1043](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/agents/openai-transport-stream.ts#L1043)
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Pinned pi-ai package forwards model.baseUrl unchanged:** The pinned dependency `@mariozechner/pi-ai@0.70.2` creates its OpenAI client with `new OpenAI({ baseURL: model.baseUrl, ... })` in `dist/providers/openai-completions.js`; it does not append `/v1` before handing control to the OpenAI SDK.
  - command: `curl -L --silent https://registry.npmjs.org/@mariozechner/pi-ai/-/pi-ai-0.70.2.tgz | tar -xzOf - package/dist/providers/openai-completions.js | sed -n '300,360p'`
- **Pinned OpenAI SDK joins overridden baseURL with /chat/completions:** The pinned `openai@6.34.0` client defaults to `https://api.openai.com/v1` only when `baseURL` is not overridden; when it is overridden, `buildURL()` uses that overridden base URL as-is, and `chat.completions.create()` posts to `'/chat/completions'`. For `https://api.z.ai/api/coding/paas/v4`, that resolves to `.../v4/chat/completions`, not `.../v4/v1/chat/completions`.
  - command: `curl -L --silent https://registry.npmjs.org/openai/-/openai-6.34.0.tgz | tar -xzOf - package/src/client.ts package/src/resources/chat/completions/completions.ts`
