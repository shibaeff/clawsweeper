## Summary

Current main already ships the supported plugin seam for this use case: `before_agent_reply` runs after inline actions, can claim the turn, and can silence it without entering the default agent run. That capability is documented, tested, and present in released `v2026.4.23`, so this PR's separate `before_agent_run` hook is redundant.
