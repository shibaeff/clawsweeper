## Summary

Current main already separates Feishu read-only inspection from runtime SecretRef resolution, so unresolved `channels.feishu.appSecret` no longer throws in doctor/status/plugin-read paths; the same implementation is present in the latest tagged release `v2026.4.22`.

## Evidence

- **Feishu read-only inspect mode no longer throws:** `resolveFeishuSecretLike` returns `undefined` in inspect mode for unresolved SecretRefs and only throws in strict mode; `resolveFeishuAccount` uses inspect mode while `resolveFeishuRuntimeAccount` keeps strict runtime behavior.
  - file: [extensions/feishu/src/accounts.ts:70](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/feishu/src/accounts.ts#L70)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Read-only channel surfaces use inspect resolver; gateway start uses strict runtime resolver:** Feishu security/message-tool/status paths call `resolveFeishuAccount` or `inspectFeishuCredentials`, while gateway startup calls `resolveFeishuRuntimeAccount(..., { requireEventSecrets: true })`.
  - file: [extensions/feishu/src/channel.ts:164](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/feishu/src/channel.ts#L164)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Regression tests cover the reported behavior split:** Tests assert that unresolved Feishu `appSecret` SecretRefs are treated as unconfigured in read-only account resolution, but still throw the typed unresolved-SecretRef error in runtime resolution.
  - file: [extensions/feishu/src/accounts.test.ts:366](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/feishu/src/accounts.test.ts#L366)

- **Changelog documents the exact read-only SecretRef fix pattern:** The changelog notes that SecretRef inspect-vs-strict handling was aligned so unresolved refs no longer crash read-only CLI flows while runtime-required refs stay strict.
