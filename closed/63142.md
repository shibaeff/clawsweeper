## Summary

Current `main` does not ship a `mac-calendar` core skill, and OpenClaw’s current scope and docs route calendar-specific integrations through ClawHub/workspace skills rather than the core repo.

## Evidence

- **Project scope keeps new optional skills out of core:** The vision doc says plugin discovery/promotion belongs in ClawHub, that new skills should be published through ClawHub first, and that new core skills should not be merged when they can live on ClawHub.
  - file: [VISION.md:72](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/VISION.md#L72)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Skills are workspace/local artifacts, not necessarily bundled core code:** The skills docs show OpenClaw loads bundled, managed, personal, project, and workspace skills, and that `openclaw skills install <slug>` installs ClawHub skills into `<workspace>/skills`. That makes skill-specific runtime bugs belong to the skill package unless a core API seam is implicated.
  - file: [docs/tools/skills.md:13](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/docs/tools/skills.md#L13)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)

  - file: [docs/tools/clawhub.md:10](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/docs/tools/clawhub.md#L10)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
