## Summary

Current `main` already covers the reported Telegram DM stale-session failure mode: inbound Telegram DMs recreate missing direct-session entries, `/new` rotates to a fresh `sessionId` on the same session key, and the per-channel DM path retires stale main-session delivery routing. The repo also now carries direct regression tests for deleted Telegram DM session recreation and `/new` rollover behavior.

## Evidence
