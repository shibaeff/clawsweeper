## Summary

Current `main` already gives the TUI non-idle activity states during agent work. The send path sets `sending` then `waiting`, lifecycle events set `running`, chat deltas set `streaming` even for commentary/tool-only deltas, and the status renderer treats those values as busy states with a spinner and elapsed time instead of `connected | idle`. The docs and tests also reflect that behavior.
