## Summary

Current `main` already contains a targeted fix for session-switch desynchronization in the web UI: session switches clear visible chat state immediately, stale `chat.history` responses are version/session guarded, and stale avatar metadata is similarly suppressed. The relevant regression tests are present on `main` in commit `9ece33c50506ba031df382f69bde65dfda19bea5`.

## Evidence

- **Session switch resets visible chat state immediately:** `resetChatStateForSessionSwitch` updates `sessionKey`, clears messages/draft/stream state, and persists the new active session before reloading history.
  - file: [ui/src/ui/app-render.helpers.ts:68](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/ui/src/ui/app-render.helpers.ts#L68)
