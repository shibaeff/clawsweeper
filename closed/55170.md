## Summary

Current `main` already has the image-tool model-resolution behavior this issue asked for: explicit `agents.defaults.imageModel` is honored as-is, provider-local vision models can be inferred from configured provider catalogs without forced `anthropic/` rewriting, and there is direct test coverage for custom providers, Moonshot, and MiniMax Portal image execution paths.
