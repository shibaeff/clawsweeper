## Summary

Current `main` already fixes the npm-distribution crash through a safer centralized fallback in `extensions/qa-lab/src/scenario-catalog.ts`, so this PR's `try/catch` in `discovery-eval.ts` is obsolete.

## Evidence

- **Central fallback already on main:** `readQaScenarioPack()` now treats a missing `qa/scenarios/index.md` as an optional npm-distribution case and returns an empty pack instead of throwing, which directly prevents the module-init crash described in the PR.
  - file: [extensions/qa-lab/src/scenario-catalog.ts:331](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/qa-lab/src/scenario-catalog.ts#L331)

[defaults]`; with the central empty-pack fallback, `readQaScenarioExecutionConfig()` returns `undefined` instead of throwing, so the built-in refs already win without a blanket `catch` in `discovery-eval.ts`.
