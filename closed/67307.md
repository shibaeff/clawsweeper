## Summary

This request is best handled as a ClawHub plugin/skill, not OpenClaw core. Current main is explicitly plugin-first for optional capabilities, the plugin SDK already exposes the needed hook/lifecycle seams, and there is already a bundled ownership-coordination precedent (`thread-ownership`) implemented outside core.

## Evidence

## Risks / Open Questions

- Maintainers may still want a first-party repo/issue claim API later if multiple built-in autonomous workflows start competing for the same SCM targets.
- Current plugin seams are sufficient for this workflow, but there is not yet a dedicated built-in repo+issue+branch claim contract.
