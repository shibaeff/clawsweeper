## Summary

Current `main` already has a supported silent-reply mechanism for tool-driven turns: exact `NO_REPLY` / `no_reply` is treated as an intentional no-visible-reply token, WebChat omits those rows, streaming suppresses leaked `NO...` lead fragments, and plugin hooks exist to force or rewrite the post-tool assistant message instead of relying on free-form `TOOLS.md` / `SOUL.md` instructions.

## Risks / Open Questions

- Free-form prompt instructions in `TOOLS.md` / `SOUL.md` are not the supported contract by themselves; a plugin that wants deterministic silence still needs to emit exact `NO_REPLY` or use the documented hook-based rewrite/suppression path.
