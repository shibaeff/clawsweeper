## Summary

Current `main` already treats vLLM as a proxy-style OpenAI-compatible backend, suppresses unsupported `strict`/`store` request shaping for that path, and also contains a Nemotron/vLLM stream fix for `finish_reason: "tool_calls"` with an empty tool list, which matches the blank-output symptom described in the issue.

## Evidence

- **vLLM path is classified as non-native/proxy-like and strict mode is disabled by default:** `resolveOpenAICompletionsCompatDefaults` turns off `supportsStrictMode` for configured non-native OpenAI-compatible endpoints; local vLLM URLs resolve as configured non-native endpoints, so strict shaping is not used automatically.

  - file: [src/agents/openai-transport-stream.ts:1584](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/openai-transport-stream.ts#L1584)
