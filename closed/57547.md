## Summary

Current `main` already implements WebChat image rendering for assistant/media replies, including the read-tool image case via tool-result media extraction and UI image block rendering. The implementation is present on `main` but does not appear to be part of the latest tagged release `v2026.4.23` yet.

## Evidence

- **Read-tool image fallback now extracts a usable media path:** `src/agents/pi-embedded-subscribe.tools.ts:240` documents the fallback strategy for tool-result media, and `src/agents/pi-embedded-subscribe.tools.ts:332` falls back to `details.path` when image content exists but no `MEDIA:` token is present. This matches the reported `Read image file [image/jpeg]` scenario.
