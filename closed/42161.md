## Summary

Current `main` already avoids the reported manual cron deadlock through a narrower lane-routing fix: manual cron runs still enqueue on `cron`, but nested embedded-agent work is remapped to `nested` instead of re-entering the active cron lane. That behavior is covered by current tests and is already present in release `v2026.4.22`, so this PR is obsolete on `main`.
