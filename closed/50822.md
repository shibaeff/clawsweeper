## Summary

Keep open. Current `main` still sends a separate Feishu streaming-card close for each distinct `final` payload, and the regression tests explicitly lock that behavior in, so this PR's stated fix is not already implemented on `main`.

## Evidence

- **Current main still closes on every distinct final:** `deliver()` only skips exact duplicate final text via `deliveredFinalTexts.has(text)`, then for any streamed card `final` it merges the text and immediately `await closeStreaming()`. That still produces one closed card per distinct final payload instead of deferring intermediate finals until idle.
