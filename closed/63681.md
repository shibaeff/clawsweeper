## Summary

Current `main` already resolves memory embedding providers from plugin capability contracts when the process-global registry is cold, and it includes an explicit regression test for resolving `ollama` in that state. The same implementation is present in release `v2026.4.23`, so this issue can be closed as already fixed/shipped.

## Evidence

- **Cold-registry memory provider fallback on main:** `listMemoryEmbeddingProviders` and `getMemoryEmbeddingProvider` no longer rely only on globally registered adapters; they merge/fallback to manifest-declared capability providers via `resolvePluginCapabilityProviders` / `resolvePluginCapabilityProvider`, which addresses the CLI cold-registry failure mode.
