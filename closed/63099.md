## Summary

Current main already resets the underlying conversation state on `/reset` instead of reusing stale CLI/session bindings, and it now has regression coverage plus incomplete-turn fallback handling. The reported behavior was on OpenClaw 2026.4.2; the shipped changelog entry indicates the reset/session-binding fix was included in v2026.4.5.
