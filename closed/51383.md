## Summary

Current `main` already provides WebChat session-based parallel contexts, session switching, and session browsing/search/navigation, which covers the core request well enough. The original premise that WebChat only has a single continuous stream and no way to work in parallel no longer matches shipped behavior.

- The existing implementation is session-based rather than a dedicated tabs/sidebar 'project/thread' UX, so users wanting cross-session full-text transcript search or per-session SOUL overrides may still prefer a narrower follow-up issue.
- I did not find evidence of a dedicated cross-session transcript search feature; the close is based on the core parallel-context/session-management request already being implemented.
