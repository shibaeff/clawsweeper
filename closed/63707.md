## Summary

Current `main` no longer uses in-place writes for `exec-approvals.json`; it writes to a temp file and renames atomically, and the repo now includes a regression test covering atomic replacement. The same implementation is present in release `v2026.4.23`, so the reported truncate/read race is already fixed in shipped code.

## Evidence

- **Atomic write on current main:** `saveExecApprovals()` now delegates to `writeExecApprovalsRaw()`, which writes JSON to a unique temp file with `flag: "wx"` and then `renameSync()`s it into place. That avoids the truncate-before-write window described in the issue.
