## Summary

Issue #50686 requests a new Zoom Team Chat core/bundled channel integration, but current OpenClaw direction is to keep optional commercial integrations out of core and ship them as external plugins. Current `main` has channel-plugin APIs and docs for exactly this path, and no Zoom implementation is present.

## Evidence

- **Core scope guardrail:** Project direction says optional capability should usually ship as plugins, be hosted in their own repo, and that the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:55](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/VISION.md#L55)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Commercial integration guardrail:** The current vision explicitly lists commercial service integrations that do not clearly fit the model-provider category under 'What We Will Not Merge (For Now)'. Zoom Team Chat fits that bucket more than core runtime work.
  - file: [VISION.md:103](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/VISION.md#L103)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **External plugin path is documented:** The plugin guide says new plugins do not need to be added to the OpenClaw repository and should be published to ClawHub or npm instead.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/plugins/building-plugins.md#L16)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Channel-plugin API exists for new messaging platforms:** The channel-plugin guide explicitly covers building a messaging channel plugin that connects OpenClaw to a messaging platform, with core owning shared dispatch and the plugin owning channel-specific behavior.
  - file: [docs/plugins/sdk-channel-plugins.md:11](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/plugins/sdk-channel-plugins.md#L11)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Community channel plugins are already the expected distribution path:** Community plugins docs describe third-party channel plugins published on ClawHub or npm, including messaging integrations like DingTalk, QQbot, and WeCom.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/plugins/community.md#L9)
