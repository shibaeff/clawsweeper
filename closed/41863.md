## Summary

Current `main` already resolves the reported overflow path more completely than this PR: image-containing tool results are now pruned by converting images to text placeholders during soft-trim and fully hard-clearing them when needed, with regression tests covering both behaviors.

## Close Comment

Closing this as implemented after Codex review.

Current `main` already prunes image-bearing tool results during context pruning, with explicit test coverage for soft-trim, image-only replacement, and hard-clear behavior. The open PR’s narrower fix is superseded by the implementation now in `src/agents/pi-hooks/context-pruning/` on `main`.

What I checked:

- **Image tool results are now pruned on main:** `softTrimToolResultMessage()` detects image blocks, replaces them with `[image removed during context pruning]`, and returns a pruned text payload instead of skipping those tool results. ([`src/agents/pi-hooks/context-pruning/pruner.ts:225`](https://github.com/openclaw/openclaw/blob/7ac35b4f6900/src/agents/pi-hooks/context-pruning/pruner.ts#L225), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))
- **No image-result exemption remains in pruning loop:** `pruneContextMessages()` adds every prunable `toolResult` to `prunableToolIndexes` and applies trimming/clearing without any `hasImageBlocks(...) => continue` guard. ([`src/agents/pi-hooks/context-pruning/pruner.ts:309`](https://github.com/openclaw/openclaw/blob/7ac35b4f6900/src/agents/pi-hooks/context-pruning/pruner.ts#L309), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))

([`src/agents/pi-hooks/context-pruning/pruner.test.ts:224`](https://github.com/openclaw/openclaw/blob/7ac35b4f6900/src/agents/pi-hooks/context-pruning/pruner.test.ts#L224), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))
