## Summary

Current `main` already implements this fix via `extraSystemPromptStatic`, which keeps volatile inbound metadata out of the CLI session reuse hash while still injecting the full prompt at runtime. The PR's core behavior is present, the DM empty-static edge case from review is handled, and the changelog records the fix separately as `#70122`, so this PR is redundant on `main`.

## Evidence
