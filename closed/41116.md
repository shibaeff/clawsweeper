## Summary

Current main already handles Discord DM ACP thread-bound spawns by resolving the DM to the current conversation and binding it with `placement: "current"` instead of trying child-thread creation. The shipped `v2026.4.22` tag contains the same implementation and regression coverage.

## Evidence

- **Discord DM conversation identity resolves to user target:** `resolveDiscordCurrentConversationIdentity(...)` returns the sender `user:<id>` for `chatType === "direct"`, which gives DM ACP commands a current-conversation identity instead of a channel/thread target.
