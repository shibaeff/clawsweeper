## Summary

Current OpenClaw already chunks long Telegram outbound text before calling the Bot API, including the normal Telegram reply-delivery path. The same chunking logic is present in the shipped `v2026.4.23` release, with tests and docs covering the 4000-character Telegram text cap.

## Evidence

- **Current direct Telegram send path chunks oversized text:** `sendMessageTelegram` builds a chunk plan with `splitTelegramHtmlChunks(rawText, 4000)` / plain-text fallback chunks and then sends each chunk sequentially with `sendTelegramTextChunks(...)`.
  - file: [extensions/telegram/src/send.ts:751](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/telegram/src/send.ts#L751)
