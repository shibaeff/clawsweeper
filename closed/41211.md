## Summary

Current main already updates the generated workspace `AGENTS.md` startup section to tell agents to use runtime-provided startup context first and avoid rereading `AGENTS.md`/`SOUL.md`/`USER.md` unless the user asks, context is missing, or a deeper follow-up read is needed. The runtime seeds new workspaces from that template, so the requested cleanup is implemented on main.
