## Summary

Current `main` already avoids treating zero-valued poll defaults as poll-creation intent, and the same fix is present in the shipped `v2026.4.22` release.

## Evidence

- **Current guard ignores zero poll durations:** `hasPollCreationParams()` only treats numeric poll fields as intent when the parsed value is finite and non-zero, so `pollDurationHours: 0` / `pollDurationSeconds: 0` no longer trigger the send-path poll rejection.
