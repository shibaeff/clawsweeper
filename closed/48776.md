## Summary

Current `main` has a core queue/session-rotation fix that updates the active reply operation and any queued followups when compaction rotates a session ID, plus regression tests covering the previously stuck queued-followup path. That matches the reported cross-channel symptom of fresh long messages getting queued and then never draining.

## Evidence
