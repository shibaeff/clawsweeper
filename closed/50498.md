## Summary

Current `main` already implements the requested behavior by scoping the runtime model catalog to the configured `agents.defaults.models` allowlist, which is what `/model` and the Control UI consume.

## Evidence

- **Gateway filters model list through allowlist:** `models.list` loads the gateway catalog, runs it through `buildAllowedModelSet(...)`, and returns the filtered `allowedCatalog` instead of the full built-in catalog.
  - file: [src/gateway/server-methods/models.ts:28](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/server-methods/models.ts#L28)

- **Allowlist is built from configured model entries:** `buildAllowedModelSetWithFallbacks(...)` reads `cfg.agents?.defaults?.models`, collects allowed keys, and filters the catalog down to those configured refs, adding synthetic rows for configured models missing from the built-in catalog.
  - file: [src/agents/model-selection-shared.ts:468](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/model-selection-shared.ts#L468)
