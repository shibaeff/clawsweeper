## Summary

Current `main` already contains Telegram duplicate-delivery protections that cover both likely causes in this report: duplicate inbound update handling and duplicate outbound finalization of DM preview messages. The shipped `v2026.4.22` release tag also contains these protections, so the `2026.3.13` report is already implemented on newer OpenClaw.

## Evidence

- **Inbound Telegram updates are deduped and safe watermarks are persisted:** `createTelegramBotCore` skips repeated Telegram updates by `update_id`/derived update key, tracks pending/failed update ids, and only persists a safe watermark after completion so restarts do not replay already-handled updates.
