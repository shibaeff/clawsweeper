## Close Comment

Closing this as implemented after Codex review.

Current `main` already implements the underlying Codex OAuth proxy fix through a broader env-proxy bootstrap path, so this PR is superseded.

What I checked:

- **Live Codex OAuth path bootstraps proxy dispatcher before login:** `loginOpenAICodexOAuth()` now calls `ensureGlobalUndiciEnvProxyDispatcher()` immediately before running the OpenAI Codex OAuth flow, then invokes `loginOpenAICodex(...)` on that path. ([`src/plugins/provider-openai-codex-oauth.ts:130`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/src/plugins/provider-openai-codex-oauth.ts#L130), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
- **Dispatcher helper implements the proxy-aware global bootstrap:** `ensureGlobalUndiciEnvProxyDispatcher()` checks HTTPS proxy env, preserves existing env-proxy dispatchers, avoids overriding unsupported custom dispatchers, and installs `EnvHttpProxyAgent` when needed. ([`src/infra/net/undici-global-dispatcher.ts:91`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/src/infra/net/undici-global-dispatcher.ts#L91), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
- **Regression coverage exists for bootstrap, idempotency, and custom dispatcher preservation:** Current tests cover installing the env proxy dispatcher, not overriding unsupported custom proxy dispatchers, retrying after dispatcher changes, and idempotent reapplication. ([`src/infra/net/undici-global-dispatcher.test.ts:207`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/src/infra/net/undici-global-dispatcher.test.ts#L207), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
- **Public provider-auth runtime points at the new OAuth implementation:** The exported provider-auth runtime now resolves `loginOpenAICodexOAuth` from [`src/plugins/provider-openai-codex-oauth.ts`](https://github.com/openclaw/openclaw/blob/7ac35b4f6900/src/plugins/provider-openai-codex-oauth.ts), so the proxy-aware path is the live one on `main`. ([`src/plugin-sdk/provider-auth-login.runtime.ts:2`](https://github.com/openclaw/openclaw/blob/7ac35b4f6900/src/plugin-sdk/provider-auth-login.runtime.ts#L2), [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f6900))
- **CLI startup also applies the same proxy bootstrap globally:** `run-main` calls `ensureGlobalUndiciEnvProxyDispatcher()` during CLI startup, which is broader coverage than the PR’s temporary wrapper around one OAuth call. ([`src/cli/run-main.ts:187`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/src/cli/run-main.ts#L187), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
