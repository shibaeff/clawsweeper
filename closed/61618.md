## Summary

Current `main` already addresses the PR's underlying Windows dual-module-instance failure by sharing plugin runtime state across duplicate module graphs, while intentionally keeping Windows native Jiti loading disabled. That makes this PR's specific loader change obsolete and contrary to the current tested direction.

## Evidence
