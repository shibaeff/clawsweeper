## Summary

Current `main` already fixes the reported bad Web UI LCM context percentages by using fresh `totalTokens`, suppressing stale totals, and covering the regression with targeted UI tests; the same code is present in release `v2026.4.22`.

## Risks / Open Questions

## Close Comment

Closing from a Codex review: current `main` already fixes the reported bad Web UI LCM context percentages. In `ui/src/ui/chat/context-notice.ts:53-65`, the UI now skips stale `totalTokensFresh === false` data, computes the ratio from `session.totalTokens`, and clamps the displayed percentage to `100`, so it no longer falls back to cumulative transcript usage that could show `0%` or `>100%`. The regression is covered in `ui/src/ui/chat/context-notice.test.ts:32-58` and `ui/src/ui/chat/context-notice.test.ts:86-98`, including the large-input/smaller-total case from this report and the stale-token case. This implementation is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
