## Summary

Current `main` already avoids the reported false `disabled` outcome for successful main-session cron heartbeats. The main-session cron path enqueues the system event, calls `runHeartbeatOnce(..., heartbeat: { target: "last" })`, and maps a completed heartbeat (`status: "ran"`) to cron `status: "ok"`. In current `runHeartbeatOnce`, `disabled` is checked only at the start of invocation; successful heartbeat paths return `status: "ran"`, not `disabled`.

## Evidence
