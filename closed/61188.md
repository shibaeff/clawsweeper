## Summary

Current `main` already removed the heuristic host-exec obfuscation gate described in the issue, so the requested config knob is no longer needed.

## Evidence

- **Approval predicate now uses explicit policy only:** `requiresExecApproval(...)` only considers `ask`, `security`, allowlist analysis, and durable approvals. There is no obfuscation or command-length input in the approval predicate.
  - file: [src/infra/exec-approvals.ts:772](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/infra/exec-approvals.ts#L772)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Node host wiring has no generic obfuscation gate:** Node-host exec computes `requiresAsk` from `requiresExecApproval(...)` and only ORs in strict inline-eval approval, not a generic obfuscation/length detector.
  - file: [src/agents/bash-tools.exec-host-node.ts:194](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/bash-tools.exec-host-node.ts#L194)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)

  - file: [src/agents/bash-tools.exec-host-gateway.ts:173](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/bash-tools.exec-host-gateway.ts#L173)
