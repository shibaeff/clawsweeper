## Summary

Current `main` no longer contains the Chrome extension relay path this PR targets. OpenClaw migrated host-local browser control to Chrome MCP `existing-session`, removes legacy relay config, and rejects `driver: "extension"`, so the iframe-child-target bug described here is obsolete on `main`.
