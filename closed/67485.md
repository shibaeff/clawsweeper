## Summary

Current `main` already routes package updates through the installed package's owning npm binary, runs a real global `npm i -g` update, and has regression coverage for the mismatched-PATH/global-root case that would strand version-manager installs. The exact report environment is also obsolete on current OpenClaw because Node 16 is no longer supported.

## Evidence

- **Package updates now use the package manager install path:** `runPackageInstallUpdate` resolves a global install target and executes `globalInstallArgs(installTarget, installSpec)`, then re-resolves and verifies the installed package root afterward.
  - file: [src/cli/update-cli/update-command.ts:347](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/cli/update-cli/update-command.ts#L347)

- **For npm, the updater runs `npm i -g`:** `globalInstallArgs` maps npm updates to `[resolved.command, "i", "-g", spec, ...]`, so the updater is using npm's normal global install flow rather than only swapping package files in place.
  - file: [src/infra/update-global.ts:561](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/infra/update-global.ts#L561)
