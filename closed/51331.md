## Summary

Current main already implements the practical outcome requested: OpenClaw has a documented slow-model timeout control via `agents.defaults.llm.idleTimeoutSeconds`, falls back to `agents.defaults.timeoutSeconds`, and now surfaces whether the failure was an LLM idle timeout versus a broader request timeout. This behavior was already present in release `v2026.4.22`.

## Risks / Open Questions

- There is still no dedicated provider-scoped knob like `models.providers.ollama.timeout`; the shipped solution is agent/run-scoped rather than provider-scoped.
- The documentation is currently in the agent-loop docs rather than the Ollama provider page, so discoverability could still be improved without changing behavior.
