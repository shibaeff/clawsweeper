## Summary

Current main no longer uses the LINE SDK version/path that triggered `ERR_MODULE_NOT_FOUND` for `axios` in self-built Docker images, and the shipped release state reflects that change.

## Evidence

- **Docker runtime still uses pruned production deps:** The Docker build installs with `pnpm install`, then prunes to production deps in the runtime-assets stage and copies that pruned `node_modules` into the final image, so the issue depends on the current production dependency graph.

  - file: [extensions/line/package.json:7](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/extensions/line/package.json#L7)

## Close Comment

Closing as implemented on main after Codex review. The reported startup path was `@line/bot-sdk@10.6.0` importing `dist/http-axios.js`, but current OpenClaw no longer ships that dependency graph for the bundled LINE plugin: `extensions/line/package.json:7` now pins `@line/bot-sdk` to `^11.0.0`, and `pnpm-lock.yaml:9866` shows the resolved `@line/bot-sdk@11.0.0` package no longer carries an `axios` runtime dependency. The Docker image still builds from the current production dependency set (`Dockerfile:74`, `Dockerfile:119`, `Dockerfile:173`), so the original `ERR_MODULE_NOT_FOUND` path no longer applies on current main. I also verified the latest release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` contains the same LINE package state, so this has already shipped.
