## Summary

Current `main` already implements the reported behavior for PR #40767: workspace overrides of bundled skills are classified as workspace skills in the status report, and the shipped `v2026.4.23` release already contains both the logic and a regression test for that case.

## Evidence

- **Shipped regression test covers the reported override case:** Release `v2026.4.23` includes a test named `does not mark an overridden workspace skill as bundled by bundled name alone`; it creates a bundled `peekaboo` skill, reports a workspace `peekaboo` entry, and asserts `source === "openclaw-workspace"`, `bundled === false`, `blockedByAllowlist === false`, and `eligible === true`. That is the same Control UI grouping bug described in the PR.
