## Summary

Current main already avoids the problematic macOS `launchctl bootstrap`/post-bootstrap `kickstart -k` path for the config-driven restart flow this PR targeted. Config writes now schedule a SIGUSR1 restart, the launchd-supervised restart path exits cleanly and lets `KeepAlive` relaunch, and tests explicitly cover the no-kickstart behavior on macOS.

## Close Comment

Closing from a Codex review because current `main` already fixes the reported macOS restart failure by taking a different path. Config-driven restarts now go through `scheduleGatewaySigusr1Restart()` in `src/gateway/server-methods/config.ts:592` and `:665`, and the launchd path exits cleanly and lets `KeepAlive` relaunch instead of issuing a post-bootstrap `kickstart -k` handoff (`src/infra/process-respawn.ts:30`, `src/cli/gateway-cli/run-loop.ts:82`). The macOS tests also pin that behavior and explicitly verify the launchd path does not call `triggerOpenClawRestart()` (`src/infra/process-respawn.test.ts:88`). This superseding behavior is already present in release `v2026.4.22`, so this PR is no longer needed as a separate change.
