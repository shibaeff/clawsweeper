## Summary

Current main no longer has the reported 2s client / 3s server handshake gap. Gateway handshake timeouts are now unified at 10s with env overrides, and the CLI gateway call path yields an event-loop turn before starting the WebSocket so heavy startup does not starve the incoming `connect.challenge`. The changelog shows this shipped in release 2026.3.22.

## Evidence

- **Shared 10s handshake defaults plus env overrides:** `DEFAULT_PREAUTH_HANDSHAKE_TIMEOUT_MS` is `10_000`, `resolveConnectChallengeTimeoutMs()` falls back to that shared value, and env overrides exist for both client and server timeouts.
  - file: [src/gateway/handshake-timeouts.ts:1](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/handshake-timeouts.ts#L1)
- **Client uses the shared connect-challenge timeout:** `resolveGatewayClientConnectChallengeTimeoutMs()` resolves the client timeout from the shared helper instead of a hardcoded 2s default.
  - file: [src/gateway/client.ts:167](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/client.ts#L167)
- **Server uses the shared pre-auth timeout:** The WebSocket handshake timer reads `getPreauthHandshakeTimeoutMsFromEnv()`, so the server side is no longer fixed at the old 3s default described in the report.
  - file: [src/gateway/server/ws-connection.ts:366](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/server/ws-connection.ts#L366)
- **CLI path yields before opening the gateway request:** The CLI gateway call path explicitly yields one event-loop turn before starting the WebSocket connection to avoid starving `connect.challenge` processing during heavy synchronous startup.
  - file: [src/gateway/call.ts:479](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/call.ts#L479)
- **Regression coverage for the event-loop yield:** There is a test asserting that CLI pairing requests yield one event-loop turn before starting.
  - file: [src/gateway/call.test.ts:489](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/call.test.ts#L489)
- **Release note shows this shipped in 2026.3.22:** The changelog entry under `## 2026.3.22` says Gateway/WS handshake now uses a 10-second pre-auth timeout and adds `OPENCLAW_HANDSHAKE_TIMEOUT_MS` as an override.
  - file: [CHANGELOG.md:2446](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/CHANGELOG.md#L2446)
