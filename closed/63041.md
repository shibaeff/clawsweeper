## Summary

Current `main` already prevents repeated calls to non-existent tools via shipped unknown-tool loop detection plus a stream-level guard that rewrites over-threshold missing-tool calls before dispatch. The PR’s root-cause analysis matches the bug, but the functionality is already implemented and further hardened on `main`, so this PR is superseded.
