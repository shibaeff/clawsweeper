## Summary

Current main and the shipped v2026.4.23 release already handle Telegram turns that end with no visible final reply via the outbound silent-reply policy path, so this PR is superseded. The branch-specific condition discussed in the PR was also called out for regressions in review, while main uses a broader policy-based implementation with coverage for direct-message fallback and allowed-silent group turns.
