## Summary

Current `main` no longer matches the PR’s reported bug: exec approvals now default to 30 minutes end-to-end, so the old hardcoded 2-minute expiry is obsolete. The proposed Discord-specific `channels.discord.execApprovals.timeoutMs` setting is not present on `main`, but the specific behavior this PR was fixing does not reproduce against current code.
