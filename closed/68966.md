## Summary

Current main treats WSL2 Gateway -> Windows Chrome as a remote-CDP setup, not the local managed `openclaw` profile used in the report. The logged SIGKILL matches expected cleanup for a failed local managed-browser launch, and current main also includes WSL2 networking hardening plus dedicated split-host troubleshooting docs.
