## Summary

Current `main` now persists CLI and ACP child-run outputs into session transcripts before announce/delivery. That addresses the reported empty announce payload, empty `sessions_history(...)`, and missing transcript-file symptoms caused by child output not being written into session history.

## Evidence

- **CLI runs now persist transcript output:** `agent-command` calls `persistCliTurnTranscript(...)` after CLI-backed runs complete, so child CLI output is written into the session transcript instead of existing only in streamed events.
