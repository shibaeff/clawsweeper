## Summary

Current main treats `qqbot` and `minimax` as valid canonical bundled plugin ids, documents that `plugins.allow` uses plugin ids rather than package names, and normalizes `minimax-portal` back to the `minimax` plugin. The report's claimed "invalid plugin ids" premise does not match current code/docs/tests.

## Evidence
