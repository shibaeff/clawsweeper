## Summary

Current `main` already does the reporter’s suggested fix for Claude CLI session continuity: it reads the CLI-reported session id from structured output, uses that as the effective session id, persists it into the session store, and now defaults `claude-cli` to warm stdio reuse with transcript-verified resume safeguards.

## Evidence

- **CLI output parser reads actual session id from Claude CLI JSON/JSONL:** `pickCliSessionId` accepts `session_id`/`conversation_id` fields and `parseCliJson` carries the parsed session id forward from CLI output instead of relying only on the pre-sent OpenClaw UUID.
  - file: [src/agents/cli-output.ts:225](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/cli-output.ts#L225)

- **CLI runner prefers returned session id over the originally supplied one:** `effectiveCliSessionId` is computed as `output.sessionId ?? context.reusableCliSession.sessionId`, and that effective id is written into `agentMeta.sessionId` and `cliSessionBinding.sessionId`. This matches the issue’s suggested fix.
  - file: [src/agents/cli-runner.ts:257](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/cli-runner.ts#L257)
