## Summary

Current `main` already fixes the reported Dreaming-ingestion pollution: session export now strips heartbeat/direct-cron/system wrapper noise, isolated cron-run transcripts are classified and skipped, and regression tests cover both the heartbeat example and cron-run exclusion. The exact persisted `SessionEntry.trigger` proposal was not implemented, but the user-facing Dreaming bug described in #67272 is already addressed on `main`.

## Evidence

- **Session sanitizer drops heartbeat and cron wrapper noise:** `sanitizeSessionText` removes generated system wrappers, direct `[cron:...]` prompts, heartbeat prompts, assistant-side `HEARTBEAT_OK`, and exec-completion events before transcript lines are exported for Dreaming ingestion.

  - sha: [e5babbb5e75c](https://github.com/openclaw/openclaw/commit/e5babbb5e75cdc7adaf4bb626db3ce838b2430c5)
- **Cron-run transcripts are classified from the session store:** Session transcript classification marks `agent:*:cron:*:run:*` store entries as cron-run transcripts, which gives Dreaming a concrete way to identify isolated cron sessions without relying on in-band text.

## Risks / Open Questions

- The fix on `main` addresses the Dreaming-ingestion problem via transcript sanitization and cron-session classification, but it does not add a persisted generic `trigger` field to `SessionEntry`; if maintainers still want broader plugin-visible session-origin metadata, that would be separate follow-up work.
- The implementation landed after the latest tagged release `v2026.4.23`, so the fix is on `main` but not yet associated with a shipped release tag.
