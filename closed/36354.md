## Summary

The PR proposes a useful OpenMork integration path, but it adds a vendor-specific adapter and docs directly to OpenClaw core. Current main explicitly routes optional vendor/model integrations through external provider plugins and agent harnesses, with discovery/promotion in ClawHub rather than new core surfaces.
