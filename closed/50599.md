## Summary

Current `main` already contains explicit fixes for the two behaviors in #50599: local authenticated gateway calls/probes now stay device-bound so `operator.read` detail RPCs remain available, and the Windows gateway call path yields before WebSocket startup to avoid `connect.challenge` handshake timeouts. The CLI/docs/changelog also now classify scope-limited probe results as degraded instead of silently healthy, and the changelog shows the status/probe reporting fix shipped in `v2026.4.20`.
