## Summary

Current main and the latest release already contain the Ollama/tool-handoff and silent-reply fixes that match this report: Ollama tool calls are preserved from intermediate stream chunks, `/v1` Ollama URLs are normalized back to native `/api/chat`, empty `stopReason="error"` turns are retried instead of ending with no reply, and leaked tool scaffolding is stripped from visible replies.

## Evidence

- **Ollama stream now preserves intermediate tool calls:** The Ollama stream path accumulates `tool_calls` from non-final NDJSON chunks and reattaches them to the final response before building the assistant message, which directly addresses live-turn tool-handoff loss.

- **Regression coverage for Ollama intermediate tool-call chunks:** Release code includes tests asserting that Ollama `tool_calls` can arrive on `done:false` chunks and must be accumulated across chunks, matching the failure class described in the issue.
