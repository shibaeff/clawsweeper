## Summary

Issue #63032 describes behavior in the external WeChat channel plugin, not an OpenClaw core bug. Current main documents WeChat as an external `@tencent-weixin/openclaw-weixin` plugin, says DM access uses normal OpenClaw pairing approval, and keeps channel/plugin ownership outside core.

## Evidence

- **WeChat is explicitly external-plugin-owned:** Current WeChat docs state that OpenClaw connects through the external `@tencent-weixin/openclaw-weixin` plugin and that WeChat-specific runtime, login, Tencent iLink API calls, media handling, and account monitoring are owned by that plugin rather than core.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/docs/channels/wechat.md#L10)

- **WeChat DM flow is documented as pairing approval, not auto-created allowFrom on login:** The same doc says direct messages use the normal OpenClaw pairing/allowlist model and instructs operators to run `openclaw pairing list openclaw-weixin` and `openclaw pairing approve openclaw-weixin <CODE>` for new senders.
  - file: [docs/channels/wechat.md:84](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/docs/channels/wechat.md#L84)
