## Close Comment

Closing this as implemented after Codex review.

Current main already supersedes this PR. The skills watcher now watches only explicit `SKILL.md` targets instead of recursively watching whole skill trees, which addresses the FD-exhaustion case more directly than adding `depth: 2`. The memory watcher on main also keeps recursive `extraPaths` coverage while ignoring heavy directories like `node_modules` and `.git`, matching the follow-up concern noted on the PR.

What I checked:

- **Skills watcher narrowed to SKILL.md-only globs:** `resolveWatchTargets()` now builds only [`<root>/SKILL.md`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/%3Croot%3E/SKILL.md) and [`<root>/*/SKILL.md`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/%3Croot%3E/*/SKILL.md) targets, with an inline comment stating this avoids traversing unrelated large trees that can exhaust FDs. ([`src/agents/skills/refresh.ts:81`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/src/agents/skills/refresh.ts#L81), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
- **Active watcher uses the narrowed target set:** `ensureSkillsWatcher()` passes `watchTargets` to `chokidar.watch(...)`; there is no recursive directory watch of the whole skills tree anymore. ([`src/agents/skills/refresh.ts:132`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/src/agents/skills/refresh.ts#L132), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
- **Regression test locks the new watcher shape:** The test asserts the watcher receives only `SKILL.md` and [`*/SKILL.md`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/*/SKILL.md) targets and that every watched target includes `SKILL.md`. ([`src/agents/skills/refresh.test.ts:33`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/src/agents/skills/refresh.test.ts#L33), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
- **Memory watcher already ignores runaway directories:** Current main ignores `.git`, `node_modules`, `.pnpm-store`, `.venv`, `venv`, `.tox`, and `__pycache__` in the memory watcher path filter. ([`extensions/memory-core/src/memory/manager-sync-ops.ts:81`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/extensions/memory-core/src/memory/manager-sync-ops.ts#L81), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))
- **Memory watcher keeps recursive extraPaths coverage:** The memory watcher still watches `path.join(entry, "**", "*.md")` for directory `extraPaths` and does not apply a hard depth cap, which matches the PR discussion that memory recursion should remain intact. ([`extensions/memory-core/src/memory/manager-sync-ops.ts:375`](https://github.com/openclaw/openclaw/blob/29d0e930bb3d/extensions/memory-core/src/memory/manager-sync-ops.ts#L375), [29d0e930bb3d](https://github.com/openclaw/openclaw/commit/29d0e930bb3d))

So I’m closing this as already implemented rather than keeping a duplicate issue open.
