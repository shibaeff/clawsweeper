## Summary

Current `main` already supersedes this PR's timeout bump. The gateway client no longer falls back to a hard-coded 2000ms timeout; it resolves its connect-challenge timeout from a shared handshake-timeout helper whose default is 10000ms, and that behavior is covered by tests and present in the `v2026.4.22` release tag.
