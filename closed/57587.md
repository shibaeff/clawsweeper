## Summary

Current `main` does not show a reproducible duplicate-by-path problem in the bootstrap cache layer. The cached list is unique by construction, the cache is process-local, and the only current path that appends bootstrap files runs after the cache, so this PR's dedup site does not match the live behavior.

## Evidence

- **Cache layer is a plain in-memory passthrough:** `src/agents/bootstrap-cache.ts:3-16` stores a module-local `Map<string, WorkspaceBootstrapFile[]>` and returns the exact array from `loadWorkspaceBootstrapFiles(...)` without any persistence or restart carry-over.

  - file: [src/agents/workspace.ts:480](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/workspace.ts#L480)
