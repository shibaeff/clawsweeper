## Summary

Current main already guards shared-session routing from heartbeat/webchat metadata poisoning, and the fix was shipped in v2026.4.14. The reported behavior on 2026.4.5 matches an older session-route corruption class that is now covered by implementation and regression tests.

## Evidence

- **Shipped fix in changelog:** The 2026.4.14 changelog explicitly says Gateway/sessions stopped heartbeat, cron-event, and exec-event turns from overwriting shared-session routing and origin metadata, preventing synthetic `heartbeat` targets from poisoning later user delivery.
