## Summary

Current `main` already covers the PR's reported behavior through landed work from other changes: SecretRef config-save hardening is present, the Control UI no longer stringifies SecretRef objects into `[object Object]`, and the agents overview now uses effective runtime model metadata so defaulted models do not fall back to `Not set`.

## Evidence

- **Discord token hint is now explicitly sensitive:** The Discord channel config UI hint now marks `token` as `sensitive: true`, matching the PR's root-cause analysis for SecretRef restore on config save.
  - file: [extensions/discord/src/config-ui-hints.ts:212](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/discord/src/config-ui-hints.ts#L212)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **SecretRef round-trip restore is implemented in main:** `maybeRestoreSecretRefId(...)` restores redacted SecretRef ids back from the original value, and the lookup restore path calls it before recursing. This covers the broken `channels.discord.token.id` round-trip case described in the PR.
  - file: [src/config/redact-snapshot.ts:570](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/redact-snapshot.ts#L570)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Regression test covers the Discord SecretRef case:** There is now a targeted restore test for `channels.discord.token` with a redacted SecretRef id, asserting the original env ref id is restored instead of failing validation.
  - file: [src/config/redact-snapshot.restore.test.ts:231](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/redact-snapshot.restore.test.ts#L231)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Control UI prevents SecretRef object corruption:** `renderTextInput` detects structured SecretRef values, keeps them read-only, and uses the structured-value placeholder instead of stringifying them. The browser test asserts the UI does not render `[object Object]` and does not write a corrupted patch back.
  - file: [ui/src/ui/views/config-form.node.ts:678](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/ui/src/ui/views/config-form.node.ts#L678)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Agents overview now uses effective runtime model metadata:** `agents.list` computes each agent's effective model with `resolveAgentEffectiveModelPrimary(...)`, and the overview falls back to `agent.model` when defaults are not present in the local config form. This matches the shipped fix for defaulted models showing `Not set`.
  - file: [src/gateway/session-utils.ts:662](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/gateway/session-utils.ts#L662)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
