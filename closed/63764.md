## Summary

The PR’s reported shell-injection vulnerability does not reproduce on current `main`. `model_usage.py` already uses list-form `subprocess.check_output(...)`, and `--provider` is constrained to fixed enum choices, so the proposed `shell=False` change is a no-op rather than a real security fix.

## Evidence
