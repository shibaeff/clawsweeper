## Summary

Current `main` already covers the reported Control UI image path: pasted images are previewed before send, user/history media renders inline instead of as raw `media://...` text, and managed image previews are fetched through a cached blob-URL path. Existing tests also cover the inline rendering and canonical inbound-media preview behavior.
