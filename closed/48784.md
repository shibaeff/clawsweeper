## Summary

Current `main` already supports same-gateway agent-to-agent delivery into another session via `sessions_send`, with explicit cross-agent gating and persisted inter-session provenance. The core problem in #48784 is implemented on `main`; I could not prove it is in a tagged release from this checkout.

## Evidence

- **Direct session injection via sessions_send:** `sessions_send` now calls the gateway `agent` method against the resolved target `sessionKey`, with `deliver: false`, an internal channel, and `inputProvenance.kind = "inter_session"`. That is a real turn in the target session context, not an isolated relay session.
