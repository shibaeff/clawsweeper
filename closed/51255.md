## Summary

Current `main` already has plugin-level session lifecycle hooks. `session_end`/`session_start` are part of the hook contract, `session_end` fires on `/new`/`/reset` rollovers, compaction-driven session rotation, idle/daily rollovers, and deletes, and the lifecycle is documented as a supported plugin seam.

## Risks / Open Questions

- The implementation is a plugin lifecycle hook (`session_end`/`session_start`), not a new internal `hooks.internal` event named `session:end`.
- This landed on `main` after the listed latest release `v2026.4.22`, so there is no released tag to point to yet.
- Like the original discussion noted, lifecycle hooks only cover graceful/session-managed endings; they do not guarantee cleanup after crashes or hard disconnects.
