## Summary

Current main routes the persisted session thinking level directly into status rendering and the status renderer displays that value without clamping. The same implementation is present in release v2026.4.22, so this issue is already fixed on shipped code.

## Evidence

- **Session status uses persisted thinkingLevel:** The session status tool passes `statusSessionEntry.thinkingLevel` into `buildStatusText` as `resolvedThinkLevel`, so the status card now starts from the persisted session value instead of recomputing or normalizing it.
  - file: [src/agents/tools/session-status-tool.ts:561](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/agents/tools/session-status-tool.ts#L561)
