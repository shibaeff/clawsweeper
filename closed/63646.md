## Summary

Close as implemented on current `main`: the embedded OpenAI path now removes unsupported disabled reasoning payloads for GPT-5 models such as `gpt-5-mini`, maps thinking through model-supported efforts, has regression tests for the embedded/off and `gpt-5-mini` cases, and the fix shipped in `v2026.4.20` with related issue/PR discussion pointing to PR #61982.

## Evidence

- **Embedded runner now deletes disabled reasoning for OpenAI sub-agent runs:** `createOpenAIThinkingLevelWrapper()` removes `payload.reasoning` when `thinkingLevel` resolves to `off`, which directly prevents the reported `reasoning.effort: "none"` request from being sent on embedded/sub-agent OpenAI runs.

- **Current OpenAI model map excludes `none` for plain GPT-5 models:** The current reasoning-effort table defines plain GPT-5 models as supporting `minimal|low|medium|high` only, matching the issue's `gpt-5-mini` error and excluding the invalid `none` default.
  - file: [src/agents/openai-reasoning-effort.ts:24](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/openai-reasoning-effort.ts#L24)
