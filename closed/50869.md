## Summary

Issue #50869 is already implemented on current `main`: `openclaw daemon start` on macOS now attempts LaunchAgent recovery when the service is installed but not loaded, matching the requested stop→start bootout recovery behavior.

## Evidence

- **Darwin start path now includes recovery hook:** `runDaemonStart` passes `onNotLoaded: recoverInstalledLaunchAgent({ result: "started" })` on macOS, so the start command no longer immediately falls back to install hints when the LaunchAgent is merely unloaded.
  - file: [src/cli/daemon-cli/lifecycle.ts:152](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cli/daemon-cli/lifecycle.ts#L152)
  - command: `nl -ba src/cli/daemon-cli/lifecycle.ts | sed -n '148,165p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Recovery logic checks for existing plist and re-bootstrap:** `recoverInstalledLaunchAgent` first checks whether the LaunchAgent plist still exists, then calls `repairLaunchAgentBootstrap`; on success it reports the service as re-bootstrapped and loaded.
  - file: [src/cli/daemon-cli/launchd-recovery.ts:14](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cli/daemon-cli/launchd-recovery.ts#L14)
