## Summary

Current `main` already handles the reported isolated-cron + `message` tool + `NO_REPLY` case by recognizing a matching direct message-tool send as the completed announce delivery, skipping the fallback cron delivery path, and keeping the run status `ok` unless there is a real fatal payload error. The shipped changelog entry for `v2026.4.20` matches this behavior.

## Evidence
