## Summary

Current `main` and the released `v2026.4.23` code already bound subagent orphan recovery retries with exponential backoff, sanitize leaked internal subagent completion context from user-facing surfaces, and enforce `sessions.json` maintenance. The issue's destructive data-loss path comes from the external cleanup script in the report, not from the current OpenClaw core code path.

## Evidence

- **Release tag already has bounded orphan-recovery retries:** `v2026.4.23` contains `MAX_RECOVERY_RETRIES = 3` plus exponential retry backoff in `scheduleOrphanRecovery()`, so the infinite 10-second retry loop described in the report is not present in the shipped release I inspected.
  - file: [src/agents/subagent-orphan-recovery.ts:283](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/subagent-orphan-recovery.ts#L283)
