## Summary

Current `main` has a Windows-specific gateway restart path that relaunches via Scheduled Task handoff instead of relying on the old Unix-only `SIGUSR1` behavior, and the new path is covered by targeted tests. I could not tie the fix to a tagged release yet, so this is fixed on `main` but `fixedRelease` remains null.

## Evidence

- **Current win32 restart path uses Scheduled Task relaunch:** `triggerOpenClawRestart()` now branches on `process.platform === "win32"` and returns `relaunchGatewayScheduledTask(process.env)` instead of using the old signal-only path on Windows.
  - file: [src/infra/restart.ts:391](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/infra/restart.ts#L391)

  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Windows helper performs detached schtasks handoff with fallback:** `relaunchGatewayScheduledTask()` writes a temporary `cmd` script that queries/runs the Gateway Scheduled Task, retries, and falls back to the installed `gateway.cmd` launcher before cleaning itself up.
  - file: [src/infra/windows-task-restart.ts:24](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/infra/windows-task-restart.ts#L24)
