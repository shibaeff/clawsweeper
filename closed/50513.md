## Summary

The requested post-response GitHub issue work-loop is a good automation idea, but current main already provides the needed post-turn hook and follow-up run APIs for plugins, and project scope says this kind of optional workflow should live as a plugin/ClawHub skill rather than new core behavior.

## Risks / Open Questions

- There is not a bundled example for this exact GitHub issue-resolution workflow yet; someone still has to package the behavior as a plugin/ClawHub skill.
- A plugin implementation would still need explicit operator setup for GitHub auth/repo targeting and safe close/comment policy.
