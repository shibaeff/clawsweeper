## Summary

Current `main` now turns no-reply timeout runs into explicit error payloads, preserves that timeout path through failover instead of dropping it, and documents the supported OpenClaw timeout settings for slow local models. That addresses the reported Ollama timeout causing the web chat to end badly on `2026.4.2`.

## Evidence
