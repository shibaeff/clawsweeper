## Summary

Current `main` already accepts legacy flattened `browser(action="act", kind=..., ...)` parameters, converts them into the internal act request shape, and has regression coverage for the affected `type`, `fill`, and `evaluate` paths. The changelog records this compatibility fix as shipped in release `v2026.3.2`.
