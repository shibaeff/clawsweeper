## Summary

Current `main` and released `v2026.4.23` already support proactive heartbeat delivery with configurable cadence, active hours, custom heartbeat prompts, and explicit channel/recipient routing, which covers the feature requested in #67338.

## Evidence

- **Heartbeat docs already describe proactive, configurable heartbeat delivery:** The heartbeat guide documents configurable cadence, active-hours windows, a custom `prompt`, and explicit delivery routing via `target`, `to`, and `accountId`. It also describes heartbeat as a periodic agent turn that can surface anything needing attention.

## Risks / Open Questions

- The issue body proposed `targetChannelId`/`targetUserId`; OpenClaw ships the equivalent capability under `heartbeat.target` plus optional `heartbeat.to` instead of those exact field names.
- For exact wall-clock reminders or fully detached scheduled sends, current docs steer users toward cron/automation rather than heartbeat, so some use cases are better served there even though proactive heartbeat delivery already exists.
