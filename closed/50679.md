## Summary

Current main no longer exposes a send composer outside the Chat tab, so the reported flow of sending from Overview/Sessions and remaining on that tab is obsolete. The only `onSend` wiring lives inside the `state.tab === "chat"` render branch, and other session navigation entry points already switch back to Chat.

## Evidence

- **Chat composer only renders on Chat tab:** `renderChat(...)` is gated behind `state.tab === "chat"`, so the bottom chat composer is not shown on Overview/Sessions/other tabs.
  - file: [ui/src/ui/app-render.ts:2209](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/ui/src/ui/app-render.ts#L2209)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '2206,2275p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Send handler is only wired inside that Chat render branch:** The `onSend` callback calling `state.handleSendChat()` is passed only inside the Chat-tab render path.
  - file: [ui/src/ui/app-render.ts:2262](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/ui/src/ui/app-render.ts#L2262)
  - command: `nl -ba ui/src/ui/app-render.ts | sed -n '2206,2275p'`
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Other session navigation surfaces already auto-switch to Chat:** Sessions and Cron navigation callbacks call `switchChatSession(...)` and then `state.setTab("chat")`, so jumping into an active conversation already lands on the Chat tab.
  - file: [ui/src/ui/app-render.ts:1641](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/ui/src/ui/app-render.ts#L1641)
