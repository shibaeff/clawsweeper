## Summary

Current main already implements the reported Telegram group-text path: group sender auth falls back from `groupAllowFrom` to `allowFrom`, the runtime only skips unauthorized/blocked group turns, and current regression tests cover plain-text group messages with `requireMention:false` for both explicit-group and `allowFrom`-fallback setups. This makes the report against OpenClaw 2026.4.8 obsolete relative to current main.
