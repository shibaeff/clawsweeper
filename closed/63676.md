## Summary

Current main already handles metadata-triggered empty model turns by retrying once with a continuation prompt and then surfacing an explicit user-visible error instead of leaving a silent blank Feishu reply. The changelog says this shipped in v2026.4.14.

## Evidence

- **Changelog entry shipped the empty-turn recovery:** The 2026.4.14 changelog explicitly says embedded GPT-style runs now recover from reasoning-only or empty turns with bounded continuation and incomplete-turn fallback.
  - file: [CHANGELOG.md:786](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/CHANGELOG.md#L786)
  - command: `nl -ba CHANGELOG.md | sed -n '700,800p'`
- **Current runtime retries empty responses:** The embedded runner detects an empty response, logs it, injects a visible-answer continuation instruction, and retries once instead of letting the turn end silently.
  - file: [src/agents/pi-embedded-runner/run.ts:1948](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-runner/run.ts#L1948)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '1948,2086p'`
- **Current runtime surfaces an explicit fallback after retry exhaustion:** If empty-response retries are exhausted, the embedded runner marks the turn as incomplete and returns a user-visible error rather than no reply.
  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:200](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L200)
