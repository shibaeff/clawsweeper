## Summary

Current `main` already contains a shipped fix for the exact reported `"Agent listener invoked outside active run"` subagent race. OpenClaw now disables late exec `onUpdate` callbacks both when the exec process settles and when the parent tool run is aborted, and the fix is called out in the changelog as shipped in `v2026.4.23`.

## Evidence

- **Shipped changelog entry matches reported error:** The release-tagged changelog explicitly says OpenClaw fixed the gateway crash `("Agent listener invoked outside active run")` when a subagent exec tool emits stdout/stderr after the agent run has ended or been aborted.
  - file: [CHANGELOG.md:942](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L942)

  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Late exec updates are suppressed after run completion:** `runExecProcess` tracks `updatesDisabled` and makes `emitUpdate()` a no-op once the exec run settles, with an inline note that this prevents calling into a disposed agent run and avoids the `outside active run` crash.
  - file: [src/agents/bash-tools.exec-runtime.ts:561](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/bash-tools.exec-runtime.ts#L561)
