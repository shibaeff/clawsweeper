## Summary

Current `main` already contains the packaging/startup hardening this issue asked for: missing `qa/scenarios/` no longer causes a fatal read during QA discovery setup, completion cache generation tolerates subcommand registration failures, release checks explicitly forbid shipping raw `qa/scenarios/` artifacts, and the fix is documented as shipped in `v2026.4.11`.

## Evidence

- **Missing QA scenario pack now degrades instead of crashing:** `readTextFile()` returns an empty string when `qa/scenarios/index.md` is absent, and `readQaScenarioPack()` converts that into an empty optional pack with a non-crashing kickoff message instead of throwing.
