## Summary

Current `main` does not match the reported failure mode: the dashboard config form is seeded from the full config snapshot, agent edits mutate only targeted paths in that full form, and config-write code/tests now preserve untouched config or reject destructive replacements.

## Evidence

- **UI form starts from full config snapshot:** `applyConfigSnapshot` copies the full `config.get` payload into `configForm`, and `mutateConfigForm` clones the existing full form or full snapshot before applying a path edit, so unrelated keys are retained during agent-page edits.
