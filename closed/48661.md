## Summary

Current OpenClaw already exposes public extension seams that cover the requested dispatch/subagent lifecycle and memory-retrieval integration points, and those seams were already present in release v2026.4.22.

## Risks / Open Questions

- There is not a literal `memory:retrieve` hook name; the equivalent extension point is the exclusive active-memory runtime/capability, which gives the memory plugin control of retrieval/scoring rather than adding a separate post-fetch hook.
