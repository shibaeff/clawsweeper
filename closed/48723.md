## Summary

Current `main` already restores the prior session activity timestamp for heartbeat-only outcomes and documents that heartbeat-only replies do not keep the session alive, which addresses the reported re-trigger loop from tool-using `HEARTBEAT.md` runs. The latest release tag and current `main` differ in the inspected heartbeat files only by an import-path change, so this implementation is already shipped, although the exact first fixing commit/release was not determinable from the shallow checkout.

## Evidence
