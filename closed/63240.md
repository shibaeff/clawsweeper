## Summary

Current `main` still appears to have the reported bug. The Google music and video providers pass `models.providers.google.baseUrl` through `normalizeGoogleApiBaseUrl()`, which preserves or adds `/v1beta`, then hand that value to `GoogleGenAI` as `httpOptions.baseUrl`. The bundled `@google/genai` client for Gemini API separately defaults `apiVersion` to `v1beta` and builds request URLs as `baseUrl + '/' + apiVersion`, so a configured base URL ending in `/v1beta` would become `/v1beta/v1beta/...`. This issue is recent and specific, with a clear code-path diagnosis and no evidence on current `main` that it has been fixed or covered by regression tests.
