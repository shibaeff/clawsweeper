## Summary

Current `main` no longer matches the reported failure mode. Invalid `security: "disabled"` values in `exec-approvals.json` are normalized away, gateway/node exec now defaults back to `security=full` and `ask=off` when those host approval values are unset/invalid, and the `exec denied: allowlist miss` path only triggers when the effective host security is still `allowlist`. Discord native command gating also now honors `guild:<id>` entries.

## Evidence

- **Invalid exec approval security values are stripped:** `normalizeExecSecurity` only accepts `deny|allowlist|full`, and `sanitizeExecApprovalPolicy` drops anything else such as `disabled`.
  - file: [src/infra/exec-approvals.ts:44](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/infra/exec-approvals.ts#L44)
- **Gateway/node exec falls back to full/off when approvals defaults are unset or invalid:** `createExecTool` reads `loadExecApprovals().defaults`; if `security` was normalized away, gateway/node fall back to `full`, and `ask` falls back to `off`.
  - file: [src/agents/bash-tools.exec.ts:1480](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/bash-tools.exec.ts#L1480)
- **Allowlist miss only happens in effective allowlist mode:** The gateway path throws `exec denied: allowlist miss` only when `hostSecurity === "allowlist"` and the allowlist did not satisfy the command.
  - file: [src/agents/bash-tools.exec-host-gateway.ts:82](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/bash-tools.exec-host-gateway.ts#L82)
- **Discord guild allowFrom entries are honored on main:** Native Discord command auth checks `commands.allowFrom.discord` for `guild:<id>` entries before user matching.
  - file: [extensions/discord/src/monitor/native-command.ts:175](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/discord/src/monitor/native-command.ts#L175)

## Risks / Open Questions

- If the reporter still has `tools.exec.security=allowlist` (or a session-level `/exec security=allowlist`) set somewhere outside the snippets in the issue, they could still legitimately hit `allowlist miss`.
- This review was read-only and code-based; I did not run a live Discord reproduction in this checkout.
