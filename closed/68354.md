## Summary

Current main already addresses the reported dreaming-session leak: narrative runs delete their subagent session in `finally`, the gateway `sessions.delete` path removes the session-store entry and archives the transcript, and memory-core now has a regression test that scrubs stale `dreaming-narrative-*` entries. The shipped changelog for `v2026.4.23` explicitly records this leak fix under `#67023`.

## Risks / Open Questions
