## Close Comment

Closing this as implemented after Codex review.

Current `main` already fixes the proxy-dispatcher regression for Codex runs with a stronger implementation than this PR: it bootstraps an `EnvHttpProxyAgent` before timeout tuning, then preserves that proxy-aware dispatcher when applying global Undici stream timeouts.

What I checked:

- **Proxy bootstrap exists on main:** `ensureGlobalUndiciEnvProxyDispatcher()` upgrades the global dispatcher to `EnvHttpProxyAgent` whenever env HTTP proxy settings are present, without touching unsupported custom proxy dispatchers. ([`src/infra/net/undici-global-dispatcher.ts:91`](https://github.com/openclaw/openclaw/blob/a903438707f1/src/infra/net/undici-global-dispatcher.ts#L91), [a903438707f1](https://github.com/openclaw/openclaw/commit/a903438707f1))
- **Timeout tuning preserves env-proxy dispatcher:** `ensureGlobalUndiciStreamTimeouts()` detects `env-proxy` dispatchers and rebuilds them as `EnvHttpProxyAgent` with the requested timeout/connect options instead of downgrading them to a plain `Agent`. ([`src/infra/net/undici-global-dispatcher.ts:118`](https://github.com/openclaw/openclaw/blob/a903438707f1/src/infra/net/undici-global-dispatcher.ts#L118), [a903438707f1](https://github.com/openclaw/openclaw/commit/a903438707f1))
- **Codex embedded runs call proxy bootstrap before timeout tuning:** `configureEmbeddedAttemptHttpRuntime()` explicitly calls `ensureGlobalUndiciEnvProxyDispatcher()` before `ensureGlobalUndiciStreamTimeouts()`, with an inline comment explaining this ordering avoids replacing the active proxy dispatcher with a bare one. ([`src/agents/pi-embedded-runner/run/attempt-http-runtime.ts:7`](https://github.com/openclaw/openclaw/blob/a903438707f1/src/agents/pi-embedded-runner/run/attempt-http-runtime.ts#L7), [a903438707f1](https://github.com/openclaw/openclaw/commit/a903438707f1))
