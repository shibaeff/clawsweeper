## Summary

Current `main` already suppresses the reported retry-noise case by clearing `lastToolError` when the same mutating action later succeeds, and that behavior is covered by a regression test for the exact `edit` mismatch-then-retry flow. The linked issue #39916 was already closed as implemented, and release `v2026.4.22` contains the same shipped behavior, so this PR is redundant.
