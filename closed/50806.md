## Summary

Current `main` already implements the reported Discord behavior: bot-authored messages are accepted under `allowBots: "mentions"` when they carry a real mention, including the previously failing case where the gateway payload text contains `<@...>` but the inbound mention metadata is missing and must be hydrated from Discord REST before mention gating runs.

## Evidence

- **Main now hydrates inbound Discord messages when mention syntax exists but metadata is missing:** `shouldHydrateDiscordMessage()` checks for `<@...>`, role mentions, `@everyone`, or `@here` in the inbound text and `hydrateDiscordMessageIfNeeded()` fetches the full Discord message via REST before preflight continues.
