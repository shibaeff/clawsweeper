## Summary

Current `main` already contains the late-stdio exec crash fix: the changelog records it in `2026.4.10`, the supervisor now waits for child close/drain before settling, and exec runtime code suppresses late updates after exit/abort with regression coverage for stale stderr after run end.

## Evidence

- **Issue targets an older version:** The report says the bug was seen on OpenClaw `2026.4.5`, which predates the changelog entry for the landed fix.
- **Changelog entry for this exact crash:** `CHANGELOG.md` under `## 2026.4.10` says: `Agents/exec: prevent gateway crash ("Agent listener invoked outside active run") when a subagent exec tool produces stdout/stderr after the agent run has ended or been aborted. (#62821)`
  - file: [CHANGELOG.md:1024](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/CHANGELOG.md#L1024)

- **Supervisor now settles on close/drain, not early exit:** The child-process adapter tracks `stdoutDrained`/`stderrDrained`, waits for stream `end`/`close`, records `exit`, and only settles from `close` or the Windows drained-stream fallback. That directly addresses the reported `stderr after run ends` timing window.
