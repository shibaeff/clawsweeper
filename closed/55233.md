## Summary

Current `main` already includes Simplified Chinese (`zh-CN`) localization support for the Control UI, including shipped locale data, browser-locale detection, a user-facing language picker, persisted locale selection, docs, and regression tests. The changelog records this as shipped in `v2026.4.5`.

## Close Comment

Closing as implemented on current `main` after Codex review. OpenClaw already ships Simplified Chinese Control UI localization: `ui/src/i18n/lib/registry.ts:13` registers `zh-CN` and auto-detects browser `zh*` locales, `ui/src/i18n/locales/zh-CN.ts:4` contains the shipped Simplified Chinese strings, `ui/src/ui/views/overview.ts:349` exposes the language picker in Overview -> Gateway Access, and `ui/src/i18n/lib/translate.ts:49` handles saved-locale startup and persistence. The docs also describe this support in `docs/web/control-ui.md:94`, and `CHANGELOG.md:1200` records the feature in release `v2026.4.5`. If there is still a specific untranslated screen or string on a current build, please open a new issue with the exact page/text and version so maintainers can track the remaining gap.
