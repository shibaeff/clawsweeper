## Summary

Current `main` already uses fully qualified provider/model keys in the Web UI model picker and patches that exact value through `sessions.patch`, with regression coverage for stale-provider and cross-provider selection cases. The same code is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Picker sends the selected qualified model value:** `renderChatModelSelect` binds each option's `value` from resolved picker options, and the change handler passes that exact trimmed value into `switchChatModel`, which calls `sessions.patch` with `model: nextModel`.
