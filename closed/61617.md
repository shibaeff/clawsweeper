## Summary

Current main already contains multiple scheduler-side guards for duplicate cron execution, and the same protections are present in the released v2026.4.23 tree. The release changelog explicitly calls out duplicate-fire prevention, the cron scheduler advances past same-second matches, applies a 2s refire floor, serializes store access, persists running markers before releasing the lock, and includes regression tests for shared-store and timer/manual overlap cases.
