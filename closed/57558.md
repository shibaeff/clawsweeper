## Summary

Current `main` already covers the reported failure mode: the Control UI now supports provider `apiKey`/`api` fields in `models.providers.*`, redacted secrets round-trip safely through `config.get`/`config.set`, and runtime auth resolution explicitly uses configured `models.providers.<provider>.apiKey`. The changelog shows these fixes shipped in `v2026.4.5`.
