## Summary

Current `main` already has deterministic Telegram photo handling, retries transient `getFile` failures, and emits a user-facing fallback when media download fails. The shipped `v2026.4.23` release also contains this handling.

## Evidence

- **Photo path is handled deterministically:** Inbound Telegram media detection and file-id resolution explicitly treat `photo[]` as media and consistently use the last `photo` entry, which is Telegram's highest-size variant.
  - file: [extensions/telegram/src/bot-handlers.media.ts:16](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/telegram/src/bot-handlers.media.ts#L16)
  - command: `git show v2026.4.23:extensions/telegram/src/bot-handlers.media.ts | nl -ba | sed -n '1,45p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Photo placeholder and download metadata use the same largest-photo rule:** Both placeholder generation and media metadata resolution pick `msg.photo?.[msg.photo.length - 1]`, so compressed-photo handling is deterministic instead of depending on array ordering elsewhere in the pipeline.
  - file: [extensions/telegram/src/bot/body-helpers.ts:34](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/telegram/src/bot/body-helpers.ts#L34)

- **Transient getFile failures are retried and photo messages are not dropped:** `resolveTelegramFileWithRetry()` retries `ctx.getFile()` three times, and the dedicated test covers `photo` specifically with the expectation `returns null for photo when getFile exhausts retries so message is not dropped`.
  - file: [extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts:218](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts#L218)

- **Download failures produce a deterministic user-facing fallback:** When media resolution throws, the Telegram handler sends `⚠️ Failed to download media. Please try again.` instead of silently swallowing the failure.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:1111](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/telegram/src/bot-handlers.runtime.ts#L1111)
