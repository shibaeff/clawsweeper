## Summary

Current `main` already persists session transcripts and reuses them across gateway restarts. The issue request matches existing session storage behavior rather than a missing feature.

## Evidence

- **Session docs: transcripts are persisted on disk:** The session management docs state that all session state is owned by the gateway and stored under `~/.openclaw/agents/<agentId>/sessions/`, with transcripts in per-session `.jsonl` files.
  - file: [docs/concepts/session.md:74](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/concepts/session.md#L74)
  - command: `nl -ba docs/concepts/session.md | sed -n '50,90p'`
  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Migration docs: sessions survive machine moves:** The migration guide explicitly says copying the OpenClaw state directory preserves `Sessions -- conversation history and agent state`, which implies restart/relaunch continuity is already a supported behavior.
  - file: [docs/install/migrating.md:13](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/install/migrating.md#L13)
  - command: `nl -ba docs/install/migrating.md | sed -n '1,40p'`

  - file: [src/gateway/server-methods/chat.ts:1486](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/gateway/server-methods/chat.ts#L1486)

- **Gateway reads persisted transcript history back:** `chat.history` loads the stored session entry and reads messages from the persisted transcript via `readSessionMessages(...)`, so history is restored from disk rather than kept only in memory.
  - file: [src/gateway/server-methods/chat.ts:1891](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/gateway/server-methods/chat.ts#L1891)
