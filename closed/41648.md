## Close Comment

Closing this as implemented after Codex review.

Current `main` already supersedes this PR. Both affected extensions now require `openclaw >=2026.4.20`, and the lockfile resolves their local `openclaw` dependency as a workspace link instead of a published `2026.3.x` package, so the audit path this PR was addressing no longer exists.

What I checked:

- **Google Chat manifest supersedes PR floor bump:** `@openclaw/googlechat` now has `devDependencies.openclaw = "workspace:*"` and `peerDependencies.openclaw = ">=2026.4.20"`, which is stricter than the PR's proposed `>=2026.3.7`. ([`extensions/googlechat/package.json:12`](https://github.com/openclaw/openclaw/blob/00bd2cf7a376/extensions/googlechat/package.json#L12), [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376))
- **Memory Core manifest supersedes PR floor bump:** `@openclaw/memory-core` also now has `devDependencies.openclaw = "workspace:*"` and `peerDependencies.openclaw = ">=2026.4.20"`. ([`extensions/memory-core/package.json:11`](https://github.com/openclaw/openclaw/blob/00bd2cf7a376/extensions/memory-core/package.json#L11), [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376))
- **Lockfile no longer resolves published 2026.3.x openclaw for these importers:** In `pnpm-lock.yaml`, both `extensions/googlechat` and `extensions/memory-core` resolve `openclaw` as `specifier: workspace:*` with `version: link:../..`, so there is no importer-level resolution to `openclaw@2026.3.2` or `openclaw@2026.3.8`. ([`pnpm-lock.yaml:638`](https://github.com/openclaw/openclaw/blob/00bd2cf7a376/pnpm-lock.yaml#L638), [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376))
- **Old vulnerable version strings are gone:** A repo search for `2026.3.2`, `2026.3.7`, and `2026.3.8` in the two package manifests and `pnpm-lock.yaml` returns no matches. ([bae7b54a85b1](https://github.com/openclaw/openclaw/commit/bae7b54a85b1))
- **Released state contains the superseding implementation:** The latest release tag points at commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, and that tagged tree already contains the stricter peer ranges plus workspace-linked lockfile entries for these extensions. ([00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376))
