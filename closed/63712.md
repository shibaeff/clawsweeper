## Summary

Current `main` already implements the requested behavior: auto-persisted fallback model overrides are cleared before the next turn, so OpenClaw retries the configured primary/channel-resolved model instead of staying permanently pinned to the fallback. Regression tests cover both the self-healing path and preservation of explicit user `/models` picks, and the changelog records this as shipped in `v2026.4.20`.

## Evidence

- **Current model-selection logic clears transient auto failover overrides each turn:** `createModelSelectionState(...)` detects `modelOverrideSource === "auto"`, clears the stored session override, writes the cleared entry back to the session store, and resets in-memory `provider`/`model` to the configured defaults so the next turn retries the primary model.
