## Summary

The request is an optional persona/policy-enforcement feature that fits the existing plugin/hook extension surface better than OpenClaw core. Core already injects the relevant identity/persona workspace files, and the remaining validation/logging/alerting behavior is already expressible through plugin hooks.

## Evidence

- **Core already injects persona/identity files:** OpenClaw documents `SOUL.md`, `IDENTITY.md`, and `USER.md` as workspace bootstrap files that are injected into agent context on new sessions.
  - file: [docs/concepts/agent.md:27](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/concepts/agent.md#L27)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Re-injection timing already has a core config:** `agents.defaults.contextInjection` already controls whether bootstrap files are re-injected every turn or skipped on safe continuations, covering the core part of the request about identity-file persistence across turns.
  - file: [docs/gateway/config-agents.md:70](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/gateway/config-agents.md#L70)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Prompt injection is already a plugin hook:** `before_prompt_build` can inject `systemPrompt`, `prependContext`, `prependSystemContext`, and `appendSystemContext`, which is the right extension point for custom identity/persona reinforcement.
  - file: [src/plugins/hook-before-agent-start.types.ts:21](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-before-agent-start.types.ts#L21)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Outbound enforcement/logging hooks already exist:** OpenClaw exposes plugin hooks for `before_agent_reply`, `llm_output`, `agent_end`, and outbound message hooks; `message_sending` can modify or cancel content before delivery.
  - file: [docs/concepts/agent-loop.md:93](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/docs/concepts/agent-loop.md#L93)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Message hook contract supports reply filtering:** `PluginHookMessageSendingResult` supports rewritten `content` or terminal `cancel`, which fits symbol filtering/blocking; `llm_output` and `agent_end` expose model outputs for logging/alerting workflows.
  - file: [src/plugins/hook-message.types.ts:45](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/plugins/hook-message.types.ts#L45)
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Project vision says optional capability should stay out of core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and new skills should go to ClawHub first.
