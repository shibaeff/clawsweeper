## Summary

Current `main` already fixes isolated heartbeat session rollover: heartbeat runs use forced fresh-session resolution, and fresh isolated sessions clear `sessionFile` so old heartbeat transcripts do not accumulate and compact across runs. The shipped `v2026.4.23` release already contains this behavior and regression coverage.

## Evidence

- **Heartbeat forces a fresh isolated session:** `runHeartbeatOnce` routes `heartbeat.isolatedSession` through `resolveCronSession(... forceNew: true)` before persisting the `:heartbeat` entry, so the logical key stays stable while the underlying session is recreated each run.
