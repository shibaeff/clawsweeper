## Summary

Current `main` still uses raw `workspaceDir` inside `loadSkillEntries()`, but the shipped OpenClaw call paths that reach skill discovery already expand `~` upstream with `resolveUserPath()`. I could not verify a current user-facing reproduction for the linked bug, so this PR looks like a defensive cleanup rather than a fix for present behavior.

## Evidence

- **Current helper still uses raw workspaceDir:** `loadSkillEntries()` still derives `workspace/skills`, plugin skill dirs, and `.agents/skills` from the raw `workspaceDir`, which explains the PR, but this is an internal helper rather than the user-facing entrypoint.
