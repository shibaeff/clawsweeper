## Summary

Issue #41212 describes an optional, personalized automation layer that fits OpenClaw's plugin/skill ecosystem, not OpenClaw core. The current project vision explicitly keeps optional capability out of core and routes new skills/plugins to ClawHub.

## Evidence

- **Issue scope is optional persona/workflow automation:** The issue proposes an always-on 'Shadow Clone' that learns the user's workflow and tone, then operates in dev/content spaces. The issue explicitly says there is no concrete problem to solve ('nothing i just thought it was a cool idea'), which makes it a product-scope suggestion rather than a core bug or contract gap.
- **Core scope excludes this kind of feature:** `VISION.md` says optional capability should usually ship as plugins, new skills should be published to ClawHub first, and 'What We Will Not Merge (For Now)' includes new core skills that can live on ClawHub plus heavy orchestration layers that duplicate existing agent/tool infrastructure.
