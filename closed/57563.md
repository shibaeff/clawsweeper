## Summary

Current `main` already has a dedicated WebUI model-selection path that qualifies model values with their provider, resolves stale provider/model combinations against the catalog, and prevents duplicate bare-vs-qualified picker entries. The issue’s reported WebUI-only provider/model conflict is therefore already implemented on `main`.

## Evidence
