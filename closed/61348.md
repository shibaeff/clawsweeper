## Summary

Current `main` already suppresses the fallback main/webchat `System:` post for isolated cron runs once announce delivery is attempted, and the same behavior is present in the released `v2026.4.23` tag.

## Evidence

- **User-facing cron semantics:** The cron docs describe `announce` as fallback delivery to the configured target, with explicit `--channel` / `--to` targeting for channel-only delivery.
  - file: [docs/automation/cron-jobs.md:131](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/docs/automation/cron-jobs.md#L131)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Redundant main fallback suppressed during isolated delivery:** The isolated cron delivery path marks delivery as attempted specifically to avoid a redundant `enqueueSystemEvent` fallback to the main requester/webchat.
  - file: [src/cron/isolated-agent/delivery-dispatch.ts:740](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/cron/isolated-agent/delivery-dispatch.ts#L740)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Isolated run returns deliveryAttempted to scheduler:** The scheduler path for detached cron jobs forwards `delivered` and `deliveryAttempted` from the isolated run result instead of treating it like a main-session system event.
  - file: [src/cron/service/timer.ts:1313](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/cron/service/timer.ts#L1313)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Regression test matches the reported symptom:** There is explicit regression coverage asserting that an isolated cron job with announce delivery attempted does not post a summary to main.
  - file: [src/cron/service.runs-one-shot-main-job-disables-it.test.ts:589](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/src/cron/service.runs-one-shot-main-job-disables-it.test.ts#L589)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Behavior is present in the latest release tag:** The same regression test and delivery code are already present in release `v2026.4.23`, so this is not just unreleased `main` behavior.
  - file: [src/cron/service.runs-one-shot-main-job-disables-it.test.ts:589](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cron/service.runs-one-shot-main-job-disables-it.test.ts#L589)

## Risks / Open Questions

- This was a read-only source review; I did not run a live cron reproduction against a gateway instance.
- OpenClaw still documents isolated cron chat delivery as 'shared' for agent-initiated `message` tool sends; this close is about the reported fallback `System:` duplication into webchat, which is covered by the current code and regression tests.
