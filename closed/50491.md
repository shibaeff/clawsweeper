## Summary

Current `main` intentionally scopes `sessions_list` to visible sessions while `openclaw status` reads the full session-store summary, and `main` now explicitly documents that distinction in tool copy and docs.

## Evidence

- **Session tools still default to tree visibility:** `resolveSessionToolsVisibility()` returns `"tree"` when unset, matching the issue's single-session result when only the current session tree is visible.
  - file: [src/agents/tools/sessions-access.ts:27](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/tools/sessions-access.ts#L27)

- **sessions_list filters out non-visible sessions:** `sessions_list` runs each gateway session row through `visibilityGuard.check(key)` and skips rows that are not allowed.
  - file: [src/agents/tools/sessions-list-tool.ts:148](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/tools/sessions-list-tool.ts#L148)

  - file: [src/agents/tool-description-presets.ts:13](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/tool-description-presets.ts#L13)
