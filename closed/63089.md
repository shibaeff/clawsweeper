## Summary

Current main does not match the reported crash path: `openrouter/auto` is modeled as `openai-completions`, embedded runs swap supported APIs onto OpenClaw boundary-aware transports, and the in-tree Anthropic transport now guards malformed blocks/text instead of assuming `text` is present.

## Evidence

- **OpenRouter models use OpenAI-compatible API:** The bundled OpenRouter provider registers dynamic and static models with `api: "openai-completions"`; `openrouter/auto` is not an `anthropic-messages` model on current main.
  - file: [extensions/openrouter/index.ts:57](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/openrouter/index.ts#L57)

- **OpenRouter catalog defines auto on OpenAI-compatible base:** The OpenRouter provider catalog sets `baseUrl: "https://openrouter.ai/api/v1"` and `api: "openai-completions"` for the bundled provider, including the `auto` model entry.
  - file: [extensions/openrouter/provider-catalog.ts:38](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/openrouter/provider-catalog.ts#L38)

- **Boundary-aware transport covers OpenAI and Anthropic APIs:** Current main maps `openai-completions` to `createOpenAICompletionsTransportStreamFn()` and `anthropic-messages` to `createAnthropicMessagesTransportStreamFn()`, so supported runs do not need the upstream default provider path.
  - file: [src/agents/provider-transport-stream.ts:75](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/provider-transport-stream.ts#L75)

  - file: [src/agents/anthropic-transport-stream.ts:811](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/anthropic-transport-stream.ts#L811)
