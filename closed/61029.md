## Summary

Current `main` already fixes this by having `image_generate` return the exact saved media paths and structured `details.media.mediaUrls`, and the tool-result delivery path now consumes those structured media URLs. A regression test covers the reported Gemini `.jpg` path under `tool-image-generation/` and asserts the exact `MEDIA:` path is emitted instead of a fabricated `media/output/*.png` path.

## Evidence
