## Summary

Current main already implements the requested behavior: proactive/default auto-compaction before hard context failure, configurable pre-compaction memory flush/checkpoint-style persistence, and transcript/tool-result guards for the orphaned `tool_use_id` failure mode. The same implementation is already present in the shipped `v2026.4.22` release tree.

## Evidence

- **Auto-compaction is already a documented default behavior:** The compaction docs state that auto-compaction is on by default, runs when a session nears the context limit, retries after overflow, and preserves recent messages while summarizing older ones.

  - file: [docs/reference/session-management-compaction.md:230](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/reference/session-management-compaction.md#L230)
