## Summary

Current `main` already routes Feishu `message(..., media=...)` through the plugin media upload path, preserves allowed local roots for local files, and includes regression tests for the reported behavior. The fix appears to be on `main` only and not yet in a tagged release.

## Evidence

- **Feishu send action consumes `media` and forwards local-root context:** The Feishu message-action handler reads `params.media` and, for `send` / `thread-reply`, calls `sendMedia` with `mediaUrl` plus `mediaLocalRoots`. That matches the reported `message(action=send, channel=feishu, media="/path/to/image.png")` path and no longer treats the path as plain text.
  - file: [extensions/feishu/src/channel.ts:697](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/feishu/src/channel.ts#L697)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '60,90p;680,745p'`
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Feishu media sender uploads local paths instead of echoing them:** `sendMediaFeishu` explicitly supports `mediaUrl` as a URL or local path, loads the file via `loadWebMedia(...)` with `localRoots`, then sends images through Feishu `image_key` and other media through `file_key` / `msg_type` routing.
  - file: [extensions/feishu/src/media.ts:576](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/feishu/src/media.ts#L576)

- **Regression test covers the exact `media: "/tmp/image.png"` send path:** The Feishu channel test now exercises `action: "send"` with `media: "/tmp/image.png"` and asserts that the plugin calls outbound media send with `mediaUrl: "/tmp/image.png"` and `mediaLocalRoots: ["/tmp"]`.
  - file: [extensions/feishu/src/channel.test.ts:439](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/feishu/src/channel.test.ts#L439)
