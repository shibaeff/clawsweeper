## Summary

Close as implemented on main: Feishu webhook traffic already uses the shared reply pipeline, `agent_end` is emitted by the core runner rather than channel-specific hook registration, and the released plugin-loader path re-initializes the global hook runner for cached runtime loads.

## Evidence

- **Feishu registers one shared inbound message handler:** `registerEventHandlers(...)` wires `im.message.receive_v1` to `createFeishuMessageReceiveHandler(...)` and `handleFeishuMessage`, so Feishu inbound traffic goes through the normal shared handler path rather than a webhook-only codepath.

  - file: [extensions/feishu/src/monitor.account.ts:447](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/extensions/feishu/src/monitor.account.ts#L447)
