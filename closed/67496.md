## Summary

Current `main` already implements the OpenClaw-side Slack Socket Mode fix: it builds proxy-aware Slack client options from `HTTP(S)_PROXY`/`NO_PROXY`, passes them into Bolt's Socket Mode receiver, and has regression tests for that path. The reported standalone raw `@slack/socket-mode` reproduction bypasses the OpenClaw gateway integration that applies this fix.

## Evidence

- **Shipped release note:** The changelog for `2026.4.8` explicitly says Slack now honors ambient HTTP(S) proxy settings for Socket Mode WebSocket connections, including `NO_PROXY`, so proxy-only deployments can connect without a monkey patch.
  - file: [CHANGELOG.md:1270](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/CHANGELOG.md#L1270)
  - command: `nl -ba CHANGELOG.md | sed -n '1264,1272p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Current proxy-aware Slack client options:** Current main constructs an `HttpsProxyAgent` from proxy env vars and documents that this fixes Socket Mode in environments that must use an HTTP CONNECT proxy.
  - file: [extensions/slack/src/client-options.ts:50](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/slack/src/client-options.ts#L50)
  - command: `nl -ba extensions/slack/src/client-options.ts | sed -n '1,120p'`

  - file: [extensions/slack/src/monitor/provider.ts:186](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/slack/src/monitor/provider.ts#L186)
  - command: `nl -ba extensions/slack/src/monitor/provider.ts | sed -n '175,205p'`

- **Receiver wiring:** `createSlackBoltApp` constructs `SocketModeReceiver` with `installerOptions.clientOptions` and the `App` with the same `clientOptions`, which is the integration layer the standalone raw `SocketModeClient` script bypasses.
  - file: [extensions/slack/src/monitor/provider-support.ts:171](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/slack/src/monitor/provider-support.ts#L171)
