## Summary

Current main already enforces WhatsApp DM allowlists at inbox ingress, so unauthorized numbers are blocked before agent routing/LLM reply generation; the same code is present in release v2026.4.22.

## Evidence

- **Ingress access-control blocks unauthorized allowlist DMs:** `checkInboundAccessControl` resolves the WhatsApp policy, applies `resolveDmGroupAccessWithLists`, and returns `allowed: false` for any DM whose decision is not `allow`, including unauthorized `dmPolicy: "allowlist"` senders.
  - file: [extensions/whatsapp/src/inbound/access-control.ts:76](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/whatsapp/src/inbound/access-control.ts#L76)
