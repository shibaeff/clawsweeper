## Summary

Current `main` already contains a generic streamed-reply dedupe fix that covers the reported Feishu queued-message duplication path: repeated `message_end`/`text_end` assistant output is suppressed both for outbound block replies and for persisted `assistantTexts`, with regression tests covering the duplicate cases.

## Evidence

- **Per-message transcript dedupe:** `shouldSkipAssistantText` compares trimmed and normalized assistant text within the current assistant message, and `pushAssistantText` skips duplicates before appending to `assistantTexts`. That directly prevents the same final assistant text from being persisted twice in one turn.
