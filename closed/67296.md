## Summary

Current main and the shipped `v2026.4.23` release no longer use the reported Dreaming/default write path that collided with memory flushes on `2026.4.14`: Dreaming now defaults to separate storage and the docs/tests reflect that supported behavior.

## Evidence

- **Dreaming default storage is now separate:** `DEFAULT_MEMORY_DREAMING_STORAGE_MODE` is `"separate"`, and `resolveMemoryDreamingConfig()` uses that default unless `dreaming.storage` is explicitly overridden.
  - file: [src/memory-host-sdk/dreaming.ts:15](https://github.com/openclaw/openclaw/blob/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac/src/memory-host-sdk/dreaming.ts#L15)

  - file: [docs/concepts/dreaming.md:18](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/concepts/dreaming.md#L18)
