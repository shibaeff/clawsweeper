## Summary

Keep open. Current `main` and the latest shipped release still omit `tool_choice` when `openai-completions` tools are present unless the caller explicitly sets `options.toolChoice`, so this PR is addressing a real unshipped behavior change rather than a stale or already-fixed item. The existing proxy-detection seam (`usesExplicitProxyLikeEndpoint`) also exists on `main`, which makes the requested behavior plausible, but the PR still needs normal maintainer review because the discussion already identified a redundant second compat lookup in the proposed implementation.
