## Summary

Current main already implements the stale-session `/new` recovery path for the bundled `session-memory` hook, and the same code is present in the released `v2026.4.22` tag. The gateway now passes the pre-reset entry as `previousSessionEntry`, the hook prefers that entry over the new session, and transcript recovery explicitly falls back to rotated `.jsonl.reset.*` files when the active file is empty or missing.

## Evidence
