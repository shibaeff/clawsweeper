## Summary

Current `main` already propagates `audioAsVoice` through the shared outbound pipeline, but the Telegram channel plugin still drops that field before calling `sendMessageTelegram`. The low-level Telegram sender already supports `asVoice`, so this PR's remaining adapter work is still relevant and the item should stay open.
