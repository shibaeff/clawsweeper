## Summary

Current `main` already implements the reported fix: reply runs build a stable `extraSystemPromptStatic` without volatile inbound metadata, forward that field into CLI execution, and hash the static prompt for session reuse. The same implementation is present in release `v2026.4.23`, so the heartbeat-triggered `reason=system-prompt` resets described in #68471 are already addressed in shipped code.

## Evidence
