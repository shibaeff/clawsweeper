## Summary

Current `main` already includes `.jsonl.reset.*` and `.jsonl.deleted.*` session transcripts in the real memory indexing path. The open PR's remaining change is only a `memory status` file-count tweak in `extensions/memory-core/src/cli.runtime.ts`, so the reported indexing bug is already implemented on `main` and shipped in `v2026.4.23`.

## Risks / Open Questions

- `memory status` still appears to undercount reset/deleted transcript artifacts because `scanSessionFiles()` uses `.endsWith(".jsonl")`; that is a reporting mismatch, not evidence that the indexer itself is missing those files.
