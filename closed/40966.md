## Summary

Current `main` already fixes the PR’s stated bug: zero-valued poll duration defaults no longer trigger poll-intent detection for `action=send`. The only remaining delta is this PR’s stricter treatment of negative durations, but `main` now documents and tests the opposite behavior intentionally so invalid non-zero durations still stay on the poll-only validation path.

## Close Comment

Closing this as implemented after Codex review.

Current `main` already fixes the reported zero-duration false positive for `message` send actions, with tests covering zero-valued camelCase and snake_case poll params. The implementation differs slightly from this PR by treating any finite non-zero numeric value as explicit poll intent, but the reported bug is resolved on `main`.

What I checked:

- **Main implementation ignores zero defaults:** `hasPollCreationParams` now treats numeric poll params as intent only when they are finite and non-zero, so `0` and `"0"` no longer trigger poll detection. ([`src/poll-params.ts:67`](https://github.com/openclaw/openclaw/blob/84fc00afdacb/src/poll-params.ts#L67), [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacb))
- **Tests cover zero-valued poll params:** Current tests explicitly assert that `pollDurationHours: 0`, `pollDurationSeconds: 0`, `pollDurationHours: "0"`, `poll_duration_seconds: 0`, and `poll_duration_hours: "0"` all return `false`. ([`src/poll-params.test.ts:37`](https://github.com/openclaw/openclaw/blob/84fc00afdacb/src/poll-params.test.ts#L37), [84fc00afdacb](https://github.com/openclaw/openclaw/commit/84fc00afdacb))
- **Reported failure path depends on poll detection:** `message` send validation still throws only when `action === "send" && hasPollCreationParams(params)`. Because zero-valued defaults now return `false`, the reported send-action rejection no longer reproduces on current `main`. ([`src/infra/outbound/message-action-runner.ts:927`](https://github.com/openclaw/openclaw/blob/b0d2003fe65b/src/infra/outbound/message-action-runner.ts#L927), [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65b))

So I’m closing this as already implemented rather than keeping a duplicate issue open.
