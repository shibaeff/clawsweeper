## Summary

Current main already supports this use case through `/acp steer`: Discord ACP-bound sessions can receive harness-native slash commands like `/help` or `/compact` without adding a quote-stripping layer.

## Evidence

- **Discord slash routing:** Discord native slash commands are intentionally handled in isolated slash-command sessions, which matches the issue's premise that bare `/help` is intercepted before it can be normal chat text.
  - file: [docs/channels/discord.md:269](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/channels/discord.md#L269)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **ACP steer resolves to the bound session:** The ACP docs say `/acp steer` resolves against the current thread/conversation binding when no explicit `--session` is provided, so a thread-bound ACP session can be targeted directly from Discord.
  - file: [docs/tools/acp-agents.md:442](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/tools/acp-agents.md#L442)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Steer preserves arbitrary instruction text:** `parseSteerInput` only peels off `--session`; all remaining tokens are joined into `instruction` unchanged, so inputs like `/help` or `/compact` survive intact.
  - file: [src/auto-reply/reply/commands-acp/shared.ts:327](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/auto-reply/reply/commands-acp/shared.ts#L327)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Steer forwards verbatim to ACP:** `runAcpSteer` sends `params.instruction` directly to `acpManager.runTurn({ text: params.instruction, mode: "steer" })`, which means `/acp steer /help` is forwarded to the harness session as the requested slash command.
  - file: [src/auto-reply/reply/commands-acp/lifecycle.ts:749](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/auto-reply/reply/commands-acp/lifecycle.ts#L749)
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Shipped in release tag:** The `v2026.4.22` release commit contains the same `/acp steer` implementation, so this support is not just on unreleased main.
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:src/auto-reply/reply/commands-acp/lifecycle.ts | sed -n '745,827p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- This closes on the basis of the existing steer-based solution, not the exact quoted-message preprocessing proposed in the issue.
- If maintainers still want bare quoted messages like `'/help'` to auto-unwrap, that would be a UX improvement rather than a missing core capability.
