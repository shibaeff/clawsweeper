## Summary

Current `main` already covers the substantive behavior from PR #45611. `config.openFile` no longer shells a raw path string; it resolves a platform-specific command and calls `execFile(...)`, with a hardened Windows PowerShell `-LiteralPath` path and tests for quoting/non-interpolation. The SSRF part is also already satisfied functionally because `metadata.azure.internal` matches the existing `.internal` hostname block rule, so the PR's added explicit hostname is redundant defense-in-depth rather than missing protection.
