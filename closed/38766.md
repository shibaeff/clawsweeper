## Summary

Keep open. Current `main` still has multiple unguarded cron `job.payload` dereferences in the Control UI and CLI, and the cron normalization path can still leave malformed jobs without a usable payload. The PR’s guard changes are not present on `main`, and the current surface has grown beyond the PR’s original scope.
