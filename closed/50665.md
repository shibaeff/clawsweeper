## Summary

Current main already addresses the reported gateway handshake timeout gap: the server default is 10s, production now supports `OPENCLAW_HANDSHAKE_TIMEOUT_MS`, and the client-side connect-challenge timeout also uses the shared 10s default with its own runtime override path.
