## Summary

Current main no longer matches the reported startup hang: the gateway binds HTTP/WebSocket before awaiting plugin/channel sidecars, exposes pending startup via readiness, and the changelog records this startup-sidecar fix in release 2026.4.10.

## Evidence

- **Gateway binds before post-attach sidecars:** `startListening()` runs before `startGatewayPostAttachRuntime(...)`, so the port bind is no longer blocked on plugin/channel sidecar startup.
  - file: [src/gateway/server.impl.ts:808](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server.impl.ts#L808)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '785,860p'`
- **Pending startup is surfaced explicitly:** Readiness returns `startup-sidecars` while sidecars are still pending instead of leaving startup state implicit.
  - file: [src/gateway/server/readiness.ts:48](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server/readiness.ts#L48)
  - command: `nl -ba src/gateway/server/readiness.ts | sed -n '1,220p'`
- **Scheduled startup work is deferred intentionally:** Heartbeat/cron/delivery recovery are kept inert during initial runtime setup and only activated after sidecars are ready.
  - file: [src/gateway/server-runtime-services.ts:81](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server-runtime-services.ts#L81)
  - command: `nl -ba src/gateway/server-runtime-services.ts | sed -n '1,220p'`
- **Regression coverage for sidecar-gated startup:** Tests cover startup-gated methods staying unavailable until sidecars resume, then re-enabling once startup completes.
  - file: [src/gateway/server-startup-post-attach.test.ts:149](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server-startup-post-attach.test.ts#L149)
  - command: `nl -ba src/gateway/server-startup-post-attach.test.ts | sed -n '145,220p'`
- **Release notes document the fix:** The 2026.4.10 changelog says gateway startup now keeps WebSocket RPC available while channels/plugins start and moves Tailscale/update-check startup ahead of sidecars to avoid operator lockout when sidecars stall.
  - file: [CHANGELOG.md:895](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/CHANGELOG.md#L895)
