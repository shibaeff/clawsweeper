## Summary

Current main already routes Telegram native commands in forum topics to topic-qualified session keys, and the `/stop` abort path explicitly targets `CommandTargetSessionKey`. The same implementation is present in the released `v2026.4.22` tag.

## Evidence

- **Telegram native commands use the routed topic session as the command target:** The native-command handler derives `sessionKey` from the Telegram conversation route and passes it as `targetSessionKey` into `resolveNativeCommandSessionTargets`, then stores the result in `CommandTargetSessionKey`.
  - file: [extensions/telegram/src/bot-native-commands.ts:833](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/telegram/src/bot-native-commands.ts#L833)
  - command: `nl -ba extensions/telegram/src/bot-native-commands.ts | sed -n '833,905p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Forum-topic coverage asserts a topic-qualified command target:** The Telegram native-command session-meta test verifies that a forum-topic command resolves `CommandTargetSessionKey` to `agent:zu:telegram:group:-1001234567890:topic:42`.
  - file: [extensions/telegram/src/bot-native-commands.session-meta.test.ts:598](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/extensions/telegram/src/bot-native-commands.session-meta.test.ts#L598)
