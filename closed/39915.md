## Summary

Current `main` already implements Telegram preview streaming with dedicated Telegram draft/message-edit logic, so this PR's premise is obsolete. The proposed `blockStreaming` flip is not the control that enables Telegram preview streaming on current `main`, and the remaining throttle-floor change would still stop previews on send/edit errors without retry/backoff.
