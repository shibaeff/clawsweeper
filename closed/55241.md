## Summary

PR #55241 is superseded. Current `main` already installs Linux systemd gateway units with `EnvironmentFile=` support and avoids re-embedding dotenv-backed secrets inline, with tests and changelog coverage showing this shipped in `v2026.4.14` via a later fix path (`#66249`).

## Close Comment

Closing as already implemented on `main` after Codex review. OpenClaw now writes dotenv-backed systemd values to a separate `gateway.systemd.env` file and references that with `EnvironmentFile=` instead of re-embedding matching secrets inline (`src/daemon/systemd.ts:457`, `src/daemon/systemd-unit.ts:38`). The current tests cover both the split-file behavior and the inline-override case (`src/daemon/systemd.test.ts:682`), and the shipped changelog entry for `v2026.4.14` records this fix under `#66249` (`CHANGELOG.md:680`). This PR looks superseded by that later landed implementation.
