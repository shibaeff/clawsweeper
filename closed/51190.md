## Summary

Current main already contains the Discord gateway lifecycle work for this failure mode: it marks Discord disconnected on close/reconnect events, force-stops sockets that open without reaching READY, and restarts startup connections that stall before READY. Regression tests cover both the stuck reconnect case and recovery after restart.

## Evidence

- **Main lifecycle now force-stops open-without-READY reconnects:** The Discord gateway status observer starts a READY watch on websocket open, pushes `connected: false` on reconnect/close events, and raises a force-stop error if the socket opens but never reaches READY within 30s.
