## Summary

Keep open. Current `main` still routes Google video generation through the GenAI SDK `generateVideos` path with no `predictLongRunning` fallback, while docs still advertise `google/veo-3.1-fast-generate-preview` as the default supported Google video model. The issue discussion's root-cause analysis still matches the codebase, and the claimed fix from `#62343` is not present in this checkout.
