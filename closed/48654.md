## Summary

Current main already preserves explicit `delivery.channel: "feishu"` and supports prefixed Feishu DM targets like `user:ou_*`/`dm:ou_*`, so the fallback `"Channel is required (no configured channels detected)"` path described in #48654 no longer matches the code on `main`.

## Evidence

- **Explicit channel is preserved before fallback:** `resolveDeliveryTarget` only invokes generic channel auto-selection when `resolveSessionDeliveryTarget` returns no channel. With an explicit `jobPayload.channel`, the fallback error path is skipped unless channel resolution failed first.

  - file: [src/infra/outbound/targets-session.ts:106](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/infra/outbound/targets-session.ts#L106)
