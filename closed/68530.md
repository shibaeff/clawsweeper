## Summary

Current `main` fixes #68530 by moving managed dreaming off the old heartbeat-coupled main-session shape, handling the sweep as an isolated cron agent turn, persisting dream diary output to `DREAMS.md`, and cleaning up `dreaming-narrative-*` sessions. The fix appears on `main` after `v2026.4.23`, so it is implemented but not yet in the latest tagged release.

## Evidence

- **Managed dreaming job is now isolated cron work:** `buildManagedDreamingCronJob` now creates one managed cron job with `sessionTarget: "isolated"`, `payload.kind: "agentTurn"`, and `delivery.mode: "none"`, replacing the old heartbeat-coupled main-session/system-event shape described in the report.
  - file: [extensions/memory-core/src/dreaming.ts:153](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/memory-core/src/dreaming.ts#L153)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Runtime only handles managed dreaming on the managed token/trigger path:** The runtime now accepts both `heartbeat` and `cron` triggers, but only runs dreaming when the managed token is present and the trigger is either the queued managed heartbeat event or the cron trigger itself. This removes the old dependence on every heartbeat poll.
  - file: [extensions/memory-core/src/dreaming.ts:769](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/memory-core/src/dreaming.ts#L769)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Dream diary output is persisted:** Narrative output is appended into `DREAMS.md` via `appendNarrativeEntry`, so the prose is no longer transient UI-only output.
  - file: [extensions/memory-core/src/dreaming-narrative.ts:620](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/memory-core/src/dreaming-narrative.ts#L620)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
- **Narrative sessions are cleaned up after the run:** `generateAndAppendDreamNarrative` deletes the subagent session in `finally` and runs the scrubber for stale/orphaned `dreaming-narrative-*` artifacts, addressing the lingering-session part of the report.
  - file: [extensions/memory-core/src/dreaming-narrative.ts:843](https://github.com/openclaw/openclaw/blob/b84e57fca38deacfa9834a82f7e270b32562ae1b/extensions/memory-core/src/dreaming-narrative.ts#L843)
  - sha: [b84e57fca38d](https://github.com/openclaw/openclaw/commit/b84e57fca38deacfa9834a82f7e270b32562ae1b)
