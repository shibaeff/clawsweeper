## Summary

Current `main` already routes config writes through source-preserving merge logic that avoids deleting authored fields during startup-auth-triggered rewrites, and the same implementation is present in release `v2026.4.22`.

## Close Comment

Closing from a Codex review because this is already implemented on current `main` and in `v2026.4.22`. The config write path now runs through `resolvePersistCandidateForWrite` (`src/config/io.ts:1613`, `src/config/io.write-prepare.ts:143`), which computes the caller’s diff against the runtime snapshot and reapplies it onto the authored source config instead of deleting source fields during write-back. There is also direct regression coverage in `src/config/io.write-prepare.test.ts:13` asserting that a partial gateway-auth write does not leak or clobber unrelated config fields. The startup-auth restart flow still goes through the persisted write path (`src/gateway/server-startup-config.ts:251`), so the reported restart-triggered scenario is covered by the hardened implementation. This implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
