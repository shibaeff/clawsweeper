## Summary

Current `main` already handles the reported path as structured `approval-unavailable` state instead of a raw gateway error, and Telegram group tool-summary suppression prevents that plain-text unavailable message from being injected into group chats. The same behavior is present in the shipped `v2026.4.23` release.

## Evidence

- **No-route approvals resolve as typed unavailable state:** Gateway approval handling expires requests as `no-approval-route` and responds with `decision: null` when there is no approval client, no turn-source route, and no forwarded delivery, rather than surfacing a transport error.
