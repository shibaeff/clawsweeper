## Summary

Current `main` already addresses the reported stale skills cache: skill directories are watched for `SKILL.md` add/change/remove events, restart-to-zero snapshot versions are treated as stale, existing session entries refresh persisted `skillsSnapshot` data, and persistent Claude live sessions restart when the skills snapshot fingerprint changes. The relevant implementation landed in commit `355c92d69b3803644a6a56247e5e9273bc1d17b9` on April 25, 2026; I did not find a released tag containing that commit, so this appears fixed on `main` but not yet in a tagged release.

## Evidence
