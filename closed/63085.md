## Summary

Current OpenClaw already supports remote/self-hosted Ollama hosts on `models.providers.ollama.baseUrl`, normalizes `/v1` to the native API root, supplies synthetic local auth for explicit remote Ollama configs, and has regression coverage for provider-level remote streaming. The report’s remaining failure signal is model/tool capability specific, not missing remote-host support.
