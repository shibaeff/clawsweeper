## Summary

Current main already uses file-backed device identity for `openclaw tui`; the report's `localStorage` premise does not match the shipped TUI gateway client anymore.

## Evidence

- **TUI uses the Node gateway client:** `openclaw tui` constructs `GatewayClient` directly, identifies as `openclaw-tui`, and only disables device identity when the explicit insecure-local UI bypass is enabled.
  - file: [src/tui/gateway-chat.ts:116](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/src/tui/gateway-chat.ts#L116)
  - command: `nl -ba src/tui/gateway-chat.ts | sed -n '109,145p'`
  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)
- **GatewayClient loads device identity from disk by default:** Unless the caller explicitly passes `deviceIdentity: null`, `GatewayClient` resolves `deviceIdentity` via `loadOrCreateDeviceIdentity()` instead of any browser storage API.
  - file: [src/gateway/client.ts:203](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/src/gateway/client.ts#L203)

  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)

  - file: [src/infra/device-identity.ts:20](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/src/infra/device-identity.ts#L20)
