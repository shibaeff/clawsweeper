## Summary

Keep open. Current `main` still hardcodes `http://`/`ws://` for Control UI links, while `gateway status` already switches the probe target to `wss://` when `gateway.tls.enabled=true`. The report matches the code on `main`, and the issue timeline already links an open fix PR (#71499), so this is not conservative-close material.
