## Summary

Current `main` already handles missing QA scenario packs without crashing completion-cache generation, and the npm pack checks now explicitly forbid shipping `qa/` artifacts. That matches the reported packaged-install regression in #63684 closely enough to close as fixed on `main`.

## Evidence

- **Missing QA pack now degrades to an empty pack:** `readQaScenarioPack()` checks whether `qa/scenarios/index.md` exists; when it does not, it returns an empty scenario pack with a fallback kickoff task instead of throwing. This directly addresses packaged installs where QA scenario files are absent.
