## Summary

Current `main` does not match the reported regression path. Feishu still carries pending group history into the next dispatch, but the empty-body guard in `get-reply-run` is reached with `BodyStripped` sourced from Feishu `BodyForAgent`, which is rebuilt with speaker/message metadata and is not empty for a mention-only trigger.

## Evidence

- **Feishu strips only the bot mention from raw content:** `normalizeMentions` replaces the bot's own mention with an empty string, but only at raw-content normalization time.
  - file: [extensions/feishu/src/bot-content.ts:267](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/bot-content.ts#L267)
  - command: `nl -ba extensions/feishu/src/bot-content.ts | sed -n '250,285p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Feishu rebuilds a non-empty agent body even when content is empty:** `buildFeishuAgentBody` starts from `ctx.content`, then prefixes speaker identity and message id. A mention-only Feishu turn therefore still produces non-empty `BodyForAgent` on current `main`.
  - file: [extensions/feishu/src/bot.ts:160](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/bot.ts#L160)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '145,195p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Pending group history is still attached on the next Feishu dispatch:** Feishu wraps the current message with pending history via `buildPendingHistoryContextFromMap` and also passes `InboundHistory` through `finalizeInboundContext`.
  - file: [extensions/feishu/src/bot.ts:795](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/extensions/feishu/src/bot.ts#L795)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '788,823p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **Session state defaults `BodyStripped` from `BodyForAgent`:** Session initialization populates `BodyStripped` from `bodyStripped ?? BodyForAgent ?? Body ...`, so `get-reply-run` sees the Feishu-built prompt body, not just the stripped raw mention text.
  - file: [src/auto-reply/reply/session.ts:837](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/auto-reply/reply/session.ts#L837)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '833,845p'`
  - sha: [ec8dbc459558](https://github.com/openclaw/openclaw/commit/ec8dbc4595583816f71af2568df61dbe7c090e1f)
- **The empty-body guard checks `BodyStripped`/`Body`, not `InboundHistory` directly:** `get-reply-run` computes `baseBody` from `sessionCtx.BodyStripped ?? sessionCtx.Body` and only returns the empty-body reply when that prompt body is empty and there is no media. With Feishu's current `BodyForAgent` path above, the reported failure mode does not line up with current source.
  - file: [src/auto-reply/reply/get-reply-run.ts:373](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/auto-reply/reply/get-reply-run.ts#L373)
