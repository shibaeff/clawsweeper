## Summary

Current main intentionally keeps the managed `openclaw` browser isolated and sync-disabled, but the user-facing need from this issue is already implemented through the built-in `user` / `existing-session` browser profile that attaches to a real signed-in Chrome session. That supported path is in code, documented, and recorded in the 2026.3.13 changelog.

## Evidence

## Risks / Open Questions

- This does not enable Google Sync inside the managed `openclaw` profile itself; it closes on the basis that the requested user outcome is already supported through `profile="user"` / `existing-session`.
- The existing-session path has documented limits versus the managed browser (for example some actions still require a managed/CDP profile).
