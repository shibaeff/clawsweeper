## Summary

Current `main` already fixes this packaged-postinstall failure by skipping `dist/extensions/*/node_modules` during dist traversal, and it includes a stronger symlink-based regression test that covers the original `unsafe dist entry` crash scenario. The PR's goal is therefore already implemented on `main`, via a different approach than the proposed prune helper.

## Evidence

- **Current main skips staged plugin runtime node_modules during dist traversal:** `listInstalledDistFiles` now recognizes `dist/extensions/<plugin>/node_modules` as staged runtime content and skips traversing those directories before the symlink safety check runs. That directly avoids the `unsafe dist entry` failure mode described in the PR body.
  - file: [scripts/postinstall-bundled-plugins.mjs:184](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/scripts/postinstall-bundled-plugins.mjs#L184)
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)
- **Regression test uses an actual symlink under dist plugin node_modules:** Current tests create `dist/extensions/slack/node_modules/.bin/fxparser` as a symlink and assert `pruneInstalledPackageDist(...)` succeeds while pruning only unrelated stale files. This is stronger than the PR's reviewed test, which Greptile noted used a regular file and did not reproduce the symlink-triggered crash.
  - file: [test/scripts/postinstall-bundled-plugins.test.ts:419](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/test/scripts/postinstall-bundled-plugins.test.ts#L419)
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)

That means the behavior is present on current `main` but not in the latest tagged release provided in the task context.
