## Summary

Current main already normalizes agent-emitted local `MEDIA:` paths into managed outbound attachments before Telegram delivery, and Telegram outbound delivery then loads and sends those attachments. Main also replaces the old silent-drop behavior with a visible `⚠️ Media failed.` fallback when attachment staging fails.

## Evidence

- **Reply media normalizer stages local MEDIA paths:** `createReplyMediaPathNormalizer` resolves workspace-relative and sandbox-relative media paths, then persists them through `resolveOutboundAttachmentFromUrl` before outbound channel delivery.
  - file: [src/auto-reply/reply/reply-media-paths.ts:127](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/auto-reply/reply/reply-media-paths.ts#L127)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Regression tests cover the reported path shapes:** Tests assert `./out/photo.png` and absolute workspace image paths are converted into outbound-managed files; current main also asserts a visible `⚠️ Media failed.` fallback when staging fails instead of dropping the attachment silently.
  - file: [src/auto-reply/reply/reply-media-paths.test.ts:38](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/auto-reply/reply/reply-media-paths.test.ts#L38)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **CLI --deliver uses the same normalization:** The command delivery path explicitly normalizes relative `MEDIA:` tokens before outbound loading, with an in-code note that this avoids `Local media path is not under an allowed directory` failures.
  - file: [src/agents/command/delivery.ts:306](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/command/delivery.ts#L306)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
