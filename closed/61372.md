## Summary

Keep open. Current `main` still matches the review-thread bug: Discord DM audio is excluded from preflight, the Discord preflight call does not carry chat-scope metadata, and the final Discord process path has no field/path to carry a preflight transcript or prune the transcribed attachment. The analogous Telegram end-to-end transcript threading already exists on `main`, which reinforces that this Discord PR is addressing a real missing path rather than dead code.

## Evidence

- **Discord preflight still excludes DMs:** `resolveDiscordPreflightAudioMentionContext()` still requires `!params.isDirectMessage` in its preflight gate, so audio-only direct messages do not enter Discord audio preflight on current `main`.
  - file: [extensions/discord/src/monitor/preflight-audio.ts:45](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/discord/src/monitor/preflight-audio.ts#L45)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Discord preflight call still lacks scope metadata:** The preflight transcription call only builds a temporary context with `MediaUrls` and `MediaTypes`; it does not pass Discord chat metadata such as `ChatType`, `SessionKey`, `Surface`, or `Provider`, matching the PR discussion about audio-scope enforcement gaps.
  - file: [extensions/discord/src/monitor/preflight-audio.ts:73](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/discord/src/monitor/preflight-audio.ts#L73)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Preflight context type has no transcript/index fields:** `DiscordMessagePreflightContext` currently carries `baseText`, `messageText`, and `wasMentioned`, but no `preflightTranscript` or `preflightTranscribedAttachmentIndex` field, so the process step has no typed path to consume a DM preflight transcript.
  - file: [extensions/discord/src/monitor/message-handler.preflight.types.ts:56](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/discord/src/monitor/message-handler.preflight.types.ts#L56)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Preflight computes transcript but does not return it:** `preflightDiscordMessage()` reads `preflightTranscript` from the preflight-audio helper earlier in the function, but the returned success object does not include that transcript or any attachment index, so the downstream process stage cannot thread it into the final inbound context.
  - file: [extensions/discord/src/monitor/message-handler.preflight.ts:1085](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/discord/src/monitor/message-handler.preflight.ts#L1085)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Process path still uses only base/message text:** `processDiscordMessage()` finalizes inbound context from `combinedBody`, `baseText`, and `messageText`; there is no pruning step for an already-preflight-transcribed audio attachment and no transcript substitution path in the final payload.
  - file: [extensions/discord/src/monitor/message-handler.process.ts:488](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/discord/src/monitor/message-handler.process.ts#L488)
  - sha: [7b5307acfc34](https://github.com/openclaw/openclaw/commit/7b5307acfc347a4e6d6647156600ee4b8d6eaa37)
- **Telegram already has the end-to-end pattern on main:** The existing Telegram implementation on `main` preflights audio, substitutes `bodyText` with `preflightTranscript` for pure audio DMs, and carries that transcript into later mention/body handling. That matches the reviewer comparison cited in this PR and highlights the missing parity on Discord.
  - file: [extensions/telegram/src/bot-message-context.body.ts:200](https://github.com/openclaw/openclaw/blob/7b5307acfc347a4e6d6647156600ee4b8d6eaa37/extensions/telegram/src/bot-message-context.body.ts#L200)
