## Summary

Current main already provides a supported pre-reset persistence path: the bundled `session-memory` hook runs on `/new` and `/reset`, receives the previous session entry before reset, and writes a memory snapshot into `<workspace>/memory`. `/restart` is a gateway restart command, not the session-reset alias.
