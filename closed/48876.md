## Summary

Current `main` already normalizes same-provider persisted model overrides before embedded runs and session model resolution use them, so a stored selection like `openai-codex/gpt-5.4` under provider `openai-codex` resolves to bare model ID `gpt-5.4` instead of sending the duplicated provider prefix downstream.

## Evidence

- **Normalization strips duplicated provider prefix from stored overrides:** `normalizeStoredOverrideModel` removes a leading `<provider>/` from `modelOverride` when `providerOverride` already stores that provider.
