## Summary

Current main already implements the core request: the agent prompt now mandates scanning available skills before replying, and the runtime has configurable tool-loop detection that warns or blocks repeated no-progress patterns.

## Risks / Open Questions

- Loop detection is configurable and disabled by default for most detectors, so users only get the full guardrail when tools.loopDetection is enabled.
- The implementation does not add a separate keyword-triggered skill preloader; it relies on the mandatory skill-scan prompt plus loop-detection guardrails.
