## Summary

Current `main` contains code and regression tests that address both parts of the report: main-session cron/system-event runs now route through the last active session target and surface final chat output back to Control UI/webchat, while explicit outbound `send`/announce requests to `channel=webchat` are now rejected as unsupported internal-only routing instead of falling through to channel delivery.

## Evidence

- **Main-session agent delivery defaults to best-effort session-only on main/global sessions:** `src/gateway/server-methods/agent.ts:732`-`739` sets `bestEffortDeliver = true` by default for main/global session agent runs, which is the path used for Control UI/webchat main-session automation.
  - file: [src/gateway/server-methods/agent.ts:732](https://github.com/openclaw/openclaw/blob/1e8dc2389e424ef90eab001c17c54f0a86d50d5d/src/gateway/server-methods/agent.ts#L732)
  - command: `git blame -L 737,739 src/gateway/server-methods/agent.ts`
  - sha: [1e8dc2389e42](https://github.com/openclaw/openclaw/commit/1e8dc2389e424ef90eab001c17c54f0a86d50d5d)
- **Internal webchat delivery is downgraded to session-only instead of hard-failing:** `src/gateway/server-methods/agent.ts:828`-`849` downgrades internal `webchat` delivery to session-only when best-effort is enabled, and `src/gateway/server.agent.gateway-server-agent-b.test.ts:376`-`393` asserts this exact behavior for `lastChannel: "webchat"`.
  - file: [src/gateway/server.agent.gateway-server-agent-b.test.ts:376](https://github.com/openclaw/openclaw/blob/1e8dc2389e424ef90eab001c17c54f0a86d50d5d/src/gateway/server.agent.gateway-server-agent-b.test.ts#L376)

- **Main-session cron jobs now wake heartbeat with target=last:** `src/cron/service.main-job-passes-heartbeat-target-last.test.ts:56`-`89` verifies main cron jobs pass `heartbeat.target = "last"`, so reminder/system-event responses follow the last active route instead of trying external announce delivery blindly.

Use \`chat.send\`...`, and `src/gateway/server-methods/send.test.ts:345`-`366` covers that response, replacing the older fallthrough behavior that could surface downstream channel errors like `channel_not_found`.
  - file: [src/gateway/server-methods/send.ts:118](https://github.com/openclaw/openclaw/blob/1e8dc2389e424ef90eab001c17c54f0a86d50d5d/src/gateway/server-methods/send.ts#L118)
