## Summary

Current OpenClaw main already uses provider-qualified model refs in the Control UI chat model picker, preserves already-qualified refs instead of re-prefixing them with the active provider, and has regression tests covering stale-provider cases. The changelog shows this class of bogus-provider-prefix bug was shipped in 2026.4.10.

## Evidence

- **Picker options are built from fully qualified provider/model refs:** `buildQualifiedChatModelValue` prefixes catalog entries with their actual provider, and `resolvePreferredServerChatModelValue` preserves already-qualified refs or corrects stale provider state from the catalog instead of blindly prepending the current provider.
  - file: [ui/src/ui/chat-model-ref.ts:5](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/ui/src/ui/chat-model-ref.ts#L5)
  - command: `nl -ba ui/src/ui/chat-model-ref.ts | sed -n '1,170p'`
- **Control UI select submits the chosen qualified value directly:** The chat model `<select>` uses `entry.value` as the option value and `switchChatModel` sends `nextModel` unchanged in `sessions.patch`, so the frontend is no longer reconstructing refs from the active provider.
  - file: [ui/src/ui/chat/session-controls.ts:95](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/ui/src/ui/chat/session-controls.ts#L95)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '80,120p'; nl -ba ui/src/ui/chat/session-controls.ts | sed -n '234,254p'`
- **Select-state logic rejects bogus re-prefixed refs:** Regression coverage asserts that a stale provider does not produce `zai/openai/gpt-5-mini` and instead preserves `openai/gpt-5-mini`, which matches the bug report’s failure mode.
  - file: [ui/src/ui/chat-model-select-state.test.ts:88](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/ui/src/ui/chat-model-select-state.test.ts#L88)
  - command: `nl -ba ui/src/ui/chat-model-select-state.test.ts | sed -n '88,116p'`
- **UI test verifies the browser sends the full qualified ref:** The session-controls test asserts that changing the picker sends `model: "openai/gpt-5-mini"` via `sessions.patch`, not a bare model id or a ref rebuilt from the current provider.
  - file: [ui/src/ui/chat/session-controls.test.ts:139](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/ui/src/ui/chat/session-controls.test.ts#L139)
  - command: `nl -ba ui/src/ui/chat/session-controls.test.ts | sed -n '138,167p'`
- **Shipped release notes match this bug class:** The 2026.4.10 changelog explicitly says session/model selection now preserves already-qualified session model refs so reloads do not produce bogus provider prefixes.
  - file: [CHANGELOG.md:863](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/CHANGELOG.md#L863)
