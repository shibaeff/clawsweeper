## Summary

Close as implemented on `main`. The reported isolated cron + announce timeout/deadlock path is already addressed by the nested-lane fix shipped in `v2026.3.13`, and current `main` still contains both the code change and regression coverage.

## Evidence

- **Changelog shipped fix:** `CHANGELOG.md` for release `2026.3.13` explicitly says: `Cron/isolated sessions: route nested cron-triggered embedded runner work onto the nested lane so isolated cron jobs no longer deadlock when compaction or other queued inner work runs.` This matches the issue’s reported timeout/no-session-output regression after `2026.3.8`.
  - file: [CHANGELOG.md:2555](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/CHANGELOG.md#L2555)
  - command: `nl -ba CHANGELOG.md | sed -n '2499,2560p'`
- **Current main routes cron embedded runs to nested lane:** The isolated cron executor now calls `runEmbeddedPiAgent` with `lane: resolveNestedAgentLane(params.lane)`, so cron-triggered inner work no longer reuses the outer cron lane.
  - file: [src/cron/isolated-agent/run-executor.ts:179](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/cron/isolated-agent/run-executor.ts#L179)
