## Summary

Current `main` already provides the requested capability through `/reset soft`: it preserves the current session/transcript, clears the cached bootstrap snapshot, reruns reset/startup prompt loading in place, and current skill snapshots already refresh mid-session on later turns when the version changes.

## Evidence

  - file: [src/auto-reply/reply/commands-reset.ts:56](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/src/auto-reply/reply/commands-reset.ts#L56)

## Risks / Open Questions

- There is no dedicated `/refresh` alias; the shipped equivalent is `/reset soft`.
- ACP-bound sessions currently reject `/reset soft`, so this equivalence applies to normal OpenClaw sessions rather than ACP-bound ones.
