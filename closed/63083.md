## Summary

Current `main` already routes transcript writes through `SessionManager.appendMessage(...)`, returns/logs transcript write failures instead of silently swallowing them, and exposes per-message transcript sequence metadata to session-history/control-ui consumers. The behavior is also present in release `v2026.4.23`, so this issue is already implemented on shipped code.

## Evidence
