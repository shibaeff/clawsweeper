## Summary

Current `main` no longer has a supported plugin-runtime hot-reload path, and the plugin loader either reuses a cached compatible registry without re-running registration or creates a fresh registry before calling `registerFull`. That makes the PR's duplicate-registration premise obsolete on `7d1891e6e63dc35dd9dcb688ba308049e68fe695`.

## Evidence

- **Plugin runtime changes restart, not hot-reload:** Docs now state config changes require a gateway restart and there is no supported hot-reload path for plugin runtime code, `register(api)` hooks, or tools.
  - file: [docs/tools/plugin.md:151](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/docs/tools/plugin.md#L151)

  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Feishu full registration is plain registration:** The Feishu bundled entry still calls all tool registrars from `registerFull(api)`, but there is no local dedupe latch in `main`; whether duplicates can happen is therefore controlled by loader behavior, not this file.
  - file: [extensions/feishu/index.ts:73](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/extensions/feishu/index.ts#L73)

- **Compatible reloads reuse cached registry:** `loadOpenClawPlugins` returns the cached registry for the computed plugin-load cache key before any new registration runs.
  - file: [src/plugins/loader.ts:1859](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/plugins/loader.ts#L1859)

  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Cache key excludes most channel config and keys on plugin/activation state:** The plugin-load cache key is built from plugin config, activation metadata, scope, and load mode. It is not keyed on per-channel runtime config values like Feishu account settings, so ordinary channel config edits do not force a same-process re-registration path.
  - file: [src/plugins/loader.ts:772](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/plugins/loader.ts#L772)

- **Cache misses build a fresh registry:** When a reload is not cache-compatible, the loader creates a new plugin registry via `createPluginRegistry(...)` before registration, so registration does not append tools onto an old registry instance.
