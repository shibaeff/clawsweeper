## Summary

Current `main` already implements macOS app activity status from live gateway `agent` job/tool events, with matching tests, and the same implementation is present in the shipped `v2026.4.23` release tag. The report was against `2026.4.2` and cites `memory/task_checkpoint.json`, but current app status is driven by the control-channel event stream instead of workspace task files.

## Evidence

- **Control channel routes live agent activity into UI status:** The macOS app listens for `agent` push events and forwards `job`/`tool` updates into `WorkActivityStore`, which is the status pipeline used for activity state.
