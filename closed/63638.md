## Summary

Current `main` already documents and supports a stable release lane, beta prereleases, dev-on-`main`, and maintainer-cut `release/YYYY.M.D` branches, which satisfies the request for a stable version path well enough.

## Evidence

- **Public release-channel docs define stable/beta/dev and recommend stable for most users:** The user-facing release-channel doc says OpenClaw ships `stable`, `beta`, and `dev`; `stable` is recommended for most users, `dev` is the moving head of `main`, and stable builds are promoted separately from beta.
