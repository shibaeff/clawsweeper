## Summary

Current `main` does not show a reachable path for non-JSON-serializable Swift values to enter `MacNodeBrowserProxy.makeRequest`. The `browser.proxy` request crosses the gateway as a JSON string, is decoded back through `OpenClawProtocol.AnyCodable`'s JSON-only cases, and `foundationValue` rehydrates only Foundation JSON containers. The sanitizer in PR #45895 would guard a value shape that the current `main` pipeline does not produce.

## Close Comment

Closing after Codex review because I could not reproduce the reported `__SwiftValue` crash path on current `main`. On current code, `browser.proxy` params are serialized to `paramsJSON` with `JSON.stringify` in `src/gateway/node-registry.ts:126`, then decoded in `apps/macos/Sources/OpenClaw/NodeMode/MacNodeBrowserProxy.swift:88` via `JSONDecoder`. `OpenClawProtocol.AnyCodable` only decodes JSON primitives / arrays / objects in `apps/shared/OpenClawKit/Sources/OpenClawProtocol/AnyCodable.swift:11`, and `foundationValue` reconstructs those into Foundation JSON containers in `apps/shared/OpenClawKit/Sources/OpenClawKit/AnyCodable+Helpers.swift:74`. That makes the non-serializable Swift values this PR sanitizes unreachable on the current mainline path, so I do not have evidence to keep the PR open. If someone can still reproduce a current-main crash with a concrete `paramsJSON` payload or another caller that bypasses this JSON boundary, please open a new issue or PR with that repro.
