## Summary

Current main already fixes this cron misclassification path. `resolveCronPayloadOutcome` now treats earlier error payloads as non-fatal when later successful text exists, and `finalizeCronRun` only emits `status: "error"` plus an `error` field when that fatal condition remains. New tests on main cover the exact regression shape.

## Evidence

- **Outcome classifier now downgrades earlier error payloads when success follows:** `resolveCronPayloadOutcome` computes `hasSuccessfulPayloadAfterLastError` and only keeps `hasFatalErrorPayload` true when no later non-error text exists; it also omits `embeddedRunError` unless the payload is still fatal.
