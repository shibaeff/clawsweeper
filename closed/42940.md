## Summary

Keep open. Current `main` does not implement the PR’s proposed one-card-per-turn behavior: the Feishu dispatcher still closes the streaming card on every `final`, and the current test suite explicitly expects two distinct finals to create two streaming sessions. There is also a real integration risk in the PR approach because the dispatcher idle path does not await async `onIdle` work, so moving final card closure entirely into `onIdle` can leave close requests outside the awaited reply-delivery chain.

## Evidence
