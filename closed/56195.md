## Summary

Current `main` already preserves operator scopes for the supported direct-local `gateway-client`/`backend` shared-auth path without device pairing, and it intentionally still clears unbound scopes for other device-less shared-auth sessions. The regression described by this PR does not match current `main` behavior.

## Evidence

- **Current tests cover the reported backend path:** `server.auth.compat-baseline` asserts that direct-local backend shared-token and shared-password connects without device identity preserve requested scopes and can execute a scoped admin RPC.
