## Summary

Current `main` already implements this behavior, and the fix is present in the shipped `v2026.4.22` release. The active-registry compatibility path in `src/plugins/loader.ts` now handles `default` requests against an active `gateway-bindable` registry by recomputing the normalized gateway-bindable cache key and only reusing the registry when the full normalized key matches, which covers the `agent_end` case more safely than this PR's broader string-match fallback.
