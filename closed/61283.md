## Summary

Current `main` implements `openclaw plugins doctor` as a local, non-activating diagnostics path. The command no longer depends on gateway startup semantics, and the long-lived plugin service behavior described here (including the linked MemOS Memory Viewer sidecar) is only started from gateway startup code, not from `plugins doctor`.
