## Summary

Current `main` already resolves explicit cron delivery channels without falling back to the ambiguous multi-channel path, and Discord `user:<id>` targets still resolve through the DM-channel send path. The shipped `v2026.4.23` release contains that production code.

## Evidence

- **Explicit cron channels bypass multi-channel fallback:** `resolveDeliveryTarget()` uses the requested `jobPayload.channel` first, and only calls `resolveMessageChannelSelection()` when `preliminary.channel` is missing. That means `delivery.channel: "discord"` does not go through the ambiguous multi-channel selector.
  - file: [src/cron/isolated-agent/delivery-target.ts:68](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cron/isolated-agent/delivery-target.ts#L68)
- **Regression coverage for explicit Discord cron delivery with multiple channels:** The cron direct-delivery regression test registers multiple outbound channel plugins, includes a Discord case, runs an isolated cron job with `delivery.mode: "announce"`, and asserts the run is marked delivered and the Discord send dependency is called.
  - file: [src/cron/isolated-agent.direct-delivery-core-channels.test.ts:225](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cron/isolated-agent.direct-delivery-core-channels.test.ts#L225)
- **Discord DM targets still use the DM-channel path:** Discord outbound resolution still treats `user:<id>` as a DM recipient and creates a DM channel via `Routes.userChannels()`. The paired unit test verifies `resolveDiscordTargetChannelId("user:U1", ...)` resolves to `{ channelId: "dm-1", dm: true }`.
  - file: [extensions/discord/src/send.shared.ts:220](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/extensions/discord/src/send.shared.ts#L220)
- **Shipped release already contains the production fix path:** Comparing `v2026.4.23` to `HEAD` for the inspected files shows no production diff in `src/cron/isolated-agent/delivery-target.ts` or `extensions/discord/src/send.shared.ts`; only additional test coverage changed afterward.
  - command: `git diff --stat v2026.4.23..HEAD -- src/cron/isolated-agent/delivery-target.ts extensions/discord/src/send.shared.ts src/cron/isolated-agent.direct-delivery-core-channels.test.ts`
  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)
