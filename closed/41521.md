## Summary

Current main already fixes the reported manual `cron.run` self-deadlock by rerouting cron-triggered inner embedded work off the cron lane, while keeping the outer manual-run queue on `CommandLane.Cron`. This PR’s exact one-line production change is not on main, but the underlying bug is already addressed by later lane-routing changes plus regression coverage, and the PR’s proposed move to `CommandLane.Nested` conflicts with current nested/session-lane behavior.

## Close Comment

Closing this as implemented after Codex review.

Current `main` already fixes the cron self-deadlock through a different mainline path: cron-triggered isolated runs are still initiated with `lane: "cron"`, but the embedded runner now remaps cron-lane inner work onto `CommandLane.Nested`, and that behavior has dedicated regression coverage. This makes the PR's manual `enqueueRun()` lane swap redundant on current `main`.

What I checked:

- **Gateway cron still marks isolated runs as cron-lane work:** `runIsolatedAgentJob` calls `runCronIsolatedAgentTurn(..., lane: "cron")`, so the production cron path still starts from the cron lane. (`src/gateway/server-cron.ts:340`)
- **Embedded runner moves cron-triggered inner work off the cron lane:** `resolveGlobalLane()` explicitly maps `CommandLane.Cron` to `CommandLane.Nested` with a deadlock comment, so the inner embedded agent run no longer waits behind the cron lane slot held by the outer cron execution. (`src/agents/pi-embedded-runner/lanes.ts:8`)
- **Regression coverage exists on main:** `runCronIsolatedAgentTurn lane selection` asserts that a cron-lane isolated run is executed on `nested`, covering the deadlock-avoidance behavior on current `main`. (`src/cron/isolated-agent.lane.test.ts:85`)
- **Shipped release note:** The changelog entry for `2026.3.13` says cron-triggered embedded runner work was routed onto the nested lane so isolated cron jobs no longer deadlock on queued inner work. (`CHANGELOG.md:2500`)

So I’m closing this as already implemented rather than keeping a duplicate issue open.
