## Summary

Current `main` already surfaces inter-session activity as structured transcript data through `sessions_history`: inter-session messages carry `provenance` with `sourceSessionKey`, `sourceChannel`, and `sourceTool`, and the history tool returns sanitized message objects without stripping that metadata.

## Evidence

- **Structured inter-session provenance exists in the stored message shape:** `InputProvenance` includes `sourceSessionKey`, `sourceChannel`, and `sourceTool`, not just `kind`.
  - file: [src/sessions/input-provenance.ts:12](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/sessions/input-provenance.ts#L12)
  - command: `nl -ba src/sessions/input-provenance.ts | sed -n '1,220p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Inter-session sends populate all requested provenance fields:** `sessions_send` writes `inputProvenance: { kind: "inter_session", sourceSessionKey, sourceChannel, sourceTool: "sessions_send" }` into the routed message.
  - file: [src/agents/tools/sessions-send-tool.ts:283](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/tools/sessions-send-tool.ts#L283)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '260,310p'`

  - file: [docs/reference/transcript-hygiene.md:80](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/reference/transcript-hygiene.md#L80)

## Risks / Open Questions

- There is no dedicated top-level `recentInterSessionActivity` array; consumers still need to filter `sessions_history.messages` by `message.provenance.kind === "inter_session"`. If maintainers want a more ergonomic summary surface later, that would be a follow-up enhancement rather than a blocker for this issue.
