## Summary

Current main supports the requested additive behavior via `agents.list[].tools.alsoAllow`; `tools.allow` is now explicitly documented and validated as the restrictive allowlist layer applied after a profile, so the reported configuration is no longer the intended contract.

## Evidence

- **Additive field exists in config types:** `ToolPolicyConfig` defines `alsoAllow` as additional entries merged into the effective allowlist, specifically for additive configuration on top of an existing allowlist or profile.
  - file: [src/config/types.tools.ts:184](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/types.tools.ts#L184)
  - command: `nl -ba src/config/types.tools.ts | sed -n '184,205p'`
- **Per-agent additive semantics are documented:** Schema help text says `agents.list[].tools.alsoAllow` is a per-agent additive allowlist on top of global and profile policy.
  - file: [src/config/schema.help.ts:600](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/schema.help.ts#L600)
  - command: `nl -ba src/config/schema.help.ts | sed -n '593,605p'`
- **Docs define profile-before-allow behavior:** Gateway/tool docs state `tools.profile` is the base allowlist and `allow`/`deny` are applied afterward, which matches the current restrictive semantics of `allow`.
  - file: [docs/gateway/config-tools.md:18](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/gateway/config-tools.md#L18)
  - command: `nl -ba docs/gateway/config-tools.md | sed -n '16,62p'`
- **Validation enforces the new contract:** Agent runtime schema rejects setting `allow` and `alsoAllow` together and tells users to remove `allow` and use `profile + alsoAllow` for additive behavior.
  - file: [src/config/zod-schema.agent-runtime.ts:615](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/zod-schema.agent-runtime.ts#L615)
  - command: `nl -ba src/config/zod-schema.agent-runtime.ts | sed -n '612,625p'`
- **Tests cover the contract:** Config tests verify `allow + alsoAllow` is rejected, including at `agents.list[].tools`, and that `profile + alsoAllow` is accepted.
  - file: [src/config/config.tools-alsoAllow.test.ts:6](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/config.tools-alsoAllow.test.ts#L6)
