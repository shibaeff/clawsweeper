## Summary

Current main already preserves external delivery routes for inter-session `sessions_send` traffic, resolves ACP-bound announce targets from stored session delivery context, and only skips the A2A announce path for the parent->own one-shot ACP-child case. The same code is present in the released `v2026.4.22` tree.

## Evidence

- **A2A skip is now narrowly scoped to parent->own background ACP child:** `sessions_send` only suppresses the announce flow when the requester is the parent of its own parent-owned one-shot ACP child; otherwise it starts the normal A2A follow-up/announce flow.
  - file: [src/agents/tools/sessions-send-tool.ts:294](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/tools/sessions-send-tool.ts#L294)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **A2A flow explicitly runs announce step and delivers to resolved target:** After waiting for the target reply, the A2A runner builds the announce prompt, runs `Agent-to-agent announce step.`, and calls gateway `send` with the resolved channel/to/account/thread target.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:64](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/tools/sessions-send-tool.a2a.ts#L64)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **ACP-shaped session keys are hydrated from stored delivery context:** When the session key itself does not encode a deliverable route, announce target resolution falls back to `sessions.list` and reads `deliveryContext`, `lastChannel`, `lastTo`, `accountId`, and `threadId`. That matches ACP-bound sessions like `agent:...:acp:binding:...`.
  - file: [src/agents/tools/sessions-announce-target.ts:34](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/tools/sessions-announce-target.ts#L34)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Current tests cover webchat nested runs plus external announce delivery:** The `sessions_send` tests expect nested agent calls to run with `channel: "webchat"` and still verify the final announce is sent to the external target/channel, including delivery-context hydration for non-key-based targets.
  - file: [src/agents/openclaw-tools.sessions.test.ts:1065](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/agents/openclaw-tools.sessions.test.ts#L1065)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Inter-session routing guard preserves external channel instead of overwriting with webchat:** Reply routing now explicitly preserves an established external route for inter-session sends because `sessions_send` turns arrive as `channel=webchat`; the regression test names this fix directly.
  - file: [src/auto-reply/reply/session-delivery.ts:113](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/auto-reply/reply/session-delivery.ts#L113)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Shipped release contains the implementation:** `CHANGELOG.md` documents the relevant shipped fixes: preserving external routes for `sessions_send` announce traffic and keeping normal A2A delivery for non-parent senders to ACP children. A direct tree comparison showed no code diff for the relevant implementation files between release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` (`v2026.4.22`) and current `HEAD`.
