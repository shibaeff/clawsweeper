## Summary

Current `main` already has the Google Chat outbound/threading guards that make this March 2026 report obsolete, and the same runtime code is present in the released `v2026.4.22` tag.

## Evidence

- **Outbound send only threads when a thread id is present:** `sendGoogleChatMessage` builds `body.thread` and `messageReplyOption` only when `thread` is truthy, so a missing Google Chat thread no longer reaches object-conversion paths during typing or reply sends.
  - file: [extensions/googlechat/src/api.ts:141](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/googlechat/src/api.ts#L141)
  - sha: [b0d2003fe65b](https://github.com/openclaw/openclaw/commit/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503)
- **Inbound Google Chat handling uses optional thread access end-to-end:** The monitor stores and reuses `message.thread?.name` with optional chaining for inbound context, typing indicator sends, and later outbound reply delivery.
  - file: [extensions/googlechat/src/monitor.ts:226](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/googlechat/src/monitor.ts#L226)
