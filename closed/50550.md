## Summary

Current `main` already ships the core behavior this issue asks for: config backups, destructive-write rejection, automatic restore from backup/last-known-good on bad config, stale plugin reference tolerance, and user-facing recovery notices.

## Evidence

- **Destructive config writes are blocked and backups are rotated:** Config writes reject clobber patterns such as dropping required gateway shape or large size drops, save the rejected payload as `openclaw.json.rejected.*`, and maintain `openclaw.json.bak` backups before replacing the live config.
