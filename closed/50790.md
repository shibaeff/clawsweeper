## Summary

Current `main` already has first-class session-scoped reset/forget operations for a single session key: chat `/new` and `/reset` are scoped to the active chat/group session, the Gateway protocol exposes targeted `sessions.reset` and `sessions.delete` methods, ACP has `--reset-session`, and Control UI can delete selected sessions while archiving their transcripts.

## Evidence

- **Manual reset is documented as a per-session operation:** `/new` and `/reset` are documented as manual session reset controls, and group docs say those directives apply only to that group session.
  - file: [docs/concepts/session.md:65](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/concepts/session.md#L65)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Group/chat scope is explicit:** The WhatsApp group-session guide states `/new` or `/reset` apply only to that group’s session and do not touch the personal DM session.
  - file: [docs/channels/group-messages.md:70](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/channels/group-messages.md#L70)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Gateway exposes targeted session-maintenance APIs:** The protocol docs list `sessions.reset`, `sessions.delete`, and `sessions.compact`, and the schema defines `SessionsResetParamsSchema` and `SessionsDeleteParamsSchema` keyed by one session key.
  - file: [docs/gateway/protocol.md:349](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/gateway/protocol.md#L349)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Reset implementation rotates only the targeted session key:** `sessions.reset` dispatches to `performGatewaySessionReset()`, which mints a fresh `sessionId` for the same store key, preserves scoped metadata, archives the previous transcript, and emits session lifecycle hooks for that key.
  - file: [src/gateway/session-reset-service.ts:544](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/session-reset-service.ts#L544)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **Forget/delete is also first-class and operator-facing:** `sessions.delete` removes one targeted session entry and archives its transcript, and Control UI wires a bulk Delete action to `sessions.delete` with transcript archival confirmation.
  - file: [src/gateway/server-methods/sessions.ts:1376](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/gateway/server-methods/sessions.ts#L1376)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)
- **ACP also supports same-key fresh-session reset:** ACP documents `--reset-session` as 'mint a fresh session id for that key (same key, new transcript)'.
  - file: [docs/cli/acp.md:268](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/docs/cli/acp.md#L268)
  - sha: [ebd7f19a3b20](https://github.com/openclaw/openclaw/commit/ebd7f19a3b20001960474ec9ba9cc424e45f7b23)

## Risks / Open Questions

- There is still no dedicated documented `openclaw sessions reset` CLI subcommand in `docs/cli/sessions.md`; the current surfaces are slash commands, Gateway API, ACP, and Control UI delete/archive.
- Control UI evidence covers delete/archive and compaction checkpoint restore, but not a distinct one-click "reset session" button.
