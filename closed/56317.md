## Summary

Current main already fixes the reported OpenRouter auto-compaction failure by prechecking prompt size locally and triggering compaction before the provider call, so it no longer depends on OpenRouter returning a context-overflow error or exposing a context-compression toggle.

## Evidence

- **Precheck compaction is implemented in runtime:** The embedded runner estimates prompt size before submission and converts over-budget prompts into a local precheck overflow path, which triggers compaction/truncation without waiting for a provider error.
