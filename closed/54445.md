## Summary

Current `main` does not fully fix this for sandboxed sessions. `image_generate` still returns absolute files under the shared OpenClaw media store, and the reply-media normalizer still sends sandboxed local media through `resolveSandboxedMediaSource()`, which rejects host paths outside the sandbox root before the managed-media bypass can apply. The new managed-media test only covers the non-sandbox path.

## Evidence

- **image_generate still emits shared-media absolute paths:** `image_generate` saves outputs with `saveMediaBuffer(..., "tool-image-generation", ...)` and returns those absolute paths in `details.media.mediaUrls` and `details.paths`.
  - file: [src/agents/tools/image-generate-tool.ts:706](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/tools/image-generate-tool.ts#L706)
