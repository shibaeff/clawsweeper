## Summary

Issue #50789 reports `openclaw browser extension install` failing on OpenClaw 2026.3.13, but current `main` no longer supports the legacy Chrome extension relay/install flow. The browser plugin now exposes only the modern `openclaw browser` command set, the old extension relay helper is an intentional no-op, and the changelog/docs record the relay-path removal and migration to `existing-session`/`user` profiles. This is an obsolete report against a removed surface, not a current-main bug.

## Close Comment

Closing this as a Codex review because the report no longer matches current OpenClaw behavior. On current `main`, the browser plugin only exposes the modern `openclaw browser ...` command set (`extensions/browser/src/cli/browser-cli.ts:20`), and the old Chrome extension relay helper is now an intentional no-op because that path was removed (`extensions/browser/src/browser/server-lifecycle.ts:9`). Since this issue is about `openclaw browser extension install` on 2026.3.13, it is obsolete against current `main` rather than a live bug in the supported browser path. If you are still seeing a current Docker/browser problem with the managed `openclaw` profile or `existing-session` setup on a recent release, please open a new issue with the exact current version, command, expected vs actual behavior, and logs.
