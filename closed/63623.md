## Summary

Current `main` already contains the iMessage replay-loop hardening that matches this report: outbound replies are sanitized and cached with message ids/text, inbound handling distinguishes real self-chat from normal DM rows using `destination_caller_id`, reflected assistant content is dropped before dispatch, and regression tests cover the self-message replay cases. The changelog says this shipped in `v2026.4.10`.

## Evidence

- **Inbound path now blocks reflected/self-chat replays:** `resolveIMessageInboundDecision` uses `destination_caller_id` to distinguish true self-chat from normal DM rows, drops `is_from_me` self-chat echoes via outbound echo-cache lookup, and drops reflected assistant-content markers before dispatch.
