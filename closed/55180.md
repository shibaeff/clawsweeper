## Summary

The reported `Unable to resolve plugin runtime module` failure matches the bundled runtime-dependency staging/packaged-install regression that has since been fixed on `main` and shipped in `v2026.4.22`. Current code now probes writability with a real write, falls back to external staging roots (`OPENCLAW_PLUGIN_STAGE_DIR` / `STATE_DIRECTORY`), repairs bundled runtime deps before loading, and remaps bundled plugin runtime roots so packaged installs can resolve `openclaw/plugin-sdk/*` correctly.
